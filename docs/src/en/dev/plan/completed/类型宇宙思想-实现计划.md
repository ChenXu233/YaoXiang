# Type Universe Implementation Plan: Removing Fn AST Node

> Document version: 1.3
> Created: 2026-03-18
> Updated: 2026-03-21
> Task: Unify function/type constructor/interface concepts as Binding nodes, defer function determination to type checking phase

---

## Background and Goals

According to the Type Universe design document, the following core changes need to be implemented:

1. **Remove Fn AST node**: All declarations unified as `Binding` nodes
2. **Function determination deferred to type checking phase**: Parser only handles syntax, semantics determined by return value type
3. **Unified syntax model**: Functions, type constructors, and interfaces all use `identifier: type = expression` form

## Stage 1: Frontend - AST Unified as Binding Node

### Step 1.1: Modify AST Node Definition

**Modification target**: Replace `StmtKind::Fn` and `StmtKind::MethodBind` with unified `StmtKind::Binding`

**Design decisions (user confirmed)**:
- **MethodBind is also Binding**: Method bindings use unified syntax model with regular functions
- **Expr::Lambda separately retained**: Lambda expressions are independent syntax, but function Binding depends on Lambda
- **Lambda simplification**: Since type header already contains parameter information, Lambda can be simplified to only keep body
- **Complete replacement**: No need for backward compatibility with old syntax

**Key files** (original plan: 1 → actual: 2):
- `src/frontend/core/parser/ast.rs`
- `src/frontend/core/parser/tests/concurrency.rs` ⚠️

**Acceptance criteria**:
- [x] `StmtKind::Fn` variant removed
- [x] `StmtKind::MethodBind` variant removed, unified as `StmtKind::Binding`
- [x] New `StmtKind::Binding` variant added, containing: name, type_name, method_type, generic_params, type_annotation, eval, params, body, is_pub
- [ ] `Expr::Lambda` variant retained, but simplified structure (remove Lambda header, only keep body)
- [ ] `Type::Fn` variant retained (for function types in type expressions)

**Status**: ✅ Completed (2026-03-20)

**New syntax form summary**:

Generic function unified syntax: `fn name<T: Type>(args) -> ret` is equivalent to `name: (T: Type) -> fn(args) -> ret`

| Old Syntax | New Syntax |
|-------|-------|
| `Type[T, N]` | `(T: Type, N: Type) -> Type` |
| `fn identity<T: Type>(x: T) -> T` | `identity: (T: Type) -> fn(x: T) -> T` |
| `Type -> Type` | `() -> Type` (type constructor empty parameters) |
| `Type[T] -> Type[N]` | `(T: Type) -> (N: Type) -> Type` |
