# Type Constructor Rule Implementation Check

> Check Target: Verify whether the compiler correctly implements the rule "Has `: Type` → Type Constructor, Otherwise → Function/Variable"
>
> Created: 2026-03-21

---

## Functionality Description

### Core Rule

| Syntax | Compiler Behavior |
|--------|-------------------|
| `x: Type = {...}` | **Type Constructor** (`: Type` annotation returns Type) |
| `f = {...}` | **Function/Variable** (`{...}` is parsed as code block) |

**Key Principle**: Omitting `: Type` is never parsed as a type, and `{...}` is always parsed as a code block.

### Ambiguous Scenarios

| Form | With `: Type` | Without `: Type` |
|------|---------------|-----------------|
| `{ x: Float }` | ✅ Type literal | ✅ Code block (expression, meaningless) |
| `{ a = 1 + 1 }` | ✅ Type, field default value is 2 | ✅ Code block (assignment expression, meaningless) |
| `{}` | ✅ Empty record type | ✅ Empty code block `() -> Void` |

**Key Point**:
- With `: Type` → `{ ... }` is parsed as type literal
- Without `: Type` → `{ ... }` is parsed as code block (even in type position!)

## Check Points

### Check 1: Type Constructor Must Eventually Return `Type`

**Key Rule**: It's not "has `Type` equals type", but rather **the final return type must be `Type`** to be a type constructor.

> **Compile-time Certainty**: Everything related to Type is **compile-time determined**. Even for generic functions, T is not passed at runtime, but **compile-time monomorphized**.

| Signature | Return Type | Meaning |
|-----------|-------------|---------|
| `(T: Type) -> Type` | `Type` | ✅ **Type Constructor** (compile-time) |
| `(T: Type) -> (() -> Void)` | `() -> Void` | ❌ **Generic Function** (compile-time monomorphized) |
| `(T: Type) -> (T -> Void)` | `T -> Void` | ❌ **Generic Function** (compile-time monomorphized) |

**Test Code**:
```yaoxiang
// ✅ Type constructor: ultimately returns Type
Point: Type = { x: Float, y: Float }
List: (T: Type) -> Type = { data: Array[T] }
Option: (T: Type) -> Type = { some: (T) -> Self, none: () -> Self }

// ❌ Generic function: ultimately returns a function type, not Type
maybe: (T: Type) -> (() -> Void) = (T) => { println("test") }
factory: (T: Type) -> (T -> Void) = (T) => { (x) => println(x) }
```

**Check Items**:
- [ ] Can `Point: Type = { ... }` compile?
- [ ] Can `List: (T: Type) -> Type = { ... }` compile?
- [ ] Can `Option: (T: Type) -> Type = { ... }` compile?

**Expected Result**: All compile successfully

### Check 2: Omitting `: Type` Is Inferred as Function

**Key Rule**: When there are input parameters, type must be annotated in **either the type annotation** or **the lambda head**, not omitted from both.

```yaoxiang
// ✅ Correct: type annotation position annotated
add: (a: Int, b: Int) -> Int = (a, b) => a + b

// ✅ Correct: lambda head position annotated
add = (a: Int, b: Int) => a + b

// ✅ Correct: both sides annotated
add: (a: Int, b: Int) -> Int = (a: Int, b: Int) => a + b

// ✅ Correct: omit lambda head (when signature is complete)
add: (a: Int, b: Int) -> Int = a + b

// ❌ Wrong: no type annotation on either side
add = (a, b) => a + b

// ✅ Correct: empty block doesn't need parameter types
main = { println("Hello") }
```

**Check Items**:
- [ ] `main = { println("Hello") }` → type `() -> Void`?
- [ ] `add = (a: Int, b: Int) => a + b` → type `(a: Int, b: Int) -> Int`?
- [ ] `add: (a: Int, b: Int) -> Int = a + b` → omit lambda head, type `(a: Int, b: Int) -> Int`?
- [ ] `add = (a, b) => a + b` → ❌ Compilation fails? No type annotation on either side

## Summary

**Core Rules**:

| Syntax | Return Type | Meaning |
|--------|-------------|---------|
| `x: Type = {...}` | `Type` | ✅ Type constructor |
| `f = {...}` | `Void` | ✅ Function/variable (`{...}` is code block) |
| `f: (T: Type) -> Type = ...` | `Type` | ✅ Type constructor |
| `f: (T: Type) -> (() -> Void) = ...` | `() -> Void` | ❌ Generic function (not type) |

**Key Distinction**:
- It's not "has `Type` equals type constructor"
- Rather, "**ultimately returns `Type`** equals type constructor"
- `(T: Type) -> (() -> Void)` returns a function type, which is a generic function

**Compile-time Certainty**:
> Everything related to Type is **compile-time determined**. Even for generic functions, T is **not passed at runtime**, but **compile-time monomorphized**.

**Implementation Points**:

1. **Parser**: Has `: Type` → mark as type constructor; No `: Type` → mark as function/variable
2. **Type Checker**:
   - Has `: Type` → `{ ... }` is parsed as type literal
   - No `: Type` → `{ ... }` is parsed as code block
3. **HM Inference**: Correctly infer function types for shorthand forms
4. **Error Handling**: Clear error messages when expecting type but getting function
