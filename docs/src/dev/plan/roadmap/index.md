## 项目概述

YaoXiang 是一门面向未来的编程语言，采用字节码编译 + VM 解释执行架构。当前处于 v0.7.x 阶段，核心编译器和运行时已基本完成。

## 核心发现

### 已完成的领域

1. **编译器前端**：词法分析、解析、类型检查全部完成，测试覆盖优秀
2. **所有权系统**：完整的借用检查器，14/15 子模块功能完整
3. **工具链**：LSP、包管理、格式化器基本完成
4. **文档站点**：多语言支持，CI/CD 自动部署

### 主要缺口

1. **并作模型**（RFC-001）：DAG 依赖分析器未实现，并发控制注解不完整
2. **Runtime 三层架构**（RFC-008）：仅阶段 A 完成，LLVM AOT 后端未实现
3. **值依赖类型**（RFC-011）：编译期求值引擎未实现
4. **配置系统**（RFC-015）：用户级配置未实现
5. **标准库测试**：仅 8 个单元测试，严重不足

#### 已知问题修复

| 问题 | 模块 | 说明 |
|------|------|------|
| `os.chdir` 未实际切换目录 | std | 只检查目录是否存在，未调用 `std::env::set_current_dir()` |
| `string.len` 返回字节数 | std | `native_len` 使用 `s.len()` 返回字节数而非字符数 |
| `weak` 模块无法导入 | std | 缺少 `StdModule` trait 实现 |
| 空代码块输出 `{\n}` | formatter | 应输出 `{}`（§6.3） |
| `single_quote` 配置未生效 | formatter | 字段存在但未读取（§8.3） |
| 参数列表不支持自动换行 | formatter | 始终输出单行（§4.2） |
