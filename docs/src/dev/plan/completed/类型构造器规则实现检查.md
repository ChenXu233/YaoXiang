# 类型构造器规则实现检查

> 检查目标：验证编译器是否正确实现"有 `: Type` 是类型构造器，否则是函数/变量"规则
>
> 创建日期：2026-03-21

---

## 功能描述

### 核心规则

| 写法 | 编译器行为 |
|------|------------|
| `x: Type = {...}` | **类型构造器**（`: Type` 标注返回 Type） |
| `f = {...}` | **函数/变量**（`{...}` 解析为代码块） |

**关键原则**：省略 `: Type` 永远不会被解析为类型，`{...}` 永远解析为代码块。

### 歧义场景

| 形式 | 有 `: Type` 时 | 无 `: Type` 时 |
|------|----------------|----------------|
| `{ x: Float }` | ✅ 类型字面量 | ✅ 代码块（表达式，无意义） |
| `{ a = 1 + 1 }` | ✅ 类型，字段默认值为 2 | ✅ 代码块（赋值表达式，无意义） |
| `{}` | ✅ 空记录类型 | ✅ 空代码块 `() -> Void` |

**关键点**：
- 有 `: Type` → `{ ... }` 解析为类型字面量
- 无 `: Type` → `{ ... }` 解析为代码块（即使在类型位置也是代码块！）

---

## 检查要点

### 检查 1：类型构造器必须最终返回 `Type`

**关键规则**：不是"有 `Type` 就是类型"，而是**最终返回类型必须是 `Type`** 才算类型构造器。

> **编译期确定性**：Type 相关的内容都是**编译期确定**的。即使是泛型函数，T 也不是运行时传入的，而是**编译期单态化**。

| 签名 | 返回类型 | 含义 |
|------|----------|------|
| `(T: Type) -> Type` | `Type` | ✅ **类型构造器**（编译期） |
| `(T: Type) -> (() -> Void)` | `() -> Void` | ❌ **泛型函数**（编译期单态化） |
| `(T: Type) -> (T -> Void)` | `T -> Void` | ❌ **泛型函数**（编译期单态化） |

**测试代码**：
```yaoxiang
// ✅ 类型构造器：最终返回 Type
Point: Type = { x: Float, y: Float }
List: (T: Type) -> Type = { data: Array[T] }
Option: (T: Type) -> Type = { some: (T) -> Self, none: () -> Self }

// ❌ 泛型函数：最终返回的是函数类型，不是 Type
maybe: (T: Type) -> (() -> Void) = (T) => { println("test") }
factory: (T: Type) -> (T -> Void) = (T) => { (x) => println(x) }
```

**检查项**：
- [ ] `Point: Type = { ... }` 能否编译？
- [ ] `List: (T: Type) -> Type = { ... }` 能否编译？
- [ ] `Option: (T: Type) -> Type = { ... }` 能否编译？

**预期结果**：全部编译通过

---

### 检查 2：省略 `: Type` 会被推断为函数

**关键规则**：有输入参数时，必须在**类型注释**或 **lambda 头** **之一**标注类型，不能两边都省略。

```yaoxiang
// ✅ 正确：类型注释位置标注
add: (a: Int, b: Int) -> Int = (a, b) => a + b

// ✅ 正确：lambda 头位置标注
add = (a: Int, b: Int) => a + b

// ✅ 正确：两边都标注
add: (a: Int, b: Int) -> Int = (a: Int, b: Int) => a + b

// ✅ 正确：省略 lambda 头（签名完整时）
add: (a: Int, b: Int) -> Int = a + b

// ❌ 错误：两边都没标注类型
add = (a, b) => a + b

// ✅ 正确：空块无需参数类型
main = { println("Hello") }
```

**检查项**：
- [ ] `main = { println("Hello") }` → 类型 `() -> Void`？
- [ ] `add = (a: Int, b: Int) => a + b` → 类型 `(a: Int, b: Int) -> Int`？
- [ ] `add: (a: Int, b: Int) -> Int = a + b` → 省略 lambda 头，类型 `(a: Int, b: Int) -> Int`？
- [ ] `add = (a, b) => a + b` → ❌ 编译失败？两边都没标注类型

---

### 检查 3：空块 `{}` 的类型

**测试代码**：
```yaoxiang
// ✅ 正确：空块 {} 推导为 () -> Void

void_func = { println("test") }

// ❌ 错误：期望 Int 但得到 Void

```

**检查项**：
- [ ] `empty_func = {}` 能否编译？类型是否为 `() -> Void`？
- [ ] `void_func = { println("test") }` 能否编译？类型是否为 `() -> Void`？
- [ ] `bad: Int = {}` 是否报错？错误信息是否清晰？

**预期结果**：
- 前两个编译通过
- 第三个报错：类型不匹配（Void vs Int）

- **核心**：省略 `: Type` 永远不会是类型定义

## 边界条件测试

### 边界 1：类型构造器 vs 函数都可以用 `-> Type`

```yaoxiang
// 这是类型构造器
Id: (T: Type) -> Type = { x: T }

// 这是函数，返回 Type 类型的值
id_func: (T: Type) -> Type = (T) => { x: T }
```

**检查项**：
- [ ] `Id: (T: Type) -> Type = { x: T }` 是类型构造器
- [ ] `id_func: (T: Type) -> Type = (T) => { x: T }` 是函数

---

### 边界 2：泛型省略 `: Type` 是函数，不是类型

```yaoxiang
// ✅ 类型构造器：最终返回 Type
List: (T: Type) -> Type = { data: Array[T] }

// ✅ 泛型函数：最终返回函数类型，不是 Type
List = (T: Type) => { data: Array[T] }  // 这是函数，不是类型！
```

**检查项**：
- [ ] `List = (T: Type) => { data: Array[T] }` 是否编译为函数？
- [ ] 这个函数的类型是什么？

---

### 边界 3：接口定义

```yaoxiang
// ✅ 正确
Clone: Type = { clone: (Self) -> Self }
Drawable: Type = { draw: (Surface) -> Void }
```

**检查项**：
- [ ] 接口定义是否必须使用 `: Type`？
- [ ] 不写 `: Type` 是否报错？

## 总结

**核心规则**：

| 写法 | 返回类型 | 含义 |
|------|----------|------|
| `x: Type = {...}` | `Type` | ✅ 类型构造器 |
| `f = {...}` | `Void` | ✅ 函数/变量（`{...}` 是代码块） |
| `f: (T: Type) -> Type = ...` | `Type` | ✅ 类型构造器 |
| `f: (T: Type) -> (() -> Void) = ...` | `() -> Void` | ❌ 泛型函数（不是类型） |

**关键区别**：
- 不是"有 `Type` 就是类型构造器"
- 而是"**最终返回 `Type`** 才是类型构造器"
- `(T: Type) -> (() -> Void)` 返回函数类型，是泛型函数

**编译期确定性**：
> Type 相关的内容都是**编译期确定**的。即使是泛型函数，T 也**不是运行时传入**的，而是**编译期单态化**。

**实现要点**：

1. **解析器**：有 `: Type` → 标记为类型构造器；无 `: Type` → 标记为函数/变量
2. **类型检查器**：
   - 有 `: Type` → `{ ... }` 解析为类型字面量
   - 无 `: Type` → `{ ... }` 解析为代码块
3. **HM 推断**：正确推断省略形式的函数类型
4. **错误处理**：当期望类型但实际是函数时，报错信息清晰
