## 概述
`yaoxiang check` 是一个命令行工具，用于对 YaoXiang 源码进行静态检查（类型检查、所有权检查、lint），不生成任何代码。它应提供快速的反馈，适合集成到 CI 流程或作为开发过程中的验证步骤。

## 目标
- 实现 `yaoxiang check` 命令，接受一个或多个文件/目录作为输入。
- 复用编译器的类型检查模块，输出错误和警告信息到标准错误。
- 支持 `--json` 选项，以 JSON 格式输出诊断信息，便于其他工具集成。
- 支持 `--watch` 模式（可选），监视文件变化并重新检查。
- 错误信息应包含文件名、行号、列号、错误等级和描述。

## 具体步骤
1. **设计命令行接口**（3 天）
   - 使用 `clap` 或类似库解析参数：`yaoxiang check [OPTIONS] <PATH>...`。
   - 支持 `--json`、`--watch`、`--color` 等选项。

2. **集成类型检查器**（1 周）
   - 将类型检查模块封装为可独立调用的函数，接收源代码字符串，返回诊断列表。
   - 支持多文件检查：需要解析所有文件并构建全局符号表。
   - 处理标准库的路径和内置定义。

3. **输出格式化**（3 天）
   - 实现类似 Rustc 的彩色错误输出，包含错误位置、错误等级和描述。
   - 实现 JSON 格式输出，遵循常见规范（如类似 `rustc --error-format=json`）。

4. **支持监视模式（可选）**（1 周）
   - 使用文件系统监控库（如 `notify`）监视文件变化，自动重新检查。
   - 避免重复全量检查，尽可能增量检查。

5. **测试与集成**（1 周）
   - 编写测试用例：对有错误的文件执行检查，验证输出是否符合预期。
   - 集成到 CI 流程中，确保每次提交都能通过检查。

## 验收标准
- 对单个无错误的文件执行 `yaoxiang check`，应无输出（或输出成功信息），退出码为 0。
- 对包含类型错误的文件执行检查，应输出清晰的错误信息（文件名、行号、列号、描述），退出码非 0。
- `--json` 选项输出有效的 JSON 数组，每个元素包含错误位置和消息。
- 检查速度应接近编译器类型检查的速度，适合频繁使用。
- 能够正确处理项目中的多个文件，并识别跨文件引用。

## 依赖
- 类型检查模块必须支持从文件系统读取文件并构建全局环境。
- 标准库的定义需要可访问（可能内置于编译器或通过路径指定）。
