# RFC-003：バージョン計画

> **ステータス**: 長期レビュー中
> **著者**: 晨煦
> **作成日**: 2025-01-05
> **最終更新**: 2025-01-06

## 概要

YaoXiang のバージョンリリース計画、v0.1 から v1.0 へのロードマップ。

**コア目標**：
1. **バイトコードコンパイル**：REPL と段階的コンパイルをサポート
2. **ブートストラップ (Bootstrap)**：YaoXiang で YaoXiang コンパイラを記述
3. **AOT コンパイル**：バイトコードからネイティブマシンコードへのコンパイル

## 一、動機

### なぜバージョ計画が必要か？

1. **プロジェクト管理**：目標を実装可能なマイルストーンに分解
2. **ユーザー期待**：ユーザーが言語の発展段階を理解できるようにする
3. **リソース配分**：各段階の重点を明確化
4. **リスク管理**：問題を早期発見し方向性を調整

### コア設計方針

- **バイトコード優先**：まずインタープリタ実行を実装し、その後に AOT を検討
- **段階的デリバリー**：各バージョンに実用的な機能を含める
- **後方互換性**：v1.0 前の API は変更される可能性ありつつも事前告知
- **ブートストラップ検証**：ブートストラップにより言語の表現力を証明
- **パフォーマンス分层**：まず動作させ、その後最適化

## 四、コンパイル戦略三層設計

| 階層 | バージョン | 入力 | 出力 | 説明 |
|------|------|------|------|------|
| L1: バイトコード | v0.1+ | ソースコード (.yx) | バイトコード (.yxb) | VM による解釈実行 |
| L2: ブートストラップ | v0.9+ | YaoXiang ソースコード | バイトコード | 自己コンパイル |
| L3: AOT | v1.0+ | ソースコード/バイトコード | 機械語 | ネイティブパフォーマンス |

**バイトコード優先の理由**：
1. **REPL サポート**：コード入力を即座にコンパイルし、対話型開発を実現
2. **段階的コンパイル**：1つの関数を修正してもその部分のみ再コンパイル
3. **プラットフォーム非依存**：.yxb ファイルはプラットフォーム間で実行可能で対応プラットフォームの VM のみ必要

## 五、依存関係戦略

**短期**：Rust ライブラリの呼び出しで crates.io を再利用（Cargo 寄生）

**現在の依存関係**：
- 并发：parking_lot, crossbeam, rayon
- データ構造：indexmap, hashbrown, smallvec
- ネットワーク：tokio
- シリアライゼーション：serde, ron

**長期**：独自の標準ライブラリとパッケージマネージャーの構築
