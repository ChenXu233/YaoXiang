# RFC-003：バージョニング計画

> **状態**: 長期レビュー中
> **著者**: 晨煦
> **作成日**: 2025-01-05
> **最終更新**: 2025-01-06

## 摘要

YaoXiang のバージョンリリース計画。v0.1 から v1.0 へのロードマップ。

**コア目標**：
1. **バイトコードコンパイル**：REPL と漸進的コンパイルをサポート
2. **自己ブートストラップ (Bootstrap)**：YaoXiang で YaoXiang コンパイラを記述
3. **AOT コンパイル**：バイトコードからネイティブマシンコードへのコンパイル

## 一、動機

### バージョニング計画が必要な理由

1. **プロジェクト管理**：目標を実行可能なマイルストーンに分解
2. **ユーザー期待**：ユーザーが言語の発展段階を理解できるようにする
3. **リソース配分**：各段階の重点を明確化
4. **リスク管理**：問題を早期に発見し方向性を調整

### コア設計判断

- **バイトコード優先**：まずインタープリタ実行を実装し、AOT は後回し
- **漸進的リリース**：各バージョンに実用的な機能を必ず 포함
- **後方互換性**：v1.0 前の API は変更될 수 있지만、事前に公告
- **自己ブート検証**：自己ブートにより言語の表現力を証明
- **パフォーマンス層別化**：まず動作させ、それから最適化

## 四、コンパイル戦略の三層設計

| レイヤー | バージョン | 入力 | 出力 | 説明 |
|----------|------------|------|------|------|
| L1: バイトコード | v0.1+ | ソースコード (.yx) | バイトコード (.yxb) | VM interpret 実行 |
| L2: 自己ブート | v0.9+ | YaoXiang ソースコード | バイトコード | 自己コンパイル |
| L3: AOT | v1.0+ | ソースコード/バイトコード | 機械語 | ネイティブパフォーマンス |

**バイトコード優先の理由**：
1. **REPL サポート**：入力コードを即座にコンパイル、対話型開発
2. **漸進的コンパイル**：単一関数を修正只需重新编译该部分只需再コンパイル
3. **プラットフォーム非依存**：.yxb ファイルはクロスプラットフォーム対応、対象プラットフォームの VM 只需

## 五、依存関係戦略

**短期**：Rust ライブラリの呼び出しによる crates.io の再利用（Cargo 寄生）

**現在の依存関係**：
- 並行処理：parking_lot, crossbeam, rayon
- データ構造：indexmap, hashbrown, smallvec
- ネットワーク：tokio
- シリアライゼーション：serde, ron

**長期**：独自の標準ライブラリとパッケージマネージャー構築

**v0.x シリーズ**：
- 機能的には完全だが、エッジケースに問題がある 可能

- API 凍結
- プロダクション使用に適合
