# RFC-003：バージョンプラン

> **ステータス**: 長期レビュー中
> **著者**: 晨煦
> **作成日**: 2025-01-05
> **最終更新**: 2025-01-06

## 摘要

YaoXiang のバージョンリリース計画。v0.1 から v1.0 までのロードマップ。

**コア目標**：
1. **バイトコードコンパイル**：REPL と漸進的コンパイルをサポート
2. **自举 (Bootstrap)**：YaoXiang で YaoXiang コンパイラを記述
3. **AOT コンパイル**：バイトコードからネイティブマシンコードへのコンパイル

## 一、動機

### バージョンプランが必要な理由

1. **プロジェクト管理**：目標を実行可能なマイルストーンに分解
2. **ユーザー期待**：言語の発展段階をユーザーに理解してもらう
3. **リソース配分**：各段階の重点を明確化
4. **リスク管理**：問題を早期発見し方向性を調整

### コア設計方針

- **バイトコード優先**：まずインタプリタ実行を実装し、その後 AOT を検討
- **漸進的デリバリー**：各バージョンで常に使用可能な機能を提供
- **後方互換性**：v1.0 前の API は変更される可能性があるが事前告知を行う
- **自举検証**：自举によって言語の表現力を証明
- **パフォーマンス分层**：まず動作させ、その後最適化

## 四、コンパイル戦略の三層設計

| レイヤ | バージョン | 入力 | 出力 | 説明 |
|------|------|------|------|------|
| L1: バイトコード | v0.1+ | ソースコード (.yx) | バイトコード (.yxb) | VM 解釈実行 |
| L2: 自举 | v0.9+ | YaoXiang ソースコード | バイトコード | 自己コンパイル |
| L3: AOT | v1.0+ | ソースコード/バイトコード | 機械語 | ネイティブパフォーマンス |

**バイトコード優先の理由**：
1. **REPL サポート**：入力コードを即座にコンパイル、対話式開発
2. **漸進的コンパイル**：単一の関数を修正した場合、その部分のみ再コンパイルすればよい
3. **プラットフォーム非依存**：.yxb ファイルはクロスプラットフォームで実行可能で対応プラットフォームの VM のみ必要

## 五、依存関係戦略

**短期**：Rust ライブラリの呼び出しで crates.io を再利用（Cargo 寄生）

**現在の依存関係**：
- 並发：parking_lot, crossbeam, rayon
- データ構造：indexmap, hashbrown, smallvec
- ネットワーク：tokio
- シリアライゼーション：serde, ron

**長期**：自作の標準ライブラリとパッケージマネージャ

- 機能が完全だが、エッジケースに問題がある可能性がある

- API が凍結
- 本番利用に適する
