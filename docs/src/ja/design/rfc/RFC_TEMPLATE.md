# RFC [番号]: [提案タイトル]

> **リファレンス**: RFCの書き方については [完全版サンプル](EXAMPLE_full_feature_proposal.md) を参照してください。

## 概要

このRFCが解決する問題と提案する解決策を1〜2文で要約してください。

## 動機

### なぜこの機能/変更が必要なのか？

このRFCが必要な理由や、解決する問題・満たすべきニーズについて説明してください。

### 現在の問題

現在の状態に存在する問題を、具体的な例（あれば）を使って記述してください。

## 提案

### コア設計

コアコンセプトやデータ構造を含め、提案する解決策を記述してください。

### 例

提案の使用方法を説明するためのコード例を提示してください：

```yaoxiang
# サンプルコード
```

### 文法の変更

提案が文法の変更を含む場合、変更前と変更後を対比して列挙してください：

| 変更前 | 変更後 |
|--------|--------|
| 文法 A | 文法 B |

## 詳細設計

### 型システムへの影響

型システムへの影響を記述してください。新しい型の導入や型制約などを含みます。

### ランタイム動作

ランタイム動作の変化（あれば）について記述してください。

### コンパイラの変更

修正が必要なコンパイラのコンポーネントを列挙してください。

### 後方互換性

後方互換性があるかどうかを評価し、既存コードの扱いについて記述してください。

#### 議論管理ルール

| 段階 | 操作 |
|------|------|
| 議論中 | 付録に記録し、オープン状態を維持する |
| 解決済み | 決議を更新し、ステータスをマークし、**本文の関連内容を更新する** |
| 破棄 | 破棄理由をマークし、参考のため記録を保持する |

> **重要**: 議論が確定したら、本文の関連内容を必ず更新してください！議論の記録は歴史的参考として保持できます。
