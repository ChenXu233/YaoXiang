# check コマンド設計ドキュメント

`yaoxiang check` は YaoXiang コンパイラの静的検査ツールであり、型検査、ファイル間分析、インクリメンタル検査機能を提供します。

## 設計原則

1. **誤報ゼロ**：報告されるすべてのエラーは実際のエラーでなければならない
2. **ファイル間対応**：モジュールをまたぐ型エラーと未定義参照を正しく検出
3. **インクリメンタル優先**：watch モードでは影響を受けたファイルのみ再検査
4. **自己文書化**：エラーコード、メッセージテンプレート、ヘルプテキストはすべて i18n で管理

## ドキュメントナビゲーション

- [診断システム](./diagnostic-system.md) — エラーコード体系、Diagnostic データ構造、Emitter 出力
- [ファイル間分析](./cross-file-analysis.md) — 共有型環境、依存関係グラフ、トポロジカルソート
- [インクリメンタル検査](./incremental-checking.md) — CheckSession、affected_modules、watch モード

## 他のシステムとの境界

| システム | 責務 | check との関係 |
|------|------|-----------------|
| コンパイラ (`yaoxiang build`) | 完全コンパイル（パース → 型検査 → コード生成） | check は前半の2段階のみ実行 |
| LSP | エディタ統合（補完、ジャンプ、診断） | check の診断結果は再利用可 |
| フォーマット (`yaoxiang fmt`) | コードスタイル | 独立しており、CI では並行使用 |
