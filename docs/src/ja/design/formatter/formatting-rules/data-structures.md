# データ構造のフォーマット規則

---

## §8 リテラル

**§8.1 整数リテラル。** 整数リテラルはそのまま出力する。

```
// ✅ 正しい
let x = 42;
```

**§8.2 浮動小数点リテラル。** 浮動小数点リテラルは小数点を含める必要がある。

```
// ✅ 正しい
let x = 3.14;
let y = 42.0;  // 小数点が必要

// ❌ 間違い
let y = 42;    // 整数であり、浮動小数点数ではない
```

**§8.3 文字列リテラル。** デフォルトでは二重引用符を使用する。`single_quote = true` の場合は単一引用符を使用する。

```
// デフォルト（二重引用符）
let s = "hello";

// single_quote = true
let s = 'hello';
```

**§8.4 ブールリテラル。** ブールリテラルは小文字を使用する。

```
// ✅ 正しい
let x = true;
let y = false;

// ❌ 間違い
let x = True;
let y = FALSE;
```

## §10 リストと辞書

**§10.1 リストのフォーマット。** リストは `[]` で囲み、要素間はカンマで区切る。

```
// ✅ 正しい
let x = [1, 2, 3];

// ❌ 間違い
let x = [1,2,3];
```

**§10.2 辞書のフォーマット。** 辞書は `{}` で囲み、キーと値のペアは `key: value` 形式を使用する。

```
// ✅ 正しい
let x = {"a": 1, "b": 2};

// ❌ 間違い
let x = {"a":1, "b":2};
```

**§10.3 リスト内包表記。** リスト内包表記は `[expr for var in iterable]` 形式を使用する。

```
// ✅ 正しい
let x = [i * 2 for i in range(10)];

// 条件付き
let x = [i for i in range(10) if i > 5];
```

## §11 Match式

**§11.1 Matchのフォーマット。** `match` キーワードと式の間にスペースを挿入する。

```
// ✅ 正しい
match x { ... }

// ❌ 間違い
match(x) { ... }
```

**§11.2 Patternのアライメント。** 複数のPatternはアラインし、スペースで埋める。

```
// ✅ アライン
match x {
    1    => "one",
    2    => "two",
    100  => "hundred",
    _    => "other",
}
```

**§11.3 Pattern过长の折り返し。** Pattern过长の場合、Patternを折り返し、`=>` とbodyはアラインする。

```
// ✅ 折り返し
match x {
    VeryLongPatternName { field1, field2 }
        => handle_case(field1, field2),
    _ => default_case(),
}
```

## §11.4 タプル

**§11.4.1 タプルのフォーマット。** タプルは `()` で囲み、要素間はカンマで区切る。

```
// ✅ 正しい
let t = (1, "hello", true);
let t = (1,);  // 単一要素タプル

// ❌ 間違い
let t = (1, "hello", true);  // カンマ後にスペースがない
let t = (1,"hello",true);  // カンマ後にスペースがない
```

**§11.4.2 空のタプル。** 空のタプルは `()` で表す。

```
// ✅ 正しい
let t = ();
```

## §11.5 インデックスアクセス

**§11.5.1 インデックスのフォーマット。** インデックスは `expr[index]` 形式を使用する。

```
// ✅ 正しい
let x = arr[0];
let y = matrix[i][j];

// ❌ 間違い
let x = arr [0];  // 余分なスペース
let y = matrix[ i ][ j ];  // 余分なスペース
```

---

## §11.6 フィールドアクセス

**§11.6.1 フィールドアクセスのフォーマット。** フィールドアクセスは `expr.field` 形式を使用する。

```
// ✅ 正しい
let x = obj.field;
let y = obj.method();

// ❌ 間違い
let x = obj . field;  // 余分なスペース
let y = obj. field;  // 余分なスペース
```

**§11.6.2 チェーンのフィールドアクセス。** チェーンのフィールドアクセスが行幅を超える場合、メソッド呼び出しは1行に1つずつにする。

```
// 行幅を超える場合
let result = object.method1().method2().method3().method4();

// フォーマット後
let result = object
    .method1()
    .method2()
    .method3()
    .method4();
```
