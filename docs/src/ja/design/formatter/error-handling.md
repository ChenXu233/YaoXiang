# エラーの処理

---

## §E1 フォーマットエラー

**§E1.1 構文エラー。** ソースコードに構文エラーが含まれている場合、フォーマッタは次のことを行う必要がある：

1. 正しい部分を可能な限りフォーマットする
2. エラーノードの元の内容を保持する
3. エラー位置に `/* error */` プレースホルダを挿入する

**§E1.2 設定エラー。** 設定ファイルの形式が不正な場合、明確なエラーメッセージを返すべきである。

---

## §E2 終了コード

| 終了コード | 意味 |
|--------|------|
| 0 | 成功 |
| 1 | `--check` モードでフォーマットされていないファイルが見つかった |
| 2 | ファイルが見つからないか設定エラー |

---

## §E3 エラーからの回復

**§E3.1 エラープレースホルダ。** ソースコードに解析できない式が含まれている場合、フォーマッタは `/* error */` プレースホルダを挿入する必要がある。

```
// 元のコード（構文エラーあり）
let x = ;

// フォーマット後
let x = /* error */;
```

**§E3.2 エラーへの許容対応。** フォーマッタは正しい部分を可能な限りフォーマットし、エラーノードの元の식을保持해야 한다.

```
// 元のコード
fn foo() {
    let x = 1;
    let y = ;  // 構文エラー
    let z = 3;
}

// フォーマット後
fn foo() {
    let x = 1;
    let y = /* error */;
    let z = 3;
}
```
