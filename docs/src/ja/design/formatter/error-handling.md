# エラー処理

---

## §E1 フォーマットエラー

**§E1.1 構文エラー。** ソースコードに構文エラーが含まれている場合、フォーマッタは以下を行うべきである:

1. `parse()` 関数を用いて解析を行う
2. 解析でエラーが発生した場合、エラーメッセージをそのまま返す
3. プレースホルダは一切挿入しない

**§E1.2 設定エラー。** 設定ファイルの形式が不正な場合は、明確なエラーメッセージを返すべきである。

---

## §E2 終了コード

| 終了コード | 意味 |
|--------|------|
| 0 | 成功 |
| 1 | `--check` モードで未フォーマットのファイルを検出 |
| 2 | ファイルが見つからない、または設定エラー |

---

## §E3 エラー処理

**§E3.1 エラー報告。** フォーマッタは `parse()` 関数を用いてソースコードを解析する。解析中にエラーが発生した場合、フォーマッタはフォーマットを行わず、エラーメッセージをそのまま返す。

**§E3.2 プレースホルダ禁止。** フォーマッタはエラー位置にプレースホルダ（例: `/* error */`）を挿入しない。構文エラーに遭遇した場合、フォーマッタはエラーを直接報告し、処理を終了する。
