# リリーステンプレート

> `/release` コマンドはこのテンプレートに従って変更履歴を生成する。  
> 変更履歴は**人が読むための変更説明**であり、コミットリストではない。

## フォーマット規約

```
:bookmark: V<バージョン番号>: <タイトル>
```

## 📦 バージョン情報

| 項目         | 値                            |
| ---------- | ---------------------------- |
| リリース日    | YYYY-MM-DD                    |
| バージョン変更 | `<旧バージョン>` → `<新バージョン>` |
| コミット数    | N 個のコミット                    |

## タイトルルール

核心的な変更を一文で要約し、50文字以内：

```
:bookmark: V0.7.2: REPL書き直しと型システム改善
:bookmark: V0.7.3: 型システム修正と所有権モデル改善
:bookmark: V0.8.0: 並行モデルとジェネリクスシステム
```

## 分類ルール

`type` プレフィックスで分類し、空の分類は省略：
| type        | 分類                       | プレフィックス            |
|:-----------:|:-------------------------:|:------------------:|
| `feat`      | ✨ 新機能                    | `:sparkles:`       |
| `fix`       | 🐛 バグ修正                  | `:bug:`            |
| `refactor`  | ♻️ リファクタリング             | `:recycle:`        |
| `perf`      | ⚡ パフォーマンス最適化         | `:zap:`            |
| `docs`      | 📝 ドキュメント              | `:memo:`           |
| `style`     | 🎨 フォーマット              | `:art:`            |
| `test`      | ✅ テスト                    | `:white_check_mark:` |
| `chore`     | 🔧 ビルド/ツール              | `:wrench:`         |
| `ci`        | 💚 CI/CD                   | `:green_heart:`    |

## コミット記録の取得

```bash
git log <最新タグ>..HEAD --oneline --no-merges
```

## 📦 バージョン情報

## 📋 今回の更新概要

今回のリリースは型システムと所有権モデルの安定性修正に焦点を当てている。
ジェネリクス型のインスタンス化（例：`List(Int) = List(1, 2, 3)`）が
正しく動作するようになり、`&T` フィールドアクセスとタプル分解代入の
複数の境界ケースが修正された。同時に所有権検査の統一リファクタリング
を完了し、後の move セマンティクスの改善の基盤を整えた。

## ✨ 新機能

### ジェネリクス型のインスタンス化

`List(Int) = List(1, 2, 3)` 構文をサポートし、型コンストラクタが
Struct として正しく登録される。ジェネリクス関数の複数回呼び出し時
に型推論が衝突しなくなり、呼び出しごとに独立した型変数のインスタ
ンスを取得する。

- 型コンストラクタが Struct として登録され、インスタンス化の根本原因を修正
- ジェネリクス関数の複数回呼び出しで独立した推論
- インターフェースメソッドバインディングの修正

### Move 後の再バインド

変数が move された後、新しい値を再バインド可能。代入優先検索が
moved 状態を正しく処理するようになり、誤った「変数は移動済み」
エラーの報告を回避する。

- VarInfo に moved 状態フラグを追加
- 代入優先検索に moved 分岐を追加

### Resource マーカートレイト

`Resource` マーカートレイトを追加し、IO 副作用を実装する型をマー
クする。後の副作用追跡と並行安全性分析の基盤を提供する。

### PLDI SRC demo

PLDI SRC demo MVP の全タスクを完了し、エンドツーエンドの型検査と
コード生成デモを含む。

## 🐛 バグ修正

### &T フィールドアクセス

不変参照型のフィールドアクセスにおける型検査の複数の問題を修正。
フィールド代入や参照経由のコンストラクタ呼び出しの推論を含む。

- `&T` フィールドアクセスの型検査修正
- フィールド代入のターゲット型推論
- 参照経由のコンストラクタ呼び出しの推論

### LSP セマンティックハイライト

`spawn {}` ブロック内の変数とタプル分解代入 `(a, b) = ...` のセマ
ンティックハイライト欠落を修正。AST 内の `DestructureAssign.names`
が各変数名の位置情報を保持するようになった。

- spawn ブロック内の変数が正しくハイライト
- タプル分解代入の変数名が正しくハイライト
- 後の move セマンティクスの LSP サポートの基盤を整える

### freeze の削除

非推奨の `freeze` 関数を削除し、関連するテストケースをクリーン
アップ。

## ♻️ リファクタリング

### 診断システムの統一

残り 5 つのエラー列挙を統一された `ErrorCodeDefinition` 診断シス
テムに移行し、エラーコードのカバレッジを 60% から 80% に向上。
すべてのコンパイルエラーが統一されたフォーマットとエラーコード規
約に準拠するようになった。

### 所有権検査の統一

分散していた所有権検査ロジックを `OwnershipPass` に統一し、単一
エントリで borrow、move、ライフタイム検査を処理。メンテナンスコ
ストを削減し、エラーの一貫性を向上。

### 並行モデルの簡素化

Send/Sync 制約システムを削除し、`spawn {}` をグループ実行モデル
にリファクタリング。`@block`/`@eager`/`@auto` デコレータ関連
コードを削除（RFC-024 phase 1）。並行モデルを簡素化し、後の拡張
可能な並行プリミティブ設計への道を開く。
