# タスク：yaoxiang fmt コード整形ツールの実装

## 概要
`yaoxiang fmt` は、YaoXiang ソースコードを自動的に整形するツールであり、コミュニティのコードスタイルを統一し、フォーマットに関する議論を減らすすることを目的としています。このツールは、言語仕様（RFC-010 など）とユーザーの設定に基づいて、コードを再整形します。

## 目標
- `yaoxiang fmt` コマンドを実装し、ファイルまたはディレクトリを入力として受け取り、コードを整形して出力（またはその場で変更）する。
- `--check` モードをサポート：コードがすでに整形されているかどうかを確認し、ファイルを修正せず、0 以外の終了コードを返す。
- 設定ファイル（例：`yaoxiang.toml`）をサポートして、改行幅や折り返しスタイルなど、整形ルールをカスタマイズ可能にする。
- コメントと元の空行の意図を保持（高度な処理）。
- 生成されたコードは意味を変えずに保ち、言語構文に準拠する必要がある。

## 具体的な手順

1. **整形ルールの設計**（1 週間）
   - RFC-010 の統一構文に基づいて、インデント、スペース、折り返しの約束事を決定する。
   - Rustfmt の設計を参照し、設定可能なオプションを定義：インデントサイズ、最大行幅、フィールド配置など。
   - コミュニティでの議論のための設計ドキュメントを執筆する。

2. **AST からコードへの再印刷の実装**（2 週間）
   - 完全な AST を取得するために既存の解析器を活用する（コメントの位置情報を含む。当前の解析器がコメントを破棄している場合は、解析器を拡張する必要がある）。
   - 整形後のコードを出力するために、ルールに従って AST を走査する「印刷器」（Printer）を設計する。
   - 特殊情况の処理：複数行文字列、インラインコメント、空行など。

3. **設定のサポート**（1 週間）
   - プロジェクトルートの `yaoxiang.toml` を読み込み、デフォルト設定とマージする。
   - 設定を整形器に渡して、インデントや折り返しなどの動作に影響を与える。

4. **`--check` モードの実装**（3 日間）
   - ファイルを整形した後、元の内容和整形後の内容を比較し、差異がある場合はエラーとして報告し、差異のあるファイルを一覧表示する。
   - 終了コードが 0 以外の場合少なくとも 1 つのファイルが整形を必要とすることを意味する。

5. **テストと統合**（1 週間）
   - すべての構文特性をカバーする大量のテストケースを記述し、整形前と整形後で意味が一致することを保証する。
   - 様々な状況で `--check` モードが正しく動作することをテストする。
   - 十分なパフォーマンスを確保し、毎秒数千行のコードを処理できるようにする。

## 验收基準
- 変数、関数、型定義、メソッド、制御フローを含む完全な YaoXiang ファイルを整形できること。
- 整形後のコードが解析と型チェックに合格すること。
- コメントと空行は合理的に保持され、誤って削除または移動されないこと。
- `--check` モードが未整形のファイルを正しく検出して報告できること。
- カスタムインデント幅（2 スペース/4 スペース/タブ）をサポートすること。
- 一般的なエッジケースの処理：複数行関数呼び出し、長い式の折り返し、フィールドリストの配置など。

## 依存関係
- 解析器はコメントと空白情報を保持する必要がある（または AST からコードを再構築できるようにソースマッピングを提供する）。当前の解析器がこれらの情報を破棄している場合は、まず解析器を拡張する必要がある。
- 言語の完全な構文木構造を理解する必要がある。
