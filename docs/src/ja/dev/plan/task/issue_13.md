# タスク：yaoxiang fmt コードフォーマットの実装

## 概要
`yaoxiang fmt` は、YaoXiang ソースコードを自動的にフォーマットするツールであり、コミュニティのコードスタイルを統一し、フォーマットの議論を減らすことを目的としている。言語仕様（RFC-010 など）とユーザー設定に基づき、コードを再整形する。

## 目標
- `yaoxiang fmt` コマンドを実装し、ファイルまたはディレクトリを入力として受け入れ、コードをフォーマットして出力（またはインプレースで修正）する。
- `--check` モードをサポート：コードがすでにフォーマットされているかをチェックし、ファイルは変更せず、非ゼロの終了コードを返す。
- 設定ファイル（例：`yaoxiang.toml`）をサポートし、フォーマットルール（インデント幅、改行スタイルなど）をカスタマイズ可能にする。
- コメントと元の空行の意図を保持する（インテリジェントな処理）。
- 生成されたコードはセマンティックに変更がなく、言語の構文に準拠する必要がある。

## 具体的な手順

### 1. フォーマットルールの設計（1週間）
- RFC-010 の統一された構文に基づき、インデント、スペース、改行の約束事を決定する。
- Rustfmt の設計を参照し、構成可能なオプションを定義する：インデントサイズ、最大行幅、フィールド配置など。
- コミュニティでの議論のための設計ドキュメントを作成する。

### 2. AST からコードへの再プリントの実装（2週間）
- 完全な AST を取得するために既存の解析器を活用する（コメントの位置情報を含む。現在の解析器がコメントを破棄している場合は、增强が必要）。
- AST を走査し、ルールに従ってフォーマットされたコードを出力する「プリンター」を設計する。
- 特殊な情况的対処：複数行文字列、インラインコメント、空行など。

### 3. 設定のサポート（1週間）
- プロジェクトルートにある `yaoxiang.toml` を読み込み、デフォルト設定とマージする。
- 設定を外字プリンターに渡して、インデント、改行などの動作に影響を与える。

### 4. `--check` モードの実装（3日間）
- ファイルをフォーマットした後、元の内容和フォーマット後の内容を比較し、差異がある場合はエラー報告し、差異のあるファイルを列表する。
- 終了コードが非ゼロの場合は少なくとも1つのファイルがフォーマットが必要であることを意味する。

### 5. テストと統合（1週間）
- すべての構文特性をカバーする大量のテストケースを作成し、フォーマット前後のセマンティクスの整合性を確保する。
- `--check` モードのさまざまな状況での正確性をテストする。
- パフォーマンスが十分に高速であることを确保し、毎秒数千行のコードを処理できる。

## 受け入れ基準
- 変数、関数、型定義、メソッド、制御フローittiを含む完全な YaoXiang ファイルをフォーマットできる。
- フォーマットされたコードは解析と型チェックを通過できる。
- コメントと空行は合理的に保持され、誤って削除または移動されない。
- `--check` モードは未フォーマットのファイルを正しく検出し報告できる。
- カスタムインデント幅（2スペース/4スペース/タブ）をサポートする。
- 一般的なエッジケース的处理：複数行関数呼び出し、長い式iiiii改行、フィールドリスト配置など。

## 依存関係
- 解析器はコメントと空白情報を保持する必要がある（または AST からコードを再構築できるソースマッピングを提供する）。現在の解析器がこれらの情報を破棄している場合は、まず解析器を拡張する必要がある。
- 言語の完全な構文木構造を理解する必要がある。
