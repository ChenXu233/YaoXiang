# タスク：LSP（Language Server Protocol）サポートの実装

## 概要

YaoXiang の开发者体験を強化するため、Language Server Protocol に準拠した言語サーバーを実装します。このサーバーはコンパイラー前端に組み込まれ、リアルタイムのコード解析、补全、ジャンプ、診断などの機能を提供し、主要なエディター（VS Code、Vim、Emacs など）との統合を可能にします。

## 目標

- YaoXiang 言語サーバーバイナリ `yaoxiang-lsp` を実装します。
- 以下の LSP リクエストをサポートします：
  - `textDocument/didOpen` / `didChange` / `didSave` / `didClose`：ファイル内容の同期
  - `textDocument/documentSymbol`：ドキュメントシンボル（関数、型、変数など）のリスト表示
  - `textDocument/completion`：スコープ内シンボルに基づくコード补全候補の提供
  - `textDocument/definition`：定義位置へのジャンプ
  - `textDocument/hover`：ホバー時の型情報とドキュメントコメントの表示
  - `textDocument/publishDiagnostics`：リアルタイムの構文エラー、型エラーなどの診断情報のプッシュ
- コンパイラーの增量コンパイル/チェック機能を実装し、常に全量解析するのを避けます。

## 具体的なステップ

### 1. サーバーアーキテクチャの設計（1 週）

- 既存のコンパイラー前端ライブラリ（字句解析、パーサー、型チェック）に基づいて構築します。
- 非同期 I/O（Tokio など）を使用して LSP リクエストを処理します。
- ドキュメントキャッシュと增量更新のデータ構造を設計します。

### 2. 基本 LSP プロトコル通信の実装（1 週）

- JSON-RPC メッセージの送受信を実装します。
- 初期化リクエスト、关闭リクエストなどのライフサイクル管理を処理します。

### 3. コンパイラー前端の統合（2 週）

- 字句解析、パーサー、型チェックを再entrant なライブラリ関数としてラップします。
- 各ドキュメントに対して「コンパイルセッション」を維持し、增量更新（変更部分のみを再解析）をサポートします。
- ジャンプとホバー用の AST ノードからソースコード位置へのマッピングを実装します。

### 4. 診断機能の実装（1 週）

- 型チェック 과정에서発生したエラーと警告をキャプチャし、LSP 診断メッセージに変換します。
- ファイル変更後に自動的に診断をプッシュします。

### 5. 补全とシンボルナビゲーションの実装（2 週）

- スコープシンボルテーブルを構築し、补全候補リスト（キーワード、組み込み型、現在のスコープ変数など）を提供します。
- 定義へのジャンプを実装します：カーソル位置に基づいてシンボルが定義されているファイルと位置を見つけます。
- ホバー情報を実装します：型シグネチャと関連するドキュメントコメント（サポートされている場合）を表示します。

### 6. テストと統合（1 週）

- テスト用の VS Code クライアントプラグイン（シンプルなラッパー）を記述します。
- すべての機能が一般的なシナリオで正常使用可能であることを確認し、明確なパフォーマンス問題がないことを保証します。

## 验收基準

- サーバーが正常に起動し、初期化リクエストに応答できる。
- YaoXiang ファイルを開くと、診断情報（構文エラー、型エラー）がエディターにリアルタイムで表示される。
- 変数名または関数名にホバーすると、その型情報が表示される。
- 識別子から定義にジャンプすると、対応するファイルが開かれ、定義行にカーソルが移動する。
- ドット（.）を入力した後、レコード型のフィールドを补完できる（型情報が必要）。
- 大型ファイル（数千行）でのサーバーの応答時間が許容範囲内（<500ms）である。

## 依存関係

- コンパイラー前端ライブラリは安定しており、AST、シンボルテーブル、型情報を取得するための API を提供する必要があります。
- 增量コンパイル機能（少なくともドキュメントレベルのキャッシュ）の実装が必要です。
