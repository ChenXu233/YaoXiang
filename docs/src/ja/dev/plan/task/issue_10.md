# タスク：言語特性を検証し、エコシステムを蓄積するための小型アプリケーション開発

## 概要

YaoXiang のフロントエンド、ミドルエンド、インタープリター后台はすでに完成しており、コンパイラ后台は開発中である。言語の実用性をテストし、隠れたバグを発見し、サンプルコードを蓄積し、早期ユーザーを獲得するために、一連の小型アプリケーションを開発する必要がある。これらのアプリケーションは解释器上でスムーズに動作し、コンパイラ后台の高性能最適化に依存せず、同時に YaoXiang の統一的構文、所有権モデル、フィールドレベル可変性などのコアデザインを十分に展示する必要がある。

## 目標

少なくとも3つの異なる分野の小型アプリケーションを開発し、脚本ツール、アルゴリズム実装、設定解析などのシナリオをカバーする。

各アプリケーションには、完全なソースコード、テストケース、簡潔なドキュメントを含めること。

開発プロセスを通じて、言語設計の問題をフィードバックし、標準ライブラリとツールチェーンを完善する。

## 推奨アプリケーション方向（選択可または自行設計）

### ファイルバッチリネームツール

機能：指定ディレクトリを巡回し、正規表現でファイル名をマッチングし、リネーム（置換、プレフィックス/サフィックス追加など）を実行する。

涉及的特性：ファイルシステム操作（FFIまたは標準ライブラリが必要）、文字列処理、所有権管理（リソースリーク防止）。

出力：実行可能スクリプト、コマンドライン引数をサポート。

### クイックソートアルゴリズムデモ

機能：古典的なクイックソートアルゴリズムを実装し、整数配列のソートをサポート。

涉及的特性：再帰、配列/リスト操作、泛型（対応している場合）、所有権フロー（配列要素の移動）。

出力：関数ライブラリ + テストプログラム、算法的正確性を展示。

### TOML 設定解析プロトタイプ

機能：単純な TOML 設定ファイルを解析（基本型とテーブルのみサポート）、YaoXiang 記録类型にマッピング。

涉及的特性：文字列解析、記録类型、エラー処理。

出力：解析関数 + サンプル設定ファイルと使用コード。

### ターミナル三目並べゲーム

機能：ターミナルで2人が对战する三目並べ、盘面表示、手を置く、勝敗判定をサポート。

涉及的特性：状態管理（フィールド可変性）、入出力、循环制御。

出力：実行可能なゲームプログラム。

## 検収基準

アプリケーションが予想通りに動作し、すべての手动テストに合格すること。

コードが YaoXiang スタイル規範（該当する場合）に準拠すること。

メモリ安全エラーがないこと（所有権規則が自動的に保証）。

開発プロセス中发现された言語バグや標準ライブラリ欠落は отдельным issueとして記録済みであること。

## 依存関係

インタープリター后台が安定して利用可能であること。

標準ライブラリのファイル I/O、文字列操作などのモジュールが必要になる場合がある。
