# 小規模アプリケーションによる言語機能の検証とエコシステムの蓄積

## 概要

現在、YaoXiang のフロントエンド、ミドルエンド、インタープリタ後端は完成しており、コンパイラ後端は開発中です。言語の実用性をテストし、隠れたバグを発見し、サンプルコードを蓄積し、早期ユーザーを獲得するためには、一連の小規模アプリケーションを開発する必要があります。これらのアプリケーションは解释器上でスムーズに動作し、コンパイラ後端の高性能最適化に依存せず、YaoXiang の統一構文、所有権モデル、フィールドレベルの可変性などのコアデザインをよく展示する必要があります。

## 目標

3つ以上の異なる分野小規模アプリケーションを開発し、スクリプトツール、アルゴリズム実装、設定解析などのシナリオをカバーすること。

各アプリケーションには、完全なソースコード、テストケース、簡潔なドキュメントを含めること。

開発プロセスを通じて、言語設計の問題点をフィードバックし、標準ライブラリとツールチェーンを完善すること。

## 推奨アプリケーション方向（選択可または自行設計）

### ファイル一括リネームツール

機能：指定されたディレクトリを走査し、正規表現に従ってファイル名をマッチングし、リネーム（置換、接頭辞/接尾辞の追加など）を実行する。

関連する特性：ファイルシステム操作（FFI または標準ライブラリが必要）、文字列処理、所有権管理（リソースリークの防止）。

出力：実行可能スクリプト、コマンドライン引数のサポート。

### クイックソートアルゴリズムデモンストレーション

機能：古典的なクイックソートアルゴリズムを実装し、整数配列のソートをサポート。

関連する特性：再帰、配列/リスト操作、泛型（サポートされている場合）、所有権フロー（配列要素の移動）。

出力：関数ライブラリ + テストプログラム、アルゴリズムの正確性を展示。

### TOML 設定解析プロトタイプ

機能：単純な TOML 設定ファイルを解析（基本型とテーブルのみをサポート）し、YaoXiang 記録タイプにマッピング。

関連する特性：文字列解析、記録タイプ、エラー処理。

出力：解析関数 + サンプル設定ファイルと使用コード。

### ターミナル三目並べゲーム

機能：ターミナルで2人が対戦する三目並べ、盤面の表示、手の配置、勝敗判断をサポート。

関連する特性：状態管理（フィールドの可変性）、入出力、 ループ制御。

出力：実行可能なゲームプログラム。

## 検収基準

アプリケーションが予想通りに動作し、すべてのmanualテストに合格すること。

コードが YaoXiang スタイル規範（該当する場合）に準拠すること。

メモリ安全エラーがないこと（所有権ルールが自動的に保証）。

開発プロセス中发现された言語バグや標準ライブラリの欠落は、個別の issue として記録されること。

## 依存関係

インタープリタ後端が安定して利用可能であること。

標準ライブラリのファイル I/O、文字列操作などのモジュールおそらく必要であること。
