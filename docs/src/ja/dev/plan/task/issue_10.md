# タスク：小型アプリケーション開発による言語機能の検証とエコシステム蓄積

## 概要

現在、YaoXiang のフロントエンド、ミドルエンド、インタプリタバックエンドは完成しており、バックエンドコンパイラは開発中です。言語の実用性をテストし、隠れたバグを発見し、サンプルコードを蓄積し、早期ユーザーを獲得するために、一連の小型アプリケーション開発が必要です。これらのアプリケーションは、インタプリタ上でスムーズに動作し、コンパイラバックエンドの高性能最適化に依存せず、YaoXiang の統合構文、所有権モデル、フィールドレベルの可変性などのコアデザインを十分に展示する必要があります。

## 目標

3 つ以上の異なる分野にわたる小型アプリケーションを開発し、スクリプトツール、アルゴリズム実装、設定解析などのシナリオをカバーします。

各アプリケーションには、完全なソースコード、テストケース、簡潔なドキュメントを含める必要があります。

開発プロセスを通じて、言語設計の問題フィードバック、標準ライブラリとツールチェーンの完善を行います。

## 推奨アプリケーション方向（選択可または独自設計）

### ファイル一括リネームツール

機能：指定されたディレクトリを走査し、正規表現でファイル名をマッチングして、リネーム（置換、プレフィックス/サフィックスの追加など）を実行します。

関連する特性：ファイルシステム操作（FFI または標準ライブラリが必要）、文字列処理、所有権管理（リソースリークの防止）。

出力：実行可能スクリプト、コマンドライン引数をサポート。

### クイックソートアルゴリズムデモ

機能：クラシックなクイックソートアルゴリズムを実装し、整数配列のソートをサポート。

関連する特性：再帰、配列/リスト操作、ジェネリクス（対応している場合）、所有権フロー（配列要素の移動）。

出力：関数ライブラリ + テストプログラム、アルゴリズムの正確性を展示。

### TOML 設定解析プロトタイプ

機能：シンプルな TOML 設定ファイルを解析（基本型とテーブルのみ対応）し、YaoXiang レコード型にマッピング。

関連する特性：文字列解析、レコード型、エラー処理。

出力：パーサー関数 + サンプル設定ファイルと使用コード。

### ターミナル三目並べゲーム

機能：ターミナルで二人のプレイヤーが对战する三目並べ、盤面表示、石を置く、勝敗判定をサポート。

関連する特性：状態管理（フィールドの可変性）、入出力、ループ制御。

出力：実行可能なゲームプログラム。

## 検収基準

アプリケーションは期待通りに動作し、すべてのマニュアルテストに合格します。

コードは YaoXiang スタイル規範（もしあれば）を遵守します。

メモリ安全エラーなし（所有権規則が自動的に保証）。

開発プロセスで発見された言語バグや標準ライブラリの欠落は、 отдельный issue として記録済み。

## 依存関係

インタプリタバックエンドは安定して利用可能である必要があります。

ファイル I/O、文字列操作などの標準ライブラリモジュール、おそらく必要。
