# タイプ宇宙思想実装計画：Fn ASTノードの除去

> ドキュメントバージョン：1.3
> 作成日：2026-03-18
> 更新日：2026-03-21
> タスク：関数/タイプコンストラクタ/インターフェースなどの概念をBindingノードに統一し、関数判定を型チェック段階に延期

---

## 背景と目標

タイプ宇宙思想ドキュメントに基づき、以下のコア変更を実装する：

1. **Fn ASTノードの除去**：すべての宣言を`Binding`ノードに統一
2. **関数判定を型チェック段階に延期**：パーサーは構文のみを担当し、意味は戻り値类型で決定
3. **統一構文モデル**：関数、タイプコンストラクタ、インターフェースはすべて`identifier: type = expression`形式を使用

---

## 段階一：フロントエンド - ASTをBindingノードに統一

### ステップ 1.1：ASTノード定義の変更

**変更目標**：`StmtKind::Fn`と`StmtKind::MethodBind`を統一して`StmtKind::Binding`に置き換える

**設計判断（ユーザー確認済み）**：
- **MethodBindもBindingである**：メソッドバインディングは通常の関数と統一構文モデルを使用
- **Expr::Lambdaは個別に保持**：Lambda式は独立した構文だが、関数BindingはLambdaに依存
- **Lambda簡略化**：型ヘッダがパラメータ情報を含むため、Lambdaはbodyのみを保持するように簡略化可能
- **完全置換**：旧構文の後方互換は不要

**主要ファイル**（元計画：1個 → 実際：2個）：
- `src/frontend/core/parser/ast.rs`
- `src/frontend/core/parser/tests/concurrency.rs` ⚠️

**受入基準**：
- [x] `StmtKind::Fn` Variantsが除去済み
- [x] `StmtKind::MethodBind` Variantsが除去済み、`StmtKind::Binding`に統一
- [x] 新規`StmtKind::Binding` Variants包含：name, type_name, method_type, generic_params, type_annotation, eval, params, body, is_pub
- [ ] `Expr::Lambda` Variantsは保持、構造簡略化（Lambdaヘッダ除去、bodyのみ保持）
- [ ] `Type::Fn` Variants保持（型式中の関数类型用）

**状態**：✅ 完了（2026-03-20）

**実施記録**：
- 完了日時：2026-03-20
- 新規`StmtKind::Binding` Variants追加、Fn、TypeDef、MethodBindを統一
- 22ファイルの参照点をすべて更新済み
