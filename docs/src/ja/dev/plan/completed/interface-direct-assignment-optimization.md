## 一、概要

### 1.1 目標

インターフェース（制約）型の直接代入構文を再実装し、vtable オーバーヘッドを排除するためのコンパイル時最適化を実現する：

1. **直接インターフェース代入的支持**：`d: Drawable = Circle(1)`
2. **コンパイル時の型推論**：具体的型を可能な限り確定する
3. **ゼロオーバーヘッド最適化**：具体的型が確定している場合は直接呼び出しを使用
4. **vtable フォールバック**：具体的型が確定できない場合は vtable を使用

### 1.2 現在の状態

| 機能 | 状態 |
|------|------|
| vtable ランタイム実装 | ✅ 実装済み |
| 仮想メソッド呼び出し IR | ✅ 実装済み |
| ジェネリクス制約 `[T: Drawable]` | ✅ 実装済み |
| 直接インターフェース代入 `s: Drawable = point` | ✅ 実装済み |
| 制約サブタイプチェック | ✅ 実装済み |
| コンパイル時型推論最適化 | ✅ 実装済み |
| CallVirt エクゼキュータ実装 | ✅ 実装済み |

### 1.3 設計変更

**変更前**：
```yaoxiang
d: Drawable = Circle(1)  # ❌ コンパイルエラー
```

**変更後**：
```yaoxiang
d: Drawable = Circle(1)  # ✅ 許可

# コンパイル時推論：
# - Circle が具体的型で確定可能 → 直接呼び出し（ゼロオーバーヘッド）
# - 型が確定できない → vtable を使用
```

| 位置 | 変更内容 |
|------|----------|

**新規セクション内容**:

新規セクション「六、コンパイル時最適化戦略」：

1. **ゼロオーバーヘッドインターフェース呼び出し**：インターフェース代入の右辺が具体的型で確定可能な場合、コンパイラはゼロオーバーヘッド呼び出しに最適化

   - ソースコード：`d: Drawable = Circle(1); d.draw(screen)`
   - コンパイル後：`circle_draw(screen)` を直接呼び出し、vtable なし

2. **推論ルール**：

   | シナリオ | 推論結果 | 呼び出し方式 |
   |------|----------|----------|
   | 具体的型の直接代入 | 具体的型 Circle | 直接呼び出し |
   | 関数戻り値 | 不明 | vtable |
   | 異種混成コレクション | 複数型 | vtable |

3. **vtable フォールバック**：具体的型が確定できない場合は vtable メカニズムを使用

**受入方式**:
