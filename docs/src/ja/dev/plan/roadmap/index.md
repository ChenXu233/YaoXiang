> **現在のバージョン**：v0.7.2

---

## プロジェクト概要

YaoXiang は未来的なプログラミング言語であり、バイトコードコンパイル + VM インタプリタ実行アーキテクチャを採用しています。現在の v0.7.x 段階では、コアコンパイラとランタイムの基本部分は既に完成しています。

## コア発見

### 完了した領域

1. **コンパイラフロントエンド**：字句解析、パース、型チェックがすべて完成、テストカバー率は優秀
2. **所有権システム**：完全な借用検査器、15 中 14 のサブモジュール機能が完成
3. **ツールチェーン**：LSP、パッケージ管理、フォーマッターが基本完成
4. **ドキュメントサイト**：多言語サポート、CI/CD 自動デプロイ

### 主要なギャップ

1. **並作モデル**（RFC-001）：DAG 依存関係アナライザ未実装、並行制御アノテーションが不完全
2. **ランタイム三層アーキテクチャ**（RFC-008）：フェーズ A のみ完成、LLVM AOT バックエンド未実装
3. **値依存型**（RFC-011）：コンパイル時評価エンジンが未実装
4. **設定システム**（RFC-015）：ユーザー設定が未実装
5. **標準ライブラリテスト**：わずか 8 ユニットテスト、严重不足

#### 既知の問題の修正

| 問題 | モジュール | 説明 |
|------|------|------|
| `os.chdir` が実際のディレクトリ切替を行わない | std | ディレクトリ存在するかのチェックのみで、`std::env::set_current_dir()` を呼び出していない |
| `string.len` がバイト数を返す | std | `native_len` が `s.len()` を使用してバイト数を返すが、文字数ではない |
| `weak` モジュールがインポートできない | std | `StdModule` trait 実装が不足 |
