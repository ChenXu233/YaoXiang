## プロジェクト概要

YaoXiang は未来志向のプログラミング言語で、バイトコードコンパイル + VM 解释実行アーキテクチャを採用しています。現在 v0.7.x 段階にあり、コアコンパイラとランタイムは基本的に完成しています。

## コアの発見

### 完了した領域

1. **コンパイラ前端**：字句解析、パース、型検査がすべて完了、テストカバレッジは優秀
2. **所有権システム**：完全な借用検査器、15 件中 14 件のサブモジュール機能が完成
3. **ツールチェーン**：LSP、パッケージ管理、フォーマッターが基本的に完成
4. **ドキュメントサイト**：多言語サポート、CI/CD 自動デプロイ

### 主要なギャップ

1. **spawnモデル**（RFC-001）：DAG 依存解析器が未実装、並列制御注釈が不完全
2. **Runtime 三層アーキテクチャ**（RFC-008）：フェーズ A のみ完成、LLVM AOT バックエンドが未実装
3. **値依存型**（RFC-011）：コンパイル時評価エンジンが未実装
4. **設定システム**（RFC-015）：ユーザー级别設定が未実装
5. **標準ライブラリテスト**：ユニットテストが 8 件のみ、严重的に不足

#### 既知の問題修正

| 問題 | モジュール | 説明 |
|------|------|------|
| `os.chdir` が実際にディレクトリを切り替えない | std | ディレクトリ是否存在のみチェック、`std::env::set_current_dir()` を呼ばない |
| `string.len` がバイト数を返す | std | `native_len` が `s.len()` を使用し、文字数而非字节数を返す |
| `weak` モジュールがインポートできない | std | `StdModule` trait 実装が欠落 |
