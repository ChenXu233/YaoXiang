## プロジェクト概要

YaoXiang は未来的なプログラミング言語であり、バイトコードコンパイル + VM 解释実行アーキテクチャを採用しています。現在の v0.7.x 段階では、コアコンパイラとランタイム已基本的に完成しています。

## コア発見

### 完成済みの領域

1. **コンパイラフロントエンド**：字句解析、パース、型チェックが全て完成し、テスト覆盖は優秀
2. **所有権システム**：完全な借用検査器、15 中 14 のサブモジュール機能が完整
3. **ツールチェーン**：LSP、パッケージ管理、フォーマッターが完成
4. **ドキュメントサイト**：多言語サポート、CI/CD 自動デプロイ

### 主要な缺口

1. **並作モデル**（RFC-001）：DAG 依存解析器未実装、并发制御アノテーションが不完整
2. **Runtime 三層アーキテクチャ**（RFC-008）：段階 A のみ完成、LLVM AOT バックエンド未実装
3. **値依存型**（RFC-011）：编译期計算エンジン未実装
4. **設定システム**（RFC-015）：ユーザーレベル設定未実装
5. **標準ライブラリテスト**：わずか 8 件のユニットテスト、严重的に不足

## 次ステップの計画

### 短期（v0.8）— テストと品質

**目標**：テスト覆盖を補い、既知の問題を修正

#### テスト補填（最高優先度）

| モジュール | 現在のテスト | 目標 | 具体的なタスク |
|------|----------|------|----------|
| 標準ライブラリ | 8 | 100+ | io/math/string/list/dict/os/time/concurrent にユニットテストを追加 |
| REPL | 0 | 20+ | 評価エンジン、コマンドシステム、自动補完テストを補足 |
| lexer | 31（150+ 未コンパイル） | 180+ | tests/ ディレクトリ下の 11 のテストファイルを有効化 |
| codegen | 13 | 30+ | translator.rs のユニットテストを補足 |
| lifetime | 83 | 100+ | drop_semantics/clone/mut_check/ref_semantics/unsafe_check テストを補足 |
| interpreter | ~60 | 80+ | 境界条件和エラー経路テストを補足 |
| runtime | 22 | 30+ | facade.rs と task.rs のテストを補足 |

#### 既知の問題の修正

| 問題 | モジュール | 説明 |
|------|------|------|
| `os.chdir` が実際はディレクトリを切换しない | std | ディレクトリ是否存在のみチェックし、`std::env::set_current_dir()` を呼び出していない |
| `string.len` がバイト数を返す | std | `native_len` が `s.len()` を使用しており、バイト数を返している（文字数ではない） |
| `weak` モジュールがインポートできない | std | `StdModule` trait の実装が不足 |
| 空コードブロックが `{\n}` を出力 | formatter | `{}` を出力すべき（§6.3） |
| `single_quote` 設定が有効にならない | formatter | フィールドが存在するが読み込まれていない（§8.3） |
| パラメータリストが自动改行をサポートしない | formatter | 常に出力を単一行にする（§4.2） |

#### CTE 编译期計算エンジン（Phase 1-2）

- [ ] `CTValue` 列挙型と `EvalEnv` 構造体を定義
- [ ] `eval()` の基本経路を実装：字句型、変数、二項演算、条件、コードブロック
- [ ] 純度分析器の第一版を実装
- [ ] 型チェッカーに CTE 呼び出しポイントを挿入
- [ ] 定数畳み込み：`1 + 2 * 3` を编译期に `7` として計算
- [ ] 関数インライン評価を実装
- [ ] `//! decreases` の解析と終了検証を実装
