> **現在のバージョン**: v0.7.2
> **最終更新**: 2026-06-04
> **ステータス**: 活発に開発中

---

## プロジェクト概要

YaoXiang は未来志向のプログラミング言語であり、バイトコードコンパイル + VM インタープリタ実行アーキテクチャを採用しています。現在 v0.7.x 段階にあり、コアコンパイラとランタイムは基本的に完成しています。

## コアな発見

### 完了した領域

1. **コンパイラフロントエンド**: 字句解析、解析、型検査はすべて完了し、テストカバレッジは優秀
2. **所有権システム**: 完全な借用検査器、14/15 サブモジュールが機能完全
3. **ツールチェーン**: LSP、パッケージ管理、フォーマッターはほぼ完了
4. **ドキュメントサイト**: 多言語サポート、CI/CD 自動デプロイ

### 主要なギャップ

1. **spawn モデル**（RFC-001）: DAG 依存関係アナライザーが未実装、並行制御アノテーションが不完全
2. **ランタイム 3 層アーキテクチャ**（RFC-008）: フェーズ A のみ完了、LLVM AOT バックエンドが未実装
3. **値依存型**（RFC-011）: コンパイル時評価エンジンが未実装
4. **設定システム**（RFC-015）: ユーザーレベル設定が未実装
5. **標準ライブラリテスト**: ユニットテストわずか 8 個で深刻に不足

## 次のステップ計画

### 短期（v0.8）— テストと品質

**目標**: テストカバレッジを埋め、既知の問題を修正する

#### テスト補完（最高優先度）

| モジュール | 現在のテスト | 目標 | 具体的なタスク |
|------|----------|------|----------|
| 標準ライブラリ | 8 | 100+ | io/math/string/list/dict/os/time/concurrent にユニットテストを追加 |
| REPL | 0 | 20+ | 評価エンジン、コマンドシステム、自動補完のテストを補足 |
| lexer | 31（150+ 未コンパイル） | 180+ | tests/ ディレクトリ下の 11 個のテストファイルをアクティブ化 |
| codegen | 13 | 30+ | translator.rs のユニットテストを補足 |
| lifetime | 83 | 100+ | drop_semantics/clone/mut_check/ref_semantics/unsafe_check のテストを補足 |
| interpreter | ~60 | 80+ | 境界条件とエラーパスのテストを補足 |
| runtime | 22 | 30+ | facade.rs と task.rs のテストを補足 |

#### 既知の問題修正

| 問題 | モジュール | 説明 |
|------|------|------|
| `os.chdir` が実際にディレクトリを切り替えしていない | std | ディレクトリの存在確認のみで、`std::env::set_current_dir()` を呼び出していない |
| `string.len` がバイト数を返す | std | `native_len` は `s.len()` を使ってバイト数を返し、文字数ではない |
| `weak` モジュールがインポートできない | std | `StdModule` trait の実装が不足 |

#### CTE コンパイル時計算エンジン（Phase 1-2）

- [ ] `CTValue` enum と `EvalEnv` 構造を定義
- [ ] `eval()` の基本パスを実装: リテラル、変数、二項演算、条件、コードブロック
- [ ] 純度アナライザーの初版を実装
- [ ] 型検査器に CTE 呼び出しポイントを挿入
- [ ] 定数畳み込み: `1 + 2 * 3` をコンパイル時に `7` に計算
- [ ] 関数インライン評価を実装
- [ ] `//! decreases` の解析と停止検証を実装
