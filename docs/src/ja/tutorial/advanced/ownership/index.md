# 所有権モデル

YaoXiang はガベージコレクション（GC）もライフタイム注釈も使用しない。メモリ安全性は**5つの概念、1つのグラデーション**の上に成り立っている。

## 5つの概念、1つのグラデーション

```
覗く/その場で変更     奪う           共有所有          複製する         システムレベル
    │              │              │              │              │
   &T            Move           ref          clone()        unsafe
  &mut T         ゼロコピー     コンパイラ自動  明示的ディープコピー *T
  ゼロサイズトークン デフォルト   Rc/Arcを選択                ユーザ責任
```

## Move：デフォルトの所有権移転

YaoXiang では、**代入 = 所有権移転**。これはデフォルトの動作であり、ゼロコピーである：

```yaoxiang
p = Point(1.0, 2.0)
p2 = p              # Move! p の所有権は p2 に移転
                    # この後 p は読み取れない

# p2 を変更したい？ mut を使う
mut p3 = Point(3.0, 4.0)
shift(p3, 1.0, 1.0)    # その場で変更
```

関数の引数渡しと戻り値も Move である：

```yaoxiang
# 引数：Move で渡される
process: (p: Point) -> Point = {
    p.transform()
    p                  # Move で返す——ゼロコピー
}

# 呼び出し
p = Point(1.0, 2.0)
result = process(p)    # p は Move された
```

## &T / &mut T：借用トークン

所有権を奪わず、一時的に「覗く」（`&T`）または「その場で変更する」（`&mut T`）だけの場合、コンパイラは自動的に**ゼロサイズの借用トークン**を生成する：

```yaoxiang
data = [1, 2, 3, 4, 5]

# コンパイラが自動的に &List(Int) トークンを渡す——所有権を奪わない
println(data.len())    # 5
println(data)          # ✅ data はまだ存在する、覗いただけ
```

`&T` と `&mut T` は**ゼロサイズ型**である——コンパイル時に存在し、ランタイムには消える。手動で `&` を書く必要はなく、コンパイラが使用状況に応じて自動的に判断する：

```yaoxiang
# 読み取り専用アクセス → 自動的に &T
print: (point: &Point) -> Void = {
    println("(${point.x}, ${point.y})")
}

# 可変変更 → 自動的に &mut T
shift: (point: &mut Point, dx: Float, dy: Float) -> Void = {
    point.x = point.x + dx
    point.y = point.y + dy
}

mut p = Point(1.0, 2.0)
print(p)                # &Point を渡す
shift(p, 1.0, 1.0)      # &mut Point を渡す
```

**重要な違い**：`&T` はコピー可能（共有読み取り専用）、`&mut T` はコピー不可（排他的可変）。これは特別なルールではなく、2つの型属性である。

## ref：スコープをまたいだ共有

複数の場所で同時に値を**保持**する必要がある場合、`ref` を使う：

```yaoxiang
data = [1, 2, 3, 4, 5]

# ref は共有所有を作成する
shared = ref data

# コンパイラが自動的に参照カウンタを選択する：
# - タスクをまたがない → Rc（シングルスレッド参照カウント）
# - タスクをまたぐ → Arc（アトミック参照カウント）
spawn {
    use(shared)    # タスクをまたぐ！コンパイラが自動的に Arc を使用
}

# Rc と Arc の違いを知る必要はない
# コンパイラが自動的に選択してくれる
```

## clone()：明示的ディープコピー

独立したコピーが必要な場合、`clone()` を明示的に呼び出す：

```yaoxiang
original = [1, 2, 3]
backup = original.clone()   # ディープコピー——独立したコピーを所有

# それぞれ独立
original[0] = 10
println(backup[0])    # 1——影響を受けない
```

`clone()` は明示的である——コピーしたい意図を明確に示す。一部の言語のようにデフォルトでコピーされることはない。

## ライフタイムなし

YaoXiang にはライフタイム `'a` がない。この設計選択は、重要な観察に基づいている：

> 借用衝突の問題は本質的にホーア命題検証と同値である。これを型検査器の証明パイプラインに統合して解決すれば、追加の借用検査フレームワークは不要である。

`'a` を注釈したり、ライフタイムを理解したりする必要はない——コンパイラが型検査フェーズで自動的に所有権安全性を検証する。

## GC なし

所有権モデル全体にガベージコレクションはない。すべてのメモリの解放タイミングはコンパイル時に決定される：

- **Move 後** → 元の変数は使用不可、RAII により自動解放
- **ref 後** → 参照カウントがゼロになった時点で解放
- **スコープ終了時** → スタック変数を自動解放

GC ポーズゼロ、ランタイムオーバーヘッドゼロ。

## まとめ

| 操作 | キーワード/構文 | コピー？ | 使用する場面 |
|------|------------|--------|--------|
| 所有権を奪う | デフォルト動作 | ゼロコピー | 関数の引数渡し、代入 |
| 覗く | 自動 `&T` | ゼロサイズトークン | 読み取り専用アクセス |
| その場で変更 | 自動 `&mut T` | ゼロサイズトークン | 可変変更 |
| 共有所有 | `ref` | 参照カウント | スコープ/タスクをまたぐ |
| 明示的コピー | `.clone()` | ディープコピー | 独立したコピーが必要な場合 |
| 生ポインタ | `unsafe` + `*T` | 手動 | システムレベル操作 |

**覚えておいてほしいこと**：Move がデフォルト、ref が共有、clone が例外。この3つのルールで、GC とは永遠にさよならできる。
