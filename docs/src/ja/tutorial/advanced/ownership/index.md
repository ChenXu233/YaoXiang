# 所有権モデル

YaoXiangはガベージコレクション（GC）を使用せず、ライフタイム注釈も必要ありません。そのメモリ安全性は**5つの概念、1つのグラデーション**の上に成り立っています。

## 5つの概念、1つのグラデーション

```
覗く/その場変更    持ち出す         共有保持          複製する        システムレベル
    │              │              │              │              │
   &T            Move           ref          clone()        unsafe
  &mut T         ゼロコピー     コンパイラ自動  明示的ディープコピー  *T
  ゼロサイズ令牌  デフォルト    Rc/Arcを選択               ユーザ責任
```

## Move：デフォルトの所有権移転

YaoXiangでは、**代入 = 所有権の移転**です。これはデフォルトの動作で、ゼロコピーです：

```yaoxiang
p = Point(1.0, 2.0)
p2 = p              // Move! p の所有権が p2 に移転
                    // この後 p は読み取れない

// p2 を変更したい？ mut を使う
mut p3 = Point(3.0, 4.0)
shift(p3, 1.0, 1.0)    // その場で変更
```

関数への引数渡しと戻り値も Move です：

```yaoxiang
// 引数：Move で渡す
process: (p: Point) -> Point = {
    p.transform()
    p                  // Move で返す——ゼロコピー
}

// 呼び出し
p = Point(1.0, 2.0)
result = process(p)    // p は Move された
```

## &T / &mut T：借用トークン

所有権を持ち出したくない場合、単に一時的に「覗く」（`&T`）または「その場で変更する」（`&mut T`）場合、コンパイラは自動的に**ゼロサイズの借用トークン**を生成します：

```yaoxiang
data = [1, 2, 3, 4, 5]

// コンパイラが自動的に &List(Int) トークンを渡す——所有権は持ち出さない
print(data.len())    // 5
print(data)          // ✅ data はまだある、覗いただけ
```

`&T` と `&mut T` は**ゼロサイズ型**です——コンパイル時に存在し、実行時には消えます。手動で `&` を書く必要はなく、コンパイラが利用シーンに応じて自動的に決定します：

```yaoxiang
// 読み取り専用アクセス → 自動的に &T
print: (point: &Point) -> Void = {
    print("({point.x}, {point.y})")
}

// 可変変更 → 自動的に &mut T
shift: (point: &mut Point, dx: Float, dy: Float) -> Void = {
    point.x = point.x + dx
    point.y = point.y + dy
}

mut p = Point(1.0, 2.0)
print(p)                // &Point を渡す
shift(p, 1.0, 1.0)      // &mut Point を渡す
```

**重要な違い**：`&T` はコピー可能（共有読み取り専用）、`&mut T` はコピー不可（排他的可変）。これは特別なルールではなく、**2つの型属性**です。

## ref：スコープを跨いだ共有

複数の場所で同時に値を保持する必要がある場合、`ref` を使用します：

```yaoxiang
data = [1, 2, 3, 4, 5]

// ref で共有保持を作成
shared = ref data

// コンパイラが自動的に参照カウンタを選択：
// - タスクを跨がない → Rc（シングルスレッド参照カウント）
// - タスクを跨ぐ → Arc（アトミック参照カウント）
spawn {
    use(shared)    // タスクを跨ぐ！コンパイラが自動的に Arc を使用
}

// Rc と Arc の違いを知っておく必要はない
// コンパイラが自動的に選んでくれる
```

## clone()：明示的ディープコピー

独立したコピーが必要な場合、明示的に `clone()` を呼び出します：

```yaoxiang
original = [1, 2, 3]
backup = original.clone()   // ディープコピー——独立したコピーを所有

// それぞれ独立
original[0] = 10
print(backup[0])    // 1——影響を受けない
```

`clone()` は明示的です——コピーしたいことを明確に示します。デフォルトでコピーする言語とは異なります。

## ライフタイムなし

YaoXiangにはライフタイム `'a` がありません。この設計選択は、重要な観察に基づいています：

> 借用競合の問題は本質的にホーア命題検証と等価です。型チェッカーの証明パイプラインに統一的に解決させるべきであり、追加の借用チェックフレームワークは必要ありません。

`'a` を注釈する必要はなく、ライフタイムを理解する必要もありません。コンパイラが型チェックフェーズで自動的に所有権安全性を検証します。

## GC なし

この所有権モデル全体にはガベージコレクションがありません。すべてのメモリの解放タイミングはコンパイル時に決定されます：

- **Move 後** → 元の変数は使用不可、RAII が自動的に解放
- **ref 後** → 参照カウントがゼロになった時点で解放
- **スコープ終了** → スタック変数が自動的に解放

GC ポーズゼロ、ランタイムオーバーヘッドゼロ。

## まとめ

| 操作 | キーワード/構文 | コピー？ | 使用する場面 |
|------|------------|--------|--------|
| 所有権を持ち出す | デフォルト動作 | ゼロコピー | 関数引数、代入 |
| 覗く | 自動 `&T` | ゼロサイズトークン | 読み取り専用アクセス |
| その場で変更 | 自動 `&mut T` | ゼロサイズトークン | 可変変更 |
| 共有保持 | `ref` | 参照カウント | スコープ/タスクを跨ぐ |
| 明示的コピー | `.clone()` | ディープコピー | 独立したコピーが必要 |
| 生ポインタ | `unsafe` + `*T` | 手動 | システムレベル操作 |

**覚えておいてください**：Move がデフォルト、ref が共有、clone が例外。3つのルールで GC とおさらばです。
