# 変数宣言

本章ではYaoXiangにおける変数宣言の中核構文を紹介します。もし他のプログラミング言語の経験があれば、YaoXiangの変数システムが非常に簡潔であることに気づくでしょう——すべての宣言が同じ構文モデルを共有しています。

## 統一構文モデル

YaoXiangの設計哲学は「万物皆統一」です。整数の宣言、関数の定義、型の作成のいずれであっても、すべて同じ構文を使用します：

```
name: type = value
```

これはYaoXiangの最も中核的な設計理念です。いくつかの例を見れば、この一貫性が伝わるでしょう：

```yaoxiang
// 変数宣言
x: Int = 42
name: String = "YaoXiang"

// 関数定義
add: (a: Int, b: Int) -> Int = a + b

// 型定義
Point: Type = { x: Float, y: Float }
```

言語仕様における変数宣言の形式定義は次のとおりです：

```
('mut')? Identifier (':' TypeExpr)? '=' Expr
```

分かりやすく言うと：任意の`mut`キーワード、その後ろに変数名、そして任意の`: 型`、最後に`= 値`です。この構造は言語全体に通じており、一度学べば十分です。

## 不変変数（デフォルトの動作）

YaoXiangでは、すべての変数は**デフォルトで不変**です。一度代入したら、もう変更できません。これは言語の安全設計です。

```yaoxiang
x = 10
// x = 20   // コンパイルエラー！x は不変です
```

`=`を使って宣言された変数に対して、コンパイラはスコープチェーンに沿って外側に向かって同名の変数を検索します。見つかれば代入を試み、見つからなければ現在のスコープ内に新しい不変変数を作成します。

```yaoxiang
x = 1       // 外側スコープに x がないため、新しい変数として宣言
x = 2       // 外側の x が見つかった、代入を試みる → コンパイルエラー！x は不変
```

これは直感に反するように見えるかもしれません——他の言語を学んだことがあれば、「見つかったのに、なぜ代入できないのか？」と思うでしょう。これはYaoXiangが安全性を最優先しているためです。デフォルトで不変ということは、コードのどこかの隅で変数が意図せず変更される心配がないということです。

## mut 可変変数

変数を変更する必要がある場合は、`mut`キーワードで明示的に宣言します：

```yaoxiang
mut counter = 0
counter = counter + 1   // 変更可能
counter = 100           // これも可能
```

`mut`にはいくつかの重要なルールがあります：

**ルール1**：`mut`は明示的な新規宣言であり、コンパイラは外側スコープで同名の変数を検索しません。

```yaoxiang
mut x = 10      // 現在のスコープに新しい可変変数 x を作成
mut x = 20      // コンパイルエラー！同スコープ内に x は既に宣言されている
```

**ルール2**：`mut`で宣言した変数は外側スコープの変数と同名にできません（シャドーイング禁止）。

```yaoxiang
x = 10
{
    mut x = 20   // コンパイルエラー！x は外側で既に宣言されており、シャドーイングは許可されない
}
```

**ルール3**：同じスコープ内で、各名前は一度しか宣言できません——`=`を使っても`mut`を使っても関係ありません。

```yaoxiang
x = 10
mut x = 20   // コンパイルエラー！x は既に宣言されている
```

これらのルールにより、各変数名が現在のスコープ内で一意であることが保証され、同名変数がどれを指しているのかという混乱に遭遇することは決してありません。

## 型推論 vs 明示的な型注釈

YaoXiangは Hindley-Milner（HM）型推論アルゴリズムを採用しています。コンパイラは記述された値から自動的に型を推論できるため、ほとんどの場合手動で型を書く必要はありません。

```yaoxiang
x = 42              // コンパイラが Int と推論
name = "YaoXiang"   // String と推論
pi = 3.14159        // Float と推論
is_valid = true     // Bool と推論
```

型を明示的に注釈したい場合（例えばコードの可読性を高めるため、あるいはコンパイラが推論できない場合）は、`: Type`構文を使用します：

```yaoxiang
count: Int = 100
greeting: String = "Hello"
ratio: Float = 0.618
```

両方の書き方は完全に等価です。型を省略してコード書き始め、必要になった時点で型注釈を追加することができます。これによりプロトタイプ開発が非常に高速になり、かつ最終的なコードの型安全性を損なうこともありません。

## 基本型一覧

YaoXiangには、日常プログラミングのほとんどのシナリオをカバーするいくつかの基本型が組み込まれています。

### Int（整数）

```yaoxiang
a = 42              // 十進数
b = 0o52            // 八進数（0o プレフィックス）
c = 0x2A            // 十六進数（0x プレフィックス）
d = 0b101010        // 二進数（0b プレフィックス）
e = 1_000_000       // 可読性のため数字をアンダースコアで区切れる
```

### Float（浮動小数点数）

```yaoxiang
pi = 3.14159
speed = 2.998e8         // 科学的記数法：2.998 × 10^8
tiny = 1.6e-19
```

### String（文字列）

```yaoxiang
name = "YaoXiang"
empty = ""              // 空文字列
escape = "Hello\nWorld" // エスケープ対応：\n 改行、\t タブ、\\ バックスラッシュ、\" ダブルクォート
unicode = "\u{4F60}\u{597D}"  // Unicode エスケープ
```

### Bool（ブール）

```yaoxiang
is_ready = true
is_done = false
```

ブール値は通常条件判定で使用されます：

```yaoxiang
if is_ready {
    print("処理開始")
}
```

## 変数スコープ

スコープは変数の可視範囲を決定します。YaoXiangのスコープルールは非常にシンプルです：**各`{}`ブロックが新しいスコープを作成します**。

### 基本ルール

```yaoxiang
{
    x = 10
    print(x)   // アクセス可能：x は現在のスコープ内にある
}
// print(x)    // エラー：x はスコープ外では不可視
```

内側から外側の変数にアクセスできます：

```yaoxiang
outer = "私は外側にいます"
{
    print(outer)   // 外側の outer にアクセス可能
    inner = "私は内側にいます"
}
// print(inner)    // エラー：inner はスコープ外では不可視
```

### 関数パラメータのスコープ

```yaoxiang
greet: (name: String) -> Void = {
    print("こんにちは, " + name)
    // name はこの関数本体内で可視
}
// name は関数外では不可視
```

### ブロック式

YaoXiangの`{}`ブロックも式であり、値を返すことができます：

```yaoxiang
result = {
    x = 10
    y = 20
    return x + y   // 30 を外側スコープに返す
}
// result の値は 30
// x と y はブロック外では不可視
```

ブロックの戻り値の詳細については、続く関数チャプターを参照してください。ここで覚えておくべきことは：**波括弧はスコープを作成し、内側から外側が見えるが、外側から内側は見えない**ということです。

## まとめ

YaoXiangの変数システムの中核概念を習得しました：

| 概念 | 要点 |
|------|------|
| 統一構文モデル | `name: type = value`、変数・関数・型すべてに共通 |
| デフォルトで不変 | `x = 10`の後、`x`は変更不可 |
| 可変変数 | `mut`を使って明示的に宣言 `mut x = 10` |
| シャドーイング禁止 | 同じ名前は同じスコープで一度しか宣言できない |
| 型推論 | HM アルゴリズムによる自動推論、`: Type`での明示注釈も可能 |
| スコープ | 各`{}`がスコープを作成、内から外が見える、外から内は不可視 |
