# 変数宣言

本章では、YaoXiang における変数宣言の核心的な構文を紹介します。他のプログラミング言語の経験があれば、YaoXiang の変数システムが非常にシンプルであることに気づくでしょう——すべての宣言が同じ構文モデルを共有しています。

## 統一構文モデル

YaoXiang の設計哲学は「万物は皆統一」です。整数を宣言するにしても、関数を定義するにしても、型を作成するにしても、すべて同じ構文を使用します：

```
name: type = value
```

これは YaoXiang の中核を成す設計理念です。いくつかの例を見れば、この一貫性を感じ取ることができるでしょう：

```yaoxiang
# 変数宣言
x: Int = 42
name: String = "YaoXiang"

# 関数定義
add: (a: Int, b: Int) -> Int = a + b

# 型定義
type Point = { x: Float, y: Float }
```

構文仕様における変数宣言の形式定義は次の通りです：

```
('mut')? Identifier (':' TypeExpr)? '=' Expr
```

平たく言えば：オプションの `mut` キーワード、続いて変数名、その後にオプションの `: 型`、最後に `= 値` という構成です。この構造は言語全体を通じて一貫しており、一度学べば十分です。

## 不変変数（デフォルトの動作）

YaoXiang では、すべての変数は**デフォルトで不変**です。一度代入したら、もう変更することはできません。これは言語の安全設計です。

```yaoxiang
x = 10
# x = 20   // コンパイルエラー！x は不変です
```

`=` で宣言された変数について、コンパイラはスコープチェーンを外側に向かって同じ名前の変数を検索します。見つかれば、それに代入します。見つからなければ、現在のスコープ内に新しい不変変数を作成します。

```yaoxiang
x = 1       // 外側スコープに x がないので、新しい変数として宣言
x = 2       // 外側の x が見つかった、代入を試みる → コンパイルエラー！x は不変
```

これは直感に反するように見えるかもしれません——他の言語を学んだことがあれば、「見つかったのになぜ代入できないのか？」と思うかもしれません。これは YaoXiang が安全性を最優先しているためです：デフォルトで不変ということは、コードのどこかで変数が誤って変更される心配をする必要がないということです。

## mut 可変変数

実際に変数を変更する必要がある場合は、`mut` キーワードで明示的に宣言します：

```yaoxiang
mut counter = 0
counter = counter + 1   // 変更可能
counter = 100           // これも可能
```

`mut` にはいくつかの重要なルールがあります：

**ルール一**：`mut` は明示的な新規宣言であり、コンパイラは外側スコープで同じ名前の変数を検索しません。

```yaoxiang
mut x = 10      // 現在のスコープに新しい可変変数 x を作成
mut x = 20      // コンパイルエラー！同一スコープ内で x はすでに宣言済み
```

**ルール二**：`mut` で宣言された変数は、外側スコープの変数と同じ名前にすることはできません（シャドーイング禁止）。

```yaoxiang
x = 10
{
    mut x = 20   // コンパイルエラー！x は外側で宣言済み、シャドーイングは許可されない
}
```

**ルール三**：同一スコープ内で、各名前は一度しか宣言できません——`=` を使っても `mut` を使っても関係ありません。

```yaoxiang
x = 10
mut x = 20   // コンパイルエラー！x はすでに宣言済み
```

これらのルールにより、各変数名が現在のスコープ内で一意であることが保証され、同じ名前の変数が一体どれを指しているのかという混乱に遭遇することは決してありません。

## 型推論 vs 明示的な型注釈

YaoXiang は Hindley-Milner（HM）型推論アルゴリズムを使用します。コンパイラは記述された値から自動的に型を推論できるため、ほとんどの場合、手動で型を書く必要はありません。

```yaoxiang
x = 42              // コンパイラが Int と推論
name = "YaoXiang"   // String と推論
pi = 3.14159        // Float と推論
is_valid = true     // Bool と推論
```

型を明示的に注釈したい場合（例えばコードの可読性を高めるため、あるいはコンパイラが推論できない場合）は、`: Type` 構文を使用します：

```yaoxiang
count: Int = 100
greeting: String = "Hello"
ratio: Float = 0.618
```

この2つの書き方は完全に等価です。型を省略して書き始め、必要に応じて型注釈を追加することができます。これにより、プロトタイプ開発が非常に迅速になり、最終的なコードの型安全性を犠牲にすることもありません。

## 基本型一覧

YaoXiang には日常プログラミングのほとんどのシナリオをカバーする、いくつかの基本型が組み込まれています。

### Int（整数）

```yaoxiang
a = 42              // 十進数
b = 0o52            // 八進数（0o プレフィックス）
c = 0x2A            // 十六進数（0x プレフィックス）
d = 0b101010        // 二進数（0b プレフィックス）
e = 1_000_000       // アンダースコアで数字を区切ることができ、可読性が向上
```

### Float（浮動小数点数）

```yaoxiang
pi = 3.14159
speed = 2.998e8         // 指数表記：2.998 × 10^8
tiny = 1.6e-19
```

### String（文字列）

```yaoxiang
name = "YaoXiang"
empty = ""              // 空文字列
escape = "Hello\nWorld" // エスケープをサポート：\n 改行、\t タブ、\\ バックスラッシュ、\" ダブルクォート
unicode = "\u{4F60}\u{597D}"  // Unicode エスケープ
```

### Bool（ブール）

```yaoxiang
is_ready = true
is_done = false
```

ブール値は通常条件分岐で使用されます：

```yaoxiang
if is_ready {
    println("処理を開始")
}
```

## 変数のスコープ

スコープは変数の可視範囲を決定します。YaoXiang のスコープルールは非常にシンプルです：**各 `{}` ブロックが新しいスコープを作成します**。

### 基本ルール

```yaoxiang
{
    x = 10
    println(x)   // アクセス可能：x は現在のスコープ内にある
}
// println(x)    // エラー：x はスコープ外ではアクセス不可
```

内側からは外側の変数にアクセスできます：

```yaoxiang
outer = "私は外側にいます"
{
    println(outer)   // 外側の outer にアクセス可能
    inner = "私は内側にいます"
}
// println(inner)    // エラー：inner はスコープ外ではアクセス不可
```

### 関数パラメータのスコープ

```yaoxiang
greet: (name: String) -> Void = {
    println("こんにちは, " + name)
    // name はこの関数本体内でアクセス可能
}
// name は関数外ではアクセス不可
```

### ブロック式

YaoXiang の `{}` ブロックは式でもあり、値を返すことができます：

```yaoxiang
result = {
    x = 10
    y = 20
    return x + y   // 30 を外側スコープに返す
}
// result の値は 30
// x と y はブロック外ではアクセス不可
```

ブロックの戻り値に関する詳細については、後続の関数の章を参照してください。ここで覚えておく必要があるのは：**中括弧はスコープを作成し、内側から外側は見えるが、外側から内側は見えない**ということです。

## まとめ

YaoXiang の変数システムの核心概念を習得しました：

| 概念 | 要点 |
|------|------|
| 統一構文モデル | `name: type = value` で、変数、関数、型すべてに使用 |
| デフォルトで不変 | `x = 10` の後、`x` は変更できない |
| 可変変数 | `mut` で明示的に宣言 `mut x = 10` |
| シャドーイング禁止 | 同一スコープ内で同じ名前は一度しか宣言できない |
| 型推論 | HM アルゴリズムが自動推論、`: Type` で明示的に注釈も可能 |
| スコープ | 各 `{}` がスコープを作成、内から外は見えるが、外から内は不可 |
