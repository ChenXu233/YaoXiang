# 変数宣言

本章では YaoXiang における変数宣言の中核となる構文を紹介します。他のプログラミング言語の経験がある人的话、YaoXiang の変数システムは 매우 簡潔であることがわかるでしょう——すべての宣言は同じ構文モデルを共有しています。

## 統一構文モデル

YaoXiang の設計哲学は「万物皆統一」です。整数を宣言するにせよ、関数を定義するせよ、型を作成するにせよ、すべて同じ一套の構文を使用します：

```
name: type = value
```

これが YaoXiang の最も核となる設計思想です。いくつかの例を見ればこの一貫性が感じられるでしょう：

```yaoxiang
// 変数宣言
x: Int = 42
name: String = "YaoXiang"

// 関数定義
add: (a: Int, b: Int) -> Int = a + b

// 型定義
Point: Type = { x: Float, y: Float }
```

構文仕様における変数宣言の形式的定義は：

```
('mut')? Identifier (':' TypeExpr)? '=' Expr
```

日本語に訳すと：オプションで `mut` キーワードを1つ、その後に変数名、そしてオプションの `: 型`、最後に `= 値` です。この構造は言語全体に貫かれており、一回学べばそれで十分です。

## 変更不能変数（デフォルトの動作）

YaoXiang では、すべての変数は**デフォルトで変更不能**です。一度代入すると、もう変更できません。これは言語の安全性のための設計です。

```yaoxiang
x = 10
// x = 20   // コンパイルエラー！x は変更不能
```

`=` で宣言された変数は、コンパイラがスコープチェーンを外に向かって検索して同名の変数を探します。見つかればそれに代入し、見つからなければ現在のスコープ内に新しい変更不能変数を作成します。

```yaoxiang
x = 1       // 外側のスコープに x がないので、新しい変数として宣言
x = 2       // 外側の x を見つけ、代入を試みる → コンパイルエラー！x は変更不能
```

これは直感に反するように見えるかもしれません——他の言語を経験したことがあるなら「見つかるのにどうして代入できないの？」と思うかもしれません。これは YaoXiang が安全性を最優先にしているからです：デフォルトで変更不能とは、コードのどこかで変数が予期せず変更される心配をしなくてよいということです。

## mut 変更可能変数

変数を действительно 修改する必要がある場合は、`mut` キーワードを明示的に使用します：

```yaoxiang
mut counter = 0
counter = counter + 1   // 修改可能
counter = 100           // 也可以
```

`mut` にはいくつかの重要なルールがあります：

**ルール1**：`mut` は明示的な新規宣言であり、コンパイラは外側のスコープを検索しません。

```yaoxiang
mut x = 10      // 現在のスコープに新しい変更可能変数 x を作成
mut x = 20      // コンパイルエラー！同スコープ内で x はすでに宣言済み
```

**ルール2**：`mut` で宣言された変数は、外側のスコープの変数と同名を付けることはできません（シャドーイング禁止）。

```yaoxiang
x = 10
{
    mut x = 20   // コンパイルエラー！x はすでに外側で宣言されており、シャドーイングは不允许
}
```

**ルール3**：同一スコープ内で、各名前は一度しか宣言できません——`=` でも `mut` でもありません。

```yaoxiang
x = 10
mut x = 20   // コンパイルエラー！x はすでに宣言済み
```

これらのルールにより、各変数名は現在のスコープ内で一意であることが保証されます。同名の変数が一体どれを指すのかという混乱に決して出会いません。

## 型推論 vs 明示的型注釈

YaoXiang は Hindley-Milner（HM）型推論アルゴリズムを使用します。コンパイラはあなたが書いた値から 型を自動的に推論できるため、多くの場合で手動で型を書く必要はありません。

```yaoxiang
x = 42              // コンパイラは Int と推論
name = "YaoXiang"   // String と推論
pi = 3.14159        // Float と推論
is_valid = true     // Bool と推論
```

型を明確に标注したい場合（例えばコードの可読性を向上させるため、あるいはコンパイラが推論できない場合など）は、`: Type` 構文を使用します：

```yaoxiang
count: Int = 100
greeting: String = "Hello"
ratio: Float = 0.618
```

 두 가지 방법은 완전히同等です。型を省略してコードを作成し、必要なときに型注釈を追加することができます。これによりプロトタイプ開発が非常に速くなり、最終的なコードの 型安全性を犠牲にすることもありません。

## 基本 型一覧

YaoXiang には、日常的なプログラミングのほぼすべてのシナリオをカバーするいくつかの基本 型が組み込まれています。

### Int（整数）

```yaoxiang
a = 42              // 10進数
b = 0o52            // 8進数（0o プレフィックス）
c = 0x2A            // 16進数（0x プレフィックス）
d = 0b101010        // 2進数（0b プレフィックス）
e = 1_000_000       // アンダースコアで数字を区切ることができる
```

### Float（浮動小数点数）

```yaoxiang
pi = 3.14159
speed = 2.998e8         // 科学的記数法：2.998 × 10^8
tiny = 1.6e-19
```

### String（文字列）

```yaoxiang
name = "YaoXiang"
empty = ""              // 空文字列
escape = "Hello\nWorld" // エスケープをサポート：\n 改行、\t タブ、\\ バックスラッシュ、\" ダブルクォート
unicode = "\u{4F60}\u{597D}"  // Unicode エスケープ
```

### Bool（真偽値）

```yaoxiang
is_ready = true
is_done = false
```

真偽値は通常、条件判断中使用されます：

```yaoxiang
if is_ready {
    print("処理を開始")
}
```

## 変数のスコープ

スコープは変数の可视範囲を決定します。YaoXiang のスコープ規則は非常にシンプルです：**各 `{}` ブロックが新しいスコープを作成します**。

### 基本ルール

```yaoxiang
{
    x = 10
    print(x)   // アクセス可能：x は現在のスコープ内
}
// print(x)    // エラー：x はスコープ外では見えない
```

内側は外側の変数にアクセスできます：

```yaoxiang
outer = "私は外にいる"
{
    print(outer)   // 外側の outer にアクセス可能
    inner = "私は内側にいる"
}
// print(inner)    // エラー：inner はスコープ外では見えない
```

### 関数 引数のスコープ

```yaoxiang
greet: (name: String) -> Void = {
    print("你好, " + name)
    // name はこの関数本体 내에서 見える
}
// name は関数体の外では見えない
```

### ブロック式

YaoXiang の `{}` ブロックも式であり、値を返すことができます：

```yaoxiang
result = {
    x = 10
    y = 20
    return x + y   // 上一つのスコープに 30 を返す
}
// result の値は 30
// x と y はブロック外では見えない
```

ブロックの返回值の詳細については、後続の関数章を参照してください。ここでは覚えておいてほしいこと：**波括弧はスコープを作成し、内側から外側が見えるが、外側から内側は見えない**。

## 小括

あなたはすでに YaoXiang 変数システムの核となる概念を掌握しました：

| 概念 | ポイント |
|------|----------|
| 統一構文モデル | `name: type = value`、変数、関数、 型はすべてこれを使用 |
| デフォルトで変更不能 | `x = 10` の後 `x` は変更不可 |
| 変更可能変数 | `mut` で明示的に宣言 `mut x = 10` |
| シャドーイング禁止 | 同一の名前は同一スコープ内で一度しか宣言できない |
| 型推論 | HM アルゴリズムが自動推論、`: Type` で明示标注も可能 |
| スコープ | 各 `{}` がスコープを作成し、内は外を見られるが、外は内を見られない |
