# while ループ

`for` は「既知のコレクションを走査する」場面に適しているのに対し、`while` は別の状況——**ループ回数が分からず、いつ停止すべきかだけを知っている**場合に適しています。

## 基本構文

構文仕様における `while` 文の定義：

```
while Expr Block
```

構造はとてもシンプルです：`while` の後に条件式が続き、その後にループ本体のコードブロックが続きます。条件が `true` である限り、ループ本体は繰り返し実行されます。

```yaoxiang
mut count = 1

while count <= 5 {
    println(count)
    count = count + 1
}
// 出力：1 2 3 4 5
```

`mut count` で変数を宣言している点に注意してください——`count` はループ内で変更する必要があるためです。`count = 1`（不変）と書くと、ループ本体内の `count = count + 1` でエラーになります。

## while の実行フロー

`while` ループの実行手順は以下の通りです：

1. 条件式を評価する
2. 条件が `true` なら、ループ本体を実行し、手順 1 に戻る
3. 条件が `false` なら、ループを終了し、後続のコードを実行する

条件は**各反復の開始前**に評価されます。最初から条件が `false` であれば、ループ本体は一度も実行されません：

```yaoxiang
mut n = 0
while n > 0 {
    println("この文章は決して表示されません")
    n = n - 1
}
// 条件 n > 0 は最初から false なので、ループ本体はスキップされる
```

## break：ループを途中で抜ける

ループの途中で抜け出したい場合があります——たとえば、探しているターゲットを見つけたときです：

```yaoxiang
numbers = [3, 7, 2, 9, 5]
mut found = false
mut index = 0

while index < 5 {
    if numbers[index] == 9 {
        found = true
        break      # 見つかったので、探索を続ける必要はない
    }
    index = index + 1
}

println("見つかりましたか？" + found.to_string())  # "見つかりましたか？true"
```

`break` は現在のループから直ちに抜け出し、ループ後のコードの実行を続けます。

## continue：現在の反復をスキップする

`continue` は `break` とは異なります——ループを抜けるのではなく、現在の反復の残りの部分をスキップして、次の条件評価に直接進みます：

```yaoxiang
mut n = 0
while n < 5 {
    n = n + 1
    if n == 3 {
        continue   # 3 をスキップして表示しない
    }
    println(n)
}
// 出力：1 2 4 5
```

このコードでは、`n` が 3 のとき、`continue` が `println(n)` をスキップし、`while n < 5` の条件評価に直接戻ります。

## 永久ループの回避

`while` を使うときには特に注意が必要です——ループ条件が最終的に `false` になることを保証してください。そうしないと、プログラムは無限にハングアップします：

```yaoxiang
# 危険！永久ループ——条件が永遠に true
# mut x = 1
# while x > 0 {
#     x = x + 1     # x はどんどん大きくなり、決して <= 0 にならない
# }

# 正しい例——明確な終了条件がある
mut x = 1
while x <= 5 {
    println(x)
    x = x + 1        # x は徐々に大きくなり、最終的に x > 5 でループが終了する
}
```

## while を使った入力の読み取り

`while` の典型的な使用例は、長さが不定な入力を処理することです——ユーザーが何回入力するかが分からず、「入力が空なら停止する」ということだけを知っています：

```yaoxiang
# 擬似コードの例——while の典型的な用法を示す
# read_line は空行を読むと空文字列を返す
mut line = read_line()
while line != "" {
    process(line)
    line = read_line()
}
```

この「条件をチェック → データを処理 → 条件を更新」というパターンは、`while` の中心的な使用パラダイムです。

## まとめ

| ポイント | 説明 |
|------|------|
| 適用場面 | ループ回数が分からず、終了条件だけを知っている |
| 構文 | `while 条件 { ... }` |
| 実行フロー | 条件を先に評価し、それからループ本体を実行する |
| `break` | ループを直ちに抜ける |
| `continue` | 現在の反復をスキップし、条件評価に戻る |
| 注意事項 | 条件が最終的に `false` になることを保証し、永久ループを避ける |
