# while ループ

`for` は「既知の集合を反復処理する」場面に適しているのに対し、`while` は別の状況に適しています——**繰り返し回数が分からず、いつ停止すべきかだけを知っている**場合です。

## 基本構文

構文仕様における `while` 文の定義：

```
while Expr Block
```

構造は単純です：`while` の後に条件式が続き、その後にループ本体のコードブロックが続きます。条件が `true` である限り、ループ本体は実行され続けます。

```yaoxiang
mut count = 1

while count <= 5 {
    print(count)
    count = count + 1
}
// 出力：1 2 3 4 5
```

`mut count` で変数を宣言している点に注目してください——`count` はループ内で変更する必要があるためです。もし `count = 1`（不変）と書いた場合、ループ本体内の `count = count + 1` はエラーになります。

## while の実行フロー

`while` ループの実行ステップは以下の通りです：

1. 条件式を評価する
2. 条件が `true` なら、ループ本体を実行し、ステップ 1 に戻る
3. 条件が `false` なら、ループを終了し、後続のコードに進む

条件は**各反復の開始前に**チェックされます。最初から条件が `false` の場合、ループ本体は一度も実行されません：

```yaoxiang
mut n = 0
while n > 0 {
    print("この文は決して表示されない")
    n = n - 1
}
// 条件 n > 0 は最初から false なので、ループ本体はスキップされる
```

## break：ループを途中で抜け出す

ループの途中で抜け出す必要がある場合があります——例えば、検索対象を見つけた場合です：

```yaoxiang
numbers = [3, 7, 2, 9, 5]
mut found = false
mut index = 0

while index < 5 {
    if numbers[index] == 9 {
        found = true
        break      // 見つかったので、これ以上探す必要はない
    }
    index = index + 1
}

print("見つかりましたか？" + found.to_string())  // "見つかりましたか？true"
```

`break` は、プログラムに現在のループを直ちに抜け出させ、ループの後続のコードの実行を続けさせます。

## continue：現在の反復をスキップする

`continue` は `break` とは異なります——ループを抜け出すのではなく、現在の反復の残りの部分をスキップして、次の条件チェックに直接進みます：

```yaoxiang
mut n = 0
while n < 5 {
    n = n + 1
    if n == 3 {
        continue   // 3 をスキップして表示しない
    }
    print(n)
}
// 出力：1 2 4 5
```

このコードでは、`n` が 3 に等しい時、`continue` が `println(n)` をスキップし、`while n < 5` の条件チェックに直接戻ります。

## 無限ループを避ける

`while` を使用する際には特に注意が必要です——ループ条件が最終的に `false` になることを保証してください。そうしないと、プログラムが永久に固まってしまいます：

```yaoxiang
// 危険！無限ループ——条件が永遠に true になる
// mut x = 1
// while x > 0 {
//     x = x + 1     // x は大きくなり続け、決して <= 0 にならない
// }

// 正しい例——明確な終了条件がある
mut x = 1
while x <= 5 {
    print(x)
    x = x + 1        // x は徐々に増大し、最終的に x > 5 になるとループが終了する
}
```

## while を使って入力を読み取る

`while` の古典的な使用場面は、長さが不定の入力を処理することです——ユーザーが何回入力するかが分からず、「入力が空になったら停止する」ことだけが分かっています：

```yaoxiang
// 擬似コードの例——while の典型的な使用方法を示す
// read_line は空行を読み込んだ時に空文字列を返す
mut line = read_line()
while line != "" {
    process(line)
    line = read_line()
}
```

この「条件チェック → データ処理 → 条件更新」というパターンは、`while` の中心的な使用パラダイムです。

## まとめ

| ポイント | 説明 |
|------|------|
| 適用場面 | 繰り返し回数が分からず、終了条件のみが分かっている |
| 構文 | `while 条件 { ... }` |
| 実行フロー | 先に条件をチェックし、それからループ本体を実行する |
| `break` | ループを直ちに抜け出す |
| `continue` | 現在の反復をスキップして条件チェックに戻る |
| 注意事項 | 条件が最終的に `false` になることを保証し、無限ループを避ける |
