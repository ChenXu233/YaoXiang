# for ループ

一連の要素に対して同じ処理を行いたい場合、`for` ループがまさにそのための道具です。YaoXiang の `for` ループは簡潔で正確に設計されています。

## 基本構文

言語仕様における `for` 文の形式定義：

```
for 'mut'? Identifier 'in' Expr Block
```

日本語に訳すと：`for` で始まり、オプションの `mut`、続いてループ変数名、`in` と走査対象の式、最後にループ本体のコードブロックとなります。

## 数値範囲の走査

最も一般的な使い方は、`..` で範囲を作成し、それを `for` で走査することです：

```yaoxiang
# 0 から 4 まで（5 は含まない）
for i in 0..5 {
    println(i)
}
# 出力：0 1 2 3 4
```

`0..5` は 0 から始まり（含める）5 で終わる（含めない）範囲を表します。これは計算機科学の分野で広く使われている慣習——半開区間です。

変数を使って範囲の始点と終点を定義することもできます：

```yaoxiang
start = 10
end = 15
for n in start..end {
    println(n)
}
# 出力：10 11 12 13 14
```

## リストの走査

`for` は数値範囲だけでなく、リストや配列などのコレクションも直接走査できます：

```yaoxiang
colors = ["赤", "橙", "黄", "緑", "青"]

for color in colors {
    println("現在の色: " + color)
}
# 出力：
# 現在の色: 赤
# 現在の色: 橙
# ... 以下同様
```

## for の独特なセマンティクス：各反復で新しい値を束縛

YaoXiang の `for` ループには他の言語と異なる設計があります：**各反復では同じ変数を変更するのではなく、新しい束縛を作成します**。

表で理解しましょう：

| 反復 | 起きること |
|------|-----------|
| 1 回目 | 新しい束縛 `i = 0` を作成し、ループ本体を実行、その後束縛を破棄 |
| 2 回目 | 新しい束縛 `i = 1`（まったく新しい束縛）を作成し、ループ本体を実行、その後破棄 |
| 3 回目 | 新しい束縛 `i = 2` を作成し、ループ本体を実行、その後破棄 |
| ... | ... |
| ループ終了 | 範囲を使い果たしてループを終了 |

つまり、各反復のループ変数は独立した新しい値となります。これは安全性にとって非常に有効です——ループ変数が誤って変更される心配がありません：

```yaoxiang
for i in 1..5 {
    # i = i + 1   # エラー：デフォルトでは不変で、i を変更できない
    println(i)
}
```

## for mut：変更が必要な場合は明示的に宣言

ループ本体内でループ変数を変更する必要がある場合（例えばアキュムレータとして使う場合）、`for mut` を使います：

```yaoxiang
# for mut を使うとループ本体内で束縛を変更できる
for mut i in 0..5 {
    i = i * 2
    println(i)
}
# 出力：0 2 4 6 8
```

注意：`for mut` を使っても、各反復は依然として新しい束縛です。`for mut` は新しい束縛自体を可変にするだけで、前の反復の変更が次の反復に引き継がれるわけではありません。

```yaoxiang
for mut i in 1..5 {
    i = i + 100
    println(i)        # 毎回 101, 102, 103, 104 が出力される
}
# 各反復で i は範囲の値から新たに始まり、前回の変更は次回に影響しない
```

## ループ変数は外側の変数をシャドウイングできない

YaoXiang では変数のシャドウイングが禁止されています。`for` のループ変数は外側のスコープの変数と同じ名前にできません：

```yaoxiang
# 間違った例
i = 10
# for i in 1..5 {     # コンパイルエラー！i はすでに外側で宣言されている
#     println(i)
# }

# 正しい書き方——別の名前を使う
i = 10
for j in 1..5 {
    println(j)
}
```

このルールにより、「このコード中の変数はいったいどれを指しているのか」と混乱することがなくなります。

## 他の言語との比較

| 言語 | for ループ変数のセマンティクス |
|------|-----------------|
| YaoXiang | 各反復で新しい値を束縛 |
| Rust | 同じ変数を変更（mut が必要） |
| Python | 同じ変数を変更 |
| C/C++ | 同じ変数を変更 |

YaoXiang の設計は人間の直感により近いものです——「集合の各要素に対して何かを行う」——各要素は独立した存在です。

## まとめ

| 要点 | 説明 |
|------|------|
| 範囲の走査 | `for i in 0..5`、半開区間 |
| コレクションの走査 | `for item in list`、要素を一つずつ取得 |
| 束縛セマンティクス | 各反復で新しい束縛を作成、同じ変数の変更ではない |
| デフォルトは不変 | ループ変数は変更不可、誤操作を防止 |
| `for mut` | 変更が必要な場合は明示的に宣言 |
| シャドウイング禁止 | ループ変数は外側の変数と同じ名前にできない |
