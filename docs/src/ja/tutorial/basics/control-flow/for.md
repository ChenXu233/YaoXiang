# for ループ

一連の要素に対して同じことをひとつずつ行いたい場合、`for` ループがまさにそのための道具です。YaoXiang の `for` ループは、簡潔かつ精密に設計されています。

## 基本構文

言語仕様における `for` 文の形式定義：

```
for 'mut'? Identifier 'in' Expr Block
```

これは次のような意味です：`for` で始まり、オプションの `mut` をひとつ、それに続いてループ変数名、`in`、走査対象の式、そして最後にループ本体のブロックが続きます。

## 数値範囲の走査

最も一般的な使い方は、`..` を使って範囲を作成し、それを `for` で走査することです：

```yaoxiang
// 0 から 4 まで（5 は含まない）
for i in 0..5 {
    print(i)
}
// 出力：0 1 2 3 4
```

`0..5` は、0 から始まり（含む）5 で終わる（含まない）範囲を表します。これはコンピュータサイエンスの一般的な慣習——**左閉右開区間**に従ったものです。

範囲の始点と終点に変数を使うこともできます：

```yaoxiang
start = 10
end = 15
for n in start..end {
    print(n)
}
// 出力：10 11 12 13 14
```

## リストの走査

`for` は数値範囲だけでなく、リストや配列などのコレクションを直接走査することもできます：

```yaoxiang
colors = ["红", "橙", "黄", "绿", "蓝"]

for color in colors {
    print("当前颜色: " + color)
}
// 出力：
// 当前颜色: 红
// 当前颜色: 橙
// ... 以下同様に続く
```

## for の独特なセマンティクス：反復ごとに新しい値をバインドする

YaoXiang の `for` ループには、他の言語とは異なる設計があります：**反復ごとに同じ変数を変更するのではなく、新しいバインドを作成する**のです。

表で整理すると次のようになります：

| 反復回数 | 起きること |
|------|-----------|
| 1 回目 | 新しいバインド `i = 0` を作成し、ループ本体を実行し、その後バインドを破棄 |
| 2 回目 | 新しいバインド `i = 1`（まったく新しいバインド）を作成し、ループ本体を実行し、その後破棄 |
| 3 回目 | 新しいバインド `i = 2` を作成し、ループ本体を実行し、その後破棄 |
| ... | ... |
| ループ終了 | 範囲が尽き、ループが終了する |

つまり、各反復のループ変数は独立した新しい値だということです。これは安全性に大いに役立ちます——ループ変数が誤って変更される心配がありません：

```yaoxiang
for i in 1..5 {
    // i = i + 1   // エラー：デフォルトは不変なので、 i を変更できない
    print(i)
}
```

## for mut：変更が必要な場合は明示的に宣言する

ループ本体内でループ変数を変更する必要がある場合（たとえばアキュムレータとして使う場合）は、`for mut` を使用します：

```yaoxiang
// for mut を使えば、ループ本体内でバインドの変更が許可される
for mut i in 0..5 {
    i = i * 2
    print(i)
}
// 出力：0 2 4 6 8
```

注意：`for mut` を使ったとしても、各反復は依然として新しいバインドです。`for mut` はあくまで新しいバインド自体を可変にするだけであり、前の反復での変更が次の反復に引き継がれることはありません。

```yaoxiang
for mut i in 1..5 {
    i = i + 100
    print(i)        // 毎回 101, 102, 103, 104 と表示される
}
// 各反復で i は範囲の値から新たに始まるため、前の変更は次には影響しない
```

## ループ変数は外側の変数をシャドウイングできない

YaoXiang では変数のシャドウイングを禁止しています。`for` のループ変数は、外側のスコープにある変数と同じ名前にすることはできません：

```yaoxiang
// 誤った例
i = 10
// for i in 1..5 {     // コンパイルエラー！ i はすでに外側で宣言されている
//     print(i)
// }

// 正しい書き方——別の名前を使う
i = 10
for j in 1..5 {
    print(j)
}
```

このルールによって、「今このコードで参照している変数はどれなのか」という混乱が起きることはありません。

## 他の言語との比較

| 言語 | for ループ変数のセマンティクス |
|------|-----------------|
| YaoXiang | 反復ごとに新しい値をバインドする |
| Rust | 同じ変数を変更する（mut が必要） |
| Python | 同じ変数を変更する |
| C/C++ | 同じ変数を変更する |

YaoXiang の設計は人間の直感——「コレクションの各要素に対して何かを行う」——により近いものです。各要素は独立した存在です。

## まとめ

| 要点 | 説明 |
|------|------|
| 範囲の走査 | `for i in 0..5`、左閉右開 |
| コレクションの走査 | `for item in list`、要素をひとつずつ取り出す |
| バインドのセマンティクス | 同じ変数を変更するのではなく、反復ごとに新しいバインドを作成する |
| デフォルトで不変 | ループ変数は変更できず、偶発的な変更を防止する |
| `for mut` | 変更が必要な場合は明示的に宣言する |
| シャドウイング禁止 | ループ変数は外側の変数と同じ名前にできない |
