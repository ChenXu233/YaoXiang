# match の基礎

`match` は YaoXiang において最も強力な制御フロー構造です。値の形に基づいて、異なる処理パスを選択できます。他の言語の `switch` を使ったことがあるなら、`match` が全面的にアップグレードされた版であることがわかるでしょう。

## 基本構文

構文仕様における `match` 式の定義：

```
match Expr { MatchArm+ }
MatchArm : Pattern ('|' Pattern)* ('if' Expr)? '=>' Expr ','
```

分解すると：

- `match` の後に照合する値を記述
- `{}` 内に1つ以上の**マッチアーム**（MatchArm）を含む
- 各マッチアーム：**パターン**の後に `=>`、次に**結果式**
- 各アームはカンマで終わる

最もシンプルな例：

```yaoxiang
number = 2

text = match number {
    0 => "零",
    1 => "一",
    2 => "二",
}
print(text)  // "二"
```

## match は式

`if` 一样，`match` 也是**表达式**——它计算出一个值。所有匹配臂的返回值类型必须一致：

```yaoxiang
score = 85

grade = match score {
    90..100 => "A",    // 范围模式（进阶内容）
    80..89 => "B",
    70..79 => "C",
    60..69 => "D",
    _ => "F",          // 通配符：匹配所有剩余情况
}
print(grade)  // "B"
```

> **注意**：範囲パターン `90..100` などは高度な内容に属し、[パターン照合の高度な用法](../pattern-matching.md) で詳しく説明します。この章では基本的なパターンに焦点を当てます。

## 基本パターン

### リテラルパターン

具体的な値で照合します：

```yaoxiang
response = 404

message = match response {
    200 => "OK",
    301 => "Moved",
    404 => "Not Found",
    500 => "Server Error",
    _ => "Unknown",
}
print(message)  // "Not Found"
```

### 識別子パターン

変数名で照合した値をキャプチャします：

```yaoxiang
result: Result(Int, String) = ok(42)

description = match result {
    ok(value) => "成功，值是: " + value.to_string(),
    err(error) => "失败，原因: " + error,
}
print(description)  // "成功，值是: 42"
```

`ok(value)` の `value` は識別子パターンです——`ok` に包まれた実際の値をキャプチャし、`=>` の後の式で使用できます。

### ワイルドカードパターン

`_` はワイルドカードで、**任意の値**にマッチします。通常最後に配置してフォールバックとして使用します：

```yaoxiang
command = "exit"

action = match command {
    "start" => "启动服务",
    "stop" => "停止服务",
    "restart" => "重启服务",
    _ => "未知指令: " + command,
}
print(action)  // "未知指令: exit"
```

## 照合は包括的でなければならない

YaoXiang の `match` は、すべての可能性のあるケースをカバーすることを要求します——コンパイラが一部の値を漏らしているを発見した場合、直にエラーを報告します。これは `match` の安全性の表れです。

```yaoxiang
// このコードはコンパイルに失敗します
// value = true
// result = match value {
//     true => "是",
//     // false 分岐が不足——コンパイルエラー！
// }

// 正しい——_ を使用してフォールバック
value = true
result = match value {
    true => "是",
    _ => "否",      // _ で false も処理されることを保証
}
```

限られた種類のケースしかないことが明らかな場合（例えば列挙型を照合する場合）、コンパイラは各バリアントがカバーされているかをチェック帮助你。这是防止漏写分支bug的利器。

## 複数パターンの組み合わせ

1つのマッチアームは `|` で区切って複数のパターンにマッチできます：

```yaoxiang
day = "sunday"

type = match day {
    "monday" | "tuesday" | "wednesday" | "thursday" | "friday" => "工作日",
    "saturday" | "sunday" => "休息日",
    _ => "无效",
}
print(type)  // "休息日"
```

## マッチアームは順序実行される

`match` は最初のアームから照合を開始し、**最初にマッチに成功した分支が生效**し、后续不会被执行：

```yaoxiang
number = 5

result = match number {
    _ => "其他",     // ワイルドカードはすべてにマッチ、ここではマッチする
    5 => "五",       // 決して実行されない——上で既にマッチしている
}
print(result)  // "其他"
```

この特性から、**ワイルドカード `_` を最後に配置する**是个好习惯。

## 小まとめ

| 要点 | 説明 |
|------|------|
| 構文 | `match 値 { パターン => 式, ... }` |
| 式 | `match` は値を計算し、すべての分岐の型は一致する必要がある |
| リテラルパターン | 具体的な値との完全照合：`200 => "OK"` |
| 識別子パターン | 値をキャプチャして変数に：`ok(value) => ...` |
| ワイルドカード `_` | 任意の値にマッチ、フォールバック用 |
| 包括性 | すべての可能性をカバーする必要があり、コンパイラがチェックする |
| 複数パターン | `パターン1 \| パターン2 => 式` |
| 順序実行 | 上から下に、最初のマッチした分支が生效 |

> **次のステップ**：この記事は `match` の基本用法をカバーしました。より高度なパターン（ネストされたパターン、ガード式、構造体のデストラクトなど）は [パターン照合の高度な用法](../pattern-matching.md) を参照してください。
