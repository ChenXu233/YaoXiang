# 警告コード

本文書は YaoXiang コンパイラが生成する可能性のある警告コードについて説明します。警告はコンパイルを阻止しませんが、コードに潜在的な問題があることを示している場合があります。

## 設定

`yaoxiang.toml` で警告の動作を設定できます：

```toml
[lint]
# 死コード警告レベル：off | warn | deny
dead-code = "warn"
```

- `off`：警告を無効化
- `warn`：警告を表示（デフォルト）
- `deny`：警告をエラーとして扱う

## 警告リスト

### W1001: 未使用のエクスポート関数

**原因**：エクスポートされた関数がどのコードからも呼び出されていません。

**例**：
```yaoxiang
pub fn dead_function() { }  // W1001: 未使用のエクスポート関数

fn main() {
    // dead_function が呼び出されたことがない
}
```

**提案**：
- 関数を外部から使用する必要がある場合は、`pub` 修飾子を削除する
- 関数を保持する必要があるが今は使用していない場合は、設定で `dead-code = "off"` を設定する

---

### W1002: 未使用のエクスポート型

**原因**：エクスポートされた型（type エイリアスまたはカスタム型）が使用されていません。

**例**：
```yaoxiang
pub type DeadType = Int  // W1002: 未使用のエクスポート型

fn main() {
    let x: Int = 42;
}
```

**提案**：
- 不要な `pub` 修飾子を削除する
- 型をエクスポートする必要があるが今は使用していない場合は、この警告を無視する

---

### W1003: 未使用のインポート

**原因**：`use` 文でインポートされたモジュールまたはシンボルが使用されていません。

**例**：
```yaoxiang
use std.json  // W1003: 未使用のインポート

fn main() {
    // json モジュールが使用されたことがない
}
```

**提案**：
- コードの整洁を保つために、未使用のインポートを削除する
- インポートを保持する必要がある場合（副作用のためなど）は、`use std.json.*` を使用するか、コメントで理由を追加することを検討する

---

### W1004: 未使用のエクスポート変数

**原因**：`pub let` でエクスポートされた変数が読み取られたことがありません。

**例**：
```yaoxiang
pub let dead_var = 42  // W1004: 未使用のエクスポート変数

fn main() {
    // dead_var が読み取られたことがない
}
```

**提案**：
- 不要な `pub` 修飾子を削除する
- 変数をエクスポートする必要があるが今は使用していない場合は、この警告を無視する

---

### W1005: 未使用のエクスポートメソッド

**原因**：型の上でエクスポートされたメソッドが呼び出されたことがありません。

**例**：
```yaoxiang
type Foo { value: Int }

pub fn Foo.dead_method(self) { }  // W1005: 未使用のエクスポートメソッド

fn main() {
    let foo = Foo { value: 1 };
    // dead_method が呼び出されたことがない
}
```

**提案**：
- 不要な `pub` 修飾子を削除する
- メソッドを保持する必要があるが今は使用していない場合は、この警告を無視する

---

## 警告レベルの詳細

| レベル | 効果 |
|------|------|
| `off` | この警告を完全に無効化 |
| `warn` | 警告を表示するがコンパイルを続行（デフォルト） |
| `deny` | 警告をエラーとして扱い、コンパイルを阻止 |

### 使用シナリオ

- **開発中**：`warn` レベルでコードの潜在的な問題を把握する
- **リリース前**：`deny` レベルで未使用コードがないことを確認する
- **レガシーコード**：`off` レベルで一時的に警告を無視する

---

## エラーコードとの違い

警告コードは `W` プレフィックス（例：W1001）を使用し、エラーコードは `E` プレフィックス（例：E1001）を使用します。

- **エラー (Error)**：コンパイルを阻止し、修正が必須
- **警告 (Warning)**：潜在的な問題を示唆し、修正は任意
