# 警告コード

本文書は YaoXiang コンパイラが生成する可能性のある警告コードを一覧で示しています。警告はコンパイルを阻止しませんが、コードに潜在的な問題があることを示している可能性があります。

## 設定

`yaoxiang.toml` で警告の動作を設定できます：

```toml
[lint]
# 死コード警告レベル：off | warn | deny
dead-code = "warn"
```

- `off`：警告を無効化
- `warn`：警告を表示（デフォルト）
- `deny`：警告をエラーとして扱う

## 警告リスト

### W1001: 未使用のエクスポート関数

**原因**：エクスポートされた関数がどのコードからも呼び出されていない。

**例**：
```yaoxiang
pub fn dead_function() { }  // W1001: 未使用のエクスポート関数

fn main() {
    // dead_function が呼び出されていない
}
```

**提案**：
- 関数が外部使用を必要としない場合、`pub` 修飾子を削除
- 関数を保持する必要があるが今は使用していない場合、設定で `dead-code = "off"` を指定

---

### W1002: 未使用のエクスポート型

**原因**：エクスポートされた型（型の別名またはカスタム型）が使用されていない。

**例**：
```yaoxiang
pub type DeadType = Int  // W1002: 未使用のエクスポート型

fn main() {
    let x: Int = 42;
}
```

**提案**：
- 不要な `pub` 修飾子を削除
- 型をエクスポートする必要があるが今は使用していない場合、この警告を無視

---

### W1003: 未使用のインポート

**原因**：`use` 文でインポートされたモジュールまたはシンボルが使用されていない。

**例**：
```yaoxiang
use std.json  // W1003: 未使用のインポート

fn main() {
    // json モジュールが使用されていない
}
```

**提案**：
- 使用されていないインポートを削除してコードのクリーンさを維持
- インポートを保持する必要がある場合（副作用のためなど）、`use std.json.*` を使用するか、コメントで説明を追加

---

### W1004: 未使用のエクスポート変数

**原因**：`pub let` でエクスポートされた変数が読み取られたことがない。

**例**：
```yaoxiang
pub let dead_var = 42  // W1004: 未使用のエクスポート変数

fn main() {
    // dead_var が読み取られたことがない
}
```

**提案**：
- 不要な `pub` 修飾子を削除
- 変数をエクスポートする必要があるが今は使用していない場合、この警告を無視

---

### W1005: 未使用のエクスポートメソッド

**原因**：型にエクスポートされたメソッドが呼び出されたことがない。

**例**：
```yaoxiang
type Foo { value: Int }

pub fn Foo.dead_method(self) { }  // W1005: 未使用のエクスポートメソッド

fn main() {
    let foo = Foo { value: 1 };
    // dead_method が呼び出されていない
}
```

**提案**：
- 不要な `pub` 修飾子を削除
- メソッドを保持する必要があるが今は使用していない場合、この警告を無視

---

## 警告レベルの詳細

| レベル | 効果 |
|--------|------|
| `off` | この警告を完全に無効化 |
| `warn` | 警告を表示するがコンパイルは継続（デフォルト） |
| `deny` | 警告をエラーとして扱い、コンパイルを阻止 |

### 使用シナリオ

- **開発中**：`warn` レベルを使用してコード内の潜在的な問題を把握
- **リリース前**：`deny` レベルを使用して未使用のコードがないことを確認
- **レガシーコード**：`off` レベルを使用して警告を一時的に無視

---

## エラーコードとの違い

警告コードは `W` プレフィックスを使用します（例：W1001）。エラーコードは `E` プレフィックスを使用します（例：E1001）。

- **エラー (Error)**：コンパイルを阻止し、修正が必要
- **警告 (Warning)**：潜在的な問題を示唆し、修正は任意
