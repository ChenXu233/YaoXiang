# 错误处理

---

## §E1 格式化错误

**§E1.1 语法错误。** 当源代码包含语法错误时，格式化器应该：

1. 使用 `parse()` 函数进行解析
2. 如果解析有错误，直接返回错误信息
3. 不插入任何占位符

**§E1.2 配置错误。** 当配置文件格式错误时，应该返回明确的错误信息。

---

## §E2 退出码

| 退出码 | 含义 |
|--------|------|
| 0 | 成功 |
| 1 | `--check` 模式发现未格式化的文件 |
| 2 | 文件未找到或配置错误 |

---

## §E3 错误处理

**§E3.1 错误报告。** 格式化器使用 `parse()` 函数解析源代码。如果解析过程中出现错误，格式化器直接返回错误信息，不进行格式化。

**§E3.2 无占位符。** 格式化器不在错误位置插入任何占位符（如 `/* error */`）。当遇到语法错误时，格式化器直接报告错误并终止。
