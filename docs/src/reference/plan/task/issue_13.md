# Task: 实现 yaoxiang fmt 代码格式化工具

## 概述
`yaoxiang fmt` 是一个自动格式化 YaoXiang 源代码的工具，旨在统一社区代码风格，减少格式争议。它应根据语言规范（RFC-010 等）和用户配置，重新排版代码。

## 目标
- 实现 `yaoxiang fmt` 命令，接受文件或目录作为输入，格式化代码并输出（或原地修改）。
- 支持 `--check` 模式：检查代码是否已格式化，不修改文件，返回非 0 退出码。
- 支持配置文件（如 `yaoxiang.toml`）自定义格式化规则（缩进宽度、换行风格等）。
- 保留注释和原始空行的意图（智能处理）。
- 生成的代码应保持语义不变，且符合语言语法。

## 具体步骤
1. **设计格式化规则**（1 周）
   - 基于 RFC-010 的统一语法，确定缩进、空格、换行的约定。
   - 参考 Rustfmt 的设计，定义可配置选项：缩进大小、最大行宽、字段对齐等。
   - 编写设计文档，供社区讨论。

2. **实现 AST 到代码的重新打印**（2 周）
   - 利用现有的解析器获得完整的 AST（包括注释的位置信息，如果当前解析器丢弃了注释，需要增强）。
   - 设计一个“打印器”遍历 AST，根据规则输出格式化后的代码。
   - 处理特殊情况：多行字符串、内联注释、空行等。

3. **支持配置**（1 周）
   - 读取项目根目录下的 `yaoxiang.toml`，合并默认配置。
   - 将配置传递给打印器，影响缩进、换行等行为。

4. **实现 `--check` 模式**（3 天）
   - 格式化文件后比较原始内容和格式化后内容，如有差异则报错并列出差异文件。
   - 退出码为非 0 表示至少有一个文件需要格式化。

5. **测试与集成**（1 周）
   - 编写大量测试用例，覆盖所有语法特性，确保格式化前后语义一致。
   - 测试 `--check` 模式在各种情况下的正确性。
   - 确保性能足够快，至少每秒处理数千行代码。

## 验收标准
- 能够格式化包含变量、函数、类型定义、方法、控制流的完整 YaoXiang 文件。
- 格式化后的代码能通过解析和类型检查。
- 注释和空行应被合理保留，不会被意外删除或移动。
- `--check` 模式能正确检测未格式化的文件并报告。
- 支持自定义缩进宽度（2 空格/4 空格/tab）。
- 处理常见边缘情况：多行函数调用、长表达式换行、字段列表对齐等。

## 依赖
- 解析器必须保留注释和空白信息（或提供源映射，使得可以从 AST 重建代码）。如果当前解析器丢弃了这些信息，需要先扩展解析器。
- 需要了解语言的完整语法树结构。
