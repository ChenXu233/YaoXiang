# Task: 开发小型应用以验证语言特性与积累生态

## 概述

当前 YaoXiang 的前端、中端和解释器后端已经完成，编译器后端正在开发中。为了测试语言的实用性、发现隐藏的 bug、积累示例代码并吸引早期用户，需要开发一系列小型应用。这些应用应在解释器上流畅运行，不依赖编译器后端的高性能优化，同时能充分展示 YaoXiang 的统一语法、所有权模型、字段级可变性等核心设计。

## 目标
开发至少 3 个不同领域的小型应用，覆盖脚本工具、算法实现、配置解析等场景。

每个应用应包含完整的源代码、测试用例和简要文档。

通过开发过程，反馈语言设计问题，完善标准库和工具链。

## 建议应用方向（可任选或自行设计）

### 文件批量重命名工具

功能：遍历指定目录，根据正则表达式匹配文件名，进行重命名（替换、添加前缀/后缀等）。

涉及特性：文件系统操作（需 FFI 或标准库）、字符串处理、所有权管理（防止资源泄漏）。

输出：可执行脚本，支持命令行参数。

### 快速排序算法演示

功能：实现经典快速排序算法，支持对整数数组排序。

涉及特性：递归、数组/列表操作、泛型（如果支持）、所有权流动（数组元素移动）。

输出：函数库 + 测试程序，展示算法正确性。

### TOML 配置解析器原型

功能：解析简单的 TOML 配置文件（仅支持基本类型和表），映射到 YaoXiang 记录类型。

涉及特性：字符串解析、记录类型、错误处理。

输出：解析器函数 + 示例配置文件及使用代码。

### 终端井字棋游戏

功能：双人在终端对战的井字棋，支持棋盘显示、落子、胜负判断。

涉及特性：状态管理（字段可变性）、输入输出、循环控制。

输出：可运行的游戏程序。

## 验收标准

应用能按预期运行，通过所有手动测试。

代码遵循 YaoXiang 风格规范（如有）。

无内存安全错误（所有权规则自动保证）。

开发过程中发现的任何语言 bug 或标准库缺失，已记录为单独 issue。

## 依赖

解释器后端必须稳定可用。

可能需要标准库中的文件 I/O、字符串操作等模块。
