# 类型宇宙思想实现计划：去除 Fn AST 节点

> 文档版本：1.1
> 创建日期：2026-03-18
> 更新日期：2026-03-18
> 任务：将函数/类型构造器/接口等概念统一为 Binding 节点，函数判定推迟到类型检查阶段

---

## 背景与目标

根据类型宇宙思想文档，需要实现以下核心变更：

1. **去除 Fn AST 节点**：所有声明统一为 `Binding` 节点
2. **函数判定推迟到类型检查阶段**：解析器只负责语法，语义由返回值类型决定
3. **统一语法模型**：函数、类型构造器、接口都使用 `identifier: type = expression` 形式

---

## 阶段一：前端 - AST 统一为 Binding 节点

### 步骤 1.1：修改 AST 节点定义

**修改目标**：将 `StmtKind::Fn` 和 `StmtKind::MethodBind` 统一替换为 `StmtKind::Binding`

**设计决策（用户确认）**：
- **MethodBind 也是 Binding**：方法绑定与普通函数使用统一的语法模型
- **Expr::Lambda 单独保留**：Lambda 表达式是独立的语法，但函数 Binding 依赖 Lambda
- **Lambda 简化**：由于类型头已包含参数信息，Lambda 可简化为只保留 body
- **完全替换**：无需向后兼容旧语法

**关键文件**（原计划：1个 → 实际：2个）：
- `src/frontend/core/parser/ast.rs`
- `src/frontend/core/parser/tests/concurrency.rs` ⚠️

**验收标准**：
- [x] `StmtKind::Fn` 变体已移除
- [x] `StmtKind::MethodBind` 变体已移除，统一为 `StmtKind::Binding`
- [x] 新增 `StmtKind::Binding` 变体，包含：name, type_name, method_type, generic_params, type_annotation, eval, params, body, is_pub
- [ ] `Expr::Lambda` 变体保留，但简化结构（去除 Lambda 头，只保留 body）
- [ ] `Type::Fn` 变体保留（用于类型表达式中的函数类型）

**状态**：✅ 已完成（2026-03-20）

**实施记录**：
- 完成时间：2026-03-20
- 新增 `StmtKind::Binding` 变体，统一了 Fn、TypeDef、MethodBind
- 22个文件的引用点已全部更新

## 用户确认的设计决策

1. ✅ **MethodBind 也是 Binding**：方法绑定与普通函数使用统一的语法模型
2. ✅ **Expr::Lambda 单独保留**：Lambda 表达式是独立的语法，但函数 Binding 依赖 Lambda
3. ✅ **Lambda 简化**：由于类型头已包含参数信息，Lambda 可简化为只保留 body
4. ✅ **完全替换**：无需向后兼容旧语法
5. ✅ **顶层允许函数调用**：用于编译期求值，入口是 main，函数调用永远不会执行
6. ✅ **编译期求值需要解决**：这是实现值依赖类型的关键
