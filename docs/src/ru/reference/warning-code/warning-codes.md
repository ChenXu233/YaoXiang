# Коды предупреждений

В данном документе перечислены коды предупреждений, которые могут быть сгенерированы компилятором YaoXiang. Предупреждения не останавливают компиляцию, но могут указывать на потенциальные проблемы в коде.

## Конфигурация

Поведение предупреждений можно настроить через `yaoxiang.toml`:

```toml
[lint]
# Уровень предупреждения для мёртвого кода: off | warn | deny
dead-code = "warn"
```

- `off`: отключить предупреждение
- `warn`: показать предупреждение (по умолчанию)
- `deny`: считать предупреждение ошибкой

## Список предупреждений

### W1001: Неиспользуемая экспортируемая функция

**Причина**: Экспортируемая функция ни разу не была вызвана никаким кодом.

**Пример**:
```yaoxiang
pub fn dead_function() { }  // W1001: Неиспользуемая экспортируемая функция

fn main() {
    // dead_function ни разу не была вызвана
}
```

**Рекомендация**:
- Если функция не должна использоваться извне, уберите модификатор `pub`
- Если функцию нужно сохранить, но пока не использовать, можно установить `dead-code = "off"` в конфигурации

---

### W1002: Неиспользуемый экспортируемый тип

**Причина**: Экспортируемый тип (type-псевдоним или пользовательский тип) ни разу не использовался.

**Пример**:
```yaoxiang
pub type DeadType = Int  // W1002: Неиспользуемый экспортируемый тип

fn main() {
    let x: Int = 42;
}
```

**Рекомендация**:
- Уберите ненужный модификатор `pub`
- Если тип нужно экспортировать, но пока не использовать, можно игнорировать это предупреждение

---

### W1003: Неиспользуемый импорт

**Причина**: Модуль или символ, импортированный оператором `use`, ни разу не использовался.

**Пример**:
```yaoxiang
use std.json  // W1003: Неиспользуемый импорт

fn main() {
    // модуль json ни разу не использовался
}
```

**Рекомендация**:
- Удалите неиспользуемые импорты для поддержания чистоты кода
- Если импорт нужно сохранить (для побочных эффектов), рассмотрите использование `use std.json.*` или добавьте комментарий с пояснением

---

### W1004: Неиспользуемая экспортируемая переменная

**Причина**: Переменная, экспортированная с помощью `pub let`, ни разу не была прочитана.

**Пример**:
```yaoxiang
pub let dead_var = 42  // W1004: Неиспользуемая экспортируемая переменная

fn main() {
    // dead_var ни разу не была прочитана
}
```

**Рекомендация**:
- Уберите ненужный модификатор `pub`
- Если переменную нужно экспортировать, но пока не использовать, можно игнорировать это предупреждение

---

### W1005: Неиспользуемый экспортируемый метод

**Причина**: Метод, экспортированный на типе, ни разу не вызывался.

**Пример**:
```yaoxiang
type Foo { value: Int }

pub fn Foo.dead_method(self) { }  // W1005: Неиспользуемый экспортируемый метод

fn main() {
    let foo = Foo { value: 1 };
    // dead_method ни разу не вызывался
}
```

**Рекомендация**:
- Уберите ненужный модификатор `pub`
- Если метод нужно сохранить, но пока не использовать, можно игнорировать это предупреждение

---

## Подробное описание уровней предупреждений

| Уровень | Эффект |
|---------|--------|
| `off` | Полностью отключает данное предупреждение |
| `warn` | Показывает предупреждение, но продолжает компиляцию (по умолчанию) |
| `deny` | Считает предупреждение ошибкой, останавливает компиляцию |

### Сценарии использования

- **В процессе разработки**: используйте уровень `warn`, чтобы знать о потенциальных проблемах в коде
- **Перед релизом**: используйте уровень `deny`, чтобы убедиться в отсутствии неиспользуемого кода
- **Устаревший код**: используйте уровень `off`, чтобы временно игнорировать предупреждения

---

## Отличие от кодов ошибок

Коды предупреждений имеют префикс `W` (например, W1001), а коды ошибок — префикс `E` (например, E1001).

- **Ошибка (Error)**: останавливает компиляцию, требует исправления
- **Предупреждение (Warning)**: указывает на потенциальные проблемы, исправление необязательно
