# Коды предупреждений

В данном документе перечислены коды предупреждений, которые могут быть сгенерированы компилятором YaoXiang. Предупреждения не останавливают компиляцию, но могут указывать на потенциальные проблемы в коде.

## Конфигурация

Поведение предупреждений можно настроить через `yaoxiang.toml`:

```toml
[lint]
# Уровень предупреждения для мёртвого кода: off | warn | deny
dead-code = "warn"
```

- `off`: отключить предупреждение
- `warn`: показать предупреждение (по умолчанию)
- `deny`: рассматривать предупреждение как ошибку

## Список предупреждений

### W1001: Неиспользуемая экспортированная функция

**Причина**: Экспортированная функция нигде не вызывается.

**Пример**:
```yaoxiang
pub fn dead_function() { }  // W1001: Неиспользуемая экспортированная функция

fn main() {
    // dead_function нигде не вызывается
}
```

**Рекомендации**:
- Если функция не должна использоваться извне, уберите модификатор `pub`
- Если функцию нужно сохранить, но она пока не используется, можно задать `dead-code = "off` в конфигурации

---

### W1002: Неиспользуемый экспортированный тип

**Причина**: Экспортированный тип (type-псевдоним или пользовательский тип) нигде не используется.

**Пример**:
```yaoxiang
pub type DeadType = Int  // W1002: Неиспользуемый экспортированный тип

fn main() {
    let x: Int = 42;
}
```

**Рекомендации**:
- Уберите ненужный модификатор `pub`
- Если тип нужно экспортировать, но он пока не используется, игнорируйте данное предупреждение

---

### W1003: Неиспользуемый импорт

**Причина**: Модуль или символ, импортированный оператором `use`, нигде не используется.

**Пример**:
```yaoxiang
use std.json  // W1003: Неиспользуемый импорт

fn main() {
    // модуль json нигде не используется
}
```

**Рекомендации**:
- Удалите неиспользуемые импорты для поддержания чистоты кода
- Если импорт нужно сохранить (для побочных эффектов), рассмотрите использование `use std.json.*` или добавьте комментарий

---

### W1004: Неиспользуемая экспортированная переменная

**Причина**: Переменная, экспортированная с помощью `pub let`, нигде не читается.

**Пример**:
```yaoxiang
pub let dead_var = 42  // W1004: Неиспользуемая экспортированная переменная

fn main() {
    // dead_var нигде не читается
}
```

**Рекомендации**:
- Уберите ненужный модификатор `pub`
- Если переменную нужно экспортировать, но она пока не используется, игнорируйте данное предупреждение

---

### W1005: Неиспользуемый экспортированный метод

**Причина**: Метод, экспортированный на типе, нигде не вызывается.

**Пример**:
```yaoxiang
type Foo { value: Int }

pub fn Foo.dead_method(self) { }  // W1005: Неиспользуемый экспортированный метод

fn main() {
    let foo = Foo { value: 1 };
    // dead_method нигде не вызывается
}
```

**Рекомендации**:
- Уберите ненужный модификатор `pub`
- Если метод нужно сохранить, но он пока не используется, игнорируйте данное предупреждение

---

## Подробнее об уровнях предупреждений

| Уровень | Эффект |
|---------|--------|
| `off` | Полностью отключает данное предупреждение |
| `warn` | Показывает предупреждение, но продолжает компиляцию (по умолчанию) |
| `deny` | Считает предупреждение ошибкой и останавливает компиляцию |

### Сценарии использования

- **В процессе разработки**: используйте уровень `warn` для выявления потенциальных проблем в коде
- **Перед релизом**: используйте уровень `deny` для проверки отсутствия неиспользуемого кода
- **Унаследованный код**: используйте уровень `off` для временного игнорирования предупреждений

---

## Отличие от кодов ошибок

Коды предупреждений имеют префикс `W` (например, W1001), а коды ошибок — префикс `E` (например, E1001).

- **Ошибка (Error)**: останавливает компиляцию, требует исправления
- **Предупреждение (Warning)**: указывает на потенциальную проблему, исправление необязательно
