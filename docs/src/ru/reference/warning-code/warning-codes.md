# Коды предупреждений

В этом документе перечислены коды предупреждений, которые может генерировать компилятор YaoXiang. Предупреждения не останавливают компиляцию, но могут указывать на потенциальные проблемы в коде.

## Конфигурация

Поведение предупреждений можно настроить через `yaoxiang.toml`:

```toml
[lint]
# 死代码警告级别：off | warn | deny
dead-code = "warn"
```

- `off`: отключить предупреждения
- `warn`: показывать предупреждения (по умолчанию)
- `deny`: считать предупреждения ошибками

## Список предупреждений

### W1001: Неиспользуемая экспортированная функция

**Причина**: экспортированная функция никогда не вызывается ни одним участком кода.

```yaoxiang
pub dead_function: () -> Void = { }  // W1001: 未使用的导出函数

main = {
    // dead_function 从未被调用
}
```

**Рекомендации**:
- Если функция не предназначена для внешнего использования, удалите модификатор `pub`
- Если функцию нужно сохранить, но она временно не используется, можно задать `dead-code = "off"` в конфигурации

---

### W1002: Неиспользуемый экспортированный тип

**Причина**: экспортированный тип (псевдоним type или пользовательский тип) никогда не используется.

**Пример**:
```yaoxiang
DeadType: Type = Int  // W1002: 未使用的导出类型

main = {
    x = 42
}
```

**Рекомендации**:
- Если тип нужно экспортировать, но он временно не используется, проигнорируйте это предупреждение

---

### W1003: Неиспользуемый импорт

**Причина**: модуль или символы, импортированные с помощью оператора `use`, никогда не используются.

```yaoxiang
use std.json  // W1003: 未使用的导入

main = {
    // json 模块从未被使用
}
```

**Рекомендации**:
- Удалите неиспользуемые импорты, чтобы поддерживать чистоту кода
- Если импорт нужно сохранить (для побочных эффектов), рассмотрите использование `use std.json.*` или добавьте комментарий с пояснением

---

### W1004: Неиспользуемая экспортированная переменная

**Причина**: переменная, экспортированная с помощью `pub`, никогда не читается.

**Пример**:
```yaoxiang
pub dead_var = 42  // W1004: 未使用的导出变量

main = {
    // dead_var 从未被读取
}
```

**Рекомендации**:
- Удалите ненужный модификатор `pub`
- Если переменную нужно экспортировать, но она временно не используется, проигнорируйте это предупреждение

---

### W1005: Неиспользуемый экспортированный метод

**Причина**: метод, экспортированный на типе, никогда не вызывается.

**Пример**:
```yaoxiang
Foo: Type = { value: Int }

pub Foo.dead_method: (self: Foo) -> Void = { }  // W1005: 未使用的导出方法

main = {
    foo = Foo(1)
    // dead_method 从未被调用
}
```

**Рекомендации**:
- Удалите ненужный модификатор `pub`
- Если метод нужно сохранить, но он временно не используется, проигнорируйте это предупреждение

---

## Подробное описание уровней предупреждений

| Уровень | Эффект |
|------|------|
| `off` | Полностью отключает это предупреждение |
| `warn` | Показывает предупреждение, но продолжает компиляцию (по умолчанию) |
| `deny` | Считает предупреждение ошибкой и останавливает компиляцию |

### Сценарии использования

- **В процессе разработки**: используйте уровень `warn`, чтобы отслеживать потенциальные проблемы в коде
- **Перед выпуском**: используйте уровень `deny`, чтобы убедиться в отсутствии неиспользуемого кода
- **В устаревшем коде**: используйте уровень `off`, чтобы временно игнорировать предупреждения

---

## Отличие от кодов ошибок

Коды предупреждений используют префикс `W` (например, W1001), коды ошибок — префикс `E` (например, E1001).

- **Ошибка (Error)**: останавливает компиляцию, требует исправления
- **Предупреждение (Warning)**: указывает на потенциальную проблему, исправление по желанию
