# Коды предупреждений

В данном документе перечислены коды предупреждений, которые могут быть сгенерированы компилятором YaoXiang. Предупреждения не блокируют компиляцию, но могут указывать на потенциальные проблемы в коде.

## Конфигурация

Поведение предупреждений можно настроить через `yaoxiang.toml`:

```toml
[lint]
# Уровень предупреждения для мёртвого кода: off | warn | deny
dead-code = "warn"
```

- `off`：отключить предупреждение
- `warn`：показать предупреждение (по умолчанию)
- `deny`：считать предупреждение ошибкой

## Список предупреждений

### W1001: Неиспользуемая экспортируемая функция

**Причина**：Экспортируемая функция нигде не вызывается.

**Пример**：
```yaoxiang
pub fn dead_function() { }  // W1001: Неиспользуемая экспортируемая функция

fn main() {
    // dead_function нигде не вызывается
}
```

**Рекомендация**：
- Если функция не требуется использовать извне, уберите модификатор `pub`
- Если функцию нужно сохранить, но пока не использовать, можно установить `dead-code = "off"` в конфигурации

---

### W1002: Неиспользуемый экспортируемый тип

**Причина**：Экспортируемый тип (type alias или пользовательский тип) нигде не используется.

**Пример**：
```yaoxiang
pub type DeadType = Int  // W1002: Неиспользуемый экспортируемый тип

fn main() {
    let x: Int = 42;
}
```

**Рекомендация**：
- Уберите ненужный модификатор `pub`
- Если тип нужно экспортировать, но пока не использовать, можно игнорировать это предупреждение

---

### W1003: Неиспользуемый импорт

**Причина**：Модуль или символ, импортированный через `use`, нигде не используется.

**Пример**：
```yaoxiang
use std.json  // W1003: Неиспользуемый импорт

fn main() {
    // модуль json нигде не используется
}
```

**Рекомендация**：
- Удалите неиспользуемые импорты для поддержания чистоты кода
- Если импорт нужно сохранить (для побочных эффектов), рассмотрите использование `use std.json.*` или добавьте комментарий

---

### W1004: Неиспользуемая экспортируемая переменная

**Причина**：Переменная, экспортированная через `pub let`, нигде не читается.

**Пример**：
```yaoxiang
pub let dead_var = 42  // W1004: Неиспользуемая экспортируемая переменная

fn main() {
    // dead_var нигде не читается
}
```

**Рекомендация**：
- Уберите ненужный модификатор `pub`
- Если переменную нужно экспортировать, но пока не использовать, можно игнорировать это предупреждение

---

### W1005: Неиспользуемый экспортируемый метод

**Причина**：Метод, экспортированный на типе, нигде не вызывается.

**Пример**：
```yaoxiang
type Foo { value: Int }

pub fn Foo.dead_method(self) { }  // W1005: Неиспользуемый экспортируемый метод

fn main() {
    let foo = Foo { value: 1 };
    // dead_method нигде не вызывается
}
```

**Рекомендация**：
- Уберите ненужный модификатор `pub`
- Если метод нужно сохранить, но пока не использовать, можно игнорировать это предупреждение

---

## Подробнее об уровнях предупреждений

| Уровень | Эффект |
|---------|--------|
| `off` | Полностью отключить данное предупреждение |
| `warn` | Показать предупреждение, но продолжить компиляцию (по умолчанию) |
| `deny` | Считать предупреждение ошибкой, заблокировать компиляцию |

### Сценарии использования

- **В процессе разработки**：используйте уровень `warn`, чтобы видеть потенциальные проблемы в коде
- **Перед релизом**：используйте уровень `deny`, чтобы убедиться в отсутствии неиспользуемого кода
- **Унаследованный код**：используйте уровень `off`, чтобы временно игнорировать предупреждения

---

## Отличие от кодов ошибок

Коды предупреждений имеют префикс `W` (например, W1001), а коды ошибок имеют префикс `E` (например, E1001).

- **Ошибка (Error)**：блокирует компиляцию, требует исправления
- **Предупреждение (Warning)**：указывает на потенциальные проблемы, можно игнорировать
