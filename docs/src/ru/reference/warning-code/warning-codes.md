# Коды предупреждений

В данном документе перечислены коды предупреждений, которые могут быть сгенерированы компилятором YaoXiang. Предупреждения не препятствуют компиляции, но могут указывать на потенциальные проблемы в коде.

## Конфигурация

Поведение предупреждений можно настроить через `yaoxiang.toml`:

```toml
[lint]
# Уровень предупреждения для мёртвого кода: off | warn | deny
dead-code = "warn"
```

- `off`: отключить предупреждение
- `warn`: показать предупреждение (по умолчанию)
- `deny`: считать предупреждение ошибкой

## Список предупреждений

### W1001: Неиспользуемая экспортированная функция

**Причина**: Экспортированная функция нигде не вызывается.

```yaoxiang
pub dead_function: () -> Void = { }  // W1001: неиспользуемая экспортированная функция

main = {
    // dead_function нигде не вызывается
}
```

**Рекомендации**:
- Если функция не должна использоваться извне, уберите модификатор `pub`
- Если функцию нужно сохранить, но она пока не используется, можно отключить предупреждение в конфигурации: `dead-code = "off"`

---

### W1002: Неиспользуемый экспортированный тип

**Причина**: Экспортированный тип (псевдоним типа или пользовательский тип) нигде не используется.

**Пример**:
```yaoxiang
DeadType: Type = Int  // W1002: неиспользуемый экспортированный тип

main = {
    x = 42
}
```

**Рекомендации**:
- Если тип нужно экспортировать, но он пока не используется, игнорируйте это предупреждение

---

### W1003: Неиспользуемый импорт

**Причина**: Модуль или символ, импортированный через `use`, нигде не используется.

```yaoxiang
use std.json  // W1003: неиспользуемый импорт

main = {
    // модуль json нигде не используется
}
```

**Рекомендации**:
- Удалите неиспользуемые импорты для поддержания чистоты кода
- Если импорт нужно сохранить (для побочных эффектов), рассмотрите использование `use std.json.*` или добавьте комментарий

---

### W1004: Неиспользуемая экспортированная переменная

**Причина**: Переменная, экспортированная с помощью `pub`, нигде не читается.

**Пример**:
```yaoxiang
pub dead_var = 42  // W1004: неиспользуемая экспортированная переменная

main = {
    // dead_var нигде не читается
}
```

**Рекомендации**:
- Уберите ненужный модификатор `pub`
- Если переменную нужно экспортировать, но она пока не используется, игнорируйте это предупреждение

---

### W1005: Неиспользуемый экспортированный метод

**Причина**: Метод, экспортированный на типе, нигде не вызывается.

**Пример**:
```yaoxiang
Foo: Type = { value: Int }

pub Foo.dead_method: (self: Foo) -> Void = { }  // W1005: неиспользуемый экспортированный метод

main = {
    foo = Foo(1)
    // dead_method нигде не вызывается
}
```

**Рекомендации**:
- Уберите ненужный модификатор `pub`
- Если метод нужно сохранить, но он пока не используется, игнорируйте это предупреждение

---

## Подробнее об уровнях предупреждений

| Уровень | Эффект |
|---------|--------|
| `off` | Полностью отключает данное предупреждение |
| `warn` | Показывает предупреждение, но продолжает компиляцию (по умолчанию) |
| `deny` | Считает предупреждение ошибкой, останавливает компиляцию |

### Сценарии использования

- **Во время разработки**: используйте уровень `warn`, чтобы знать о потенциальных проблемах в коде
- **Перед релизом**: используйте уровень `deny`, чтобы убедиться в отсутствии неиспользуемого кода
- **Устаревший код**: используйте уровень `off`, чтобы временно игнорировать предупреждения

---

## Отличие от кодов ошибок

Коды предупреждений имеют префикс `W` (например, W1001), а коды ошибок — префикс `E` (например, E1001).

- **Ошибка (Error)**: препятствует компиляции, должна быть исправлена
- **Предупреждение (Warning)**: указывает на потенциальную проблему, исправление необязательно
