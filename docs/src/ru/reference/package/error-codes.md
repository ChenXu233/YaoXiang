# Коды ошибок

В данном документе перечислены ошибки, которые может возвращать менеджер пакетов YaoXiang, а также способы их устранения.

## Список ошибок

### E0100: Проект уже существует

```
Error: Project already exists: <path>
```

**Причина**: Каталог проекта, который пытались создать, уже существует.

**Способ устранения**:
1. Выбрать другое имя проекта
2. Удалить или переместить существующий каталог

---

### E0101: Не является допустимым проектом

```
Error: Not a YaoXiang project: yaoxiang.toml not found
```

**Причина**: Текущий каталог или указанный каталог не содержит файл `yaoxiang.toml`.

**Способ устранения**:
1. Убедиться, что команды выполняются внутри каталога проекта
2. Использовать `yaoxiang init` для создания нового проекта

---

### E0102: Зависимость не существует

```
Error: Dependency not found: <name>
```

**Причина**: Предпринята попытка выполнить операцию с несуществующей зависимостью.

**Способ устранения**:
1. Проверить правильность написания имени зависимости
2. Использовать `yaoxiang list` для просмотра существующих зависимостей

---

### E0103: Зависимость уже существует

```
Error: Dependency already exists: <name>
```

**Причина**: Предпринята попытка добавить уже существующую зависимость.

**Способ устранения**:
1. Если требуется обновить версию, сначала удалить с помощью `yaoxiang rm`
2. Либо просто использовать существующую зависимость

---

### E0104: Недопустимый формат manifest

```
Error: Invalid yaoxiang.toml format: <details>
```

**Причина**: Формат файла `yaoxiang.toml` некорректен.

**Способ устранения**:
1. Проверить правильность синтаксиса TOML
2. Убедиться, что все обязательные поля присутствуют
3. Проверить наличие синтаксических ошибок (например, пропущенные кавычки, запятые и т.д.)

---

### E0105: Ошибка ввода-вывода

```
Error: IO error: <details>
```

**Причина**: Ошибка при операциях чтения или записи файлов.

**Распространенные причины**:
- Недостаточно места на диске
- Недостаточно прав доступа
- Файл занят другой программой

**Способ устранения**:
1. Проверить свободное место на диске
2. Проверить права доступа к файлам
3. Закрыть другие программы, которые могут использовать файлы

---

### E0106: Ошибка разбора TOML

```
Error: TOML parse error: <details>
```

**Причина**: Ошибка формата TOML файла.

**Способ устранения**:
1. Проверить синтаксис TOML
2. Убедиться, что специальные символы правильно экранированы

---

## Часто задаваемые вопросы

### В: Что делать, если установка зависимостей не удалась?

1. Проверить сетевое подключение
2. Убедиться в правильности имени и версии зависимости
3. Попробовать использовать `yaoxiang update` для обновления

### В: Что делать при конфликте версий?

Проверить наличие несовместимых требований к версиям зависимостей в `yaoxiang.toml`.

### В: Что делать, если каталог vendor поврежден?

Удалить каталог `vendor` и повторно выполнить `yaoxiang install`.
