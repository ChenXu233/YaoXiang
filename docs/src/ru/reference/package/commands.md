# Интерфейс командной строки

## yaoxiang init

Инициализация нового проекта YaoXiang.

### Использование

```bash
yaoxiang init <название-проекта>
```

### Параметры

| Параметр | Описание |
|------|------|
| название-проекта | Название нового проекта |

### Опции

| Опция | Описание |
|------|------|
| `--help` | Показать справочную информацию |

### Примеры

```bash
# Создание нового проекта
yaoxiang init my-project

# Результат:
# ✨ Проект создан: my-project
#   my-project/yaoxiang.toml
#   my-project/yaoxiang.lock
#   my-project/src/main.yx
#   my-project/.gitignore
```

## yaoxiang add

Добавление зависимости в проект.

### Использование

```bash
yaoxiang add <имя-пакета> [версия]
yaoxiang add <имя-пакета> --dev
```

### Параметры

| Параметр | Описание |
|------|------|
| имя-пакета | Название добавляемой зависимости |
| версия | Номер версии (необязательно, по умолчанию `*`) |

### Опции

| Опция | Описание |
|------|------|
| `--dev`, `-D` | Добавить как зависимость для разработки |

### Примеры

```bash
# Добавить последнюю версию
yaoxiang add http

# Добавить указанную версию
yaoxiang add http 1.0.0

# Добавить диапазон версий
yaoxiang add json ">=2.0.0"

# Добавить зависимость для разработки
yaoxiang add test-utils --dev
yaoxiang add benchmark -D
```

## yaoxiang rm

Удаление зависимости из проекта.

### Использование

```bash
yaoxiang rm <имя-пакета>
yaoxiang rm <имя-пакета> --dev
```

### Параметры

| Параметр | Описание |
|------|------|
| имя-пакета | Название удаляемой зависимости |

### Опции

| Опция | Описание |
|------|------|
| `--dev`, `-D` | Удалить зависимость для разработки |

### Примеры

```bash
# Удалить обычную зависимость
yaoxiang rm http

# Удалить зависимость для разработки
yaoxiang rm test-utils --dev
```

## yaoxiang install

Установка зависимостей проекта.

### Использование

```bash
yaoxiang install
```

### Описание

- Читает объявления зависимостей из `yaoxiang.toml`
- Скачивает все зависимости в каталог `vendor`
- Создаёт/обновляет `yaoxiang.lock` с зафиксированными версиями
- Обнаруживает конфликты версий зависимостей

### Примеры

```bash
# Установить все зависимости
yaoxiang install

# Пример вывода:
# 📦 Разрешение зависимостей...
#   http (1.0.0) [установлено]
#   json (2.0.0) [в кэше]
# ✅ Зависимости установлены, файл блокировки обновлён
```

## yaoxiang update

Обновление зависимостей проекта.

### Использование

```bash
yaoxiang update
yaoxiang update <имя-пакета>
```

### Параметры

| Параметр | Описание |
|------|------|
| имя-пакета | Конкретная зависимость для обновления (необязательно) |

### Описание

- Без параметра: обновить все зависимости
- С параметром: обновить только указанную зависимость

### Примеры

```bash
# Обновить все зависимости
yaoxiang update

# Пример вывода:
# 📦 Обновление зависимостей...
#   http (0 → 1.1.0)
# ✅ Обновлена 1 зависимость, файл блокировки обновлён

# Обновить одну зависимость
yaoxiang update http
```

## yaoxiang list

Вывод списка всех зависимостей проекта.

### Использование

```bash
yaoxiang list
```

### Описание

Показывает все runtime-зависимости и зависимости для разработки, а также их версии и источники.

### Примеры

```bash
# Вывести список зависимостей
yaoxiang list

# Пример вывода:
# 📦 Зависимости проекта
#
# Runtime-зависимости:
#   http        1.0.0    registry
#   json        2.0.0    registry
#
# Зависимости для разработки:
#   test-utils  0.5.0    registry
```
