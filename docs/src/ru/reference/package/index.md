# Менеджер пакетов

Встроенный в YaoXiang менеджер пакетов, обеспечивающий инициализацию проекта, управление зависимостями, блокировку версий и другие функции.

## Обзор

Менеджер пакетов YaoXiang (сокращённо YPM) использует принципы проектирования, аналогичные Cargo:

- **Декларативные зависимости**: объявление необходимых зависимостей в `yaoxiang.toml`
- **Детерминированная сборка**: блокировка версий через `yaoxiang.lock`, обеспечение воспроизводимой сборки
- **Локальный кэш**: зависимости загружаются в директорию `vendor`, поддержка автономного использования

## Быстрый старт

```bash
# 1. Создать новый проект
yaoxiang init my-project

# 2. Добавить зависимость
cd my-project
yaoxiang add http

# 3. Установить зависимости
yaoxiang install

# 4. Запустить проект
yaoxiang run src/main.yx
```

## Список команд

| Команда | Описание |
|------|------|
| [`yaoxiang init`](./commands#yaoxiang-init) | Инициализировать новый проект |
| [`yaoxiang add`](./commands#yaoxiang-add) | Добавить зависимость |
| [`yaoxiang rm`](./commands#yaoxiang-rm) | Удалить зависимость |
| [`yaoxiang install`](./commands#yaoxiang-install) | Установить зависимости |
| [`yaoxiang update`](./commands#yaoxiang-update) | Обновить зависимости |
| [`yaoxiang list`](./commands#yaoxiang-list) | Список зависимостей |

## Структура проекта

```
my-project/
├── yaoxiang.toml      # Манифест проекта (обязательно)
├── yaoxiang.lock      # Файл блокировки зависимостей (автоматически генерируется)
├── vendor/            # Директория хранения зависимостей (автоматически генерируется)
└── src/
    └── main.yx       # Точка входа
```
