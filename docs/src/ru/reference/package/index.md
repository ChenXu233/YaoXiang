# Менеджер пакетов

Встроенный в YaoXiang менеджер пакетов, предоставляющий возможности инициализации проекта, управления зависимостями, блокировки версий и других функций.

## Обзор

Менеджер пакетов YaoXiang (далее YPM) использует концепцию проектирования, аналогичную Cargo:

- **Декларативные зависимости**: объявление необходимых зависимостей в `yaoxiang.toml`
- **Детерминированная сборка**: блокировка версий через `yaoxiang.lock` для обеспечения воспроизводимости сборки
- **Локальный кэш**: зависимости загружаются в директорию `vendor`, поддержка автономного использования

## Быстрый старт

```bash
# 1. Создание нового проекта
yaoxiang init my-project

# 2. Добавление зависимости
cd my-project
yaoxiang add http

# 3. Установка зависимостей
yaoxiang install

# 4. Запуск проекта
yaoxiang run src/main.yx
```

## Список команд

| Команда | Описание |
|---------|----------|
| [`yaoxiang init`](./commands#yaoxiang-init) | Инициализация нового проекта |
| [`yaoxiang add`](./commands#yaoxiang-add) | Добавление зависимости |
| [`yaoxiang rm`](./commands#yaoxiang-rm) | Удаление зависимости |
| [`yaoxiang install`](./commands#yaoxiang-install) | Установка зависимостей |
| [`yaoxiang update`](./commands#yaoxiang-update) | Обновление зависимостей |
| [`yaoxiang list`](./commands#yaoxiang-list) | Список зависимостей |

## Структура проекта

```
my-project/
├── yaoxiang.toml      # Манифест проекта (обязательно)
├── yaoxiang.lock      # Файл блокировки зависимостей (автогенерация)
├── vendor/            # Директория хранения зависимостей (автогенерация)
└── src/
    └── main.yx       # Точка входа
```
