# Синтаксическая спецификация

В данном документе определяется синтаксическая спецификация языка программирования YaoXiang, включая лексическую структуру, синтаксические правила и приоритет операторов.

---

## Глава I: Лексическая структура

### 1.1 Исходный файл

Исходные файлы YaoXiang должны использовать кодировку UTF-8. Исходные файлы обычно имеют расширение `.yx`.

### 1.2 Классификация лексических единиц

| Категория | Описание | Пример |
|------|------|------|
| Идентификатор | Начинается с буквы или подчёркивания | `x`, `_private`, `my_var` |
| Ключевое слово | Языковые предопределённые зарезервированные слова | `Type`, `pub`, `use` |
| Литерал | Фиксированное значение | `42`, `"hello"`, `true` |
| Оператор | Знак операции | `+`, `-`, `*`, `/` |
| Разделитель | Синтаксический разделитель | `(`, `)`, `{`, `}`, `,` |

### 1.3 Ключевые слова

YaoXiang определяет минимальный набор ключевых слов:

```
pub    use    spawn
ref    mut    if     elif
else   match  while  for    return
break  continue as     in     unsafe
```

Эти ключевые слова имеют специальное значение в любом контексте и не могут использоваться в качестве идентификаторов.

### 1.4 Зарезервированные слова

«Зарезервированные слова» YaoXiang разделены на три уровня, которые распознаются парсером (parser) и проверщиком типов (type checker) на разных этапах:

#### 1.4.1 Литеральные зарезервированные слова

Идентификаторы литералов, имеющие собственные токены в парсере и не могущие использоваться как обычные идентификаторы:

| Идентификатор | Принадлежащий тип | Описание |
|--------|---------|------|
| `Type` | — | Ключевое слово меta type |
| `true` | Bool | Логическое значение «истина» |
| `false` | Bool | Логическое значение «ложь» |
| `void` | Void | Литерал Void (значение Unit). Строчный `void` — литерал значения; заглавный `Void` — имя типа (см. §1.4.3).|

#### 1.4.2 Конструкторы выражений

Следующие конструкторы распознаются парсером в контекстах pattern matching и выражений:

| Конструктор | Принадлежащий тип | Описание |
|--------|---------|------|
| `some(T)` | Option | Конструкция value variant типа Option |
| `ok(T)` | Result | Успешный variant Result |
| `err(E)` | Result | Variant ошибки Result |

#### 1.4.3 Встроенные имена типов

Следующие имена типов предварительно зарегистрированы проверщиком типов и могут использоваться в позициях типов без импорта. Парсер рассматривает их как обычные идентификаторы — **не являются зарезервированными словами, могут быть затенены (shadow) локальными привязками (не рекомендуется)**.

| Имя типа | Логическое соответствие | Описание |
|--------|---------|------|
| `Void` | ⊤ (истина/Unit) | Тип-произведение с нулём полей, имеющий ровно одного обитателя (литерал `void`, см. §1.4.1) |
| `Never` | ⊥ (ложь/пустой тип) | Тип с нулём variants, имеющий ноль обитателей. Никакое выражение не может породить значение `Never`. `Never <: T` выполняется для всех `T` (принцип взрыва).|
| `Int` | — | Знаковое целое число |
| `Float` | — | Число с плавающей точкой |
| `Bool` | — | Логическое значение: `true` / `false` |
| `Char` | — | Unicode-символ |
| `String` | — | Строка |

### 1.5 Идентификаторы

Идентификаторы начинаются с буквы или подчёркивания, последующие символы могут быть буквами, цифрами или подчёркиваниями. Идентификаторы чувствительны к регистру.

Специальные идентификаторы:
- `_` используется как placeholder, обозначающий игнорирование значения
- Идентификаторы, начинающиеся с подчёркивания, обозначают приватные члены

### 1.6 Литералы

#### 1.6.1 Целые числа

```
Decimal     ::= [0-9][0-9_]*
Octal       ::= 0o[0-7][0-7_]*
Hex         ::= 0x[0-9a-fA-F][0-9a-fA-F_]*
Binary      ::= 0b[01][01_]*
```

#### 1.6.2 Числа с плавающей точкой

```
Float       ::= [0-9][0-9_]* '.' [0-9][0-9_]* ([eE][+-]?[0-9][0-9_]*)?
```

#### 1.6.3 Строки

```
String      ::= '"' ([^"\\] | EscapeSequence)* '"'
Escape      ::= '\\' ([nrt'"\\] | UnicodeEscape)
Unicode     ::= 'u' '{' HexDigit+ '}'
```

#### 1.6.4 Коллекции

```
List        ::= '[' Expr (',' Expr)* ']'
Dict        ::= '{' String ':' Expr (',' String ':' Expr)* '}'
Set         ::= '{' Expr (',' Expr)* '}'
```

#### 1.6.5 List comprehension

```
ListComp    ::= '[' Expr 'for' Identifier 'in' Expr (',' Expr)* ('if' Expr)? ']'
```

#### 1.6.6 Проверка членства

```
Membership  ::= Expr 'in' Expr
```

### 1.7 Комментарии

```
// однострочный комментарий

/* многострочный комментарий
   может занимать несколько строк */
```

### 1.8 Правила отступов

Код должен использовать отступ в 4 пробела; использование символов табуляции запрещено. Это обязательное синтаксическое правило.

## Глава II: Синтаксические правила

### 2.1 Классификация выражений

```
Expr        ::= Literal
              | Identifier
              | FnCall
              | MemberAccess
              | IndexAccess
              | UnaryOp
              | BinaryOp
              | TypeCast
              | RangeExpr
              | ErrorPropagate
              | RefExpr
              | IfExpr
              | MatchExpr
              | Block
              | Lambda
```

### 2.2 Приоритет операторов

| Приоритет | Оператор | Ассоциативность |
|--------|--------|--------|
| 1 | `()` `[]` `.` `?` | Слева направо |
| 2 | `as` | Слева направо |
| 3 | `*` `/` `%` | Слева направо |
| 4 | `+` `-` | Слева направо |
| 5 | `..` | Слева направо |
| 6 | `<<` `>>` | Слева направо |
| 7 | `&` `\|` `^` | Слева направо |
| 8 | `==` `!=` `<` `>` `<=` `>=` | Слева направо |
| 9 | `not` | Справа налево |
| 10 | `and` `or` | Слева направо |
| 11 | `if...else` | Справа налево |
| 12 | `=` `+=` `-=` `*=` `/=` | Справа налево |

### 2.3 Вызов функции

```
FnCall      ::= Expr '(' ArgList? ')'
ArgList     ::= Expr (',' Expr)* (',' NamedArg)* | NamedArg (',' NamedArg)*
NamedArg    ::= Identifier ':' Expr
```

### 2.4 Доступ к членам

```
MemberAccess::= Expr '.' Identifier
```

### 2.5 Индексный доступ

```
IndexAccess ::= Expr '[' Expr ']'
```

### 2.6 Приведение типов

```
TypeCast    ::= Expr 'as' TypeExpr
```

### 2.7 Условное выражение

```
IfExpr      ::= 'if' Expr Block ('elif' Expr Block)* ('else' Block)?
```

### 2.8 Pattern matching

```
MatchExpr   ::= 'match' Expr '{' MatchArm+ '}'
MatchArm    ::= Pattern ('|' Pattern)* ('if' Expr)? '=>' Expr ','
Pattern     ::= Literal
              | Identifier
              | Wildcard
              | StructPattern
              | TuplePattern
              | EnumPattern
              | OrPattern
```

### 2.9 Блочное выражение

```
Block       ::= '{' Stmt* Expr? '}'
```

**Унифицированная семантика**: семантика `return` для всех блоков `{}` согласована:

| Тип блока | Семантика return | Возврат по умолчанию |
|--------|------------|----------|
| Обычный `{}` | Возвращает значение | Void |
| `unsafe {}` | Возвращает определение типа | Void |
| `spawn {}` | Возвращает результат | Void |

**Основные принципы**:
- `return` в `{}` всегда возвращает содержимое в область видимости выше
- По умолчанию без `return` возвращается `Void`
- Форма выражения `= expr` возвращает значение напрямую

```yaoxiang
# Обычный блок {}: return возвращает значение
result = {
    x = compute()
    return x  # возвращает значение в область видимости выше
}

# Блок unsafe {}: return возвращает определение типа
SqliteDb = unsafe {
    SqliteDb: Type = {
        handle: *Void
    }
    return SqliteDb  # возвращает определение типа в область видимости выше
}

# Блок spawn {}: return возвращает результат
(a, b) = spawn {
    result1 = fetch("url1"),
    result2 = fetch("url2")
    return (result1, result2)  # возвращает результат в область видимости выше
}
```

### 2.10 Лямбда-выражение

```
Lambda      ::= '(' ParamList? ')' '=>' Expr
            |  '(' ParamList? ')' '=>' Block
```

### 2.11 Оператор error propagation

```
ErrorPropagate ::= Expr '?'
```

Оператор `?` является постфиксным, его приоритет совпадает с `.`. Для типа `Result(T, E)`:
- При `Ok(v)` извлекает значение `v` и продолжает выполнение
- При `Err(e)` распространяет ошибку вверх (`return Err(e)`)

```yaoxiang
process: (data: Data) -> Result(Data, Error) = {
    validated = validate(data)?     // при успехе извлекает значение, при ошибке распространяет вверх
    transform(validated)
}
```

### 2.12 Выражение диапазона

```
RangeExpr   ::= Expr '..' Expr
```

`..` создаёт тип диапазона, используемый в циклах `for` и срезах.

```yaoxiang
for i in 0..10 { print(i) }
slice = array[0..5]
```

### 2.13 Выражение ref

```
RefExpr     ::= 'ref' Expr
```

`ref` создаёт разделяемое владение. Компилятор автоматически выбирает Rc (в пределах одной задачи) или Arc (между задачами); пользователю не нужно беспокоиться о деталях реализации.

```yaoxiang
data = ref heavy_data
spawn { use(data) }   // межзадачно: компилятор автоматически выберет Arc
```

### 2.14 Выражение unsafe

```
UnsafeExpr  ::= 'unsafe' Block
```

Блок `unsafe` используется для определения непрозрачных типов и работы с сырыми указателями. Используйте `return` для возврата определения типа в область видимости выше.

**Семантика**:
- В `unsafe {}` можно определять типы и работать с сырыми указателями
- Возвращённый тип доступен за пределами `unsafe {}`
- Доступ к полям типа требует привилегий unsafe

```yaoxiang
# Определение непрозрачного типа в блоке unsafe
SqliteDb = unsafe {
    SqliteDb: Type = {
        handle: *Void  # сырой указатель
    }
    return SqliteDb
}

# SqliteDb доступен за пределами блока unsafe
db = sqlite3_open("test.db")
```

### 2.15 Область видимости

**Основные правила**:
- Каждый блок `{}` создаёт область видимости
- Внутренняя область видимости может обращаться к переменным внешней
- Внешняя область видимости не может обращаться к переменным внутренней
- Объявление переменных следует принципу «приоритет присваивания»

```yaoxiang
# Область видимости блока
{
    x = 10
    # x виден в данной области видимости
}
# x не виден за пределами данной области видимости

# Область видимости функции
add: (a: Int, b: Int) -> Int = {
    result = a + b
    return result
}
# result не виден за пределами функции
```

**Объявление и затенение переменных**:
- `x = value`: поиск `x` по цепочке областей видимости наружу; если найден — присваивание, если нет — новое объявление
- `mut x = value`: явное новое объявление изменяемой переменной, запрещено совпадение имён с внешней
- В одной области видимости каждое имя может быть объявлено только один раз

> **Подробное определение**: полные правила области видимости, объявления переменных и механизма затенения см. в [Спецификации модульной системы](./modules.md#第四章作用域).

## Глава III: Операторы

### 3.1 Классификация операторов

```
Stmt        ::= LetStmt
              | ExprStmt
              | ReturnStmt
              | BreakStmt
              | ContinueStmt
              | IfStmt
              | MatchStmt
              | WhileStmt
              | ForStmt
              | SpawnStmt
```

### 3.2 Объявление переменных

```
LetStmt     ::= ('mut')? Identifier (':' TypeExpr)? '=' Expr
```

### 3.3 Оператор return

```
ReturnStmt  ::= 'return' Expr?
```

**Семантика**: `return` используется для возврата значения из блока кода. При отсутствии `return` блок кода по умолчанию возвращает `Void`.

### 3.4 Оператор break

```
BreakStmt   ::= 'break' Identifier?
```

### 3.5 Оператор continue

```
ContinueStmt::= 'continue'
```

### 3.6 Оператор if

```
IfStmt      ::= 'if' Expr Block ('elif' Expr Block)* ('else' Block)?
```

### 3.7 Оператор match

```
MatchStmt   ::= 'match' Expr '{' MatchArm+ '}'
```

### 3.8 Оператор while

```
WhileStmt   ::= 'while' Expr Block
```

### 3.9 Оператор for

```
ForStmt     ::= 'for' 'mut'? Identifier 'in' Expr Block
```

#### 3.9.1 Семантика: каждая итерация — это новое связывание значения

Семантика цикла for в YaoXiang отличается от традиционных языков: **каждая итерация — это новое связывание значения, а не изменение одной и той же переменной**.

```yaoxiang
// Пример: for i in 1..5
for i in 1..5 {
    print(i)
}
```

**Процесс выполнения**:

| Итерация | Поведение переменной цикла |
|------|----------------|
| 1-я | Создаётся новая привязка `i = 1`, выполняется тело цикла, печатается 1 |
| 2-я | Создаётся новая привязка `i = 2` (предыдущая привязка уничтожена), выполняется тело цикла, печатается 2 |
| 3-я | Создаётся новая привязка `i = 3`, выполняется тело цикла, печатается 3 |
| 4-я | Создаётся новая привязка `i = 4`, выполняется тело цикла, печатается 4 |
| Завершение | Тело цикла завершается, привязка уничтожается |

**Ключевой момент**: после завершения каждой итерации созданная в этой итерации привязка уничтожается. Следующая итерация — это совершенно новая привязка, не имеющая никакой связи с привязкой предыдущей итерации.

#### 3.9.2 Различие между for и for mut

| Синтаксис | Изменяемость переменной цикла | Описание |
|------|----------------|------|
| `for i in 1..5` | Неизменяемая | В теле цикла нельзя изменять привязку |
| `for mut i in 1..5` | Изменяемая | В теле цикла можно изменять привязку |

```yaoxiang
// Допустимо: каждая итерация связывает новое значение, модификация не требуется
for i in 1..5 {
    print(i)  // чтение значения i
}

// Ошибка: неизменяемая привязка, нельзя изменять
for i in 1..5 {
    i = i + 1  // ошибка: нельзя изменять неизменяемую привязку
}

// Допустимо: использование for mut разрешает изменение привязки
for mut i in 1..5 {
    i = i + 1  // модификация разрешена
}
```

#### 3.9.3 Проверка затенения

YaoXiang запрещает затенение переменных. Переменная цикла for не может иметь то же имя, что и переменная во внешней области видимости:

```yaoxiang
// Ошибка: i уже объявлен во внешней области
i = 10
for i in 1..5 {
    print(i)
}

// Правильно: использование разных имён переменных
i = 10
for j in 1..5 {
    print(j)
}
```

Это правило применимо ко всем блокам кода, подробности см. в [4.3 Правила затенения](./modules.md#43-遮蔽规则).

#### 3.9.4 Сравнение с другими языками

| Язык | Семантика переменной цикла for |
|------|------------------|
| YaoXiang | Каждая итерация — новая привязка значения |
| Rust | Изменение одной и той же переменной (требуется mut) |
| Python | Изменение одной и той же переменной (mut не требуется) |
| C/C++ | Изменение одной и той же переменной (требуется указатель или ссылка) |

**Обоснование дизайна**: YaoXiang использует семантику связывания по следующим причинам:

1. **Более естественная семантика**
   В естественном языке «для каждого элемента x в коллекции» означает, что каждый x — это самостоятельная сущность. `for i in 1..5` в YaoXiang читается как «для каждого i от 1 до 5», и i в каждой итерации — это совершенно новая привязка, что согласуется с интуитивным пониманием человека.

2. **Предотвращение случайных модификаций**
   Семантика неизменяемого связывания по умолчанию означает, что в теле цикла невозможно случайно модифицировать переменную цикла. Не нужно беспокоиться о том, что где-то в сложном теле цикла случайно напишется `i = ...`, что приведёт к трудноотслеживаемой ошибке.

3. **Высокопроизводительные решения под рукой**
   Когда действительно необходимо переиспользовать переменную между итерациями (например, аккумулятор, кэш), достаточно использовать объявление `for mut` для переключения в режим изменяемого связывания. Это яснее неявного разделения состояния — намерение выражается явно через синтаксис, а не скрывается в поведении во время выполнения.

### 3.10 Оператор spawn

```
SpawnBlock  ::= '(' Pattern (',' Pattern)* ')' '=' 'spawn' '{' Expr (',' Expr)* '}'
SpawnFor    ::= Identifier '=' 'spawn' 'for' 'mut'? Identifier 'in' Expr '{' Expr '}'
SpawnStmt   ::= SpawnBlock | SpawnFor
```

**Блок spawn**: явное объявление параллельной области; выражения внутри блока выполняются параллельно.

```yaoxiang
(result_a, result_b) = spawn {
    parse(fetch("url1")),
    parse(fetch("url2"))
}
```

**Цикл spawn**: цикл с параллелизмом по данным.

```yaoxiang
results = spawn for item in items {
    process(item)
}
```

## Приложение: Краткий справочник по синтаксису

### A.1 Управление потоком

```
if Expr Block (elif Expr Block)* (else Block)?
match Expr { MatchArm+ }
while Expr Block
for 'mut'? Identifier 'in' Expr Block
```

### A.2 Обработка ошибок

```
Expr '?'              // error propagation (тип Result)
```

### A.3 Синтаксис match

```
match value {
    pattern1 => expr1,
    pattern2 if guard => expr2,
    _ => default_expr,
}
```
