# Спецификация синтаксиса

В данном документе определяется спецификация синтаксиса языка программирования YaoXiang, включая лексическую структуру, синтаксические правила и приоритет операторов.

---

## Глава 1: Лексическая структура

### 1.1 Исходные файлы

Исходные файлы YaoXiang должны использовать кодировку UTF-8. Исходные файлы обычно имеют расширение `.yx`.

### 1.2 Классификация лексем

| Категория | Описание | Пример |
|-----------|----------|--------|
| Идентификатор | Начинается с буквы или подчёркивания | `x`, `_private`, `my_var` |
| Ключевое слово | Зарезервированное слово языка | `Type`, `pub`, `use` |
| Литерал | Фиксированное значение | `42`, `"hello"`, `true` |
| Оператор | Символ операции | `+`, `-`, `*`, `/` |
| Разделитель | Синтаксический разделитель | `(`, `)`, `{`, `}`, `,` |

### 1.3 Ключевые слова

YaoXiang определяет минимальное количество ключевых слов:

```
pub    use    spawn
ref    mut    if     elif
else   match  while  for    return
break  continue as     in     unsafe
```

Эти ключевые слова имеют особое значение в любом контексте и не могут использоваться как идентификаторы.

### 1.4 Зарезервированные слова

| Зарезервированное слово | Тип | Описание |
|------------------------|-----|----------|
| `Type` | Type | 元类型 |
| `true` | Bool | 布尔真值 |
| `false` | Bool | 布尔假值 |
| `void` | Void | 空值 |
| `some(T)` | Option | Option 值变体 |
| `ok(T)` | Result | Result 成功变体 |
| `err(E)` | Result | Result 错误变体 |

### 1.5 Идентификаторы

Идентификаторы начинаются с буквы или подчёркивания, последующие символы могут быть буквами, цифрами или подчёркиваниями. Идентификаторы чувствительны к регистру.

Специальные идентификаторы:
- `_` используется как заполнитель, означающий игнорирование значения
- Идентификаторы, начинающиеся с подчёркивания, обозначают приватные члены

### 1.6 Литералы

#### 1.6.1 Целые числа

```
Decimal     ::= [0-9][0-9_]*
Octal       ::= 0o[0-7][0-7_]*
Hex         ::= 0x[0-9a-fA-F][0-9a-fA-F_]*
Binary      ::= 0b[01][01_]*
```

#### 1.6.2 Числа с плавающей точкой

```
Float       ::= [0-9][0-9_]* '.' [0-9][0-9_]* ([eE][+-]?[0-9][0-9_]*)?
```

#### 1.6.3 Строки

```
String      ::= '"' ([^"\\] | EscapeSequence)* '"'
Escape      ::= '\\' ([nrt'"\\] | UnicodeEscape)
Unicode     ::= 'u' '{' HexDigit+ '}'
```

#### 1.6.4 Коллекции

```
List        ::= '[' Expr (',' Expr)* ']'
Dict        ::= '{' String ':' Expr (',' String ':' Expr)* '}'
Set         ::= '{' Expr (',' Expr)* '}'
```

#### 1.6.5 Генератор списка

```
ListComp    ::= '[' Expr 'for' Identifier 'in' Expr (',' Expr)* ('if' Expr)? ']'
```

#### 1.6.6 Проверка принадлежности

```
Membership  ::= Expr 'in' Expr
```

### 1.7 Комментарии

```
// Однострочный комментарий

/* Многострочный комментарий
   может занимать несколько строк */
```

### 1.8 Правила отступов

Код должен использовать отступ в 4 пробела, использование символов Tab запрещено. Это обязательное синтаксическое правило.

## Глава 2: Синтаксические правила

### 2.1 Классификация выражений

```
Expr        ::= Literal
              | Identifier
              | FnCall
              | MemberAccess
              | IndexAccess
              | UnaryOp
              | BinaryOp
              | TypeCast
              | RangeExpr
              | ErrorPropagate
              | RefExpr
              | IfExpr
              | MatchExpr
              | Block
              | Lambda
```

### 2.2 Приоритет операторов

| Приоритет | Операторы | Ассоциативность |
|-----------|-----------|-----------------|
| 1 | `()` `[]` `.` `?` | слева направо |
| 2 | `as` | слева направо |
| 3 | `*` `/` `%` | слева направо |
| 4 | `+` `-` | слева направо |
| 5 | `..` | слева направо |
| 6 | `<<` `>>` | слева направо |
| 7 | `&` `\|` `^` | слева направо |
| 8 | `==` `!=` `<` `>` `<=` `>=` | слева направо |
| 9 | `not` | справа налево |
| 10 | `and` `or` | слева направо |
| 11 | `if...else` | справа налево |
| 12 | `=` `+=` `-=` `*=` `/=` | справа налево |

### 2.3 Вызов функции

```
FnCall      ::= Expr '(' ArgList? ')'
ArgList     ::= Expr (',' Expr)* (',' NamedArg)* | NamedArg (',' NamedArg)*
NamedArg    ::= Identifier ':' Expr
```

### 2.4 Доступ к члену

```
MemberAccess::= Expr '.' Identifier
```

### 2.5 Доступ по индексу

```
IndexAccess ::= Expr '[' Expr ']'
```

### 2.6 Приведение типа

```
TypeCast    ::= Expr 'as' TypeExpr
```

### 2.7 Условное выражение

```
IfExpr      ::= 'if' Expr Block ('elif' Expr Block)* ('else' Block)?
```

### 2.8 Сопоставление с образцом

```
MatchExpr   ::= 'match' Expr '{' MatchArm+ '}'
MatchArm    ::= Pattern ('|' Pattern)* ('if' Expr)? '=>' Expr ','
Pattern     ::= Literal
              | Identifier
              | Wildcard
              | StructPattern
              | TuplePattern
              | EnumPattern
              | OrPattern
```

### 2.9 Блочное выражение

```
Block       ::= '{' Stmt* Expr? '}'
```

**Семантика**: внутри блока кода необходимо использовать `return` для возврата значения; без `return` по умолчанию возвращается `Void`. Форма выражения `= expr` возвращает значение напрямую.

### 2.10 Лямбда-выражение

```
Lambda      ::= '(' ParamList? ')' '=>' Expr
            |  '(' ParamList? ')' '=>' Block
```

### 2.11 Оператор распространения ошибок

```
ErrorPropagate ::= Expr '?'
```

Оператор `?` является постфиксным оператором с тем же приоритетом, что и `.`. Для типа `Result(T, E)`:
- При `Ok(v)` извлекается значение `v` и выполнение продолжается
- При `Err(e)` ошибка распространяется вверх (`return Err(e)`)

```yaoxiang
process: (data: Data) -> Result(Data, Error) = {
    validated = validate(data)?     // При успехе извлекает значение, при неудаче распространяет вверх
    transform(validated)
}
```

### 2.12 Выражение диапазона

```
RangeExpr   ::= Expr '..' Expr
```

`..` создаёт тип диапазона, используемый в циклах `for` и срезах.

```yaoxiang
for i in 0..10 { print(i) }
slice = array[0..5]
```

### 2.13 Выражение ref

```
RefExpr     ::= 'ref' Expr
```

`ref` создаёт разделяемое владение. Компилятор автоматически выбирает Rc (внутри одной задачи) или Arc (между задачами), пользователю не нужно заботиться о деталях реализации.

```yaoxiang
data = ref heavy_data
spawn { use(data) }   // Между задачами: компилятор автоматически выбирает Arc
```

## Глава 3: Операторы

### 3.2 Объявление переменной

```

```

### 3.3 Оператор return

```
ReturnStmt  ::= 'return' Expr?
```

**Семантика**: `return` используется для возврата значения из блока кода. Без `return` блок кода по умолчанию возвращает `Void`.

### 3.4 Оператор break

```

```

### 3.5 Оператор continue

```
ContinueStmt::= 'continue'
```

### 3.6 Оператор if

```
IfStmt      ::= 'if' Expr Block ('elif' Expr Block)* ('else' Block)?
```

### 3.7 Оператор match

```
MatchStmt   ::= 'match' Expr '{' MatchArm+ '}'
```

### 3.8 Оператор while

```
WhileStmt   ::= 'while' Expr Block
```

### 3.9 Оператор for

```
ForStmt     ::= 'for' 'mut'? Identifier 'in' Expr Block
```

#### 3.9.1 Семантика: каждая итерация создаёт новое связывание

Семантика цикла for в YaoXiang отличается от традиционных языков: **каждая итерация создаёт новое связывание, а не изменяет одну и ту же переменную**.

```yaoxiang
// Пример: for i in 1..5
for i in 1..5 {
    print(i)
}
```

**Процесс выполнения**:

| Итерация | Поведение переменной цикла |
|----------|----------------------------|
| 1-я | Создаётся новое связывание `i = 1`, выполняется тело цикла, печатается 1 |
| 2-я | Создаётся новое связывание `i = 2` (предыдущее связывание уничтожено), выполняется тело цикла, печатается 2 |
| 3-я | Создаётся новое связывание `i = 3`, выполняется тело цикла, печатается 3 |
| 4-я | Создаётся новое связывание `i = 4`, выполняется тело цикла, печатается 4 |
| Конец | Тело цикла завершено, связывание уничтожено |

**Ключевой момент**: после окончания каждой итерации связывание, созданное в эту итерацию, уничтожается. Следующая итерация — это совершенно новое связывание, не имеющее никакой связи со связыванием предыдущей итерации.

#### 3.9.2 Разница между for и for mut

| Синтаксис | Изменяемость переменной цикла | Описание |
|-----------|-------------------------------|----------|
| `for i in 1..5` | Неизменяемо | Нельзя изменять связывание в теле цикла |
| `for mut i in 1..5` | Изменяемо | Можно изменять связывание в теле цикла |

```yaoxiang
// Допустимо: каждая итерация связывает новое значение, изменение не требуется
for i in 1..5 {
    print(i)  // Читает значение i
}

// Ошибка: неизменяемое связывание, нельзя изменять
for i in 1..5 {
    i = i + 1  // Ошибка: нельзя изменять неизменяемое связывание
}

// Допустимо: использование for mut позволяет изменять связывание
for mut i in 1..5 {
    i = i + 1  // Разрешено
}
```

#### 3.9.3 Проверка затенения

YaoXiang запрещает затенение переменных. Переменная цикла for не может иметь то же имя, что и переменная во внешней области видимости:

```yaoxiang
// Ошибка: i уже объявлена во внешней области
i = 10
for i in 1..5 {
    print(i)
}

// Правильно: использовать другое имя переменной
i = 10
for j in 1..5 {
    print(j)
}
```

Это правило применяется ко всем блокам кода, подробнее см. [4.3 Правила затенения](./modules.md#43-правила-затенения).

#### 3.9.4 Сравнение с другими языками

| Язык | Семантика переменной цикла for |
|------|--------------------------------|
| YaoXiang | Каждая итерация создаёт новое связывание |
| Rust | Изменяет одну и ту же переменную (нужен mut) |
| Python | Изменяет одну и ту же переменную (mut не нужен) |
| C/C++ | Изменяет одну и ту же переменную (нужен указатель или ссылка) |

**Обоснование дизайна**: YaoXiang использует семантику связывания, потому что:

1. **Более соответствует естественной семантике**
   В естественном языке «для каждого элемента x из множества» означает, что каждый x — независимая сущность. YaoXiang `for i in 1..5` читается как «для каждого i от 1 до 5», и каждая итерация i — это совершенно новое связывание, что соответствует человеческой интуиции.

2. **Предотвращает случайное изменение**
   Неизменяемая по умолчанию семантика связывания означает, что внутри тела цикла невозможно случайно изменить переменную цикла. Не нужно беспокоиться о том, что где-то в сложном теле цикла случайно написано `i = ...`, что может вызвать трудноотслеживаемую ошибку.

3. **Высокопроизводительные решения легко доступны**
   Когда действительно необходимо повторно использовать переменную между итерациями (например, аккумулятор, кэш), можно использовать `for mut` для переключения в режим изменяемого связывания. Это яснее, чем неявное разделение состояния — намерение выражается синтаксически, а не скрывается в поведении во время выполнения.

### 3.10 Оператор spawn

```
SpawnBlock  ::= '(' Pattern (',' Pattern)* ')' '=' 'spawn' '{' Expr (',' Expr)* '}'
SpawnFor    ::= Identifier '=' 'spawn' 'for' 'mut'? Identifier 'in' Expr '{' Expr '}'
SpawnStmt   ::= SpawnBlock | SpawnFor
```

**spawn блок**: явно объявляет границу параллелизма, выражения внутри блока выполняются параллельно.

```yaoxiang
(result_a, result_b) = spawn {
    parse(fetch("url1")),
    parse(fetch("url2"))
}
```

**spawn цикл**: цикл с параллелизмом данных.

```yaoxiang
results = spawn for item in items {
    process(item)
}
```

## Приложение: Краткая справка по синтаксису

### A.1 Управление потоком выполнения


```
Expr '?'              // Распространение ошибки (тип Result)
```

### A.3 Синтаксис match

```
match value {
    pattern1 => expr1,
    pattern2 if guard => expr2,
    _ => default_expr,
}
```
