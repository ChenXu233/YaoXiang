# Спецификация синтаксиса

В данном документе определяется спецификация синтаксиса языка программирования YaoXiang, включая лексическую структуру, синтаксические правила и приоритет операторов.

---

## Глава 1: Лексическая структура

### 1.1 Исходные файлы

Исходные файлы YaoXiang должны использовать кодировку UTF-8. Исходные файлы обычно имеют расширение `.yx`.

### 1.2 Классификация лексем

| Категория | Описание | Пример |
|-----------|----------|---------|
| Идентификатор | Начинается с буквы или подчёркивания | `x`, `_private`, `my_var` |
| Ключевое слово | Зарезервированное слово языка | `Type`, `pub`, `use` |
| Литерал | Фиксированное значение | `42`, `"hello"`, `true` |
| Оператор | Символ операции | `+`, `-`, `*`, `/` |
| Разделитель | Синтаксический разделитель | `(`, `)`, `{`, `}`, `,` |

### 1.3 Ключевые слова

YaoXiang определяет минимальное количество ключевых слов:

```
pub    use    spawn
ref    mut    if     elif
else   match  while  for    return
break  continue as     in     unsafe
```

Эти ключевые слова имеют особое значение в любом контексте и не могут использоваться как идентификаторы.

### 1.4 Зарезервированные слова

| Зарезервированное слово | Тип | Описание |
|------------------------|-----|----------|
| `Type` | Type | Метатип |
| `true` | Bool | Булево истинное значение |
| `false` | Bool | Булево ложное значение |
| `void` | Void | Пустое значение |
| `some(T)` | Option | Вариант значения Option |
| `ok(T)` | Result | Вариант успеха Result |
| `err(E)` | Result | Вариант ошибки Result |

### 1.5 Идентификаторы

Идентификаторы начинаются с буквы или подчёркивания, последующие символы могут быть буквами, цифрами или подчёркиванием. Идентификаторы чувствительны к регистру.

Специальные идентификаторы:
- `_` используется как заполнитель, означающий игнорирование значения
- Идентификаторы, начинающиеся с подчёркивания, обозначают приватные члены

### 1.6 Литералы

#### 1.6.1 Целые числа

```
Decimal     ::= [0-9][0-9_]*
Octal       ::= 0o[0-7][0-7_]*
Hex         ::= 0x[0-9a-fA-F][0-9a-fA-F_]*
Binary      ::= 0b[01][01_]*
```

#### 1.6.2 Числа с плавающей точкой

```
Float       ::= [0-9][0-9_]* '.' [0-9][0-9_]* ([eE][+-]?[0-9][0-9_]*)?
```

#### 1.6.3 Строки

```
String      ::= '"' ([^"\\] | EscapeSequence)* '"'
Escape      ::= '\\' ([nrt'"\\] | UnicodeEscape)
Unicode     ::= 'u' '{' HexDigit+ '}'
```

#### 1.6.4 Коллекции

```
List        ::= '[' Expr (',' Expr)* ']'
Dict        ::= '{' String ':' Expr (',' String ':' Expr)* '}'
Set         ::= '{' Expr (',' Expr)* '}'
```

#### 1.6.5 Генератор списка

```
ListComp    ::= '[' Expr 'for' Identifier 'in' Expr (',' Expr)* ('if' Expr)? ']'
```

#### 1.6.6 Проверка вхождения

```
Membership  ::= Expr 'in' Expr
```

### 1.7 Комментарии

```
// Однострочный комментарий

/* Многострочный комментарий
   может занимать несколько строк */
```

### 1.8 Правила отступов

Код должен использовать отступ в 4 пробела, использование символа Tab запрещено. Это обязательное синтаксическое правило.

## Глава 2: Синтаксические правила

### 2.1 Классификация выражений

```
Expr        ::= Literal
              | Identifier
              | FnCall
              | MemberAccess
              | IndexAccess
              | UnaryOp
              | BinaryOp
              | TypeCast
              | IfExpr
              | MatchExpr
              | Block
              | Lambda
```

### 2.2 Приоритет операторов

| Приоритет | Оператор | Ассоциативность |
|-----------|----------|-----------------|
| 1 | `()` `[]` `.` | Слева направо |
| 2 | `as` | Слева направо |
| 3 | `*` `/` `%` | Слева направо |
| 4 | `+` `-` | Слева направо |
| 5 | `<<` `>>` | Слева направо |
| 6 | `&` `\|` `^` | Слева направо |
| 7 | `==` `!=` `<` `>` `<=` `>=` | Слева направо |
| 8 | `not` | Справа налево |
| 9 | `and` `or` | Слева направо |
| 10 | `if...else` | Справа налево |
| 11 | `=` `+=` `-=` `*=` `/=` | Справа налево |

### 2.3 Вызов функции

```
FnCall      ::= Expr '(' ArgList? ')'
ArgList     ::= Expr (',' Expr)* (',' NamedArg)* | NamedArg (',' NamedArg)*
NamedArg    ::= Identifier ':' Expr
```

### 2.4 Доступ к члену

```
MemberAccess::= Expr '.' Identifier
```

### 2.5 Доступ по индексу

```
IndexAccess ::= Expr '[' Expr ']'
```

### 2.6 Преобразование типа

```
TypeCast    ::= Expr 'as' TypeExpr
```

### 2.7 Условное выражение

```
IfExpr      ::= 'if' Expr Block ('elif' Expr Block)* ('else' Block)?
```

### 2.8 Сопоставление с образцом

```
MatchExpr   ::= 'match' Expr '{' MatchArm+ '}'
MatchArm    ::= Pattern ('|' Pattern)* ('if' Expr)? '=>' Expr ','
Pattern     ::= Literal
              | Identifier
              | Wildcard
              | StructPattern
              | TuplePattern
              | EnumPattern
              | OrPattern
```

### 2.9 Блочное выражение

```
Block       ::= '{' Stmt* Expr? '}'
```

### 2.10 Лямбда-выражение

```
Lambda      ::= '(' ParamList? ')' '=>' Expr
            |  '(' ParamList? ')' '=>' Block
```

```
              | BreakStmt
              | ContinueStmt
              | IfStmt
              | MatchStmt
              | LoopStmt
              | WhileStmt

```

### 3.2 Объявление переменной

```

```

### 3.3 Оператор return

```
ReturnStmt  ::= 'return' Expr?
```

### 3.4 Оператор break

```
BreakStmt   ::= 'break' Identifier?
```

### 3.5 Оператор continue

```
ContinueStmt::= 'continue'
```

### 3.6 Оператор if

```
IfStmt      ::= 'if' Expr Block ('elif' Expr Block)* ('else' Block)?
```

### 3.7 Оператор match

```
MatchStmt   ::= 'match' Expr '{' MatchArm+ '}'
```

### 3.8 Оператор while

```
WhileStmt   ::= 'while' Expr Block
```

### 3.9 Оператор for

```
ForStmt     ::= 'for' 'mut'? Identifier 'in' Expr Block
```

#### 3.9.1 Семантика: каждая итерация связывает новое значение

Семантика цикла for в YaoXiang отличается от традиционных языков: **каждая итерация связывает новое значение, а не изменяет одну и ту же переменную**.

```yaoxiang
// Пример: for i in 1..5
for i in 1..5 {
    print(i)
}
```

**Процесс выполнения**:

| Итерация | Поведение переменной цикла |
|----------|---------------------------|
| 1-я | Создаётся новая привязка `i = 1`, выполняется тело цикла, печатается 1 |
| 2-я | Создаётся новая привязка `i = 2` (предыдущая привязка уничтожена), выполняется тело цикла, печатается 2 |
| 3-я | Создаётся новая привязка `i = 3`, выполняется тело цикла, печатается 3 |
| 4-я | Создаётся новая привязка `i = 4`, выполняется тело цикла, печатается 4 |
| Конец | Тело цикла завершено, привязка уничтожена |

**Ключевой момент**: в конце каждой итерации привязка, созданная в этой итерации, уничтожается. Следующая итерация — это совершенно новая привязка, не имеющая никакой связи с привязкой предыдущей итерации.

#### 3.9.2 Различия между for и for mut

| Синтаксис | Изменяемость переменной цикла | Описание |
|-----------|-------------------------------|----------|
| `for i in 1..5` | Неизменяемая | Нельзя изменять привязку в теле цикла |
| `for mut i in 1..5` | Изменяемая | Можно изменять привязку в теле цикла |

```yaoxiang
// Допустимо: каждая итерация связывает новое значение, изменение не требуется
for i in 1..5 {
    print(i)  // Чтение значения i
}

// Ошибка: неизменяемая привязка, нельзя изменять
for i in 1..5 {
    i = i + 1  // Ошибка: нельзя изменять неизменяемую привязку
}

// Допустимо: использование for mut позволяет изменять привязку
for mut i in 1..5 {
    i = i + 1  // Разрешено изменение
}
```

#### 3.9.3 Проверка затенения

Переменные цикла for не могут затенять уже существующие переменные во внешней области видимости:

```yaoxiang
// Ошибка: i уже объявлена во внешней области
i = 10
for i in 1..5 {
    print(i)
}

// Правильно: использовать другое имя переменной
i = 10
for j in 1..5 {
    print(j)
}
```

Код с ошибкой: `E2013 - Cannot shadow existing variable`

#### 3.9.4 Сравнение с другими языками

| Язык | Семантика переменной цикла for |
|------|--------------------------------|
| YaoXiang | Каждая итерация связывает новое значение |
| Rust | Изменяет одну и ту же переменную (требуется mut) |
| Python | Изменяет одну и ту же переменную (mut не требуется) |
| C/C++ | Изменяет одну и ту же переменную (требуются указатели или ссылки) |

**Обоснование дизайна**: YaoXiang использует семантику привязки, потому что:
1. Переменные в теле цикла уничтожаются после завершения каждой итерации
2. Следующая итерация — это совершенно новая привязка
3. Это безопаснее, не нужно учитывать состояние между итерациями

## Приложение: Краткая справка по синтаксису

### А.1 Управление потоком выполнения

```
if Expr Block (elif Expr Block)* (else Block)?

while Identifier in Expr Block Expr Block
for
```

### А.2 Синтаксис match

```
match value {
    pattern1 => expr1,
    pattern2 if guard => expr2,
    _ => default_expr,
}
```
