# Спецификация синтаксиса

В данном документе определяется спецификация синтаксиса языка программирования YaoXiang, включая лексическую структуру, синтаксические правила и приоритет операторов.

---

## Глава 1: Лексическая структура

### 1.1 Исходные файлы

Исходные файлы YaoXiang должны использовать кодировку UTF-8. Исходные файлы обычно имеют расширение `.yx`.

### 1.2 Классификация лексем

| Категория | Описание | Пример |
|-----------|----------|--------|
| Идентификаторы | Начинаются с буквы или подчёркивания | `x`, `_private`, `my_var` |
| Ключевые слова | Зарезервированные слова языка | `Type`, `pub`, `use` |
| Литералы | Фиксированные значения | `42`, `"hello"`, `true` |
| Операторы | Знаки операций | `+`, `-`, `*`, `/` |
| Разделители | Синтаксические разделители | `(`, `)`, `{`, `}`, `,` |

### 1.3 Ключевые слова

YaoXiang определяет минимальное количество ключевых слов:

```
pub    use    spawn
ref    mut    if     elif
else   match  while  for    return
break  continue as     in     unsafe
```

Эти ключевые слова имеют особое значение в любом контексте и не могут использоваться как идентификаторы.

### 1.4 Зарезервированные слова

| Зарезервированное слово | Тип | Описание |
|------------------------|-----|----------|
| `Type` | Type | Метатип |
| `true` | Bool | Булево истинное значение |
| `false` | Bool | Булево ложное значение |
| `void` | Void | Пустое значение |
| `some(T)` | Option | Вариант-значение типа Option |
| `ok(T)` | Result | Вариант успеха типа Result |
| `err(E)` | Result | Вариант ошибки типа Result |

### 1.5 Идентификаторы

Идентификаторы начинаются с буквы или подчёркивания, последующие символы могут быть буквами, цифрами или подчёркиваниями. Идентификаторы чувствительны к регистру.

Специальные идентификаторы:
- `_` используется как заполнитель, обозначающий игнорирование некоторого значения
- Идентификаторы, начинающиеся с подчёркивания, обозначают приватные члены

### 1.6 Литералы

#### 1.6.1 Целые числа

```
Decimal     ::= [0-9][0-9_]*
Octal       ::= 0o[0-7][0-7_]*
Hex         ::= 0x[0-9a-fA-F][0-9a-fA-F_]*
Binary      ::= 0b[01][01_]*
```

#### 1.6.2 Числа с плавающей точкой

```
Float       ::= [0-9][0-9_]* '.' [0-9][0-9_]* ([eE][+-]?[0-9][0-9_]*)?
```

#### 1.6.3 Строки

```
String      ::= '"' ([^"\\] | EscapeSequence)* '"'
Escape      ::= '\\' ([nrt'"\\] | UnicodeEscape)
Unicode     ::= 'u' '{' HexDigit+ '}'
```

#### 1.6.4 Коллекции

```
List        ::= '[' Expr (',' Expr)* ']'
Dict        ::= '{' String ':' Expr (',' String ':' Expr)* '}'
Set         ::= '{' Expr (',' Expr)* '}'
```

#### 1.6.5 Генератор списка

```
ListComp    ::= '[' Expr 'for' Identifier 'in' Expr (',' Expr)* ('if' Expr)? ']'
```

#### 1.6.6 Проверка принадлежности

```
Membership  ::= Expr 'in' Expr
```

### 1.7 Комментарии

```
// Однострочный комментарий

/* Многострочный комментарий
   может занимать несколько строк */
```

### 1.8 Правила отступов

Код должен использовать отступ в 4 пробела, использование символа Tab запрещено. Это обязательное синтаксическое правило.

## Глава 2: Синтаксические правила

### 2.1 Классификация выражений

```
Expr        ::= Literal
              | Identifier
              | FnCall
              | MemberAccess
              | IndexAccess
              | UnaryOp
              | BinaryOp
              | TypeCast
              | RangeExpr
              | ErrorPropagate
              | RefExpr
              | IfExpr
              | MatchExpr
              | Block
              | Lambda
```

### 2.2 Приоритет операторов

| Приоритет | Оператор | Ассоциативность |
|-----------|----------|-----------------|
| 1 | `()` `[]` `.` `?` | Слева направо |
| 2 | `as` | Слева направо |
| 3 | `*` `/` `%` | Слева направо |
| 4 | `+` `-` | Слева направо |
| 5 | `..` | Слева направо |
| 6 | `<<` `>>` | Слева направо |
| 7 | `&` `\|` `^` | Слева направо |
| 8 | `==` `!=` `<` `>` `<=` `>=` | Слева направо |
| 9 | `not` | Справа налево |
| 10 | `and` `or` | Слева направо |
| 11 | `if...else` | Справа налево |
| 12 | `=` `+=` `-=` `*=` `/=` | Справа налево |

### 2.3 Вызов функции

```
FnCall      ::= Expr '(' ArgList? ')'
ArgList     ::= Expr (',' Expr)* (',' NamedArg)* | NamedArg (',' NamedArg)*
NamedArg    ::= Identifier ':' Expr
```

### 2.4 Обращение к члену

```
MemberAccess::= Expr '.' Identifier
```

### 2.5 Обращение по индексу

```
IndexAccess ::= Expr '[' Expr ']'
```

### 2.6 Преобразование типа

```
TypeCast    ::= Expr 'as' TypeExpr
```

### 2.7 Условное выражение

```
IfExpr      ::= 'if' Expr Block ('elif' Expr Block)* ('else' Block)?
```

### 2.8 Сопоставление с образцом

```
MatchExpr   ::= 'match' Expr '{' MatchArm+ '}'
MatchArm    ::= Pattern ('|' Pattern)* ('if' Expr)? '=>' Expr ','
Pattern     ::= Literal
              | Identifier
              | Wildcard
              | StructPattern
              | TuplePattern
              | EnumPattern
              | OrPattern
```

### 2.9 Блочное выражение

```
Block       ::= '{' Stmt* Expr? '}'
```

### 2.10 Lambda-выражение

```
Lambda      ::= '(' ParamList? ')' '=>' Expr
            |  '(' ParamList? ')' '=>' Block
```

### 2.11 Оператор распространения ошибки

```
ErrorPropagate ::= Expr '?'
```

Оператор `?` является постфиксным оператором, имеющим тот же приоритет, что и `.`. Для типа `Result(T, E)`:
- При `Ok(v)` извлекается значение `v` и выполнение продолжается
- При `Err(e)` ошибка распространяется вверх (`return Err(e)`)

```yaoxiang
process: (data: Data) -> Result(Data, Error) = {
    validated = validate(data)?     // При успехе извлекает значение, при неудаче распространяет вверх
    transform(validated)
}
```

### 2.12 Выражение диапазона

```
RangeExpr   ::= Expr '..' Expr
```

`..` создаёт тип диапазона, используемый в циклах `for` и срезах.

```yaoxiang
for i in 0..10 { print(i) }
slice = array[0..5]
```

### 2.13 Выражение ref

```
RefExpr     ::= 'ref' Expr
```

`ref` создаёт совместно владеющую ссылку. Компилятор автоматически выбирает Rc (для одной задачи) или Arc (между задачами), пользователю не нужно заботиться о деталях реализации.

```yaoxiang
data = ref heavy_data
spawn { use(data) }   // Между задачами: компилятор автоматически выбирает Arc
```

```
ForStmt     ::= 'for' 'mut'? Identifier 'in' Expr Block
```

#### 3.9.1 Семантика: каждая итерация связывает новое значение

Семантика цикла for в YaoXiang отличается от традиционных языков: **каждая итерация связывает новое значение, а не модифицирует одну и ту же переменную**.

```yaoxiang
// Пример: for i in 1..5
for i in 1..5 {
    print(i)
}
```

**Процесс выполнения**:

| Итерация | Поведение переменной цикла |
|----------|----------------------------|
| 1-я | Создаётся новая привязка `i = 1`, выполняется тело цикла, печатается 1 |
| 2-я | Создаётся новая привязка `i = 2` (предыдущая привязка уничтожена), выполняется тело цикла, печатается 2 |
| 3-я | Создаётся новая привязка `i = 3`, выполняется тело цикла, печатается 3 |
| 4-я | Создаётся новая привязка `i = 4`, выполняется тело цикла, печатается 4 |
| Конец | Тело цикла завершено, привязка уничтожается |

**Ключевой момент**: после завершения каждой итерации привязка, созданная в этой итерации, уничтожается. Следующая итерация — это совершенно новая привязка, не имеющая никакой связи с привязкой предыдущей итерации.

#### 3.9.2 Разница между for и for mut

| Синтаксис | Изменяемость переменной цикла | Описание |
|-----------|-------------------------------|----------|
| `for i in 1..5` | Неизменяемая | В теле цикла нельзя модифицировать привязку |
| `for mut i in 1..5` | Изменяемая | В теле цикла можно модифицировать привязку |

```yaoxiang
// Допустимо: каждая итерация связывает новое значение, модификация не требуется
for i in 1..5 {
    print(i)  // Чтение значения i
}

// Ошибка: неизменяемая привязка, нельзя модифицировать
for i in 1..5 {
    i = i + 1  // Ошибка: нельзя модифицировать неизменяемую привязку
}

// Допустимо: использование for mut позволяет модифицировать привязку
for mut i in 1..5 {
    i = i + 1  // Разрешено
}
```

#### 3.9.3 Проверка затенения

YaoXiang запрещает затенение переменных. Переменная цикла for не может иметь то же имя, что и переменная во внешней области видимости:

```yaoxiang
// Ошибка: i уже объявлена внешне
i = 10
for i in 1..5 {
    print(i)
}

// Правильно: использовать другое имя переменной
i = 10
for j in 1..5 {
    print(j)
}
```

Это правило применяется ко всем блокам кода, подробнее см. [4.3 Правила затенения](./modules.md#43-правила-затенения).

#### 3.9.4 Сравнение с другими языками

| Язык | Семантика переменной цикла for |
|------|--------------------------------|
| YaoXiang | Каждая итерация связывает новое значение |
| Rust | Модифицирует одну и ту же переменную (нужен mut) |
| Python | Модифицирует одну и ту же переменную (mut не требуется) |
| C/C++ | Модифицирует одну и ту же переменную (нужен указатель или ссылка) |

**Обоснование дизайна**: YaoXiang использует семантику связывания, потому что:

1. **Более соответствует естественной семантике**
   В естественном языке "для каждого элемента x из множества" означает, что каждый x — независимая сущность. YaoXiang `for i in 1..5` читается как "для каждого i от 1 до 5", и каждый i в каждой итерации — это совершенно новая привязка, что соответствует человеческой интуиции.

2. **Предотвращает случайные модификации**
   Семантика неизменяемых привязок по умолчанию означает, что невозможно случайно модифицировать переменную цикла в теле цикла. Не нужно беспокоиться о том, что где-то в сложном теле цикла случайно написано `i = ...`, что приведёт к трудно отслеживаемым ошибкам.

3. **Высокопроизводительные решения всегда доступны**
   Когда действительно необходимо повторно использовать переменную между итерациями (например, аккумулятор, кэш), достаточно объявить `for mut` для переключения в режим изменяемой привязки. Это яснее, чем неявное совместное состояние — намерение выражено синтаксически, а не скрыто в поведении во время runtime.

### 3.10 Оператор spawn

```
SpawnBlock  ::= '(' Pattern (',' Pattern)* ')' '=' 'spawn' '{' Expr (',' Expr)* '}'
SpawnFor    ::= Identifier '=' 'spawn' 'for' 'mut'? Identifier 'in' Expr '{' Expr '}'
SpawnStmt   ::= SpawnBlock | SpawnFor
```

**spawn блок**: явно объявляет область параллелизма, выражения внутри блока выполняются параллельно.

```yaoxiang
(result_a, result_b) = spawn {
    parse(fetch("url1")),
    parse(fetch("url2"))
}
```

**spawn цикл**: цикл с параллелизмом данных.

```yaoxiang
results = spawn for item in items {
    process(item)
}
```

```

    pattern1 => expr1,
    pattern2 if guard => expr2,
    _ => default_expr,
}
```
