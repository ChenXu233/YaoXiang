# Спецификация системы модулей

Настоящий документ определяет спецификацию системы модулей языка программирования YaoXiang, включая определение модулей, импорт, экспорт и области видимости.

---

## Глава 1. Основы модулей

### 1.1 Основы модулей

Модули ограничиваются файлами. Каждый файл `.yx` является модулем.

```
// Имя файла является именем модуля
// Math.yx
pub pi: Float = 3.14159
pub sqrt(Float) -> Float = (x) => { ... }
```

### 1.2 Правила именования модулей

- Имя модуля определяется именем файла
- Расширение файла `.yx` не входит в имя модуля
- Имена модулей используют PascalCase

## Глава 2. Импорт модулей

### 2.1 Синтаксис импорта

```
Import       ::= 'use' ModuleRef ImportSpec?
ImportSpec   ::= ('{' ImportItems '}') ('as' AliasList)?
              |  'as' AliasList
ImportItems  ::= Identifier (',' Identifier)* ','?
AliasList    ::= Identifier (',' Identifier)*
```

### 2.2 Способы импорта

| Синтаксис | Описание | Пример |
|-----------|----------|--------|
| `use path;` | Импорт модуля, доступ через последнюю часть | `use std.io;` -> `io.print` |
| `use path.{a, b};` | Импорт указанных элементов | `use std.io.{print};` -> `print` |
| `use path as alias;` | Импорт с переименованием | `use std.io as io;` -> `io.print` |
| `use path.{i1, i2} as a, b;` | Импорт указанных элементов с переименованием | `use std.io.{print, read} as p, r;` -> `p`, `r` |

### 2.3 Примеры импорта

```yaoxiang
// Импорт всего модуля
use std.io
io.print("Hello")

// Импорт указанных элементов
use std.io.{print, read}
print("Hello")

// Импорт с переименованием
use std.io as io_module
io_module.print("Hello")

// Импорт указанных элементов с переименованием
use std.io.{print, read} as p, r
p("Hello")
```

## Глава 3. Экспорт модулей

### 3.1 Ключевое слово pub

Для объявления экспортируемых элементов используется ключевое слово `pub`:

```yaoxiang
// Math.yx
pub pi: Float = 3.14159
pub sqrt: (x: Float) -> Float = (x) => { ... }

// Приватные элементы (не экспортируются)
internal_value: Int = 42
```

### 3.2 Правила экспорта

- По умолчанию все элементы являются приватными
- Элементы, объявленные с `pub`, доступны из других модулей
- Приватные элементы доступны только внутри текущего модуля

### 3.3 Автоматическое связывание pub

Для функций, объявленных с `pub`, компилятор автоматически выполняет связывание с типами, определёнными в том же файле:

```yaoxiang
// Использование pub, компилятор автоматически связывает
pub distance: (p1: Point, p2: Point) -> Float = {
    dx = p1.x - p2.x
    dy = p1.y - p2.y
    (dx * dx + dy * dy).sqrt()
}

// Компилятор автоматически выводит:
    // 1. Point определён в текущем файле
    // 2. Параметры функции содержат Point
    // 3. Выполняется Point.distance = distance[0]

// Вызовы
d = distance(p1, p2)           // Функциональный стиль
d2 = p1.distance(p2)           // Синтаксический сахар ООП
```

## Глава 4. Области видимости

### 4.1 Область видимости модуля

Каждый модуль имеет собственную область видимости; элементы внутри модуля по умолчанию недоступны извне.

### 4.2 Вложенные области видимости

```yaoxiang
// Область видимости блока
{
    x = 10
    // x видим в этой области
}
// x невидим за пределами этой области

// Область видимости функции
add: (a: Int, b: Int) -> Int = {
    result = a + b
    return result
}
// result невидим за пределами функции
```

### 4.3 Объявление переменных и затенение

В YaoXiang нет ключевого слова `let`. Что представляет собой `x = value` — объявление или присваивание? Действует принцип:

**Присваивание имеет приоритет.** Объявление происходит один раз, а присваивание — многократно. Пусть часто выполняемая операция идёт по кратчайшему пути.

```
x = value:
  Поиск x по цепочке областей видимости
    → Найден mut x          : присваивание, OK (через токен &mut)
    → Найден x (moved)       : считается "действительная привязка не найдена", переобъявление в текущей области
    → Найден x (неизменяемый, живой) : E2010 Нельзя переприсвоить
    → Не найден              : новое объявление в текущей области (единственный путь объявления)

mut x = value:
    → x уже существует в текущей области : E2002 Повторное определение
    → x существует во внешней области     : E2013 Затенение запрещено (явное новое объявление не может совпадать с внешним)
    → Конфликтов нет                       : новое изменяемое объявление
```

- **Одна область видимости**: любое имя может быть объявлено только один раз (E2002)
- **Во внутренней области без `mut`**: приоритетный поиск во внешней, присваивание или ошибка
- **Во внутренней области с `mut`**: явное новое объявление, запрещено совпадение с внешним (E2013)

#### Одна область видимости

```yaoxiang
x = 10
x = 20              // E2002: 'x' уже определён в этой области

mut y = 10
y = 20              // OK: та же привязка, переприсваивание
mut y = 30          // E2002: 'y' уже определён в этой области

z = 10
mut z = 20          // E2002: 'z' уже определён в этой области (mut не может перекрыть существующее объявление)
```

#### Перепривязка после Move

Неизменяемая переменная, если она владеет значением, после своего move (потребления) переходит в состояние **moved** — имя по-прежнему занимает слот области видимости, но значение недоступно. В этом случае `x = value` — не изменение старой привязки, а переобъявление `x` в той же области видимости.

```
Ветвь "найден moved" при приоритетном поиске присваивания:
  x существует в текущей области, но в состоянии moved
    → Компилятор считает "действительная привязка не найдена"
    → Переобъявление x в текущей области (перекрытие старого слота moved)
```

**Основной механизм:** после потребления старого значения привязка прекращает существование, имя возвращается в состояние "неинициализированного". Это единственный путь.

**Отличие от затенения:**
- Затенение (Rust `let x = ...`): старая привязка всё ещё существует, просто скрыта новой
- Перепривязка после move: старая привязка уже потреблена, имя возвращается в неинициализированное состояние, переобъявление — единственный выход

**Ограничения:**
- Только владеющие значения могут быть moved. Ссылки (`&T`, `&mut T`) копируются, а не перемещаются
- Проверка move выполняется во время компиляции; чтение переменной в состоянии moved в любом выражении вызывает E2014
- IDE может отображать серую подсказку на moved-переменных, указывая, что имя находится в неинициализированном состоянии

```yaoxiang
// Чтение после move → Ошибка
data = fetch()
result = process(data)   // data moved
print(data)              // E2014: 'data' была перемещена, не может использоваться

// Ссылки не вызывают move
ref_data = &value
copy1 = ref_data         // Копирование ссылки, ref_data всё ещё доступен
copy2 = ref_data          // OK

// Пересечение областей видимости: состояние moved проходит через границы
data = fetch()
{
    data = transform(data)  // move внешней data → перепривязка (новое объявление во внутренней области)
    print(data)             // OK: используется внутренняя data
}
print(data)                 // E2014: внешняя data была перемещена
```

#### Пересечение областей видимости

```yaoxiang
// Внешняя область неизменяема, внутренняя присваивает → нельзя переприсвоить неизменяемую переменную
x = 10
{
    x = 20          // E2010: 'x' неизменяема, нельзя переприсвоить
}
{
    mut x = 20      // E2013: нельзя затенять существующую переменную 'x' (явное объявление новой привязки)
}

// Внешняя mut, внутренняя присваивает → изменение той же привязки
mut y = 10
{
    y = 20          // OK: та же привязка, изменение через токен &mut
}
print(y)            // 20

// Внешняя mut, во внутренней нельзя объявить одноимённую
mut z = 10
{
    z = 30          // OK: та же привязка
}
{
    mut z = 30      // E2013: нельзя затенять существующую переменную 'z'
}

// Множественная вложенность: mut проходит через все уровни
mut a = 0
{
    {
        a = 10      // OK
    }
}
print(a)            // 10

// Неизменяемость через все уровни тоже не позволяет переприсвоить
b = 0
{
    {
        b = 10      // E2010: 'b' неизменяема, нельзя переприсвоить
    }
}
```

#### Цикл for

```yaoxiang
// Переменная цикла каждый раз — новая привязка, не изменение
for i in 1..5 {
    print(i)        // OK: каждая итерация связывает новое значение
    i = 10          // E2010: неизменяемая переменная цикла, нельзя переприсвоить
}

for mut i in 1..5 {
    i = 10          // OK: изменяемая переменная цикла
}

// Переменная цикла не может затенять внешнюю
i = 0
for i in 1..5 {     // E2013: нельзя затенять существующую переменную 'i'
}

// mut внешний аккумулятор можно изменять в теле цикла
mut sum = 0
for i in 1..5 {
    sum = sum + i   // OK: та же привязка, изменение через токен &mut
}
print(sum)          // 15

// Неизменяемый внешний нельзя изменять в теле цикла
sum2 = 0
for i in 1..5 {
    sum2 = sum2 + i // E2010: 'sum2' неизменяема, нельзя переприсвоить
}
```

#### Коды ошибок

| Код ошибки | Сообщение | Ситуация |
|------------|-----------|----------|
| E2002 | `'{name}' уже определён в этой области видимости` | Повторное объявление в одной области (независимо от mut) |
| E2010 | `Нельзя присвоить неизменяемой переменной '{name}'` | Присваивание во внутренней области без `mut`, когда внешняя переменная неизменяема и не moved |
| E2013 | `Нельзя затенять существующую переменную '{name}'` | Явное объявление (`mut x` или `x: Type`) во внутренней области совпадает с внешним именем |
| E2014 | `'{name}' была перемещена и не может использоваться` | Чтение переменной в состоянии moved |

## Глава 5. Организация модулей

### 5.1 Структура каталогов

```
src/
├── main.yx          // Главный модуль
├── math/
│   ├── index.yx     // Входной модуль математики
│   ├── vector.yx    // Модуль векторов
│   └── matrix.yx    // Модуль матриц
└── utils/
    ├── index.yx     // Входной модуль утилит
    └── string.yx    // Утилиты для строк
```

### 5.2 Входной модуль

Файл `index.yx` в каталоге служит входной точкой модуля:

```yaoxiang
// math/index.yx
use math.vector
use math.matrix

pub Vector = vector.Vector
pub Matrix = matrix.Matrix
```

### 5.3 Относительный импорт

```yaoxiang
// В math/vector.yx
use math.matrix  // Абсолютный импорт
use .matrix      // Относительный импорт (тот же каталог)
```
