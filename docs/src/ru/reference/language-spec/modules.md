# Система модулей: спецификация

Настоящий документ определяет спецификацию системы модулей языка программирования YaoXiang, включая определение модулей, импорт/экспорт и области видимости.

---

## Глава I: Определение модулей

### 1.1 Основы модулей

Границей модуля служит файл. Каждый файл `.yx` является модулем.

```
// Имя файла совпадает с именем модуля
// Math.yx
pub pi: Float = 3.14159
pub sqrt: (x: Float) -> Float = (x) => { ... }
```

### 1.2 Правила именования модулей

- Имя модуля определяется именем файла
- Расширение `.yx` не участвует в имени модуля
- Имена модулей используют стиль PascalCase

## Глава II: Импорт модулей

### 2.1 Синтаксис импорта

```
Import       ::= 'use' ModuleRef ImportSpec?
ImportSpec   ::= ('{' ImportItems '}') ('as' AliasList)?
              |  'as' AliasList
ImportItems  ::= Identifier (',' Identifier)* ','?
AliasList    ::= Identifier (',' Identifier)*
```

### 2.2 Способы импорта

| Синтаксис | Описание | Пример |
|------|------|------|
| `use path;` | Импорт модуля, доступ через последнюю часть | `use std.io;` -> `io.print` |
| `use path.{a, b};` | Импорт указанных элементов | `use std.io.{print};` -> `print` |
| `use path as alias;` | Импорт с переименованием | `use std.io as io;` -> `io.print` |
| `use path.{i1, i2} as a, b;` | Импорт указанных элементов с переименованием | `use std.io.{print, read} as p, r;` -> `p`, `r` |

### 2.3 Примеры импорта

```yaoxiang
// Импорт всего модуля
use std.io
io.print("Hello")

// Импорт указанных элементов
use std.io.{print, read}
print("Hello")

// Импорт с переименованием
use std.io as io_module
io_module.print("Hello")

// Импорт указанных элементов с переименованием
use std.io.{print, read} as p, r
p("Hello")
```

## Глава III: Экспорт модулей

### 3.1 Ключевое слово `pub`

Для объявления экспортируемых элементов используется ключевое слово `pub`:

```yaoxiang
// Math.yx
pub pi: Float = 3.14159
pub sqrt: (x: Float) -> Float = (x) => { ... }

// Приватный элемент (не экспортируется)
internal_value: Int = 42
```

### 3.2 Правила экспорта

- По умолчанию все элементы являются приватными
- Элементы, объявленные с `pub`, доступны из других модулей
- Приватные элементы доступны только внутри текущего модуля

### 3.3 Автоматическое связывание `pub`

Для функций, объявленных с `pub`, компилятор автоматически выполняет привязку к типам, определённым в том же файле:

```yaoxiang
// Объявление с pub — компилятор выполнит автоматическую привязку
pub distance: (p1: Point, p2: Point) -> Float = {
    dx = p1.x - p2.x
    dy = p1.y - p2.y
    (dx * dx + dy * dy).sqrt()
}

// Компилятор автоматически выводит:
// 1. Point определён в текущем файле
// 2. Параметры функции содержат Point
// 3. Выполняется Point.distance = distance[0]

// Вызов
d = distance(p1, p2)           // Функциональный стиль
d2 = p1.distance(p2)           // Синтаксический сахар ООП
```

## Глава IV: Области видимости

### 4.1 Область видимости модуля

Каждый модуль имеет собственную область видимости; элементы модуля по умолчанию недоступны извне.

### 4.2 Вложенные области видимости

```yaoxiang
// Блочная область видимости
{
    x = 10
    // x виден в этой области видимости
}
// x недоступен за пределами этой области видимости

// Область видимости функции
add: (a: Int, b: Int) -> Int = {
    result = a + b
    return result
}
// result недоступен за пределами функции
```

### 4.3 Объявление переменных и затенение

В YaoXiang нет ключевого слова `let`. Является ли `x = value` объявлением или присваиванием? Действует один принцип:

**Приоритет у присваивания.** Объявление выполняется лишь единожды, а присваивание — многократно. Пусть наиболее частая операция идёт по кратчайшему пути.

```
x = value:
  Поиск x по цепочке областей видимости наружу
    → Найден mut x          : присваивание, OK (через токен &mut)
    → Найден x (moved)      : считается «не найдено валидной привязки», повторное объявление в текущей области видимости
    → Найден x (неизменяемый, живой) : E2010 невозможно присвоить
    → Не найден             : новое объявление в текущей области видимости (единственный путь объявления)

mut x = value:
    → x уже существует в текущей области видимости : E2002 повторное определение
    → x существует во внешней области видимости   : E2013 запрещено затенение (явное новое объявление не может иметь то же имя, что и внешнее)
    → Нет конфликта         : новое изменяемое объявление
```

- **Одна область видимости**: любое имя может быть объявлено лишь однажды (E2002)
- **Во внутренней области нет `mut`**: сначала поиск во внешней области, присваивание или ошибка
- **Во внутренней области есть `mut`**: явное новое объявление, запрещено совпадение имени с внешней областью (E2013)

#### Одна область видимости

```yaoxiang
x = 10
x = 20              // E2002: 'x' уже определён в этой области видимости

mut y = 10
y = 20              // OK: та же привязка, повторное присваивание
mut y = 30          // E2002: 'y' уже определён в этой области видимости

z = 10
mut z = 20          // E2002: 'z' уже определён в этой области видимости (mut не может перезаписать существующее объявление)
```

#### Повторная привязка после move

Если неизменяемая переменная владеет ресурсом, и её значение было move (поглощено), исходная привязка переходит в состояние **moved** — имя по-прежнему занимает слот в области видимости, но значение становится недоступным. В этом случае `x = value` не является изменением старой привязки, а представляет собой повторное объявление `x` в той же области видимости.

```
Ветка «moved» при приоритете присваивания:
  x существует в текущей области видимости, но в состоянии moved
    → Компилятор считает «валидная привязка не найдена»
    → Повторное объявление x в текущей области видимости (перезаписывает старый слот moved)
```

**Ключевой механизм:** после поглощения старого значения привязка становится недействительной, и имя возвращается в состояние «объявляемое». Это не затенение — старой привязки уже не существует.

```yaoxiang
// Конвейерная обработка данных: каждый шаг поглощает старое значение и порождает новое
data = fetch()           // Неизменяемая, владеет ресурсом
data = transform(data)   // move data → старая data становится недействительной, новая data привязывается заново
data = filter(data)      // То же самое
process(data)

// Эквивалент в явной форме (для сравнения):
data1 = fetch()
data2 = transform(data1)  // data1 перемещён, повторное использование невозможно
data3 = filter(data2)     // data2 перемещён, повторное использование невозможно
process(data3)
```

**Разделение семантики:**

| Операция | Значение | Механизм | Синтаксис |
|------|------|------|------|
| **Повторная привязка** | Старое значение исчезает, новое значение рождается | move + повторное объявление | `x = f(x)` |
| **Изменение на месте** | Изменение значения в той же ячейке памяти | присваивание mut | `mut x; x = v` |

**Почему это отличается от затенения:**
- Затенение (`let x = ...` в Rust): старая привязка продолжает существовать, просто скрыта новой
- Повторная привязка после move: старая привязка уже поглощена, имя возвращается в неинициализированное состояние, повторное объявление — единственный выход

**Ограничения:**
- Только значения, владеющие ресурсом, могут быть перемещены (move). Ссылки (`&T`, `&mut T`) копируются, а не перемещаются
- Проверка move выполняется на этапе компиляции: чтение переменной в состоянии moved в любом выражении приводит к ошибке E2014
- IDE может отображать переменные в состоянии moved серым цветом, указывая, что имя находится в неинициализированном состоянии

```yaoxiang
// Чтение после move → ошибка
data = fetch()
result = process(data)   // data перемещён
print(data)              // E2014: 'data' был перемещён и не может быть использован

// Ссылки не вызывают move
ref_data = &value
copy1 = ref_data         // Копирование ссылки, ref_data по-прежнему доступна
copy2 = ref_data         // OK

// Состояние moved проникает через области видимости
data = fetch()
{
    data = transform(data)  // move внешнего data → повторная привязка (новое объявление во внутренней области)
    print(data)             // OK: используется внутренняя data
}
print(data)                 // E2014: внешняя data была перемещена
```

#### Между областями видимости

```yaoxiang
// Внешняя неизменяемая, внутреннее присваивание → неизменяемую переменную нельзя присвоить заново
x = 10
{
    x = 20          // E2010: 'x' неизменяем, невозможно присвоить заново
}
{
    mut x = 20      // E2013: невозможно затенить существующую переменную 'x' (явное объявление новой привязки)
}

// Внешняя mut, внутреннее присваивание → изменение той же привязки
mut y = 10
{
    y = 20          // OK: та же привязка, изменение через токен &mut
}
print(y)            // 20

// Внешняя mut, внутреннее объявление с тем же именем невозможно
mut z = 10
{
    z = 30          // OK: та же привязка
}
{
    mut z = 30      // E2013: невозможно затенить существующую переменную 'z'
}

// Многоуровневая вложенность: mut пронизывает все уровни
mut a = 0
{
    {
        a = 10      // OK
    }
}
print(a)            // 10

// Неизменяемость также пронизывает все уровни — переприсваивание невозможно
b = 0
{
    {
        b = 10      // E2010: 'b' неизменяем, невозможно присвоить заново
    }
}
```

#### Цикл `for`

```yaoxiang
// Переменная цикла на каждой итерации — новая привязка, а не модификация
for i in 1..5 {
    print(i)        // OK: каждая итерация привязывает новое значение
    i = 10          // E2010: неизменяемая переменная цикла, невозможно присвоить заново
}

for mut i in 1..5 {
    i = 10          // OK: изменяемая переменная цикла
}

// Переменная цикла не может затенять внешнюю
i = 0
for i in 1..5 {     // E2013: невозможно затенить существующую переменную 'i'
}

// Изменяемый внешний аккумулятор можно модифицировать в теле цикла
mut sum = 0
for i in 1..5 {
    sum = sum + i   // OK: та же привязка, изменение через токен &mut
}
print(sum)          // 15

// Неизменяемый внешний аккумулятор нельзя модифицировать в теле цикла
sum2 = 0
for i in 1..5 {
    sum2 = sum2 + i // E2010: 'sum2' неизменяем, невозможно присвоить заново
}
```

#### Связанные коды ошибок

| Код | Сообщение | Сценарий |
|--------|------|----------|
| E2002 | `'{name}' is already defined in this scope` | Повторное объявление в одной области видимости (независимо от mut) |
| E2010 | `Cannot assign to immutable variable '{name}'` | Внутреннее присваивание без `mut`, когда внешняя переменная неизменяема и не moved |
| E2013 | `Cannot shadow existing variable '{name}'` | Внутреннее явное объявление (`mut x` или `x: Type`) с тем же именем, что и внешнее |
| E2014 | `'{name}' has been moved and cannot be used` | Чтение переменной, находящейся в состоянии moved |

## Глава V: Организация модулей

### 5.1 Структура каталогов

```
src/
├── main.yx          // Главный модуль
├── math/
│   ├── index.yx     // Точка входа модуля math
│   ├── vector.yx    // Модуль векторов
│   └── matrix.yx    // Модуль матриц
└── utils/
    ├── index.yx     // Точка входа модуля utils
    └── string.yx    // Утилиты для строк
```

### 5.2 Точка входа модуля

Файл `index.yx` в каталоге служит точкой входа модуля:

```yaoxiang
// math/index.yx
use math.vector
use math.matrix

pub Vector = vector.Vector
pub Matrix = matrix.Matrix
```

### 5.3 Относительный импорт

```yaoxiang
// В файле math/vector.yx
use math.matrix  // Абсолютный импорт
use .matrix      // Относительный импорт (тот же каталог)
```
