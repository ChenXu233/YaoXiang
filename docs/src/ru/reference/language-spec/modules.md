# Спецификация системы модулей

В данном документе определяется спецификация системы модулей языка программирования YaoXiang, включая определение модулей, импорт/экспорт и области видимости.

---

## Глава 1: Определение модулей

### 1.1 Основы модулей

Модули ограничиваются файлами. Каждый файл `.yx` является модулем.

```
// Имя файла является именем модуля
// Math.yx
pub pi: Float = 3.14159
pub sqrt(Float) -> Float = (x) => { ... }
```

### 1.2 Правила именования модулей

- Имя модуля определяется именем файла
- Расширение файла `.yx` не входит в имя модуля
- Имена модулей используют PascalCase

## Глава 2: Импорт модулей

### 2.1 Синтаксис импорта

```
Import       ::= 'use' ModuleRef ImportSpec?
ImportSpec   ::= ('{' ImportItems '}') ('as' AliasList)?
              |  'as' AliasList
ImportItems  ::= Identifier (',' Identifier)* ','?
AliasList    ::= Identifier (',' Identifier)*
```

### 2.2 Способы импорта

| Синтаксис | Описание | Пример |
|-----------|----------|--------|
| `use path;` | Импорт модуля, доступ через последнюю часть | `use std.io;` -> `io.print` |
| `use path.{a, b};` | Импорт указанных элементов | `use std.io.{print};` -> `print` |
| `use path as alias;` | Импорт с переименованием | `use std.io as io;` -> `io.print` |
| `use path.{i1, i2} as a, b;` | Импорт указанных элементов с переименованием | `use std.io.{print, read} as p, r;` -> `p`, `r` |

### 2.3 Примеры импорта

```yaoxiang
// Импорт всего модуля
use std.io
io.print("Hello")

// Импорт указанных элементов
use std.io.{print, read}
print("Hello")

// Импорт с переименованием
use std.io as io_module
io_module.print("Hello")

// Импорт указанных элементов с переименованием
use std.io.{print, read} as p, r
p("Hello")
```

## Глава 3: Экспорт модулей

### 3.1 Ключевое слово pub

Для объявления экспортируемых элементов используется ключевое слово `pub`:

```yaoxiang
// Math.yx
pub pi: Float = 3.14159
pub sqrt: (x: Float) -> Float = (x) => { ... }

// Приватные элементы (не экспортируются)
internal_value: Int = 42
```

### 3.2 Правила экспорта

- По умолчанию все элементы являются приватными
- Элементы, объявленные с `pub`, доступны другим модулям
- Приватные элементы доступны только внутри текущего модуля

### 3.3 Автоматическая привязка pub

Для функций, объявленных с `pub`, компилятор автоматически привязывает их к типам, определённым в том же файле:

```yaoxiang
// Объявление с pub, компилятор выполняет автоматическую привязку
pub distance: (p1: Point, p2: Point) -> Float = {
    dx = p1.x - p2.x
    dy = p1.y - p2.y
    (dx * dx + dy * dy).sqrt()
}

// Компилятор автоматически выводит:
// 1. Point определён в текущем файле
// 2. Параметры функции содержат Point
// 3. Выполняется Point.distance = distance[0]

// Вызовы
d = distance(p1, p2)           // Функциональный стиль
d2 = p1.distance(p2)           // ООП-синтаксис
```

## Глава 4: Область видимости

### 4.1 Область видимости модуля

Каждый модуль имеет собственную область видимости; элементы внутри модуля по умолчанию недоступны извне.

### 4.2 Вложенная область видимости

```yaoxiang
// Блочная область видимости
{
    x = 10
    // x видим в этой области видимости
}
// x невидим за пределами этой области видимости

// Область видимости функции
add: (a: Int, b: Int) -> Int = {
    result = a + b
    return result
}
// result невидим за пределами функции
```

### 4.3 Объявление переменных и затенение

В YaoXiang нет ключевого слова `let`. Что означает `x = value` — объявление или присваивание? Действует принцип:

**Присваивание имеет приоритет.** Объявление происходит один раз, а присваивание — сотни раз. Пусть операция с высокой частотой идёт по кратчайшему пути.

```
x = value:
  Искать x по цепочке областей видимости
    → Найден mut x    : присваивание, OK (через токен &mut)
    → Найден x (неизменяемый): E2010 Невозможно переприсвоить
    → Не найден       : новое объявление в текущей области (единственный путь объявления)

mut x = value:
    → x уже существует в текущей области : E2002 Повторное определение
    → x существует во внешней области    : E2013 Затенение запрещено (явное новое объявление не может совпадать с внешним)
    → Конфликтов нет                      : новое изменяемое объявление
```

- **Одна область видимости**: любое имя может быть объявлено только один раз (E2002)
- **Во внутренней области без `mut`**: приоритетный поиск во внешней области, присваивание или ошибка
- **Во внутренней области с `mut`**: явное новое объявление, запрещено совпадать с внешним (E2013)

#### Одна область видимости

```yaoxiang
x = 10
x = 20              // E2002: 'x' уже определён в этой области видимости

mut y = 10
y = 20              // OK: та же привязка, переприсваивание
mut y = 30          // E2002: 'y' уже определён в этой области видимости

z = 10
mut z = 20          // E2002: 'z' уже определён в этой области видимости (mut не может перекрыть существующее объявление)
```

#### Межобластная видимость

```yaoxiang
// Внешняя область неизменяемая, внутренняя присваивает → нельзя переприсвоить неизменяемую переменную
x = 10
{
    x = 20          // E2010: 'x' неизменяемый, нельзя переприсвоить
}
{
    mut x = 20      // E2013: нельзя затенить существующую переменную 'x' (явное объявление новой привязки)
}

// Внешняя mut, внутренняя присваивает → модификация той же привязки
mut y = 10
{
    y = 20          // OK: та же привязка, модификация через токен &mut
}
print(y)            // 20

// Внешняя mut, внутренняя не может объявить одноимённую
mut z = 10
{
    z = 30          // OK: та же привязка
}
{
    mut z = 30      // E2013: нельзя затенить существующую переменную 'z'
}

// Многоуровневое вложение: mut проходит через все уровни
mut a = 0
{
    {
        a = 10      // OK
    }
}
print(a)            // 10

// Неизменяемость проходит через все уровни и не может быть переприсвоена
b = 0
{
    {
        b = 10      // E2010: 'b' неизменяемый, нельзя переприсвоить
    }
}
```

#### Цикл for

```yaoxiang
// Переменная цикла при каждой итерации — новая привязка, не модификация
for i in 1..5 {
    print(i)        // OK: каждая итерация связывает новое значение
    i = 10          // E2010: неизменяемая переменная цикла, нельзя переприсвоить
}

for mut i in 1..5 {
    i = 10          // OK: изменяемая переменная цикла
}

// Переменная цикла не может затенить внешнюю
i = 0
for i in 1..5 {     // E2013: нельзя затенить существующую переменную 'i'
}

// mut внешний аккумулятор можно модифицировать в теле цикла
mut sum = 0
for i in 1..5 {
    sum = sum + i   // OK: та же привязка, модификация через токен &mut
}
print(sum)          // 15

// Неизменяемый внешний нельзя модифицировать в теле цикла
sum2 = 0
for i in 1..5 {
    sum2 = sum2 + i // E2010: 'sum2' неизменяемый, нельзя переприсвоить
}
```

#### Коды связанных ошибок

| Код ошибки | Сообщение | Условие возникновения |
|------------|-----------|----------------------|
| E2002 | `'{name}' is already defined in this scope` | Повторное объявление в одной области видимости (независимо от mut) |
| E2010 | `Cannot assign to immutable variable '{name}'` | Присваивание во внутренней области без `mut`, когда внешняя переменная неизменяема |
| E2013 | `Cannot shadow existing variable '{name}'` | Явное объявление во внутренней области (`mut x` или `x: Type`) совпадает с внешним именем |

## Глава 5: Организация модулей

### 5.1 Структура каталогов

```
src/
├── main.yx          // Главный модуль
├── math/
│   ├── index.yx     // Точка входа математического модуля
│   ├── vector.yx    // Модуль векторов
│   └── matrix.yx    // Модуль матриц
└── utils/
    ├── index.yx     // Точка входа модуля утилит
    └── string.yx    // Утилиты для строк
```

### 5.2 Точка входа модуля

Файл `index.yx` в каталоге служит точкой входа модуля:

```yaoxiang
// math/index.yx
use math.vector
use math.matrix

pub Vector = vector.Vector
pub Matrix = matrix.Matrix
```

### 5.3 Относительный импорт

```yaoxiang
// В math/vector.yx
use math.matrix  // Абсолютный импорт
use .matrix      // Относительный импорт (тот же каталог)
```
