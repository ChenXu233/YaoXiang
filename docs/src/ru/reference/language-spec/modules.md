# Спецификация системы модулей

В данном документе определяется спецификация системы модулей языка программирования YaoXiang, включая определение модулей, импорт/экспорт и области видимости.

---

## Глава 1: Определение модулей

### 1.1 Основы модулей

Модули граничат файлами. Каждый файл `.yx` является модулем.

```
// Имя файла является именем модуля
// Math.yx
pub pi: Float = 3.14159
pub sqrt(Float) -> Float = (x) => { ... }
```

### 1.2 Правила именования модулей

- Имя модуля определяется именем файла
- Расширение файла `.yx` не входит в имя модуля
- Имена модулей используют PascalCase

## Глава 2: Импорт модулей

### 2.1 Синтаксис импорта

```
Import       ::= 'use' ModuleRef ImportSpec?
ImportSpec   ::= ('{' ImportItems '}') ('as' AliasList)?
              |  'as' AliasList
ImportItems  ::= Identifier (',' Identifier)* ','?
AliasList    ::= Identifier (',' Identifier)*
```

### 2.2 Способы импорта

| Синтаксис | Описание | Пример |
|------|------|------|
| `use path;` | Импорт модуля, доступ через последнюю часть | `use std.io;` -> `io.print` |
| `use path.{a, b};` | Импорт указанных элементов | `use std.io.{print};` -> `print` |
| `use path as alias;` | Импорт с переименованием | `use std.io as io;` -> `io.print` |
| `use path.{i1, i2} as a, b;` | Импорт указанных элементов с переименованием | `use std.io.{print, read} as p, r;` -> `p`, `r` |

### 2.3 Примеры импорта

```yaoxiang
// Импорт всего модуля
use std.io
io.print("Hello")

// Импорт указанных элементов
use std.io.{print, read}
print("Hello")

// Импорт с переименованием
use std.io as io_module
io_module.print("Hello")

// Импорт указанных элементов с переименованием
use std.io.{print, read} as p, r
p("Hello")
```

## Глава 3: Экспорт модулей

### 3.1 Ключевое слово pub

Для объявления экспортируемых элементов используется ключевое слово `pub`:

```yaoxiang
// Math.yx
pub pi: Float = 3.14159
pub sqrt: (x: Float) -> Float = (x) => { ... }

// Приватные элементы (не экспортируются)
internal_value: Int = 42
```

### 3.2 Правила экспорта

- По умолчанию все элементы являются приватными
- Элементы, объявленные с `pub`, доступны из других модулей
- Приватные элементы доступны только внутри текущего модуля

### 3.3 Автоматическое связывание pub

Для функций, объявленных с `pub`, компилятор автоматически привязывает их к типам, определённым в том же файле:

```yaoxiang
// Использование pub, компилятор автоматически привязывает
pub distance: (p1: Point, p2: Point) -> Float = {
    dx = p1.x - p2.x
    dy = p1.y - p2.y
    (dx * dx + dy * dy).sqrt()
}

// Компилятор автоматически выводит:
// 1. Point определён в текущем файле
// 2. Функция принимает Point как параметр
// 3. Выполняется Point.distance = distance[0]

// Вызовы
d = distance(p1, p2)           // Функциональный стиль
d2 = p1.distance(p2)           // Синтаксический сахар ООП
```

## Глава 4: Области видимости

### 4.1 Область видимости модуля

Каждый модуль имеет собственную область видимости, элементы модуля по умолчанию недоступны извне.

### 4.2 Вложенные области видимости

```yaoxiang
// Блочная область видимости
{
    x = 10
    // x виден в этой области видимости
}
// x не виден за пределами этой области видимости

// Область видимости функции
add: (a: Int, b: Int) -> Int = {
    result = a + b
    return result
}
// result не виден за пределами функции
```

### 4.3 Правила затенения

```yaoxiang
// Затенение переменной
x = 10
x = 20  // Затеняет предыдущий x

// Затенение в области видимости
x = 10
{
    x = 20  // Затеняет внешний x
    // Здесь x равен 20
}
// Здесь x равен 10
```

## Глава 5: Организация модулей

### 5.1 Структура каталогов

```
src/
├── main.yx          // Главный модуль
├── math/
│   ├── index.yx     // Точка входа математического модуля
│   ├── vector.yx    // Модуль векторов
│   └── matrix.yx    // Модуль матриц
└── utils/
    ├── index.yx     // Точка входа модуля утилит
    └── string.yx    // Утилиты для работы со строками
```

### 5.2 Точка входа модуля

Файл `index.yx` в каталоге служит точкой входа модуля:

```yaoxiang
// math/index.yx
use math.vector
use math.matrix

pub Vector = vector.Vector
pub Matrix = matrix.Matrix
```

### 5.3 Относительный импорт

```yaoxiang
// В math/vector.yx
use math.matrix  // Абсолютный импорт
use .matrix      // Относительный импорт (тот же каталог)
```
