# Спецификация модели параллелизма

Настоящий документ определяет спецификацию модели параллелизма языка программирования YaoXiang, включая асинхронное программирование, примитивы параллелизма и модель памяти.

---

## Глава 1: Spawn-функции

### 1.1 Функции spawn (并作函数)

```
SpawnFn     ::= Identifier ':' FnType 'spawn' '=' Lambda
FnType      ::= '(' ParamTypes? ')' '->' TypeExpr ('@' Annotation)?
Annotation  ::= 'block' | 'eager'
```

**Аннотации функций**:

| Аннотация | Позиция | Поведение |
|-----------|---------|-----------|
| `@block` | После типа возврата | Отключает оптимизацию параллелизма, полностью последовательное выполнение |
| `@eager` | После типа возврата | Принудительное энергичное вычисление |

**Синтаксические примеры**:

```
// 并作函数：可并发执行
fetch_data: (url: String) -> JSON spawn = { ... }

// @block 同步函数：完全顺序执行
main: () -> Void @block = { ... }

// @eager 急切函数：立即执行
compute: (n: Int) -> Int @eager = { ... }
```

### 1.2 Spawn-блок (并作块)

Явно объявленная область параллелизма, задачи внутри блока выполняются с помощью spawn:

```
SpawnBlock  ::= '(' Pattern (',' Pattern)* ')' '=' 'spawn' '{' Expr (',' Expr)* '}'
```

**Примеры**:

```
// 并作块：显式并发
(result_a, result_b) = spawn {
    parse(fetch("url1")),
    parse(fetch("url2"))
}
```

### 1.3 Spawn-цикл (并作循环)

Цикл с данным параллелизмом, тело цикла выполняется параллельно для всех элементов данных:

```
SpawnFor    ::= Identifier '=' 'spawn' 'for' Identifier 'in' Expr '{' Expr '}'
```

**Пример**:

```
// 并作循环：数据并行
results = spawn for item in items {
    process(item)
}
```

### 1.4 Оператор распространения ошибок (错误传播运算符)

```
ErrorPropagate ::= Expr '?'
```

**Пример**:

```
process: (p: Point) -> Result[Data, Error] = {
    data = fetch_data()?      // 自动传播错误
    transform(data)?
}
```

## Глава 2: Управление памятью (内存管理)

### 2.1 Модель владения (所有权模型)

YaoXiang использует **модель владения** для управления памятью, каждое значение имеет единственного владельца:

| Семантика | Описание | Синтаксис |
|-----------|----------|-----------|
| **Move** | Семантика по умолчанию, передача владения | `p2 = p` |
| **ref** | Совместное использование (подсчёт ссылок Arc) | `shared = ref p` |
| **clone()** | Явное копирование | `p2 = p.clone()` |

### 2.2 Семантика Move (默认)

```yaoxiang
// Присваивание = Move (нулевое копирование)
p: Point = Point(1.0, 2.0)
p2 = p              // Move, p становится недействительным

// Передача параметров в функцию = Move
process: (p: Point) -> Void = {
    // Владение p передаётся внутрь
}

// Возврат значения = Move
create: () -> Point = {
    p = Point(1.0, 2.0)
    return p        // Move, владение передаётся
}
```

### 2.3 Ключевое слово ref (Arc)

Ключевое слово `ref` создаёт **указатель с подсчётом ссылок** (Arc), используемый для безопасного совместного использования:

```yaoxiang
// Создание Arc
p: Point = Point(1.0, 2.0)
shared = ref p      // Arc, потокобезопасный

// Совместный доступ
spawn(() => print(shared.x))   // Безопасно

// Arc автоматически управляет жизненным циклом
// При выходе shared из области видимости счётчик становится нулём и происходит освобождение
```

**Особенности**:
- Потокобезопасный подсчёт ссылок
- Автоматическое управление жизненным циклом
- Безопасное пересечение границ spawn

### 2.4 Явное копирование clone() (clone() 显式复制)

```yaoxiang
// Явное копирование значения
p: Point = Point(1.0, 2.0)
p2 = p.clone()      // p и p2 независимы

// Оба могут быть изменены без влияния друг на друга
p.x = 0.0           // Корректно
p2.x = 0.0          // Корректно
```

### 2.5 Блоки unsafe (unsafe 代码块)

Блоки `unsafe` позволяют использовать голые указатели для системного программирования:

```yaoxiang
// Тип голого указателя
PtrType ::= '*' TypeExpr

// Блок unsafe
UnsafeBlock ::= 'unsafe' '{' Stmt* '}'
```

**Пример**:

```yaoxiang
p: Point = Point(1.0, 2.0)

// Голые указатели можно использовать только в блоках unsafe
unsafe {
    ptr: *Point = &p     // Получение голого указателя
    (*ptr).x = 0.0       // Разыменование
}
```

**Ограничения**:
- Голые указатели можно использовать только в блоках `unsafe`
- Пользователь гарантирует отсутствие висячих указателей и использования после освобождения
- Не участвуют в проверках Send/Sync

### 2.6 BNF синтаксиса владения

```bnf
// === Выражения владения ===

// Move (по умолчанию)
MoveExpr     ::= Expr

// ref Arc
RefExpr      ::= 'ref' Expr

// clone
CloneExpr    ::= Expr '.clone' '(' ')'

// === Голые указатели (только unsafe) ===

PtrType       ::= '*' TypeExpr
UnsafeBlock   ::= 'unsafe' '{' Stmt* '}'
```

## Глава 3: Безопасность параллелизма (并发安全)

### 3.1 Ограничения Send / Sync

| Ограничение | Семантика | Описание |
|-------------|-----------|----------|
| **Send** | Можно безопасно передавать между потоками | Значение можно переместить в другой поток |
| **Sync** | Можно безопасно совместно использовать между потоками | Неизменяемая ссылка может быть передана в другой поток |

**Автоматический вывод**:

```
// Правила вывода Send
Struct[T1, T2]: Send ⇐ T1: Send и T2: Send

// Правила вывода Sync
Struct[T1, T2]: Sync ⇐ T1: Sync и T2: Sync
```

**Ограничения типов**:

| Тип | Send | Sync | Описание |
|-----|------|------|----------|
| `T` (значение) | ✅ | ✅ | Неизменяемые данные |
| `ref T` | ✅ | ✅ | Arc потокобезопасен |
| `*T` | ❌ | ❌ | Голый указатель небезопасен |

### 3.2 Иерархия ограничений Send/Sync

```
Send ──► Можно безопасно передавать между потоками
  │
  └──► Sync ──► Можно безопасно совместно использовать между потоками
       │
       └──► Типы, удовлетворяющие Send + Sync, автоматически поддерживают параллелизм

Arc[T] реализует Send + Sync (потокобезопасный подсчёт ссылок)
Mutex[T] обеспечивает внутреннюю изменяемость
```

### 3.3 Потокобезопасные типы

| Тип | Семантика | Безопасность параллелизма | Описание |
|-----|-----------|---------------------------|----------|
| `T` | Неизменяемые данные | ✅ Безопасно | Тип по умолчанию, чтение несколькими задачами без гонок |
| `Ref[T]` | Изменяемая ссылка | ⚠️ Требуется синхронизация | Отмечен как допускающий параллельное изменение, компилятор проверяет использование блокировок |
| `Atomic[T]` | Атомарный тип | ✅ Безопасно | Низкоуровневые атомарные операции, безблокировочный параллелизм |
| `Mutex[T]` | Обёртка мьютекса | ✅ Безопасно | Автоматическая блокировка/разблокировка, гарантируется компилятором |
| `RwLock[T]` | Обёртка блокировки чтения-записи | ✅ Безопасно | Оптимизация для сценариев с большим количеством чтения и малым количеством записи |

**Синтаксис**:

```
Mutex[T]    // Обёртка изменяемых данных с мьютексом
Atomic[T]   // Атомарный тип (только для Int, Float и т.д.)
RwLock[T]   // Обёртка блокировки чтения-записи
```

**Синтаксический сахар with**:

```
with mutex.lock() {
    // Критическая секция: защищена Mutex
    ...
}
```

## Приложение: Краткий справочник синтаксиса параллелизма

### A.1 Синтаксис spawn

```yaoxiang
// Spawn-функция
fetch_data: (url: String) -> JSON spawn = { ... }

// Spawn-блок
(result_a, result_b) = spawn {
    parse(fetch("url1")),
    parse(fetch("url2"))
}

// Spawn-цикл
results = spawn for item in items {
    process(item)
}
```

### A.2 Синтаксис владения

```yaoxiang
// Move (по умолчанию)
p2 = p

// ref Arc
shared = ref p

// clone
p2 = p.clone()

// unsafe
unsafe {
    ptr: *Point = &p
    (*ptr).x = 0.0
}
```

### A.3 Потокобезопасные типы

```yaoxiang
// Мьютекс
mutex: Mutex[Int] = Mutex(0)
with mutex.lock() {
    // Критическая секция
}

// Атомарный тип
counter: Atomic[Int] = Atomic(0)
counter.increment()

// Блокировка чтения-записи
data: RwLock[Data] = RwLock(data)
with data.read() {
    // Операция чтения
}
```
