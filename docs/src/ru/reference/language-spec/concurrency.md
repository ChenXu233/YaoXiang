# Спецификация модели параллелизма

В данном документе определяется спецификация модели параллелизма языка программирования YaoXiang, включая асинхронное программирование, примитивы параллелизма и модель памяти.

---

## Глава 1: Функции spawn

### 1.1 Функции spawn (функции параллельного выполнения)

```
SpawnFn     ::= Identifier ':' FnType 'spawn' '=' Lambda
FnType      ::= '(' ParamTypes? ')' '->' TypeExpr ('@' Annotation)?
Annotation  ::= 'block' | 'eager'
```

**Аннотации функций**:

| Аннотация | Позиция | Поведение |
|-----------|---------|-----------|
| `@block` | После типа возврата | Отключает оптимизацию параллелизма, полностью последовательное выполнение |
| `@eager` | После типа возврата | Принудительное энергичное вычисление |

**Примеры синтаксиса**:

```
// Функция spawn: может выполняться параллельно
fetch_data: (url: String) -> JSON spawn = { ... }

// @block синхронная функция: полностью последовательное выполнение
main: () -> Void @block = { ... }

// @eager энергичная функция: немедленное выполнение
compute: (n: Int) -> Int @eager = { ... }
```

### 1.2 Блок spawn

Явно объявленная область параллелизма, задачи внутри блока выполняются параллельно:

```
SpawnBlock  ::= '(' Pattern (',' Pattern)* ')' '=' 'spawn' '{' Expr (',' Expr)* '}'
```

**Пример**:

```
// Блок spawn: явный параллелизм
(result_a, result_b) = spawn {
    parse(fetch("url1")),
    parse(fetch("url2"))
}
```

### 1.3 Цикл spawn

Цикл с data-level параллелизмом, тело цикла выполняется параллельно над всеми элементами данных:

```
SpawnFor    ::= Identifier '=' 'spawn' 'for' Identifier 'in' Expr '{' Expr '}'
```

**Пример**:

```
// Цикл spawn: data-level параллелизм
results = spawn for item in items {
    process(item)
}
```

### 1.4 Оператор распространения ошибок

```
ErrorPropagate ::= Expr '?'
```

**Пример**:

```
process: (p: Point) -> Result[Data, Error] = {
    data = fetch_data()?      // Автоматическое распространение ошибки
    transform(data)?
}
```

## Глава 2: Управление памятью

### 2.1 Модель владения

YaoXiang использует **модель владения** для управления памятью, каждое значение имеет единственного владельца:

| Семантика | Описание | Синтаксис |
|-----------|----------|-----------|
| **Move** | Семантика по умолчанию, передача владения | `p2 = p` |
| **ref** | Совместное использование (Arc с подсчётом ссылок) | `shared = ref p` |
| **clone()** | Явное копирование | `p2 = p.clone()` |

### 2.2 Семантика Move (по умолчанию)

```yaoxiang
// Присваивание = Move (нулевое копирование)
p: Point = Point(1.0, 2.0)
p2 = p              // Move, p становится недействительным

// Передача аргумента в функцию = Move
process: (p: Point) -> Void = {
    // Владение p передаётся внутрь
}

// Возврат значения = Move
create: () -> Point = {
    p = Point(1.0, 2.0)
    return p        // Move, владение передаётся
}
```

### 2.3 Ключевое слово ref (Arc)

Ключевое слово `ref` создаёт **указатель с подсчётом ссылок** (Arc) для безопасного совместного использования:

```yaoxiang
// Создание Arc
p: Point = Point(1.0, 2.0)
shared = ref p      // Arc, потокобезопасный

// Совместный доступ
spawn(() => print(shared.x))   // Безопасно

// Arc автоматически управляет временем жизни
// При выходе shared из области видимости счётчик становится нулём и происходит освобождение
```

**Особенности**:
- Потокобезопасный подсчёт ссылок
- Автоматическое управление временем жизни
- Безопасное использование за границами spawn

### 2.4 Явное копирование через clone()

```yaoxiang
// Явное копирование значения
p: Point = Point(1.0, 2.0)
p2 = p.clone()      // p и p2 независимы

// Оба могут быть изменены без взаимного влияния
p.x = 0.0           // Корректно
p2.x = 0.0          // Корректно
```

### 2.5 Блоки unsafe

Блок `unsafe` позволяет использовать голые указатели для системного программирования:

```yaoxiang
// Тип голого указателя
PtrType ::= '*' TypeExpr

// Блок unsafe
UnsafeBlock ::= 'unsafe' '{' Stmt* '}'
```

**Пример**:

```yaoxiang
p: Point = Point(1.0, 2.0)

// Голые указатели можно использовать только в блоке unsafe
unsafe {
    ptr: *Point = &p     // Получение голого указателя
    (*ptr).x = 0.0       // Разыменование
}
```

**Ограничения**:
- Голые указатели можно использовать только в блоках `unsafe`
- Пользователь гарантирует отсутствие висячих указателей и использования после освобождения
- Не участвуют в проверках Send/Sync

### 2.6 BNF синтаксиса владения

```bnf
// === Выражения владения ===

// Move (по умолчанию)
MoveExpr     ::= Expr

// ref Arc
RefExpr      ::= 'ref' Expr

// clone
CloneExpr    ::= Expr '.clone' '(' ')'

// === Голые указатели (только unsafe) ===

PtrType       ::= '*' TypeExpr
UnsafeBlock   ::= 'unsafe' '{' Stmt* '}'
```

## Глава 3: Безопасность параллелизма

### 3.1 Ограничения Send / Sync

| Ограничение | Семантика | Описание |
|-------------|-----------|----------|
| **Send** | Безопасная межпотоковая передача | Значение можно переместить в другой поток |
| **Sync** | Безопасное межпотоковое совместное использование | Неизменяемая ссылка может быть передана в другой поток |

**Автоматический вывод**:

```
// Правила вывода Send
Struct[T1, T2]: Send ⇐ T1: Send 且 T2: Send

// Правила вывода Sync
Struct[T1, T2]: Sync ⇐ T1: Sync 且 T2: Sync
```

**Ограничения типов**:

| Тип | Send | Sync | Описание |
|-----|------|------|----------|
| `T` (значение) | ✅ | ✅ | Неизменяемые данные |
| `ref T` | ✅ | ✅ | Arc потокобезопасен |
| `*T` | ❌ | ❌ | Голый указатель небезопасен |

### 3.2 Иерархия ограничений Send/Sync

```
Send ──► Безопасная межпотоковая передача
  │
  └──► Sync ──► Безопасное межпотоковое совместное использование
       │
       └──► Типы, удовлетворяющие Send + Sync, автоматически поддерживают параллелизм

Arc[T] реализует Send + Sync (потокобезопасный подсчёт ссылок)
Mutex[T] обеспечивает внутреннюю изменяемость
```

### 3.3 Типы для безопасного параллелизма

| Тип | Семантика | Безопасность параллелизма | Описание |
|-----|-----------|---------------------------|----------|
| `T` | Неизменяемые данные | ✅ Безопасно | Тип по умолчанию, множественное чтение несколькими задачами без гонок |
| `Ref[T]` | Изменяемая ссылка | ⚠️ Требуется синхронизация | Помечен как изменяемый для параллелизма, компилятор проверяет использование блокировок |
| `Atomic[T]` | Атомарный тип | ✅ Безопасно | Низкоуровневые атомарные операции, безблокировочный параллелизм |
| `Mutex[T]` | Обёртка мьютекса | ✅ Безопасно | Автоматическая блокировка/разблокировка, гарантируется компилятором |
| `RwLock[T]` | Обёртка读写锁 | ✅ Безопасно | Оптимизация для сценариев с большим количеством чтения и малым количеством записи |

**Синтаксис**:

```
Mutex[T]    // Обёртка мьютекса над изменяемыми данными
Atomic[T]   // Атомарный тип (только для Int, Float и т.д.)
RwLock[T]   // Обёртка读写锁
```

**Синтаксический сахар with**:

```
with mutex.lock() {
    // Критическая секция: защищена Mutex
    ...
}
```

## Приложение: Краткий справочник синтаксиса параллелизма

### A.1 Синтаксис spawn

```yaoxiang
// Функция spawn
fetch_data: (url: String) -> JSON spawn = { ... }

// Блок spawn
(result_a, result_b) = spawn {
    parse(fetch("url1")),
    parse(fetch("url2"))
}

// Цикл spawn
results = spawn for item in items {
    process(item)
}
```

### A.2 Синтаксис владения

```yaoxiang
// Move (по умолчанию)
p2 = p

// ref Arc
shared = ref p

// clone
p2 = p.clone()

// unsafe
unsafe {
    ptr: *Point = &p
    (*ptr).x = 0.0
}
```

### A.3 Типы безопасного параллелизма

```yaoxiang
// Мьютекс
mutex: Mutex[Int] = Mutex(0)
with mutex.lock() {
    // Критическая секция
}

// Атомарный тип
counter: Atomic[Int] = Atomic(0)
counter.increment()

// Читатель-писатель锁
data: RwLock[Data] = RwLock(data)
with data.read() {
    // Операция чтения
}
```
