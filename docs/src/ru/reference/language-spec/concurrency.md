# Спецификация модели параллелизма

Настоящий документ определяет спецификацию модели параллелизма языка программирования YaoXiang, включая асинхронное программирование, примитивы параллелизма и модель памяти.

---

## Глава 1: Функции spawn

### 1.1 Функции spawn

```
SpawnFn     ::= Identifier ':' FnType 'spawn' '=' Lambda
FnType      ::= '(' ParamTypes? ')' '->' TypeExpr ('@' Annotation)?
Annotation  ::= 'block' | 'eager'
```

**Аннотации функций**:

| Аннотация | Позиция | Поведение |
|-----------|---------|-----------|
| `@block` | После типа возврата | Отключает оптимизацию параллелизма, полностью последовательное выполнение |
| `@eager` | После типа возврата | Принудительное энергичное вычисление |

**Примеры синтаксиса**:

```
// Функция spawn: потенциально параллельное выполнение
fetch_data: (url: String) -> JSON spawn = { ... }

// Синхронная функция @block: полностью последовательное выполнение
main: () -> Void @block = { ... }

// Функция @eager: немедленное выполнение
compute: (n: Int) -> Int @eager = { ... }
```

### 1.2 Блоки spawn

Явно объявленная область параллелизма, задачи внутри блока выполняются через spawn:

```
SpawnBlock  ::= '(' Pattern (',' Pattern)* ')' '=' 'spawn' '{' Expr (',' Expr)* '}'
```

**Примеры**:

```
// Блок spawn: явный параллелизм
(result_a, result_b) = spawn {
    parse(fetch("url1")),
    parse(fetch("url2"))
}
```

### 1.3 Циклы spawn

Цикл для данных-параллелизма, тело цикла выполняется параллельно для всех элементов данных:

```
SpawnFor    ::= Identifier '=' 'spawn' 'for' Identifier 'in' Expr '{' Expr '}'
```

**Примеры**:

```
// Цикл spawn: данные-параллелизм
results = spawn for item in items {
    process(item)
}
```

### 1.4 Оператор распространения ошибок

```
ErrorPropagate ::= Expr '?'
```

**Примеры**:

```
process: (p: Point) -> Result[Data, Error] = {
    data = fetch_data()?      // Автоматическое распространение ошибки
    transform(data)?
}
```

## Глава 2: Управление памятью

### 2.1 Модель владения

YaoXiang использует **модель владения** для управления памятью, каждый объект имеет единственного владельца:

| Семантика | Описание | Синтаксис |
|-----------|----------|-----------|
| **Move** | Семантика по умолчанию, передача владения | `p2 = p` |
| **ref** | Разделение (Arc с подсчётом ссылок) | `shared = ref p` |
| **clone()** | Явное копирование | `p2 = p.clone()` |

### 2.2 Семантика Move (по умолчанию)

```yaoxiang
// Присваивание = Move (нулевое копирование)
p: Point = Point(1.0, 2.0)
p2 = p              // Move, p становится недействительным

// Передача аргумента = Move
process: (p: Point) -> Void = {
    // Владение p передаётся
}

// Возврат значения = Move
create: () -> Point = {
    p = Point(1.0, 2.0)
    return p        // Move, владение передаётся
}
```

### 2.3 Ключевое слово ref (Arc)

Ключевое слово `ref` создаёт **указатель с подсчётом ссылок** (Arc) для безопасного разделения:

```yaoxiang
// Создание Arc
p: Point = Point(1.0, 2.0)
shared = ref p      // Arc, потокобезопасный

// Разделённый доступ
spawn(() => print(shared.x))   // Безопасно

// Arc автоматически управляет жизненным циклом
// При выходе shared из области видимости счётчик обнуляется и происходит освобождение
```

**Характеристики**:
- Потокобезопасный подсчёт ссылок
- Автоматическое управление жизненным циклом
- Безопасность при пересечении границ spawn

### 2.4 clone() Явное копирование

```yaoxiang
// Явное копирование значения
p: Point = Point(1.0, 2.0)
p2 = p.clone()      // p и p2 независимы

// Оба могут быть изменены без влияния друг на друга
p.x = 0.0           // Корректно
p2.x = 0.0          // Корректно
```

### 2.5 Блоки unsafe

Блоки `unsafe` позволяют использовать сырые указатели для системного программирования:

```yaoxiang
// Тип сырого указателя
PtrType ::= '*' TypeExpr

// Блок unsafe
UnsafeBlock ::= 'unsafe' '{' Stmt* '}'
```

**Примеры**:

```yaoxiang
p: Point = Point(1.0, 2.0)

// Сырые указатели можно использовать только в блоке unsafe
unsafe {
    ptr: *Point = &p     // Получение сырого указателя
    (*ptr).x = 0.0       // Разыменование
}
```

**Ограничения**:
- Сырые указатели можно использовать только в блоках `unsafe`
- Пользователь гарантирует отсутствие висячих указателей и использования после освобождения
- Не участвуют в проверках Send/Sync

### 2.6 BNF синтаксиса владения

```bnf
// === Выражения владения ===

// Move (по умолчанию)
MoveExpr     ::= Expr

// ref Arc
RefExpr      ::= 'ref' Expr

// clone
CloneExpr    ::= Expr '.clone' '(' ')'

// === Сырые указатели (только unsafe) ===

PtrType       ::= '*' TypeExpr
UnsafeBlock   ::= 'unsafe' '{' Stmt* '}'
```

## Глава 3: Безопасность параллелизма

### 3.1 Ограничения Send / Sync

| Ограничение | Семантика | Описание |
|-------------|-----------|----------|
| **Send** | Безопасная передача между потоками | Значение можно переместить в другой поток |
| **Sync** | Безопасное разделение между потоками | Неизменяемая ссылка может быть разделена в другой поток |

**Автоматический вывод**:

```
// Правила вывода Send
Struct[T1, T2]: Send ⇐ T1: Send и T2: Send

// Правила вывода Sync
Struct[T1, T2]: Sync ⇐ T1: Sync и T2: Sync
```

**Ограничения типов**:

| Тип | Send | Sync | Описание |
|-----|------|------|----------|
| `T` (значение) | ✅ | ✅ | Неизменяемые данные |
| `ref T` | ✅ | ✅ | Arc потокобезопасный |
| `*T` | ❌ | ❌ | Сырой указатель небезопасен |

### 3.2 Иерархия ограничений Send/Sync

```
Send ──► Безопасная передача между потоками
  │
  └──► Sync ──► Безопасное разделение между потоками
       │
       └──► Типы, удовлетворяющие Send + Sync, автоматически параллельны

Arc[T] реализует Send + Sync (потокобезопасный подсчёт ссылок)
Mutex[T] обеспечивает внутреннюю изменяемость
```

### 3.3 Потокобезопасные типы

| Тип | Семантика | Параллельно безопасен | Описание |
|-----|-----------|----------------------|----------|
| `T` | Неизменяемые данные | ✅ Безопасно | Тип по умолчанию, чтение несколькими задачами без гонок |
| `Ref[T]` | Изменяемая ссылка | ⚠️ Требуется синхронизация | Помечен как изменяемый несколькими задачами, компилятор проверяет использование блокировок |
| `Atomic[T]` | Атомарный тип | ✅ Безопасно | Низкоуровневые атомарные операции, безблокировочный параллелизм |
| `Mutex[T]` | Обёртка мьютекса | ✅ Безопасно | Автоматическая блокировка/разблокировка, гарантируется компилятором |
| `RwLock[T]` | Обёртка блокировки чтения-записи | ✅ Безопасно | Оптимизация для сценариев с частыми чтениями и редкими записями |

**Синтаксис**:

```
Mutex[T]    // Обёртка мьютекса для изменяемых данных
Atomic[T]   // Атомарный тип (только Int, Float и т.д.)
RwLock[T]   // Обёртка блокировки чтения-записи
```

**Синтаксический сахар with**:

```
with mutex.lock() {
    // Критическая секция: защищена Mutex
    ...
}
```

## Приложение: Краткий справочник по синтаксису параллелизма

### А.1 Синтаксис spawn

```yaoxiang
// Функция spawn
fetch_data: (url: String) -> JSON spawn = { ... }

// Блок spawn
(result_a, result_b) = spawn {
    parse(fetch("url1")),
    parse(fetch("url2"))
}

// Цикл spawn
results = spawn for item in items {
    process(item)
}
```

### А.2 Синтаксис владения

```yaoxiang
// Move (по умолчанию)
p2 = p

// ref Arc
shared = ref p

// clone
p2 = p.clone()

// unsafe
unsafe {
    ptr: *Point = &p
    (*ptr).x = 0.0
}
```

### А.3 Потокобезопасные типы

```yaoxiang
// Мьютекс
mutex: Mutex[Int] = Mutex(0)
with mutex.lock() {
    // Критическая секция
}

// Атомарный тип
counter: Atomic[Int] = Atomic(0)
counter.increment()

// Блокировка чтения-записи
data: RwLock[Data] = RwLock(data)
with data.read() {
    // Операция чтения
}
```
