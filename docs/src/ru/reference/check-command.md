# yaoxiang check

Выполнение статической проверки исходного кода YaoXiang (проверка типов, проверка владения), без генерации кода.

## Использование

```
yaoxiang check [OPTIONS] [PATH]...
```

## Аргументы

| Аргумент | Описание |
|------|------|
| `PATH` | Один или несколько путей к файлам или каталогам. Если не указан, проверяется текущий проект. |

## Опции

| Опция | Описание | Значение по умолчанию |
|------|------|--------|
| `--json` | Вывод диагностической информации в формате JSON | Нет |
| `-w`, `--watch` | Отслеживание изменений файлов и автоматическая повторная проверка | Нет |
| `--color <MODE>` | Режим цветного вывода: `auto`, `always`, `never` | `auto` |
| `--exclude <PATH>` | Исключить указанный путь (можно использовать несколько раз) | Нет |
| `--no-progress` | Подавить сообщения о ходе выполнения и сводке | Нет |

## Коды завершения

| Код завершения | Описание |
|--------|------|
| `0` | Нет ошибок |
| `1` | Обнаружены ошибки при проверке |
| `2` | Файлы `.yx` не найдены |

## Межфайловый анализ

`yaoxiang check` поддерживает проверку типов между файлами. При проверке нескольких файлов:

1. Параллельный парсинг всех файлов `.yx`
2. Построение графа зависимостей модулей
3. Обнаружение циклических зависимостей (ошибка)
4. Проверка в порядке топологической сортировки
5. Использование общей среды типов для корректной проверки межфайловых ссылок

```bash
# Проверить весь проект (автоопределение межфайловых ссылок)
yaoxiang check src/

# Проверить указанные файлы
yaoxiang check src/main.yx src/lib.yx
```

## Инкрементная проверка (режим watch)

Используйте `-w` или `--watch` для включения режима отслеживания файлов. При изменении файлов автоматически выполняется повторная проверка.

```bash
yaoxiang check --watch
```

## Формат вывода JSON

При использовании `--json` формат вывода следующий:

```json
{
  "error_count": 0,
  "warning_count": 0,
  "diagnostics": [
    {
      "file": "src/main.yx",
      "severity": "error",
      "code": "E1001",
      "message": "Unknown variable: 'x'",
      "line": 5,
      "column": 3,
      "end_line": 5,
      "end_column": 4,
      "lsp": { ... }
    }
  ]
}
```

## Примеры

```bash
# Проверить текущий проект
yaoxiang check

# Проверить указанные файлы
yaoxiang check src/main.yx

# Проверить каталог и вывести JSON
yaoxiang check src/ --json

# Режим отслеживания
yaoxiang check --watch

# Режим CI (без цвета, без прогресса)
yaoxiang check --color never --no-progress

# Исключить каталог тестов
yaoxiang check src/ --exclude tests/
```

## Интеграция с CI

```yaml
# GitHub Actions
- name: Type check
  run: yaoxiang check --color never --no-progress
```

Подробная конфигурация CI описана в [Руководстве по интеграции CI](../guide/ci-integration.md).
