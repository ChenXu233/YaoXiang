# yaoxiang check

Выполняет статическую проверку исходного кода YaoXiang (проверка типов, проверка владения) без генерации какого-либо кода.

## Использование

```
yaoxiang check [OPTIONS] [PATH]...
```

## Аргументы

| Аргумент | Описание |
|------|------|
| `PATH` | Один или несколько путей к файлам или каталогам. Если не указан, проверяется текущий проект. |

## Параметры

| Параметр | Описание | Значение по умолчанию |
|------|------|--------|
| `--json` | Вывод диагностической информации в формате JSON | нет |
| `-w`, `--watch` | Отслеживать изменения файлов и автоматически перепроверять | нет |
| `--color <MODE>` | Режим цветного вывода: `auto`, `always`, `never` | `auto` |
| `--exclude <PATH>` | Исключить указанный путь (можно указывать несколько раз) | нет |
| `--no-progress` | Подавить сообщения о прогрессе и сводке | нет |

## Коды завершения

| Код завершения | Описание |
|--------|------|
| `0` | Ошибок нет |
| `1` | В ходе проверки обнаружены ошибки |
| `2` | Файлы `.yx` не найдены |

## Межфайловый анализ

`yaoxiang check` поддерживает межфайловую проверку типов. При проверке нескольких файлов:

1. Параллельно разбирает все файлы `.yx`
2. Строит граф зависимостей модулей
3. Обнаруживает циклические зависимости (с ошибкой)
4. Выполняет проверку в порядке топологической сортировки
5. Использует общее окружение типов для корректного обнаружения межфайловых ссылок

```bash
# Проверить весь проект (автоматическое обнаружение межфайловых ссылок)
yaoxiang check src/

# Проверить указанные файлы
yaoxiang check src/main.yx src/lib.yx
```

## Инкрементальная проверка (режим watch)

Используйте `-w` или `--watch` для включения режима наблюдения за файлами. При изменении файлов проверка запускается автоматически.

```bash
yaoxiang check --watch
```

## Формат вывода JSON

При использовании `--json` вывод имеет следующий формат:

```json
{
  "error_count": 0,
  "warning_count": 0,
  "diagnostics": [
    {
      "file": "src/main.yx",
      "severity": "error",
      "code": "E1001",
      "message": "Unknown variable: 'x'",
      "line": 5,
      "column": 3,
      "end_line": 5,
      "end_column": 4,
      "lsp": { ... }
    }
  ]
}
```

## Примеры

```bash
# Проверить текущий проект
yaoxiang check

# Проверить указанный файл
yaoxiang check src/main.yx

# Проверить каталог и вывести JSON
yaoxiang check src/ --json

# Режим наблюдения
yaoxiang check --watch

# Режим CI (без цвета, без прогресса)
yaoxiang check --color never --no-progress

# Исключить каталог тестов
yaoxiang check src/ --exclude tests/
```

## Интеграция с CI

```yaml
# GitHub Actions
- name: Type check
  run: yaoxiang check --color never --no-progress
```

Подробную конфигурацию CI см. в [Руководстве по интеграции с CI](../guide/ci-integration.md).
