# yaoxiang check

Выполнение статической проверки (type check, ownership check) исходного кода YaoXiang без генерации кода.

## Использование

```
yaoxiang check [OPTIONS] [PATH]...
```

## Аргументы

| Аргумент | Описание |
|----------|----------|
| `PATH` | Один или несколько путей к файлам или каталогам. Если не указано, проверяется текущий проект. |

## Опции

| Опция | Описание | Значение по умолчанию |
|-------|----------|------------------------|
| `--json` | Вывод диагностической информации в формате JSON | Нет |
| `-w`, `--watch` | Наблюдение за изменениями файлов и автоматическая повторная проверка | Нет |
| `--color <MODE>` | Режим цветного вывода: `auto`, `always`, `never` | `auto` |
| `--exclude <PATH>` | Исключить указанный путь (можно использовать несколько раз) | Нет |
| `--no-progress` | Скрыть сообщения о прогрессе и сводке | Нет |

## Коды завершения

| Код завершения | Описание |
|----------------|----------|
| `0` | Ошибок нет |
| `1` | При проверке обнаружены ошибки |
| `2` | Файлы `.yx` не найдены |

## Межфайловый анализ

`yaoxiang check` поддерживает межфайловый type check. При проверке нескольких файлов:

1. Параллельный парсинг всех файлов `.yx`
2. Построение графа зависимостей модулей
3. Обнаружение циклических зависимостей (ошибка)
4. Проверка в порядке топологической сортировки
5. Использование общей type environment для корректного обнаружения межфайловых ссылок

```bash
# Проверить весь проект (автоматическое обнаружение межфайловых ссылок)
yaoxiang check src/

# Проверить указанные файлы
yaoxiang check src/main.yx src/lib.yx
```

## Инкрементальная проверка (режим watch)

Используйте `-w` или `--watch` для включения режима наблюдения за файлами. При изменении файлов проверка выполняется автоматически повторно.

```bash
yaoxiang check --watch
```

## Формат вывода JSON

При использовании `--json` формат вывода следующий:

```json
{
  "error_count": 0,
  "warning_count": 0,
  "diagnostics": [
    {
      "file": "src/main.yx",
      "severity": "error",
      "code": "E1001",
      "message": "Unknown variable: 'x'",
      "line": 5,
      "column": 3,
      "end_line": 5,
      "end_column": 4,
      "lsp": { ... }
    }
  ]
}
```

## Примеры

```bash
# Проверить текущий проект
yaoxiang check

# Проверить указанные файлы
yaoxiang check src/main.yx

# Проверить каталог и вывести JSON
yaoxiang check src/ --json

# Режим наблюдения
yaoxiang check --watch

# CI-режим (без цвета, без прогресса)
yaoxiang check --color never --no-progress

# Исключить каталог тестов
yaoxiang check src/ --exclude tests/
```

## Интеграция с CI

```yaml
# GitHub Actions
- name: Type check
  run: yaoxiang check --color never --no-progress
```

Подробную конфигурацию CI см. в [Руководстве по интеграции CI](../guide/ci-integration.md).
