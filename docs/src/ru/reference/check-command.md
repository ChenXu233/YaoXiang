# yaoxiang check

Выполнение статической проверки исходного кода YaoXiang (проверка типов, проверка владения) без генерации кода.

## Использование

```
yaoxiang check [OPTIONS] [PATH]...
```

## Параметры

| Параметр | Описание |
|----------|----------|
| `PATH` | Один или несколько путей к файлам или каталогам. При отсутствии проверяется текущий проект. |

## Опции

| Опция | Описание | Значение по умолчанию |
|-------|----------|----------------------|
| `--json` | Вывод диагностической информации в формате JSON | Нет |
| `-w`, `--watch` | Наблюдение за изменениями файлов и автоматическая повторная проверка | Нет |
| `--color <MODE>` | Режим цветного вывода: `auto`, `always`, `never` | `auto` |
| `--exclude <PATH>` | Исключить указанный путь (можно использовать несколько раз) | Нет |
| `--no-progress` | Подавить сообщения о ходе выполнения и сводки | Нет |

## Коды завершения

| Код завершения | Описание |
|----------------|----------|
| `0` | Ошибок нет |
| `1` | Обнаружены ошибки при проверке |
| `2` | Файлы `.yx` не найдены |

## Формат JSON-вывода

При использовании `--json` формат вывода следующий:

```json
{
  "error_count": 0,
  "warning_count": 0,
  "diagnostics": [
    {
      "file": "src/main.yx",
      "severity": "error",
      "code": "E1001",
      "message": "Unknown variable: 'x'",
      "line": 5,
      "column": 3,
      "end_line": 5,
      "end_column": 4,
      "lsp": { ... }
    }
  ]
}
```

## Примеры

```bash
# Проверить текущий проект
yaoxiang check

# Проверить указанный файл
yaoxiang check src/main.yx

# Проверить каталог с выводом в JSON
yaoxiang check src/ --json

# Режим наблюдения
yaoxiang check --watch

# Режим CI (без цвета, без прогресса)
yaoxiang check --color never --no-progress

# Исключить каталог тестов
yaoxiang check src/ --exclude tests/
```

## Интеграция с CI

```yaml
# GitHub Actions
- name: Type check
  run: yaoxiang check --color never --no-progress
```
