# Дорожная карта реализации системы заимствования токенов

## Цель

Полная реализация RFC-009 v9 системы заимствования токенов вместо старого упрощённого заимствования.

## Цепочка зависимостей

```
RFC-009 v9 (дизайн токенов заимствования) ← Завершено
    │
    ├── 1. Система атрибутов типов [дизайн завершён → type-property-system-dup.md]
    │      ├── Определение и реализация трейта Dup
    │      ├── Рекурсивная проверка struct в trait solver
    │      └── Рекурсивная проверка полей в auto-derive
    │
    ├── 2. Модель захвата в замыканиях [дизайн завершён → closure-capture-model.md]
    │      ├── Анализ захвата переменных
    │      ├── Анализ побега
    │      └── Заполнение env в MakeClosure
    │
    └── 3. Реализация токенов заимствования [после завершения этапов 1, 2]
           ├── MonoType::Ref { mutable, inner }
           ├── borrow checker pass (middle/passes/lifetime/)
           ├── Обнаружение конфликтов токенов (потоково-чувствительный анализ активности)
           └── ZST оптимизация (токены исчезают после компиляции)
```

## Этапы

### Этап 1: Система атрибутов типов

**Статус**: Дизайн завершён → [type-property-system-dup.md](type-property-system-dup.md)

**Область действия**:
- Трейт Dup зарегистрирован как встроенный marker trait
- Примитивные типы автоматически помечены как Dup
- struct/перечисление/кортежи автоматически выводятся: все поля Dup → тип Dup
- trait solver поддерживает рекурсивную проверку struct/перечислений/кортежей
- auto-derive поддерживает поля обобщённых контейнеров (`List(Int)` и т.д.)
- Удаление Send/Sync (не видно пользователю, полностью автоматическая обработка компилятором)

**Связанные файлы**:
- `src/frontend/core/types/base/mono.rs`
- `src/frontend/core/typecheck/traits/std_traits.rs`
- `src/frontend/core/typecheck/traits/auto_derive.rs`
- `src/frontend/core/typecheck/traits/solver.rs`

### Этап 2: Модель захвата в замыканиях

**Статус**: Ожидает дизайна

**Область действия**:
- Анализ внешних переменных, на которые ссылается lambda при проверке типов
- Определение способа захвата для каждой переменной (токен заимствования vs Move)
- Заполнение env в MakeClosure при генерации IR
- Поддержка распространения токенов заимствования в замыканиях

### Этап 3: Реализация токенов заимствования

**Статус**: Ожидает завершения этапов 1, 2

**Область действия**:
- AST: `Type::Ref`, `Expr::Borrow`
- Лексер: токены `&` и `&mut`
- MonoType: `Ref { mutable, inner }`
- IR: инструкции заимствования (по необходимости)
- Passes: `BorrowChecker` (потоково-чувствительный анализ активности)
- ZST оптимизация: токены устраняются после компиляции
