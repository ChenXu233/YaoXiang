# RFC-009 v9 Отчёт об аудите полноты реализации

## Область аудита

Проверка полноты реализации компилятора на соответствие документу проектирования системы заимствования токенов RFC-009 v9. Охватывает шесть измерений: система типов, парсер, проверка заимствований, генерация IR, система Dup и захват замыканий.

---

## Список проблем (по приоритету)

### P0 — Блокирующие (заимствование токенов не работает во время выполнения)

| # | Проблема | Расположение | Влияние | Направление исправления |
|---|----------|--------------|---------|-------------------------|
| P0-1 | В IR нет инструкций Borrow/Release | `ir.rs` | Заимствование токенов нельзя представить в IR | Добавить инструкции `Borrow { dst, src, mutable }` и `Release { src }` |
| P0-2 | Отсутствует генерация IR для `Expr::Borrow` | `ir_gen.rs` | `&expr` игнорируется в IR-фазе | Добавить ветвь Borrow в `generate_expr_ir`, генерировать инструкции Borrow |
| P0-3 | Интерпретатор не обрабатывает Borrow | `execute.rs` | Невозможно выполнить заимствование во время выполнения | Добавить обработку `BytecodeInstr::Borrow`/`Release` |

### P1 — Важные (семантическая корректность)

| # | Проблема | Расположение | Влияние | Направление исправления |
|---|----------|--------------|---------|-------------------------|
| P1-1 | `&T` не удовлетворяет Dup | `solver.rs` L201-233 | Нарушение ядра RFC: "&T можно свободно копировать" | Добавить ветвь `MonoType::Ref { mutable: false, .. } => true` |
| P1-2 | Отсутствует механизм брендов | `borrow_checker.rs` | Невозможно отслеживать производные отношения токенов | Добавить уникальный ID времени компиляции и цепочку производных брендов |
| P1-3 | IR-инструкции для `&mut T` недостижимы | `borrow_checker.rs` | Обнаружение конфликтов изменяемых заимствований никогда не срабатывает | Автоматически решится после добавления MutRef-инструкций в IR |

### P2 — Улучшения (качество кода)

| # | Проблема | Расположение | Влияние | Направление исправления |
|---|----------|--------------|---------|-------------------------|
| P2-1 | Оптимизация ZST в MakeClosure | `ir_gen.rs` L3196-3198 | При захвате токенов возникают бессмысленные накладные расходы | ZST-типы пропускают env |
| P2-2 | Остатки Send/Sync | `solver.rs`, `auto_derive.rs`, `send_sync.rs` | Остатки кода | Удалить `check_send_trait`/`check_sync_trait`, очистить `BUILTIN_DERIVES` |
| P2-3 | Bytecode From<MonoType> заглушка | `bytecode.rs` L1418-1424 | Все типы маппятся в Void | Реализовать настоящую конвертацию типов |

## Рекомендуемый порядок реализации

1. **P1-1**: В `solver.rs` добавить сопоставление Dup для `MonoType::Ref` (1 строка)
2. **P2-2**: Очистить остатки Send/Sync
3. **P0-1**: В `ir.rs` добавить IR-инструкции Borrow/Release
4. **P0-2**: В `ir_gen.rs` добавить генерацию IR для `Expr::Borrow`

7. **P1-2**: Механизм брендов (можно отложить, текущего отслеживания по именам переменных достаточно для базовой функциональности)
