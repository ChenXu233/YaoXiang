# Документация сайта: редизайн страниц

## Цели

Переработать вёрстку 6 пользовательских страниц, унифицировать визуальный язык, улучшить оформление всех страниц.

## Область охвата

### Пользовательские вёрстки (6 шт.)

| Страница | Файл | Текущий компонент | Описание |
|----------|------|-------------------|----------|
| Главная | `index.md` | `Home.vue` | Сохранить текущий дизайн как базовый |
| Скачать | `download.md` | `Download.vue` | Переработать |
| Сообщество | `community.md` | `Community.vue` | Переработать |
| Площадка | `playground.md` | `Playground.vue` | Переработать |
| Инструменты | `tools.md` | Нет (голый HTML) | Создать `Tools.vue` |
| Блог | `blog/index.md` | Нет (голый HTML) | Создать `Blog.vue` |

### Глобальное оформление

Все стандартные страницы VitePress (учебники, руководства, справка, документация для разработчиков и т. д.) — скорректировать глобальную цветовую схему CSS, шрифты, отступы, без изменения структуры вёрстки.

## Архитектурные решения

## Стратегия компонентизации

Функциональная логика daisyUI-компонентов (btn/badge/card/dropdown и т. д.) сохраняется, визуальное оформление единообразно перекрывается токенами темы `yaoxiang`, дополнительная обёртка не требуется.

В Vue-компоненты выделяются только действительно повторяющиеся структуры страниц:

| Компонент | Повторное использование | Props |
|-----------|-------------------------|-------|
| `HeroSection.vue` | Скачать/Сообщество/Площадка/Инструменты/Блог — одинаковый Hero на 5 страницах | `badge`, `title`, `description`, слот по умолчанию (дополнительное содержимое под badge) |
| `TerminalWindow.vue` | Главная (показ кода) / Скачать (команда установки) / Сообщество (git log) | `title?`, `shadow` (primary/secondary), слот по умолчанию |
| `TimelineTrack.vue` | Главная (tracks) / Блог (список статей) | `dotColor?`, `tag?`, `date?`, слот по умолчанию |
| `FeatureCard.vue` | Скачать (платформы) / Инструменты (карточки) / Сообщество (входы) | `icon?`, `title`, `description`, `status?`, слот по умолчанию (действия внизу) |

Не выделяются: PageShell (VitePress достаточно), UnderConstruction (inline достаточно).

### Совместимость с i18n

Весь контент, требующий перевода, остаётся в markdown frontmatter, Vue-компоненты только отвечают за рендеринг. Список статей блога предоставляется через виртуальный модуль `virtual:blog-index`.

### Базовая линия дизайна

За основу берётся `Home.vue`:
- Шрифты: `Space Mono` (глобально) + `Microsoft YaHei` (заголовки на китайском) + `JetBrains Mono` (код)
- Смещённая тень: `shadow-[Xpx_Xpx_0px_color]` (резкая тень без размытия)
- Карточка: `card bg-base-200 rounded-none shadow-[6px_6px_0px_var(--p)]` (острые углы + грубая смещённая тень + приподнятый фон страницы)
- Ось времени: основа `w-1 bg-base-300` + круглые узлы `rounded-full`
- Наклонные метки: `rotate-2` + `shadow-lg` + `rounded-none`
- Подвал: `bg-neutral text-neutral-content rounded-none`
- Стратегия скруглений: только окно кода `rounded-lg`, круглые узлы `rounded-full`, всё остальное `rounded-none`

## Цветовая схема

### Чернильно-алая палитра

Метафорой служат традиционная каллиграфия, печати-чжуинь и эпиграфические оттиски: светлая тема — тёплая, как киноварная печать на рисовой бумаге, тёмная — холодная, как оттиск на каменной стеле.

Кастомные темы daisyUI v5 с использованием формата OKLCH через `@plugin "daisyui/theme"`.

### Разделение ролей primary/secondary

- **Primary (кинодарь)**: Hero Badge, CTA-кнопки, тень карточек 1-го уровня, badge заголовков секций, ключевые акценты
- **Secondary (печатная краска / стельно-серый)**: тень карточек 2-го уровня, второстепенные chip-метки, второстепенные узлы оси времени
- **Иерархия карточек**: страница `base-100` → карточка `base-200` (разница яркости ~5–6 %), карточки 1-го уровня `shadow-primary`, карточки 2-го уровня `shadow-secondary`
- **Accent (бирюзово-зелёный)**: сохранение подсветки ключевых слов в синтаксисе кода

### Шрифтовая система (соответствует `index.md`)

| Назначение | Шрифтовой стек |
|------------|----------------|
| Глобальный основной текст | `Space Mono`, monospace |
| Заголовки на китайском | `Microsoft YaHei`, `SimHei`, `PingFang SC`, `Heiti SC`, sans-serif |
| Код | `JetBrains Mono`, monospace |

Загрузка из Google Fonts: `JetBrains Mono:wght@400;700` + `Space Mono:ital,wght@0,400;0,700;1,400`

## Индекс блога

### Плагин `virtual:blog-index`

Собственный плагин VitePress (виртуальный модуль Vite), без зависимости от CI.

- Путь: `.vitepress/plugins/blog-index.ts`
- На этапе сборки сканирует `blog/*.md`, извлекает frontmatter (title, date, description)
- Сортирует по дате, выдаёт JSON
- В режиме разработки отслеживает изменения файлов, HMR автоматически обновляет данные
- Страница блога получает данные через `import { blogPosts } from 'virtual:blog-index'`

## Новые плагины

| Плагин | Назначение | Доп. настройки |
|--------|------------|----------------|
| `vitepress-plugin-group-icons` | Иконки типов файлов в блоках кода | Нет |
| `vitepress-plugin-back-to-top` | Кнопка «наверх» | Нет (стили перекрываются темой) |
| `vitepress-plugin-nprogress` | Полоса прогресса загрузки страницы | `color: oklch(55% 0.24 28)` |
| `@vuepress/plugin-reading-time` | Время чтения + счётчик слов | Нет (только для страниц блога) |
| `vitepress-plugin-tabs` | Синтаксис tab в Markdown | Нет |
| `@nolebase/vitepress-plugin-git-changelog` | Контрибьюторы + changelog внизу каждой страницы | `maxGitLogCount: 5` |

Внимание: совместимость плагинов с VitePress 1.6.4 необходимо проверять поочерёдно, при необходимости — заменять альтернативами.

## Схемы вёрстки страниц

### Страница «Инструменты» (Tools.vue) · новая

Перевод с голого HTML на Vue-компонент.

| Элемент | Решение |
|---------|---------|
| Hero | Badge TOOLS + заголовок жирным шрифтом + описание |
| Блок карточек | Три карточки возможностей (компилятор/форматер/LSP), иконка + заголовок + описание + метка статуса |
| Статусы | Компилятор/форматер → `доступно` (primary badge), LSP → `в разработке` (полупрозрачный badge cool) |

### Страница списка блога (Blog.vue) · новая

Вёрстка в виде оси времени, в духе главной страницы (Tracks):

| Элемент | Решение |
|---------|---------|
| Hero | Badge BLOG + заголовок жирным шрифтом + описание |
| Ось времени | Основа (w-1 bg-base-300) + узлы `rounded-full` |
| Карточки статей | bg-base-200 + жёсткая тень shadow-secondary + дата + заголовок + краткое описание + наклонная метка NEW |
| Источник данных | Виртуальный модуль `virtual:blog-index` |
| Пустое состояние | Пунктирный плейсхолдер «Скоро появятся новые статьи...» |

### Страница «Сообщество» (Community.vue)

Полная переработка, очистка старой структуры. Принципы: обильные пустоты, сдержанность; карточки только для терминала и доски объявлений; Badge пронизывает всю страницу; данные рассредоточены в смысловых зонах.

**Вёрстка (сверху вниз):**

| Зона | Решение |
|------|---------|
| Hero | Сетчатый фон + заголовок жирным шрифтом + tagline + ряд выразительных Badge (каждый со своим лёгким поворотом и характерной анимацией, при наведении — выпрямление и увеличение) |
| Мейнтейнеры | Левая граница `border-l-3 border-primary` + аватары (с тенью, увеличение при наведении) + Badge CORE/SOLO рядом с именем + естественное вкрапление числа контрибьюторов в описание |
| Контрибьюторы | Две колонки PR / Issue, пустое состояние — пунктирная рамка с приглашением к участию |
| Последние события | Две колонки: терминал git log (число коммитов встроено внизу) + доска объявлений (Meetup/Conf/Contributor Day, статусные Badge встроены в заголовок) |
| Присоединиться | Три ссылки с подчёркиванием (рядом с GitHub — Badge Stars, рядом с руководством контрибьютора — Badge Open Issues), при наведении — стрелка сдвигается вправо + цвет подчёркивания меняется |

**Система Badge (переиспользуемые классы):**

6 вариантов — hot (алый + тень + наклон), cool (вторичный цвет + wobble), teal (бирюзово-зелёный + наклон), glow (пульсация border), muted (серый + наклон), soft (мелкий серый). По всей странице смешанное использование, без выделения в отдельный блок.

**Что больше не используется:**
- терминал `mockup-window` (заменяется на mini-console)
- отдельный блок статистики (данные рассредоточены)
- описательный блок с `border-l-4`
- статус online у аватаров

### Страница «Площадка» (Playground.vue)

Сохраняется ядро CodeMirror-редактора, оболочка заменяется полностью.

| Элемент | Решение |
|---------|---------|
| Hero | Badge PLAYGROUND + заголовок жирным шрифтом + описание |
| Редактор | Три точки терминала + имя файла `main.yx` + нижняя строка состояния (строка/столбец/отступ/кодировка) |
| Верхний правый угол | Пометка версии компилятора `v0.1.0 · WASM` |
| Кнопка запуска | Фон primary + смещённая тень + смещение при нажатии |
| Панель вывода | Тень вторичного цвета + строка заголовка с временем компиляции |
| Горячие клавиши | Под кнопкой запуска три группы (Ctrl+Enter — запуск / Ctrl+S — поделиться / Ctrl+K — форматирование) |
| Путь | Исправление пути Playground.vue (путь import в `theme/index.js` против фактического расположения файла) |

Сохраняется существующая функциональность, цвета приводятся в соответствие, структура вёрстки не меняется.

## Прочие пункты реализации

### Глобальная переработка CSS (tailwind.css)

1. `@import "tailwindcss"`
2. `@plugin "daisyui/theme"` светлая `yaoxiang` (default, prefersdark: false)
3. `@plugin "daisyui/theme"` тёмная `yaoxiang-dark` (prefersdark: true)
4. Переменные VitePress `--vp-c-brand` и т. п. пробрасываются на primary-токены
5. Цвет бегунка полосы прокрутки `oklch(55% 0.24 28)`
6. Сохраняется исправление `VPNavBarMenu` панели навигации

### Оптимизация слежения за мышью

Home.vue + Download.vue: обернуть в `useIntersectionObserver` или `v-if`, активировать `useMouse` только когда компонент виден, при невидимости — приостанавливать расчёт 3D-наклона.

### Исправление пути Playground.vue

Путь import в строке L9 файла `theme/index.js` не совпадает с фактическим расположением файла, привести к единому месту.
