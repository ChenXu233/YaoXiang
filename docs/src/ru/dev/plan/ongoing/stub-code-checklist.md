|------|----------|----------|------|
| 32-34 | `fn step(&mut self) -> ExecutorResult<()>` | Пошаговое выполнение одной инструкции | `todo!()` |
| 36-38 | `fn step_over(&mut self) -> ExecutorResult<()>` | Шаг с обходом (без захода внутрь функций) | `todo!()` |
| 40-42 | `fn step_out(&mut self) -> ExecutorResult<()>` | Шаг с выходом (выполнить до конца текущей функции) | `todo!()` |
| 44-46 | `fn run(&mut self) -> ExecutorResult<()>` | Продолжить выполнение до следующей точки останова | `todo!()` |

**Контекст**: Остальные методы trait `DebuggableExecutor` (`set_breakpoint`, `has_breakpoint`, `current_ip`, `current_function`, `breakpoints`) реализованы, не реализовано только управление пошаговым выполнением.

**Рекомендации по реализации**:
- `step()`: выполнить инструкцию по текущему IP, IP++
- `step_over()`: если текущая инструкция — вызов функции, установить временную точку останова на следующей инструкции, затем вызвать run
- `step_out()`: запомнить текущую глубину стека вызовов, вызывать run, пока глубина стека не уменьшится
- `run()`: циклически выполнять, пока не встретится точка останова или программа не завершится
