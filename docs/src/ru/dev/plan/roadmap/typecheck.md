## Обзор модуля

Типо-чекер отвечает за семантический анализ AST и выведение типов. Используется трёхпроходная архитектура сканирования: сбор определений типов → сбор сигнатур функций → проверка тел функций. Реализован полный алгоритм выведения типов Hindley-Milner.

**Объём кода**: 28 153 строки (15 383 строки реализации + 12 770 строк тестов)

---

## Список функций

### Основной типо-чекер (checker.rs - 1 116 строк)

- ✅ Оркестрация проверки типов на уровне модуля
- ✅ Трёхпроходная архитектура сканирования

- ✅ Регистрация стандартных trait (Clone, Dup, Equal, Debug, Iterator)

- ✅ Паттерн сбора ошибок (для сценариев диагностики LSP)
- ✅ Сбор семантических токенов (для подсветки кода)

- ✅ **Проверка операторов** (statements.rs - 1 364 строки): let связывания, определения функций, use операторы, привязки внешних методов, return операторы

- ✅ **Проверка подтипов** (subtyping.rs): Int→Float подтип, ковариантность/контравариантность, структурированные подтипы (уткотипирование)
- ✅ **Совместимость типов** (compatibility.rs): совместимость function типов, совместимость контейнерных типов
- ✅ **Проверка присваивания** (assignment.rs): проверка изменяемости, constrained присваивание
- ✅ **Управление областями видимости** (scope.rs): унифицированный менеджер областей видимости
- ✅ **Анализ захвата в замыканиях** (capture.rs): анализ побега, выведение паттернов захвата (Read/Write/Move)

- ✅ **Специализация** (specialization/): алгоритм специализации generic функций, создание экземпляров, замена типов

- ✅ **Реализация семантических токенов** (semantic_tokens_impl.rs - 1 653 строки): аннотация семантических типов для идентификаторов исходного кода
