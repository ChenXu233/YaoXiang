## Обзор модуля

Модуль генерации кода отвечает за перевод IR (промежуточного представления) в байт-код. Поддерживает полный формат файла байт-кода `.yx` (.42 формат), включая сегмент отладочной информации.

**Объём кода**: около 2400 строк (7 файлов исходного кода)

### Формат файла байт-кода (bytecode.rs)

- ✅ Полная структура `BytecodeFile`: заголовок файла + таблица типов + пул констант + кодовый сегмент + необязательный отладочный сегмент
- ✅ Заголовок файла: магическое число `YXBC` (0x59584243), версия 2
- ✅ Поддержка смешанного порядка байтов: магическое число в big-endian, данные в little-endian
- ✅ Поддержка типов констант: Void, Bool, Int, Float, Char, String, Bytes
- ✅ **Отладочный сегмент** (DebugSection): поддержка отображения позиций исходного кода (IP -> Span)

### Система кодов операций (opcode.rs, shared)

- ✅ Определено **более 80 Opcode**, разделённых на 12 категорий
- ✅ Полная реализация `TryFrom<u8>`
- ✅ Вспомогательные методы проверки: `is_numeric_op`, `is_call_op`, `is_jump_op` и др.

## Нереализованные/заглушки инструкции

| IR-инструкция | Текущая реализация | Примечание |
|----------------|--------------------|------------|
| ShareRef | Nop | В коде есть TODO-комментарий |
| Free | Nop | Без операции |
| Dup, Swap | Nop | Операции со стеком пока не реализованы |
| UnsafeBlockStart/End | Nop | Маркеры unsafe-блоков |
| PtrFromRef/PtrDeref/PtrStore/PtrLoad | все Nop | Операции с сырыми указателями пока не поддерживаются |
| TypeTest | заглушка TypeCheck | Операнды захардкожены как [0, 0, 0] |
| Cast | операнды захардкожены | Целевой тип не закодирован |

## Покрытие тестами

**13 юнит-тестов**, все проходят:

| Файл тестов | Кол-во тестов | Область покрытия |
|-------------|---------------|-------------------|
| `mod.rs` | 1 | Базовая инициализация контекста |
| `buffer.rs` | 2 | Добавление/получение из пула констант, эмиссия в буфер байт-кода |
| `emitter.rs` | 2 | Эмиссия инструкций и отображения, ожидающие заполнения переходы |
| `operand.rs` | 2 | Преобразование регистров, обнаружение переполнения |
| `flow.rs` | 5 | Генератор меток, распределитель регистров, менеджер потока, базовые таблицы символов, вложенные области видимости |
| `bytecode.rs` | 1 | Round-trip тест кодирования/декодирования DebugSection |

**Недостатки тестирования**:
- ❌ Нет отдельных тестов для translator.rs
- ❌ Нет интеграционных тестов для полного процесса `generate()`
- ❌ Нет сквозных тестов (исходный код -> байт-код -> десериализация и проверка)
