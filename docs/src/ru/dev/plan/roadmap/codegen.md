# Генерация кода (Codegen)

> **Статус модуля**: Завершён (базовая функциональность)
> **Расположение**: `src/middle/passes/codegen/`
> **Последнее обновление**: 2026-06-01

---

## Обзор модуля

Модуль генерации кода отвечает за преобразование IR (промежуточного представления) в байткод. Поддерживает полный формат файла байткода `.yx` (формат .42), включая сегмент отладочной информации.

**Объём кода**: около 2400 строк (7 исходных файлов)

---

## Список функций

### Транслятор (translator.rs - 1 073 строки)

**Арифметические/побитовые операции (всё реализовано)**:
- ✅ Add, Sub, Mul, Div, Mod, And, Or, Xor, Shl, Shr, Sar, Neg

**Операции сравнения (всё реализовано)**:
- ✅ Eq, Ne, Lt, Le, Gt, Ge

**Управление потоком выполнения (всё реализовано)**:
- ✅ Jmp, JmpIf, JmpIfNot, Ret, с механизмом обратного заполнения смещений переходов

**Вызовы функций (всё реализовано)**:
- ✅ CallStatic, CallNative, CallVirt, CallDyn, TailCall

**Операции с переменными (всё реализовано)**:
- ✅ Move, Load, Store

**Операции с памятью/объектами (всё реализовано)**:
- ✅ Alloc, AllocArray, HeapAlloc
- ✅ LoadField, StoreField, LoadIndex, StoreIndex
- ✅ CreateStruct

**Система владения (всё реализовано)**:
- ✅ Drop, ArcNew, ArcClone, ArcDrop
- ✅ Borrow, Release (токены заимствования)
- ✅ ShareRef (временно реализован как Nop)

**Замыкания/Upvalue (всё реализовано)**:
- ✅ MakeClosure, LoadUpvalue, StoreUpvalue, CloseUpvalue

**Строковые операции (всё реализовано)**:
- ✅ StringLength, StringConcat, StringGetChar, StringFromInt, StringFromFloat

**Параллелизм (частично реализовано)**:
- ✅ Spawn
- ✅ EvalPush, EvalPop, Yield

### Формат файла байткода (bytecode.rs)

- ✅ Полная структура `BytecodeFile`: заголовок файла + таблица типов + пул констант + кодовый сегмент + опциональный сегмент отладочной информации
- ✅ Заголовок файла: магическое число `YXBC` (0x59584243), версия 2
- ✅ Поддержка смешанного порядка байтов: магическое число в big-endian, данные в little-endian
- ✅ Поддерживаемые типы констант: Void, Bool, Int, Float, Char, String, Bytes
- ✅ **Сегмент отладочной информации** (DebugSection): поддержка отображения позиций в исходном коде (IP -> Span)

### Система кодов операций (opcode.rs, shared)

- ✅ Определено **более 80 Opcode**, разделённых на 12 категорий
- ✅ Полная реализация `TryFrom<u8>`
- ✅ Вспомогательные методы проверки: `is_numeric_op`, `is_call_op`, `is_jump_op` и др.

## Нереализованные/заглушки инструкции

| IR-инструкция | Текущая реализация | Описание |
|----------------|---------------------|----------|
| ShareRef | Nop | В коде есть TODO-комментарий |
| Free | Nop | Без операции |
| Dup, Swap | Nop | Операции со стеком пока не реализованы |
| UnsafeBlockStart/End | Nop | Маркеры unsafe-блоков |
| PtrFromRef/PtrDeref/PtrStore/PtrLoad | Всё Nop | Операции с сырыми указателями пока не поддерживаются |
| TypeTest | Заглушка TypeCheck | Операнды захардкожены как [0, 0, 0] |
| Cast | Операнды захардкожены | Целевой тип не закодирован |

## Покрытие тестами

**13 юнит-тестов**, все проходят:

| Тестовый файл | Количество тестов | Область покрытия |
|---------------|-------------------|------------------|
| `mod.rs` | 1 | Базовое создание контекста |
| `buffer.rs` | 2 | Добавление/получение из пула констант, эмиссия буфера байткода |
| `emitter.rs` | 2 | Эмиссия инструкций и маппинг, отложенное заполнение переходов |
| `operand.rs` | 2 | Преобразование регистров, обнаружение переполнения |
| `flow.rs` | 5 | Генератор меток, распределитель регистров, менеджер потока, базовая таблица символов, вложенность областей видимости |
| `bytecode.rs` | 1 | Круговой тест кодирования/декодирования DebugSection |

**Недостатки тестирования**:
- ❌ translator.rs не имеет отдельных тестов
- ❌ Нет интеграционных тестов для полного процесса `generate()`
- ❌ Нет сквозных тестов (исходный код -> байткод -> десериализация и проверка)

## Оценка качества кода

| Измерение | Оценка | Описание |
|-----------|--------|----------|
| Полнота функционала | 85% | Основной трансляционный процесс полный, несколько инструкций-заглушек |
| Покрытие тестами | Недостаточно | Около 30%, translator не протестирован |
| Документация | Средняя | Есть文档 по модулям и типам, не хватает спецификации формата и пользовательской документации |
| Качество кода | Хорошее | Чёткая архитектура, разделение ответственности, но translator.rs слишком большой |
| Соответствие RFC | В основном соответствует | Удовлетворяет потребностям VM-бэкенда, LLVM-бэкенд ещё предстоит реализовать |
