# Лексический анализатор (Lexer)

> **Статус модуля**: стабильный (3 пункта к улучшению)
> **Расположение**: `src/frontend/core/lexer/`
> **Последнее обновление**: 2026-06-01

---

## Обзор модуля

Лексический анализатор отвечает за преобразование строки исходного кода в поток токенов. Используется классическая архитектура, управляемая потоком символов, поддерживающая полную лексическую спецификацию языка YaoXiang.

**Объём кода**: около 800 строк (7 исходных файлов)

---

## Сравнение с RFC

| RFC | Статус реализации | Описание |
|-----|----------|------|
| RFC-004 Синтаксис привязки | ✅ Реализован | `[` `]` корректно распознаются, BindingValidator полностью реализован |
| RFC-010 Единый синтаксис типов | ⚠️ Частичная реализация | Для дженериков используются `()` вместо `<>`, но отсутствуют ключевые слова `where`, `trait`, `interface`, `impl`, `forall`, `exists` |
| RFC-011 Система дженериков | ⚠️ Валидатор реализован | `TypeSystemValidator` реализован, но отсутствует поддержка ключевых слов, связанных с типами высшего порядка |
| RFC-012 F-String | ✅ Реализован | Полный лексический анализ `f"..."` |
