# Лексер (Lexer)

> **Состояние модуля**：Завершено
> **Расположение**：`src/frontend/core/lexer/`
> **Последнее обновление**：2026-06-01

---

## Обзор модуля

Лексер отвечает за преобразование строки исходного кода в поток Token. Используется классическая архитектура, управляемая символьным потоком, с поддержкой полной спецификации лексики языка YaoXiang.

**Объём кода**：около 800 строк (7 исходных файлов)

## Сравнение с RFC

| RFC | Состояние реализации | Описание |
|-----|----------|------|
| RFC-004 Синтаксис привязки | ✅ Реализовано | `[` `]` корректно распознаются, BindingValidator полностью реализован |
| RFC-010 Унифицированный синтаксис типа | ⚠️ Частично реализовано | generics используют `()`, а не `<>`, но отсутствуют ключевые слова `where`, `trait`, `interface`, `impl`, `forall`, `exists` |
| RFC-011 Система generics | ⚠️ Реализация валидатора | TypeSystemValidator реализован, но отсутствует поддержка ключевых слов для высших типов |
| RFC-012 F-String | ✅ Реализовано | Полная лексическая обработка `f"..."` |
