# Лексер (Lexer)

> **Состояние модуля**: Стабильный (3 улучшения в待完成列表)
> **Расположение**: `src/frontend/core/lexer/`
> **Последнее обновление**: 2026-06-01

---

## Обзор модуля

Лексер отвечает за преобразование строки исходного кода в поток Token. Использует классическую архитектуру, управляемую потоком символов, с поддержкой полной лексической спецификации языка YaoXiang.

**Объём кода**: ~800 строк (7 исходных файлов)

## Сравнение с RFC

| RFC | Состояние реализации | Описание |
|-----|----------------------|----------|
| RFC-004 Синтаксис связывания | ✅ Реализовано | `[` `]` корректно распознаются, BindingValidator полностью реализован |
| RFC-010 Унифицированный синтаксис типов | ⚠️ Частично реализовано | Обобщённые типы используют `()`, а не `<>`, но отсутствуют ключевые слова `where`, `trait`, `interface`, `impl`, `forall`, `exists` |
| RFC-011 Система обобщённых типов | ⚠️ Реализация валидатора | TypeSystemValidator реализован, но отсутствует поддержка ключевых слов для высших типов |
| RFC-012 F-String | ✅ Реализовано | Полная лексема `f"..."` |
