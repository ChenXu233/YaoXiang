# Интерпретатор (Interpreter)

> **Статус модуля**：Завершено
> **Расположение**：`src/backends/interpreter/`
> **Последнее обновление**：2026-06-01

---

## Обзор модуля

Интерпретатор отвечает за выполнение байткода. Использует архитектуру регистровой виртуальной машины, поддерживает полный набор из 39 байткодовых инструкций, полностью согласован с RFC-008 (модель параллелизма) и RFC-009 (модель владения).

**Объём кода**：3 768 строк (9 файлов исходного кода)

---

## Список функций

### Основной исполнительный движок (execute.rs - 1 308 строк)

**Управление потоком выполнения (10 видов)**：
- ✅ Nop, Return, ReturnValue, Yield (пустая операция), EvalPush, EvalPop, Spawn, Jmp, JmpIf, JmpIfNot, Switch (упрощённый)

**Операции с регистрами (5 видов)**：
- ✅ Mov, LoadConst, LoadLocal, StoreLocal, LoadArg

**Арифметические/логические операции (11 BinaryOp + 1 UnaryOp)**：
- ✅ Add/Sub/Mul/Div/Rem/And/Or/Xor/Shl/Sar/Shr, поддержка как Int, так и Float
- ⚠️ UnaryOp реализован только для Int (отрицание)

**Сравнение (6 видов CompareOp)**：
- ✅ Eq/Ne/Lt/Le/Gt/Ge, поддержка как Int, так и String

**Операции с памятью (9 видов)**：
- ✅ StackAlloc (пустая операция), HeapAlloc, Drop (пустая операция), GetField, SetField, LoadElement, StoreElement, NewListWithCap, CreateStruct

**Операции Arc/Weak (5 видов)**：
- ✅ ArcNew, ArcClone, ArcDrop (пустая операция), WeakNew, WeakUpgrade

**Токены заимствования (2 вида)**：
- ✅ Borrow (ZST, во время выполнения эквивалентен Mov), Release (ZST, во время выполнения эквивалентен Nop)

**Вызовы функций (7 видов)**：
- ✅ CallStatic, CallNative, CallVirt, CallDyn, MakeClosure, LoadUpvalue, StoreUpvalue, CloseUpvalue

**Операции со строками (6 видов)**：
- ✅ StringLength, StringConcat, StringEqual, StringGetChar, StringFromInt, StringFromFloat

**Обработка исключений (3 вида)**：
- ✅ TryBegin (пустая операция), TryEnd (пустая операция), Throw

**Отладка/типы (4 вида)**：
- ⚠️ BoundsCheck (пустая операция), TypeCheck (пустая операция), Cast (прозрачная передача), TypeOf (заглушка)

### Основные архитектурные возможности

- ✅ **Куча (Heap)**：динамическое выделение List/Tuple/Array/Dict/Struct
- ✅ **Стек вызовов (Frame)**：файл регистров + локальные переменные + upvalue + eval-стек + группа spawn
- ✅ **Пул констант**：межмодульный общий доступ
- ✅ **Таблица функций**：по имени (HashMap) и по индексу (Vec), поддержка вызова замыканий по ID
- ✅ **FFI-реестр**：предварительная загрузка серии функций `std.io.*`, расширяемость пользовательскими нативными функциями
- ✅ **Планирование задач DAG (LocalRuntime)**：ленивое/параллельное вычисление на основе RFC-008
- ✅ **Три стратегии вычисления**：Block (синхронный), Auto (ленивый/параллельный), Eager (жадный)
- ✅ **Структурированный параллелизм**：отслеживание группы spawn, ожидание завершения всех задач при выходе из области видимости, каскадная отмена при сбое зависимостей

## Сравнение с RFC

### RFC-008 (Параллельная модель Runtime)

| Требование к дизайну | Статус реализации | Пояснение |
|----------|----------|------|
| Трёхуровневая среда выполнения: Embedded / Standard / Full | ✅ Реализовано | Настраивается через `RuntimeMode` |
| Три стратегии вычисления: Block / Auto / Eager | ✅ Реализовано | |
| Планирование задач DAG (`LocalRuntime`) | ✅ Реализовано | |
| Отслеживание зависимостей задач, распространение отмены, структурированный параллелизм | ✅ Реализовано | |
| Синхронный = частный случай планирования (режим Embedded) | ✅ Реализовано | |

### RFC-009 (Модель владения)

| Требование к дизайну | Статус реализации | Пояснение |
|----------|----------|------|
| Borrow/Release как токены нулевого размера (ZST) | ✅ Реализовано | Во время выполнения эквивалентны Mov/Nop |
| ArcNew/ArcClone/ArcDrop реализуют семантику ключевого слова `ref` | ✅ Реализовано | |
| WeakNew/WeakUpgrade реализуют слабые ссылки | ✅ Реализовано | |
| Семантика перемещения (поведение по умолчанию) | ✅ Реализовано | |
| `clone()` обрабатывается на уровне компиляции | ✅ Реализовано | Специальные инструкции во время выполнения не требуются |

## Упрощённые/заглушечные реализации

| Инструкция | Текущее поведение | Дизайнерский замысел |
|------|----------|------|
| Switch | Прямое продвижение IP | Должен распределять переходы по значению |
| TypeOf | Возвращает заглушку длины type_table | Должен возвращать информацию о типе во время выполнения |
| Cast | Прозрачная передача значения (без реального преобразования) | Должен преобразовывать по целевому типу |
| BoundsCheck / TypeCheck | Пустые операции | В режиме отладки должны выполнять проверки во время выполнения |
| StringGetChar | Берёт только первый символ, игнорирует параметр index | Должен брать символ по index |
| UnaryOp | Только отрицание Int, игнорирует тип op | Должен поддерживать больше унарных операций |
| step/step_over/step_out/run | `todo!()` | Функции пошаговой отладки не реализованы |
