# Интерпретатор (Interpreter)

> **Состояние модуля**: стабильный (5 улучшений в очереди)
> **Расположение**: `src/backends/interpreter/`
> **Последнее обновление**: 2026-06-01

---

## Обзор модуля

Интерпретатор отвечает за выполнение байткода. Использует архитектуру регистровой виртуальной машины, поддерживает полный набор из 39 байткодовых инструкций, полностью соответствует RFC-008 (модель параллелизма) и RFC-009 (модель владения).

**Объём кода**: 3 768 строк (9 исходных файлов)

---

## Список функций

### Основной исполнительный движок (execute.rs - 1 308 строк)

**Управление потоком выполнения (10 видов)**:
- ✅ Nop, Return, ReturnValue, Yield (пустая операция), EvalPush, EvalPop, Spawn, Jmp, JmpIf, JmpIfNot, Switch (упрощённый)

**Операции с регистрами (5 видов)**:
- ✅ Mov, LoadConst, LoadLocal, StoreLocal, LoadArg

**Арифметические/логические операции (11 BinaryOp + 1 UnaryOp)**:
- ✅ Add/Sub/Mul/Div/Rem/And/Or/Xor/Shl/Sar/Shr, поддержка как Int, так и Float
- ⚠️ UnaryOp реализован только для Int (отрицание)

**Сравнение (6 видов CompareOp)**:
- ✅ Eq/Ne/Lt/Le/Gt/Ge, поддержка как Int, так и String

**Операции с памятью (9 видов)**:
- ✅ StackAlloc (пустая операция), HeapAlloc, Drop (пустая операция), GetField, SetField, LoadElement, StoreElement, NewListWithCap, CreateStruct

**Операции Arc/Weak (5 видов)**:
- ✅ ArcNew, ArcClone, ArcDrop (пустая операция), WeakNew, WeakUpgrade

**Токены заимствования (2 вида)**:
- ✅ Borrow (ZST, в runtime эквивалентен Mov), Release (ZST, в runtime эквивалентен Nop)

**Вызовы функций (7 видов)**:
- ✅ CallStatic, CallNative, CallVirt, CallDyn, MakeClosure, LoadUpvalue, StoreUpvalue, CloseUpvalue

**Строковые операции (6 видов)**:
- ✅ StringLength, StringConcat, StringEqual, StringGetChar, StringFromInt, StringFromFloat

**Обработка исключений (3 вида)**:
- ✅ TryBegin (пустая операция), TryEnd (пустая операция), Throw

**Отладка/типы (4 вида)**:
- ⚠️ BoundsCheck (пустая операция), TypeCheck (пустая операция), Cast (прозрачная передача), TypeOf (заглушка)

### Основные архитектурные возможности

- ✅ **Куча (Heap)**: динамическое выделение List/Tuple/Array/Dict/Struct
- ✅ **Стек вызовов (Frame)**: файл регистров + локальные переменные + upvalue + eval-стек + группа spawn
- ✅ **Пул констант**: разделяется между модулями
- ✅ **Таблица функций**: по имени (HashMap) и по индексу (Vec), двойная таблица, поддержка вызова замыканий по ID
- ✅ **Реестр FFI**: предварительная загрузка функций серии `std.io.*`, расширяемость пользовательскими нативными функциями
- ✅ **Планировщик задач на основе DAG (LocalRuntime)**: ленивые/并作 вычисления на основе RFC-008
- ✅ **Три стратегии вычислений**: Block (синхронный), Auto (ленивый/并作), Eager (энергичный)
- ✅ **Структурированный 并作**: отслеживание групп spawn, ожидание завершения всех задач при выходе из области видимости, каскадная отмена при сбое зависимостей

## Сравнение с RFC

### RFC-008 (Runtime-модель параллелизма)

| Требование дизайна | Состояние реализации | Примечание |
|-------------------|----------------------|------------|
| Три уровня рантайма: Embedded / Standard / Full | ✅ Реализовано | Настраивается через `RuntimeMode` |
| Три стратегии вычислений: Block / Auto / Eager | ✅ Реализовано | |
| Планировщик задач DAG (`LocalRuntime`) | ✅ Реализовано | |
| Отслеживание зависимостей задач, распространение отмены, структурированный 并作 | ✅ Реализовано | |
| Синхронный = частный случай планирования (режим Embedded) | ✅ Реализовано | |

### RFC-009 (модель владения)

| Требование дизайна | Состояние реализации | Примечание |
|-------------------|----------------------|------------|
| Borrow/Release как токены нулевого размера (ZST) | ✅ Реализовано | В runtime эквивалентны Mov/Nop |
| ArcNew/ArcClone/ArcDrop реализуют семантику ключевого слова `ref` | ✅ Реализовано | |
| WeakNew/WeakUpgrade реализуют слабые ссылки | ✅ Реализовано | |
| Move-семантика (поведение по умолчанию) | ✅ Реализовано | |
| `clone()` обрабатывается на уровне компиляции | ✅ Реализовано | В runtime не требуется специальная инструкция |

## Упрощённые/заглушечные реализации

| Инструкция | Текущее поведение | Задуманное поведение |
|------------|-------------------|---------------------|
| Switch | Прямое продвижение IP | Должен выполнять dispatch по значению |
| TypeOf | Возвращает размер type_table как заглушку | Должен возвращать информацию о типе в runtime |
| Cast | Прозрачная передача значения (без реального преобразования) | Должен выполнять преобразование к целевому типу |
| BoundsCheck / TypeCheck | Пустая операция | В debug-режиме должны выполнять проверки в runtime |
| StringGetChar | Берёт только первый символ, игнорирует параметр index | Должен брать символ по index |
| UnaryOp | Только отрицание Int, игнорирует тип op | Должен поддерживать больше унарных операций |
| step/step_over/step_out/run | `todo!()` | Функции пошаговой отладки не реализованы |
