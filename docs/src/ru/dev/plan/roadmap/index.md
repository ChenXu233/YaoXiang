## Обзор проекта

YaoXiang — это перспективный язык программирования с архитектурой компиляции в байт-код + интерпретация в VM. В настоящее время находится в стадии v0.7.x, базовый компилятор и среда выполнения практически завершены.

## Ключевые находки

### Завершённые области

1. **Фронтенд компилятора**：Лексер, парсер, проверка типов — всё завершено, отличное тестовое покрытие
2. **Система владения**：Полный проверка заимствований, 14/15 подмодулей полностью функциональны
3. **Инструментарий**：LSP, менеджер пакетов, форматтер — в основном завершены
4. **Документационный сайт**：Многоязычная поддержка, CI/CD авторазвёртывание

### Основные пробелы

1. **Модель 并作**（RFC-001）：Анализатор зависимостей DAG не реализован, аннотации управления 并发 неполны
2. **Трёхуровневая архитектура Runtime**（RFC-008）：Завершён только этап A, LLVM AOT-бэкенд не реализован
3. **Зависимые типы значений**（RFC-011）：Вычислительный движок этапа компиляции не реализован
4. **Система конфигурации**（RFC-015）：Конфигурация уровня пользователя не реализована
5. **Тестирование стандартной библиотеки**：Только 8 юнит-тестов, крайне недостаточно

**Цель**：Восполнить тестовое покрытие, исправить известные проблемы

| Модуль | Текущие тесты | Цель | Конкретные задачи |

| REPL | 0 | 20+ | Дополнить тесты движка вычислений, системы команд, автодополнения |
| Лексер | 31（150+ не скомпилировано） | 180+ | Активировать 11 тестовых файлов в каталоге tests/ |
| Codegen | 13 | 30+ | Дополнить юнит-тесты translator.rs |

| Runtime | 22 | 30+ | Дополнить тесты facade.rs и task.rs |

#### Исправление известных проблем

| Проблема | Модуль | Описание |
|------|------|----------|
| `os.chdir` фактически не меняет директорию | std | Только проверяет существование директории, не вызывает `std::env::set_current_dir()` |
| `string.len` возвращает количество байтов | std | `native_len` использует `s.len()` и возвращает байты, а не символы |
| Модуль `weak` невозможно импортировать | std | Отсутствует реализация `StdModule` trait |
| Пустой блок кода выводит `{\n}` | formatter | Должен выводить `{}`（§6.3） |
| Конфигурация `single_quote` не применяется | formatter | Поле существует, но не читается（§8.3） |
| Списки параметров не поддерживают автоперенос строки | formatter | Всегда выводится в одну строку（§4.2） |

#### CTE Вычислительный движок этапа компиляции（Фазы 1-2）

- [ ] Определить `CTValue` enum и структуру `EvalEnv`
- [ ] Реализовать базовые пути `eval()`: 字面量, переменные, 二元ные операции, условия, блоки кода
- [ ] Реализовать первую версию анализатора чистоты
- [ ] Вставить точки вызова CTE в проверщик типов
- [ ] Константная свёртка：`1 + 2 * 3` вычисляется на этапе компиляции как `7`
- [ ] Реализовать вычисление встраиваемых функций
