> **Последнее обновление**: 2026-06-04

---

## Обзор проекта

YaoXiang — это перспективный язык программирования, использующий архитектуру компиляции в байт-код + интерпретации VM. В настоящее время проект находится на этапе v0.7.x, основной компилятор и среда выполнения в основном завершены.

## Основные выводы

### Завершённые области

1. **Фронтенд компилятора**: лексический анализ, парсинг, проверка типов полностью завершены, покрытие тестами отличное
2. **Система владения**: полный проверщик заимствований, 14/15 подмодулей функционально завершены
3. **Цепочка инструментов**: LSP, менеджер пакетов, форматер в основном завершены
4. **Сайт документации**: многоязычная поддержка, автоматическое развёртывание CI/CD

### Основные пробелы

1. **Модель spawn** (RFC-001): DAG-анализатор зависимостей не реализован, аннотации управления параллелизмом неполные
2. **Трёхуровневая архитектура Runtime** (RFC-008): завершён только этап A, LLVM AOT бэкенд не реализован
3. **Зависимые от значений типы** (RFC-011): механизм вычислений на этапе компиляции не реализован
4. **Система конфигурации** (RFC-015): пользовательская конфигурация не реализована
5. **Тесты стандартной библиотеки**: всего 8 модульных тестов, серьёзно недостаточно

**Цель**: заполнить покрытие тестами, исправить известные проблемы

#### Исправление известных проблем

| Проблема | Модуль | Описание |
|------|------|------|
| `os.chdir` фактически не переключает директорию | std | Только проверяет, существует ли директория, не вызывает `std::env::set_current_dir()` |
| `string.len` возвращает количество байтов | std | `native_len` использует `s.len()` для возврата количества байтов вместо количества символов |
| Модуль `weak` не может быть импортирован | std | Отсутствует реализация трейта `StdModule` |
