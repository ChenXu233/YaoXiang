# Парсер (Parser)

> **Статус модуля**：Стабильный (5 улучшений в待改进)
> **Расположение**：`src/frontend/core/parser/`
> **Последнее обновление**：2026-06-01

---

## Обзор модуля

Парсер отвечает за преобразование потока токенов в АСТ (абстрактное синтаксическое дерево). Использует классический алгоритм Pratt Parsing (rekursivный descent с приоритетом операторов), поддерживающий полную спецификацию языка YaoXiang.

**Объём кода**：около 5000 строк (31 исходный файл, из них 14 тестовых)

---

## Список функций

### Разбор выражений (Pratt Parser)

**Префиксные (nud)**：
- ✅ Все 字面量：Int, Float, String, Char, Bool, FString
- ✅ Идентификаторы/ссылки на переменные
- ✅ Унарные операторы：`-`, `+`, `not`, `*`（разыменование）
- ✅ Выражения заимствования：`&expr`, `&mut expr`
- ✅ Группировка/кортежи：`(expr)`, `(a, b, c)`
- ✅ Списковые 字面量 и списковые включения：`[1,2,3]`, `[x*x for x in items]`
- ✅ Блочные выражения：`{ stmts; expr }`
- ✅ Управление потоком：`if/elif/else`, `match`, `while`, `for`
- ✅ Ключевое слово `ref`（создание Arc）
- ✅ Блоки `unsafe`
- ✅ Аннотации стратегии вычисления `@block/@auto/@eager`
- ✅ 并发块 `spawn`
- ✅ `return`, `break`, `continue`（с необязательными метками）

**Инфиксные (led)**：
- ✅ Все бинарные операторы：`+`, `-`, `*`, `/`, `%`, `==`, `!=`, `<`, `<=`, `>`, `>=`, `and`, `or`, `..`
- ✅ Присваивание：`=`
- ✅ Вызовы функций：`f(a, b)`, с именованными параметрами `f(x=1, y=2)`
- ✅ Доступ к полям：`obj.field`（цепочки：`a.b.c`）
- ✅ Индексный доступ：`arr[0]`（цепочки：`m[i][j]`）
- ✅ Приведение типов：`expr as Type`
- ✅ Ошибка распространение：`expr?`
- ✅ Lambda：`x => expr`, `(a, b) => expr`, `(x: Int) => x + 1`

**Уровни приоритета (10 уровней)**：Lowest(0) < Assign/Range(1) < Or(2) < And(3) < Eq(4) < Cmp(5) < Add(6) < Mul(7) < Unary/Cast(8) < Call(9) < Highest(10)

### Разбор операторов

- ✅ Объявление переменных：`x = 42`, `x: Int = 42`, `mut x: Int = 0`, `pub x: Int = 42`
- ✅ Определение функций (RFC-010)：`add: (a: Int, b: Int) -> Int = a + b`
- ✅ Определение типов (RFC-010)：`Name: Type = { ... }`
- ✅ Определение методов (RFC-010)：`Point.draw: (self: Point, s: Surface) -> Void = ...`
- ✅ Внешнее связывание (RFC-004)：`Point.distance = distance[0]`
- ✅ Управление потоком：`if/elif/else`, `while`, `for [mut] item in iter`, `return`, `break [label]`, `continue [label]`
- ✅ Импорт：`use path`, `use path.{a, b}`, `use path as alias`
- ✅ Аннотации стратегии вычисления (RFC-001/008)：`@block`, `@auto`, `@eager`
- ✅ Модификатор видимости `pub`

### 类型系统解析

- ✅ Именованные типы：`Int`, `String`, `Bool`, `Float`
- ✅ Метатипы (MetaType)：`Type`（RFC-010 core）
- ✅ Функциональные типы：`(Int, Float) -> Bool`
- ✅ Кортежные типы：`(Int, String, Bool)`
- ✅ Структурные типы：`{ x: Float, y: Float }`
- ✅ Перечисление/вариантные типы：`{ red | green | blue }`, `{ ok(Int) | err(String) }`
- ✅ 泛型 типы：`List(Int)`, `Map(String, Int)`
- ✅ Голые указатели：`*Int`
- ✅ Ссылочные типы：`&T`, `&mut T`
- ✅ Ассоциированные типы：`T::Item`
- ✅ 字面量 типы (const generics)：`n: n`

### Восстановление после ошибок

- ✅ `parse()`：при первой ошибке возвращает `Err`
- ✅ `parse_with_recovery()`：всегда возвращает `ParseResult`, на позиции ошибок вставляются узлы-заглушки `StmtKind::Error` / `Expr::Error`
- ✅ Метод `synchronize()`：переходит к следующей границе оператора для восстановления
