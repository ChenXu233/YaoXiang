# Парсер (Parser)

> **Статус модуля**: стабильный (5 задач к улучшению)
> **Расположение**: `src/frontend/core/parser/`
> **Последнее обновление**: 2026-06-01

---

## Обзор модуля

Парсер отвечает за преобразование потока Token в AST (абстрактное синтаксическое дерево). Используется классический алгоритм Pratt Parsing (нисходящий парсинг с приоритетом операторов), поддерживается полная спецификация синтаксиса языка YaoXiang.

**Объём кода**: около 5000 строк (31 исходный файл, из них 14 тестовых файлов)

---

## Список функций

### Разбор выражений (Pratt Parser)

**Префикс (nud)**:
- ✅ Все литералы: Int, Float, String, Char, Bool, FString
- ✅ Идентификатор/ссылка на переменную
- ✅ Унарные операторы: `-`, `+`, `not`, `*` (разыменование)
- ✅ Выражения заимствования: `&expr`, `&mut expr`
- ✅ Группировка/кортеж: `(expr)`, `(a, b, c)`
- ✅ Литералы списков и list comprehension: `[1,2,3]`, `[x*x for x in items]`
- ✅ Блочные выражения: `{ stmts; expr }`
- ✅ Поток управления: `if/elif/else`, `match`, `while`, `for`
- ✅ Ключевое слово `ref` (создание Arc)
- ✅ `unsafe` блок
- ✅ `spawn` параллельный блок (RFC-024)
- ✅ `return`, `break`, `continue` (с опциональными метками)

**Инфикс (led)**:
- ✅ Все бинарные операторы: `+`, `-`, `*`, `/`, `%`, `==`, `!=`, `<`, `<=`, `>`, `>=`, `and`, `or`, `..`
- ✅ Присваивание: `=`
- ✅ Вызов функции: `f(a, b)`, с именованными параметрами `f(x=1, y=2)`
- ✅ Доступ к полю: `obj.field` (цепочечный: `a.b.c`)
- ✅ Доступ по индексу: `arr[0]` (цепочечный: `m[i][j]`)
- ✅ Приведение типа: `expr as Type`
- ✅ Распространение ошибки: `expr?`
- ✅ Lambda: `x => expr`, `(a, b) => expr`, `(x: Int) => x + 1`

**Уровни приоритета (10 уровней)**: Lowest(0) < Assign/Range(1) < Or(2) < And(3) < Eq(4) < Cmp(5) < Add(6) < Mul(7) < Unary/Cast(8) < Call(9) < Highest(10)

### Разбор операторов

- ✅ Объявление переменной: `x = 42`, `x: Int = 42`, `mut x: Int = 0`, `pub x: Int = 42`
- ✅ Определение функции (RFC-010): `add: (a: Int, b: Int) -> Int = a + b`
- ✅ Определение типа (RFC-010): `Name: Type = { ... }`
- ✅ Определение метода (RFC-010): `Point.draw: (self: Point, s: Surface) -> Void = ...`
- ✅ Внешнее связывание (RFC-004): `Point.distance = distance[0]`
- ✅ Поток управления: `if/elif/else`, `while`, `for [mut] item in iter`, `return`, `break [label]`, `continue [label]`
- ✅ Импорт: `use path`, `use path.{a, b}`, `use path as alias`
- ✅ Модификатор видимости `pub`

### Разбор type system

- ✅ Именованные типы: `Int`, `String`, `Bool`, `Float`
- ✅ Meta type (MetaType): `Type` (ядро RFC-010)
- ✅ Function type: `(Int, Float) -> Bool`
- ✅ Tuple type: `(Int, String, Bool)`
- ✅ Тип структуры: `{ x: Float, y: Float }`
- ✅ Enum/variant type: `{ red | green | blue }`, `{ ok(Int) | err(String) }`
- ✅ Generics типы: `List(Int)`, `Map(String, Int)`
- ✅ Сырой указатель: `*Int`
- ✅ Ссылочный тип: `&T`, `&mut T`
- ✅ Associated type: `T::Item`
- ✅ Literal типы (const generics): `n: n`

### Восстановление после ошибок

- ✅ `parse()`: возвращает `Err` при первой ошибке
- ✅ `parse_with_recovery()`: всегда возвращает `ParseResult`, в позицию ошибки вставляются узлы-заполнители `StmtKind::Error` / `Expr::Error`
- ✅ Метод `synchronize()`: переход к границе следующего оператора для восстановления

| RFC | Статус реализации | Описание |
|-----|----------|------|
| RFC-001 Модель параллелизма | ✅ Реализовано | Аннотации `EvalMode` (Block/Auto/Eager) |
| RFC-004 Curry много позиционное связывание | ✅ Реализовано | Синтаксис внешнего связывания `Type.method = func[0,1]` |
| RFC-007 Унификация синтаксиса функций | ✅ Реализовано | Lambda `(a, b) => body`, HM вывод |
| RFC-008 Модель параллелизма runtime | ✅ Реализовано | `spawn { ... }` блок |
| RFC-010 Унифицированный синтаксис типов | ✅ Реализовано | Унифицированная модель `name: type = value`, meta type `Type` |
| RFC-011 Generics type system | ✅ Реализовано | Generics синтаксис `(T: Type, N: Int) -> Type` |
| RFC-012 F-string шаблонные строки | ✅ Реализовано | `f"Hello {name}"` разбирается в узел FString |
| RFC-017 Поддержка LSP | ✅ Реализовано | `parse_with_recovery()` + Error узлы-заполнители |
