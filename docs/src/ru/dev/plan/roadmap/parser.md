# Парсер (Parser)

> **Статус модуля**：Завершено
> **Расположение**：`src/frontend/core/parser/`
> **Последнее обновление**：2026-06-01

---

## Обзор модуля

Парсер отвечает за преобразование потока Token в AST (абстрактное синтаксическое дерево). Использует классический алгоритм Pratt Parsing (сверху вниз с приоритетом операторов), поддерживающий полную спецификацию синтаксиса языка YaoXiang.

**Объём кода**：около 5000 строк (31 исходный файл, из них 14 тестовых файлов)

---

## Список функций

### Разбор выражений (Pratt Parser)

**Префиксные (nud)**：
- ✅ Все 字面量：Int, Float, String, Char, Bool, FString
- ✅ Идентификаторы/ссылки на переменные
- ✅ Унарные операторы：`-`, `+`, `not`, `*`（разыменование）
- ✅ Выражения заимствования：`&expr`, `&mut expr`
- ✅ Группировка/кортежи：`(expr)`, `(a, b, c)`
- ✅ 字面量 списка и списковые включения：`[1,2,3]`, `[x*x for x in items]`
- ✅ Блочные выражения：`{ stmts; expr }`
- ✅ Управление потоком：`if/elif/else`, `match`, `while`, `for`
- ✅ Ключевое слово `ref`（создание Arc）
- ✅ Блоки `unsafe`
- ✅ Аннотации стратегии вычисления `@block/@auto/@eager`
- ✅ Блоки 并发 `spawn`
- ✅ `return`, `break`, `continue`（с необязательными метками）

**Инфиксные (led)**：
- ✅ Все бинарные операторы：`+`, `-`, `*`, `/`, `%`, `==`, `!=`, `<`, `<=`, `>`, `>=`, `and`, `or`, `..`
- ✅ Присваивание：`=`
- ✅ Вызовы функций：`f(a, b)`, с именованными параметрами `f(x=1, y=2)`
- ✅ Доступ к полям：`obj.field`（цепочка：`a.b.c`）
- ✅ Индексный доступ：`arr[0]`（цепочка：`m[i][j]`）
- ✅ Приведение типов：`expr as Type`
- ✅ 错误传播：`expr?`
- ✅ Lambda：`x => expr`, `(a, b) => expr`, `(x: Int) => x + 1`

**Уровни приоритета (10 уровней)**：Lowest(0) < Assign/Range(1) < Or(2) < And(3) < Eq(4) < Cmp(5) < Add(6) < Mul(7) < Unary/Cast(8) < Call(9) < Highest(10)

### Разбор операторов

- ✅ Объявления переменных：`x = 42`, `x: Int = 42`, `mut x: Int = 0`, `pub x: Int = 42`
- ✅ Определения функций (RFC-010)：`add: (a: Int, b: Int) -> Int = a + b`
- ✅ Определения типов (RFC-010)：`Name: Type = { ... }`
- ✅ Определения методов (RFC-010)：`Point.draw: (self: Point, s: Surface) -> Void = ...`
- ✅ Внешние 绑定 (RFC-004)：`Point.distance = distance[0]`
- ✅ Управление потоком：`if/elif/else`, `while`, `for [mut] item in iter`, `return`, `break [label]`, `continue [label]`
- ✅ Импорт：`use path`, `use path.{a, b}`, `use path as alias`
- ✅ Аннотации стратегии вычисления (RFC-001/008)：`@block`, `@auto`, `@eager`
- ✅ Модификатор видимости `pub`

### Разбор 类型系统

- ✅ Именованные типы：`Int`, `String`, `Bool`, `Float`
- ✅ 元类型 (MetaType)：`Type`（RFC-010 core）
- ✅ Функциональные типы：`(Int, Float) -> Bool`
- ✅ Типы кортежей：`(Int, String, Bool)`
- ✅ Типы структур：`{ x: Float, y: Float }`
- ✅ Типы перечислений/вариантов：`{ red | green | blue }`, `{ ok(Int) | err(String) }`
- ✅ 泛型 типы：`List(Int)`, `Map(String, Int)`
- ✅ Голые указатели：`*Int`
- ✅ Типы ссылок：`&T`, `&mut T`
- ✅ 关联类型：`T::Item`
- ✅ 字面ла типы (const generics)：`n: n`

### 错误恢复

- ✅ `parse()`：при обнаружении первой ошибки возвращает `Err`
- ✅ `parse_with_recovery()`：всегда возвращает `ParseResult`, при ошибках вставляет заполнители `StmtKind::Error` / `Expr::Error`
- ✅ Метод `synchronize()`：переход к следующей границе оператора для восстановления
