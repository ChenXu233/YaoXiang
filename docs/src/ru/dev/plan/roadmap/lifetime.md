# Проверка заимствований (Lifetime)

> **Состояние модуля**: Завершено
> **Расположение**: `src/middle/passes/lifetime/`
> **Последнее обновление**: 2026-06-01

---

## Обзор модуля

Модуль проверки заимствований представляет собой полноценную **систему анализа владения и управления жизненным циклом**, ответственную за проверку Move-семантики, конфликтов заимствования, нарушений изменяемости и других проблем, связанных с владением.

**Объём кода**: около 300 КБ исходного кода (15 подфайлов)

## Сравнение с RFC (RFC-009 Модель владения)

| Пункт дизайна RFC | Состояние реализации | Пояснение |
|-------------|---------|------|
| Move-семантика (по умолчанию) | ✅ Реализовано | MoveChecker обнаруживает UseAfterMove |
| Токены заимствования &T/&mut T | ✅ Реализовано | BorrowChecker реализует обнаружение конфликтов токенов |
| &T копируемо (Dup) | ✅ Реализовано | Несколько токенов &T могут существовать одновременно |
| &mut T линейно | ✅ Реализовано | &mut T из одного источника может иметь только один активный токен |
| Механизм заморозки | ✅ Реализовано | freeze() преобразует &mut T в замороженный &T |
| Обнаружение конфликтов токенов (потоково-чувствительный анализ активности) | ✅ Реализовано | Отслеживание состояния токенов внутри тела функции |
| Ключевое слово ref (автоматический выбор Rc/Arc) | ⚠️ Частично реализовано | Проверка семантики ref существует |
| clone() для явного глубокого копирования | ✅ Реализовано | CloneChecker обнаруживает клонирование перемещённых/высвобожденных значений |
| unsafe + *T | ✅ Реализовано | UnsafeChecker проверяет операции с голыми указателями вне блоков unsafe |
| Внутризадачные циклы: тихое разрешение | ✅ Реализовано | IntraTaskCycleTracker отслеживает в режиме предупреждений |
| Межзадачные циклы: линтер | ✅ Реализовано | CycleChecker обнаруживает циклические ссылки между spawn |
| Без жизненных циклов 'a | ✅ Соответствует дизайну | Параметры жизненного цикла не реализованы |
| Ограничения Send/Sync | ✅ Реализовано | SendSyncChecker работает независимо от OwnershipChecker |
