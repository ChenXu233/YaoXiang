# Проверщик заимствований (Lifetime)

> **Статус модуля**: переходный период — архитектура линейного сканирования v8 → конвейер холловских утверждений v9
> **Расположение**: `src/middle/passes/lifetime/`
> **Последнее обновление**: 2026-06-13
>
> **Связанные RFC**:
> - [RFC-009: Дизайн модели владения](../design/rfc/accepted/009-ownership-model.md) — принято
> - [RFC-009a: Анализ жизненного цикла токенов — конвейер доказательств на основе Хоара](../design/rfc/accepted/009a-borrow-proof-pipeline.md) — принято

---

## Обзор модуля

Модуль проверщика заимствований отвечает за анализ владения в YaoXiang — семантику Move, конфликты токенов заимствования, корректность Drop/Clone, обнаружение циклов ref, нарушения изменяемости.

**Текущая архитектура** (переходный период):
- ir_gen жёстко вставляет инструкции Borrow/Release (лексическая область видимости)
- BorrowChecker линейно сканирует IR, пассивно проверяя конфликты токенов
- ControlFlowAnalyzer существует, но его ядро пустое
- Поведение, видимое пользователю, в основном корректное, но на нижнем уровне это не конвейер холловских утверждений RFC-009a

**Целевая архитектура** (RFC-009 + RFC-009a):
- Дерево брендов отслеживает отношения порождения токенов
- Анализ потребителей управляет освобождением NLL
- Обратный BFS-анализ живости (быстрый канал покрывает 95%+ сценариев)
- Резервное логическое отсечение SMT (крайне редкие сценарии while + условия пути)
- Release управляется анализом областей видимости, не жёстко закодирован в ir_gen

**Объём кода**: около 300 КБ исходного кода (15 подфайлов)

## Статус соответствия RFC

### Пять ключевых концепций RFC-009

| Концепция | Поведение, видимое пользователю | Реализация на нижнем уровне |
|------|-------------|---------|
| **Move** | ✅ Завершено | MoveChecker, обнаружение UseAfterMove |
| **&T / &mut T** | ✅ Завершено | BorrowChecker линейно сканирует, пассивно реагирует на инструкции Borrow/Release |
| **ref** | ⚠️ Обнаружение циклов завершено, анализ побега отсутствует | ref_semantics + cycle_check + intra_task_cycle |
| **clone()** | ✅ Завершено | CloneChecker, 0 тестов |
| **unsafe + *T** | ✅ Завершено | UnsafeChecker |

### Шесть этапов RFC-009a

| Этап | Содержание | Статус | Описание |
|------|------|------|------|
| 1 | Структура данных дерева брендов | ❌ Не начато | Заменить `HashMap<String, BorrowToken>`, ID бренда + родительский узел + список потребителей |
| 2 | Анализ потребителей | ❌ Не начато | Автоматический сбор потребителей токенов при построении DAG, основа NLL |
| 3 | Обратный BFS-анализ живости | ❌ Не начато | Дерево брендов + потребители + отсечение break → покрытие 95%+ сценариев |
| 4 | Release, управляемый областью видимости | ❌ Не начато | Удалить жёсткое кодирование ir_gen, вставка LIFO в точках выхода из области видимости, автоматическая обработка `?` |
| 5 | Логическое отсечение SMT | ❌ Не начато | Заблокировано фазой 2 RFC-027, срабатывает только при while + условии пути |
| 6 | Очистка | ❌ Не начато | BorrowChecker → BorrowPredicateEmitter, удалить ControlFlowAnalyzer |
