# Проверка заимствований (Lifetime)

> **Состояние модуля**: стабильное (4 улучшения в очереди)
> **Расположение**: `src/middle/passes/lifetime/`
> **Последнее обновление**: 2026-06-01

---

## Обзор модуля

Модуль проверки заимствований представляет собой полную систему **анализа владения и управления временем жизни**, отвечающую за проверку семантики Move, конфликтов заимствования, нарушений изменяемости и других проблем, связанных с владением.

**Объём кода**: около 300 КБ исходного кода (15 подфайлов)

## Сравнение с RFC (RFC-009 модель владения)

| Пункт дизайна RFC | Состояние реализации | Пояснение |
|-------------------|----------------------|-----------|
| Семантика Move (по умолчанию) | ✅ Реализовано | MoveChecker обнаруживает UseAfterMove |
| Токены заимствования &T/&mut T | ✅ Реализовано | BorrowChecker реализует обнаружение конфликтов токенов |
| &T копируемо (Dup) | ✅ Реализовано | Несколько токенов &T могут существовать одновременно |
| &mut T линейно | ✅ Реализовано | От одного источника &mut T может быть только один активный |
| Механизм заморозки | ✅ Реализовано | freeze() замораживает &mut T в &T |
| Обнаружение конфликтов токенов (потоково-чувствительный анализ активности) | ✅ Реализовано | Отслеживание состояния токенов внутри тела функции |
| Ключевое слово ref (автоматический выбор Rc/Arc) | ⚠️ Частично реализовано | Проверка семантики ref существует |
| clone() явное глубокое копирование | ✅ Реализовано | CloneChecker обнаруживает клонирование перемещённых/освобождённых значений |
| unsafe + *T | ✅ Реализовано | UnsafeChecker проверяет операции с голыми указателями вне блока unsafe |
| Внутризадачные циклы: разрешены без ошибок | ✅ Реализовано | IntraTaskCycleTracker отслеживает в режиме предупреждений |
| Межзадачные циклы: линтер | ✅ Реализовано | CycleChecker обнаруживает циклические ссылки между spawn |
| Без времени жизни 'a | ✅ Соответствует дизайну | Параметры времени жизни не реализованы |
| Ограничения Send/Sync | ✅ Реализовано | SendSyncChecker независим от OwnershipChecker |
