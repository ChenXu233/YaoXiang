## Почти завершённые RFC (8)

Осталось мало работы, каждому RFC не хватает всего 1–2 функций.

### RFC-006: Создание сайта документации

**Завершено**: Сайт на VitePress, многоязычная поддержка (zh, en, ja, ru), автоматическое развёртывание CI/CD, настройка боковой панели/навигации, функция поиска

**Осталось**:
- ❌ Меню переключения версий (формат `/v0.5/zh/`)

### RFC-007: Унификация синтаксиса определения функций

**Завершено**: Синтаксический разбор `name: (params) -> Return = body`, синтаксис лямбда-выражений, вывод типов HM, унификация синтаксиса определения функций

**Осталось**:
- ❌ Инструмент миграции `yaoxiang-migrate`

### RFC-012: Шаблонные строки F-String

**Завершено**: Префиксный синтаксис `f"Hello {name}"`, интерполяция переменных/вычисление выражений, преобразование в строковые операции на этапе компиляции

**Осталось**:
- ❌ Полная поддержка спецификаторов форматирования (`:.2f` и т.д.)

### RFC-013: Проект спецификации кодов ошибок

**Завершено**: Четырёхзначная цифровая нумерация `Exxxx`, многоязычные файлы ресурсов (JSON i18n), общий конструктор `DiagnosticBuilder`, макрос `error!`

**Осталось**:
- ⚠️ Команда CLI `yaoxiang explain` ожидает подтверждения

### RFC-023: Модель захвата замыканий

**Завершено**: Компилятор автоматически анализирует внешние переменные, на которые ссылается тело замыкания, прямое копирование типов Dup, нулевые аннотации

**Осталось**:
- ⚠️ Полный анализ побега ожидает доработки

### RFC-009: Модель владения

**Завершено**: Семантика Move (по умолчанию), токены заимствования &T/&mut T, явное глубокое копирование clone(), unsafe + *T, обнаружение циклов между задачами, ограничения Send/Sync, обнаружение конфликтов токенов

**Осталось**:
- ❌ Механизм брендинга (уникальная идентификация внутренних токенов компилятора)
- ❌ Анализ побега для автоматического выбора Rc/Arc для ref

### RFC-010: Унифицированный синтаксис типов

**Завершено**: Унифицированный синтаксис объявлений `name: type = value`, определение record type, определение интерфейса и проверка реализации, синтаксис определения методов (Type.method), синтаксис вызова дженериков `()`, проверка структурной подтипизации

**Осталось**:
- ❌ Полная поддержка утиной типизации
- ❌ Композиция интерфейсов (тип-пересечение `Drawable & Serializable`)

### RFC-017: Поддержка языкового сервера LSP

**Завершено**: Независимый процесс LSP-сервера, коммуникация по JSON-RPC, автодополнение кода/переход к определению/диагностика/поиск ссылок/всплывающие подсказки, призрачные подсказки (Inlay Hints), семантические токены, переименование, операции с кодом

**Осталось**:
- ❌ Инкрементальная синхронизация
- ❌ Удалённая связь TCP/Unix Socket
- ❌ Адаптер отладки DAP
- ❌ Визуализация семантики владения

## RFC в процессе реализации (4)

Основные функции запущены, но остаётся большой объём незавершённой работы.

### RFC-001: Модель spawn и обработка ошибок (устарел)

**Заменён RFC-024**. Старые аннотации `@block`/`@eager`/`@auto`, trait-ы `Send`/`Sync`, схема анализа DAG всей программы устарели. Новая модель параллелизма описана в RFC-024.

### RFC-004: Многопозиционное объединённое связывание каррированных методов

**Завершено**: Базовый синтаксис связывания методов (определение Type.method), базовый синтаксический сахар вызова методов

**Осталось**:
- ❌ Синтаксис индексного связывания позиций `[positions]` (`[0]`, `[-1]` и т.д.)
- ❌ Многопозиционное объединённое связывание `[0, 1]`
- ❌ Автоматическое каррированное связывание
- ❌ Логика связывания по умолчанию

### RFC-011: Проект системы дженериков

**Завершено**: Унифицированный синтаксис сигнатур `(T: Type, R: Type) -> ...`, механизм самоописания Type, ограничения типов `T: Dup + Add`, ассоциированные типы (GAT), специализация перегрузки функций, мономорфизация, удаление мёртвого кода

**Осталось**:
- ❌ Зависимые от значений типы (`Vec: (n: Int) -> Type`)
- ❌ Движок вычислений на этапе компиляции (вычисление вызовов функций в позициях типов на этапе компиляции)
- ❌ Спецификация decreases (проверка завершаемости)
- ❌ Условные типы `If: (C: Bool, T: Type, E: Type) -> Type`
- ❌ Семейства типов

### RFC-015: Проект системы конфигурации

**Завершено**: Базовый модуль конфигурации, разбор `yaoxiang.toml` уровня проекта

**Осталось**:
- ❌ Конфигурация пользовательского уровня `~/.config/yaoxiang/config.toml`
- ❌ Логика слияния конфигураций
- ❌ Команда CLI `yaoxiang config`
- ❌ Переопределение через командную строку/переменные среды
- ❌ Расширение `[tool.*]`
- ❌ Ограничение `platform`

## Не начатые RFC (1)

### RFC-008: Отделение модели параллелизма Runtime от планировщика (Фаза A завершена)

**Завершено**: Трёхуровневый Runtime Embedded/Standard/Full, планировщик DAG, кооперативная нарезка времени, сериализация ресурсов, распространение ошибок/отмены

**Осталось**:
- ❌ Анализ DAG на этапе компиляции (покрывается RFC-024)
- ❌ AOT-бэкенд LLVM
- ❌ Универсальный интерфейс планировщика
- ❌ Линковка статической библиотеки планировщика
