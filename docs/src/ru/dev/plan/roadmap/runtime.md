# Runtime (Среда выполнения)

> **Состояние модуля**: Имеются пробелы (4 пункта требуют улучшения, этап B не начат)
> **Расположение**: `src/backends/runtime/`

---

## Обзор модуля

Модуль runtime отвечает за планирование задач и параллельное выполнение. Реализует трёхуровневую архитектуру runtime, определённую в RFC-008: Embedded / Standard / Full.

**Объём кода**: около 95 КБ (4 исходных файла)

---

## Список функций

### engine.rs — Ядро планирования DAG (реализовано)

- ✅ **Управление жизненным циклом задач**: spawn / mark_running / complete / cancel / yield_now
- ✅ **Граф зависимостей DAG**: жёсткие зависимости (hard_deps, с распространением ошибок/отмены) + контрольные зависимости (control_deps, только упорядочение без распространения ошибок)
- ✅ **Сериализация ресурсов**: механизм ResourceKey, задачи с одинаковым ключом строго сериализуются, при отмене ожидают контрольные зависимости для сохранения порядка ресурсов
- ✅ **Обнаружение циклических зависимостей**: проверка достижимости в add_dependency, возврат ошибки CycleDetected
- ✅ **Распространение ошибок/отмены**: ошибки распространяются по рёбрам жёстких зависимостей через BFS, при множественных ошибках причины отмены объединяются (primary + others)
- ✅ **Кооперативное временное срезание**: drive_until_polled поддерживает TaskPoll::Pending для передачи управления, честное переключение между задачами
- ✅ **Целевое приоритетное планирование**: next_ready_for(target) приоритизирует цепочку зависимостей цели, изолированные задачи не блокируют завершение цели
- ✅ **Статистика**: RuntimeStats (pending/running/completed/failed/cancelled/total_spawned/avg_execution_time)

### facade.rs — Трёхуровневый фасад runtime (реализовано)

- ✅ **Embedded Runtime**: немедленное выполнение, замыкание запускается сразу при spawn, без DAG, без поддержки deps/resources
- ✅ **Standard Runtime**: однопоточное DAG-планирование, поддержка обычных TaskFn и кооперативных CoopTaskFn
- ✅ **Full Runtime**: многопоточное выполнение, связь через crossbeam channel, пул рабочих потоков
- ✅ **Единый фасад Runtime**: конфигурация через RuntimeConfig(mode, workers, work_stealing)

### task.rs — Абстракция задачи (реализовано)

- ✅ **TaskId**: уникальный идентификатор, поддержка Display
- ✅ **TaskPriority**: четыре уровня — Low/Normal/High/Critical
- ✅ **TaskState**: Pending/Running/Completed/Failed/Cancelled
- ✅ **TaskConfig**: конфигурация через builder (priority/name/stack_size/parent_id)
- ✅ **Task**: сущность задачи, содержит id/config/state/result
- ✅ **TaskContext**: контекст выполнения задачи (registers/stack/locals/entry_ip)
- ✅ **Scheduler trait**: абстрактный интерфейс планировщика
- ✅ **TaskSpawner**: обёртка обобщённого планировщика задач

## Ключевые находки

1. **trait Scheduler в task.rs отделён от реального планирования в facade.rs**: task.rs определяет trait Scheduler, но facade.rs не использует этот trait, а напрямую распределяет через enum
2. **В task.rs имеются повторяющиеся определения типов**: SyncValue, TaskResult, RuntimeError, SchedulerStats определены и в engine.rs, и в task.rs
3. **WorkStealing фактически не реализован**: поле work_stealing есть в RuntimeConfig, но FullRuntime работает как простой пул потоков с channel
