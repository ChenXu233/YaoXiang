## Обзор модуля

Модуль среды выполнения отвечает за планирование задач и параллельное выполнение. Реализует трёхуровневую архитектуру среды выполнения, определённую в RFC-008: Embedded / Standard / Full.

**Объём кода**: ~95KB (4 исходных файла)

---

## Список функций

### engine.rs — Ядро планирования DAG (реализовано)

- ✅ **Управление жизненным циклом задач**: spawn / mark_running / complete / cancel / yield_now
- ✅ **График зависимостей DAG**: жёсткие зависимости (hard_deps, распространение ошибок/отмены) + контрольные зависимости (control_deps, только упорядочение без распространения ошибок)
- ✅ **Сериализация ресурсов**: механизм ResourceKey, задачи с одинаковым ключом строго сериализуются, при отмене ожидают контрольных зависимостей для сохранения порядка ресурсов
- ✅ **Обнаружение циклических зависимостей**: проверка достижимости в add_dependency, возврат ошибки CycleDetected
- ✅ **Распространение ошибок/отмены**: ошибки распространяются BFS по рёбрам жёстких зависимостей, при одновременных ошибках нескольких зависимостей причины отмены объединяются (primary + others)
- ✅ **Кооперативное временное срезание**: drive_until_polled поддерживает TaskPoll::Pending для передачи управления, справедливое переключение между задачами
- ✅ **Целевое приоритетное планирование**: next_ready_for(target) приоритетно продвигает цепочку зависимостей цели, изолированные задачи не блокируют завершение цели
- ✅ **Статистика**: RuntimeStats (pending/running/completed/failed/cancelled/total_spawned/avg_execution_time)

### facade.rs — Фасад трёхуровневой среды выполнения (реализовано)

- ✅ **Embedded Runtime**: немедленное выполнение, замыкание выполняется сразу при spawn, без DAG, не поддерживает deps/resources
- ✅ **Standard Runtime**: однопоточное планирование DAG, поддержка обычных TaskFn и кооперативных CoopTaskFn
- ✅ **Full Runtime**: многопоточное выполнение, связь через crossbeam channel, пул рабочих потоков
- ✅ **Унифицированный фасад Runtime**: конфигурация через RuntimeConfig(mode, workers, work_stealing)

### task.rs — Абстракция задачи (реализовано)

- ✅ **TaskId**: уникальный идентификатор, поддержка Display
- ✅ **TaskPriority**: четыре уровня — Low/Normal/High/Critical
- ✅ **TaskState**: Pending/Running/Completed/Failed/Cancelled
- ✅ **TaskConfig**: конфигурация паттерном builder (priority/name/stack_size/parent_id)
- ✅ **Task**: сущность задачи, содержит id/config/state/result
- ✅ **TaskContext**: контекст выполнения задачи (registers/stack/locals/entry_ip)
- ✅ **Scheduler trait**: абстрактный интерфейс планировщика
- ✅ **TaskSpawner**: обёртка универсального планировщика задач

## Ключевые находки

1. **Scheduler trait в task.rs и реальное планирование в facade.rs разделены**: task.rs определяет Scheduler trait, но facade.rs не использует этот trait, а напрямую распределяет через enum
2. **В task.rs присутствуют дублирующие определения типов**: SyncValue, TaskResult, RuntimeError, SchedulerStats определены и в engine.rs, и в task.rs
3. **WorkStealing фактически не реализован**: в RuntimeConfig есть поле work_stealing, но FullRuntime использует простую модель пула потоков + channel
