## Обзор модуля

Модуль рантайма отвечает за планирование задач и параллельное выполнение. Реализует определённую в RFC-008 трёхуровневую архитектуру рантайма: Embedded / Standard / Full.

**Объём кода**: ~95 КБ (4 файла с исходниками)

---

## Список функций

### engine.rs — ядро планирования DAG (реализовано)

- ✅ **Управление жизненным циклом задач**: spawn / mark_running / complete / cancel / yield_now
- ✅ **Граф зависимостей DAG**: жёсткие зависимости (hard_deps, с распространением ошибок/отмены) + управляющие зависимости (control_deps, только упорядочивают, не распространяют ошибки)
- ✅ **Сериализация ресурсов**: механизм ResourceKey, задачи с одинаковым ключом строго сериализуются, при отмене ожидание управляющих зависимостей для сохранения порядка ресурсов
- ✅ **Обнаружение циклических зависимостей**: при add_dependency обнаружение цикла через проверку достижимости, возврат ошибки CycleDetected
- ✅ **Распространение ошибок/отмены**: ошибки распространяются по рёбрам жёстких зависимостей через BFS, при одновременном сбое нескольких зависимостей причины отмены объединяются (primary + others)
- ✅ **Кооперативная нарезка времени**: drive_until_polled поддерживает уступку TaskPoll::Pending, честная круговая обработка нескольких задач
- ✅ **Приоритетное планирование цели**: next_ready_for(target) в первую очередь продвигает цепочку зависимостей цели, изолированные задачи не блокируют завершение цели
- ✅ **Статистика**: RuntimeStats (pending/running/completed/failed/cancelled/total_spawned/avg_execution_time)

### facade.rs — фасад трёхуровневого рантайма (реализовано)

- ✅ **Embedded Runtime**: немедленное выполнение, при spawn замыкание запускается сразу, без DAG, без поддержки deps/resources
- ✅ **Standard Runtime**: однопоточное планирование DAG, поддержка обычного TaskFn и кооперативного CoopTaskFn
- ✅ **Full Runtime**: многопоточное выполнение, коммуникация через crossbeam channel, пул потоков-воркеров
- ✅ **Унифицированный фасад Runtime**: конфигурируется через RuntimeConfig (mode, workers, work_stealing)

### task.rs — абстракция задач (реализовано)

- ✅ **TaskId**: уникальный идентификатор, поддерживает Display
- ✅ **TaskPriority**: четыре уровня — Low / Normal / High / Critical
- ✅ **TaskState**: Pending / Running / Completed / Failed / Cancelled
- ✅ **TaskConfig**: конфигурация в стиле builder (priority / name / stack_size / parent_id)
- ✅ **Task**: сущность задачи, содержит id / config / state / result
- ✅ **TaskContext**: контекст выполнения задачи (registers / stack / locals / entry_ip)
- ✅ **trait Scheduler**: абстрактный интерфейс планировщика
- ✅ **TaskSpawner**: обёртка планировщика задач с дженериками

## Ключевые находки

1. **trait Scheduler в task.rs отделён от фактического планирования в facade.rs**: task.rs определяет trait Scheduler, но facade.rs его не использует, а напрямую диспетчеризирует через enum.
2. **task.rs содержит дублирующиеся определения типов**: SyncValue, TaskResult, RuntimeError, SchedulerStats определены и в engine.rs, и в task.rs.
3. **WorkStealing фактически не реализован**: RuntimeConfig содержит поле work_stealing, но FullRuntime фактически использует простую модель «пул потоков + channel».
4. **RFC-024 изменит модель выполнения spawn**: сейчас spawn целиком оборачивается в одно замыкание и планируется DAG-ом рантайма; RFC-024 планирует анализ зависимостей непосредственных подвыражений внутри spawn-блока в compile-time, генерацию плана выполнения и параллельное выполнение по группам в рантайме.
