# План реализации FFI

## Обзор

FFI (Foreign Function Interface) — механизм, позволяющий коду YaoXiang вызывать нативные функции Rust, выступает мостом между runtime языка и системными API. В данном плане реализуется FFI с привязкой на этапе компиляции, поддерживающий:

- Вызов системных API Rust через функции стандартной библиотеки (std.io)
- Пользовательские нативные функции (native)

### Цели проектирования

| Цель | Описание |
|------|----------|
| Нулевые накладные расходы во время выполнения | Привязка на этапе компиляции, без поиска после кэширования |
| Типобезопасность | Проверка сигнатур функций компилятором |
| Расширяемость | Пользователи могут объявлять произвольные нативные функции |
| Без нового синтаксиса | Повторное использование модели `name: type = value` |

### Обзор архитектуры

```
┌─────────────────────────────────────────────────────────┐
│  Компиляция                                             │
│  ───────────────────────────────────────────────────  │
│                                                          │

                          │
```

## Шаг 1: Создание инфраструктуры FFI-реестра

### Файл

`src/backends/interpreter/ffi.rs` (новый)

### Реализуемое содержимое

| Содержимое | Описание |
|------|------|
| Тип `NativeHandler` | `fn(&[RuntimeValue]) -> Result<RuntimeValue, ExecutorError>` |
| Структура `FfiRegistry` | `handlers: HashMap<String, NativeHandler>` + кэш |
| Метод `with_std()` | Предварительная регистрация функций std.io |
| Метод `register()` | Пользовательская регистрация новых функций |
| Метод `call()` | Вызов функции с кэшированием |

### Структура основного кода

```rust
pub struct FfiRegistry {
    // Таблица обработчиков функций
    handlers: HashMap<String, NativeHandler>,
    // Кэш времени выполнения (ускорение вызовов)
    cache: Mutex<HashMap<String, NativeHandler>>,
}

impl FfiRegistry {
    // Предопределённые функции стандартной библиотеки
    pub fn with_std() -> Self { ... }

    // Пользовательская регистрация функций
    pub fn register(&mut self, name: &str, handler: NativeHandler) { ... }

    // Вызов: поиск в кэше → выполнение
    pub fn call(&self, name: &str, args: &[RuntimeValue]) -> Result<RuntimeValue> { ... }
}
```

### Критерии приёмки

- [x] `FfiRegistry::new()` возвращает реестр с функциями std.io
- [x] `register()` может добавлять новые функции
- [x] `call()` корректно вызывает зарегистрированные функции

## Шаг 7: Поддержка пользовательских нативных функций

### Файл

`src/std/ffi.rs` (новый)

### Реализуемое содержимое

| Содержимое | Описание |
|------|------|
| Определение типа `Native` | Маркер для объявления нативных функций пользователем |
| Функция `register` | Пользовательская регистрация собственной логики обработки нативных функций |

### Способ использования пользователем

**Объявление в исходном коде YaoXiang:**

```yaoxiang
# Объявление привязки нативной функции
my_add: (a: Int, b: Int) -> Int = Native("my_add")

# Вызов (компилятор автоматически генерирует байткод CallNative)
result = my_add(1, 2)
```

**Регистрация встраиваемого API на Rust:**

```rust
// Регистрация обработчика нативной функции на стороне Rust
interpreter.ffi_registry_mut().register("my_add", |args| {
    let a = args[0].to_int().unwrap_or(0);
    let b = args[1].to_int().unwrap_or(0);
    Ok(RuntimeValue::Int(a + b))
});
```

### Реализуемое содержимое

| Содержимое | Описание |
|------|------|
| Структура `NativeBinding` | Привязка нативной функции, объявленной пользователем (func_name → native_symbol) |
| `detect_native_binding()` | Обнаружение паттерна `Native("...")` в AST |
| `ModuleIR.native_bindings` | Передача информации о нативных привязках на уровне IR |
| Интеграция IR-генератора | Обнаружение `= Native("symbol")` пропуск генерации тела функции, запись привязки |
| Интеграция Translator | Автоматическая регистрация пользовательских нативных функций перед `translate_module` |
