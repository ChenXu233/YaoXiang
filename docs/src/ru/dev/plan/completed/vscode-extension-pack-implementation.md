**Цель**:
- Создать каталог `vscode-extension/` в корневом каталоге проекта
- Установить стандартную структуру пакета расширений VS Code

**Структура каталогов**:
```
vscode-extension/
├── package.json                    # Конфигурация расширения
├── language-configuration.json     # Конфигурация языка
├── syntaxes/
│   └── yaoxiang.tmLanguage.json    # Подсветка синтаксиса (опционально)
```

**Критерии приёмки**:

**Тестовые проекты**:

## Шаг 2: Создание package.json

**Цель**:
- Определить идентификатор языка YaoXiang
- Связать расширение файла `.yx`
- Настроить поддержку встроенного LSP-клиента

**Основная конфигурация**:
```json
{
  "name": "yaoxiang",
  "displayName": "YaoXiang Language",
  "description": "Поддержка языка программирования YaoXiang",
  "languages": [{
    "id": "yaoxiang",
    "aliases": ["YaoXiang", "yx"],
    "extensions": [".yx"]
  }],
  "grammars": {
    "language": "yaoxiang",
    "scopeName": "source.yaoxiang"
  }
}
```

**Критерии приёмки**:
- [ ] package.json содержит корректную конфигурацию идентификатора языка
- [ ] Расширение файла `.yx` связано
- [ ] Отображаемое имя языка — "YaoXiang"

**Тестовые проекты**:
- [ ] Проверка синтаксиса package.json
- [ ] Проверка полноты конфигурации

## Шаг 3: Создание language-configuration.json

**Цель**:
- Настроить формат однострочных комментариев `//`
- Настроить формат блочных комментариев `/* */`
- Настроить правила соответствия скобок
- Настроить правила автоматического отступа

**Основная конфигурация**:
```json
{
  "comments": {
    "lineComment": "//",
    "blockComment": ["/*", "*/"]
  },
  "brackets": [
    ["{", "}"],
    ["[", "]"],
    ["(", ")"]
  ],
  "indentationRules": {
    "increaseIndentPattern": "^.*\\{[^}]*$",
    "decreaseIndentPattern": "^\\s*\\}"
  }
}
```

**Критерии приёмки**:
- [ ] Однострочные комментарии используют `//`
- [ ] Блочные комментарии используют `/* */`
- [ ] Соответствие скобок работает корректно

**Тестовые проекты**:
- [ ] Открыть файл .yx в VS Code и проверить работу горячей клавиши комментирования (Ctrl+/)
- [ ] Проверить подсветку соответствия скобок

## Шаг 4: (Опционально) Создание подсветки синтаксиса

**Цель**:
- Создать определение синтаксиса TextMate на основе ключевых слов YaoXiang
- Поддержать подсветку ключевых слов, строк, чисел и комментариев

**Список ключевых слов YaoXiang**:
- Управление потоком выполнения (`trait`): `if`, `elif`, `else`, `match`, `while`, `for`, `in`, `return`, `break`, `continue`
- Объявления: `pub`, `use`, `spawn`, `ref`, `mut`
- Типы: `as`, `unsafe`

**Структура синтаксиса TextMate**:
```json
{
  "name": "YaoXiang",
  "patterns": [
    {
      "include": "#keywords"
    },
    {
      "include": "#strings"
    },
    {
      "include": "#numbers"
    },
    {
      "include": "#comments"
    }
  ]
}
```

**Критерии приёмки**:
- [ ] Ключевые слова подсвечиваются корректно
- [ ] Строки подсвечиваются корректно
- [ ] Числа подсвечиваются корректно
- [ ] Комментарии подсвечиваются корректно

**Тестовые проекты**:
- [ ] Открыть файл .yx и проверить эффект подсветки синтаксиса
- [ ] Проверить правильность подсветки для различных типов токенов
