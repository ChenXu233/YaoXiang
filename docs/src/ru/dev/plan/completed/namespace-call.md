# Поддержка вызовов пространств имён

## Обзор

Реализация синтаксиса вызовов пространств имён в форме `std.module.function`, позволяющего вызывать функции модулей, такие как `std.io.print` или `std.math.abs`.

## Текущее состояние

- **Проблема**: `use std.io.*` импортирует короткие имена, но вызов в форме `std.io.print` выдаёт ошибку «Unknown variable: 'std'»
- **Ожидаемое поведение**: пользователи могут вызывать функции в форме `std.<module>.<function>`

## Модули, требующие изменений

### 1. Фронтенд компилятора — парсер

Файл: `src/frontend/parser/`

Необходимо поддержать распознавание выражений-имён в форме `a.b.c` и корректно разбирать пути пространств имён.

### 2. Фронтенд компилятора — проверка типов

Файл: `src/frontend/typecheck/`

При встрече с путём пространства имён необходимо:

1. Распознать `std` как встроенное пространство имён
2. Разобрать имя последующего модуля (`io`, `math`, `net` и т.д.)
3. Найти функцию внутри модуля и проверить её тип

### 3. Генерация IR

Файл: `src/middle/passes/codegen/`

При генерации IR путь пространства имён необходимо преобразовать в целевую ссылку на функцию.

### 4. Интерпретатор/Среда выполнения

Файл: `src/backends/interpreter/executor.rs`

При выполнении обеспечить корректное разрешение пути пространства имён в FFI handler.

## Шаги реализации

1. **Изменения в парсере**: распознавание выражений доступа к полю в форме `a.b`
2. **Семантический анализ**: реализация логики разрешения пространств имён
3. **Генерация кода**: генерация корректных инструкций вызова функций
4. **Тестирование**: добавление тестовых случаев для проверки вызовов вроде `std.io.print`

## Тестовые случаи

```yaoxiang
use std.io

// должно работать
std.io.println("Hello")

// короткие имена тоже должны работать
use std.io.*
println("World")
```

## Связанные файлы

- `src/frontend/parser/` — парсер

- `src/middle/passes/codegen/` — генерация кода
- `src/backends/interpreter/` — интерпретатор
