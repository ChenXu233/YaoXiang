## 1. Обзор

### 1.1 Цели

Повторная реализация синтаксиса прямого присваивания для интерфейсных (контрактных) типов с реализацией оптимизации на этапе компиляции для устранения накладных расходов vtable:

1. **Поддержка прямого присваивания интерфейсов**: `d: Drawable = Circle(1)`
2. **Вывод типов на этапе компиляции**: максимальное определение конкретного типа
3. **Оптимизация с нулевыми накладными расходами**: использование прямых вызовов при определённом конкретном типе
4. **Резервный механизм vtable**: использование vtable при невозможности определения конкретного типа

### 1.2 Текущее состояние

| Функционал | Статус |
|------------|--------|
| Реализация vtable в runtime | ✅ Реализовано |
| IR виртуальных вызовов методов | ✅ Реализовано |
| Обобщённые ограничения `[T: Drawable]` | ✅ Реализовано |
| Прямое присваивание интерфейса `s: Drawable = point` | ✅ Реализовано |
| Проверка подтипов для ограничений | ✅ Реализовано |

| Реализация исполнителя CallVirt | ✅ Реализовано |

### 1.3 Изменения в дизайне

**До изменений**:
```yaoxiang

```

**После изменений**:
```yaoxiang

# - Тип не может быть определён → используется vtable
```

**Новое содержимое раздела**:

Новый раздел «Раздел 6: Стратегия оптимизации на этапе компиляции»:

1. **Вызовы интерфейсов с нулевыми накладными расходами**: когда правая часть присваивания интерфейса является конкретным типом и может быть определена, компилятор оптимизирует вызов до нулевых накладных расходов

   - Исходный код: `d: Drawable = Circle(1); d.draw(screen)`
   - После компиляции: прямой вызов `circle_draw(screen)`, без vtable

2. **Правила вывода**:

   | Сценарий | Результат вывода | Способ вызова |
   |----------|------------------|---------------|
   | Прямое присваивание конкретного типа | Конкретный тип Circle | Прямой вызов |
   | Возвращаемое значение функции | Неизвестен | vtable |
   | Гетерогенная коллекция | Несколько типов | vtable |

3. **Резервный механизм vtable**: при невозможности определения конкретного типа используется механизм vtable
