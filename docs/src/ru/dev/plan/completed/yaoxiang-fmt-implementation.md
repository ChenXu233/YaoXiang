> **Задача**: реализовать инструмент `yaoxiang fmt` для форматирования кода

> **Статус**:全部完成 ✅

> **Дата завершения**: 2026-03-01

## Обзор

Настоящий план, основанный на документе требований issue_13, разбивает инструмент форматирования `yaoxiang fmt` на несколько этапов реализации. Каждый этап содержит подробные цели, критерии приёмки и тестовые задания.

### Выбор архитектуры

Применяется схема **AST + исходное отображение**:
1. **Lexer** записывает позиции комментариев/пробелов → генерирует поток токенов + исходное отображение
2. **Parser** генерирует AST (с информацией о Span)
3. **Formatter** обходит AST + исходное отображение согласно конфигурации → выводит отформатированный код

**Ключевые преимущества**:
- Сохранение полной информации о комментариях
- Поддержка сложной реструктуризации кода
- Глубокая интеграция с LSP

### 0.1 Структура данных исходного отображения (SourceMap)

**Цели реализации**:
- Разработать и реализовать структуру `SourceMap` для записи отображения позиций исходного кода
- Записывать содержимое: позиции комментариев, области пробелов, позиции исходных токенов
- Поддерживать преобразование из смещения символов в строку/столбец

**Проектирование структуры данных**:
```
SourceMap {
    source: String,                    // Исходный код
    comments: Vec<Comment>,            // Список комментариев
    tokens: Vec<TokenWithSpan>,        // Токен + позиция
    line_offsets: Vec<usize>,          // Смещение начала каждой строки
    blank_lines: Vec<usize>,           // Список номеров пустых строк
}

Comment {
    content: String,
    span: Span,
    style: CommentStyle,  // SingleLine, MultiLine, Doc
}
```

- [x] SourceMap корректно анализирует исходный код
