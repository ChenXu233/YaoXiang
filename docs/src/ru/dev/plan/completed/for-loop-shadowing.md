# Документация по реализации цикла for и проверки затенения

## Обзор

В данном документе описывается план реализации дизайна изменяемости переменных цикла for и проверки затенения в языке YaoXiang.

**Предпосылки проблемы**:
- Текущий синтаксис `for i in 1..5` вызывает ошибку на этапе IR, поскольку переменная цикла не помечена как изменяемая
- Необходимо реализовать правило запрета затенения (shadowing)
- Изменяемость переменной цикла внутри цикла должна быть объявлена явно, в соответствии с правилами синтаксиса `let mut`, но поскольку в языке нет синтаксиса `let`, необходимо запретить затенение

## Принципы проектирования

1. **Запрет затенения**: Переменные, объявленные в новом пространстве имен (for, if, {} блоки и т.д.), не могут затенять переменные, уже существующие во внешней области видимости
2. **Явная изменяемость**: Изменяемость должна быть явно объявлена с помощью ключевого слова `mut`, переменные цикла по умолчанию неизменяемы
3. **Новая привязка при каждой итерации**: Переменные цикла for создают новую привязку при каждой итерации, а не модифицируют одну и ту же переменную

## Содержание реализации

### 1. Проверка запрета затенения

#### 1.1 Этап проверки типов

Во всех местах создания новых переменных выполняется проверка затенения:

- **Цикл for**: Проверяется, не объявлено ли имя переменной цикла уже в текущей или внешней области видимости
- **Декларация let**: Проверяется, не объявлено ли имя переменной уже в текущей области видимости
- **Блоки if/while и др.**: Проверяется, не затеняют ли внутренние объявления внешние переменные

#### 1.2 Место реализации

Изменяются следующие файлы:
- `src/frontend/typecheck/checking/mod.rs` — добавление логики проверки затенения

#### 1.3 Коды ошибок

Новый код ошибки `E2xxx` (предстоит определить):
```
[E2xxx] Ошибка затенения переменной
error: cannot shadow existing variable 'x'
 --> example.yx:3:5
  |
3 |     for i in 1..5 {
  |     ^ variable 'i' is already declared in outer scope
```

### 2. Изменяемость переменных цикла for

#### 2.1 Синтаксический дизайн

```yaoxiang
for i in 1..5 {      // i неизменяема (по умолчанию)
    print(i)         // OK
    i = i + 1        // Ошибка: cannot assign to immutable variable
}

for mut i in 1..5 {  // i изменяема
    i = i + 1        // OK
}
```

#### 2.2 Синтаксический анализ

Изменения в `src/frontend/core/parser/statements/control_flow.rs`:
- При разборе оператора for проверяется наличие `mut` после ключевого слова `for`
- Если `mut` присутствует, он записывается в AST узел

Изменения структуры AST:
```rust
StmtKind::For {
    var,           // Имя переменной
    var_mut: bool, // Новое: изменяемость переменной
    iterable,
    body,
    label,
}
```

#### 2.3 Генерация IR

Изменения в `generate_for_loop_ir` в `src/middle/core/ir_gen.rs`:
- Если `var_mut` равен true, переменная цикла добавляется в `current_mut_locals`
- Это позволяет MutChecker разрешить повторные присваивания данной переменной

```rust
// В generate_for_loop_ir
if var_mut {
    self.current_mut_locals.insert(var_reg);
}
```

## Примеры результатов реализации

### Пример 1: Базовый цикл for

```yaoxiang
// Ввод
for i in 1..5 {
    print(i)
}

// Вывод
1
2
3
4
```

### Пример 2: Модификация переменной цикла for

```yaoxiang
// Ввод
for mut i in 1..3 {
    i = i + 10
    print(i)
}

// Вывод
11
12
13
```

### Пример 3: Запрет затенения — цикл for

```yaoxiang
// Ввод
i = 10
for i in 1..5 {
    print(i)
}

// Ошибка вывода
error [E2xxx] cannot shadow existing variable 'i'
 --> example.yx:2:5
  |
2 |     for i in 1..5 {
  |         ^ variable 'i' is already declared in outer scope
help: consider renaming the inner variable or outer variable to avoid shadowing
```

### Пример 4: Запрет затенения в блоке {}

```yaoxiang
// Ввод
x = 1
{
    x = 2  // Ошибка!
    print(x)
}

// Ошибка вывода
error [E2xxx] cannot shadow existing variable 'x'
 --> example.yx:3:1
  |
3 |     x = 2
  |     ^ variable 'x' is already declared in outer scope
help: consider renaming the inner variable or outer variable to avoid shadowing
```

### Пример 5: Затенение в блоке if

```yaoxiang
// Ввод
x = 1
if true {
    x = 2  // Ошибка!
    print(x)
}

// Ошибка вывода
error [E2xxx] cannot shadow existing variable 'x'
 --> example.yx:4:5
  |
4 |         x = 2
  |         ^ variable 'x' is already declared in outer scope
help: consider renaming the inner variable or outer variable to avoid shadowing
```

### Пример 6: Модификация неизменяемой переменной в теле цикла

```yaoxiang
// Ввод
for i in 1..5 {
    i = i + 1
}

// Ошибка вывода
error [E2010] Cannot assign to immutable variable 'i'
 --> example.yx:2:5
  |
2 |     i = i + 1
  |     ^ cannot assign to immutable variable 'i'
help: Use 'mut' to declare a mutable variable
```

## 2. Новая привязка переменной цикла for при каждой итерации

```yaoxiang
// Ввод
for i in 1..3 {
    print(i)
}

// Здесь i при каждой итерации является новой привязкой, потому что после завершения каждой итерации i в теле цикла уничтожается, и при следующей итерации создаётся новая привязка i, а не модифицируется та же переменная.
```

## План приёмки

### Приёмка функциональности

1. **Базовая функциональность цикла for**
   - [ ] `for i in 1..5 { print(i) }` корректно выводит 1-4
   - [ ] `for mut i in 1..5 { i = i + 1; print(i) }` корректно выводит 2-5

2. **Запрет затенения**
   - [ ] При наличии внешней переменной использование одноимённой переменной в цикле for вызывает ошибку
   - [ ] При наличии внешней переменной объявление одноимённой переменной вызывает ошибку
   - [ ] Объявление переменной внутри блока if/while, затеняющее внешнюю переменную, вызывает ошибку
   - [ ] Проверка затенения через границы областей видимости функций

3. **Проверка изменяемости**
   - [ ] Переменная цикла for по умолчанию неизменяема, модификация вызывает ошибку
   - [ ] Переменная for mut может быть изменена, работает корректно

### Приёмка сообщений об ошибках

- [ ] Сообщение об ошибке затенения чётко указывает на затенённую переменную и её расположение
- [ ] Сообщение об ошибке изменяемости соответствует стилю существующего E2010
