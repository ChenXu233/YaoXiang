# План реализации идей Типовой Вселенной: Удаление узла Fn AST

> Версия документа: 1.3
> Дата создания: 2026-03-18
> Дата обновления: 2026-03-21
> Задача: Унификация функций/конструкторов типов/интерфейсов в узел Binding, определение функции отложено до этапа проверки типов

---

## Предпосылки и Цели

Согласно документу идей Типовой Вселенной, необходимо реализовать следующие ключевые изменения:

1. **Удаление узла Fn AST**: Все объявления унифицированы в узел `Binding`
2. **Определение функции отложено до этапа проверки типов**: Парсер отвечает только за синтаксис, семантика определяется типом возвращаемого значения
3. **Единая синтаксическая модель**: Функции, конструкторы типов и интерфейсы используют форму `identifier: type = expression`

## Этап первый: Фронтенд - AST унифицирован в узел Binding

### Шаг 1.1: Изменение определения AST узла

**Цель модификации**: Заменить `StmtKind::Fn` и `StmtKind::MethodBind` на `StmtKind::Binding`

**Проектные решения (подтверждены пользователем)**:
- **MethodBind также является Binding**: Привязки методов и обычные функции используют унифицированную синтаксическую модель
- **Expr::Lambda сохраняется отдельно**: Lambda-выражения являются независимым синтаксисом, но функциональный Binding зависит от Lambda
- **Упрощение Lambda**: Поскольку заголовок типа уже содержит информацию о параметрах, Lambda может быть упрощена до сохранения только body
- **Полная замена**: Обратная совместимость со старым синтаксисом не требуется

**Ключевые файлы** (первоначальный план: 1 → фактически: 2):
- `src/frontend/core/parser/ast.rs`
- `src/frontend/core/parser/tests/concurrency.rs` ⚠️

**Критерии приёмки**:
- [x] Вариант `StmtKind::Fn` удалён
- [x] Вариант `StmtKind::MethodBind` удалён, унифицирован в `StmtKind::Binding`
- [x] Добавлен вариант `StmtKind::Binding`, содержащий: name, type_name, method_type, generic_params, type_annotation, eval, params, body, is_pub
- [ ] Вариант `Expr::Lambda` сохранён, но структура упрощена (удалён заголовок Lambda, сохранён только body)
- [ ] Вариант `Type::Fn` сохранён (для типов функций в типовых выражениях)

**Статус**: ✅ Выполнено (2026-03-20)

**Запись реализации**:

- `Array: Type[T, N]` → `Array: (T: Type, N: Type) -> Type`
- `Matrix: Type[T, Rows, Cols]` → `Matrix: (T: Type, Rows: Type, Cols: Type) -> Type`
- `List: Type[T]` → `List: (T: Type) -> Type`
- BNF синтаксис: `generic_type ::= identifier ':' 'Type' generic_params? '=' type_expression`

language-spec.md (20 изменений):
- Синхронизировано обновление синтаксиса обобщённых типов и обобщённых функций в нескольких местах
