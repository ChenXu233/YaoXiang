# Задача: Реализация поддержки LSP (Language Server Protocol)

## Обзор

Для улучшения опыта разработки YaoXiang необходимо реализовать языковой сервер, соответствующий Language Server Protocol. Этот сервер будет встроен в интерфейс компилятора и обеспечит возможности анализа кода в реальном времени, автодополнения, перехода к определениям, диагностики и интеграции с основными редакторами (VS Code, Vim, Emacs и др.).

## Цели

- Реализовать бинарный файл языкового сервера YaoXiang `yaoxiang-lsp`.
- Поддержать следующие LSP-запросы:
  - `textDocument/didOpen` / `didChange` / `didSave` / `didClose`: синхронизация содержимого файлов.
  - `textDocument/documentSymbol`: перечисление символов документа (функции, типы, переменные и т.д.).
  - `textDocument/completion`: предоставление предложений по автодополнению кода (на основе символов в области видимости).
  - `textDocument/definition`: переход к месту определения.
  - `textDocument/hover`: отображение информации о типе и документационных комментариев при наведении.
  - `textDocument/publishDiagnostics`:推送 синтаксических ошибок, ошибок типов и других диагностических сообщений в реальном времени.
- Реализовать возможность инкрементной компиляции/проверки компилятором во избежание полного повторного разбора при каждом изменении.

## Конкретные шаги

1. **Проектирование архитектуры сервера** (1 неделя)
   - Построение на основе существующих библиотек интерфейса компилятора (лексический анализ, парсинг, проверка типов).
   - Использование асинхронного ввода-вывода (например, Tokio) для обработки LSP-запросов.
   - Проектирование структур данных для кэширования документов и инкрементного обновления.

2. **Реализация базового протокола LSP-коммуникации** (1 неделя)
   - Реализация отправки и получения JSON-RPC сообщений.
   - Обработка запросов инициализации, завершения и других запросов жизненного цикла.

3. **Интеграция интерфейса компилятора** (2 недели)
   - Инкапсуляция лексического анализа, парсинга и проверки типов в виде повторно используемых библиотечных функций.
   - Поддержка «сессии компиляции» для каждого документа с возможностью инкрементного обновления (только повторный разбор изменённых частей).
   - Реализация маппинга от AST-узлов к позициям в исходном коде для перехода и отображения при наведении.

4. **Реализация диагностических функций** (1 неделя)
   - Перехват ошибок и предупреждений в процессе проверки типов и их преобразование в диагностические сообщения LSP.
   - Автоматическая отправка диагностики после изменения файла.

5. **Реализация автодополнения и навигации по символам** (2 недели)
   - Построение таблицы символов области видимости для предоставления списка кандидатов автодополнения (включая ключевые слова, встроенные типы, переменные текущей области видимости).
   - Реализация перехода к определению: поиск файла и позиции определения символа по позиции курсора.
   - Реализация отображения информации при наведении: показ сигнатуры типа и связанных документационных комментариев (если поддерживаются).

6. **Тестирование и интеграция** (1 неделя)
   - Написание простого клиентского плагина VS Code для тестирования.
   - Обеспечение работоспособности всех функций в типичных сценариях без существенных проблем с производительностью.

## Критерии приёмки

- Сервер корректно запускается и отвечает на запросы инициализации.
- После открытия файла YaoXiang диагностическая информация (синтаксические ошибки, ошибки типов) отображается в редакторе в реальном времени.
- При наведении на переменную или имя функции отображается информация о её типе.
- При переходе к определению по идентификатору открывается соответствующий файл с переходом на строку определения.
- После ввода точки предлагается автодополнение полей record type (необходима информация о типе).
- Время отклика сервера на больших файлах (тысячи строк) находится в приемлемых пределах (<500 мс).

## Зависимости

- Интерфейсная библиотека компилятора должна быть стабильной и предоставлять API для получения AST, таблицы символов и информации о типах.
- Необходима реализация инкрементной компиляции (как минимум кэширование на уровне документа).
