# Задача: Разработка небольших приложений для проверки возможностей языка и накопления экосистемы

## Обзор

Фронтенд, средний уровень и интерпретатор YaoXiang уже реализованы, компиляторный бэкенд находится в разработке. Для тестирования практичности языка, обнаружения скрытых багов, накопления примеров кода и привлечения ранних пользователей необходимо разработать серию небольших приложений. Эти приложения должны работать плавно на интерпретаторе, не зависеть от высокопроизводительных оптимизаций компиляторного бэкенда, и при этом в полной мере демонстрировать основные конструкции YaoXiang: унифицированный синтаксис, модель владения, изменчивость на уровне полей и другие.

## Цели

Разработать не менее 3 приложений из разных областей, охватывающих сценарии использования: скриптовые инструменты, реализация алгоритмов, парсинг конфигурации и другие.

Каждое приложение должно содержать полный исходный код, тестовые случаи и краткую документацию.

В процессе разработки собирать обратную связь по вопросам дизайна языка, совершенствовать стандартную библиотеку и инструментарий.

## Рекомендуемые направления приложений (можно выбрать или предложить свои)

### Инструмент массового переименования файлов

Функционал: обход указанной директории, сопоставление имён файлов по регулярным выражениям, переименование (замена, добавление префикса/суффикса и т.д.).

Задействованные возможности: операции с файловой системой (нужен FFI или стандартная библиотека), обработка строк, управление владением (предотвращение утечки ресурсов).

Вывод: исполняемый скрипт с поддержкой аргументов командной строки.

### Демонстрация алгоритма быстрой сортировки

Функционал: реализация классического алгоритма быстрой сортировки для сортировки массива целых чисел.

Задействованные возможности: рекурсия, операции с массивами/списками, обобщения (если поддерживаются), передача владения (перемещение элементов массива).

Вывод: библиотека функций + тестовая программа, демонстрирующая корректность алгоритма.

### Прототип парсера конфигурации TOML

Функционал: парсинг простых TOML-файлов конфигурации (только базовые типы и таблицы), маппинг в типы записей YaoXiang.

Задействованные возможности: парсинг строк, типы записей, обработка ошибок.

Вывод: функции парсера + пример конфигурационного файла и код использования.

### Крестики-нолики в терминале

Функционал: игра для двух игроков в терминале, поддержка отображения доски, размещения фигур, определения победителя.

Задействованные возможности: управление состоянием (изменчивость полей), ввод/вывод, управление циклом.

Вывод: запускаемая игровая программа.

## Критерии приёмки

Приложения работают согласно ожиданиям, проходят все ручные тесты.

Код следует соглашениями по стилю YaoXiang (если имеются).

Отсутствуют ошибки безопасности памяти (гарантируется правилами владения).

Все обнаруженные в процессе разработки баги языка или отсутствующие возможности стандартной библиотеки зафиксированы как отдельные issue.

## Зависимости

Интерпретаторный бэкенд должен быть стабильно работоспособен.

Возможно потребуются модули стандартной библиотеки для файлового ввода/вывода, строковых операций и других.
