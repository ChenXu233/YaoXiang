# Шаблон релиза

> Команда `/release` генерирует changelog по этому шаблону.  
> changelog — это **описание изменений для людей**, а не список коммитов.

## Спецификация формата

```
:bookmark: V<номер_версии>: <заголовок>
```

| Поле                  | Значение                |

| Изменение версии      | `<старая_версия>` → `<новая_версия>` |
| Количество коммитов   | N коммитов              |

## 📋 Краткое описание обновления

Данный релиз сосредоточен на исправлениях стабильности системы типов
и модели владения. Инстанциация обобщённых типов (например,
`List(Int) = List(1, 2, 3)`) теперь работает корректно, исправлено
несколько граничных случаев доступа к полям `&T` и деструктурирующего
присваивания кортежей. Также завершён унифицированный рефакторинг
проверки владения, что закладывает основу для последующего улучшения
семантики move.

## ✨ Новые возможности

### Инстанциация обобщённых типов

Поддержка синтаксиса `List(Int) = List(1, 2, 3)`, конструктор типа
корректно регистрируется как Struct. При множественных вызовах
обобщённой функции вывод типов больше не конфликтует — каждый вызов
получает независимый экземпляр переменной типа.

- Конструктор типа регистрируется как Struct, исправлена корневая причина инстанциации
- Независимый вывод типов при множественных вызовах обобщённых функций
- Исправлена привязка методов интерфейса

### Перепривязка после Move

После того как переменная была перемещена (move), ей можно присвоить
новое значение. Присваивание с приоритетным поиском теперь корректно
обрабатывает состояние moved, избегая ложных ошибок "переменная уже
перемещена".

- В VarInfo добавлен флаг состояния moved
- В присваивании с приоритетным поиском добавлена ветка moved

### Resource marker trait

Добавлен маркерный trait `Resource`, помечающий типы, реализующие
побочные эффекты ввода-вывода. Это обеспечивает основу для последующего
отслеживания побочных эффектов и анализа безопасности параллелизма.

### PLDI SRC demo

Завершены все задачи PLDI SRC demo MVP, включая сквозную демонстрацию
проверки типов и генерации кода.

## 🐛 Исправления ошибок

### Доступ к полям &T

Исправлено несколько проблем проверки типов при доступе к полям
неизменяемой ссылки, включая присваивание полям и вывод типов при вызове
конструктора через ссылку.

- Исправлена проверка типов доступа к полям `&T`
- Вывод целевого типа при присваивании полю
- Вывод типов при вызове конструктора через ссылку

### Семантическая подсветка LSP

Исправлена отсутствующая подсветка переменных внутри блоков `spawn {}`
и деструктурирующего присваивания кортежей `(a, b) = ...`. В AST поле
`DestructureAssign.names` теперь содержит информацию о позиции каждого
имени переменной.

- Корректная подсветка переменных внутри блоков spawn
- Корректная подсветка имён переменных при деструктурирующем присваивании кортежей
- Заложена основа для последующей поддержки семантики move в LSP

### Удаление freeze

Удалена устаревшая функция `freeze`, проведена очистка связанных
тестовых случаев.

## ♻️ Рефакторинг и оптимизация

### Унифицированная система диагностики

Оставшиеся 5 перечислений ошибок перенесены в единую систему
диагностики `ErrorCodeDefinition`, охват кодами ошибок увеличен с 60%
до 80%. Все ошибки компиляции теперь следуют единому формату и
спецификации кодов ошибок.

### Унификация проверки владения

Разрозненная логика проверки владения объединена в единый
`OwnershipPass` с единственной точкой входа для проверки borrow, move
и времени жизни. Снижаются затраты на сопровождение, повышается
согласованность сообщений об ошибках.

### Упрощение модели параллелизма

Удалена система ограничений Send/Sync, `spawn {}` переработан в модель
группового выполнения. Удалён код, связанный с декораторами
`@block`/`@eager`/`@auto` (RFC-024 phase 1). Модель параллелизма
упрощена, что прокладывает путь для последующего проектирования
расширяемых примитивов параллелизма.
