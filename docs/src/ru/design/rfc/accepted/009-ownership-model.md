# RFC-009: Проектирование модели владения

> **Статус**: Принято
> **Автор**: Чэнь Сюй
> **Дата создания**: 2025-01-08
> **Последнее обновление**: 2026-05-11 (добавлено упрощённое заимствование &T/&mut T, усовершенствована градация владения)

## Аннотация

В данном документе определяется **модель владения (Ownership Model)** языка программирования YaoXiang.

**Основной проект — пять концепций, одна градация**:

```
Посмотреть/Изменить на месте    Забрать           Общее владение        Скопировать         Системный уровень
      │                           │                    │                    │                   │
     &T                          Move              ref                  clone()            unsafe
   &mut T                     Нулевое            Компилятор              Явное            *T
   Параметр                  копирование         автоматически          глубокое          Пользователь
    функции                    По умолчанию      выбирает              копирование         ответственен
   Запрещено                   RAII               Rc/Arc
    вырываться                                  Внутри задачи —
                                              бесшумная тишина
                                              Меж задачами — lint
                                              Weak в стандартной
                                               библиотеке
```

- **Move (по умолчанию)**: присваивание/передача параметров/возврат = передача владения, нулевое копирование, автоматическое освобождение через RAII
- **`&T` / `&mut T` (упрощённое заимствование)**: только как параметры функции, запрещено вырываться. Ноль аннотаций, ноль времени жизни. Компилятор автоматически заимствует на стороне вызова
- **`ref` — ключевое слово**: совместное использование между областями видимости. Компилятор автоматически выбирает Rc (не меж задач) или Arc (меж задач)
- **`clone()`**: явное глубокое копирование
- **`unsafe` + `*T`**: голый указатель, аварийный выход на системном уровне

**Устранённая сложность**:
- ❌ Нет времени жизни `'a`
- ❌ Нет проверки заимствований (запрет вместо аннотации, не требуется глобальный вывод времени жизни как в Rust)
- ❌ Нет GC
- ❌ Нет анализа потребления/возврата владения и других «мини-проверок заимствований»
- ❌ Пользователю не нужно знать разницу между Rc/Arc (компилятор выбирает автоматически)

> **Нагрузка на программиста**: три правила для `&T`/`&mut T`, одно ключевое слово `ref`, компилятор всё делает автоматически.
> **Гарантия производительности**: Move с нулевыми накладными расходами, заимствование с нулевыми накладными расходами, ref платит по факту использования, нет пауз GC.

## Мотивация

### Зачем нужна модель владения?

| Язык | Управление памятью | Проблема |
|------|--------------------|----------|
| C/C++ | Ручное | Утечки памяти, висячие указатели, двойное освобождение |
| Java/Python | GC | Задержки, накладные расходы памяти, непредсказуемые паузы |
| Rust | Владение + проверка заимствований | Крутая кривая обучения времени жизни `'a` |
| **YaoXiang** | **Move + Borrow + ref** | **Просто, определённо, без GC** |

### Проектные цели

```yaoxiang
# 1. Move по умолчанию (нулевое копирование)
p = Point(1.0, 2.0)
p2 = p                         # Move, p больше нельзя читать

# 2. &T / &mut T заимствование (нулевые накладные расходы, только чтение/модификация на месте, запрещено вырываться)
print_info(p2)                 # Компилятор автоматически заимствует &p2, возвращает после использования
shift(p2, 1.0, 1.0)           # Компилятор автоматически заимствует &mut p2

# 3. ref = совместное использование (компилятор автоматически выбирает Rc/Arc)
shared = ref p2                # Владение между областями видимости
spawn { use(shared) }          # Компилятор: меж задач → Arc

# 4. clone() = явное копирование
backup = p2.clone()            # Глубокое копирование, эксклюзивное владение

# 5. unsafe + *T = системный уровень
unsafe {
    ptr: *Point = &p
    (*ptr).x = 0.0
}
```

### Ключевые отличия от Rust

| Характеристика | Rust | YaoXiang |
|----------------|------|----------|
| Семантика по умолчанию | Заимствование `&T` (требуется явное `.clone()`) | **Move (передача значения, нулевое копирование)** |
| Заимствование | `&T`/`&mut T`, может возвращаться, требуется время жизни | **`&T`/`&mut T` только как параметры, запрещено вырываться** |
| Механизм совместного использования | `Arc::new()` + ручной Weak | **`ref` — ключевое слово (компилятор автоматически выбирает Rc/Arc)** |
| Копирование | `clone()` | `clone()` |
| Голый указатель | `*T` | `*T` |
| Время жизни | `'a` | ❌ Нет |
| Проверка заимствований | Глобальный вывод | **Только в области видимости функции** |
| Циклические ссылки | Ручной Weak | **Освобождение по завершении задачи / lint меж задач / Weak в стандартной библиотеке** |

## Предложение

### 1. Move (передача владения по умолчанию)

```yaoxiang
# Правило: присваивание / передача параметров / возврат = Move, нулевое копирование

p: Point = Point(1.0, 2.0)
p2 = p                           # Move, p больше нельзя читать

# Переменной можно переприсвоить (стиль Python, без затенения)
p = Point(3.0, 4.0)              # p перепривязывается, тип должен совпадать

# Параметры функции: Move
process: (p: Point) -> Point = {
    p.transform()
    p                            # Move при возврате
}

# Возврат из функции: Move
create: () -> Point = {
    p = Point(1.0, 2.0)
    p                            # Move при возврате, нулевое копирование
}
```

**Характеристики**:
- Нулевое копирование (компилятор перемещает указатель)
- После перемещения оригинальная привязка нечитаема (ошибка компиляции)
- RAII: автоматическое освобождение при выходе из области видимости
- Сигнатура функции `(T) -> T` сама по себе документация — потребляет T, возвращает T

### 2. &T / &mut T (упрощённое заимствование)

**Основной принцип: заимствование — это только «взглянуть» или «изменить на месте», владение не допускается.**

#### 2.1 Три правила

```
1. &T / &mut T могут появляться только как параметры функции
2. Нельзя возвращать, сохранять в структуре, присваивать локальной переменной, захватывать замыканием с последующим вырыванием
3. На стороне вызова не требуется аннотация &, компилятор автоматически выбирает заимствование или Move по сигнатуре метода
```

**Ноль аннотаций. Ноль времени жизни.** Компилятор делает только одну вещь: гарантирует, что заимствование не покидает текущую функцию. Это не требует межфункционального анализа — потому что запрет на вырывание перекрывает путь меж функциями, вывод не нужен.

#### 2.2 Базовое использование

```yaoxiang
# Сторона метода: объявление типа self, определяет способ заимствования
Point.print: (self: &Point) -> Void = {
    print(self.x)                  # Чтение полей
    print(self.y)
    # Функция заканчивается, заимствование заканчивается
}

Point.shift: (self: &mut Point, dx: Float, dy: Float) -> Void = {
    self.x = self.x + dx           # Модификация на месте
    self.y = self.y + dy
}

# Сторона вызова: компилятор автоматически выбирает заимствование или Move
p = Point(1.0, 2.0)
p.print()                          # Компилятор: &p, заимствование автоматически освобождается после print
p.shift(1.0, 1.0)                  # Компилятор: &mut p, заимствование автоматически освобождается после shift
p.print()                          # OK, p всё ещё валиден

# Свободные функции аналогично
distance: (a: &Point, b: &Point) -> Float = {
    sqrt((a.x - b.x)**2 + (a.y - b.y)**2)  # Чтение двух параметров
}
d = distance(p, p2)                     # Компилятор: &p, &p2
```

#### 2.3 Запрещённые действия

```yaoxiang
# ❌ Запрещено: заимствование покидает функцию
get_x_ref: (p: &Point) -> &Float = { p.x }      # Возврат заимствования → ошибка компиляции
store_ref: (p: &Point) -> Wrapper = {             # Сохранение в структуре → ошибка компиляции
    Wrapper { ref: p }
}

# ❌ Запрещено: заимствование присваивается локальной переменной
bad_bind: (p: &Point) -> Void = {
    q = p                         # &Point присваивается локальной переменной → ошибка компиляции
}

# ❌ Запрещено: пересечение границы задачи
bad_task: (p: &Point) -> Void = {
    spawn { print(p.x) }          # Заимствование вырывается в задачу → ошибка компиляции
}

# ❌ Запрещено: заимствованное нельзя Move
bad_move: (p: &Point) -> Void = {
    p2 = p                        # Не ваше, не имеете права передавать → ошибка компиляции
}

# ❌ Запрещено: заимствованное нельзя сохранять надолго
bad_ref: (p: &Point) -> Void = {
    shared = ref p                # Заимствование — не владение, нельзя ref → ошибка компиляции
}
```

#### 2.4 Защита от алиасинга для &mut

```yaoxiang
# ✅ Допустимо: несколько &T одновременно активны
read_both: (a: &Point, b: &Point) -> Float = { a.x + b.y }

# ✅ Допустимо: после завершения &mut можно снова заимствовать
shift_and_read: (p: &mut Point) -> Void = {
    shift(p, 1.0, 1.0)           # Заимствование &mut p в этом вызове
    print_info(p)                 # Предыдущее заимствование завершено, можно снова & 
}

# ❌ Запрещено: &mut и &T одновременно активны
# Компилятор выполняет потокочувствительный анализ в области видимости функции, гарантируя, что в каждый момент активен только один &mut
```

#### 2.5 Правила автоматического выбора заимствования

Компилятор на стороне вызова следует приоритету:

```
1. Если фактический аргумент ещё будет использоваться → предпочитается заимствование (&T или &mut T, согласно сигнатуре метода)
2. Если фактический аргумент больше не используется → Move
3. Приоритет匹配 порядка: &T < &mut T < Move
```

```yaoxiang
# Пример: автоматический выбор
p = Point(1.0, 2.0)
p.print()        # print объявляет &self → &p, после заимствования p можно использовать
p.shift(1.0, 1.0) # shift объявляет &mut self → &mut p
p2 = p           # Move, p больше не используется
```

### 3. Ключевое слово ref (автоматическая оптимизация компилятора)

`ref` — единственный способ совместного использования между областями видимости. Что под капотом — Rc или Arc, пользователю знать не нужно.

#### 3.1 Базовое использование

```yaoxiang
p: Point = Point(1.0, 2.0)
shared = ref p                   # Совместное владение, компилятор автоматически выбирает реализацию

# Совместное использование между задачами
@block
main: () -> Void = {
    data = ref heavy_data
    spawn { use(data) }           # Компилятор: меж задач → Arc
    spawn { use(data) }           # Компилятор: меж задач → Arc
}

# Совместное использование внутри задачи
@block
main: () -> Void = {
    data = ref heavy_data
    use(data)                     # Компилятор: не меж задач → Rc
}
```

**Ментальная модель пользователя**: `ref` = совместное владение. Этого достаточно.

#### 3.2 Анализ вырывания компилятора: Rc vs Arc

```
Анализ потока данных ref:

Не вырывается в другие задачи → Rc (неатомарный подсчёт ссылок, низкие накладные расходы)
Вырывается в другие задачи    → Arc (атомарный подсчёт ссылок, потокобезопасно)
```

#### 3.3 Стратегия обнаружения циклов

```
Циклы внутри задачи → бесшумное разрешение.
  ├── Структурированный параллелизм гарантирует, что при завершении задачи все ресурсы освобождаются унифицированно.
  ├── ref всегда сохраняет валидность, семантика не размывается.
  └── Пользователь имеет право создавать двунаправленные сильные ссылки внутри задачи (например, промежуточные состояния графовых вычислений).

Циклы между задачами → lint (по умолчанию warn, настраивается).
  ├── Программа работает корректно, настоящих утечек не будет (при завершении родительской задачи ресурсы дочерних задач освобождаются).
  ├── Но межзадачные сильные ссылки означают размытые границы владения — стоит остановиться и переосмыслить.
  ├── По умолчанию warn, компиляция проходит с предупреждением.
  └── Команда может установить deny в конфигурации проекта и включить в CI как требование качества.
```

**Уровни lint** (аналогично Rust clippy):

| Уровень | Поведение | Сценарий |
|---------|-----------|----------|
| `allow` | Не проверять | Личные проекты |
| `warn` (по умолчанию) | Компиляция проходит, предупреждение | Этап разработки |
| `deny` | Компиляция не проходит | CI требования команды |
| `forbid` | Компиляция не проходит, нельзя переопределить | Организационные обязательные правила |

```yaoxiang
# Циклы внутри задачи: бесшумное разрешение, двунаправленные сильные ссылки
build_graph: () -> Void = {
    a = Node("a")
    b = Node("b")
    a.next = ref b
    b.prev = ref a                # Цикл. При завершении задачи унифицированное освобождение.
}

# Циклы между задачами: lint (по умолчанию warn)
@block
parent_task: () -> Void = {
    shared_a = ref a
    shared_b = ref b
    spawn {
        shared_a.child = ref shared_b   # ⚠️ warn: циклическая ссылка между задачами
    }
}
```

**Пример конфигурации проекта**:

```toml
# yaoxiang.toml
[lints]
cross-task-cycle = "deny"    # Межзадачные циклы отклоняются в CI напрямую
```

| Тип цикла | Поведение | Причина |
|-----------|-----------|---------|
| Цикл ref внутри задачи | Не проверяется | Право пользователя, освобождение при завершении задачи |
| Цикл ref между задачами | lint (по умолчанию warn) | Напоминание переосмыслить, можно настроить deny |

#### 3.4 Weak: предоставляется стандартной библиотекой

```yaoxiang
use std.rc.Weak

# Продвинутый пользователь делает явный выбор
a.next = ref b
b.prev = Weak.new(a.next)        # Пользователь явно контролирует, какое направление слабое
```

**`Weak` не встроен в язык, это тип стандартной библиотеки.** Для повседневного использования достаточно `ref`. Продвинутые пользователи, которым нужна тонкая настройка памяти, явно импортируют `Weak`.

#### 3.5 Заимствование vs ref

| | `&T` / `&mut T` | `ref` |
|------|------|----------|
| Что делает | Взглянуть / изменить на месте | Совместное владение |
| Область действия | Параметры функции, использовать и вернуть | Между областями видимости |
| Стоимость | Нулевые накладные расходы | Rc или Arc (выбирает компилятор) |
| Вырывание | Запрещено | Для этого и создан |
| Циклы | Не затрагивает | Бесшумно разрешаются, меж задач — lint |

### 4. clone() —— Явное копирование

```yaoxiang
p: Point = Point(1.0, 2.0)
p2 = p.clone()                   # Глубокое копирование
# p и p2 независимы, не влияют друг на друга
```

**Когда использовать**: когда нужно сохранить оригинальное значение и сценарии Move или совместного использования не подходят.

### 5. unsafe + голый указатель (системное программирование)

```yaoxiang
p: Point = Point(1.0, 2.0)

unsafe {
    ptr: *Point = &p              # Голый указатель
    (*ptr).x = 0.0                # Разыменование (пользователь гарантирует безопасность)
    ptr2 = ptr + 1                # Арифметика указателей
}
```

**Ограничения**:
- Можно использовать только в блоке `unsafe`
- Пользователь гарантирует отсутствие висячих указателей, не использовать после освобождения
- Для FFI, операций с памятью и другого системного программирования

### 6. Общая градация владения

```
  Заимствование (нулевые накладные расходы)    Move (нулевые накладные расходы)    Совместное (по факту)    Копирование
   │                                              │                                 │                      │
  &T Посмотреть                               По умолчанию —                    ref Rc/Arc              clone()
  &mut T Изменить на месте                    передача владения                Компилятор              Явное
   │                                              │                             автоматически            глубокое
  Параметры функции                            Внутри области                   выбирает               копирование
  Запрещено вырываться                        видимости                                               │
  Автоматический выбор                         T -> T Возврат                ref Меж задач → Arc        В любой
                                            T -> Void Потребление           ref Не меж задач → Rc        момент
                                                  │                          Внутри задачи цикл     Независимая
                                               Возврат владения               бесшумно                 копия
                                               для цепочки                    Меж задач цикл lint
                                               вызовов                        Weak в стандартной
                                                                            библиотеке — выход
```

## Комплексный пример

```yaoxiang
Point: Type = {
    x: Float,
    y: Float,

    # &T: только чтение
    print: (self: &Point) -> Void = {
        print(self.x)
        print(self.y)
    }

    # &mut T: модификация на месте
    shift: (self: &mut Point, dx: Float, dy: Float) -> Void = {
        self.x = self.x + dx
        self.y = self.y + dy
    }

    # Move → Move: потребление с возвратом
    scale: (self: Point, f: Float) -> Point = {
        self.x = self.x * f
        self.y = self.y * f
        self                            # Забрать, изменить, вернуть
    }
}

# Комплексное использование
p = Point(1.0, 2.0)
p.print()                           # &p, заимствование
p.shift(1.0, 1.0)                   # &mut p, заимствование
p = p.scale(2.0)                    # Move → возврат
shared = ref p                      # ref совместное владение
spawn { use(shared) }

# clone независимая копия
backup = p.clone()

# Цикл внутри задачи: бесшумное разрешение
a = Node("a")
b = Node("b")
a.next = ref b
b.prev = ref a                      # Цикл, при завершении задачи унифицированное освобождение

# unsafe системный уровень
unsafe {
    ptr: *Point = &p
    (*ptr).x = 0.0
}
```

## Ограничения системы типов

### Send / Sync

| Тип | Send | Sync | Описание |
|------|------|------|----------|
| Типы значений | ✅ | ✅ | Int, Float, Point... |
| `ref T` | ✅ | ✅ | Компилятор автоматически выбирает Rc/Arc |
| `&T` / `&mut T` | ❌ | ❌ | Заимствование не может меж потоков (не разрешено вырываться) |
| `*T` | ❌ | ❌ | Голый указатель, однопоточный |

---

## Анализ производительности

| Операция | Стоимость | Описание |
|----------|-----------|----------|
| Move | Ноль | Перемещение указателя |
| `&T` / `&mut T` | Ноль | Проверка на этапе компиляции, нулевые накладные расходы runtime |
| `ref` (не меж задач)| Низкая | Компилируется в Rc, неатомарные операции |
| `ref` (меж задач)| Средняя | Компилируется в Arc, атомарные операции |
| `clone()` | Зависит от типа | Маленькие объекты быстро, большие медленно |
| `unsafe + *T` | Ноль | Прямые операции с памятью |

### Сравнение

| Язык | Механизм совместного использования | Управление памятью | Обработка циклов | Сложность |
|------|-------------------------------------|---------------------|------------------|-----------|
| Rust | Arc / Mutex + проверка заимствований | Проверка на этапе компиляции | Ручной Weak | Высокая |
| Go | chan / pointer | GC | GC | Низкая |
| C++ | shared_ptr | RAII | weak_ptr | Средняя |
| **YaoXiang** | **ref + упрощённое заимствование** | **RAII** | **Освобождение на границе задач / lint меж задач / Weak в стандартной библиотеке** | **Низкая** |

## Компромиссы

### Преимущества

1. **Простота**: нет времени жизни, нет глобального проверщика заимствований. Три правила для `&T`/`&mut T`
2. **Интеллектуальный компилятор**: ref автоматически выбирает Rc/Arc, на стороне вызова автоматически выбирается заимствование
3. **Определённость**: ref просто сохраняет валидность, не станет тихо слабой ссылкой
4. **Высокая производительность**: Move с нулевым копированием, заимствование с нулевыми накладными расходами
5. **Гибкость**: `unsafe + *T` поддерживает системное программирование

### Недостатки

1. **Накладные расходы runtime для ref**: атомарные операции имеют стоимость (но это неизбежная цена совместного использования)
2. **Риски unsafe**: пользователь должен гарантировать корректность
3. **Межзадачные циклы — lint, а не ошибка компиляции**: в отличие от Rust, где это ошибка компиляции, по умолчанию warn, требуется настройка deny командой, чтобы использовать как требование качества

## Альтернативные решения

| Решение | Почему не выбрано |
|---------|-------------------|
| GC | Накладные расходы runtime, непредсказуемые паузы |
| Проверщик заимствований Rust | Требуется время жизни `'a`, крутая кривая обучения |
| Чистый Move | Не способен обрабатывать параллельное совместное использование |
| Без голых указателей | Невозможно системное программирование |
| Раскрытие Rc/Arc пользователю | Детали реализации перекладываются на пользователя, увеличивается когнитивная нагрузка |

### Проверка перекрёстного заимствования: на точке вызова + потокочувствительный анализ

**Область анализа**: только тело функции. Потому что `&T`/`&mut T` могут быть только параметрами, на каждом звене цепочки вызовов при входе во вложенную функцию заимствование из внешней функции автоматически освобождается с завершением вызова. Межфункциональный анализ не нужен.

**Уровень 1: Проверка на точке вызова**——каждый фактический аргумент не может одновременно находиться в позиции `&mut` и в другой позиции заимствования:

```yaoxiang
alias_bad: (a: &mut Point, b: &Point) -> Void = { ... }
p = Point(1.0, 2.0)
alias_bad(p, p)    # ❌ p одновременно как &mut и как &, компилятор отклоняет
```

**Уровень 2: Потокочувствительный в теле функции**——после передачи `&mut` в вызов, после возврата вызова заимствование освобождается, можно заимствовать снова:

```yaoxiang
process_twice: (p: &mut Point) -> Void = {
    shift(p, 1.0, 1.0)    # &mut передаётся в shift, после возврата shift заимствование завершено
    print_info(p)          # Снова заимствуется &p, конфликта нет
}
```

**Что не требуется**: межфункциональное отслеживание времени жизни, глобальный анализ алиасинга, решение ограничений графа заимствования, NLL. Потому что заимствование не вырывается, не сохраняется в переменных, не возвращается——при выходе из точки вызова сразу становится недействительным.
