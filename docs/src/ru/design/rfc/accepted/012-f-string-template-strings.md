# RFC 012: F-String шаблонные строки

> **Статус**: Принято
> **Автор**: Chen Xu
> **Дата создания**: 2025-01-27
> **Последнее обновление**: 2026-02-12

## Аннотация

Добавление в язык YaoXiang поддержки f-string шаблонных строк с возможностью интерполяции переменных, вычисления выражений и форматирования вывода. F-string использует синтаксис в стиле Python (`f"..."` префикс), где выражения встраиваются в строку с помощью синтаксиса `{expression}`, а на этапе компиляции преобразуются в эффективные строковые операции.

> **Примечание**: Синтаксис и поведение f-string совместимы с Python, подробную спецификацию см. в [официальной документации Python](https://docs.python.org/3/tutorial/inputoutput.html#formatted-string-literals).

## Мотивация

### Зачем нужна эта функция?

Текущие способы конкатенации строк в YaoXiang слишком громоздки:

```yaoxiang
# Текущее состояние: использование + для конкатенации
name = "Alice"
age = 30
message = "Hello ".concat(name).concat(", age: ").concat(age.to_string())
print(message)

# Или использование функции format
message2 = format("Hello {}, age: {}", name, age)
```

### Текущие проблемы

1. **Плохая читаемость**: Конкатенация и форматирование строк требуют множества вызовов, код избыточен
2. **Подверженность ошибкам**: Ручная обработка преобразования типов, легко забыть `.to_string()`
3. **Соображения производительности**: Многократная конкатенация строк может влиять на производительность
4. **Недостаточная выразительность**: Невозможно наглядно встраивать сложные выражения в строки

## Предложение

### Основной дизайн

Введение f-string как нового префикса строковых литералов с поддержкой:
- **Интерполяция переменных**: `f"Hello {name}"`
- **Вычисление выражений**: `f"Sum: {x + y}"`
- **Спецификаторы форматирования**: `f"Pi: {pi:.2f}"`
- **Типовая безопасность**: Проверка типов выражений на этапе компиляции

### Примеры

```yaoxiang
# Базовая интерполяция
name = "Alice"
greeting = f"Hello {name}"  # "Hello Alice"

# Интерполяция выражений
x = 10
y = 20
result = f"Sum: {x + y}"    # "Sum: 30"

# Спецификаторы форматирования
pi = 3.14159
formatted = f"Pi: {pi:.2f}"  # "Pi: 3.14"

# Сложные выражения
items = [1, 2, 3]
s = f"Count: {len(items)}, sum: {sum(items)}"  # "Count: 3, sum: 6"

# Вызовы методов объектов
user = User("Bob", 25)
bio = f"Name: {user.name}, age: {user.get_age()}"
```

### Изменения синтаксиса

| Раньше | Теперь |
|--------|--------|
| `"Hello ".concat(name)` | `f"Hello {name}"` |
| `format("Value: {}", value)` | `f"Value: {value}"` |
| `format("Pi: {:.2f}", pi)` | `f"Pi: {pi:.2f}"` |

### Спецификация синтаксиса

```
FStringLiteral ::= 'f' '"' FStringContent* '"'
FStringContent ::= FStringChar | EscapeSequence | FStringInterpolation
FStringInterpolation ::= '{' Expression (':' FormatSpec)? '}'
FormatSpec      ::= [width] ['.' precision] type
width           ::= digit+
precision       ::= digit+
type            ::= 'b' | 'c' | 'd' | 'e' | 'E' | 'f' | 'F' | 'g' | 'G' | 'n' | 'o' | 's' | 'x' | 'X' | '%'
```

## Детальный дизайн

### Синтаксический анализ

Компилятор распознаёт строковые литералы с префиксом `f` на этапе лексического анализа,解析花括号内的表达式和可选的格式化说明符。

### Стратегия преобразования

F-string на этапе компиляции преобразуется в эффективные строковые операции:

**Простая интерполяция**:
```yaoxiang
f"Hello {name}"
```
Преобразуется в:
```yaoxiang
"Hello ".concat(name.to_string())
```

**Интерполяция выражений**:
```yaoxiang
f"Sum: {x + y}"
```
Преобразуется в:
```yaoxiang
"Sum: ".concat((x + y).to_string())
```

**Спецификаторы форматирования**:
```yaoxiang
f"Pi: {pi:.2f}"
```
Преобразуется в:
```yaoxiang
format("Pi: {:.2f}", pi)
```

**Множественная интерполяция**:
```yaoxiang
f"Hello {name}, you are {age} years old"
```
Преобразуется в:
```yaoxiang
"Hello ".concat(name.to_string()).concat(", you are ").concat(age.to_string()).concat(" years old")
```

### Влияние на систему типов

- Выражения интерполяции должны реализовывать интерфейс `Stringable` (автоматически реализуется для базовых типов и строк)
- Спецификаторы форматирования требуют поддержки соответствующего форматирования типом
- Компилятор проверяет соответствие типов выражений и правил форматирования

### Изменения в компиляторе

| Компонент | Изменения |
|-----------|-----------|
| lexer | Распознавание префикса f, синтаксический анализ интерполяции внутри строк |
| parser | Новый синтаксический узел FStringLiteral |
| typecheck | Проверка типов выражений интерполяции, валидация правил форматирования |
| codegen | Генерация кода конкатенации строк или вызовов форматирования |

### Обратная совместимость

- ✅ Полная обратная совместимость
- Существующие строковые литералы `"..."` остаются без изменений
- F-string — новый синтаксис, не влияющий на существующий код

## Компромиссы

### Преимущества

1. **Компактный синтаксис**: Сокращение шаблонного кода, повышение читаемости
2. **Типовая безопасность**: Проверка на этапе компиляции, уменьшение ошибок времени выполнения
3. **Оптимизация производительности**: Компилятор может оптимизировать конкатенацию строк
4. **Высокая выразительность**: Поддержка произвольных выражений и форматирования
5. **Низкий порог вхождения**: Совместимость с экосистемой Python

### Недостатки

1. **Сложность компилятора**: Необходимость добавления новой логики синтаксического анализа и преобразования
2. **Синтаксическая неоднозначность**: Требуется отличать от существующего синтаксиса строк
3. **Сложности отладки**: Преобразованный код отличается от исходного

## Альтернативные решения

| Решение | Почему не выбрано |
|---------|-------------------|
| Только интерполяция переменных | Не удовлетворяет потребностям сложного форматирования |
| Использование функционального стиля `format(...)` | Синтаксис недостаточно компактен |
| Отложить до v2.0 | У пользователей есть явная потребность в удобстве работы со строками |
| Использовать обратные кавычки или другой префикс | Несовместимость с экосистемой Python |

   - Интерполяция переменных и простых выражений
   - Базовое преобразование типов

2. **Этап 2 (v1.0)**:
   - Поддержка спецификаторов форматирования

3. **Этап 3 (v1.1)**:
   - Улучшение отладочной информации
   - Улучшение сообщений об ошибках
   - Дополнительные опции форматирования

### Зависимости

- Без внешних зависимостей
- Требуется базовая поддержка системы типов
- Требуются базовые функции библиотеки строк

### Риски

1. **Риск производительности**: Множественная интерполяция может создавать избыточные объекты строк
   - **Смягчение**: Оптимизация компилятором для объединения смежных строковых констант
2. **Сложность проверки типов**: Проверка типов для спецификаторов форматирования
   - **Смягчение**: Ориентир на реализацию Python, использование простых и прямолинейных проверок
3. **Синтаксическая неоднозначность**: Вложенное использование `{` и `}`
   - **Смягчение**: Чёткие правила синтаксиса, ограничение вложенности

## Приложения

### Приложение A: Справочник спецификаторов форматирования

| Тип | Спецификатор | Пример | Вывод |
|-----|--------------|--------|-------|
| Целое | `d` | `f"{42:d}"` | "42" |
| С плавающей точкой | `f` | `f"{3.14:.2f}"` | "3.14" |
| Научная нотация | `e` | `f"{1000:e}"` | "1.000000e+03" |
| Строка | `s` | `f"{name:s}"` | "Alice" |
| Шестнадцатеричное | `x` | `f"{255:x}"` | "ff" |

### Приложение B: Примеры сценариев использования

```yaoxiang
# Логирование
log(level: String, msg: String, count: Int) = () => {
    timestamp = get_timestamp()
    print(f"[{timestamp}] {level}: {msg} (count: {count})")
}

# Построение JSON
json = "{\n    \"name\": \"".concat(user.name).concat("\",\n    \"age\": ")
    .concat(user.age.to_string()).concat(",\n    \"email\": \"")
    .concat(user.email).concat("\"\n}")

# Построение SQL-запроса (осторожно с SQL-инъекциями)
query = f"SELECT * FROM users WHERE age > {min_age} AND status = '{status}'"

# Отладочная информация
debug_info = f"Point({x:.2f}, {y:.2f}) at {timestamp}"

# Условное форматирование
status_msg = if is_active {
    f"User {name} is active"
} else {
    f"User {name} is inactive"
}
```
