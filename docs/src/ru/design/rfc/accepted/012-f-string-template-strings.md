# RFC 012: F-String 模板字符串

## Краткое описание

Добавление в язык YaoXiang возможности шаблонных строк f-string, поддерживающей интерполяцию переменных, вычисление выражений и форматированный вывод. f-string использует синтаксис в стиле Python (префикс `f"..."`), позволяя встраивать выражения в строку с помощью синтаксиса `{expression}` и преобразуя их на этапе компиляции в эффективные строковые операции.

> **Внимание**: Синтаксис и поведение f-string соответствуют Python, конкретные спецификации см. в [официальной документации Python](https://docs.python.org/3/tutorial/inputoutput.html#formatted-string-literals).

## Мотивация

### Зачем нужна эта возможность?

Текущий способ конкатенации строк в YaoXiang довольно громоздок:

```yaoxiang
# Текущее положение: использование + для конкатенации
name = "Alice"
age = 30
message = "Hello ".concat(name).concat(", age: ").concat(age.to_string())
print(message)

# Или использование функции format
message2 = format("Hello {}, age: {}", name, age)
```

### Текущие проблемы

1. **Низкая читаемость**: конкатенация и форматирование строк требуют множества вызовов, код получается многословным
2. **Склонность к ошибкам**: ручная обработка преобразования типов, легко забыть `.to_string()`
3. **Соображения производительности**: многократная конкатенация строк может влиять на производительность
4. **Недостаточная выразительность**: нет возможности наглядно встраивать сложные выражения в строку

## Предложение

### Основной дизайн

Введение f-string в качестве нового префикса строкового литерала с поддержкой:
- **Интерполяция переменных**: `f"Hello {name}"`
- **Вычисление выражений**: `f"Sum: {x + y}"`
- **Спецификаторы форматирования**: `f"Pi: {pi:.2f}"`
- **Безопасность типов**: проверка типа выражения на этапе компиляции

### Примеры

```yaoxiang
# Базовая интерполяция
name = "Alice"
greeting = f"Hello {name}"  # "Hello Alice"

# Интерполяция выражений
x = 10
y = 20
result = f"Sum: {x + y}"    # "Sum: 30"

# Спецификаторы форматирования
pi = 3.14159
formatted = f"Pi: {pi:.2f}"  # "Pi: 3.14"

# Сложные выражения
items = [1, 2, 3]
s = f"Count: {len(items)}, sum: {sum(items)}"  # "Count: 3, sum: 6"

# Вызовы методов объекта
user = User("Bob", 25)
bio = f"Name: {user.name}, age: {user.get_age()}"
```

### Изменения в синтаксисе

| До | После |
|------|------|
| `"Hello ".concat(name)` | `f"Hello {name}"` |
| `format("Value: {}", value)` | `f"Value: {value}"` |
| `format("Pi: {:.2f}", pi)` | `f"Pi: {pi:.2f}"` |

### Спецификация синтаксиса

```
FStringLiteral ::= 'f' '"' FStringContent* '"'
FStringContent ::= FStringChar | EscapeSequence | FStringInterpolation
FStringInterpolation ::= '{' Expression (':' FormatSpec)? '}'
FormatSpec      ::= [width] ['.' precision] type
width           ::= digit+
precision       ::= digit+
type            ::= 'b' | 'c' | 'd' | 'e' | 'E' | 'f' | 'F' | 'g' | 'G' | 'n' | 'o' | 's' | 'x' | 'X' | '%'
```

## Подробный дизайн

### Синтаксический анализ

Компилятор на этапе лексического анализа распознаёт строковые литералы с префиксом `f`, разбирая выражения внутри фигурных скобок и необязательные спецификаторы форматирования.

### Стратегия трансформации

f-string на этапе компиляции преобразуются в эффективные строковые операции:

**Простая интерполяция**:
```yaoxiang
f"Hello {name}"
```
Преобразуется в:
```yaoxiang
"Hello ".concat(name.to_string())
```

**Интерполяция выражений**:
```yaoxiang
f"Sum: {x + y}"
```
Преобразуется в:
```yaoxiang
"Sum: ".concat((x + y).to_string())
```

**Спецификаторы форматирования**:
```yaoxiang
f"Pi: {pi:.2f}"
```
Преобразуется в:
```yaoxiang
format("Pi: {:.2f}", pi)
```

**Несколько интерполяций**:
```yaoxiang
f"Hello {name}, you are {age} years old"
```
Преобразуется в:
```yaoxiang
"Hello ".concat(name.to_string()).concat(", you are ").concat(age.to_string()).concat(" years old")
```

### Влияние на систему типов

- Интерполируемое выражение должно реализовывать интерфейс `Stringable` (автоматически реализуется для базовых типов и строк)
- Спецификаторы форматирования требуют, чтобы тип поддерживал соответствующее форматирование
- Компилятор проверяет соответствие типа выражения правилам форматирования

### Изменения в компиляторе

| Компонент | Изменения |
|------|------|
| lexer | Распознавание префикса f, разбор синтаксиса интерполяции внутри строки |
| parser | Добавление узла синтаксиса FStringLiteral |
| typecheck | Проверка типа интерполируемого выражения, валидация правил форматирования |
| codegen | Генерация кода конкатенации строк или вызовов форматирования |

### Обратная совместимость

- ✅ Полностью обратно совместимо
- Существующие строковые литералы `"..."` остаются без изменений
- f-string — это новый синтаксис, не затрагивающий существующий код

## Компромиссы

### Преимущества

1. **Лаконичный синтаксис**: сокращение шаблонного кода, повышение читаемости
2. **Безопасность типов**: проверка на этапе компиляции, уменьшение ошибок времени выполнения
3. **Оптимизация производительности**: компилятор может оптимизировать конкатенацию строк
4. **Высокая выразительность**: поддержка произвольных выражений и форматирования
5. **Низкий порог входа**: соответствие экосистеме Python

### Недостатки

1. **Сложность компилятора**: необходимость нового синтаксического анализа и логики трансформации
2. **Синтаксическая неоднозначность**: необходимость отличать от существующего синтаксиса строк
3. **Сложность отладки**: скомпилированный код структурно отличается от исходного

## Альтернативы

| Решение | Почему не выбрано |
|------|--------------|
| Поддержка только интерполяции переменных | Не покрывает потребности в сложном форматировании |
| Использование функционального стиля `format(...)` | Синтаксис недостаточно лаконичен |
| Отложить до v2.0 | У пользователей есть явная потребность в удобстве работы со строками |
| Использование обратных кавычек или другого префикса | Не соответствует экосистеме Python |

## Стратегия реализации

### Этапы реализации

1. **Этап 1 (v0.9)**:
   - Поддержка базового синтаксиса f-string
   - Интерполяция переменных и простых выражений
   - Преобразование базовых типов

2. **Этап 2 (v1.0)**:
   - Поддержка спецификаторов форматирования
   - Интерполяция сложных выражений
   - Оптимизация производительности

3. **Этап 3 (v1.1)**:
   - Улучшение отладочной информации
   - Улучшение сообщений об ошибках
   - Дополнительные опции форматирования

### Зависимости

- Нет внешних зависимостей
- Требуется базовая поддержка системы типов
- Требуется базовая функциональность библиотеки строк

### Риски

1. **Риск производительности**: множественные интерполяции могут приводить к избыточным строковым объектам
   - **Смягчение**: оптимизация компилятора для объединения соседних строковых констант
2. **Сложность проверки типов**: проверка типов для спецификаторов форматирования
   - **Смягчение**: использование простой и прямой проверки, ориентируясь на реализацию Python
3. **Синтаксическая неоднозначность**: вложенное использование `{` и `}`
   - **Смягчение**: чёткие правила синтаксиса, ограничение вложенности

## Открытые вопросы

- [x] Поддерживаются ли экранированные фигурные скобки? Совместимо с Python: двойная фигурная скобка обозначает одинарную, например <code v-pre>{{</code> означает <code v-pre>{</code>, <code v-pre>}}</code> означает <code v-pre>}</code>
- [x] Поддерживаются ли пользовательские функции форматирования? Совместимо с Python: поддерживается настройка поведения форматирования типов через метод `__format__`
- [x] Полная спецификация спецификаторов форматирования? Совместимо с Python, подробности см. в BNF выше
- [x] Конкретная стратегия оптимизации производительности? Совместимо с Python: конкатенация во время выполнения, специальная оптимизация не требуется
- [x] Лучшие практики диагностики ошибок? Совместимо с Python: при сообщении об ошибке отображается исходное содержимое f-string и его позиция

## Приложения

### Приложение А: Справочник по спецификаторам форматирования

| Тип | Спецификатор | Пример | Вывод |
|------|--------|------|------|
| Целое число | `d` | `f"{42:d}"` | "42" |
| Число с плавающей точкой | `f` | `f"{3.14:.2f}"` | "3.14" |
| Экспоненциальная запись | `e` | `f"{1000:e}"` | "1.000000e+03" |
| Строка | `s` | `f"{name:s}"` | "Alice" |
| Шестнадцатеричное | `x` | `f"{255:x}"` | "ff" |

### Приложение Б: Примеры сценариев использования

```yaoxiang
# Логирование
log(level: String, msg: String, count: Int) = () => {
    timestamp = get_timestamp()
    print(f"[{timestamp}] {level}: {msg} (count: {count})")
}

# Построение JSON
json = "{\n    \"name\": \"".concat(user.name).concat("\",\n    \"age\": ")
    .concat(user.age.to_string()).concat(",\n    \"email\": \"")
    .concat(user.email).concat("\"\n}")

# Построение SQL-запроса (обратите внимание на риск SQL-инъекции)
query = f"SELECT * FROM users WHERE age > {min_age} AND status = '{status}'"

# Отладочная информация
debug_info = f"Point({x:.2f}, {y:.2f}) at {timestamp}"

# Условное форматирование
status_msg = if is_active {
    f"User {name} is active"
} else {
    f"User {name} is inactive"
}
```
