# RFC 012: F-String шаблонные строки

> **Статус**: Принято
> **Автор**: Chen Xu
> **Дата создания**: 2025-01-27
> **Последнее обновление**: 2026-02-12

## Аннотация

Добавление в язык YaoXiang特性 f-string шаблонных строк, поддерживающих интерполяцию переменных, вычисление выражений и форматированный вывод. f-string использует синтаксис в стиле Python（`f"..."` префикс），в котором выражения встраиваются в строку через синтаксис `{выражение}`, а при компиляции преобразуются в эффективные строковые операции.

> **Примечание**: Синтаксис и поведение f-string соответствуют Python, подробные спецификации см. в [официальной документации Python](https://docs.python.org/3/tutorial/inputoutput.html#formatted-string-literals).

## Мотивация

### Зачем нужна эта特性?

Текущие способы конкатенации строк в YaoXiang достаточно громоздки：

```yaoxiang
# Текущее состояние: использование + для конкатенации
name = "Alice"
age = 30
message = "Hello ".concat(name).concat(", age: ").concat(age.to_string())
print(message)

# Или использование функции format
message2 = format("Hello {}, age: {}", name, age)
```

### Текущие проблемы

1. **Плохая читаемость**：Конкатенация и форматирование строк требуют множества вызовов, код избыточен
2. **Подвержено ошибкам**：Ручная обработка преобразования типов, легко пропустить `.to_string()`
3. **Проблемы производительности**：Многократная конкатенация строк может влиять на производительность
4. **Недостаточная выразительность**：Невозможно интуитивно встраивать сложные выражения в строки

## Предложение

### Основной дизайн

Введение f-string как нового типа строкового literal со следующими возможностями:
- **Интерполяция переменных**：`f"Hello {name}"`
- **Вычисление выражений**：`f"Sum: {x + y}"`
- **Спецификаторы формата**：`f"Pi: {pi:.2f}"`
- **Типовая безопасность**：Проверка типов выражений при компиляции

### Примеры

```yaoxiang
# Базовая интерполяция
name = "Alice"
greeting = f"Hello {name}"  # "Hello Alice"

# Интерполяция выражений
x = 10
y = 20
result = f"Sum: {x + y}"    # "Sum: 30"

# Спецификаторы формата
pi = 3.14159
formatted = f"Pi: {pi:.2f}"  # "Pi: 3.14"

# Сложные выражения
items = [1, 2, 3]
s = f"Count: {len(items)}, sum: {sum(items)}"  # "Count: 3, sum: 6"

# Вызовы методов объекта
user = User("Bob", 25)
bio = f"Name: {user.name}, age: {user.get_age()}"
```

### Синтаксические изменения

| Раньше | Теперь |
|--------|--------|
| `"Hello ".concat(name)` | `f"Hello {name}"` |
| `format("Value: {}", value)` | `f"Value: {value}"` |
| `format("Pi: {:.2f}", pi)` | `f"Pi: {pi:.2f}"` |

### Спецификация синтаксиса

```
FStringLiteral ::= 'f' '"' FStringContent* '"'
FStringContent ::= FStringChar | EscapeSequence | FStringInterpolation
FStringInterpolation ::= '{' Expression (':' FormatSpec)? '}'
FormatSpec      ::= [width] ['.' precision] type
width           ::= digit+
precision       ::= digit+
type            ::= 'b' | 'c' | 'd' | 'e' | 'E' | 'f' | 'F' | 'g' | 'G' | 'n' | 'o' | 's' | 'x' | 'X' | '%'
```

## Детальный дизайн

### Синтаксический анализ

Компилятор распознаёт строковые literal с префиксом `f` на этапе лексического анализа,解析рует выражения внутри фигурных скобок и необязательные спецификаторы формата.

### Стратегия преобразования

f-string компилируется в эффективные строковые операции：

**Простая интерполяция**：
```yaoxiang
f"Hello {name}"
```
Преобразуется в：
```yaoxiang
"Hello ".concat(name.to_string())
```

**Интерполяция выражений**：
```yaoxiang
f"Sum: {x + y}"
```
Преобразуется в：
```yaoxiang
"Sum: ".concat((x + y).to_string())
```

**Спецификаторы формата**：
```yaoxiang
f"Pi: {pi:.2f}"
```
Преобразуется в：
```yaoxiang
format("Pi: {:.2f}", pi)
```

**Множественная интерполяция**：
```yaoxiang
f"Hello {name}, you are {age} years old"
```
Преобразуется в：
```yaoxiang
"Hello ".concat(name.to_string()).concat(", you are ").concat(age.to_string()).concat(" years old")
```

### Влияние на системы типов

- Выражения интерполяции должны реализовывать интерфейс `Stringable`（автоматически реализуется для базовых типов и строк）
- Спецификаторы формата требуют поддержки соответствующего форматирования типом
- Компилятор проверяет соответствие типов выражений и правил форматирования

### Изменения в компиляторе

| Компонент | Изменения |
|-----------|-----------|
| lexer | Распознавание префикса f, разбор синтаксиса интерполяции в строках |
| parser | Новый синтаксический узел FStringLiteral |
| typecheck | Проверка типов выражений интерполяции, валидация правил форматирования |
| codegen | Генерация кода конкатенации строк или вызовов форматирования |

### Обратная совместимость

- ✅ Полная обратная совместимость
- Существующие строковые literal `"..."` остаются без изменений
- f-string — новый синтаксис, не влияющий на существующий код

## Компромиссы

### Преимущества

1. **Компактный синтаксис**：Сокращение шаблонного кода, улучшение читаемости
2. **Типовая безопасность**：Проверка при компиляции, уменьшение ошибок runtime
3. **Оптимизация производительности**：Компилятор может оптимизировать конкатенацию строк
4. **Высокая выразительность**：Поддержка произвольных выражений и форматирования
5. **Низкий порог вхождения**：Согласованность с экосистемой Python

### Недостатки

1. **Сложность компилятора**：Необходимость добавления логики синтаксического анализа и преобразования
2. **Синтаксическая неоднозначность**：Необходимость отличения от существующего синтаксиса строк
3. **Сложности отладки**：Преобразованный код отличается от исходного

## Альтернативные решения

| Решение | Почему не выбрано |
|---------|-------------------|
| Только интерполяция переменных | Не удовлетворяет потребностям сложного форматирования |
| Использование функционального стиля `format(...)` | Синтаксис недостаточно компактный |
| Отложить до v2.0 | Пользователи явно нуждаются в удобстве работы со строками |
| Использование обратных кавычек или другого префикса | Не согласовано с экосистемой Python |

## Стратегия реализации

### Фазы реализации

1. **Фаза 1 (v0.9)**:
   - Базовая поддержка синтаксиса f-string
   - Интерполяция переменных и простых выражений
   - Преобразование базовых типов

2. **Фаза 2 (v1.0)**:
   - Поддержка спецификаторов формата
   - Интерполяция сложных выражений
   - Оптимизация производительности

3. **Фаза 3 (v1.1)**:
   - Улучшение отладочной информации
   - Улучшение сообщений об ошибках
   - Дополнительные опции форматирования

### Зависимости

- Без внешних зависимостей
- Требуется поддержка базовой системы типов
- Требуется базовая функциональность строковой библиотеки

### Риски

1. **Риск производительности**：Множественная интерполяция может создавать избыточные объекты строк
   - **Смягчение**：Оптимизация компилятором合并相邻них строковых констант
2. **Сложность проверки типов**：Проверка типов спецификаторов формата
   - **Смягчение**：Ориентир на реализацию Python, использование простых и прямых проверок
3. **Синтаксическая неоднозначность**：Вложенное использование `{` и `}`
   - **Смягчение**：Чёткие правила синтаксиса, ограничение вложенности

## Открытые вопросы

- [x] Поддержка экранированных фигурных скобок? Согласованность с Python: использование двойных фигурных скобок для одиночных, например <code v-pre>{{</code> 表示 <code v-pre>{</code>, <code v-pre>}}</code> 表示 <code v-pre>}</code>
- [x] Поддержка пользовательских функций форматирования? Согласованность с Python: поддержка自定义格式化 поведения типов через метод `__format__`
- [x] Полная спецификация спецификаторов формата? Согласованность с Python, подробности в BNF выше
- [x] Конкретные стратегии оптимизации производительности? Согласованность с Python: конкатенация во время выполнения, без специальной оптимизации
- [x] Лучшие практики диагностики ошибок? Согласованность с Python: при ошибках показывать исходное содержимое f-string и позицию

## Приложения

### Приложение A：Справочник по спецификаторам формата

| Тип | Спецификатор | Пример | Вывод |
|------|--------------|--------|-------|
| Целые числа | `d` | `f"{42:d}"` | "42" |
| Числа с плавающей точкой | `f` | `f"{3.14:.2f}"` | "3.14" |
| Научная нотация | `e` | `f"{1000:e}"` | "1.000000e+03" |
| 字符串型 | `s` | `f"{name:s}"` | "Alice" |
| Шестнадцатеричные | `x` | `f"{255:x}"` | "ff" |

### Приложение B：Примеры сценариев использования

```yaoxiang
# Логирование
log(level: String, msg: String, count: Int) = () => {
    timestamp = get_timestamp()
    print(f"[{timestamp}] {level}: {msg} (count: {count})")
}

# Построение JSON
json = "{\n    \"name\": \"".concat(user.name).concat("\",\n    \"age\": ")
    .concat(user.age.to_string()).concat(",\n    \"email\": \"")
    .concat(user.email).concat("\"\n}")

# Построение SQL-запросов（注意 риски SQL-инъекций）
query = f"SELECT * FROM users WHERE age > {min_age} AND status = '{status}'"

# Отладочная информация
debug_info = f"Point({x:.2f}, {y:.2f}) at {timestamp}"

# Условное форматирование
status_msg = if is_active {
    f"User {name} is active"
} else {
    f"User {name} is inactive"
}
```
