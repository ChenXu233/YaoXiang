# RFC 012: Шаблонные строки F-String

> **Статус**: Принято
> **Автор**: Chen Xu
> **Дата создания**: 2025-01-27
> **Последнее обновление**: 2026-02-12

## Аннотация

Добавление в язык YaoXiang поддержки шаблонных строк f-string с интерполяцией переменных, вычислением выражений и форматированным выводом. F-string использует синтаксис в стиле Python (`f"..."` префикс), где выражения встраиваются в строку через синтаксис `{expression}`, компилируясь во время компиляции в эффективные строковые операции.

> **Примечание**: Синтаксис и поведение f-string соответствуют Python, подробные спецификации см. в [официальной документации Python](https://docs.python.org/3/tutorial/inputoutput.html#formatted-string-literals).

## Мотивация

### Зачем нужен этот функционал?

Текущие методы конкатенации строк в YaoXiang слишком громоздки:

```yaoxiang
# Текущее состояние: конкатенация через +
name = "Alice"
age = 30
message = "Hello ".concat(name).concat(", age: ").concat(age.to_string())
print(message)

# Или использование функции format
message2 = format("Hello {}, age: {}", name, age)
```

### Текущие проблемы

1. **Плохая читаемость**: Конкатенация и форматирование строк требуют множества вызовов, код избыточен
2. **Подверженность ошибкам**: Ручная обработка преобразования типов, легко пропустить `.to_string()`
3. **Вопросы производительности**: Многократная конкатенация строк может влиять на производительность
4. **Недостаточная выразительность**: Невозможно наглядно встраивать сложные выражения в строки

## Предложение

### Основной дизайн

Введение f-string как нового префикса строковых литералов с поддержкой:
- **Интерполяция переменных**: `f"Hello {name}"`
- **Вычисление выражений**: `f"Sum: {x + y}"`
- **Спецификаторы формата**: `f"Pi: {pi:.2f}"`
- **Типовая безопасность**: Проверка типов выражений на этапе компиляции

### Примеры

```yaoxiang
# Базовая интерполяция
name = "Alice"
greeting = f"Hello {name}"  # "Hello Alice"

# Интерполяция выражений
x = 10
y = 20
result = f"Sum: {x + y}"    # "Sum: 30"

# Спецификаторы формата
pi = 3.14159
formatted = f"Pi: {pi:.2f}"  # "Pi: 3.14"

# Сложные выражения
items = [1, 2, 3]
s = f"Count: {len(items)}, sum: {sum(items)}"  # "Count: 3, sum: 6"

# Вызовы методов объекта
user = User("Bob", 25)
bio = f"Name: {user.name}, age: {user.get_age()}"
```

### Синтаксические изменения

| До | После | |
|------|------|
| `"Hello ".concat(name)` | `f"Hello {name}"` | |
| `format("Value: {}", value)` | `f"Value: {value}"` | |
| `format("Pi: {:.2f}", pi)` | `f"Pi: {pi:.2f}"` | |

### Спецификация синтаксиса

```
FStringLiteral ::= 'f' '"' FStringContent* '"'
FStringContent ::= FStringChar | EscapeSequence | FStringInterpolation
FStringInterpolation ::= '{' Expression (':' FormatSpec)? '}'
FormatSpec      ::= [width] ['.' precision] type
width           ::= digit+
precision       ::= digit+
type            ::= 'b' | 'c' | 'd' | 'e' | 'E' | 'f' | 'F' | 'g' | 'G' | 'n' | 'o' | 's' | 'x' | 'X' | '%'
```

## Детальный дизайн

### Синтаксический анализ

Компилятор распознаёт строковые литералы с префиксом `f` на этапе лексического анализа,解析花括号内的表达式和可选的格式化说明符。

### Стратегия трансформации

F-string компилируется в эффективные строковые операции на этапе компиляции:

**Простая интерполяция**：
```yaoxiang
f"Hello {name}"
```
Трансформируется в:
```yaoxiang
"Hello ".concat(name.to_string())
```

**Интерполяция выражений**：
```yaoxiang
f"Sum: {x + y}"
```
Трансформируется в:
```yaoxiang
"Sum: ".concat((x + y).to_string())
```

**Спецификаторы формата**：
```yaoxiang
f"Pi: {pi:.2f}"
```
Трансформируется в:
```yaoxiang
format("Pi: {:.2f}", pi)
```

**Множественная интерполяция**：
```yaoxiang
f"Hello {name}, you are {age} years old"
```
Трансформируется в:
```yaoxiang
"Hello ".concat(name.to_string()).concat(", you are ").concat(age.to_string()).concat(" years old")
```

### Влияние на систему типов

- Выражения интерполяции должны реализовывать интерфейс `Stringable` (автоматически реализуется для базовых типов и строк)
- Спецификаторы формата требуют поддержки соответствующего форматирования типом
- Компилятор проверяет соответствие типов выражений и правил форматирования

### Изменения в компиляторе

| Компонент | Изменение | |
|------|------|
| lexer | Распознавание префикса f, синтаксический анализ интерполяции в строках | |
| parser | Новый узел синтаксиса FStringLiteral | |
| typecheck | Проверка типов интерполяционных выражений, валидация правил форматирования | |
| codegen | Генерация кода вызовов конкатенации строк или форматирования | |

### Обратная совместимость

- ✅ Полная обратная совместимость
- Существующие строковые литералы `"..."` остаются без изменений
- F-string — новый синтаксис, не влияющий на существующий код

## Компромиссы

### Преимущества

1. **Краткий синтаксис**: Сокращение шаблонного кода, улучшение читаемости
2. **Типовая безопасность**: Проверка на этапе компиляции, сокращение ошибок времени выполнения
3. **Оптимизация производительности**: Компилятор может оптимизировать конкатенацию строк
4. **Высокая выразительность**: Поддержка произвольных выражений и форматирования
5. **Низкий порог вхождения**: Согласованность с экосистемой Python

### Недостатки

1. **Сложность компилятора**: Требуется добавить новую логику синтаксического анализа и трансформации
2. **Синтаксическая неоднозначность**: Необходимость отличать от существующего синтаксиса строк
3. **Сложность отладки**: Скомпилированный код отличается от исходного по структуре

## Альтернативные решения

| Решение | Почему не выбрано | |
|------|--------------|
| Только интерполяция переменных | Не удовлетворяет потребностям сложного форматирования | |
| Функциональный стиль `format(...)` | Синтаксис недостаточно краток | |
| Отложить до v2.0 | У пользователей есть чёткие потребности в удобстве работы со строками | |
| Использование обратных кавычек или другого префикса | Не согласовано с экосистемой Python | |

## Стратегия реализации

### Фазы разделения

1. **Фаза 1 (v0.9)**:
   - Базовая поддержка синтаксиса f-string
   - Интерполяция переменных и простых выражений
   - Преобразование базовых типов

2. **Фаза 2 (v1.0)**:
   - Поддержка спецификаторов формата
   - Интерполяция сложных выражений
   - Оптимизация производительности

3. **Фаза 3 (v1.1)**:
   - Улучшение отладочной информации
   - Улучшение сообщений об ошибках
   - Дополнительные опции форматирования

### Зависимости

- Нет внешних зависимостей
- Требуется поддержка базовой системы типов
- Требуются базовые функции строковой библиотеки

### Риски

1. **Риск производительности**: Множественная интерполяция может создавать избыточные строковые объекты
   - **Смягчение**: Оптимизация компилятором —合并相邻 строковых констант
2. **Сложность проверки типов**: Проверка типов спецификаторов формата
   - **Смягчение**: Ориентир на реализацию Python, использование простых и прямолинейных проверок
3. **Синтаксическая неоднозначность**: Вложенное использование `{` и `}`
   - **Смягчение**: Чёткие правила синтаксиса, ограничение вложенности

## Открытые вопросы

- [x] Поддержка экранированных фигурных скобок? Согласованно с Python: использование двойных фигурных скобок для одинарных, например <code v-pre>{{</code> представляет <code v-pre>{</code>, <code v-pre>}}</code> представляет <code v-pre>}</code>
- [x] Поддержка пользовательских функций форматирования? Согласованно с Python: поддержка пользовательского форматирования типов через метод `__format__`
- [x] Полная спецификация спецификаторов формата? Согласованно с Python, подробности в BNF выше
- [x] Конкретные стратегии оптимизации производительности? Согласованно с Python: конкатенация во время выполнения, без специальной оптимизации
- [x] Лучшие практики диагностики ошибок? Согласованно с Python: при ошибках отображать исходное содержимое f-string и позицию

## Приложения

### Приложение A: Справочник спецификаторов формата

| Тип | Спецификатор | Пример | Вывод | |
|------|--------|------|------|
| Целое | `d` | `f"{42:d}"` | "42" | |
| С плавающей точкой | `f` | `f"{3.14:.2f}"` | "3.14" | |
| Научная нотация | `e` | `f"{1000:e}"` | "1.000000e+03" | |
| Строка | `s` | `f"{name:s}"` | "Alice" | |
| Шестнадцатеричный | `x` | `f"{255:x}"` | "ff" | |

### Приложение B: Примеры сценариев использования

```yaoxiang
# Логирование
log(level: String, msg: String, count: Int) = () => {
    timestamp = get_timestamp()
    print(f"[{timestamp}] {level}: {msg} (count: {count})")
}

# Построение JSON
json = "{\n    \"name\": \"".concat(user.name).concat("\",\n    \"age\": ")
    .concat(user.age.to_string()).concat(",\n    \"email\": \"")
    .concat(user.email).concat("\"\n}")

# Построение SQL-запроса (внимание к риску SQL-инъекции)
query = f"SELECT * FROM users WHERE age > {min_age} AND status = '{status}'"

# Отладочная информация
debug_info = f"Point({x:.2f}, {y:.2f}) at {timestamp}"

# Условное форматирование
status_msg = if is_active {
    f"User {name} is active"
} else {
    f"User {name} is inactive"
}
```
