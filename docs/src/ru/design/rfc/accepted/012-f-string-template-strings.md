# RFC 012: F-String шаблонные строки

## Краткое содержание

Добавление в язык YaoXiang特性 f-string шаблонных строк, поддерживающих вставку переменных, вычисление выражений и форматированный вывод. f-string использует синтаксис в стиле Python (`f"..."` префикс), где выражения встраиваются через синтаксис `{expression}`, а во время компиляции преобразуются в эффективные операции со строками.

> **Примечание**: Синтаксис и поведение f-string соответствуют Python. Подробную спецификацию см. в [официальной документации Python](https://docs.python.org/3/tutorial/inputoutput.html#formatted-string-literals).

## Мотивация

### Зачем нужна эта特性?

Текущие способы конкатенации строк в YaoXiang достаточно громоздки:

```yaoxiang
# Текущее состояние: использование + для конкатенации
name = "Alice"
age = 30
message = "Hello ".concat(name).concat(", age: ").concat(age.to_string())
print(message)

# Или использование функции format
message2 = format("Hello {}, age: {}", name, age)
```

### Текущие проблемы

1. **Плохая читаемость**: Конкатенация и форматирование строк требуют множества вызовов, код избыточен
2. **Подверженность ошибкам**: Ручная обработка преобразования типов, легко пропустить `.to_string()`
3. **Вопросы производительности**: Многократная конкатенация строк может влиять на производительность
4. **Недостаточная выразительность**: Невозможно интуитивно встраивать сложные выражения в строки

## Предложение

### Основной дизайн

Введение f-string как нового префикса строковых литералов с поддержкой:
- **Вставка переменных**: `f"Hello {name}"`
- **Вычисление выражений**: `f"Sum: {x + y}"`
- **Спецификаторы формата**: `f"Pi: {pi:.2f}"`
- **Безопасность типов**: Проверка типов выражений во время компиляции

### Примеры

```yaoxiang
# Базовая вставка
name = "Alice"
greeting = f"Hello {name}"  # "Hello Alice"

# Вставка выражений
x = 10
y = 20
result = f"Sum: {x + y}"    # "Sum: 30"

# Спецификаторы формата
pi = 3.14159
formatted = f"Pi: {pi:.2f}"  # "Pi: 3.14"

# Сложные выражения
items = [1, 2, 3]
s = f"Count: {len(items)}, sum: {sum(items)}"  # "Count: 3, sum: 6"

# Вызов методов объекта
user = User("Bob", 25)
bio = f"Name: {user.name}, age: {user.get_age()}"
```

### Синтаксические изменения

| До | После |
|------|------|
| `"Hello ".concat(name)` | `f"Hello {name}"` |
| `format("Value: {}", value)` | `f"Value: {value}"` |
| `format("Pi: {:.2f}", pi)` | `f"Pi: {pi:.2f}"` |

### Спецификация синтаксиса

```
FStringLiteral ::= 'f' '"' FStringContent* '"'
FStringContent ::= FStringChar | EscapeSequence | FStringInterpolation
FStringInterpolation ::= '{' Expression (':' FormatSpec)? '}'
FormatSpec      ::= [width] ['.' precision] type
width           ::= digit+
precision       ::= digit+
type            ::= 'b' | 'c' | 'd' | 'e' | 'E' | 'f' | 'F' | 'g' | 'G' | 'n' | 'o' | 's' | 'x' | 'X' | '%'
```

## Детальный дизайн

### Синтаксический анализ

Компилятор распознаёт строковые литералы с префиксом `f` на этапе лексического анализа, выполняет разбор выражений внутри фигурных скобок и необязательных спецификаторов формата.

### Стратегия преобразования

f-string во время компиляции преобразуется в эффективные операции со строками:

**Простая вставка**:
```yaoxiang
f"Hello {name}"
```
Преобразуется в:
```yaoxiang
"Hello ".concat(name.to_string())
```

**Вставка выражений**:
```yaoxiang
f"Sum: {x + y}"
```
Преобразуется в:
```yaoxiang
"Sum: ".concat((x + y).to_string())
```

**Спецификаторы формата**:
```yaoxiang
f"Pi: {pi:.2f}"
```
Преобразуется в:
```yaoxiang
format("Pi: {:.2f}", pi)
```

**Множественные вставки**:
```yaoxiang
f"Hello {name}, you are {age} years old"
```
Преобразуется в:
```yaoxiang
"Hello ".concat(name.to_string()).concat(", you are ").concat(age.to_string()).concat(" years old")
```

### Влияние на систему типов

- Выражения вставки должны реализовывать интерфейс `Stringable` (автоматически реализуется для базовых типов и строк)
- Спецификаторы формата требуют поддержки соответствующего форматирования типом
- Компилятор проверяет соответствие типов выражений и правил форматирования

### Изменения в компиляторе

| Компонент | Изменения |
|------|------|
| lexer | Распознавание префикса f, синтаксический анализ вставки в строках |
| parser | Новый узел синтаксиса FStringLiteral |
| typecheck | Проверка типов вставленных выражений, валидация правил форматирования |
| codegen | Генерация кода конкатенации строк или вызовов форматирования |

### Обратная совместимость

- ✅ Полная обратная совместимость
- Существующие строковые литералы `"..."` остаются без изменений
- f-string — новый синтаксис, не влияющий на существующий код

## Компромиссы

### Преимущества

1. **Компактный синтаксис**: Уменьшение шаблонного кода, повышение читаемости
2. **Безопасность типов**: Проверка во время компиляции, сокращение ошибок времени выполнения
3. **Оптимизация производительности**: Компилятор может оптимизировать конкатенацию строк
4. **Высокая выразительность**: Поддержка произвольных выражений и форматирования
5. **Низкий порог вхождения**: Согласованность с экосистемой Python

### Недостатки

1. **Сложность компилятора**: Необходимость добавления синтаксического анализа и логики преобразования
2. **Синтаксическая неоднозначность**: Необходимость отличения от существующего синтаксиса строк
3. **Сложности отладки**: Преобразованный код отличается от исходного

## Альтернативные решения

| Решение | Почему не выбрано |
|------|--------------|
| Только поддержка вставки переменных | Не удовлетворяет потребностям сложного форматирования |
| Использование функционального стиля `format(...)` | Синтаксис недостаточно компактный |
| Отложить до v2.0 | У пользователей есть явная потребность в удобстве работы со строками |
| Использование обратных кавычек или другого префикса | Не согласовано с экосистемой Python |

## Стратегия реализации

### Фазы

1. **Фаза 1 (v0.9)**:
   - Базовая поддержка синтаксиса f-string
   - Вставка переменных и простых выражений
   - Преобразование базовых типов

2. **Фаза 2 (v1.0)**:
   - Поддержка спецификаторов формата
   - Вставка сложных выражений
   - Оптимизация производительности

3. **Фаза 3 (v1.1)**:
   - Улучшение отладочной информации
   - Улучшение сообщений об ошибках
   - Дополнительные опции форматирования

### Зависимости

- Без внешних зависимостей
- Требуется базовая поддержка системы типов
- Требуются базовые функции библиотеки строк

### Риски

1. **Риск производительности**: Множественные вставки могут создавать избыточные объекты строк
   - **Смягчение**: Оптимизация компилятором для объединения смежных строковых констант
2. **Сложность проверки типов**: Проверка типов для спецификаторов формата
   - **Смягчение**: Ориентир на реализацию Python, использование простой прямой проверки
3. **Синтаксическая неоднозначность**: Вложенное использование `{` и `}`
   - **Смягчение**: Чёткие правила синтаксиса, ограничение вложенности

## Открытые вопросы

- [x] Поддержка экранированных фигурных скобок? Как в Python: использование двойных фигурных скобок для одиночной, например <code v-pre>{{</code> обозначает <code v-pre>{</code>, <code v-pre>}}</code> обозначает <code v-pre>}</code>
- [x] Поддержка пользовательских функций форматирования? Как в Python: поддержка настройки форматирования типов через метод `__format__`
- [x] Полная спецификация спецификаторов формата? Как в Python, подробности в BNF выше
- [x] Конкретные стратегии оптимизации производительности? Как в Python: конкатенация во время выполнения, без специальной оптимизации
- [x] Лучшие практики диагностики ошибок? Как в Python: при ошибке показывать исходное содержимое f-string и позицию

## Приложения

### Приложение А: Справочник по спецификаторам формата

| Тип | Спецификатор | Пример | Вывод |
|------|--------|------|------|
| Целое | `d` | `f"{42:d}"` | "42" |
| С плавающей точкой | `f` | `f"{3.14:.2f}"` | "3.14" |
| Научная нотация | `e` | `f"{1000:e}"` | "1.000000e+03" |
| Строка | `s` | `f"{name:s}"` | "Alice" |
| Шестнадцатеричное | `x` | `f"{255:x}"` | "ff" |

### Приложение Б: Примеры сценариев использования

```yaoxiang
# Ведение журнала
log(level: String, msg: String, count: Int) = () => {
    timestamp = get_timestamp()
    print(f"[{timestamp}] {level}: {msg} (count: {count})")
}

# Построение JSON
json = "{\n    \"name\": \"".concat(user.name).concat("\",\n    \"age\": ")
    .concat(user.age.to_string()).concat(",\n    \"email\": \"")
    .concat(user.email).concat("\"\n}")

# Построение SQL-запросов (остерегайтесь SQL-инъекций)
query = f"SELECT * FROM users WHERE age > {min_age} AND status = '{status}'"

# Отладочная информация
debug_info = f"Point({x:.2f}, {y:.2f}) at {timestamp}"

# Условное форматирование
status_msg = if is_active {
    f"User {name} is active"
} else {
    f"User {name} is inactive"
}
```
