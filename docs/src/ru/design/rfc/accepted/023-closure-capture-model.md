# RFC-023: Модель захвата замыканий

> **Статус**: Принято
> **Автор**: Чэнь Сюй
> **Дата создания**: 2026-05-29
> **Последнее обновление**: 2026-05-29

> **Ссылки**:
> - [RFC-007: Унификация синтаксиса функций](./accepted/007-function-syntax-unification.md)
> - [RFC-009: Модель владения v9](./accepted/009-ownership-model.md)
> - [RFC-011: Проектирование системы обобщённых типов](./accepted/011-generic-type-system.md) — Раздел 2.4: Встроенный marker trait Dup/Clone

## Аннотация

Данный RFC определяет **модель захвата замыканий** языка YaoXiang. Компилятор автоматически анализирует внешние переменные, на которые ссылается тело замыкания, и в зависимости от типа переменной (Dup/не-Dup) и от того, замыкает ли замыкание, автоматически выбирает способ захвата — для типов Dup выполняется прямое копирование, для не-Dup без побега выполняется заимствование, для не-Dup с побегом выполняется Move. Пользователю не нужны аннотации; используется единый набор правил, общий с автоматическим выбором заимствования при вызове функций.

## Мотивация

### Зачем это нужно?

В текущей реализации захват замыканий **не реализован** — поле `env` инструкции `MakeClosure` всегда пустое, лямбда не может ссылаться ни на какие внешние переменные. Система токенов заимствования требует, чтобы замыкания могли захватывать токены `&T` (нуль-стоимостное копирование) — это ключевой сценарий использования.

### Текущая проблема

```yaoxiang
# Такой код сейчас не компилируется — лямбда не может сослаться на threshold
filter_by: (items: List(Point), threshold: &Float) -> List(Point) = {
    items.filter(|p| p.x > threshold)  # ❌ threshold невозможно захватить
}
```

## Предложение

### Основная концепция

Захват замыканий определяется компилятором полностью автоматически. Правила **полностью идентичны** правилам автоматического выбора заимствования при вызове функций:

```
Тип переменной  Замыкание сбегает  Способ захвата
─────────────────────────────────────────────────
Dup              Любой              Копирование (побитовая копия или нуль-стоимость)
Не-Dup           Не сбегает         Автоматическое заимствование (&T или &mut T)
Не-Dup           Сбегает            Move (передача владения)
```

**Определение побега**:

```
spawn { || ... }           → Сбегает
return || ...              → Сбегает
let x = || ... ;  x  полe  → Сбегает
items.filter(|p| ...)      → Не сбегает (синхронный вызов высшего порядка)
||.method()                → Не сбегает (вызов на месте)
```

Консервативный принцип: при невозможности определения считать, что замыкание сбегает.

### Примеры

```yaoxiang
# 1. Токен Dup — прямое копирование (нуль-стоимость)
filter_by: (items: List(Point), threshold: &Float) -> List(Point) = {
    # threshold: &Float → Dup → компилятор копирует токен в замыкание
    # Токен нулевого размера, нулевые накладные расходы времени выполнения
    items.filter(|p| p.x > threshold)
}

# 2. Не-Dup + без побега — автоматическое заимствование
process: (buf: Buffer) -> Void = {
    # buf не-Dup, filter не сбегает → автоматическое создание токена &Buffer
    transform(|b| b.read())
    # После возврата замыкания токен освобождается, buf снова доступен
}

# 3. Замыкание сбегает — Move
spawn_worker: (data: Data) -> Void = {
    # data не-Dup, spawn → сбегает → Move
    spawn { use(data) }
}

# 4. Смешанный захват
complex: (items: List(Point), config: &Config, buf: Buffer) -> List(Point) = {
    # config: &Config → Dup → копирование токена
    # buf: Buffer → не-Dup, не сбегает → заимствование &mut Buffer
    items.filter(|p| {
        let threshold = config.get_threshold()
        buf.update(p)
        p.x > threshold
    })
}

# 5. Обнаружение конфликта заимствования
bad: (buf: Buffer) -> Void = {
    closure = |b| b.write()
    buf.read()  # ❌ buf уже заимствован замыканием, конфликт
}
```

### Синтаксические изменения

**Синтаксических изменений нет**. Способ захвата определяется компилятором автоматически, пользователю не нужны аннотации.

## Детальное проектирование

### Влияние на систему типов

Сигнатура типа лямбды остаётся неизменной: `(params) -> Return`. Захваченные переменные не отражаются в сигнатуре типа; они обрабатываются компилятором на этапе генерации IR.

### Изменения в компиляторе

| Компонент | Изменение | Описание |
|-----------|-----------|----------|
| `capture.rs` (новый) | Анализ захвата + анализ побега + выбор模式 | ~150 строк |
| `expressions.rs` | Вывод типа лямбды вызывает анализ захвата | ~10 строк |
| `ir_gen.rs` | Заполнение env в MakeClosure; пропуск ZST | ~80 строк |
| `ir.rs` | Возможно, нужно скорректировать тип env в MakeClosure | ~5 строк |

**Процесс анализа захвата**:

```
1. Обход AST тела лямбды
2. Сбор всех ссылок Expr::Var(name)
3. Фильтрация: только переменные из внешней области видимости замыкания
4. Классификация: Read (только чтение) / Write (чтение-запись) / Move (перемещена)
5. Проверка атрибутов типа: является ли Dup
6. Определение побега: способ использования замыкания
7. Выбор模式 захвата:
   Dup → Copy
   Не-Dup + не сбегает + Read → Borrow (&T)
   Не-Dup + не сбегает + Write → BorrowMut (&mut T)
   Не-Dup + сбегает → Move
```

**Генерация IR**:

```rust
// Текущее состояние (пусто)
Instruction::MakeClosure { dst, func, env: Vec::new() }

// Изменено на
Instruction::MakeClosure { dst, func, env: captured_env }

// Логика генерации captured_env:
for captured in captures {
    match captured.mode {
        Copy if is_zst(captured.ty) => {
            // Тип нулевого размера — не генерируем никаких инструкций
            // Замыкание напрямую ссылается на внешнюю переменную (устраняется на этапе компиляции)
        }
        Copy => {
            // Генерируем Move dst, src (поверхностное копирование для типа Dup)
        }
        Borrow => {
            // Генерируем Borrow dst, src (создание ReadToken)
        }
        BorrowMut => {
            // Генерируем Borrow dst, src (создание WriteToken)
        }
        Move => {
            // Генерируем Move dst, src (передача владения)
        }
    }
}
```

### Поведение во время выполнения

Способ захвата не влияет на производительность во время выполнения:

- **Dup + ZST** (например, токен `&T`) → Ноль инструкций, замыкание напрямую ссылается на внешнюю переменную
- **Dup + не-ZST** (например, Int) → Одно копирование в регистр
- **Borrow/BorrowMut** → Создание токена (концепция времени компиляции, нулевые накладные расходы)
- **Move** → Та же стоимость, что и обычный Move

### Обратная совместимость

Полная совместимость. В настоящее время все лямбды не могут захватывать внешние переменные; данный RFC только добавляет выразительность, не нарушая существующий код.

## Компромиссы

### Преимущества

1. **Нуль аннотаций**: Пользователю не нужно писать никаких аннотаций захвата
2. **Единство с вызовами функций**: Правила захвата = правила автоматического заимствования функций
3. **Нуль-стоимость**: Захват токенов Dup полностью устраняется на этапе компиляции
4. **Безопасность**: Анализ побега предотвращает use-after-free

### Недостатки

1. **Консервативный анализ побега**: При невозможности определения выполняется Move, что может быть излишним
2. **Неявность**: Способ захвата не отражён в коде; для отладки нужно смотреть вывод компилятора

## Альтернативные решения

| Решение | Почему не выбрано |
|---------|-------------------|
| Явное ключевое слово `move` в стиле Rust | Введение нового синтаксиса, увеличение когнитивной нагрузки на пользователей |
| Полный Move | Невозможно выразить нуль-стоимостное заимствование токенов |
| Полное заимствование | Побег замыкания приведёт к висячей ссылке |
| Ручные аннотации пользователем | Противоречит философии проектирования «компилятор автоматизирует всё» |

## Стратегия реализации

### Фазы

1. **Фаза 1**: Анализ захвата (только идентификация ссылок на внешние переменные, без различия способов захвата)
2. **Фаза 2**: Анализ побега + выбор模式
3. **Фаза 3**: Генерация IR + оптимизация ZST
4. **Фаза 4**: Интеграция обнаружения конфликтов заимствования

### Зависимости

- Зависит от RFC-011 (система обобщённых типов, раздел 2.4 трейт Dup/Clone) — требуется трейт Dup для определения копируемости переменных
- Зависит от RFC-009 v9 (токены заимствования) — режимы захвата Borrow/BorrowMut требуют типов токенов
- После реализации RFC-023 можно приступать к реализации системы токенов заимствования (RFC-009 v9)

### Риски

- Анализ побега может быть слишком консервативным, что приводит к излишнему Move; можно оптимизировать в будущем
- Анализ захвата обобщённых замыканий может потребовать дополнительной обработки
