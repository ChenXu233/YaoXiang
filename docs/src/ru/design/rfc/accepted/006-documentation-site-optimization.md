# RFC-006: Создание сайта документации

> **Ссылка**: См. [шаблон RFC](RFC_TEMPLATE.md) для ознакомления с правилами оформления RFC.

## Резюме

Создать сайт документации YaoXiang, объединить разрозненные документы, обеспечить поддержку поиска, навигации, мультиязычности и переключения версий.

## Мотивация

### Зачем нужна эта функциональность?

В настоящее время документация разбросана по нескольким каталогам и отображается только через GitHub Readme. Новым пользователям сложно найти нужную информацию, поиск отсутствует, документация на китайском и английском языках не синхронизирована.

### Текущие проблемы

```
docs/
├── README.md              # Главный указатель (ограниченное содержимое)
├── tutorial/              # Учебные пособия
├── guides/                # Руководства
├── architecture/          # Документация по архитектуре
├── design/                # Проектная документация
├── examples/              # Примеры
├── plans/                 # Планы внедрения
├── implementation/        # Документация по реализации
├── maintenance/           # Документация по сопровождению
└── archived/              # Архив
```

Проблемы:
1. Отсутствует единая точка входа, только GitHub Readme
2. Невозможен поиск
3. Нет переключения версий — пользователи могут читать устаревшую документацию
4. Файлы .obsidian попадают в систему контроля версий

### Спецификация URL-путей (основной дизайн)

| Сценарий | Формат URL | Описание |
|----------|-----------|----------|
| Последняя версия, китайский | `/zh/getting-started/` | Перенаправление на последнюю версию |
| Последняя версия, английский | `/en/getting-started/` | Перенаправление на последнюю версию |
| Указанная версия | `/v0.5/zh/getting-started/` | Префикс номера версии |
| Главная страница | `/zh/` или `/en/` | Языковая главная страница |

**Дизайн переключения версий**:
```
Выпадающее меню переключения версий:
├── v0.6 (latest)
├── v0.5
├── v0.4
└── v0.3
```

**Спецификация путей версий** (ключевое решение, сложно изменить позже):
- Последняя версия: `/zh/xxx/` → перенаправление на последнюю версию
- Указанная версия: `/v0.5/zh/xxx/` → фиксированная версия
- Переключение версий в навигации: переключение комбинации `/v0.5/` и `/zh/`

### Интеграция CI/CD

```yaml
# .github/workflows/docs-deploy.yml
name: Deploy Docs

on:
  push:
    branches: [main]
    paths: ['docs/**', '!.obsidian/**']

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
        working-directory: docs
      - run: npm run build
      - uses: actions/deploy-pages@v4
        with:
          build_dir: docs/.vitepress/dist
```

### Конфигурация сайта

```typescript
// docs/.vitepress/config.mts
import { defineConfig } from 'vitepress'
import starlight from '@astrojs/starlight'

export default defineConfig({
  title: 'YaoXiang',
  description: 'Язык программирования, ориентированный на будущее',

  locales: {
    root: { label: '中文', lang: 'zh-CN', link: '/zh/' },
    en: { label: 'English', lang: 'en-US', link: '/en/' },
  },

  // Локальный поиск
  plugins: [
    starlight({
      title: 'YaoXiang',
      localSearch: {},
    }),
  ],

  // Ссылка для редактирования
  editLink: {
    pattern: 'https://github.com/yaoxiang-lang/yaoxiang/edit/main/docs/:path',
  },
})
```

## Компромиссы

### Преимущества

- Профессиональный сайт документации повышает имидж проекта
- Пользователи быстро находят нужную информацию
- Локального поиска достаточно и он бесплатен
- Мультиязычная поддержка служит международному сообществу
- Переключение версий предотвращает чтение устаревшей документации

### Недостатки

- Стоимость сопровождения: необходима поддержка конфигурации сайта
- Введение технологического стека: Node.js

## Альтернативы

| Вариант | Почему не выбран |
|---------|------------------|
| GitHub Wiki | Плохой поиск, низкая кастомизируемость |
| Только README | Нет поиска, нет навигации |
| Docusaurus | Слишком тяжёлый, медленный запуск |
