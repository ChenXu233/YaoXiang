# RFC-006: Строительство документационного сайта

> **Статус**: Принято
> **Автор**: Чэнь Сюй
> **Дата создания**: 2025-01-05
> **Последнее обновление**: 2026-02-12

> **Справка**: Смотрите [шаблон RFC](RFC_TEMPLATE.md) для ознакомления с RFC-стандартами.

## Краткое описание

Создание документационного сайта YaoXiang для интеграции разрозненной документации с поддержкой поиска, навигации, многоязычности и переключения версий.

## Мотивация

### Зачем нужна эта функция?

Текущая документация разбросана по нескольким каталогам и представлена только через GitHub Readme, что затрудняет новым пользователям поиск нужной информации, отсутствует поиск, китайская и английская документация не синхронизированы.

### Текущие проблемы

```
docs/
├── README.md              # Основной индекс (ограниченное содержание)
├── tutorial/              # Учебные пособия
├── guides/               # Руководства
├── architecture/          # Архитектурная документация
├── design/               # Проектная документация
├── examples/             # Примеры
├── plans/                # Планы реализации
├── implementation/       # Документация реализации
├── maintenance/          # Документация по сопровождению
└── archived/             # Архив
```

Проблемы:
1. Нет единой точки входа, только GitHub Readme
2. Поиск недоступен
3. Нет переключения версий, пользователи могут читать устаревшую документацию
4. .obsidian смешан с контролем версий

### Спецификация URL-путей (основной дизайн)

| Сценарий | Формат URL | Описание |
|----------|-----------|----------|
| Последняя версия на китайском | `/zh/getting-started/` | Перенаправление на последнюю версию |
| Последняя версия на английском | `/en/getting-started/` | Перенаправление на последнюю версию |
| Указанная версия | `/v0.5/zh/getting-started/` | Префикс с номером версии |
| Главная страница | `/zh/` или `/en/` | Языковая главная страница |

**Дизайн переключения версий**:
```
Выпадающее меню переключения версий:
├── v0.6 (latest)
├── v0.5
├── v0.4
└── v0.3
```

**Спецификация путей версий** (ключевое решение, трудно изменить впоследствии):
- Последняя версия: `/zh/xxx/` → перенаправление на последнюю версию
- Указанная версия: `/v0.5/zh/xxx/` → фиксированная версия
- Переключение версий в панели навигации: переключение комбинаций `/v0.5/` и `/zh/`

### Интеграция CI/CD

```yaml
# .github/workflows/docs-deploy.yml
name: Deploy Docs

on:
  push:
    branches: [main]
    paths: ['docs/**', '!.obsidian/**']

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
        working-directory: docs
      - run: npm run build
      - uses: actions/deploy-pages@v4
        with:
          build_dir: docs/.vitepress/dist
```

### Конфигурация сайта

```typescript
// docs/.vitepress/config.mts
import { defineConfig } from 'vitepress'
import starlight from '@astrojs/starlight'

export default defineConfig({
  title: 'YaoXiang',
  description: 'Язык программирования, обращённый в будущее',

  locales: {
    root: { label: '中文', lang: 'zh-CN', link: '/zh/' },
    en: { label: 'English', lang: 'en-US', link: '/en/' },
  },

  // Локальный поиск
  plugins: [
    starlight({
      title: 'YaoXiang',
      localSearch: {},
    }),
  ],

  // Ссылка для редактирования
  editLink: {
    pattern: 'https://github.com/yaoxiang-lang/yaoxiang/edit/main/docs/:path',
  },
})
```

## Компромиссы

### Преимущества

- Профессиональный документационный сайт улучшает имидж проекта
- Пользователи быстро находят нужную информацию
- Локальный поиск бесплатен и достаточен
- Многоязычная поддержка обслуживает международное сообщество
- Переключение версий предотвращает чтение устаревшей документации

### Недостатки

- Затраты на сопровождение: требуется поддержка конфигурации сайта
- Введение технологического стека: Node.js

## Альтернативные решения

| Решение | Почему не выбрано |
|---------|-------------------|
| GitHub Wiki | Плохой поиск, низкая кастомизация |
| Только README | Нет поиска, нет навигации |
| Docusaurus | Слишком тяжёлый, медленный запуск |
