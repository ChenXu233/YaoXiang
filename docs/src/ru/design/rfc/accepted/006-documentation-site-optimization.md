# RFC-006: Создание документационного сайта

> **Справка**: Ознакомьтесь с [шаблоном RFC](RFC_TEMPLATE.md) для понимания спецификации RFC.

## Аннотация

Создание документационного сайта YaoXiang для интеграции разрозненной документации с поддержкой поиска, навигации, многоязычности и переключения версий.

## Мотивация

### Зачем нужен этот функционал?

Документация в настоящее время разбросана по нескольким каталогам и представлена только через GitHub Readme. Новым пользователям сложно найти нужную информацию, отсутствует поиск, а китайская и английская версии документации не синхронизированы.

### Текущие проблемы

```
docs/
├── README.md              # Главный индекс (ограниченное содержимое)
├── tutorial/              # Учебные пособия
├── guides/               # Руководства
├── architecture/          # Документация по архитектуре
├── design/               # Проектная документация
├── examples/             # Примеры
├── plans/                # Планы реализации
├── implementation/       # Документация реализации
├── maintenance/          # Документация по сопровождению
└── archived/             # Архив
```

Проблемы:
1. Отсутствует единая точка входа — только GitHub Readme
2. Невозможен поиск
3. Отсутствует переключение версий — пользователи могут читать устаревшую документацию
4. .obsidian смешан с системой контроля версий

### Спецификация URL-путей (основной дизайн)

| Сценарий | Формат URL | Описание |
|------|---------|------|
| Последняя версия, китайский | `/zh/getting-started/` | Перенаправление на последнюю версию |
| Последняя версия, английский | `/en/getting-started/` | Перенаправление на последнюю версию |
| Указанная версия | `/v0.5/zh/getting-started/` | Префикс с номером версии |
| Главная страница | `/zh/` или `/en/` | Языковая главная страница |

**Дизайн переключения версий**:
```
Выпадающее меню переключения версий:
├── v0.6 (latest)
├── v0.5
├── v0.4
└── v0.3
```

**Спецификация путей версий** (критическое решение, трудно изменить впоследствии):
- Последняя версия: `/zh/xxx/` → перенаправление на последнюю версию
- Указанная версия: `/v0.5/zh/xxx/` → фиксированная версия
- Переключение версий в навигационной панели: переключение комбинаций `/v0.5/` и `/zh/`

### Интеграция CI/CD

```yaml
# .github/workflows/docs-deploy.yml
name: Deploy Docs

on:
  push:
    branches: [main]
    paths: ['docs/**', '!.obsidian/**']

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
        working-directory: docs
      - run: npm run build
      - uses: actions/deploy-pages@v4
        with:
          build_dir: docs/.vitepress/dist
```


### Конфигурация сайта

```typescript
// docs/.vitepress/config.mts
import { defineConfig } from 'vitepress'
import starlight from '@astrojs/starlight'

export default defineConfig({
  title: 'YaoXiang',
  description: 'Язык программирования будущего',

  locales: {
    root: { label: '中文', lang: 'zh-CN', link: '/zh/' },
    en: { label: 'English', lang: 'en-US', link: '/en/' },
  },

  // Локальный поиск
  plugins: [
    starlight({
      title: 'YaoXiang',
      localSearch: {},
    }),
  ],

  // Ссылка для редактирования
  editLink: {
    pattern: 'https://github.com/yaoxiang-lang/yaoxiang/edit/main/docs/:path',
  },
})
```

## Компромиссы

### Преимущества

- Профессиональный документационный сайт повышает имидж проекта
- Пользователи быстро находят нужную информацию
- Локальный поиск бесплатен и достаточен
- Поддержка многоязычности обслуживает международное сообщество
- Переключение версий предотвращает чтение устаревшей документации

### Недостатки

- Затраты на сопровождение: требуется поддержка конфигурации сайта
- Введение технологического стека: Node.js

## Альтернативные решения

| Решение | Почему не выбрано |
|------|-----------|
| GitHub Wiki | Слабый поиск, низкая кастомизация |
| Только README | Отсутствие поиска и навигации |
| Docusaurus | Избыточность, медленный запуск |
