# RFC-004: Дизайн мульти-позиционного объединённого связывания каррированных методов

## Аннотация

Данный RFC предлагает совершенно новый **синтаксис мульти-позиционного объединённого связывания**, позволяющий точно привязывать функции к произвольным позициям параметров типа, поддерживающий как однопозиционное, так и мульти-позиционное объединённое связывание, и фундаментально решающий проблему «кто является вызывающим» при каррированном связывании без введения ключевого слова `self`.

## Мотивация

### Зачем нужна эта возможность?

В текущем дизайне языка при привязке независимых функций в качестве методов типа возникают следующие проблемы:

1. **Негибкая позиция вызывающего**: традиционное связывание может зафиксировать только `obj` в качестве первого аргумента в `obj.method(args)`
2. **Трудности с мультипараметрическим связыванием**: когда метод должен принимать несколько параметров одного типа, невозможно выразить это элегантно
3. **Семантическая неоднозначность каррирования**: при частичном применении трудно различить «к какой позиции привязано»

### Цель дизайна: унифицировать два взгляда на программирование

Данный дизайн направлен на **унификацию функционального и ООП взглядов на программирование**:

```yaoxiang
# Функциональный взгляд: явная передача всех аргументов
distance(p1, p2)

# ООП взгляд: неявный this
p1.distance(p2)

# Синтаксический сахар [positions] делает обе записи эквивалентными, по сути это вызовы функций
Point.distance = distance[0]   # this привязан к позиции 0
```

**Ключевые ценности**:
- Внизу — функция, наверху — синтаксис метода
- Без введения ключевого слова `self`, сохраняя простоту языка
- Полная функциональность: вызов метода по сути является передачей параметров
- `[0]`, `[1]`, `[-1]` гибко управляют позицией связывания this
- **Единый синтаксис**: определение функции использует формат `name: (params) -> Return = body`

### Текущие проблемы

```yaoxiang
# Проблемы существующего дизайна:
Point: Type = { x: Float, y: Float }
Vector: Type = { x: Float, y: Float, z: Float }

distance: (a: Point, b: Point) -> Float = { ... }
transform: (p: Point, v: Vector) -> Point = { ... }

# Можно привязать только к первому аргументу
Point.distance = distance  # Эквивалентно distance[0]
# p1.distance(p2) → distance(p1, p2) ✓

# Но что если сигнатура transform имеет вид transform(Vector, Point)?
# Невозможно выразить семантику p1.transform(v1) → transform(v1, p1)
```

## Предложение

### Основной дизайн: явное указание позиции

**Ключевое правило: без `[n]` = без связывания.** `Point.name = func` — это лишь псевдоним пространства имён, который не вызывает никакого неявного связывания. Чтобы синтаксис вызова с точкой `p.name(args)` работал, необходимо явно указать: `Point.name = func[n]`.

#### Однопозиционное связывание

```yaoxiang
# Явное связывание с первой позицией аргумента типа Point (индексация с 0)
Point.distance = distance[0]
p1.distance(p2)                     # → distance(p1, p2)

# Связывание со второй позицией аргумента типа Point
Point.compare = distance[1]         # Связывание со вторым аргументом Point
p1.compare(p2)                      # → distance(p2, p1)
```

**Без `[n]` = без связывания**:

```yaoxiang
# Без [n] → чистый псевдоним пространства имён, без синтаксиса вызова с точкой
Point.distance = distance            # Доступно только Point.distance(p1, p2)
# p1.distance(p2)  ❌  Связывание отсутствует

# Фабричные функции естественно допустимы и не требуют специальной обработки
create_point: () -> Point = { ... }
Point.create = create_point          # Point.create()   ✅
```
- Безопасность типов: связывание выполняется только при совпадении типов, что предотвращает ошибки
- Гибкое управление: `[n]` точно контролирует позицию связывания

#### Каррированное связывание

Когда количество аргументов функции превышает количество позиций связывания, автоматически генерируется каррированная функция. **Связывание всегда является явной операцией.**

```yaoxiang
Point: Type = { x: Float, y: Float }

# Базовая функция
scale: (p: Point, factor: Float) -> Point = {
    return Point(p.x * factor, p.y * factor)
}

# Явное связывание с позицией 0 → каррирование: оставшийся аргумент factor предоставляется вызывающим
Point.scale = scale[0]

# Вызов
p1 = Point(2.0, 3.0)
scaled = p1.scale(2.0)       # → scale(p1, 2.0)

# Цепочечный вызов выглядит ещё элегантнее
result = Point(2.0, 3.0).scale(2.0)  # → Point(4.0, 6.0)
```

### Синтаксис связывания по индексу позиции

Вводится синтаксис `[position]` для точного управления связью между аргументами функции и типом:

```yaoxiang
# Формат синтаксиса: Type.method = function[positions]

# === Базовое связывание ===

# Однопозиционное связывание
Point.distance = distance[1]           # Связывание с аргументом 1 (индексация с 0)
# Использование: p1.distance(p2) → distance(p2, p1)

# Мульти-позиционное объединённое связывание (деструктуризация кортежа)
Point.transform = transform[1, 2]      # Связывание с аргументами 1, 2
# Использование: p1.transform(v1) → transform(v1, p1)
# Исходная сигнатура функции: transform(Point, Vector) → Point
# После связывания: Point.transform(Vector) → Point
```

### Подробное определение синтаксиса

```
ОбъявлениеСвязывания ::= Тип '.' Идентификатор '=' ИмяФункции '[' СписокПозиций ']'

СписокПозиций ::= Позиция (',' Позиция)*
Позиция        ::= ЦелоеЧисло                  # заполнитель
                | '_'                          # пропуск этой позиции (заполнитель)
                | ЦелоеЧисло '..' ЦелоеЧисло   # диапазон позиций (будущее расширение)

ИмяФункции ::= Идентификатор
Тип         ::= Идентификатор (Дженерики)?
```

### Встроенное связывание

Связывание можно записать непосредственно в теле определения типа, без отдельного оператора связывания:

```yaoxiang
# Способ 1: прямое связывание в теле определения типа
Point: Type = {
    x: Float = 0,
    y: Float = 0,
    distance = distance[0]           # Связывание с позицией 0
}

# Способ 2: анонимная функция + позиционное связывание
Point: Type = {
    x: Float = 0,
    y: Float = 0,
    distance: ((a: Point, b: Point) -> Float)[0] = ((a, b) => {
        dx = a.x - b.x
        dy = a.y - b.y
        return (dx * dx + dy * dy).sqrt()
    })
}
# Синтаксис: ((params) => body)[position]
```

**Семантика каррирования**:
- При связывании `distance = distance[0]` исходная сигнатура функции `(a: Point, b: Point) -> Float`
- Сгенерированная сигнатура метода: `b: Point -> Float` (позиция 0 заполняется вызывающим)

### Примеры использования

```yaoxiang
# === Полные примеры ===

Point: Type = { x: Float, y: Float }
Vector: Type = { x: Float, y: Float, z: Float }

# 1. Базовое вычисление расстояния
distance: (a: Point, b: Point) -> Float = {
    dx = a.x - b.x
    dy = a.y - b.y
    return (dx * dx + dy * dy).sqrt()
}

# Связывание: Point.distance = distance[1]
# Вызов: p1.distance(p2) → distance(p2, p1)
# Но мы хотим p1.distance(p2) → distance(p1, p2), поэтому:
Point.distance = distance[0]

# 2. Операция трансформации (мульти-позиционное связывание)
transform: (p: Point, v: Vector) -> Point = {
    return Point(p.x + v.x, p.y + v.y)
}

# Связывание Point.transform = transform[1]
# Вызов: p.transform(v) → transform(v, p) ❌
# Связывание Point.transform = transform[0]
# Вызов: p.transform(v) → transform(p, v) ✓

# 3. Сложная функция со многими аргументами
multiply: (a: Point, s: Float) -> Point = {
    return Point(a.x * s, a.y * s)
}

# Связываем только 1-й аргумент (тип Point), сохраняем 3-й аргумент
Point.scale = multiply[0, _]
# Вызов: p.scale(2.0) → multiply(p, 2.0)

# 4. Межтиповое связывание
Circle: Type = { center: Point, radius: Float }

distance: (a: Circle, b: Circle) -> Float = {
    return a.center.distance(b.center) - a.radius - b.radius
}

# Привязка метода расстояния к типу Circle
Circle.distance = distance[0, 1]
# Вызов: c1.distance(c2) → distance(c1, c2)
```

### Поддержка деструктуризации кортежей

```yaoxiang
# === Связывание с деструктуризацией кортежей ===

# Функция, принимающая аргумент-кортеж
process_coordinates: (coord: (Float, Float)) -> String = {
    return match coord {
        (0.0, 0.0) -> "origin"
        (x, 0.0) -> "on x-axis at ${x}"
        (0.0, y) -> "on y-axis at ${y}"
        (x, y) -> "point at (${x}, ${y})"
    }
}

Coord: Type = { x: Float, y: Float }

# Автоматическое связывание с деструктуризацией: Coord -> (Float, Float)
Coord.describe = process_coordinates[1]
# Использование: coord.describe() → process_coordinates((coord.x, coord.y))
```

### Связывание с множественными возвращаемыми значениями

```yaoxiang
# === Связывание с множественными возвращаемыми значениями ===

min_max: (list: List(Int)) -> (Int, Int) = {
    min = list.reduce(Int.MAX, (a, b) => if a < b then a else b)
    max = list.reduce(Int.MIN, (a, b) => if a > b then a else b)
    return (min, max)
}

List.range: (T:Type)->((self: List(T)) -> (T, T)) = min_max[1]
# Использование: (min_val, max_val) = list.range()
```

### Правила проверки типов

```rust
fn check_binding_type_compatibility(
    binding: &Binding,
    func: &Function
) -> Result<(), TypeError> {
    // 1. Если это автоматический поиск позиции (явно не указана), проверить, найдено ли соответствие
    if binding.positions.is_empty() {
        return Err(TypeError::NoMatchingParameter(
            binding.type_name.clone(),
            func.name.clone()
        ));
    }

    // 2. Проверить допустимость всех индексов позиций
    for pos in &binding.positions {
        if *pos >= func.params.len() {
            return Err(TypeError::InvalidBindingPosition(*pos));
        }
    }

    // 3. Проверить совместимость типов позиций связывания
    for pos in &binding.positions {
        let param_type = &func.params[*pos].type_;
        let binding_type = &binding.type_name;

        if !isAssignable(binding_type, param_type) {
            return Err(TypeError::IncompatibleTypes(
                binding_type, param_type
            ));
        }
    }

    // 4. Проверить соответствие аргументов вызова метода оставшимся аргументам
    Ok(())
}
```

### Поведение во время выполнения

| Сценарий | Синтаксис связывания | Вызов | Преобразуется в |
|------|---------|------|--------|
| Без связывания | `Point.distance = distance` | `Point.distance(p1, p2)` | `distance(p1, p2)` |
| Одна позиция | `Point.distance = distance[0]` | `p1.distance(p2)` | `distance(p1, p2)` |
| Одна позиция | `Point.distance = distance[1]` | `p1.distance(p2)` | `distance(p2, p1)` |
| Отрицательный индекс | `Point.test = func[-1]` | `p.test(a, b)` | `func(a, b, p)` |
| Множество позиций (каррирование) | `Point.scale = scale[0]` | `p.scale(2.0)` | `scale(p, 2.0)` |
| Заполнитель | `Type.method = func[1]` | `obj.method(arg)` | `func(arg, obj)` |

**Пояснения**:
- **Без связывания**: `Point.name = func` — это лишь псевдоним пространства имён, без синтаксиса вызова с `.`
- `[0]`: вызывающий привязывается к позиции 0 (первый аргумент)
- `[1]`: вызывающий привязывается к позиции 1 (второй аргумент)
- `[-1]`: вызывающий привязывается к последней позиции (отсчёт с конца)

## Компромиссы

### Преимущества

- **Явное связывание**: `[n]` — единственный механизм связывания, без `[n]` связывания нет, никакого неявного поведения
- **Точное управление**: можно привязать к любой позиции аргумента, высокая гибкость
- **Безопасность типов**: полная проверка типов на этапе компиляции, связывание только при совпадении типов
- **Простота синтаксиса**: синтаксис `[position]` интуитивно понятен и прост в изучении
- **Отсутствие ключевого слова `self`**: сохраняет простоту языка
- **Дружелюбность к каррированию**: естественная поддержка частичного применения и цепочечных вызовов
- **Дружелюбность к ООП**: автоматическое каррирование облегчает миграцию ООП-программистов

### Недостатки

- **Стоимость обучения**: требуется понимание концепции индексов позиций
- **Сложность компиляции**: разрешение связывания и проверка типов увеличивают сложность компилятора
- **Трудность отладки**: сообщения об ошибках должны чётко указывать на проблемы с позицией связывания

## Альтернативы

| Вариант | Описание | Почему не выбран |
|------|------|--------|
| Ключевое слово `self` | Ввести `self` в стиле Python/Rust | Нарушает философию дизайна YaoXiang об отсутствии неявного `self` |
| Связывание по именованным аргументам | Использовать именованные аргументы `func(a=obj)` | Требует изменения определения сигнатуры функции, увеличивает сложность |
| Система макросов | Реализовать связывание с помощью макросов | Большие накладные расходы во время выполнения, снижение безопасности типов |
| Перегрузка операторов | Ограничить `self` определёнными позициями | Синтаксис не единообразен, семантика запутанна |

## Стратегия реализации

### Поэтапное разделение

1. **Фаза 1: Базовое связывание** (v0.3)
   - Реализация синтаксиса однопозиционного связывания `[n]` (n начинается с 0, поддержка отрицательных значений)
   - Базовая проверка типов и генерация кода
   - Покрытие модульными тестами

2. **Фаза 2: Расширенные возможности** (v0.5)
   - Поддержка синтаксиса диапазонов `[n..m]`
   - Оптимизация вычисления позиций на этапе компиляции

### Зависимости

- Внешних зависимостей нет
- Прямой связи с RFC-001 (обработка ошибок) нет
- Может быть реализовано независимо

### Риски

- Обработка совместимости с существующим синтаксисом связывания
- Стратегия оптимизации производительности (раскрытие на этапе компиляции vs поиск во время выполнения)

## Приложения

### Приложение А: Журнал проектных решений

| Решение | Определение | Обоснование |
|------|------|--------|
| База индексации | С 0 | Соответствует индексации кортежей/списков аргументов |
| Отрицательные индексы | Поддерживаются | Гибкость, отсчёт с конца |
| Заполнитель | `_` | Краткий, общепринятый символ |
| Синтаксис диапазонов | Реализуется | Пакетное связывание, например `[0..2]` |
| Стиль синтаксиса | Инфиксный `Type.method = func[positions]` | Единообразие с RFC-010 |
| **Правило связывания** | **Только явное `[n]` привязывает, без него — не привязывает** | **Отсутствие неявного поведения, ортогональность определения функции и связывания** |
| **Пространство имён** | **`Type.name` — лишь принадлежность к пространству имён, не вызывает связывание** | **Разделение определения и связывания** |
| **Синтаксис функций** | **Имена аргументов в сигнатуре `name: (params) -> Return`** | **Единообразие с RFC-010** |

### Приложение Б: Глоссарий

| Термин | Определение |
|------|------|
| Позиция связывания | Индекс в списке аргументов функции |
| Объединённое связывание | Привязка типа к нескольким позициям аргументов |
| Частичное применение | Предоставление только части аргументов с возвратом функции, ожидающей завершения вызова |
| **Единый синтаксис** | **`name: (params) -> Return = body`, имена аргументов объявляются в сигнатуре** |
| **Функция в пространстве имён** | **Синтаксис `Type.name`, функция принадлежит пространству имён Type, без подразумеваемого связывания** |
| **Явное связывание** | **`Type.name = func[n]`, единственный механизм связывания методов** |
