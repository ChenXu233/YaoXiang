# RFC-004: Дизайн многопозиционного объединённого связывания для каррированных методов

## Аннотация

Настоящий RFC предлагает совершенно новый синтаксис **многопозиционного объединённого связывания**, который позволяет точно привязывать функцию к любой позиции аргумента типа, поддерживая как однопозиционное, так и многопозиционное объединённое связывание, что фундаментально решает проблему «кто является вызывающим» при каррированном связывании без необходимости вводить ключевое слово `self`.

## Мотивация

### Зачем нужна эта возможность?

В текущем дизайне языка при привязке независимой функции в качестве метода типа возникают следующие проблемы:

1. **Негибкость позиции вызывающего**: традиционное связывание может зафиксировать только `obj` из `obj.method(args)` как первый аргумент
2. **Сложность связывания нескольких аргументов**: когда метод должен принимать несколько аргументов одного типа, невозможно выразить это элегантно
3. **Семантическая неоднозначность каррирования**: при частичном применении сложно отличить, «к какой позиции выполняется привязка»

### Цель дизайна: унификация двух подходов к программированию

Данный дизайн направлен на **унификацию функционального и ООП подходов**:

```yaoxiang
# Функциональный подход: явная передача всех аргументов
distance(p1, p2)

# ООП-подход: неявный this
p1.distance(p2)

# Синтаксический сахар [positions] делает обе записи эквивалентными, по существу обе являются вызовами функций
Point.distance = distance[0]   # this привязывается к позиции 0
```

**Ключевые преимущества**:
- Внизу лежит функция, сверху — синтаксис методов
- Не вводится ключевое слово `self`, сохраняется простота языка
- Полная функциональность: вызов метода по существу является передачей аргументов
- `[0]`, `[1]`, `[-1]` гибко управляют позицией привязки this
- **Единый синтаксис**: определение функции использует формат `name: (params) -> Return = body`

### Текущие проблемы

```yaoxiang
# Проблемы текущего дизайна:
Point: Type = { x: Float, y: Float }
Vector: Type = { x: Float, y: Float, z: Float }

distance: (a: Point, b: Point) -> Float = { ... }
transform: (p: Point, v: Vector) -> Point = { ... }

# Возможна только привязка к первому аргументу
Point.distance = distance  # эквивалентно distance[0]
# p1.distance(p2) → distance(p1, p2) ✓

# Но если сигнатура transform: transform(Vector, Point)?
# Невозможно выразить семантику p1.transform(v1) → transform(v1, p1)
```

## Предложение

### Основной дизайн: явное указание позиции

**Ключевое правило: не указано `[n]` = нет связывания.** `Point.name = func` — это лишь псевдоним пространства имён, не вызывающий никакого неявного связывания. Чтобы синтаксис вызова с `.` вида `p.name(args)` работал, необходимо явное указание: `Point.name = func[n]`.

#### Однопозиционное связывание

```yaoxiang
# Явная привязка к первой позиции аргумента Point (индексация с 0)
Point.distance = distance[0]
p1.distance(p2)                     # → distance(p1, p2)

# Привязка ко второй позиции аргумента Point
Point.compare = distance[1]         # привязка ко второму аргументу Point
p1.compare(p2)                      # → distance(p2, p1)
```

**Без `[n]` = нет связывания**:

```yaoxiang
# Без [n] → чистый псевдоним пространства имён, синтаксис . недоступен
Point.distance = distance            # только Point.distance(p1, p2)
# p1.distance(p2)  ❌  связывание отсутствует

# Фабричные функции естественно допустимы, без специальной обработки
create_point: () -> Point = { ... }
Point.create = create_point          # Point.create()   ✅
```

- Безопасность типов: связывание выполняется только при совпадении типов, что предотвращает ошибки
- Гибкое управление: `[n]` обеспечивает точный контроль позиции связывания

#### Каррированное связывание

Когда количество аргументов функции превышает количество позиций связывания, автоматически генерируется каррированная функция. **Связывание всегда является явной операцией.**

```yaoxiang
Point: Type = { x: Float, y: Float }

# Базовая функция
scale: (p: Point, factor: Float) -> Point = {
    return Point(p.x * factor, p.y * factor)
}

# Явная привязка к позиции 0 → каррирование: оставшийся аргумент factor предоставляется вызывающим
Point.scale = scale[0]

# Вызов
p1 = Point(2.0, 3.0)
scaled = p1.scale(2.0)       # → scale(p1, 2.0)

# Цепочечный вызов ещё элегантнее
result = Point(2.0, 3.0).scale(2.0)  # → Point(4.0, 6.0)
```

### Синтаксис позиционного индексированного связывания

Вводится синтаксис `[position]` для точного управления связью между аргументами функции и типом:

```yaoxiang
# Формат синтаксиса: Type.method = function[positions]

# === Базовое связывание ===

# Однопозиционное связывание
Point.distance = distance[1]           # привязка к 1-му аргументу (индексация с 0)
# Использование: p1.distance(p2) → distance(p2, p1)

# Многопозиционное объединённое связывание (деконструкция кортежа)
Point.transform = transform[1, 2]      # привязка к 1,2 аргументам
# Использование: p1.transform(v1) → transform(v1, p1)
# Исходная сигнатура: transform(Point, Vector) → Point
# После связывания: Point.transform(Vector) → Point
```

### Подробное определение синтаксиса

```
ОбъявлениеСвязывания ::= Тип '.' Идентификатор '=' ИмяФункции '[' СписокПозиций ']'

СписокПозиций ::= Позиция (',' Позиция)*
Позиция        ::= Целое                    # заполнитель
                | '_'                      # пропуск этой позиции (заполнитель)
                | Целое '..' Целое         # диапазон позиций (будущее расширение)

ИмяФункции     ::= Идентификатор
Тип            ::= Идентификатор (АргументОбобщений)?
```

### Встроенное связывание

Связывание может быть записано непосредственно в теле определения типа, без отдельного оператора связывания:

```yaoxiang
# Способ 1: непосредственная привязка в теле определения типа
Point: Type = {
    x: Float = 0,
    y: Float = 0,
    distance = distance[0]           # привязка к позиции 0
}

# Способ 2: анонимная функция + позиционное связывание
Point: Type = {
    x: Float = 0,
    y: Float = 0,
    distance: ((a: Point, b: Point) -> Float)[0] = ((a, b) => {
        dx = a.x - b.x
        dy = a.y - b.y
        return (dx * dx + dy * dy).sqrt()
    })
}
# Синтаксис: ((params) => body)[position]
```

**Семантика каррирования**:
- При связывании `distance = distance[0]` исходная сигнатура функции `(a: Point, b: Point) -> Float`
- Сгенерированная сигнатура метода: `b: Point -> Float` (позиция 0 заполняется вызывающим)

### Примеры использования

```yaoxiang
# === Полный пример ===

Point: Type = { x: Float, y: Float }
Vector: Type = { x: Float, y: Float, z: Float }

# 1. Базовое вычисление расстояния
distance: (a: Point, b: Point) -> Float = {
    dx = a.x - b.x
    dy = a.y - b.y
    return (dx * dx + dy * dy).sqrt()
}

# Связывание: Point.distance = distance[1]
# Вызов: p1.distance(p2) → distance(p2, p1)
# Но мы хотим p1.distance(p2) → distance(p1, p2), поэтому:
Point.distance = distance[0]

# 2. Операция трансформации (многопозиционное связывание)
transform: (p: Point, v: Vector) -> Point = {
    return Point(p.x + v.x, p.y + v.y)
}

# Связывание Point.transform = transform[1]
# Вызов: p.transform(v) → transform(v, p) ❌
# Связывание Point.transform = transform[0]
# Вызов: p.transform(v) → transform(p, v) ✓

# 3. Сложная функция с множеством аргументов
multiply: (a: Point, s: Float) -> Point = {
    return Point(a.x * s, a.y * s)
}

# Привязка только к 1-му аргументу (тип Point), 3-й аргумент сохраняется
Point.scale = multiply[0, _]
# Вызов: p.scale(2.0) → multiply(p, 2.0)

# 4. Межтиповое связывание
Circle: Type = { center: Point, radius: Float }

distance: (a: Circle, b: Circle) -> Float = {
    return a.center.distance(b.center) - a.radius - b.radius
}

# Привязка метода distance к типу Circle
Circle.distance = distance[0, 1]
# Вызов: c1.distance(c2) → distance(c1, c2)
```

### Поддержка деконструкции кортежей

```yaoxiang
# === Связывание с деконструкцией кортежа ===

# Функция принимает кортежный аргумент
process_coordinates: (coord: (Float, Float)) -> String = {
    return match coord {
        (0.0, 0.0) -> "origin"
        (x, 0.0) -> "on x-axis at ${x}"
        (0.0, y) -> "on y-axis at ${y}"
        (x, y) -> "point at (${x}, ${y})"
    }
}

Coord: Type = { x: Float, y: Float }

# Автоматическое связывание с деконструкцией: Coord -> (Float, Float)
Coord.describe = process_coordinates[1]
# Использование: coord.describe() → process_coordinates((coord.x, coord.y))
```

### Связывание с множественными возвращаемыми значениями

```yaoxiang
# === Связывание с множественными возвращаемыми значениями ===

min_max: (list: List(Int)) -> (Int, Int) = {
    min = list.reduce(Int.MAX, (a, b) => if a < b then a else b)
    max = list.reduce(Int.MIN, (a, b) => if a > b then a else b)
    return (min, max)
}

List.range: (T:Type)->((self: List(T)) -> (T, T)) = min_max[1]
# Использование: (min_val, max_val) = list.range()
```

### Правила проверки типов

```rust
fn check_binding_type_compatibility(
    binding: &Binding,
    func: &Function
) -> Result<(), TypeError> {
    // 1. Если выполняется автоматический поиск позиции (не указана явно), проверяем, найдено ли совпадение
    if binding.positions.is_empty() {
        return Err(TypeError::NoMatchingParameter(
            binding.type_name.clone(),
            func.name.clone()
        ));
    }

    // 2. Проверяем валидность всех позиционных индексов
    for pos in &binding.positions {
        if *pos >= func.params.len() {
            return Err(TypeError::InvalidBindingPosition(*pos));
        }
    }

    // 3. Проверяем совместимость типов в позициях связывания
    for pos in &binding.positions {
        let param_type = &func.params[*pos].type_;
        let binding_type = &binding.type_name;

        if !isAssignable(binding_type, param_type) {
            return Err(TypeError::IncompatibleTypes(
                binding_type, param_type
            ));
        }
    }

    // 4. Проверяем соответствие аргументов метода оставшимся аргументам
    Ok(())
}
```

### Поведение во время выполнения

| Сценарий | Синтаксис связывания | Вызов | Преобразуется в |
|------|---------|------|--------|
| Без связывания | `Point.distance = distance` | `Point.distance(p1, p2)` | `distance(p1, p2)` |
| Однопозиционное | `Point.distance = distance[0]` | `p1.distance(p2)` | `distance(p1, p2)` |
| Однопозиционное | `Point.distance = distance[1]` | `p1.distance(p2)` | `distance(p2, p1)` |
| Отрицательный индекс | `Point.test = func[-1]` | `p.test(a, b)` | `func(a, b, p)` |
| Многопозиционное (каррирование) | `Point.scale = scale[0]` | `p.scale(2.0)` | `scale(p, 2.0)` |
| Заполнитель | `Type.method = func[1]` | `obj.method(arg)` | `func(arg, obj)` |

**Пояснения**:
- **Без связывания**: `Point.name = func` — это просто псевдоним пространства имён, синтаксис `.` недоступен
- `[0]`: вызывающий привязывается к позиции 0 (первый аргумент)
- `[1]`: вызывающий привязывается к позиции 1 (второй аргумент)
- `[-1]`: вызывающий привязывается к последней позиции (счёт с конца)

## Компромиссы

### Преимущества

- **Явное связывание**: `[n]` — единственный механизм связывания, без указания связывание не выполняется, отсутствие неявного поведения
- **Точный контроль**: возможность привязки к любой позиции аргумента, высокая гибкость
- **Безопасность типов**: полная проверка типов во время компиляции, связывание выполняется только при совпадении типов
- **Простой синтаксис**: `[position]` интуитивно понятен и прост
- **Отсутствие ключевого слова `self`**: сохраняет простоту языка
- **Каррирование-дружественность**: естественная поддержка частичного применения и цепочечных вызовов
- **ООП-дружественность**: автоматическое каррирование позволяет ООП-программистам безболезненно мигрировать

### Недостатки

- **Стоимость обучения**: требуется понимание концепции позиционного индексирования
- **Сложность компиляции**: разрешение связывания и проверка типов увеличивают сложность компилятора
- **Сложность отладки**: сообщения об ошибках должны чётко указывать на проблемы с позицией связывания

## Альтернативы

| Подход | Описание | Почему не выбран |
|------|------|-----------|
| Ключевое слово `self` | Введение `self` в стиле Python/Rust | Нарушает философию дизайна YaoXiang об отсутствии неявного `self` |
| Связывание по именованным аргументам | Использование именованных аргументов `func(a=obj)` | Требует изменения определения сигнатуры функции, увеличивает сложность |
| Система макросов | Реализация связывания с помощью макросов | Высокие накладные расходы во время выполнения, снижение безопасности типов |
| Перегрузка операторов | Ограничение `self` в определённых позициях | Синтаксис неоднороден, семантическая путаница |

## Стратегия реализации

### Разделение на этапы

1. **Этап 1: Базовое связывание** (v0.3)
   - Реализация синтаксиса однопозиционного связывания `[n]` (n начинается с 0, поддержка отрицательных чисел)
   - Базовая проверка типов и генерация кода
   - Покрытие модульными тестами

2. **Этап 2: Продвинутые возможности** (v0.5)
   - Поддержка синтаксиса диапазона `[n..m]`
   - Оптимизация вычисления позиций во время компиляции

### Зависимости

- Внешние зависимости отсутствуют
- Прямой связи с RFC-001 (обработка ошибок) нет
- Возможна независимая реализация

### Риски

- Обработка совместимости с существующим синтаксисом связывания
- Стратегия оптимизации производительности (развёртывание во время компиляции vs. поиск во время выполнения)

## Приложения

### Приложение A: Запись проектных решений

| Решение | Решено | Причина |
|------|------|------|
| База индекса | Начинается с 0 | Соответствует индексации кортежей/списков аргументов |
| Отрицательный индекс | Поддерживается | Гибкость, счёт с конца |
| Заполнитель | `_` | Лаконичный, универсальный символ |
| Синтаксис диапазона | Реализуется | Пакетное связывание, например `[0..2]` |
| Стиль синтаксиса | Инфиксный `Type.method = func[positions]` | Унифицировано с RFC-010 |
| **Правило связывания** | **Связывание только при явном `[n]`, без указания связывание не выполняется** | **Отсутствие неявного поведения, ортогональность определения функции и связывания** |
| **Пространство имён** | **`Type.name` — лишь принадлежность к пространству имён, не запускает связывание** | **Разделение определения и связывания** |
| **Синтаксис функции** | **Имя аргумента объявляется в сигнатуре `name: (params) -> Return`** | **Унифицировано с RFC-010** |

### Приложение B: Глоссарий

| Термин | Определение |
|------|------|
| Позиция связывания | Индекс в списке аргументов функции |
| Объединённое связывание | Привязка типа к нескольким позициям аргументов |
| Частичное применение | Предоставление только части аргументов, возврат функции, ожидающей завершения вызова |
| **Унифицированный синтаксис** | **`name: (params) -> Return = body`, имя аргумента объявляется в сигнатуре** |
| **Функция пространства имён** | **Синтаксис `Type.name`, функция принадлежит пространству имён Type, без неявного связывания** |
| **Явное связывание** | **`Type.name = func[n]`, единственный механизм связывания методов** |
