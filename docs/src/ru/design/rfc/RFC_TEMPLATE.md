## Жизненный цикл и судьба

RFC имеет следующие статусы переходов:

```
┌─────────────┐
│  Черновик   │  ← Автор создаёт
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ На рассмотрении │  ← Обсуждение сообществом
└──────┬──────┘
       │
       ├──────────────────┐
       ▼                  ▼
┌─────────────┐    ┌─────────────┐
│  Принято    │    │ Отклонено   │
└──────┬──────┘    └──────┬──────┘
       │                  │
       ▼                  ▼
┌─────────────┐    ┌─────────────┐
│   accepted/ │    │     rfc/    │
│(официальный│    │(остаётся    │
│  дизайн)   │    │ на месте)   │
└─────────────┘    └─────────────┘
```

### Описание статусов

| Статус | Расположение | Описание |
|------|------|------|
| **Черновик** | `docs/design/rfc/` | Авторский черновик, ожидает подачи на рассмотрение |
| **На рассмотрении** | `docs/design/rfc/` | Открыто для обсуждения и обратной связи сообщества |
| **Принято** | `docs/design/accepted/` | Становится официальным документом дизайна, переходит в фазу реализации |
| **Отклонено** | `docs/design/rfc/` | Остаётся в каталоге RFC, статус обновляется |

### Действия после принятия

1. Переместить RFC в каталог `docs/design/accepted/`
2. Обновить имя файла на описательное (например, `enhanced-pattern-matching.md`)
3. Обновить статус на "Официальный"
4. Обновить статус на "Принято", добавить дату принятия

### Действия после отклонения

1. Оставить в каталоге `docs/design/rfc/`
2. Добавить в начало файла причину отклонения и дату
3. Обновить статус на "Отклонено"

### Действия после принятия решений по обсуждениям

Когда консенсус по открытому вопросу достигнут:

1. **Обновить приложение A**: Заполнить раздел «Решение» в теме обсуждения
2. **Обновить основную часть**: Синхронизировать решение в основной части документа
3. **Записать решение**: Добавить в «Приложение B: Журнал дизайнерских решений»
4. **Отметить вопрос**: В списке «Открытые вопросы» поставить галочку `[x]`

---

> **Примечание**: Номер RFC используется только на этапе обсуждения. После принятия номер удаляется, вместо него используется описательное имя файла.
