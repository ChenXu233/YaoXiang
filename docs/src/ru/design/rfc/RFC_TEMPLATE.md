## Жизненный цикл и судьба

RFC имеет следующие статусы:

```
┌─────────────┐
│   Черновик  │  ← Автор создаёт
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ На рассмотрении │  ← Обсуждение сообщества
└──────┬──────┘
       │
       ├──────────────────┐
       ▼                  ▼
┌─────────────┐    ┌─────────────┐
│   Принят    │    │  Отклонён   │
└──────┬──────┘    └──────┬──────┘
       │                  │
       ▼                  ▼
┌─────────────┐    ┌─────────────┐
│   accepted/ │    │     rfc/    │
│(официальный│    │(остаётся на │
│   дизайн)   │    │   месте)    │
└─────────────┘    └─────────────┘
```

### Описание статусов

| Статус | Расположение | Описание |
|--------|--------------|----------|
| **Черновик** | `docs/design/rfc/` | Авторский черновик, ожидает отправки на рассмотрение |
| **На рассмотрении** | `docs/design/rfc/` | Открыто для обсуждения и обратной связи сообщества |
| **Принят** | `docs/design/accepted/` | Становится официальным дизайн-документом, переходит в фазу реализации |
| **Отклонён** | `docs/design/rfc/` | Остаётся в директории RFC, статус обновляется |

### Действия после принятия

1. Переместить RFC в директорию `docs/design/accepted/`
2. Обновить имя файла на описательное (например, `enhanced-pattern-matching.md`)
3. Обновить статус на "Официальный"
4. Обновить статус на "Принят", добавить дату принятия

### Действия после отклонения

1. Оставить в директории `docs/design/rfc/`
2. Добавить в начало файла причину отклонения и дату
3. Обновить статус на "Отклонён"

### Действия после определения обсуждения

Когда по какому-либо открытому вопросу достигнут консенсус:

1. **Обновить приложение A**: Заполнить «Решение» в теме обсуждения
2. **Обновить основную часть**: Синхронизировать решение с основным текстом документа
3. **Записать решение**: Добавить в «Приложение B: Записи дизайнерских решений»
4. **Отметить вопрос**: Поставить галочку `[x]` в списке «Открытые вопросы»

---

> **Примечание**: Номер RFC используется только на этапе обсуждения. После принятия номер удаляется, вместо него используется описательное имя файла.
