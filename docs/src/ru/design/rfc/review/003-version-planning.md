# RFC-003：Планирование версий

> **Статус**: На длительной проверке
> **Автор**: Чэнь Сюй
> **Дата создания**: 2025-01-05
> **Последнее обновление**: 2025-01-06

## Аннотация

План выпуска версий YaoXiang — дорожная карта от v0.1 до v1.0.

**Основные цели**:
1. **Компиляция в байт-код**: Поддержка REPL и инкрементальной компиляции
2. **Самокомпиляция (Bootstrap)**: Написание компилятора YaoXiang на YaoXiang
3. **AOT-компиляция**: Компиляция байт-кода в нативный машинный код

## 一、Мотивация

### Зачем нужно планирование версий?

1. **Управление проектом**: Разделение целей на выполнимые этапы
2. **Ожидания пользователей**: Информирование о стадиях развития языка
3. **Распределение ресурсов**: Определение приоритетов каждого этапа
4. **Контроль рисков**: Своевременное выявление проблем и корректировка направления

### Ключевые проектные решения

- **Приоритет байт-кода**: Сначала реализуем интерпретацию, затем AOT
- **Постепенная поставка**: Каждая версия имеет работающий функционал
- **Обратная совместимость**: До v1.0 API может меняться, но с предварительным уведомлением
- **Верификация самокомпиляцией**: Доказательство выразительности языка через самокомпиляцию
- **Многоуровневая оптимизация производительности**: Сначала работает, потом оптимизируем

## 四、Трёхуровневая архитектура компиляции

| Уровень | Версия | Входные данные | Выходные данные | Описание |
|---------|--------|----------------|-----------------|----------|
| L1: Байт-код | v0.1+ | Исходный код (.yx) | Байт-код (.yxb) | Интерпретация VM |
| L2: Самокомпиляция | v0.9+ | Исходный код YaoXiang | Байт-код | Компилирует сам себя |
| L3: AOT | v1.0+ | Исходный код/байт-код | Машинный код | Нативная производительность |

**Обоснование приоритета байт-кода**:
1. **Поддержка REPL**: Немедленная компиляция вводимого кода, интерактивная разработка
2. **Инкрементальная компиляция**: Изменение одной функции требует перекомпиляции только этой части
3. **Кроссплатформенность**: Файлы .yxb работают на разных платформах, требуется только VM для соответствующей платформы

## 五、Стратегия зависимостей

**Краткосрочная перспектива**: Использование Rust-библиотек через crates.io (Cargo-паразитирование)

**Текущие зависимости**:
- Параллелизм: parking_lot, crossbeam, rayon
- Структуры данных: indexmap, hashbrown, smallvec
- Сетевые: tokio
- Сериализация: serde, ron

**Долгосрочная перспектива**: Собственная стандартная библиотека и менеджер пакетов
