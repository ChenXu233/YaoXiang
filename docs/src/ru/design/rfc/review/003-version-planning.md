# RFC-003：版本规划

> **Статус**: Долгосрочная проверка
> **Автор**: Чэнь Сюй
> **Дата создания**: 2025-01-05
> **Последнее обновление**: 2025-01-06

## Аннотация

План выпуска версий YaoXiang, дорожная карта от v0.1 до v1.0.

**Основные цели**:
1. **Компиляция в байт-код**: Поддержка REPL и инкрементальной компиляции
2. **Бутстрэппинг (Bootstrap)**: Написание компилятора YaoXiang на YaoXiang
3. **AOT-компиляция**: Компиляция байт-кода в машинный код

## I. Мотивация

### Зачем нужна версионная дорожная карта?

1. **Управление проектом**: Разделение целей на выполнимые этапы
2. **Ожидания пользователей**: Позволить пользователям понять стадии развития языка
3. **Распределение ресурсов**: Определение приоритетов на каждом этапе
4. **Контроль рисков**: Своевременное выявление проблем и корректировка курса

### Ключевые проектные решения

- **Приоритет байт-кода**: Сначала реализовать интерпретацию, затем AOT
- **Поэтапная поставка**: Каждая версия имеет рабочие функции
- **Обратная совместимость**: API до v1.0 может изменяться, но с предварительным уведомлением
- **Верификация через бутстрэппинг**: Доказательство выразительности языка через самокомпиляцию
- **Слои производительности**: Сначала запустить, потом оптимизировать

## IV. Трёхуровневый дизайн стратегии компиляции

| Уровень | Версия | Вход | Выход | Описание |
|---------|--------|------|-------|----------|
| L1: Байт-код | v0.1+ | Исходный код (.yx) | Байт-код (.yxb) | Интерпретация VM |
| L2: Бутстрэппинг | v0.9+ | Исходный код YaoXiang | Байт-код | Самокомпиляция |
| L3: AOT | v1.0+ | Исходный код/Байт-код | Машинный код | Нативная производительность |

**Обоснование приоритета байт-кода**:
1. **Поддержка REPL**: Незамедлительная компиляция вводимого кода, интерактивная разработка
2. **Инкрементальная компиляция**: Изменение одной функции требует перекомпиляции только этой части
3. **Кроссплатформенность**: Файлы .yxb работают на разных платформах, нужен только VM для конкретной платформы

## V. Стратегия зависимостей

**Краткосрочная перспектива**: Использование библиотек Rust, повторное использование crates.io (Cargo как паразит)

**Текущие зависимости**:
- Параллелизм: parking_lot, crossbeam, rayon
- Структуры данных: indexmap, hashbrown, smallvec
- Сети: tokio
- Сериализация: serde, ron

**Долгосрочная перспектива**: Собственная стандартная библиотека и менеджер пакетов
