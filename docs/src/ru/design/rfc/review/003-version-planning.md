# RFC-003：版本规划

> **状态**: Долгосрочная проверка
> **Автор**: Чэнь Сюй
> **Дата создания**: 2025-01-05
> **Последнее обновление**: 2025-01-06

## Аннотация

План выпуска версий YaoXiang, дорожная карта от v0.1 до v1.0.

**Основные цели**:
1. **Компиляция в байт-код**: Поддержка REPL и инкрементальной компиляции
2. **Бутстрэппинг (Bootstrap)**: Написание компилятора YaoXiang на YaoXiang
3. **AOT-компиляция**: Компиляция байт-кода в нативный машинный код

## I. Мотивация

### Зачем нужно планирование версий?

1. **Управление проектом**: Разделение целей на выполнимые этапы
2. **Ожидания пользователей**: Позволить пользователям понять стадии развития языка
3. **Распределение ресурсов**: Определение приоритетов на каждом этапе
4. **Контроль рисков**: Своевременное выявление проблем и корректировка направления

### Ключевые проектные решения

- **Приоритет байт-кода**: Сначала реализовать интерпретацию, затем AOT
- **Инкрементальная поставка**: Каждая версия имеет готовые функции
- **Обратная совместимость**: До v1.0 API может измениться, но с предварительным уведомлением
- **Верификация через бутстрэппинг**: Доказательство выразительности языка через бутстрэппинг
- **Слои производительности**: Сначала запустить, потом оптимизировать

## IV. Трёхуровневая архитектура компиляции

| Уровень | Версия | Входные данные | Выходные данные | Описание |
|---------|--------|----------------|-----------------|----------|
| L1: Байт-код | v0.1+ | Исходный код (.yx) | Байт-код (.yxb) | Интерпретация VM |
| L2: Бутстрэппинг | v0.9+ | Исходный код YaoXiang | Байт-код | Компиляция самого себя |
| L3: AOT | v1.0+ | Исходный код/байт-код | Машинный код | Нативная производительность |

**Обоснование приоритета байт-кода**:
1. **Поддержка REPL**: Немедленная компиляция вводимого кода, интерактивная разработка
2. **Инкрементальная компиляция**: При изменении одной функции достаточно перекомпилировать только её
3. **Кроссплатформенность**: Файлы .yxb работают на разных платформах, требуется только VM для соответствующей платформы

## V. Стратегия зависимостей

**Краткосрочная перспектива**: Вызов библиотек Rust, повторное использование crates.io (Cargo symbiosis)

**Текущие зависимости**:
- Concurrency: parking_lot, crossbeam, rayon
- Структуры данных: indexmap, hashbrown, smallvec
- Сетевое взаимодействие: tokio
- Сериализация: serde, ron

**Долгосрочная перспектива**: Собственная стандартная библиотека и менеджер пакетов
