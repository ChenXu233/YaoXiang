# RFC-пример: Улучшенный синтаксис сопоставления с образцом

> **Примечание**: Это шаблонный пример RFC, демонстрирующий написание полного RFC-предложения.
> Пожалуйста, используйте этот шаблон для написания собственного RFC.

## Аннотация

Добавление более мощных возможностей сопоставления с образцом в YaoXiang, включая вложенные образцы, охранные выражения и привязку переменных через `let`.

## Мотивация

### Почему эта возможность необходима?

Текущие выражения `match` имеют ограниченную функциональность и не могут обрабатывать следующие распространённые сценарии:

```yaoxiang
# Невозможность деструктуризации вложенных структур
Person: Type = { name: String, address: Address }
Address: Type = { city: String, zip: Int }
match person {
    Person(name: "Alice", address: Address(city: "Beijing", _)) => "Alice from Beijing"  # ❌ Не поддерживается
}

# Невозможность привязки переменных в образце
match result {
    ok(value) => print(value)          # ❌ Требуется явная деструктуризация
}
```

### Текущие проблемы

1. Отсутствует поддержка деструктуризации вложенных образцов
2. Невозможно использовать охранные выражения в образцах
3. Оператор `let` не поддерживает сопоставление с образцом

## Предложение

### Основной дизайн

Расширение синтаксиса выражений `match` для поддержки:

1. **Деструктуризация вложенных образцов**: деструктуризация структур любой глубины вложенности
2. **Охранные выражения**: добавление условий `if` после образца
3. **Привязка переменных в образцах**: прямая привязка переменных из образца

### Примеры

```yaoxiang
# Вложенная деструктуризация
Person: Type = { name: String, address: Address }
Address: Type = { city: String, zip: Int }

match person {
    Person(name: "Alice", address: Address(city: "Beijing", _)) => "Alice from Beijing"
    Person(name: n, address: Address(city: c, _)) => n + " from " + c
}

# Охранные выражения
match n {
    n if n > 0 && n < 10 => "1-9"
    n if n >= 10 => "10+"
    _ => "unknown"
}

# Привязка в образце
match result {
    ok(value) => print(value)          # value уже привязана
    err(e) => log_error(e)
}

# Вложенность + привязка
match data {
    User(name: first, profile: Profile(age: a)) if a >= 18 => first + " is adult"
}
```

### Сопоставление с образцом в операторе `let`

```yaoxiang
# Новый синтаксис
let Point(x: 0, y: _) = point  # Привязка только если x == 0
let Ok(value) = result         # Деструктуризация Result

# Множественная привязка
let (a, b, c) = tuple          # Деструктуризация кортежа
```

## Детальный дизайн

### Синтаксические изменения

```
MatchExpr   ::= 'match' Expr '{' MatchArm+ '}'
MatchArm    ::= Pattern ('|' Pattern)* ('if' Expr)? '=>' Expr ','
Pattern     ::= LiteralPattern
              | IdentifierPattern
              | StructPattern
              | TuplePattern
              | OrPattern
              | RestPattern

LiteralPattern ::= '_' | Literal
IdentifierPattern ::= Identifier (':' Pattern)?
StructPattern ::= Identifier '(' FieldPattern (',' FieldPattern)* ','? ')'
FieldPattern  ::= Identifier ':' Pattern | Identifier
TuplePattern  ::= '(' Pattern (',' Pattern)* ','? ')'
OrPattern     ::= Pattern '|' Pattern
RestPattern   ::= '...'
```

### Влияние на систему типов

- Необходимо расширить проверку типов при сопоставлении с образцом
- Переменные образца получают корректный тип при успешном сопоставлении

### Изменения в компиляторе

| Компонент | Изменения | описание |
|------|------|------|
| Лексер | Новые токены для образцов | Добавление новых токенов |
| Парсер | Логика разбора образцов | Новая логика разбора |
| Проверка типов | Вывод типов и привязка образцов | Типовая инференция |
| Генерация кода | Генерация кода для сопоставления | Генерация кода |

### Обратная совместимость

- ✅ Полная обратная совместимость
- Добавлен только новый синтаксис, старый синтаксис `match` не изменён

## Компромиссы

### Преимущества

- Более выразительный синтаксис, более лаконичный код
- Согласованность с主流ными языками сопоставления с образцом (Rust, Scala, Elixir)
- Снижение количества ошибок времени выполнения, раннее обнаружение несовпадений

### Недостатки

- Увеличение сложности реализации компилятора
- Незначительное повышение кривой обучения

## Альтернативные решения

| Решение | Почему не выбрано | описание |
|------|--------------|------|
| Только деструктуризация верхнего уровня | Невозможность обработки распространённых вложенных сценариев | Отсутствие гибкости |
| Использование функционального стиля | Естественное смешивание с императивным кодом затруднено | Стилистическое несоответствие |
| Отложить до v2.0 | У пользователей уже есть острая потребность | Приоритет пользователей |

## Стратегия реализации

### Фазы

1. **Фаза 1 (v0.6)**: Вложенная деструктуризация и охранные выражения
2. **Фаза 2 (v0.7)**: Привязка переменных в образцах
3. **Фаза 3 (v0.8)**: Сопоставление с образцом в `let`

### Зависимости

- Внешних зависимостей нет
- Требуется предварительное завершение базовой системы типов

### Риски

- Сложность компиляции образцов может вызвать проблемы с производительностью
- Слишком глубокая вложенность может привести к переполнению стека
