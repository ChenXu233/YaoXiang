## Жизненный цикл RFC

```
┌─────────────┐
│   Черновик   │  ← Автор создаёт
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ На рассмотрении │  ← Открытое обсуждение сообщества
└──────┬──────┘
       │
       ├──────────────────┐
       ▼                  ▼
┌─────────────┐    ┌─────────────┐
│  Принято    │    │ Отклонено  │
└──────┬──────┘    └──────┬──────┘
       │                  │
       ▼                  ▼
┌─────────────┐    ┌─────────────┐
│  accepted/  │    │ rejected/   │
│(официальный│    │ (отклонён)  │
│   дизайн)   │    │             │
└─────────────┘    └─────────────┘
```

### Описание статусов

| Статус | Расположение | Описание |
|--------|--------------|----------|
| **Черновик** | `docs/reference/rfc/draft/` | Черновик автора, ожидает подачи на рассмотрение |
| **На рассмотрении** | `docs/reference/rfc/review/` | Открытое обсуждение и обратная связь сообщества |
| **Принято** | `docs/reference/rfc/accepted/` | Становится официальным документом дизайна, переходит в фазу реализации |
| **Отклонено** | `docs/reference/rfc/rejected/` | Документы RFC, которые были отклонены |

## Подача RFC

1. Прочитайте [RFC_TEMPLATE.md](RFC_TEMPLATE.md) для ознакомления с требованиями к формату
2. Обратитесь к [EXAMPLE_full_feature_proposal.md](EXAMPLE_full_feature_proposal.md) для изучения стиля написания
3. Создайте новый файл с именем `порядковый-номер-описательное-название.md`
4. Поместите файл в каталог `docs/reference/rfc/draft/`
5. Обновите этот индексный файл, добавив новую запись RFC
6. Отправьте PR для прохождения процесса рассмотрения
