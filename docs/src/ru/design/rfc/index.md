## Жизненный цикл RFC

```
Черновик → На рассмотрении → Принят → Устарел (заменён)
                                  ↓
                            Отклонён (не прошёл)
```

### Описание статусов

| Статус | Расположение | Описание |
|------|------|------|
| **Черновик** | `rfc/draft/` | Авторский черновик, ожидает подачи на рассмотрение |
| **На рассмотрении** | `rfc/review/` | Открыто для обсуждения и обратной связи от сообщества |
| **Принят** | `rfc/accepted/` | Становится официальным проектным документом, переходит в фазу реализации |
| **Устарел** | `rfc/deprecated/` | Был принят, но заменён новым проектом |
| **Отклонён** | `rfc/rejected/` | Отклонённый документ RFC |

## Подача RFC

1. Прочтите [RFC_TEMPLATE.md](RFC_TEMPLATE.md), чтобы узнать требования к формату
2. Изучите [EXAMPLE_full_feature_proposal.md](EXAMPLE_full_feature_proposal.md) для ознакомления с примером
3. Создайте новый файл с именем `номер-описательное-название.md`
4. Поместите файл в каталог `docs/reference/rfc/draft/`
5. Обновите данный индексный файл, добавив новую запись RFC
6. Отправьте PR для входа в процесс рассмотрения
