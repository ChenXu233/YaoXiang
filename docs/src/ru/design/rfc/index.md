## Жизненный цикл RFC

```
Проект → На рассмотрении → Принято → Устарело (заменено)
                            ↓
                       Отклонено (не прошло)
```

### Описание статусов

| Статус | Расположение | Описание |
|------|------|------|
| **Проект** | `rfc/draft/` | Черновик автора, ожидает отправки на рассмотрение |
| **На рассмотрении** | `rfc/review/` | Открыто для обсуждения сообществом и обратной связи |
| **Принято** | `rfc/accepted/` | Становится официальным проектным документом, переходит в стадию реализации |
| **Устарело** | `rfc/deprecated/` | Было принято, заменено новым дизайном |
| **Отклонено** | `rfc/rejected/` | Документ отклонённого RFC |

## Подача RFC

1. Прочитайте [RFC_TEMPLATE.md](RFC_TEMPLATE.md), чтобы узнать требования к формату
2. Обратитесь к [EXAMPLE_full_feature_proposal.md](EXAMPLE_full_feature_proposal.md), чтобы изучить формат написания
3. Создайте новый файл с именем `номер-описательное-название.md`
4. Поместите файл в директорию `docs/reference/rfc/draft/`
5. Обновите данный индексный файл, добавив новую запись RFC
6. Отправьте PR для входа в процесс рассмотрения
