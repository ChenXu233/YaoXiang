## Жизненный цикл RFC

```
Черновик → На рассмотрении → Принят → Устарел (заменён)
                              ↓
                        Отклонён (не прошёл)
```

### Описание статусов

| Статус | Расположение | Описание |
|------|------|------|
| **Черновик** | `rfc/draft/` | Черновик автора, ожидает отправки на рассмотрение |
| **На рассмотрении** | `rfc/review/` | Открыт для обсуждения сообществом и обратной связи |
| **Принят** | `rfc/accepted/` | Стал официальным проектным документом, переходит в стадию реализации |
| **Устарел** | `rfc/deprecated/` | Ранее был принят, но заменён новым дизайном |
| **Отклонён** | `rfc/rejected/` | Документ RFC, который был отклонён |

## Отправка RFC

1. Прочитайте [RFC_TEMPLATE.md](RFC_TEMPLATE.md), чтобы узнать требования к формату
2. Изучите [EXAMPLE_full_feature_proposal.md](EXAMPLE_full_feature_proposal.md) в качестве примера оформления
3. Создайте новый файл с именем `номер-описательное-название.md`
4. Поместите файл в директорию `docs/reference/rfc/draft/`
5. Обновите данный индексный файл, добавив новую запись RFC
6. Отправьте PR для запуска процесса рассмотрения
