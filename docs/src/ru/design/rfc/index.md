## Жизненный цикл RFC

```
Черновик → На рассмотрении → Принят → Устарел (заменён)
                            ↓
                      Отклонён (не прошёл)
```

### Описание статусов

| Статус | Расположение | Описание |
|--------|--------------|----------|
| **Черновик** | `rfc/draft/` | Авторский черновик, ожидает подачи на рассмотрение |
| **На рассмотрении** | `rfc/review/` | Открыто для обсуждения и обратной связи сообщества |
| **Принят** | `rfc/accepted/` | Становится официальным документом дизайна, переходит в стадию реализации |
| **Устарел** | `rfc/deprecated/` | Был принят, заменён новым дизайном |
| **Отклонён** | `rfc/rejected/` | Документы отклонённых RFC |

## Подача RFC

1. Прочитайте [RFC_TEMPLATE.md](RFC_TEMPLATE.md) для ознакомления с требованиями к формату
2. Изучите [EXAMPLE_full_feature_proposal.md](EXAMPLE_full_feature_proposal.md) для изучения стиля написания
3. Создайте новый файл с именем `порядковый-номер-описательное-название.md`
4. Поместите файл в каталог `docs/reference/rfc/draft/`
5. Обновите этот индексный файл, добавив новую запись RFC
6. Отправьте PR для прохождения процесса рассмотрения
