## Жизненный цикл RFC

```
Черновик → На рассмотрении → Принят → Устарел (заменён)
                                ↓
                          Отклонён (не прошёл)
```

### Описание статусов

| Статус | Расположение | Описание |
|------|------|------|
| **Черновик** | `rfc/draft/` | Черновик автора, ожидает подачи на рассмотрение |
| **На рассмотрении** | `rfc/review/` | Открыто для обсуждения сообществом и обратной связи |

| **Устарел** | `rfc/deprecated/` | Был принят ранее, заменён новым дизайном |
| **Отклонён** | `rfc/rejected/` | Документы RFC, которые были отклонены |

## Подача RFC

1. Прочтите [RFC_TEMPLATE.md](RFC_TEMPLATE.md), чтобы узнать требования к формату
2. Изучите [EXAMPLE_full_feature_proposal.md](EXAMPLE_full_feature_proposal.md) для понимания стиля написания
3. Создайте новый файл с именем `номер-описательный-заголовок.md`
4. Поместите файл в каталог `docs/src/design/rfc/draft/`
5. Обновите данный индексный файл, добавив новую запись RFC
6. Отправьте PR для перехода в процесс рассмотрения
