## Жизненный цикл RFC

```
Черновик → На рассмотрении → Принят → Устарел (заменён)
                                  ↓
                              Отклонён (не прошёл)
```

### Описание статусов

| Статус | Расположение | Описание |
|------|------|------|
| **Черновик** | `rfc/draft/` | Авторский черновик, ожидает подачи на рассмотрение |
| **На рассмотрении** | `rfc/review/` | Открыт для обсуждения сообществом и обратной связи |
| **Принят** | `rfc/accepted/` | Становится официальным проектным документом, переходит в фазу реализации |
| **Устарел** | `rfc/deprecated/` | Был принят ранее, заменён новым дизайном |
| **Отклонён** | `rfc/rejected/` | Документ RFC, отклонённый по результатам рассмотрения |

## Подача RFC

1. Прочтите [RFC_TEMPLATE.md](RFC_TEMPLATE.md), чтобы ознакомиться с требованиями к формату
2. Изучите [EXAMPLE_full_feature_proposal.md](EXAMPLE_full_feature_proposal.md), чтобы понять структуру изложения
3. Создайте новый файл с именем `номер-описательное-название.md`
4. Поместите файл в каталог `docs/reference/rfc/draft/`
5. Обновите этот индекс-файл, добавив новую запись RFC
6. Отправьте PR для запуска процесса рассмотрения
