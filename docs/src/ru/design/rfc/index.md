## Жизненный цикл RFC

```
Черновик → На рассмотрении → Принято → Устарело (заменено)
                                    ↓
                                 Отклонено (не прошло)
```

### Описание статусов

| Статус | Расположение | Описание |
|------|------|------|
| **Черновик** | `rfc/draft/` | Черновик автора, ожидает подачи на рассмотрение |
| **На рассмотрении** | `rfc/review/` | Открыто для обсуждения и отзывов сообщества |
| **Принято** | `rfc/accepted/` | Становится официальным проектным документом, переходит в стадию реализации |
| **Устарело** | `rfc/deprecated/` | Было принято ранее, заменено новым проектом |
| **Отклонено** | `rfc/rejected/` | Документы RFC, которые были отклонены |

## Подача RFC

1. Ознакомьтесь с [RFC_TEMPLATE.md](RFC_TEMPLATE.md), чтобы узнать требования к формату
2. Изучите [EXAMPLE_full_feature_proposal.md](EXAMPLE_full_feature_proposal.md), чтобы понять стиль написания
3. Создайте новый файл с именем в формате `номер-описательное-название.md`
4. Поместите файл в каталог `docs/reference/rfc/draft/`
5. Обновите настоящий индексный файл, добавив новую запись RFC
6. Отправьте PR для запуска процесса рассмотрения
