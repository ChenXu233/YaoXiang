## Жизненный цикл RFC

```
┌─────────────┐
│   Черновик  │  ← Автор создаёт
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ На рассмотрении │  ← Открытое обсуждение сообщества и обратная связь
└──────┬──────┘
       │
       ├──────────────────┐
       ▼                  ▼
┌─────────────┐    ┌─────────────┐
│  Принят     │    │  Отклонён   │
└──────┬──────┘    └──────┬──────┘
       │                  │
       ▼                  ▼
┌─────────────┐    ┌─────────────┐
│  accepted/  │    │  rejected/  │
│(официальный │    │ (отклонён)  │
│   дизайн)   │    │             │
└─────────────┘    └─────────────┘
```

### Описание статусов

| Статус | Расположение | Описание |
|--------|--------------|----------|
| **Черновик** | `docs/reference/rfc/draft/` | Авторский черновик, ожидает подачи на рассмотрение |
| **На рассмотрении** | `docs/reference/rfc/review/` | Открытое обсуждение сообщества и обратная связь |
| **Принят** | `docs/reference/rfc/accepted/` | Становится официальным документом дизайна, переходит в стадию реализации |
| **Отклонён** | `docs/reference/rfc/rejected/` | Документы RFC, которые были отклонены |

## Подача RFC

1. Прочитайте [RFC_TEMPLATE.md](RFC_TEMPLATE.md), чтобы узнать требования к формату
2. Обратитесь к [EXAMPLE_full_feature_proposal.md](EXAMPLE_full_feature_proposal.md) для изучения стиля написания
3. Создайте новый файл с именем `порядковый-номер-описательное-название.md`
4. Поместите файл в каталог `docs/reference/rfc/draft/`
5. Обновите этот индексный файл, добавив новую запись RFC
6. Отправьте PR для прохождения процесса рассмотрения
