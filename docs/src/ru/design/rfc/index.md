## Жизненный цикл RFC

```
Черновик → На рассмотрении → Принято → Устарело (заменено)
                             ↓
                        Отклонено (не прошло)
```

### Описание статусов

| Статус | Расположение | Описание |
|--------|-------------|----------|
| **Черновик** | `rfc/draft/` | Авторский черновик, ожидает подачи на рассмотрение |
| **На рассмотрении** | `rfc/review/` | Открыто общественное обсуждение и обратная связь |
| **Принято** | `rfc/accepted/` | Принятый RFC документ |
| **Устарело** | `rfc/deprecated/` | Был принят, заменён новым дизайном |
| **Отклонено** | `rfc/rejected/` | Отклонённый RFC документ |

## Подача RFC

1. Прочитайте [RFC_TEMPLATE.md](RFC_TEMPLATE.md) для ознакомления с требованиями к формату
2. Обратитесь к [EXAMPLE_full_feature_proposal.md](EXAMPLE_full_feature_proposal.md) для изучения подхода к написанию
3. Создайте новый файл с именем `порядковый номер-описательное название.md`
4. Поместите файл в каталог `docs/src/design/rfc/draft/`
5. Обновите этот указатель, добавив новую запись RFC
6. Отправьте PR для прохождения процедуры рассмотрения
