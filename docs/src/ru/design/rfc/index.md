## Жизненный цикл RFC

```
Черновик → Рассмотрение → Принят → Устарел (заменён)
                              ↓
                         Отклонён (не прошёл)
```

### Описание статусов

| Статус | Расположение | Описание |
|------|------|------|
| **Черновик** | `rfc/draft/` | Черновик автора, ожидает подачи на рассмотрение |
| **На рассмотрении** | `rfc/review/` | Открыто для обсуждения сообществом и обратной связи |
| **Устарел** | `rfc/deprecated/` | Был принят, но заменён новой версией дизайна |
| **Отклонён** | `rfc/rejected/` | Документ RFC, который был отклонён |

## Подача RFC

1. Прочтите [RFC_TEMPLATE.md](RFC_TEMPLATE.md) для ознакомления с требованиями к формату
2. Изучите [EXAMPLE_full_feature_proposal.md](EXAMPLE_full_feature_proposal.md) в качестве образца написания
3. Создайте новый файл с именем `номер-описательное-название.md`
4. Поместите файл в каталог `docs/src/design/rfc/draft/`
5. Обновите данный индексный файл, добавив новую запись RFC
6. Отправьте PR для перехода в процесс рассмотрения
