## Жизненный цикл RFC

```
Черновик → На рассмотрении → Принято → Устарело (заменено)
                  ↓
             Отклонено (не прошло)
```

### Описание статусов

| Статус | Расположение | Описание |
|------|------|------|
| **Черновик** | `rfc/draft/` | Авторский черновик, ожидает подачи на рассмотрение |
| **На рассмотрении** | `rfc/review/` | Открыто для обсуждения сообщества и обратной связи |
| **Принято** | `rfc/accepted/` | Становится официальным документом дизайна, переходит в фазу реализации |
| **Устарело** | `rfc/deprecated/` | Было принято, заменено новым дизайном |
| **Отклонено** | `rfc/rejected/` | Документы отклонённых RFC |

## Подача RFC

1. Прочитайте [RFC_TEMPLATE.md](RFC_TEMPLATE.md) для ознакомления с требованиями к формату
2. Обратитесь к [EXAMPLE_full_feature_proposal.md](EXAMPLE_full_feature_proposal.md) для изучения стиля написания
3. Создайте новый файл с именем `порядковый-номер-описательное-название.md`
4. Поместите файл в каталог `docs/reference/rfc/draft/`
5. Обновите этот индексный файл, добавив новую запись RFC
6. Отправьте PR для начала процесса рассмотрения
