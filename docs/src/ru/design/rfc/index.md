## Жизненный цикл RFC

```
Проект → На рассмотрении → Принят → Устарел (заменён)
                  ↓
               Отклонён (не прошёл)
```

### Описание статусов

| Статус | Расположение | Описание |
|------|------|------|
| **Проект** | `rfc/draft/` | Черновик автора, ожидает подачи на рассмотрение |
| **На рассмотрении** | `rfc/review/` | Открыто для обсуждения сообществом и обратной связи |
| **Принят** | `rfc/accepted/` | Становится официальным документом дизайна, переходит в стадию реализации |
| **Устарел** | `rfc/deprecated/` | Был принят ранее, заменён новым дизайном |
| **Отклонён** | `rfc/rejected/` | Документы RFC, которые были отклонены |

## Подача RFC

1. Прочитайте [RFC_TEMPLATE.md](RFC_TEMPLATE.md) для ознакомления с требованиями к формату
2. Обратитесь к [EXAMPLE_full_feature_proposal.md](EXAMPLE_full_feature_proposal.md) для изучения стиля написания
3. Создайте новый файл с именем `номер-описательный-заголовок.md`
4. Поместите файл в каталог `docs/reference/rfc/draft/`
5. Обновите этот индексный файл, добавив новую запись RFC
6. Отправьте PR для входа в процесс рассмотрения
