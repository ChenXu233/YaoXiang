## Жизненный цикл RFC

```
Черновик → На рассмотрении → Принят → Устарел (заменён)
                              ↓
                       Отклонён (не прошёл)
```

### Описание статусов

| Статус | Расположение | Описание |
|------|------|------|
| **Черновик** | `rfc/draft/` | Авторский черновик, ожидает отправки на рассмотрение |
| **На рассмотрении** | `rfc/review/` | Открыт для обсуждения сообществом и обратной связи |
| **Принят** | `rfc/accepted/` | Становится официальным проектным документом, переходит в стадию реализации |
| **Устарел** | `rfc/deprecated/` | Был принят, но заменён новым дизайном |
| **Отклонён** | `rfc/rejected/` | Документ RFC, который был отклонён |

## Подача RFC

1. Прочтите [RFC_TEMPLATE.md](RFC_TEMPLATE.md) для ознакомления с требованиями к формату
2. Изучите [EXAMPLE_full_feature_proposal.md](EXAMPLE_full_feature_proposal.md) для понимания стиля написания
3. Создайте новый файл с именем `номер-описательный-заголовок.md`
4. Поместите файл в каталог `docs/reference/rfc/draft/`
5. Обновите этот индексный файл, добавив новую запись RFC
6. Отправьте PR для перехода в процесс рассмотрения
