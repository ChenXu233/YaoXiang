## Жизненный цикл RFC

```
Черновик → Рассмотрение → Принят → Устарел (заменён)
                          ↓
                       Отклонён (не прошёл)
```

### Описание статусов

| Статус | Расположение | Описание |
|------|------|------|
| **Черновик** | `rfc/draft/` | Черновик автора, ожидает отправки на рассмотрение |
| **На рассмотрении** | `rfc/review/` | Открыто для обсуждения и обратной связи от сообщества |
| **Принят** | `rfc/accepted/` | Становится официальным проектным документом, переходит в стадию реализации |
| **Устарел** | `rfc/deprecated/` | Был принят, но заменён новым дизайном |
| **Отклонён** | `rfc/rejected/` | Документ RFC, который был отклонён |

## Отправка RFC

1. Прочитайте [RFC_TEMPLATE.md](RFC_TEMPLATE.md), чтобы узнать требования к формату
2. Изучите [EXAMPLE_full_feature_proposal.md](EXAMPLE_full_feature_proposal.md) в качестве примера
3. Создайте новый файл с именем `номер-описательный-заголовок.md`
4. Поместите файл в каталог `docs/reference/rfc/draft/`
5. Обновите данный индексный файл, добавив новую запись RFC
6. Отправьте PR для запуска процесса рассмотрения
