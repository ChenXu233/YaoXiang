## Жизненный цикл RFC

```
Черновик → На рассмотрении → Принято → Устарело (заменено)
                              ↓
                          Отклонено (не прошло)
```

### Описание статусов

| Статус | Расположение | Описание |
|------|------|------|
| **Черновик** | `rfc/draft/` | Черновик автора, ожидает отправки на рассмотрение |
| **На рассмотрении** | `rfc/review/` | Открыто для обсуждения и обратной связи сообщества |
| **Принято** | `rfc/accepted/` | Становится официальным проектным документом, переходит в стадию реализации |
| **Устарело** | `rfc/deprecated/` | Было принято, но заменено новым дизайном |
| **Отклонено** | `rfc/rejected/` | Документ RFC, который был отклонён |

## Отправка RFC

1. Прочтите [RFC_TEMPLATE.md](RFC_TEMPLATE.md), чтобы узнать требования к формату
2. Изучите [EXAMPLE_full_feature_proposal.md](EXAMPLE_full_feature_proposal.md) для ознакомления с примером
3. Создайте новый файл с именем `номер-описательный-заголовок.md`
4. Поместите файл в каталог `docs/reference/rfc/draft/`
5. Обновите настоящий индексный файл, добавив новую запись RFC
6. Отправьте PR для перехода в процесс рассмотрения
