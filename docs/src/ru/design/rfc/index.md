## Жизненный цикл RFC

```
Черновик → На рассмотрении → Принят → Устарел (заменён)
                              ↓
                          Отклонён (не прошёл)
```

### Описание статусов

| Статус | Расположение | Описание |
|------|------|------|
| **Черновик** | `rfc/draft/` | Черновик автора, ожидает отправки на рассмотрение |
| **На рассмотрении** | `rfc/review/` | Открыто для обсуждения сообществом и обратной связи |
| **Принят** | `rfc/accepted/` | Становится официальным проектным документом, переходит в стадию реализации |
| **Устарел** | `rfc/deprecated/` | Был принят ранее, заменён новым дизайном |
| **Отклонён** | `rfc/rejected/` | Документы RFC, которые были отклонены |

## Подача RFC

1. Прочтите [RFC_TEMPLATE.md](RFC_TEMPLATE.md), чтобы узнать требования к формату
2. Обратитесь к [EXAMPLE_full_feature_proposal.md](EXAMPLE_full_feature_proposal.md), чтобы изучить стиль написания
3. Создайте новый файл с именем `номер-описательное-название.md`
4. Поместите файл в директорию `docs/reference/rfc/draft/`
5. Обновите этот файл индекса, добавив новую запись RFC
6. Отправьте PR для перехода в процесс рассмотрения
