## Жизненный цикл RFC

```
Проект → На рассмотрении → Принято → Устарело (заменено)
                            ↓
                        Отклонено (не прошло)
```

### Описание статусов

| Статус | Расположение | Описание |
|------|------|------|
| **Проект** | `rfc/draft/` | Черновик автора, ожидает отправки на рассмотрение |
| **На рассмотрении** | `rfc/review/` | Открыто для обсуждения и обратной связи от сообщества |
| **Принято** | `rfc/accepted/` | Становится официальным проектным документом, переходит в фазу реализации |
| **Устарело** | `rfc/deprecated/` | Было принято ранее, заменено новым дизайном |
| **Отклонено** | `rfc/rejected/` | Отклонённые документы RFC |

## Отправка RFC

1. Прочитайте [RFC_TEMPLATE.md](RFC_TEMPLATE.md), чтобы узнать требования к формату
2. Ознакомьтесь с [EXAMPLE_full_feature_proposal.md](EXAMPLE_full_feature_proposal.md), чтобы изучить стиль написания
3. Создайте новый файл с именем `номер-описательное-название.md`
4. Поместите файл в каталог `docs/reference/rfc/draft/`
5. Обновите этот индексный файл, добавив новую запись RFC
6. Отправьте PR для входа в процесс рассмотрения
