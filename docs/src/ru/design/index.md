# Проектная документация YaoXiang

> Дао рождает одно, одно рождает два, два рождают три, три рождают все сущее.

В этом каталоге содержатся проектные решения, предложения и обсуждения языка программирования YaoXiang.

## Базовые принципы проектирования

| Принцип | Описание |
|---------|----------|
| **Всё есть тип** | Значения, функции, модули — всё это типы; типы являются объектами первого класса |
| **Естественный синтаксис** | Читаемость на уровне Python, близость к естественному языку |
| **Модель владения** | Абстракции без накладных расходов, без GC, высокая производительность |
| **Модель 并作** | Синхронный синтаксис, асинхронная сущность, автоматический параллелизм |
| **Дружелюбие к ИИ** | Строгая структурированность, чёткий AST |

## Участие в проектных обсуждениях

### Процесс подачи предложений

```
1. Составление предложения (с использованием шаблона RFC)
   → Помещается в каталог rfc/

2. Обсуждение сообществом
   → Обсуждение в соответствующем issue rfc/REPO

3. Рассмотрение основной командой
   → Принятие → Перемещение в accepted/
   → Отклонение → Перемещение в archived/ или удаление
```

### Принципы проектирования

- **Чёткие границы**: каждое проектное решение должно иметь чёткую область применения
- **Практичность прежде всего**: решение реальных проблем, а не воображаемых угроз
- **Постепенная прозрачность**: многоуровневой дизайн модели 并作 (L1-L3)
- **Неизменность видимого поведения пользователя**: Never break userspace

## Примеры кода

```yaoxiang
// Определение типа
Point: Type = { x: Float, y: Float }
Result: Type[T, E] = { ok(T) | err(E) }

// Определение функции
add: (a: Int, b: Int) -> Int = a + b

// 并作-функция (автоматический 并作)
fetch_data: (url: String) -> JSON spawn = {
    HTTP.get(url).json()
}

// Главная функция
main: () -> Void = {
    print("Hello, YaoXiang!")
}
```

## Ключевые проектные решения

### 1. Типовая система

- **Унифицированный синтаксис типов**: упразднение `enum`, `struct`, `union`, единый синтаксис `Name: Type = {...}`
- **Конструкторы как типы**: устранение разрыва между «типом» и «значением»
- **Поддержка 泛型**: компиляционная мономорфизация, нулевые накладные расходы во время выполнения

### 2. 并作-модель

```yaoxiang
// Три уровня абстракции 并作

// L1: @blocking синхронный (параллелизм отключён)
fetch: (String) -> JSON @blocking = (url) => { ... }

// L2: spawn явный 并作
process: () -> Void spawn = () => {
    users = fetch_users()
    posts = fetch_posts()  // Автоматический 并作
}

// L3: Полная прозрачность (по умолчанию)
compute: (Int) -> Int = (n) => {
    a = heavy_calc(1)  // Система автоматически анализирует зависимости
    b = heavy_calc(2)
    c = heavy_calc(3)
    a + b + c
}
```

### 3. Обработка ошибок

```yaoxiang
Result: Type[T, E] = { ok(T) | err(E) }

process: () -> Result[Data, Error] = {
    data = fetch_data()?      // ? оператор прозрачно распространяет
    transformed = transform(data)?
    save(transformed)?
}
```
