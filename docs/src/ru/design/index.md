# Проектная документация YaoXiang

> Дао порождает единицу, единица порождает двойку, двойка порождает тройку, тройка порождает все сущее.

В данном каталоге содержатся проектные решения, предложения и обсуждения языка программирования YaoXiang.

## Основные проектные принципы

| Принцип | Описание |
|---------|----------|
| **Всё является типом** | Значения, функции, модули — всё это типы; типы являются объектами первого класса |
| **Естественный синтаксис** | Читаемость на уровне Python, приближённая к естественному языку |
| **Модель владения** | Нулевая стоимость абстракций, без сборщика мусора, высокая производительность |
| **Модель 并作** | Синхронный синтаксис, асинхронная сущность, автоматический параллелизм |
| **Дружественность к ИИ** | Строгая структурированность, чёткий AST |

## Участие в проектных обсуждениях

### Процесс подачи предложений

```
1. Составление предложения (по шаблону RFC)
   → Помещается в каталог rfc/

2. Обсуждение в сообществе
   → В соответствующем issue репозитория rfc/REPO

3. Рассмотрение основной командой
   → Принято → Переносится в accepted/
   → Отклонено → Переносится в archived/ или удаляется
```

### Проектные принципы

- **Чёткие границы**: каждое проектное решение должно иметь чёткую область применения
- **Практичность прежде всего**: решаем реальные проблемы, а не воображаемые угрозы
- **Постепенная прозрачность**: многоуровневый дизайн модели 并作 (L1-L3)
- **Неизменность видимого поведения пользователя**: Never break userspace

## Примеры кода

```yaoxiang
// Определение типа
Point: Type = { x: Float, y: Float }
Result: Type[T, E] = { ok(T) | err(E) }

// Определение функции
add: (a: Int, b: Int) -> Int = a + b

// 并作 функция (автоматический параллелизм)
fetch_data: (url: String) -> JSON spawn = {
    HTTP.get(url).json()
}

// Главная функция
main: () -> Void = {
    print("Hello, YaoXiang!")
}
```

## Ключевые проектные решения

### 1. Система типов

- **Единый синтаксис типов**: упразднение `enum`, `struct`, `union`, унификация через `Name: Type = {...}`
- **Конструкторы как типы**: устранение разрыва между «типом» и «значением»
- **Поддержка обобщений**: мономорфизация на этапе компиляции, нулевые накладные расходы во время выполнения

### 2. Модель 并作

```yaoxiang
// Три уровня абстракции параллелизма

// L1: @blocking синхронный (параллелизм отключён)
fetch: (String) -> JSON @blocking = (url) => { ... }

// L2: spawn явный параллелизм
process: () -> Void spawn = () => {
    users = fetch_users()
    posts = fetch_posts()  // Автоматический параллелизм
}

// L3: Полная прозрачность (по умолчанию)
compute: (Int) -> Int = (n) => {
    a = heavy_calc(1)  // Система автоматически анализирует зависимости
    b = heavy_calc(2)
    c = heavy_calc(3)
    a + b + c
}
```

### 3. Обработка ошибок

```yaoxiang
Result: Type[T, E] = { ok(T) | err(E) }

process: () -> Result[Data, Error] = {
    data = fetch_data()?      // Оператор ? прозрачно распространяет
    transformed = transform(data)?
    save(transformed)?
}
```
