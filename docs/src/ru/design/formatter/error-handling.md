# Обработка ошибок

---

## §E1 Ошибки форматирования

**§E1.1 Синтаксические ошибки.** Когда исходный код содержит синтаксические ошибки, форматировщик должен:

1. Использовать функцию `parse()` для разбора
2. Если разбор содержит ошибки, напрямую вернуть информацию об ошибке
3. Не вставлять никаких заполнителей

**§E1.2 Ошибки конфигурации.** Когда файл конфигурации имеет неверный формат, следует возвращать однозначное сообщение об ошибке.

---

## §E2 Коды выхода

| Код выхода | Значение |
|--------|------|
| 0 | Успех |
| 1 | В режиме `--check` обнаружены неотформатированные файлы |
| 2 | Файл не найден или ошибка конфигурации |

---

## §E3 Обработка ошибок

**§E3.1 Отчёт об ошибках.** Форматировщик использует функцию `parse()` для разбора исходного кода. Если в процессе разбора возникает ошибка, форматировщик напрямую возвращает информацию об ошибке и не выполняет форматирование.

**§E3.2 Без заполнителей.** Форматировщик не вставляет никаких заполнителей (например, `/* error */`) в места ошибок. При обнаружении синтаксической ошибки форматировщик напрямую сообщает об ошибке и завершает работу.
