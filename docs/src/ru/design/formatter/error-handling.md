# Обработка ошибок

---

## §E1 Ошибки форматирования

**§E1.1 Синтаксические ошибки.** Когда исходный код содержит синтаксические ошибки, форматировщик должен:

1. Форматировать правильные части по возможности
2. Сохранять исходное содержимое ошибочных узлов
3. Вставлять заполнитель `/* error */` в позицию ошибки

**§E1.2 Ошибки конфигурации.** При неправильном формате конфигурационного файла следует возвращать чёткое сообщение об ошибке.

---

## §E2 Коды завершения

| Код завершения | Значение |
|--------|------|
| 0 | Успех |
| 1 | Режим `--check` обнаружил неформатированные файлы |
| 2 | Файл не найден или ошибка конфигурации |

---

## §E3 Восстановление после ошибок

**§E3.1 Заполнители ошибок.** Когда исходный код содержит выражения, которые не удаётся разобрать, форматировщик должен вставить заполнитель `/* error */`.

```
// Исходный код (с синтаксической ошибкой)
let x = ;

// После форматирования
let x = /* error */;
```

**§E3.2 Устойчивость к ошибкам.** Форматировщик должен форматировать правильные части по возможности, сохраняя исходное содержимое ошибочных узлов.

```
// Исходный код
fn foo() {
    let x = 1;
    let y = ;  // Синтаксическая ошибка
    let z = 3;
}

// После форматирования
fn foo() {
    let x = 1;
    let y = /* error */;
    let z = 3;
}
```
