# Правила форматирования управляющих конструкций

---

## §5 Управляющие конструкции

**§5.1 Выражение if.** Между ключевым словом `if` и условием ставится пробел, между условием и блоком кода ставится пробел.

```
// ✅ Правильно
if condition { ... }

// ❌ Неправильно
if(condition) { ... }
if condition{ ... }
```

**§5.2 elif/else.** Между предыдущим блоком кода и `elif`/`else` ставится пробел.

```
// ✅ Правильно
if a > 0 { ... } elif a < 0 { ... } else { ... }

// ❌ Неправильно
if a > 0 { ... }elif a < 0 { ... }else { ... }
```

**§5.3 Цикл for.** Между ключевым словом `for`, переменной, ключевым словом `in` и итератором ставятся пробелы.

```
// ✅ Правильно
for item in collection { ... }

// ❌ Неправильно
for item in(collection) { ... }
for(item) in collection { ... }
```

**§5.4 Цикл while.** Между ключевым словом `while` и условием ставится пробел.

```
// ✅ Правильно
while condition { ... }

// ❌ Неправильно
while(condition) { ... }
```

**§5.5 Метки циклов.** Метка соединяется с ключевым словом цикла через `: `.

```
// ✅ Правильно
'outer: for i in range(10) { ... }

// ❌ Неправильно
'outer:for i in range(10) { ... }
'outer : for i in range(10) { ... }
```

## §5.6 Оператор return

**§5.6.1 Формат return.** Между ключевым словом `return` и выражением ставится пробел.

```
// ✅ Правильно
return 42;
return x + y;

// ❌ Неправильно
return(42);  // отсутствует пробел
return  42;  // лишний пробел
```

**§5.6.2 Пустой return.** Пустой return использует только ключевое слово `return`.

```
// ✅ Правильно
return;

// ❌ Неправильно
return ;  // лишний пробел
return void;  // void не нужен
```

## §5.7 Оператор break

**§5.7.1 Формат break.** Между ключевым словом `break` и меткой ставится пробел.

```
// ✅ Правильно
break;
break 'outer;

// ❌ Неправильно
break(outer);  // неправильный синтаксис
break  'outer;  // лишний пробел
```

continue;
continue 'outer;

// ❌ Неправильно
