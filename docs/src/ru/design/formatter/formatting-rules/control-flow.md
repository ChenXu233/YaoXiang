# Правила форматирования управляющих конструкций

---

## §5 Управляющие конструкции

**§5.1 Выражение if.** Ключевое слово `if` отделяется пробелом от условия, условие отделяется пробелом от блока кода.

```
// ✅ Правильно
if condition { ... }

// ❌ Неправильно
if(condition) { ... }
if condition{ ... }
```

**§5.2 elif/else.** Ключевые слова `elif` и `else` отделяются пробелом от предшествующего блока кода.

```
// ✅ Правильно
if a > 0 { ... } elif a < 0 { ... } else { ... }

// ❌ Неправильно
if a > 0 { ... }elif a < 0 { ... }else { ... }
```

**§5.3 Цикл for.** Ключевое слово `for`, переменная, ключевое слово `in` и итератор разделяются пробелами.

```
// ✅ Правильно
for item in collection { ... }

// ❌ Неправильно
for item in(collection) { ... }
for(item) in collection { ... }
```

**§5.4 Цикл while.** Ключевое слово `while` отделяется пробелом от условия.

```
// ✅ Правильно
while condition { ... }

// ❌ Неправильно
while(condition) { ... }
```

**§5.5 Метки циклов.** Метка соединяется с ключевым словом цикла через `: `.

```
// ✅ Правильно
'outer: for i in range(10) { ... }

// ❌ Неправильно
'outer:for i in range(10) { ... }
'outer : for i in range(10) { ... }
```
