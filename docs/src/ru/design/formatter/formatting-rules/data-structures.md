# Правила форматирования структур данных

---

## §8 Литералы

**§8.1 Целочисленные литералы.** Целочисленные литералы выводятся напрямую.

```
// ✅ Правильно
let x = 42;
```

**§8.2 Литералы с плавающей точкой.** Литералы с плавающей точкой должны содержать десятичную точку.

```
// ✅ Правильно
let x = 3.14;
let y = 42.0;  // должна быть десятичная точка

// ❌ Ошибка
let y = 42;    // целое число, а не с плавающей точкой
```

**§8.3 Строковые литералы.** По умолчанию используются двойные кавычки. При `single_quote = true` используются одинарные кавычки.

```
// По умолчанию (двойные кавычки)
let s = "hello";

// single_quote = true
let s = 'hello';
```

**§8.4 Булевы литералы.** Булевы литералы используют нижний регистр.

```
// ✅ Правильно
let x = true;
let y = false;

// ❌ Ошибка
let x = True;
let y = FALSE;
```

## §10 Списки и словари

**§10.1 Формат списков.** Списки заключаются в `[]`, элементы разделяются запятыми.

```
// ✅ Правильно
let x = [1, 2, 3];

// ❌ Ошибка
let x = [1,2,3];
```

**§10.2 Формат словарей.** Словари заключаются в `{}`, пары ключ-значение используют формат `key: value`.

```
// ✅ Правильно
let x = {"a": 1, "b": 2};

// ❌ Ошибка
let x = {"a":1, "b":2};
```

**§10.3 Списковые включения.** Списковые включения используют формат `[expr for var in iterable]`.

```
// ✅ Правильно
let x = [i * 2 for i in range(10)];

// С условием
let x = [i for i in range(10) if i > 5];
```

## §11 Выражения Match

**§11.1 Формат match.** Ключевое слово `match` отделяется от выражения пробелом.

```
// ✅ Правильно
match x { ... }

// ❌ Ошибка
match(x) { ... }
```

**§11.2 Выравнивание pattern.** Несколько pattern должны быть выровнены с использованием пробелов.

```
// ✅ Выровнено
match x {
    1    => "one",
    2    => "two",
    100  => "hundred",
    _    => "other",
}
```

**§11.3 Перенос длинных pattern.** Когда pattern слишком длинный, pattern переносится, а `=>` выравнивается с body.

```
// ✅ Перенос
match x {
    VeryLongPatternName { field1, field2 }
        => handle_case(field1, field2),
    _ => default_case(),
}
```
