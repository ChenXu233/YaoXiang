本文档定义了 `yaoxiang fmt` 代码格式化工具的行为规范。所有格式化行为必须遵循本规范。

## 原则

**原则 1：格式化是幂等的。** 对已格式化的代码再次格式化，输出必须与输入完全相同。

```rust
// 规则：format(format(code)) == format(code)
assert_eq!(format_source(input, &opts), format_source(&format_source(input, &opts).unwrap(), &opts).unwrap());
```

**原则 2：格式化不改变语义。** 格式化前后的代码必须具有相同的 AST（抽象语法树）。

**原则 3：格式化保留所有注释。** 单行注释、多行注释、文档注释必须被保留，不得删除或修改。

**原则 4：配置优先级。** 配置优先级链为：CLI 参数 > 项目级配置（`yaoxiang.toml`）> 用户级配置（`~/.config/yaoxiang/config.toml`）> 默认值。

## 适用范围

本规范适用于所有 `.yx` 源文件的格式化。

**Принципы**

**Принцип 1: Форматирование является идемпотентным.** Повторное форматирование уже отформатированного кода должно давать результат, полностью идентичный входному.

```rust
// Правило: format(format(code)) == format(code)
assert_eq!(format_source(input, &opts), format_source(&format_source(input, &opts).unwrap(), &opts).unwrap());
```

**Принцип 2: Форматирование не изменяет семантику.** Код до и после форматирования должен иметь одинаковое AST (абстрактное синтаксическое дерево).

**Принцип 3: Форматирование сохраняет все комментарии.** Однострочные, многострочные и документирующие комментарии должны быть сохранены, их запрещено удалять или изменять.

**Принцип 4: Приоритет конфигурации.** Цепочка приоритетов конфигурации: параметры CLI > конфигурация проекта (`yaoxiang.toml`) > пользовательская конфигурация (`~/.config/yaoxiang/config.toml`) > значения по умолчанию.

**Область применения**

Настоящая спецификация применяется к форматированию всех файлов исходного кода `.yx`.
