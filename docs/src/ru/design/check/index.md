# Проектная документация команды check

`yaoxiang check` — это инструмент статической проверки компилятора YaoXiang, обеспечивающий проверку типов, межфайловый анализ и инкрементальную проверку.

## Принципы проектирования

1. **Ноль ложных срабатываний**: каждая сообщённая ошибка должна быть подлинной ошибкой
2. **Межфайловая осведомлённость**: корректное обнаружение межмодульных ошибок типов и неопределённых ссылок
3. **Приоритет инкрементальности**: в режиме watch перепроверяются только затронутые файлы
4. **Самодокументация**: коды ошибок, шаблоны сообщений и справочные тексты управляются через i18n

## Навигация по документации

- [Система диагностики](./diagnostic-system.md) — система кодов ошибок, структура данных Diagnostic, вывод Emitter
- [Межфайловый анализ](./cross-file-analysis.md) — общая среда типов, граф зависимостей, топологическая сортировка
- [Инкрементальная проверка](./incremental-checking.md) — CheckSession, affected_modules, режим watch

## Границы взаимодействия с другими системами

| Система | Ответственность | Связь с check |
|---------|-----------------|---------------|
| Компилятор (`yaoxiang build`) | Полная компиляция (разбор → проверка типов → генерация кода) | check выполняет только первые два этапа |
| LSP | Интеграция с редактором (дополнение, переходы, диагностика) | диагностика check может быть переиспользована |
| Форматирование (`yaoxiang fmt`) | Стиль кода | Независимо, используется параллельно в CI |
