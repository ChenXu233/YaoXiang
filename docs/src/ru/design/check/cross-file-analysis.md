# Межфайловый анализ

## Описание проблемы

В ранних реализациях `check_files_with_diagnostics` создавал независимый `Compiler` для каждого файла, что не позволяло обнаруживать межфайловые ссылки. Функции `pub`, определённые в fileA, не распознавались в fileB.

## Решение

Использование общей `TypeEnvironment` для проверки всех модулей в порядке зависимостей.

## Процесс реализации

```text
1. Параллельный парсинг всех .yx файлов → Vec<(PathBuf, ModuleId, AST)>
2. Построение графа зависимостей с помощью ModuleDependencyGraph::build_from_ast
3. detect_cycles() проверяет циклические зависимости → ошибка
4. topological_sort() получает порядок компиляции
5. Типовая проверка по порядку:
   a. Создание общей TypeEnvironment (с модулем std)
   b. Для каждого модуля: регистрация его экспортов в общую среду → типовая проверка
   c. Сбор диагностической информации
6. Возврат CheckResult
```

## Изоляция пространства имён

Используется формат `module_name.symbol_name` для хранения экспортируемых символов во избежание конфликтов одноимённых символов из разных модулей.

## Известные ограничения

- `traits/` — заглушка (coherence/impl_check/object_safety/resolution) не реализованы
- `check_single_module` всё ещё создаёт независимый Compiler для каждого модуля (полный перенос информации о типах через общую среду ещё не реализован)
