# Менеджер пакетов

Встроенный в YaoXiang менеджер пакетов, предоставляющий полноценное управление зависимостями.

## Обзор

YaoXiang Package Manager (YPM) использует декларативное управление зависимостями:

- Объявление зависимостей проекта в `yaoxiang.toml`
- Файл `yaoxiang.lock` фиксирует точные версии, обеспечивая воспроизводимость сборки
- Зависимости загружаются в директорию `vendor`

## Быстрый старт

```bash
# Создать новый проект
yaoxiang init my-project
cd my-project

# Добавить зависимость
yaoxiang add http
yaoxiang add json

# Установить зависимости
yaoxiang install

# Запустить проект
yaoxiang run src/main.yx
```

## Структура проекта

```
my-project/
├── yaoxiang.toml      # Манифест проекта
├── yaoxiang.lock      # Lock-файл зависимостей
├── vendor/            # Хранилище зависимостей
└── src/
    └── main.yx
```

## init

Инициализирует новый проект.

### Использование

```bash
yaoxiang init <name>
```

### Параметры

| Параметр | Тип | Описание |
|------|------|------|
| `name` | string | Название проекта |

### Описание

Создаёт новый проект YaoXiang в текущей директории или в указанном пути.

### Создаваемые файлы

- `yaoxiang.toml` — манифест проекта
- `yaoxiang.lock` — lock-файл зависимостей
- `src/main.yx` — точка входа
- `.gitignore` — конфигурация игнорирования Git

### Примеры

```bash
# Создать проект в текущей директории
yaoxiang init my-project

# Вывод
# ✨ Проект создан: my-project
#   my-project/yaoxiang.toml
#   my-project/yaoxiang.lock
#   my-project/src/main.yx
#   my-project/.gitignore
```

## add

Добавляет зависимость в проект.

### Использование

```bash
yaoxiang add <name> [version]
yaoxiang add <name> --dev
```

### Параметры

| Параметр | Тип | Описание |
|------|------|------|
| `name` | string | Название пакета |
| `version` | string | Версия (необязательно, по умолчанию `*`) |

### Опции

| Опция | Описание |
|------|------|
| `--dev`, `-D` | Добавить как зависимость разработки |

### Описание

Добавляет зависимость в файл `yaoxiang.toml` проекта и обновляет `yaoxiang.lock`.

### Спецификации версий

| Спецификация | Описание | Пример |
|------|------|------|
| `*` | Любая версия | `http = "*"` |
| `1.0.0` | Точная версия | `http = "1.0.0"` |
| `>=1.0.0` | Минимальная версия | `http = ">1.0.0"` |
| `~1.0.0` | Совместимая версия | `http = "~1.0.0"` |
| `^1.0.0` | Caret-версия | `http = "^1.0.0"` |

### Источники зависимостей

#### Registry (по умолчанию)

```bash
yaoxiang add http
yaoxiang add http 1.0.0
```

#### Git-репозиторий

```bash
# В манифесте будет сгенерирована следующая конфигурация
# http = { version = "1.0.0", git = "https://github.com/example/http" }
```

#### Локальный путь

```bash
# В манифесте будет сгенерирована следующая конфигурация
# mylib = { version = "0.1.0", path = "./mylib" }
```

### Примеры

```bash
# Добавить последнюю версию
yaoxiang add http

# Добавить указанную версию
yaoxiang add http 1.0.0

# Добавить диапазон версий
yaoxiang add json ">=2.0.0"

# Добавить зависимость разработки
yaoxiang add test-utils --dev
yaoxiang add benchmark -D
```

## rm

Удаляет зависимость из проекта.

### Использование

```bash
yaoxiang rm <name>
yaoxiang rm <name> --dev
```

### Параметры

| Параметр | Тип | Описание |
|------|------|------|
| `name` | string | Название пакета |

### Опции

| Опция | Описание |
|------|------|
| `--dev`, `-D` | Удалить зависимость разработки |

### Описание

Удаляет указанную зависимость из `yaoxiang.toml` проекта и обновляет `yaoxiang.lock`.

### Примеры

```bash
# Удалить зависимость времени выполнения
yaoxiang rm http

# Удалить зависимость разработки
yaoxiang rm test-utils --dev
```

## install

Устанавливает зависимости проекта.

### Использование

```bash
yaoxiang install
```

### Описание

Читает объявления зависимостей из `yaoxiang.toml` и выполняет следующие операции:

1. Разрешение версий зависимостей
2. Обнаружение конфликтов версий
3. Загрузка зависимостей в директорию `vendor`
4. Генерация/обновление `yaoxiang.lock`

### Поведение

- Если зависимостей нет — выводит информационное сообщение и завершает работу
- Если директория `vendor` существует — проверяет и использует кэш
- При обнаружении конфликтов версий — выводит сообщение об ошибке и завершает работу

### Примеры

```bash
# Установить все зависимости
yaoxiang install

# Вывод
# 📦 Разрешение зависимостей...
#   http (1.0.0) [установлено]
#   json (2.0.0) [в кэше]
# ✅ Зависимости установлены, lock-файл обновлён
```

### Обновление lock-файла

Команда `install` обновляет `yaoxiang.lock`:

```toml
# yaoxiang.lock
[package]
version = 1

[package.http]
version = "1.0.0"
source = "registry"

[package.json]
version = "2.0.0"
source = "registry"
```

## update

Обновляет зависимости проекта.

### Использование

```bash
yaoxiang update
yaoxiang update <name>
```

### Параметры

| Параметр | Тип | Описание |
|------|------|------|
| `name` | string | Название пакета (необязательно) |

### Описание

### Полное обновление

Без параметров обновляет все зависимости:

1. Очищает текущие зафиксированные версии
2. Удаляет старые версии из директории `vendor`
3. Перезагружает все зависимости
4. Обновляет `yaoxiang.lock`

### Обновление одного пакета

С параметром обновляет только указанную зависимость:

1. Удаляет старую версию из `vendor`
2. Перезагружает новую версию
3. Обновляет соответствующую запись в `yaoxiang.lock`
4. Другие зависимости не затрагиваются

### Примеры

```bash
# Обновить все зависимости
yaoxiang update

# Вывод
# 📦 Обновление зависимостей...
#   http (1.0.0 → 1.1.0)
#   json (2.0.0 → 2.1.0)
# ✅ Обновлено 2 зависимости, lock-файл обновлён

# Обновить одну зависимость
yaoxiang update http

# Вывод
# ✅ Обновлён http (1.0.0 → 1.1.0)
```

## list

Выводит список зависимостей проекта.

### Использование

```bash
yaoxiang list
```

### Описание

Отображает все зависимости проекта, включая:

- Зависимости времени выполнения (из `[dependencies]`)
- Зависимости разработки (из `[dev-dependencies]`)
- Версию и источник каждой зависимости

### Примеры

```bash
yaoxiang list

# Вывод
# 📦 Зависимости проекта
#
# Зависимости времени выполнения:
#   http        1.0.0    registry
#   json        2.0.0    registry
#
# Зависимости разработки:
#   test-utils  0.5.0    registry
```

## Конфигурационные файлы

### yaoxiang.toml

Манифест проекта, объявляющий метаданные проекта и зависимости.

```toml
[package]
name = "my-project"
version = "0.1.0"
description = "Описание проекта"
authors = ["Автор <email@example.com>"]
license = "MIT"

[dependencies]
http = "1.0.0"
json = "*"

[dev-dependencies]
test-utils = "0.5.0"
```

### yaoxiang.lock

Lock-файл зависимостей, автоматически генерируемый менеджером пакетов.

```toml
# Автоматически сгенерировано менеджером пакетов YaoXiang

[package]
version = 1

[package.http]
version = "1.0.0"
source = "registry"
```

## Ключевые концепции

### Зависимости времени выполнения vs зависимости разработки

- **Зависимости времени выполнения** (`[dependencies]`): пакеты, необходимые для работы проекта
- **Зависимости разработки** (`[dev-dependencies]`): пакеты, нужные только для разработки и тестирования

### Источники зависимостей

| Тип | Пример конфигурации | Описание |
|------|----------|------|
| Registry | `http = "1.0.0"` | Получение из удалённого реестра пакетов |
| Git | `{ version = "1.0.0", git = "https://..." }` | Получение из Git-репозитория |
| Path | `{ version = "0.1.0", path = "./lib" }` | Получение из локального пути |

### Lock-файл

`yaoxiang.lock` автоматически генерируется менеджером пакетов. **Обязательно коммитьте его в систему контроля версий**:

- Обеспечивает использование одинаковых версий зависимостей всеми членами команды
- Обеспечивает воспроизводимость CI-сборки
- Предотвращает проблемы типа "работает на моей машине"

### Директория vendor

Зависимости после загрузки хранятся в директории `vendor`:

- Автоматически управляется командами `yaoxiang install` и `yaoxiang update`
- Можно удалить и пересоздать повторным запуском `install`
- Рекомендуется добавить в `.gitignore` — каждый участник команды управляет ей независимо

## Часто задаваемые вопросы

### Q: Что делать при конфликте версий зависимостей?

YPM обнаруживает конфликты версий и выдаёт ошибку. Решения:

1. Скорректировать требования к версии зависимости
2. Дождаться исправления от автора зависимости
3. Рассмотреть возможность удаления конфликтующей зависимости

### Q: Как использовать приватные пакеты?

Для приватных пакетов можно использовать источник Git:

```bash
# Добавить через Git URL
# Вручную отредактировать yaoxiang.toml
[dependencies]
private-pkg = { version = "1.0.0", git = "https://github.com/org/private-pkg" }
```

### Q: Можно ли удалить директорию vendor?

Да. После удаления выполните `yaoxiang install` — все зависимости будут перезагружены.

### Q: Как посмотреть информацию о пакете?

Используйте `yaoxiang list` для просмотра всех зависимостей или просмотрите файл `yaoxiang.toml`.
