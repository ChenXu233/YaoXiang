# Менеджер пакетов

Встроенный в YaoXiang менеджер пакетов, предоставляющий полноценные функции управления зависимостями.

## Обзор

YaoXiang Package Manager (YPM) использует декларативное управление зависимостями:

- Зависимости проекта объявляются в `yaoxiang.toml`
- `yaoxiang.lock` блокирует точные версии, обеспечивая воспроизводимость сборки
- Зависимости загружаются в каталог `vendor`

## Быстрый старт

```bash
# Создать новый проект
yaoxiang init my-project
cd my-project

# Добавить зависимость
yaoxiang add http
yaoxiang add json

# Установить зависимости
yaoxiang install

# Запустить проект
yaoxiang run src/main.yx
```

## Структура проекта

```
my-project/
├── yaoxiang.toml      # Манифест проекта
├── yaoxiang.lock      # Файл блокировки зависимостей
├── vendor/            # Хранилище зависимостей
└── src/
    └── main.yx
```

## init

Инициализирует новый проект.

### Использование

```bash
yaoxiang init <name>
```

### Параметры

| Параметр | Тип | Описание |
|----------|------|----------|
| `name` | string | Название проекта |

### Описание

Создаёт новый проект YaoXiang в текущем каталоге или в указанном пути.

### Создаваемые файлы

- `yaoxiang.toml` — манифест проекта
- `yaoxiang.lock` — файл блокировки зависимостей
- `src/main.yx` — точка входа
- `.gitignore` — конфигурация игнорирования для Git

### Примеры

```bash
# Создать проект в текущем каталоге
yaoxiang init my-project

# Вывод
# ✨ Проект создан: my-project
#   my-project/yaoxiang.toml
#   my-project/yaoxiang.lock
#   my-project/src/main.yx
#   my-project/.gitignore
```

## add

Добавляет зависимость в проект.

### Использование

```bash
yaoxiang add <name> [version]
yaoxiang add <name> --dev
```

### Параметры

| Параметр | Тип | Описание |
|----------|------|----------|
| `name` | string | Название пакета |
| `version` | string | Версия (необязательно, по умолчанию `*`) |

### Опции

| Опция | Описание |
|------|----------|
| `--dev`, `-D` | Добавить как зависимость для разработки |

### Описание

Добавляет зависимость в файл `yaoxiang.toml` проекта и обновляет `yaoxiang.lock`.

### Спецификации версий

| Спецификация | Описание | Пример |
|------|----------|--------|
| `*` | Любая версия | `http = "*"` |
| `1.0.0` | Точная версия | `http = "1.0.0"` |
| `>=1.0.0` | Минимальная версия | `http = ">1.0.0"` |
| `~1.0.0` | Совместимая версия | `http = "~1.0.0"` |
| `^1.0.0` | Caret-версия | `http = "^1.0.0"` |

### Источники зависимостей

#### Registry (по умолчанию)

```bash
yaoxiang add http
yaoxiang add http 1.0.0
```

#### Git-репозиторий

```bash
# В манифесте будет сгенерирована следующая конфигурация
# http = { version = "1.0.0", git = "https://github.com/example/http" }
```

#### Локальный путь

```bash
# В манифесте будет сгенерирована следующая конфигурация
# mylib = { version = "0.1.0", path = "./mylib" }
```

### Примеры

```bash
# Добавить последнюю версию
yaoxiang add http

# Добавить конкретную версию
yaoxiang add http 1.0.0

# Добавить диапазон версий
yaoxiang add json ">=2.0.0"

# Добавить зависимость для разработки
yaoxiang add test-utils --dev
yaoxiang add benchmark -D
```

## rm

Удаляет зависимость из проекта.

### Использование

```bash
yaoxiang rm <name>
yaoxiang rm <name> --dev
```

### Параметры

| Параметр | Тип | Описание |
|----------|------|----------|
| `name` | string | Название пакета |

### Опции

| Опция | Описание |
|------|----------|
| `--dev`, `-D` | Удалить зависимость для разработки |

### Описание

Удаляет указанную зависимость из `yaoxiang.toml` проекта и обновляет `yaoxiang.lock`.

### Примеры

```bash
# Удалить runtime-зависимость
yaoxiang rm http

# Удалить зависимость для разработки
yaoxiang rm test-utils --dev
```

## install

Устанавливает зависимости проекта.

### Использование

```bash
yaoxiang install
```

### Описание

Читает объявления зависимостей из `yaoxiang.toml` и выполняет следующие действия:

1. Разрешает версии зависимостей
2. Обнаруживает конфликты версий
3. Загружает зависимости в каталог `vendor`
4. Генерирует/обновляет `yaoxiang.lock`

### Поведение

- Если зависимостей нет, отображается информационное сообщение и выполняется выход
- Если каталог `vendor` уже существует, выполняется проверка и повторное использование кэша
- При обнаружении конфликтов версий отображается сообщение об ошибке и выполняется выход

### Примеры

```bash
# Установить все зависимости
yaoxiang install

# Вывод
# 📦 Разрешение зависимостей...
#   http (1.0.0) [установлен]
#   json (2.0.0) [в кэше]
# ✅ Зависимости установлены, файл блокировки обновлён
```

### Обновление файла блокировки

Команда `install` обновляет `yaoxiang.lock`:

```toml
# yaoxiang.lock
[package]
version = 1

[package.http]
version = "1.0.0"
source = "registry"

[package.json]
version = "2.0.0"
source = "registry"
```

## update

Обновляет зависимости проекта.

### Использование

```bash
yaoxiang update
yaoxiang update <name>
```

### Параметры

| Параметр | Тип | Описание |
|----------|------|----------|
| `name` | string | Название пакета (необязательно) |

### Описание

### Полное обновление

Без параметров обновляются все зависимости:

1. Очищаются текущие заблокированные версии
2. Очищаются старые версии в каталоге `vendor`
3. Повторно загружаются все зависимости
4. Обновляется `yaoxiang.lock`

### Частичное обновление

С параметром обновляется только указанная зависимость:

1. Удаляется старая версия из `vendor`
2. Повторно загружается новая версия
3. Обновляется соответствующая запись в `yaoxiang.lock`
4. Другие зависимости не затрагиваются

### Примеры

```bash
# Обновить все зависимости
yaoxiang update

# Вывод
# 📦 Обновление зависимостей...
#   http (1.0.0 → 1.1.0)
#   json (2.0.0 → 2.1.0)
# ✅ Обновлено 2 зависимости, файл блокировки обновлён

# Обновить одну зависимость
yaoxiang update http

# Вывод
# ✅ Обновлён http (1.0.0 → 1.1.0)
```

## list

Выводит список зависимостей проекта.

### Использование

```bash
yaoxiang list
```

### Описание

Отображает все зависимости проекта, включая:

- Runtime-зависимости (из `[dependencies]`)
- Зависимости для разработки (из `[dev-dependencies]`)
- Версию и источник каждой зависимости

### Примеры

```bash
yaoxiang list

# Вывод
# 📦 Зависимости проекта
#
# Runtime-зависимости:
#   http        1.0.0    registry
#   json        2.0.0    registry
#
# Зависимости для разработки:
#   test-utils  0.5.0    registry
```

## Конфигурационные файлы

### yaoxiang.toml

Манифест проекта, объявляющий метаданные проекта и зависимости.

```toml
[package]
name = "my-project"
version = "0.1.0"
description = "Описание проекта"
authors = ["Автор <email@example.com>"]
license = "MIT"

[dependencies]
http = "1.0.0"
json = "*"

[dev-dependencies]
test-utils = "0.5.0"
```

### yaoxiang.lock

Файл блокировки зависимостей, автоматически генерируемый менеджером пакетов.

```toml
# Автоматически сгенерировано YaoXiang Package Manager

[package]
version = 1

[package.http]
version = "1.0.0"
source = "registry"
```

## Основные концепции

### Runtime-зависимости vs зависимости для разработки

- **Runtime-зависимости** (`[dependencies]`): пакеты, необходимые для работы проекта
- **Зависимости для разработки** (`[dev-dependencies]`): пакеты, нужные только при разработке и тестировании

### Источники зависимостей

| Тип | Пример конфигурации | Описание |
|------|----------|--------|
| Registry | `http = "1.0.0"` | Получение из удалённого реестра пакетов |
| Git | `{ version = "1.0.0", git = "https://..." }` | Получение из Git-репозитория |
| Path | `{ version = "0.1.0", path = "./lib" }` | Получение из локального пути |

### Файл блокировки

`yaoxiang.lock` автоматически генерируется менеджером пакетов, настоятельно рекомендуется **обязательно добавить его в систему контроля версий**:

- Обеспечивает использование всеми членами команды полностью одинаковых версий зависимостей
- Гарантирует воспроизводимость сборки в CI
- Избегает проблем типа «на моей машине работает»

### Каталог vendor

Зависимости после загрузки хранятся в каталоге `vendor`:

- Автоматически управляется командами `yaoxiang install` и `yaoxiang update`
- Можно удалить и пересоздать повторным запуском `install`
- Рекомендуется добавить в `.gitignore`, чтобы разные члены команды управляли им независимо

## Часто задаваемые вопросы

### В: Что делать при конфликте версий зависимостей?

YPM обнаруживает конфликты версий зависимостей и выдаёт ошибку. Решения:

1. Скорректировать требования к версиям зависимостей
2. Дождаться исправления от автора зависимости
3. Рассмотреть возможность удаления конфликтующей зависимости

### В: Как использовать приватные пакеты?

Для приватных пакетов можно использовать Git-источник:

```bash
# Добавить через Git URL
# Вручную отредактировать yaoxiang.toml
[dependencies]
private-pkg = { version = "1.0.0", git = "https://github.com/org/private-pkg" }
```

### В: Можно ли удалить каталог vendor?

Да. После удаления выполните `yaoxiang install` — все зависимости будут повторно загружены.

### В: Как посмотреть информацию о пакете?

Используйте `yaoxiang list` для просмотра всех зависимостей или просмотрите `yaoxiang.toml`.
