## Продвинутые возможности

### Семейства типов

Семейства типов позволяют определять на уровне типов несколько ассоциированных типов. Каждый член семейства типов может иметь собственный набор ассоциированных типов, не будучи ограничен единым обобщённым ограничением.

```yao
trait Collection<T> {
    pub type Reducer<Result>;
    pub type Accumulator<State>;
}
```

Возможность ассоциированных типов в **YaoXiang** позволяет каждой реализации иметь различные ассоциированные типы, делая сигнатуры типов более соответствующими конкретным требованиям, а не скованными единым обобщённым ограничением.

### Параллелизм

Механизм `spawn` в YaoXiang работает асинхронно и автоматически распределяет работу между доступными процессорными блоками. Каждая точка `spawn` возвращает дескриптор для управления параллельной задачей.

```yao
let result = spawn compute_data(arg1, arg2);
```

#### Spawn-блоки

Spawn-блоки похожи на обычные блоки кода, но выполняются асинхронно. Они могут захватывать переменные из родительской области видимости.

```yao
let counter = 0;
let handle = spawn {
    counter = counter + 1;
    process_data(data);
};

handle.wait();
handle.cancel();
```

Когда spawn-блок изменяет захваченную переменную, это изменение не затрагивает оригинальную переменную в родительской области видимости. Если переменная родителя выходит из области видимости до завершения spawn-блока, она также становится недоступна для дочернего процесса.

#### Spawn-функции

Spawn-функции предоставляют альтернативный способ асинхронного выполнения. В отличие от блоков, они не наследуют контекст вызова.

```yao
let result = spawn async_compute(x, y, data);
```

Все переменные должны передаваться явно в качестве аргументов, и любое изменение этих переменных внутри spawn-функции не влияет на оригинальные переменные вызывающей области видимости.

#### Spawn-циклы

Spawn-циклы распределяют итерации между процессорными блоками для параллельного выполнения:

```yao
for item in data_list {
    spawn process_item(item);
}
```
