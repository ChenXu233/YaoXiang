# Определение и вызов функций

В предыдущей главе вы научились объявлять переменные. Эта глава познакомит вас с основой YaoXiang — функциями. Синтаксис функций в YaoXiang разделяет ту же модель `name: type = value` с объявлением переменных, поэтому вам всё будет знакомо.

## Функция — это Lambda

Сначала самая важная концепция: **в YaoXiang функция по сути является лямбда-выражением**. Никакого специального ключевого слова `fn`, никаких сложных ритуалов. Определить функцию — значит дать имя лямбде.

```
# Любая функция по сути является комбинацией этих четырёх вещей:
name: (params) -> Return = body
 ^       ^        ^        ^
 |       |        |        +-- Тело функции (лямбда-выражение или блок кода)
 |       |        +-- Тип возвращаемого значения
 |       +-- Список параметров (сигнатура)
 +-- Имя функции
```

Это полностью соответствует изученному в предыдущей главе `name: type = value` — просто «типом» здесь оказывается функциональный тип.

## Форма выражения: прямое возвращение значения

Самая простая функция не требует ключевого слова `return`. Когда тело функции — это одно выражение, оно напрямую становится возвращаемым значением:

```yaoxiang
# Форма выражения — прямое возвращение значения, без return
add: (a: Int, b: Int) -> Int = a + b
square: (x: Int) -> Int = x * x
greet: (name: String) -> String = "Привет, " + name
```

Вызовем их:

```yaoxiang
sum = add(3, 5)          # sum = 8
sq = square(4)           # sq = 16
msg = greet("мир")       # msg = "Привет, мир"
```

Это называется **формой выражения**. Когда тело функции — это выражение (а не блок кода `{ }`), его значение напрямую становится возвращаемым значением функции. Писать `return` не нужно, а если вы напишете — это будет ошибкой.

```yaoxiang
# Правильно: выражение напрямую становится возвращаемым значением
double: (x: Int) -> Int = x * 2

# Неправильно: писать return в форме выражения — синтаксическая ошибка
# double: (x: Int) -> Int = return x * 2   // ❌
```

## Форма блока кода: явный return

Когда функция содержит несколько шагов вычисления, оберните тело функции в блок `{ }`. **В блоке кода необходимо использовать оператор `return` для возврата значения**:

```yaoxiang
# Форма блока кода — нужно использовать return для возврата значения
factorial: (n: Int) -> Int = {
    if n <= 1 {
        return 1
    }
    return n * factorial(n - 1)
}

# Вычисление результата
f5 = factorial(5)        # f5 = 120
```

Правило простое: **форма выражения возвращает значение напрямую; форма блока кода требует явного `return`**. Если вы забыли написать `return` в блоке кода, функция по умолчанию вернёт `Void`.

```yaoxiang
# Внимание: в этой функции есть ошибка
# bad_add: (a: Int, b: Int) -> Int = {
#     a + b   # Нет return! Блок по умолчанию вернёт Void, но сигнатура требует Int → ошибка типа
# }

# Правильный вариант
good_add: (a: Int, b: Int) -> Int = {
    return a + b
}
```

Резюме:

| Форма | Синтаксис | Способ возврата значения |
|------|------|------------|
| Форма выражения | `name: ... = expr` | Значение выражения напрямую становится возвращаемым |
| Форма блока кода | `name: ... = { ... }` | Необходимо явно использовать `return` |

## Определение параметров

### Базовые параметры

Параметры записываются в сигнатуре функции, каждый параметр может быть аннотирован типом:

```yaoxiang
# Два параметра, оба с аннотацией типов
multiply: (a: Int, b: Int) -> Int = a * b
```

### Тип параметра должен быть указан в сигнатуре или в заголовке Lambda

Правило в YaoXiang таково: **при наличии входных параметров их типы должны быть явно указаны хотя бы в одном месте — в сигнатуре или в заголовке Lambda**. Если опустить типы в обоих местах, компилятор отклонит код.

```yaoxiang
# Способ 1: типы параметров в сигнатуре (заголовок Lambda опущен)
add: (a: Int, b: Int) -> Int = a + b

# Способ 2: типы параметров в заголовке Lambda (сигнатура опущена)
add = (a: Int, b: Int) => a + b

# Способ 3: полная форма (сигнатура + заголовок Lambda)
add: (a: Int, b: Int) -> Int = (a, b) => a + b

# Неправильно: типы не указаны нигде
# add = (a, b) => a + b   // ❌ Компилятор не может вывести типы параметров
```

**Рекомендуется использовать способ 1** — типы параметров записываются в сигнатуре, заголовок Lambda опускается. Это самый лаконичный и понятный вариант записи.

## Возвращаемое значение

Тип возвращаемого значения функции указывается после `->`. `->` — это маркер функционального типа, его нельзя опускать (без него выражение будет разобрано как другой тип).

```yaoxiang
# Возвращает Int
add_one: (x: Int) -> Int = x + 1

# Возвращает String
to_string: (n: Int) -> String = n.to_string()

# Возвращает Void (без возвращаемого значения)
log: (msg: String) -> Void = {
    println(msg)    # Без return, по умолчанию вернёт Void
}
```

Тип возвращаемого значения тоже можно опустить — за вас сработает вывод типов HM:

```yaoxiang
# Компилятор выводит тип возвращаемого значения как Int
add = (a: Int, b: Int) => a + b

# Компилятор выводит тип возвращаемого значения как String
greet = (name: String) => "Привет, " + name
```

## Вызов функций

### Позиционные аргументы

Самый базовый способ вызова — передача аргументов по порядку:

```yaoxiang
add: (a: Int, b: Int) -> Int = a + b

result = add(1, 2)        # result = 3
```

Формальное определение вызова функции в спецификации синтаксиса:

```
Expr '(' ArgList? ')'
```

Переводя на обычный язык: за выражением следуют круглые скобки, внутри которых может находиться список аргументов.

### Именованные аргументы

Помимо передачи аргументов по позиции, YaoXiang также поддерживает **именованные аргументы** — значение указывается по имени параметра, порядок не имеет значения:

```yaoxiang
# Именованные аргументы — после имени параметра идёт двоеточие, затем значение
result = add(a: 3, b: 5)     # result = 8
result = add(b: 5, a: 3)     # Порядок произвольный, результат тот же

# Можно смешивать с позиционными аргументами, но позиционные должны идти первыми
result = add(3, b: 5)        # OK
```

Именованные аргументы делают вызов более читаемым, что особенно полезно при большом количестве параметров:

```yaoxiang
# Сигнатура функции
send: (to: String, title: String, body: String) -> Void = {
    println("Кому: " + to)
    println("Заголовок: " + title)
    println("Тело: " + body)
}

# Именованные аргументы делают намерение вызова очевидным
send(
    to: "alice@example.com",
    title: "Уведомление о встрече",
    body: "Завтра в 3 часа дня собрание"
)
```

## Функции без параметров

Для функций без параметров список параметров можно опустить:

```yaoxiang
# Полная форма: явное объявление пустого списка параметров
hello: () -> Void = {
    println("Hello!")
}

# Кратчайшая форма: сигнатура опущена, компилятор автоматически выводит () -> Void
hello = {
    println("Hello!")
}

# Вызов функции без параметров
hello()
```

Функция `main` — самый распространённый пример функции без параметров:

```yaoxiang
# Несколько вариантов записи функции main

# Полная форма
main: () -> Void = {
    println("Hello, YaoXiang!")
}

# Кратчайшая форма (рекомендуется)
main = {
    println("Hello, YaoXiang!")
}
```

## Многострочные функции

Когда логика функции достаточно сложна, используйте форму блока кода для организации кода. YaoXiang требует отступ в 4 пробела:

```yaoxiang
# Многошаговое вычисление
calculate_stats: (numbers: List(Int)) -> Float = {
    # Объявление локальных переменных
    mut total = 0
    mut count = 0

    # Цикл для накопления
    for n in numbers {
        total = total + n
        count = count + 1
    }

    # Защита от деления на ноль
    if count == 0 {
        return 0.0
    }

    # Возврат среднего значения
    return total:as(Float) / count:as(Float)
}
```

В многострочных функциях можно писать комментарии с `#`, объявлять локальные переменные `mut`, использовать `for` и `if` для построения логики.

## pub и автоматическое связывание

В модуле функции, объявленные с ключевым словом `pub`, могут быть импортированы и использованы другими модулями. Что ещё интереснее: **`pub`-функции автоматически привязываются к типам, определённым в том же файле**, что позволяет вызывать их в ООП-стиле.

```yaoxiang
# point.yx

# Определение типа
type Point = { x: Float, y: Float }

# pub-функция: компилятор автоматически свяжет её с Point.distance
pub distance: (p1: Point, p2: Point) -> Float = {
    dx = p1.x - p2.x
    dy = p1.y - p2.y
    return (dx * dx + dy * dy).sqrt()
}

# Оба способа вызова работают
p1 = Point(x: 3.0, y: 4.0)
p2 = Point(x: 1.0, y: 2.0)

d1 = distance(p1, p2)       # Вызов в функциональном стиле
d2 = p1.distance(p2)        # Вызов в ООП-стиле (синтаксический сахар)
```

Когда компилятор видит `pub distance(p1: Point, p2: Point)` и обнаруживает, что `Point` определён в том же файле, он автоматически создаёт привязку `Point.distance`. Вам не нужно писать никакого дополнительного кода `impl`.

## Краткий справочник

```yaoxiang
# ── Сводка по синтаксису определения функций ──

# Форма выражения (самая распространённая)
add: (a: Int, b: Int) -> Int = a + b

# Форма блока кода (многошаговая логика)
factorial: (n: Int) -> Int = {
    if n <= 1 { return 1 }
    return n * factorial(n - 1)
}

# Функция без параметров (кратчайшая)
main = { println("Hello!") }

# С параметрами — сигнатура опущена
double = (x: Int) => x * 2

# С параметрами — заголовок Lambda опущен (рекомендуется)
triple: (x: Int) -> Int = x * 3

# pub-экспорт + автоматическое связывание
pub add: (a: Int, b: Int) -> Int = a + b

# ── Синтаксис вызова ──

result = add(1, 2)          # Позиционные аргументы
result = add(a: 1, b: 2)    # Именованные аргументы
result = add(1, b: 2)       # Смешанный вызов (позиционные первые)
```

## Резюме

Вы освоили основы функций в YaoXiang:

- **Единый синтаксис**: `name: (params) -> Return = body`, имеющий общую природу с объявлением переменной `name: type = value`
- **Форма выражения**: `= expr`, значение выражения напрямую становится возвращаемым, `return` не требуется
- **Форма блока кода**: `= { ...; return expr }`, внутри блока необходим явный `return`
- **Аннотация типов параметров**: типы должны быть указаны хотя бы в одном месте — в сигнатуре или в заголовке Lambda; рекомендуется указывать в сигнатуре
- **Вызов**: позиционные или именованные аргументы, порядок именованных произвольный
- **Автоматическое связывание pub**: `pub`-функции автоматически связываются с типами из того же файла, поддерживается вызов вида `obj.method()`
- **Кратчайшая запись без параметров**: `name = { ... }`, компилятор автоматически выводит `() -> Void`

Дальше вы можете перейти к изучению главы [Поток управления](./control-flow.md), чтобы узнать, как использовать `if`, `for` и `while` внутри функций.
