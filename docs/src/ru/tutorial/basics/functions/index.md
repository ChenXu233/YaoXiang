# Определение и вызов функций

В предыдущей главе вы научились объявлять переменные. Эта глава познакомит вас с ядром YaoXiang — функциями. Синтаксис функций YaoXiang разделяет ту же модель `name: type = value` с объявлением переменных, так что вы почувствуете знакомые черты.

## Функция — это Lambda

Сначала самое важное: **в YaoXiang функция по сути является лямбда-выражением**. Никакого специального ключевого слова `fn`, никаких сложных церемоний. Определить функцию — значит дать лямбде имя.

```
# Любая функция по сути представляет собой комбинацию этих четырёх элементов:
name: (params) -> Return = body
 ^       ^        ^        ^
 |       |        |        +-- Тело функции (лямбда-выражение или блок кода)
 |       |        +-- Тип возвращаемого значения
 |       +-- Список параметров (сигнатура)
 +-- Имя функции
```

Это полностью соответствует изученной в предыдущей главе модели `name: type = value` — просто здесь «типом» оказывается функциональный тип.

## Форма выражения: прямое возвращение значения

В самой простой функции ключевое слово `return` не требуется. Когда тело функции — это одно выражение, оно напрямую становится возвращаемым значением:

```yaoxiang
// Форма выражения — прямое возвращение значения, без return
add: (a: Int, b: Int) -> Int = a + b
square: (x: Int) -> Int = x * x
greet: (name: String) -> String = "Привет, " + name
```

Вызовем их:

```yaoxiang
sum = add(3, 5)          // sum = 8
sq = square(4)           // sq = 16
msg = greet("мир")       // msg = "Привет, мир"
```

Это называется **формой выражения**. Когда тело функции является выражением (а не блоком `{ }`), его значение напрямую становится возвращаемым значением функции. Писать `return` не нужно, а если написать — это будет ошибкой.

```yaoxiang
// Верно: выражение напрямую становится возвращаемым значением
double: (x: Int) -> Int = x * 2

// Ошибка: писать return в форме выражения — синтаксическая ошибка
// double: (x: Int) -> Int = return x * 2   // ❌
```

---

## Форма блока кода: явный return

Когда функция содержит несколько шагов вычислений, оберните тело функции в блок `{ }`. **Внутри блока для возврата значения необходимо использовать оператор `return`**:

```yaoxiang
// Форма блока кода — возвращаемое значение должно быть через return
factorial: (n: Int) -> Int = {
    if n <= 1 {
        return 1
    }
    return n * factorial(n - 1)
}

// Вычисляем результат
f5 = factorial(5)        // f5 = 120
```

Правило простое: **в форме выражения значение возвращается напрямую; в форме блока кода требуется явный `return`**. Если в блоке кода забыть написать `return`, функция по умолчанию вернёт `Void`.

```yaoxiang
// Внимание: в этой функции есть баг
// bad_add: (a: Int, b: Int) -> Int = {
//     a + b   // Нет return! Блок по умолчанию возвращает Void, но сигнатура требует Int → ошибка типа
// }

// Правильная запись
good_add: (a: Int, b: Int) -> Int = {
    return a + b
}
```

Резюме:

| Форма | Синтаксис | Способ возврата значения |
|------|------|------------|
| Форма выражения | `name: ... = expr` | Значение выражения напрямую становится возвращаемым |
| Форма блока кода | `name: ... = { ... }` | Необходим явный `return` |

## Определение параметров

### Базовые параметры

Параметры записываются в сигнатуре функции, каждый параметр можно аннотировать типом:

```yaoxiang
// Два параметра, оба аннотированы типами
multiply: (a: Int, b: Int) -> Int = a * b
```

### Тип параметра должен быть указан в сигнатуре или в заголовке Lambda

Правило YaoXiang таково: **при наличии входных параметров их типы должны быть явно указаны как минимум в одном из мест — в сигнатуре или в заголовке Lambda**. Пропуск типов в обоих местах будет отвергнут компилятором.

```yaoxiang
// Способ 1: типы параметров в сигнатуре (заголовок Lambda опущен)
add: (a: Int, b: Int) -> Int = a + b

// Способ 2: типы параметров в заголовке Lambda (сигнатура опущена)
add = (a: Int, b: Int) => a + b

// Способ 3: полная форма (сигнатура + заголовок Lambda)
add: (a: Int, b: Int) -> Int = (a, b) => a + b

// Ошибка: типы не указаны ни в одном из мест
// add = (a, b) => a + b   // ❌ Компилятор не может вывести типы параметров
```

**Рекомендуется использовать способ 1** — типы параметров в сигнатуре, заголовок Lambda опускается. Это самая лаконичная и ясная запись.

## Возвращаемое значение

Тип возвращаемого значения функции записывается после `->`. `->` — это маркер функционального типа, его нельзя опускать (при пропуске он будет разобран как другой тип).

```yaoxiang
// Возвращает Int
add_one: (x: Int) -> Int = x + 1

// Возвращает String
to_string: (n: Int) -> String = n.to_string()

// Возвращает Void (без возвращаемого значения)
log: (msg: String) -> Void = {
    print(msg)    // Без return, по умолчанию возвращается Void
}
```

Тип возвращаемого значения также можно опустить и доверить вывод типов HM:

```yaoxiang
// Компилятор выводит, что тип возврата — Int
add = (a: Int, b: Int) => a + b

// Компилятор выводит, что тип возврата — String
greet = (name: String) => "Привет, " + name
```

## Вызов функций

### Позиционные аргументы

Самый базовый способ вызова — передача аргументов по порядку:

```yaoxiang
add: (a: Int, b: Int) -> Int = a + b

result = add(1, 2)        // result = 3
```

Определение формы вызова функции в спецификации синтаксиса:

```
Expr '(' ArgList? ')'
```

Переводя на повседневный язык: после выражения идёт пара круглых скобок, внутри которых может располагаться список аргументов.

### Именованные аргументы

Помимо передачи аргументов по позиции, YaoXiang также поддерживает **именованные аргументы** — указание значений по имени параметра в любом порядке:

```yaoxiang
// Именованные аргументы — после имени параметра идёт двоеточие, затем значение
result = add(a: 3, b: 5)     // result = 8
result = add(b: 5, a: 3)     // Порядок произвольный, результат тот же

// Можно смешивать с позиционными аргументами, но позиционные должны идти первыми
result = add(3, b: 5)        // OK
```

Именованные аргументы делают вызов более читаемым, что особенно полезно при большом количестве параметров:

```yaoxiang
// Сигнатура функции
send: (to: String, title: String, body: String) -> Void = {
    print("Кому: " + to)
    print("Заголовок: " + title)
    print("Тело: " + body)
}

// Именованные аргументы делают намерение вызова очевидным
send(
    to: "alice@example.com",
    title: "Уведомление о встрече",
    body: "Завтра в 3 часа дня встреча"
)
```

## Функции без параметров

Для функций без параметров список параметров можно опустить:

```yaoxiang
// Полная форма: явное объявление пустого списка параметров
hello: () -> Void = {
    print("Hello!")
}

// Самая краткая форма: сигнатура опущена, компилятор автоматически выводит () -> Void
hello = {
    print("Hello!")
}

// Вызов функции без параметров
hello()
```

Функция `main` — самый распространённый пример функции без параметров:

```yaoxiang
// Несколько вариантов записи функции main

// Полная форма
main: () -> Void = {
    print("Hello, YaoXiang!")
}

// Самая краткая форма (рекомендуется)
main = {
    print("Hello, YaoXiang!")
}
```

## Многострочные функции

Когда логика функции достаточно сложна, используйте форму блока кода для организации кода. YaoXiang требует отступ в 4 пробела:

```yaoxiang
// Многошаговое вычисление
calculate_stats: (numbers: List(Int)) -> Float = {
    // Объявление локальных переменных
    mut total = 0
    mut count = 0

    // Цикл накопления
    for n in numbers {
        total = total + n
        count = count + 1
    }

    // Защита от деления на ноль
    if count == 0 {
        return 0.0
    }

    // Возврат среднего значения
    return total:as(Float) / count:as(Float)
}
```

В многострочных функциях можно писать комментарии с помощью `#`, объявлять локальные переменные с `mut`, использовать `for` и `if` для построения логики.

## pub и автоматическая привязка

В модуле функции, объявленные с ключевым словом `pub`, могут импортироваться и использоваться другими модулями. Что ещё интереснее, **функции `pub` автоматически привязываются к типам, определённым в том же файле**, что позволяет вызывать их в ООП-стиле.

```yaoxiang
// point.yx

// Определение типа
Point: Type = { x: Float, y: Float }

// pub-функция: компилятор автоматически привяжет её как Point.distance
pub distance: (p1: Point, p2: Point) -> Float = {
    dx = p1.x - p2.x
    dy = p1.y - p2.y
    return (dx * dx + dy * dy).sqrt()
}

// Оба способа вызова работают
p1 = Point(3.0, 4.0)
p2 = Point(1.0, 2.0)

d1 = distance(p1, p2)       // Вызов в функциональном стиле
d2 = p1.distance(p2)        // Вызов в ООП-стиле (синтаксический сахар)
```

Когда компилятор видит `pub distance(p1: Point, p2: Point)` и обнаруживает, что `Point` определён в том же файле, он автоматически создаёт привязку `Point.distance`. Вам не нужно писать никакого дополнительного кода `impl`.

## Краткий справочник

```yaoxiang
// ── Сводка синтаксиса определения функций ──

// Форма выражения (самая распространённая)
add: (a: Int, b: Int) -> Int = a + b

// Форма блока кода (многошаговая логика)
factorial: (n: Int) -> Int = {
    if n <= 1 { return 1 }
    return n * factorial(n - 1)
}

// Функция без параметров (самая краткая)
main = { print("Hello!") }

// С параметрами — сигнатура опущена
double = (x: Int) => x * 2

// С параметрами — заголовок Lambda опущен (рекомендуется)
triple: (x: Int) -> Int = x * 3

// pub-экспорт + автоматическая привязка
pub add: (a: Int, b: Int) -> Int = a + b

// ── Синтаксис вызова ──

result = add(1, 2)          // Позиционные аргументы
result = add(a: 1, b: 2)    // Именованные аргументы
result = add(1, b: 2)       // Смешанный вызов (позиционные первыми)
```

## Резюме

Вы освоили основы функций YaoXiang:

- **Единый синтаксис**: `name: (params) -> Return = body` — того же происхождения, что и `name: type = value` для объявления переменных
- **Форма выражения**: `= expr`, значение выражения напрямую становится возвращаемым, `return` не нужен
- **Форма блока кода**: `= { ...; return expr }`, внутри блока требуется явный `return`
- **Аннотация типов параметров**: типы должны быть указаны как минимум в одном из мест — в сигнатуре или в заголовке Lambda; рекомендуется указывать в сигнатуре
- **Вызов**: позиционные или именованные аргументы, именованные аргументы можно передавать в любом порядке
- **Автоматическая привязка pub**: функции `pub` автоматически привязываются к типам того же файла, поддерживая вызов вида `obj.method()`
- **Самая краткая запись без параметров**: `name = { ... }`, компилятор автоматически выводит `() -> Void`

Дальше вы можете перейти к главе [Поток управления](./control-flow.md), чтобы узнать, как использовать `if`, `for` и `while` внутри функций.
