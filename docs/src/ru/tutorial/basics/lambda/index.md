# Lambda-выражения

Lambda — это **анонимные функции, которые можно определить на лету**. В YaoXiang обычные функции по своей сути являются именованными Lambda.

## Синтаксис

Согласно спецификации синтаксиса:

```
Lambda      ::= '(' ParamList? ')' '=>' Expr
            |  '(' ParamList? ')' '=>' Block
```

Простейшая Lambda:

```yaoxiang
# Lambda в форме выражения
double = (x) => x * 2

println(double(5))   # 10
println(double(10))  # 20
```

## Единство Lambda и функций

Основная философия дизайна YaoXiang — единый синтаксис. **Функция — это Lambda, привязанная к имени**:

```yaoxiang
# Эти две записи полностью эквивалентны:

# Форма Lambda
add = (a, b) => a + b

# Форма функции (синтаксический сахар)
add: (a: Int, b: Int) -> Int = a + b
```

Первая строка — «присвоить Lambda переменной `add`», вторая — «определить функцию с именем `add`». Компилятор обрабатывает их практически одинаково.

## Когда использовать Lambda

Lambda лучше всего подходит для двух сценариев:

### 1. Функции высшего порядка — передача функций в качестве аргументов

```yaoxiang
# Применить операцию к каждому элементу списка
apply_to_all: (list: List(Int), op: (Int) -> Int) -> List(Int) = {
    mut result = []
    for item in list {
        result.append(op(item))
    }
    return result
}

numbers = [1, 2, 3, 4, 5]

# Передаём Lambda
doubled = apply_to_all(numbers, (x) => x * 2)
squared = apply_to_all(numbers, (x) => x * x)

println(doubled)  # [2, 4, 6, 8, 10]
println(squared)  # [1, 4, 9, 16, 25]
```

### 2. Временные одноразовые операции

Нет необходимости определять отдельную функцию для логики, которая используется только один раз:

```yaoxiang
# Сортировка — временно определяем правило сортировки
students = [
    {"name": "Alice", "score": 90},
    {"name": "Bob", "score": 85},
    {"name": "Charlie", "score": 92},
]

sorted_students = students.sort_by((a, b) => a["score"].compare(b["score"]))
```

## Блочная форма Lambda

Когда Lambda требует многострочной логики, используется блочная форма:

```yaoxiang
# Блочная Lambda: может содержать несколько инструкций
process = (data) => {
    cleaned = data.trim()
    lower = cleaned.lowercase()
    return lower
}

result = process("  Hello World  ")
println(result)  # "hello world"
```

Обратите внимание, что блочная форма требует использования `return` для возврата значения — это полностью совпадает с функциями.

## Lambda с несколькими параметрами

```yaoxiang
# Три параметра
add_three = (x, y, z) => x + y + z
println(add_three(1, 2, 3))  # 6

# Lambda без параметров
greet = () => "Hello, YaoXiang!"
println(greet())  # "Hello, YaoXiang!"
```

## Вывод типов

Типы параметров Lambda могут выводиться из контекста:

```yaoxiang
# Тип выводится из контекста использования — не нужно писать (x: Int) => x * 2
apply: (op: (Int) -> Int, value: Int) -> Int = op(value)

result = apply((x) => x + 10, 5)
println(result)  # 15
```

Компилятор знает, что тип `op` — `(Int) -> Int`, поэтому `x` в Lambda `(x) => x + 10` автоматически выводится как `Int`.

> **Примечание**: согласно правилам определения функций, типы параметров должны быть указаны как минимум в одном месте: в сигнатуре или в заголовке Lambda. Когда Lambda передаётся в качестве аргумента, тип обычно задаётся сигнатурой принимающей стороны.

## Резюме

| Суть | Описание |
|------|------|
| Синтаксис | `(params) => expr` или `(params) => { return ... }` |
| Суть | Функция = именованная Lambda |
| Функции высшего порядка | Lambda можно передавать в качестве аргумента |
| Блочная форма | Многострочная логика использует `{}` + `return` |
| Вывод типов | Типы параметров автоматически выводятся из контекста |

Lambda — это самый лаконичный способ выразить «временную логику» в YaoXiang. Овладев ею, вы сделаете свой код более гибким и компактным.
