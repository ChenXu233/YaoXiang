# Лямбда-выражения

Лямбда — это **анонимная функция, которую можно определить на лету**. В YaoXiang обычные функции по сути являются именованными лямбдами.

## Синтаксис

Согласно спецификации синтаксиса:

```
Lambda      ::= '(' ParamList? ')' '=>' Expr
            |  '(' ParamList? ')' '=>' Block
```

Простейшая лямбда:

```yaoxiang
// Лямбда в форме выражения
double = (x) => x * 2

print(double(5))   // 10
print(double(10))  // 20
```

## Единство лямбды и функции

Основная философия дизайна YaoXiang — единый синтаксис. **Функция — это лямбда, привязанная к имени**:

```yaoxiang
// Эти два варианта полностью эквивалентны:

// Лямбда-форма
add = (a, b) => a + b

// Форма функции (синтаксический сахар)
add: (a: Int, b: Int) -> Int = a + b
```

Первая строка — это «присвоить лямбду переменной `add`», вторая — «определить функцию с именем `add`». Компилятор обрабатывает их практически одинаково.

## Когда использовать лямбду

Лямбда лучше всего подходит для двух сценариев:

### 1. Функции высшего порядка — передача функции в качестве аргумента

```yaoxiang
// Применить операцию к каждому элементу списка
apply_to_all: (list: List(Int), op: (Int) -> Int) -> List(Int) = {
    mut result = []
    for item in list {
        result.append(op(item))
    }
    return result
}

numbers = [1, 2, 3, 4, 5]

// Передаём лямбду
doubled = apply_to_all(numbers, (x) => x * 2)
squared = apply_to_all(numbers, (x) => x * x)

print(doubled)  // [2, 4, 6, 8, 10]
print(squared)  // [1, 4, 9, 16, 25]
```

### 2. Временная одноразовая операция

Нет необходимости определять отдельную функцию ради логики, которая используется только один раз:

```yaoxiang
// Сортировка — задаём правило сравнения на лету
students = [
    {"name": "Alice", "score": 90},
    {"name": "Bob", "score": 85},
    {"name": "Charlie", "score": 92},
]

sorted_students = students.sort_by((a, b) => a["score"].compare(b["score"]))
```

## Лямбда в форме блока

Когда лямбда требует многострочной логики, используется блочная форма:

```yaoxiang
// Лямбда-блок: может содержать несколько операторов
process = (data) => {
    cleaned = data.trim()
    lower = cleaned.lowercase()
    return lower
}

result = process("  Hello World  ")
print(result)  // "hello world"
```

Обратите внимание, что в блочной форме для возврата значения необходимо использовать `return`, — это полностью совпадает с поведением функций.

## Лямбда с несколькими параметрами

```yaoxiang
// Три параметра
add_three = (x, y, z) => x + y + z
print(add_three(1, 2, 3))  // 6

// Лямбда без параметров
greet = () => "Hello, YaoXiang!"
print(greet())  // "Hello, YaoXiang!"
```

## Вывод типов

Типы параметров лямбды могут выводиться из контекста:

```yaoxiang
// Тип выводится из места использования — не нужно писать (x: Int) => x * 2
apply: (op: (Int) -> Int, value: Int) -> Int = op(value)

result = apply((x) => x + 10, 5)
print(result)  // 15
```

Компилятор знает, что тип `op` — это `(Int) -> Int`, поэтому `x` в лямбде `(x) => x + 10` автоматически выводится как `Int`.

> **Примечание**: согласно правилам определения функций, типы параметров должны быть указаны как минимум в одном месте — в сигнатуре или в заголовке лямбды. Когда лямбда передаётся в качестве аргумента, тип обычно задаётся сигнатурой принимающей стороны.

## Резюме

| Пункт | Описание |
|------|------|
| Синтаксис | `(params) => expr` или `(params) => { return ... }` |
| Суть | Функция = именованная лямбда |
| Функции высшего порядка | Лямбду можно передавать в качестве аргумента |
| Блочная форма | Для многострочной логики используйте `{}` + `return` |
| Вывод типов | Типы параметров автоматически выводятся из контекста |

Лямбда — это самый лаконичный способ выразить «временную логику» в YaoXiang. Освойте её, и ваш код станет гибче и компактнее.
