# Объявление переменных

В этой главе рассматривается основной синтаксис объявления переменных в YaoXiang. Если у вас есть опыт работы с другими языками программирования, вы обнаружите, что система переменных в YaoXiang очень лаконична — все объявления используют единую синтаксическую модель.

## Единая синтаксическая модель

Философия дизайна YaoXiang — «всё едино». Будь то объявление целого числа, определение функции или создание типа — для всего используется один и тот же синтаксис:

```
name: type = value
```

Это ключевая идея дизайна YaoXiang. Несколько примеров помогут прочувствовать эту согласованность:

```yaoxiang
// Объявление переменной
x: Int = 42
name: String = "YaoXiang"

// Определение функции
add: (a: Int, b: Int) -> Int = a + b

// Определение типа
Point: Type = { x: Float, y: Float }
```

Формальное определение объявления переменной в спецификации синтаксиса:

```
('mut')? Identifier (':' TypeExpr)? '=' Expr
```

Простыми словами: необязательное ключевое слово `mut`, затем имя переменной, затем необязательное `: тип`, и в конце `= значение`. Эта структура пронизывает весь язык — достаточно выучить её один раз.

## Неизменяемые переменные (поведение по умолчанию)

В YaoXiang все переменные **по умолчанию неизменяемы**. После присваивания значение изменить нельзя. Это решение в дизайне языка направлено на безопасность.

```yaoxiang
x = 10
// x = 20   // Ошибка компиляции! x — неизменяемая переменная
```

Когда переменная объявляется с помощью `=`, компилятор ищет переменную с таким же именем по цепочке областей видимости наружу. Если находит — присваивает ей значение; если не находит — создаёт новую неизменяемую переменную в текущей области видимости.

```yaoxiang
x = 1       // Во внешней области видимости нет x, поэтому объявляется новая переменная
x = 2       // Найден внешний x, выполняется попытка присваивания → Ошибка компиляции! x — неизменяемая
```

Это может показаться контринтуитивным — если вы изучали другие языки, у вас может возникнуть вопрос: «Раз нашли, почему нельзя присвоить?». Причина в том, что YaoXiang ставит безопасность на первое место: неизменяемость по умолчанию означает, что вам не нужно беспокоиться о том, что какая-то переменная будет случайно изменена в каком-нибудь уголке кода.

## mut — изменяемые переменные

Когда вам действительно нужно изменять переменную, используйте ключевое слово `mut` для явного объявления:

```yaoxiang
mut counter = 0
counter = counter + 1   // Можно изменять
counter = 100           // Тоже можно
```

У `mut` есть несколько важных правил:

**Правило первое**: `mut` — это явное новое объявление, компилятор не выполняет поиск переменной с таким же именем во внешней области видимости.

```yaoxiang
mut x = 10      // В текущей области видимости создаётся новая изменяемая переменная x
mut x = 20      // Ошибка компиляции! В той же области видимости x уже объявлен
```

**Правило второе**: переменная, объявленная через `mut`, не может иметь то же имя, что и переменная во внешней области видимости (запрет затенения).

```yaoxiang
x = 10
{
    mut x = 20   // Ошибка компиляции! x уже объявлен во внешней области, затенение запрещено
}
```

**Правило третье**: в пределах одной области видимости каждое имя может быть объявлено только один раз — независимо от того, используется `=` или `mut`.

```yaoxiang
x = 10
mut x = 20   // Ошибка компиляции! x уже объявлен
```

Эти правила гарантируют уникальность каждого имени переменной в текущей области видимости, и вы никогда не столкнётесь с путаницей в том, на какую именно переменную ссылается одно и то же имя.

## Вывод типов против явных аннотаций типов

YaoXiang использует алгоритм вывода типов Хиндли-Милнера (HM). Компилятор может автоматически выводить типы на основе записанных вами значений, и в большинстве случаев указывать тип вручную не требуется.

```yaoxiang
x = 42              // Компилятор выводит Int
name = "YaoXiang"   // Выводится String
pi = 3.14159        // Выводится Float
is_valid = true     // Выводится Bool
```

Когда вы хотите явно указать тип (например, для повышения читаемости кода или когда компилятор не может вывести тип), используйте синтаксис `: Type`:

```yaoxiang
count: Int = 100
greeting: String = "Hello"
ratio: Float = 0.618
```

Оба способа записи полностью эквивалентны. Можно начать писать код, опуская типы, и добавлять аннотации типов по мере необходимости. Это делает прототипирование очень быстрым, при этом не жертвуя типобезопасностью финального кода.

## Обзор базовых типов

YaoXiang имеет несколько встроенных базовых типов, покрывающих подавляющее большинство повседневных задач программирования.

### Int (целое число)

```yaoxiang
a = 42              // Десятичное
b = 0o52            // Восьмеричное (префикс 0o)
c = 0x2A            // Шестнадцатеричное (префикс 0x)
d = 0b101010        // Двоичное (префикс 0b)
e = 1_000_000       // Подчёркивание можно использовать как разделитель для удобства чтения
```

### Float (число с плавающей точкой)

```yaoxiang
pi = 3.14159
speed = 2.998e8         // Экспоненциальная запись: 2.998 × 10^8
tiny = 1.6e-19
```

### String (строка)

```yaoxiang
name = "YaoXiang"
empty = ""              // Пустая строка
escape = "Hello\nWorld" // Поддержка escape-последовательностей: \n — перевод строки, \t — табуляция, \\ — обратный слэш, \" — двойная кавычка
unicode = "\u{4F60}\u{597D}"  // Unicode escape-последовательности
```

### Bool (логический тип)

```yaoxiang
is_ready = true
is_done = false
```

Логические значения обычно используются в условных конструкциях:

```yaoxiang
if is_ready {
    print("Начинаем обработку")
}
```

## Область видимости переменных

Область видимости определяет диапазон, в котором переменная доступна. Правила областей видимости в YaoXiang очень просты: **каждый блок `{}` создаёт новую область видимости**.

### Основные правила

```yaoxiang
{
    x = 10
    print(x)   // Доступ есть: x находится в текущей области видимости
}
// print(x)    // Ошибка: x не виден за пределами области видимости
```

Внутренняя область видимости имеет доступ к переменным внешней:

```yaoxiang
outer = "Я снаружи"
{
    print(outer)   // Доступ к внешней переменной outer
    inner = "Я внутри"
}
// print(inner)    // Ошибка: inner не виден за пределами области видимости
```

### Область видимости параметров функции

```yaoxiang
greet: (name: String) -> Void = {
    print("Привет, " + name)
    // name виден внутри тела этой функции
}
// name не виден за пределами функции
```

### Блочные выражения

Блок `{}` в YaoXiang также является выражением и может возвращать значение:

```yaoxiang
result = {
    x = 10
    y = 20
    return x + y   // Возвращает 30 во внешнюю область видимости
}
// Значение result равно 30
// x и y не видны за пределами блока
```

Подробное описание возврата значений из блоков приведено в последующей главе о функциях. Здесь достаточно запомнить: **фигурные скобки создают область видимости, внутренний код видит внешний, внешний — не видит внутренний**.

## Итоги

Вы освоили ключевые концепции системы переменных YaoXiang:

| Концепция | Основные моменты |
|------|------|
| Единая синтаксическая модель | `name: type = value`, используется для переменных, функций и типов |
| Неизменяемость по умолчанию | После `x = 10` значение `x` нельзя изменить |
| Изменяемые переменные | Явно объявляются через `mut`: `mut x = 10` |
| Запрет затенения | В пределах одной области видимости имя можно объявить только один раз |
| Вывод типов | Алгоритм HM выполняет автоматический вывод, либо можно явно указать `: Type` |
| Область видимости | Каждый `{}` создаёт область видимости: внутренний код видит внешний, внешний — не видит внутренний |
