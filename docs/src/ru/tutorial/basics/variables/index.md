# Объявление переменных

В этой главе рассматривается основной синтаксис объявления переменных в YaoXiang. Если у вас есть опыт работы с другими языками программирования, вы обнаружите, что система переменных в YaoXiang очень лаконична — все объявления используют единую синтаксическую модель.

## Единая синтаксическая модель

Философия дизайна YaoXiang — «всё едино». Будь то объявление целого числа, определение функции или создание типа — везде используется один и тот же синтаксис:

```
name: type = value
```

Это самая ключевая идея дизайна YaoXiang. Несколько примеров помогут ощутить эту согласованность:

```yaoxiang
# Объявление переменной
x: Int = 42
name: String = "YaoXiang"

# Определение функции
add: (a: Int, b: Int) -> Int = a + b

# Определение типа
type Point = { x: Float, y: Float }
```

Формальное определение объявления переменной в спецификации синтаксиса:

```
('mut')? Identifier (':' TypeExpr)? '=' Expr
```

Переводя на человеческий язык: может идти необязательное ключевое слово `mut`, затем имя переменной, затем необязательное `: Тип`, и в конце `= значение`. Эта структура пронизывает весь язык — достаточно выучить её один раз.

## Неизменяемые переменные (поведение по умолчанию)

В YaoXiang все переменные **по умолчанию неизменяемые**. После присваивания значение нельзя изменить. Это элемент безопасности языка.

```yaoxiang
x = 10
# x = 20   // Ошибка компиляции! x является неизменяемой
```

Для переменных, объявленных через `=`, компилятор выполняет поиск переменной с тем же именем по цепочке областей видимости наружу. Если переменная найдена, выполняется попытка присваивания; если не найдена — в текущей области видимости создаётся новая неизменяемая переменная.

```yaoxiang
x = 1       // Во внешней области видимости нет x, поэтому объявляется новая переменная
x = 2       // Найден внешний x, выполняется попытка присваивания → Ошибка компиляции! x неизменяем
```

Это может показаться контринтуитивным — если вы изучали другие языки, у вас может возникнуть мысль: «Раз переменная найдена, почему нельзя присвоить?». Это потому, что YaoXiang ставит безопасность на первое место: неизменяемость по умолчанию означает, что вам не нужно беспокоиться о случайном изменении какой-либо переменной в каком-нибудь уголке кода.

## mut — изменяемые переменные

Когда вам действительно нужно изменять переменную, явно объявите её с помощью ключевого слова `mut`:

```yaoxiang
mut counter = 0
counter = counter + 1   # Можно изменять
counter = 100           # Тоже можно
```

У `mut` есть несколько важных правил:

**Правило первое**: `mut` — это явное новое объявление, компилятор не выполняет поиск переменной с тем же именем во внешней области видимости.

```yaoxiang
mut x = 10      # Создаётся новая изменяемая переменная x в текущей области видимости
mut x = 20      # Ошибка компиляции! x уже объявлен в этой области видимости
```

**Правило второе**: переменная, объявленная через `mut`, не может иметь то же имя, что и переменная во внешней области видимости (запрещено затенение).

```yaoxiang
x = 10
{
    mut x = 20   # Ошибка компиляции! x уже объявлен во внешней области, затенение запрещено
}
```

**Правило третье**: в одной области видимости каждое имя может быть объявлено только один раз — независимо от того, используется `=` или `mut`.

```yaoxiang
x = 10
mut x = 20   # Ошибка компиляции! x уже объявлен
```

Эти правила гарантируют, что каждое имя переменной в текущей области видимости уникально, и вы никогда не столкнётесь с путаницей относительно того, на какую именно переменную ссылается одно и то же имя.

## Вывод типов против явных аннотаций типов

YaoXiang использует алгоритм вывода типов Хиндли-Милнера (HM). Компилятор может автоматически вывести тип из записанного вами значения, и в большинстве случаев указывать тип вручную не требуется.

```yaoxiang
x = 42              # Компилятор выводит Int
name = "YaoXiang"   # Выводится String
pi = 3.14159        # Выводится Float
is_valid = true     # Выводится Bool
```

Когда вы хотите явно указать тип (например, для повышения читаемости кода или когда компилятор не может вывести тип), используйте синтаксис `: Type`:

```yaoxiang
count: Int = 100
greeting: String = "Hello"
ratio: Float = 0.618
```

Обе записи полностью эквивалентны. Вы можете начать писать код, опуская типы, и при необходимости добавить аннотации типов позже. Это делает прототипирование очень быстрым, не жертвуя при этом типобезопасностью финального кода.

## Обзор основных типов

YaoXiang имеет несколько встроенных базовых типов, которые покрывают подавляющее большинство повседневных задач программирования.

### Int (целое число)

```yaoxiang
a = 42              # Десятичное
b = 0o52            # Восьмеричное (префикс 0o)
c = 0x2A            # Шестнадцатеричное (префикс 0x)
d = 0b101010        # Двоичное (префикс 0b)
e = 1_000_000       # Можно разделять цифры подчёркиванием для удобства чтения
```

### Float (число с плавающей точкой)

```yaoxiang
pi = 3.14159
speed = 2.998e8         # Экспоненциальная запись: 2,998 × 10^8
tiny = 1.6e-19
```

### String (строка)

```yaoxiang
name = "YaoXiang"
empty = ""              # Пустая строка
escape = "Hello\nWorld" # Поддержка экранирования: \n — перевод строки, \t — табуляция, \\ — обратный слэш, \" — двойная кавычка
unicode = "\u{4F60}\u{597D}"  # Экранирование Unicode
```

### Bool (логический тип)

```yaoxiang
is_ready = true
is_done = false
```

Логические значения обычно используются в условных проверках:

```yaoxiang
if is_ready {
    println("Начинаем обработку")
}
```

## Область видимости переменных

Область видимости определяет диапазон, в котором переменная доступна. Правила областей видимости в YaoXiang очень просты: **каждый блок `{}` создаёт новую область видимости**.

### Базовые правила

```yaoxiang
{
    x = 10
    println(x)   # Доступ есть: x находится в текущей области видимости
}
# println(x)    # Ошибка: x не виден за пределами области видимости
```

Внутренняя область имеет доступ к переменным внешней области:

```yaoxiang
outer = "Я снаружи"
{
    println(outer)   # Доступ к внешней переменной outer есть
    inner = "Я внутри"
}
# println(inner)    # Ошибка: inner не виден за пределами области видимости
```

### Область видимости параметров функции

```yaoxiang
greet: (name: String) -> Void = {
    println("Привет, " + name)
    # name виден внутри тела этой функции
}
# name не виден за пределами функции
```

### Блочные выражения

Блок `{}` в YaoXiang также является выражением и может возвращать значение:

```yaoxiang
result = {
    x = 10
    y = 20
    return x + y   # Возвращается 30 во внешнюю область видимости
}
# Значение result равно 30
# x и y не видны за пределами блока
```

Подробное описание возврата значений из блоков приведено в последующей главе о функциях. Здесь нужно запомнить только одно: **фигурные скобки создают область видимости, внутренняя область видит внешнюю, внешняя не видит внутреннюю**.

## Резюме

Вы освоили основные концепции системы переменных YaoXiang:

| Концепция | Ключевые моменты |
|------|------|
| Единая синтаксическая модель | `name: type = value`, используется для переменных, функций и типов |
| Неизменяемость по умолчанию | После `x = 10` значение `x` нельзя изменить |
| Изменяемые переменные | Явно объявляются через `mut`: `mut x = 10` |
| Запрет затенения | В одной области видимости имя может быть объявлено только один раз |
| Вывод типов | Алгоритм HM выполняет вывод автоматически, либо можно явно указать `: Type` |
| Область видимости | Каждый `{}` создаёт область видимости: внутренняя видит внешнюю, внешняя не видит внутреннюю |
