# Сопоставление шаблонов

В главе [основы match](../control-flow/match.md) вы изучили базовое использование `match` — литералы, идентификаторы, подстановочный знак. Теперь мы углубимся во все возможности сопоставления шаблонов в YaoXiang.

## Полный набор типов шаблонов

Согласно спецификации синтаксиса, полное определение `Pattern` выглядит так:

```
Pattern     ::= Literal       # шаблон литерала: 42, "hello"
            | Identifier      # шаблон идентификатора: захватывает значение
            | Wildcard        # подстановочный знак: _
            | StructPattern   # шаблон структуры: деструктурирует запись
            | TuplePattern    # шаблон кортежа: деструктурирует кортеж
            | EnumPattern     # шаблон перечисления: деструктурирует вариант
            | OrPattern       # шаблон «или»: шаблон1 | шаблон2
```

Первые три базовых шаблона вы уже изучили в предыдущей главе. Эта глава посвящена четырём продвинутым шаблонам.

## Шаблон перечисления

Шаблон перечисления — наиболее часто используемая продвинутая возможность `match`. Он позволяет деструктурировать варианты перечисления и извлекать внутренние данные.

### Базовое сопоставление перечислений

```yaoxiang
# Определение типа Result
Result: (T: Type, E: Type) -> Type = ok(T) | err(E)

# Функция обрабатывает Result с помощью match
handle: (result: Result(Int, String)) -> String = match result {
    ok(value) => "Успех! Полученное значение: ${value}",
    err(msg) => "Ошибка: ${msg}",
}

a = ok(42)
b = err("Таймаут соединения")

println(handle(a))  # Успех! Полученное значение: 42
println(handle(b))  # Ошибка: Таймаут соединения
```

### Тип Option

```yaoxiang
# Использование Option вместо null
# Встроенный тип: Option: (T: Type) -> Type = some(T) | none

describe: (opt: Option(Int)) -> String = match opt {
    some(n) => "Есть значение: ${n}",
    none => "Ничего нет",
}

println(describe(some(100)))  # Есть значение: 100
println(describe(none))       # Ничего нет
```

### Пользовательское перечисление

```yaoxiang
# Определение перечисления цветов
type Color = red | green | blue | rgb(Int, Int, Int)

to_hex: (c: Color) -> String = match c {
    red => "#FF0000",
    green => "#00FF00",
    blue => "#0000FF",
    rgb(r, g, b) => "#${r.to_hex()}${g.to_hex()}${b.to_hex()}",
}

println(to_hex(red))                # #FF0000
println(to_hex(rgb(128, 128, 128))) # #808080
```

В `rgb(r, g, b)` идентификаторы `r`, `g`, `b` — это шаблоны идентификаторов, они захватывают три значения внутри варианта `rgb`.

## Шаблон структуры (деструктуризация записи)

Шаблон структуры позволяет напрямую извлекать нужные поля из структуры:

```yaoxiang
type Point = { x: Float, y: Float }
type Rect = { x: Float, y: Float, width: Float, height: Float }

# Деструктуризация с помощью шаблона структуры
area: (shape: Rect) -> Float = match shape {
    { x: _, y: _, width: w, height: h } => w * h,
}

r = Rect(x: 0.0, y: 0.0, width: 10.0, height: 20.0)
println(area(r))  # 200.0
```

`{ width: w, height: h }` означает «из записи взять поле `width` и связать с переменной `w`, взять поле `height` и связать с переменной `h`». `x: _` и `y: _` означают «эти поля существуют, но их значения не важны».

**Сокращённая запись**: если имя поля совпадает с именем переменной, можно использовать краткую форму — компилятор автоматически деструктурирует в одноимённую переменную:

```yaoxiang
describe_point: (p: Point) -> String = match p {
    { x: 0.0, y: 0.0 } => "Начало координат",
    { x, y } => "Координаты (${x}, ${y})",
}

println(describe_point(Point(x: 0.0, y: 0.0)))  # Начало координат
println(describe_point(Point(x: 3.0, y: 4.0)))  # Координаты (3.0, 4.0)
```

## Шаблон кортежа

Шаблон кортежа деструктурирует отдельные элементы кортежа:

```yaoxiang
type Pair = (Int, String)

first: (p: Pair) -> Int = match p {
    (n, _) => n,
}

second: (p: Pair) -> String = match p {
    (_, s) => s,
}

p = (42, "hello")
println(first(p))   # 42
println(second(p))  # "hello"
```

## Шаблон «или»

С помощью `|` можно объединить несколько шаблонов: совпадение засчитывается, если подходит любой из них:

```yaoxiang
type Token = number(Int) | plus | minus | times | divide | eof

# Объединяем несколько вариантов в класс «операторов»
is_operator: (t: Token) -> Bool = match t {
    plus | minus | times | divide => true,
    _ => false,
}

println(is_operator(plus))      # true
println(is_operator(number(5))) # false
```

## Охранные выражения (if-стражи)

Добавьте `if условие` после ветки сопоставления — тогда ветка сработает только при совпадении шаблона **и** выполнении условия:

```yaoxiang
type Age = adult(Int) | child(Int)

# Охранное выражение добавляет дополнительное условие
can_drive: (a: Age) -> Bool = match a {
    adult(n) if n >= 18 => true,
    adult(n) if n < 18 => false,
    child(_) => false,
}

println(can_drive(adult(20)))  # true
println(can_drive(adult(16)))  # false
```

Переменные в охранном выражении берутся из предшествующего шаблона — в `adult(n) if n >= 18` сначала `n` захватывает значение, затем `n >= 18` выполняет проверку.

## Проверка полноты

Компилятор YaoXiang гарантирует, что `match` покрывает все возможные случаи. Если ветвь пропущена, компилятор сообщит об ошибке:

```yaoxiang
type Direction = north | south | east | west

# ✅ Корректно: все четыре направления покрыты
turn: (d: Direction) -> Direction = match d {
    north => east,
    east => south,
    south => west,
    west => north,
}

# ❌ Ошибка компиляции: пропущен west
# broken: (d: Direction) -> Direction = match d {
#     north => east,
#     east => south,
#     south => west,
#     # west не обработан → ошибка компиляции
# }
```

Это важный механизм YaoXiang для защиты от неожиданностей во время выполнения — при добавлении нового варианта компилятор напомнит вам обновить все места с `match`.

## Вложенные шаблоны

Настоящая мощь шаблонов раскрывается в **вложенности** — внутри одного шаблона можно расположить другой:

```yaoxiang
type Expr = literal(Int) | add(Expr, Expr) | mul(Expr, Expr)

# Вложенный шаблон: внутри add снова сопоставляем literal
simplify: (e: Expr) -> Expr = match e {
    add(literal(0), right) => right,  # 0 + x = x
    add(left, literal(0)) => left,    # x + 0 = x
    mul(literal(1), right) => right,  # 1 * x = x
    mul(left, literal(1)) => left,    # x * 1 = x
    other => other,
}

e = add(literal(0), literal(5))
println(simplify(e))  # literal(5)
```

В `add(literal(0), right)` внешний слой — это шаблон перечисления `add`, внутренний — шаблон литерала `literal(0)` — два уровня вложенности за одно сопоставление.

## Итоги

| Тип шаблона | Синтаксис | Назначение |
|-------------|-----------|------------|
| Литерал | `42`, `"hi"` | Точное совпадение значения |
| Идентификатор | `x` | Захват сопоставленного значения |
| Подстановочный знак | `_` | Универсальное совпадение |
| Перечисление | `ok(value)` | Деструктуризация варианта перечисления |
| Структура | `{ x, y }` | Деструктуризация полей записи |
| Кортеж | `(a, b)` | Деструктуризация элементов кортежа |
| «Или» | `a \| b \| c` | Совпадение по одному из вариантов |
| Охранное выражение | `шаблон if условие` | Дополнительная проверка условия |

`match` + сопоставление шаблонов = самый мощный инструмент управления потоком выполнения в YaoXiang. Освоив его, вы будете писать более безопасный и понятный код.
