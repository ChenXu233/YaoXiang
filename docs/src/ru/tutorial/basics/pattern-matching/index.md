# Сопоставление с образцом

В [основах match](../control-flow/match.md) вы изучили базовое использование `match` — литералы, идентификаторы, подстановочные знаки. Теперь мы углубимся во все возможности сопоставления с образцом в YaoXiang.

## Полные типы образцов

Согласно спецификации синтаксиса, полное определение `Pattern` выглядит так:

```
Pattern     ::= Literal       # образец-литерал: 42, "hello"
            | Identifier      # образец-идентификатор: захват значения
            | Wildcard        # подстановочный знак: _
            | StructPattern   # образец структуры: деструктуризация записи
            | TuplePattern    # образец кортежа: деструктуризация кортежа
            | EnumPattern     # образец перечисления: деструктуризация варианта
            | OrPattern       # образец-или: pattern1 | pattern2
```

Первые три базовых образца вы уже изучили в предыдущей главе. Эта глава сосредоточена на четырёх продвинутых образцах.

## Образцы перечислений

Образцы перечислений — наиболее часто используемая продвинутая возможность `match`. Они позволяют деструктурировать варианты перечислений и извлекать внутренние данные.

### Базовое сопоставление перечислений

```yaoxiang
// Определение типа Result
Result: (T: Type, E: Type) -> Type = { ok(T) | err(E) }

// Функция обрабатывает Result с помощью match
handle: (result: Result(Int, String)) -> String = match result {
    ok(value) => "Успех! Полученное значение: {value}",
    err(msg) => "Произошла ошибка: {msg}",
}

a = ok(42)
b = err("время соединения истекло")

print(handle(a))  // Успех! Полученное значение: 42
print(handle(b))  // Произошла ошибка: время соединения истекло
```

### Тип Option

```yaoxiang
// Использование Option во избежание null
// Встроенный тип: Option: (T: Type) -> Type = some(T) | none

describe: (opt: Option(Int)) -> String = match opt {
    some(n) => "Есть значение: {n}",
    none => "Ничего нет",
}

print(describe(some(100)))  // Есть значение: 100
print(describe(none))       // Ничего нет
```

### Пользовательские перечисления

```yaoxiang
// Определение перечисления цветов
Color: Type = { red | green | blue | rgb(Int, Int, Int) }

to_hex: (c: Color) -> String = match c {
    red => "#FF0000",
    green => "#00FF00",
    blue => "#0000FF",
    rgb(r, g, b) => "#{r.to_hex()}{g.to_hex()}{b.to_hex()}",
}

print(to_hex(red))                // #FF0000
print(to_hex(rgb(128, 128, 128))) // #808080
```

`r`, `g`, `b` в `rgb(r, g, b)` — это образцы-идентификаторы: они захватывают три значения внутри варианта `rgb`.

## Образцы структур (деструктуризация записи)

Образцы структур позволяют извлекать нужные поля непосредственно из структуры:

```yaoxiang
Point: Type = { x: Float, y: Float }
Rect: Type = { x: Float, y: Float, width: Float, height: Float }

// Деструктуризация образцом структуры
area: (shape: Rect) -> Float = match shape {
    { x: _, y: _, width: w, height: h } => w * h,
}

r = Rect(0.0, 0.0, 10.0, 20.0)
print(area(r))  // 200.0
```

`{ width: w, height: h }` означает «извлечь поле `width` из записи и связать с переменной `w`, извлечь поле `height` и связать с переменной `h`». `x: _` и `y: _` означают «эти поля существуют, но их значения не важны».

**Сокращённая запись**: когда имя поля совпадает с именем переменной, можно использовать сокращение — компилятор автоматически выполнит деструктуризацию в одноимённую переменную:

```yaoxiang
describe_point: (p: Point) -> String = match p {
    { x: 0.0, y: 0.0 } => "Начало координат",
    { x, y } => "Координаты ({x}, {y})",
}

print(describe_point(Point(0.0, 0.0)))  // Начало координат
print(describe_point(Point(3.0, 4.0)))  // Координаты (3.0, 4.0)
```

## Образцы кортежей

Образцы кортежей деструктурируют отдельные элементы кортежа:

```yaoxiang
Pair: Type = (Int, String)

first: (p: Pair) -> Int = match p {
    (n, _) => n,
}

second: (p: Pair) -> String = match p {
    (_, s) => s,
}

p = (42, "hello")
print(first(p))   // 42
print(second(p))  // "hello"
```

## Образцы или

Используйте `|` для объединения нескольких образцов, чтобы соответствовать любому из них:

```yaoxiang
Token: Type = { number(Int) | plus | minus | times | divide | eof }

// Объединение нескольких вариантов в категорию "оператор"
is_operator: (t: Token) -> Bool = match t {
    plus | minus | times | divide => true,
    _ => false,
}

print(is_operator(plus))      // true
print(is_operator(number(5))) // false
```

## Защитные выражения (if-стражи)

Добавьте `if условие` после ветви сопоставления, чтобы она срабатывала только тогда, когда образец совпадает **и** условие выполнено:

```yaoxiang
Age: Type = { adult(Int) | child(Int) }

// Защитное выражение добавляет дополнительное условие
can_drive: (a: Age) -> Bool = match a {
    adult(n) if n >= 18 => true,
    adult(n) if n < 18 => false,
    child(_) => false,
}

print(can_drive(adult(20)))  // true
print(can_drive(adult(16)))  // false
```

Переменные в защитном выражении берутся из предыдущего образца — `adult(n) if n >= 18` сначала захватывает значение в `n`, затем проверяет `n >= 18`.

## Проверка полноты (исчерпываемости)

Компилятор YaoXiang гарантирует, что `match` покрывает все возможные случаи. Если ветвь пропущена, компилятор выдаст ошибку:

```yaoxiang
Direction: Type = { north | south | east | west }

// ✅ Верно: все четыре направления покрыты
turn: (d: Direction) -> Direction = match d {
    north => east,
    east => south,
    south => west,
    west => north,
}

// ❌ Ошибка компиляции: пропущен west
// broken: (d: Direction) -> Direction = match d {
//     north => east,
//     east => south,
//     south => west,
//     // west не обработан → ошибка компиляции
// }
```

Это важный механизм YaoXiang для предотвращения непредвиденных ситуаций во время выполнения — как только вы добавите новый вариант, компилятор напомнит вам обновить все места с `match`.

## Вложенные образцы

Истинная мощь образцов заключается в **вложенности** — вы можете вкладывать один образец в другой:

```yaoxiang
Expr: Type = { literal(Int) | add(Expr, Expr) | mul(Expr, Expr) }

// Вложенный образец: внутри add снова сопоставляем literal
simplify: (e: Expr) -> Expr = match e {
    add(literal(0), right) => right,  // 0 + x = x
    add(left, literal(0)) => left,    // x + 0 = x
    mul(literal(1), right) => right,  // 1 * x = x
    mul(left, literal(1)) => left,    // x * 1 = x
    other => other,
}

e = add(literal(0), literal(5))
print(simplify(e))  // literal(5)
```

В `add(literal(0), right)` внешний слой — это образец перечисления `add`, внутренний — образец-литерал `literal(0)`: два уровня вложенности за одно сопоставление.

## Итоги

| Тип образца | Синтаксис | Назначение |
|----------|------|------|
| Литерал | `42`, `"hi"` | Точное сопоставление значения |
| Идентификатор | `x` | Захват сопоставленного значения |
| Подстановочный знак | `_` | Универсальное сопоставление |
| Перечисление | `ok(value)` | Деструктуризация варианта перечисления |
| Структура | `{ x, y }` | Деструктуризация полей записи |
| Кортеж | `(a, b)` | Деструктуризация элементов кортежа |
| Или | `a \| b \| c` | Сопоставление с одним из вариантов |
| Защитное выражение | `pattern if cond` | Дополнительная проверка условия |

`match` + сопоставление с образцом = самый мощный инструмент управления потоком выполнения в YaoXiang. Освоив его, вы будете писать более безопасный и понятный код.
