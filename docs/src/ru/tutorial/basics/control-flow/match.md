# Основы match

`match` — это самая мощная управляющая конструкция в YaoXiang. Она позволяет выбирать различные пути обработки в зависимости от формы **значения**. Если вы использовали `switch` в других языках, то обнаружите, что `match` — его значительно улучшенная версия.

## Базовый синтаксис

Определение выражения `match` в спецификации синтаксиса:

```
match Expr { MatchArm+ }
MatchArm : Pattern ('|' Pattern)* ('if' Expr)? '=>' Expr ','
```

Если разложить по частям:

- после `match` идёт значение для сопоставления;
- `{}` содержат одну или несколько **ветвей сопоставления** (MatchArm);
- каждая ветвь: за **шаблоном** следует `=>`, затем — **выражение-результат**;
- каждая ветвь заканчивается запятой.

Простейший пример:

```yaoxiang
number = 2

text = match number {
    0 => "ноль",
    1 => "один",
    2 => "два",
}
print(text)  // "два"
```

## match — это выражение

Как и `if`, `match` тоже является **выражением** — оно вычисляет значение. Типы возвращаемых значений всех ветвей сопоставления должны совпадать:

```yaoxiang
score = 85

grade = match score {
    90..100 => "A",    // шаблон диапазона (продвинутая тема)
    80..89 => "B",
    70..79 => "C",
    60..69 => "D",
    _ => "F",          // шаблон-джокер: соответствует всем оставшимся случаям
}
print(grade)  // "B"
```

> **Примечание**: шаблоны диапазонов, такие как `90..100`, относятся к продвинутым темам и будут подробно рассмотрены в разделе [Продвинутое сопоставление с образцом](../pattern-matching.md). В данной главе мы сосредоточимся на базовых шаблонах.

## Базовые шаблоны

### Шаблон литерала

Сопоставление с конкретным значением:

```yaoxiang
response = 404

message = match response {
    200 => "OK",
    301 => "Moved",
    404 => "Not Found",
    500 => "Server Error",
    _ => "Unknown",
}
print(message)  // "Not Found"
```

### Шаблон идентификатора

Использование имени переменной для захвата сопоставленного значения:

```yaoxiang
result: Result(Int, String) = ok(42)

description = match result {
    ok(value) => "Успех, значение: " + value.to_string(),
    err(error) => "Неудача, причина: " + error,
}
print(description)  // "Успех, значение: 42"
```

`value` в `ok(value)` — это шаблон идентификатора: он захватывает фактическое значение, обёрнутое в `ok`, и вы можете использовать его в выражении после `=>`.

### Шаблон-джокер

`_` — это шаблон-джокер, который соответствует **любому значению**. Обычно его помещают в конец в качестве запасного варианта:

```yaoxiang
command = "exit"

action = match command {
    "start" => "Запустить сервис",
    "stop" => "Остановить сервис",
    "restart" => "Перезапустить сервис",
    _ => "Неизвестная команда: " + command,
}
print(action)  // "Неизвестная команда: exit"
```

## Сопоставление должно быть исчерпывающим

`match` в YaoXiang требует покрытия всех возможных случаев: если компилятор обнаружит, что какие-то возможные значения пропущены, он сразу же выдаст ошибку. Это отражает безопасность `match`.

```yaoxiang
// Этот код не скомпилируется
// value = true
// result = match value {
//     true => "Да",
//     // Отсутствует ветвь false — ошибка компиляции!
// }

// Правильно — использовать _ в качестве запасного варианта
value = true
result = match value {
    true => "Да",
    _ => "Нет",      // _ обеспечивает обработку и false
}
```

Когда вы точно знаете, что существует лишь конечное число случаев (например, при сопоставлении с перечислением), компилятор поможет вам проверить, что каждый вариант покрыт. Это мощное средство защиты от ошибок из-за пропущенных ветвей.

## Комбинирование нескольких шаблонов

Одна ветвь сопоставления может соответствовать нескольким шаблонам, разделённым `|`:

```yaoxiang
day = "sunday"

type = match day {
    "monday" | "tuesday" | "wednesday" | "thursday" | "friday" => "Рабочий день",
    "saturday" | "sunday" => "Выходной",
    _ => "Неверно",
}
print(type)  // "Выходной"
```

## Ветви сопоставления выполняются по порядку

`match` начинает проверку шаблонов с первой ветви — **срабатывает первая успешно сопоставленная ветвь**, а последующие не выполняются:

```yaoxiang
number = 5

result = match number {
    _ => "Другое",     // шаблон-джокер соответствует всему, сработает здесь
    5 => "Пять",       // никогда не выполнится — уже сопоставлено выше
}
print(result)  // "Другое"
```

Эта особенность означает, что **помещать шаблон-джокер `_` в конец** — хорошая практика.

## Резюме

| Пункт | Описание |
|------|------|
| Синтаксис | `match значение { шаблон => выражение, ... }` |
| Выражение | `match` вычисляет значение, типы всех ветвей должны совпадать |
| Шаблон литерала | Точное сопоставление с конкретным значением: `200 => "OK"` |
| Шаблон идентификатора | Захват значения в переменную: `ok(value) => ...` |
| Шаблон-джокер `_` | Соответствует любому значению, служит запасным вариантом |
| Исчерпывающее сопоставление | Должно покрывать все возможные случаи, компилятор проверяет это |
| Несколько шаблонов | `шаблон1 \| шаблон2 => выражение` |
| Последовательное выполнение | Сверху вниз, срабатывает первая совпавшая ветвь |

> **Следующий шаг**: в этой статье рассмотрены базовые возможности `match`. Более продвинутые шаблоны (вложенные шаблоны, охранные выражения, деструктуризация структур и т.д.) описаны в разделе [Продвинутое сопоставление с образцом](../pattern-matching.md).
