# Основы match

`match` — самая мощная управляющая конструкция в YaoXiang. Она позволяет выбирать разные пути обработки в зависимости от **формы** значения. Если вы использовали `switch` в других языках, вы обнаружите, что `match` — это全面升级 (полностью усовершенствованная) версия.

## Базовый синтаксис

Определение выражения `match` в спецификации синтаксиса:

```
match Expr { MatchArm+ }
MatchArm : Pattern ('|' Pattern)* ('if' Expr)? '=>' Expr ','
```

Разберём по частям:

- После `match` указывается значение для сопоставления
- `{}` содержит одну или несколько **ветвей сопоставления** (MatchArm)
- Каждая ветвь: **паттерн**, затем `=>`, затем **выражение-результат**
- Каждая ветвь заканчивается запятой

Простейший пример:

```yaoxiang
number = 2

text = match number {
    0 => "ноль",
    1 => "один",
    2 => "два",
}
println(text)  # "два"
```

## match — это выражение

Как и `if`, `match` тоже является **выражением** — он вычисляет значение. Типы возвращаемых значений всех ветвей сопоставления должны быть одинаковыми:

```yaoxiang
score = 85

grade = match score {
    90..100 => "A",    # паттерн диапазона (продвинутый материал)
    80..89 => "B",
    70..79 => "C",
    60..69 => "D",
    _ => "F",          # подстановочный знак: соответствует всем оставшимся случаям
}
println(grade)  # "B"
```

> **Внимание**: паттерны диапазонов `90..100` и подобные относятся к продвинутому материалу и будут подробно рассмотрены в [продвинутом pattern matching](../pattern-matching.md). В этой главе сосредоточимся на базовых паттернах.

## Базовые паттерны

### Паттерны-литералы

Сопоставление по конкретному значению:

```yaoxiang
response = 404

message = match response {
    200 => "OK",
    301 => "Moved",
    404 => "Not Found",
    500 => "Server Error",
    _ => "Unknown",
}
println(message)  # "Not Found"
```

### Паттерны-идентификаторы

Использование имени переменной для захвата сопоставленного значения:

```yaoxiang
result: Result(Int, String) = ok(42)

description = match result {
    ok(value) => "Успех, значение: " + value.to_string(),
    err(error) => "Неудача, причина: " + error,
}
println(description)  # "Успех, значение: 42"
```

`value` в `ok(value)` — это паттерн-идентификатор: он захватывает фактическое значение, обёрнутое в `ok`, и вы можете использовать его в выражении после `=>`.

### Подстановочный паттерн

`_` — это подстановочный знак, который соответствует **любому значению**. Обычно ставится в конце как «ловушка» для всех остальных случаев:

```yaoxiang
command = "exit"

action = match command {
    "start" => "Запустить сервис",
    "stop" => "Остановить сервис",
    "restart" => "Перезапустить сервис",
    _ => "Неизвестная команда: " + command,
}
println(action)  # "Неизвестная команда: exit"
```

## Сопоставление должно быть исчерпывающим

`match` в YaoXiang требует покрытия всех возможных случаев — если компилятор обнаружит, что вы пропустили какие-то возможные значения, он сразу выдаст ошибку. Это проявление безопасности `match`.

```yaoxiang
# Этот код не скомпилируется
# value = true
# result = match value {
#     true => "да",
#     # Отсутствует ветвь false — ошибка компиляции!
# }

# Правильно — использование _ в качестве «ловушки»
value = true
result = match value {
    true => "да",
    _ => "нет",      # _ гарантирует обработку false
}
```

Когда вы точно знаете, что существует лишь ограниченное количество случаев (например, при сопоставлении enum), компилятор проверяет, что каждый вариант покрыт. Это мощное средство защиты от багов, связанных с пропущенными ветвями.

## Комбинирование нескольких паттернов

Одна ветвь сопоставления может соответствовать нескольким паттернам, разделённым `|`:

```yaoxiang
day = "sunday"

type = match day {
    "monday" | "tuesday" | "wednesday" | "thursday" | "friday" => "рабочий день",
    "saturday" | "sunday" => "выходной",
    _ => "неверно",
}
println(type)  # "выходной"
```

## Ветви сопоставления выполняются по порядку

`match` начинает перебор ветвей с первой, **срабатывает первая успешно совпавшая ветвь**, последующие не выполняются:

```yaoxiang
number = 5

result = match number {
    _ => "другое",     # подстановочный знак соответствует всему — сработает здесь
    5 => "пять",       # никогда не выполнится — выше уже сработало совпадение
}
println(result)  # "другое"
```

Из этого свойства следует, что **ставить подстановочный знак `_` в конец** — хорошая привычка.

## Резюме

| Пункт | Описание |
|------|------|
| Синтаксис | `match значение { паттерн => выражение, ... }` |
| Выражение | `match` вычисляет значение, типы всех ветвей совпадают |
| Паттерн-литерал | Точное соответствие конкретному значению: `200 => "OK"` |
| Паттерн-идентификатор | Захват значения в переменную: `ok(value) => ...` |
| Подстановочный `_` | Соответствует любому значению, служит «ловушкой» |
| Исчерпывающность | Необходимо покрыть все варианты, компилятор проверяет |
| Несколько паттернов | `паттерн1 \| паттерн2 => выражение` |
| Порядок выполнения | Сверху вниз, срабатывает первая совпавшая ветвь |

> **Следующий шаг**: эта статья охватывает базовое использование `match`. Более продвинутые паттерны (вложенные паттерны, защитные выражения, деструктуризация структур и т.д.) описаны в [Продвинутом pattern matching](../pattern-matching.md).
