# Основы match

`match` — наиболее мощная управляющая конструкция YaoXiang. Она позволяет выбрать различные пути обработки в зависимости от **формы значения**. Если вы использовали `switch` в других языках, вы обнаружите, что `match` — это его значительно улучшенная версия.

## Базовый синтаксис

Определение выражения `match` в грамматике:

```
match Expr { MatchArm+ }
MatchArm : Pattern ('|' Pattern)* ('if' Expr)? '=>' Expr ','
```

Разбор по частям:

- после `match` следует значение для сопоставления
- `{}` содержит одну или несколько **ветвей сопоставления** (MatchArm)
- каждая ветвь: **образец** за которым следует `=>`, затем **выражение-результат**
- каждая ветвь завершается запятой

Простейший пример:

```yaoxiang
number = 2

text = match number {
    0 => "零",
    1 => "一",
    2 => "二",
}
print(text)  // "二"
```

## match как выражение

Как и `if`, `match` является **выражением** — оно вычисляет значение. Типы возвращаемых значений всех ветвей должны совпадать:

```yaoxiang
score = 85

grade = match score {
    90..100 => "A",    // диапазон (продвинутая тема)
    80..89 => "B",
    70..79 => "C",
    60..69 => "D",
    _ => "F",          // подстановочный знак: всё остальное
}
print(grade)  // "B"
```

> **Примечание**: диапазоны вроде `90..100` рассматриваются в разделе [Продвинутые техники сопоставления](../pattern-matching.md). Здесь мы сосредоточимся на базовых образцах.

## Базовые образцы

### Образцы-литералы

Сопоставление с конкретными значениями:

```yaoxiang
response = 404

message = match response {
    200 => "OK",
    301 => "Moved",
    404 => "Not Found",
    500 => "Server Error",
    _ => "Unknown",
}
print(message)  // "Not Found"
```

### Идентификатор-образцы

Захват сопоставленного значения в переменную:

```yaoxiang
result: Result(Int, String) = ok(42)

description = match result {
    ok(value) => "成功，值是: " + value.to_string(),
    err(error) => "失败，原因: " + error,
}
print(description)  // "成功，值是: 42"
```

В `ok(value)` переменная `value` — это идентификатор-образец, который захватывает значение, обёрнутое в `ok`. Вы можете использовать его в выражении после `=>`.

### Подстановочный образец (wildcard)

`_` — это подстановочный знак, который сопоставляется с **любым значением**. Обычно его размещают в конце как запасной вариант:

```yaoxiang
command = "exit"

action = match command {
    "start" => "启动服务",
    "stop" => "停止服务",
    "restart" => "重启服务",
    _ => "未知指令: " + command,
}
print(action)  // "未知指令: exit"
```

## Сопоставление должно быть исчерпывающим

В YaoXiang `match` требует покрытия всех возможных случаев — если компилятор обнаружит, что вы пропустили какие-то значения, он выдаст ошибку. Это проявление безопасности `match`.

```yaoxiang
// Этот код не скомпилируется
// value = true
// result = match value {
//     true => "是",
//     // отсутствует ветвь false — ошибка компиляции!
// }

// Правильно — используем _ как запасной вариант
value = true
result = match value {
    true => "是",
    _ => "否",      // _ гарантирует обработку false
}
```

Когда вы точно знаете, что существует ограниченное число вариантов (например, при сопоставлении с перечислением), компилятор помогает проверить, покрыт ли каждый вариант. Это мощный инструмент защиты от ошибок.

## Комбинация нескольких образцов

Одна ветвь сопоставления может соответствовать нескольким образцам, разделённым `|`:

```yaoxiang
day = "sunday"

type = match day {
    "monday" | "tuesday" | "wednesday" | "thursday" | "friday" => "工作日",
    "saturday" | "sunday" => "休息日",
    _ => "无效",
}
print(type)  // "休息日"
```

## Ветви сопоставления выполняются последовательно

`match` начинает проверку с первой ветви, **первая успешно сопоставленная ветвь выполняется**, остальные игнорируются:

```yaoxiang
number = 5

result = match number {
    _ => "其他",     // подстановочный знак соответствует всему, здесь сработает
    5 => "五",       // никогда не выполнится — уже сопоставлено выше
}
print(result)  // "其他"
```

Эта особенность означает, что **размещение подстановочного знака `_` в конце** — хорошая практика.

## Итоги

| Понятие | Описание |
|---------|----------|
| Синтаксис | `match значение { образец => выражение, ... }` |
| Выражение | `match` вычисляет значение, все ветви имеют одинаковый тип |
| Образец-литерал | Точное сопоставление со значением: `200 => "OK"` |
| Идентификатор-образец | Захват значения в переменную: `ok(value) => ...` |
| Подстановочный `_` | Сопоставление с любым значением, запасной вариант |
| Исчерпывающесть | Должно покрывать все возможные случаи, компилятор проверяет |
| Несколько образцов | `образец1 \| образец2 => выражение` |
| Последовательность | Сверху вниз, первая сопоставленная ветвь выполняется |
