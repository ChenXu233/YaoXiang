# Управление потоком выполнения

В предыдущей главе вы научились определять функции. Теперь давайте узнаем, как наделить код способностью «принимать решения» и «повторять выполнение» — это и есть **управление потоком выполнения**.

YaoXiang предоставляет пять конструкций управления потоком, каждая из которых выполняет свою задачу:

| Конструкция | Назначение | Краткое описание |
|--------|------|------------|
| `if-elif-else` | Условное ветвление | Выбор пути выполнения в зависимости от условия |
| `for` | Цикл обхода | Последовательная обработка каждого элемента коллекции |
| `while` | Условный цикл | Повторение, пока условие истинно |
| `break` / `continue` | Управление циклом | Досрочный выход из цикла или пропуск текущей итерации |
| `match` | Сопоставление с образцом | Ветвление в зависимости от структуры значения |

Краткое знакомство:

```yaoxiang
# if — это выражение, способное возвращать значение
status = if score >= 60 { "сдал" } else { "не сдал" }

# for — обход диапазона
for i in 0..5 {
    println(i)
}

# while — условный цикл
mut n = 3
while n > 0 {
    println(n)
    n = n - 1
}

# match — сопоставление с образцом
description = match number {
    0 => "ноль",
    1 => "один",
    _ => "другое",
}
```

Все эти конструкции управления потоком могут использоваться как **выражения** — они способны вычислять значение. Это важное отличие YaoXiang от многих традиционных языков.

В следующих разделах каждая конструкция управления потоком будет рассмотрена подробно. Рекомендуется читать их по порядку, поскольку между ними существует естественная преемственность.
