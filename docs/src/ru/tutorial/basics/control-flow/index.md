# Управление потоком

В предыдущей главе вы научились определять функции. Теперь мы изучим, как дать коду возможность «принимать решения» и «повторять выполнение» — это и есть **управление потоком**.

YaoXiang предоставляет пять структур управления потоком, каждая со своей задачей:

| Управление потоком | Назначение | Краткое описание |
|--------|------|------------|
| `if-elif-else` | Условное ветвление | Выбор пути выполнения в зависимости от условия |
| `for` | Цикл обхода | Последовательная обработка каждого элемента коллекции |
| `while` | Условный цикл | Повторяется, пока условие истинно |
| `break` / `continue` | Управление циклом | Досрочный выход из цикла или пропуск текущей итерации |
| `match` | Сопоставление с образцом | Ветвление в зависимости от структуры значения |

Краткое знакомство:

```yaoxiang
// if — это выражение, может возвращать значение
status = if score >= 60 { "сдал" } else { "не сдал" }

// for обход диапазона
for i in 0..5 {
    print(i)
}

// while условный цикл
mut n = 3
while n > 0 {
    print(n)
    n = n - 1
}

// match сопоставление с образцом
description = match number {
    0 => "ноль",
    1 => "один",
    _ => "другое",
}
```

Все эти структуры управления потоком могут использоваться как **выражения** — они могут вычислять значение. Это важное отличие YaoXiang от многих традиционных языков.
