# Цикл for

Когда вам нужно выполнить одно и то же действие для каждого элемента последовательности, цикл `for` — это именно то, что нужно. Цикл `for` в YaoXiang спроектирован лаконично и точно.

## Базовый синтаксис

Формальное определение оператора `for` в спецификации синтаксиса:

```
for 'mut'? Identifier 'in' Expr Block
```

Переводя на русский: начинается с `for`, может иметь необязательный `mut`, затем имя переменной цикла, потом `in` и выражение для обхода, и в конце — блок кода тела цикла.

## Обход числового диапазона

Самое распространённое использование — создание диапазона с помощью `..` и его обход через `for`:

```yaoxiang
# от 0 до 4 (не включая 5)
for i in 0..5 {
    println(i)
}
# Вывод: 0 1 2 3 4
```

`0..5` означает диапазон от 0 (включительно) до 5 (не включительно). Это общепринятое соглашение в компьютерных науках — полуоткрытый интервал.

Вы также можете использовать переменные для задания начала и конца диапазона:

```yaoxiang
start = 10
end = 15
for n in start..end {
    println(n)
}
# Вывод: 10 11 12 13 14
```

## Обход списков

`for` может не только обходить числовые диапазоны, но и напрямую перебирать коллекции вроде списков и массивов:

```yaoxiang
colors = ["红", "橙", "黄", "绿", "蓝"]

for color in colors {
    println("当前颜色: " + color)
}
# Вывод:
# 当前颜色: 红
# 当前颜色: 橙
# ... и так далее
```

## Уникальная семантика for: новая привязка на каждой итерации

У цикла `for` в YaoXiang есть особенность, отличающая его от других языков: **на каждой итерации создаётся новая привязка, а не изменяется одна и та же переменная**.

Поймём это с помощью таблицы:

| Итерация | Что происходит |
|----------|----------------|
| 1-я | Создаётся новая привязка `i = 0`, выполняется тело цикла, затем привязка уничтожается |
| 2-я | Создаётся новая привязка `i = 1` (совершенно новая), выполняется тело цикла, затем уничтожается |
| 3-я | Создаётся новая привязка `i = 2`, выполняется тело цикла, затем уничтожается |
| ... | ... |
| Конец цикла | Диапазон исчерпан, цикл завершается |

Это означает, что переменная цикла на каждой итерации представляет собой независимое новое значение. Это очень полезно для безопасности — вам не нужно беспокоиться о случайном изменении переменной цикла:

```yaoxiang
for i in 1..5 {
    # i = i + 1   // Ошибка: по умолчанию неизменяемая, нельзя изменять i
    println(i)
}
```

## for mut: явное объявление при необходимости изменения

Если вам действительно нужно изменить переменную цикла внутри тела (например, использовать её как аккумулятор), используйте `for mut`:

```yaoxiang
# for mut разрешает изменение привязки внутри тела цикла
for mut i in 0..5 {
    i = i * 2
    println(i)
}
# Вывод: 0 2 4 6 8
```

Обратите внимание: даже при использовании `for mut` на каждой итерации по-прежнему создаётся новая привязка. `for mut` лишь делает саму новую привязку изменяемой, но не переносит изменения предыдущей итерации в следующую.

```yaoxiang
for mut i in 1..5 {
    i = i + 100
    println(i)        # Каждый раз выводит 101, 102, 103, 104
}
# На каждой итерации i заново берётся из диапазона, изменения прошлой итерации не влияют на следующую
```

## Переменная цикла не может затенять внешнюю переменную

YaoXiang запрещает затенение переменных (shadowing). Переменная цикла `for` не может иметь то же имя, что и переменная во внешней области видимости:

```yaoxiang
# Неправильный пример
i = 10
# for i in 1..5 {     // Ошибка компиляции! i уже объявлена во внешней области
#     println(i)
# }

# Правильный вариант — используйте другое имя
i = 10
for j in 1..5 {
    println(j)
}
```

Это правило избавляет вас от путаницы «на какую именно переменную ссылается код в данный момент».

## Сравнение с другими языками

| Язык | Семантика переменной цикла for |
|------|---------------------------------|
| YaoXiang | Новая привязка на каждой итерации |
| Rust | Изменение одной и той же переменной (требуется mut) |
| Python | Изменение одной и той же переменной |
| C/C++ | Изменение одной и той же переменной |

Дизайн YaoXiang ближе к человеческой интуиции — «для каждого элемента коллекции выполнить что-то» — каждый элемент является самостоятельной сущностью.

## Резюме

| Пункт | Описание |
|-------|----------|
| Обход диапазона | `for i in 0..5`, полуоткрытый интервал |
| Обход коллекции | `for item in list`, поэлементный перебор |
| Семантика привязки | Новая привязка на каждой итерации, а не изменение одной переменной |
| По умолчанию неизменяемая | Переменную цикла нельзя изменять — защита от случайных правок |
| `for mut` | Явное объявление при необходимости изменения |
| Запрет затенения | Переменная цикла не может иметь то же имя, что и внешняя переменная |
