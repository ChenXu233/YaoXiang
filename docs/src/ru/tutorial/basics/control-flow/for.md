# Цикл for

Когда вам нужно выполнить одно и то же действие над каждым элементом последовательности, `for` — именно тот инструмент, который нужен. Цикл `for` в YaoXiang спроектирован лаконично и точно.

## Базовый синтаксис

Формальное определение оператора `for` в спецификации синтаксиса:

```
for 'mut'? Identifier 'in' Expr Block
```

Если перевести: начинается с `for`, далее может идти необязательный `mut`, затем имя переменной цикла, потом `in` и перебираемое выражение, и в конце — блок кода тела цикла.

## Перебор числового диапазона

Чаще всего `for` используется с диапазоном, созданным оператором `..`:

```yaoxiang
// от 0 до 4 (не включая 5)
for i in 0..5 {
    print(i)
}
// Вывод: 0 1 2 3 4
```

`0..5` означает диапазон от 0 (включительно) до 5 (не включительно). Это общепринятое соглашение в компьютерных науках — интервал, закрытый слева и открытый справа.

Начало и конец диапазона можно задавать переменными:

```yaoxiang
start = 10
end = 15
for n in start..end {
    print(n)
}
// Вывод: 10 11 12 13 14
```

## Перебор списка

`for` способен перебирать не только числовые диапазоны, но и коллекции вроде списков или массивов:

```yaoxiang
colors = ["红", "橙", "黄", "绿", "蓝"]

for color in colors {
    print("当前颜色: " + color)
}
// Вывод:
// 当前颜色: 红
// 当前颜色: 橙
// ... и так далее
```

## Уникальная семантика for: каждая итерация создаёт новое связывание

У цикла `for` в YaoXiang есть особенность, отличающая его от других языков: **каждая итерация создаёт новое связывание, а не изменяет одну и ту же переменную**.

Чтобы понять, рассмотрим таблицу:

| Итерация | Что происходит |
|------|-----------|
| 1-я | Создаётся новое связывание `i = 0`, выполняется тело цикла, затем связывание уничтожается |
| 2-я | Создаётся новое связывание `i = 1` (совершенно новое), выполняется тело цикла, затем уничтожается |
| 3-я | Создаётся новое связывание `i = 2`, выполняется тело цикла, затем уничтожается |
| ... | ... |
| Конец цикла | Диапазон исчерпан, цикл завершается |

Это означает, что переменная цикла в каждой итерации — независимое новое значение. Это очень полезно для безопасности — вам не нужно беспокоиться о случайном изменении переменной цикла:

```yaoxiang
for i in 1..5 {
    // i = i + 1   // Ошибка: по умолчанию неизменяемое, i нельзя изменять
    print(i)
}
```

## for mut: явное объявление при необходимости изменения

Если вам действительно нужно изменить переменную цикла внутри тела (например, использовать её как аккумулятор), применяйте `for mut`:

```yaoxiang
// for mut разрешает изменение связывания внутри тела цикла
for mut i in 0..5 {
    i = i * 2
    print(i)
}
// Вывод: 0 2 4 6 8
```

Обратите внимание: даже при использовании `for mut` каждая итерация по-прежнему создаёт новое связывание. `for mut` лишь делает само новое связывание изменяемым и не передаёт изменения предыдущей итерации в следующую.

```yaoxiang
for mut i in 1..5 {
    i = i + 100
    print(i)        // каждый раз выводит 101, 102, 103, 104
}
// в каждой итерации i начинается со значения из диапазона заново, изменения предыдущей итерации не влияют на следующую
```

## Переменная цикла не может затенять внешнюю переменную

YaoXiang запрещает затенение переменных. Переменная цикла `for` не может иметь то же имя, что и переменная во внешней области видимости:

```yaoxiang
// Неправильный пример
i = 10
// for i in 1..5 {     // Ошибка компиляции! i уже объявлена во внешней области
//     print(i)
// }

// Правильный вариант — используйте другое имя
i = 10
for j in 1..5 {
    print(j)
}
```

Это правило гарантирует, что вы никогда не запутаетесь в том, на какую именно переменную ссылается текущий код.

## Сравнение с другими языками

| Язык | Семантика переменной цикла for |
|------|-----------------|
| YaoXiang | Каждая итерация создаёт новое связывание |
| Rust | Изменяется одна и та же переменная (требуется mut) |
| Python | Изменяется одна и та же переменная |
| C/C++ | Изменяется одна и та же переменная |

Дизайн YaoXiang ближе к человеческой интуиции — «для каждого элемента в коллекции выполнить что-то» — где каждый элемент является отдельной сущностью.

## Резюме

| Пункт | Пояснение |
|------|------|
| Перебор диапазона | `for i in 0..5`, интервал закрыт слева и открыт справа |
| Перебор коллекции | `for item in list`, поэлементное извлечение |
| Семантика связывания | Каждая итерация создаёт новое связывание, а не изменяет одну и ту же переменную |
| Неизменяемость по умолчанию | Переменная цикла не может быть изменена для защиты от случайных правок |
| `for mut` | Явное объявление, когда изменение необходимо |
| Запрет затенения | Переменная цикла не может иметь то же имя, что и внешняя переменная |
