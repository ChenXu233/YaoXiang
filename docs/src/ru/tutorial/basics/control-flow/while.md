# цикл while

`for` подходит для сценариев «обхода известной коллекции», тогда как `while` подходит для другой ситуации — **когда вы не знаете, сколько раз нужно повторять, а знаете только, когда следует остановиться**.

## Базовый синтаксис

Определение оператора `while` в спецификации синтаксиса:

```
while Expr Block
```

Структура проста: после `while` идёт условное выражение, затем блок кода тела цикла. Пока условие истинно (`true`), тело цикла выполняется.

```yaoxiang
mut count = 1

while count <= 5 {
    println(count)
    count = count + 1
}
// Вывод: 1 2 3 4 5
```

Обратите внимание, что мы объявляем переменную как `mut count` — поскольку `count` нужно изменять внутри цикла. Если написать `count = 1` (неизменяемая), то строка `count = count + 1` в теле цикла вызовет ошибку.

## Порядок выполнения `while`

Шаги выполнения цикла `while` следующие:

1. Проверить условное выражение.
2. Если условие истинно (`true`), выполнить тело цикла, затем вернуться к шагу 1.
3. Если условие ложно (`false`), завершить цикл и продолжить выполнение последующего кода.

Условие проверяется **перед началом каждой итерации**. Если изначально условие ложно (`false`), тело цикла не выполнится ни разу:

```yaoxiang
mut n = 0
while n > 0 {
    println("Эта строка никогда не будет напечатана")
    n = n - 1
}
// Условие n > 0 с самого начала false, тело цикла пропускается
```

## break: досрочный выход из цикла

Иногда нужно досрочно выйти из цикла посередине — например, когда искомый элемент найден:

```yaoxiang
numbers = [3, 7, 2, 9, 5]
mut found = false
mut index = 0

while index < 5 {
    if numbers[index] == 9 {
        found = true
        break      # Нашли, продолжать не нужно
    }
    index = index + 1
}

println("Найдено? " + found.to_string())  # "Найдено? true"
```

`break` заставляет программу немедленно выйти из текущего цикла и продолжить выполнение кода после него.

## continue: пропуск текущей итерации

`continue` отличается от `break` — он не выходит из цикла, а пропускает оставшуюся часть текущей итерации и сразу переходит к следующей проверке условия:

```yaoxiang
mut n = 0
while n < 5 {
    n = n + 1
    if n == 3 {
        continue   # Пропускаем 3, не печатаем
    }
    println(n)
}
// Вывод: 1 2 4 5
```

В этом коде, когда `n` равно 3, `continue` пропускает `println(n)` и сразу возвращается к проверке условия `while n < 5`.

## Как избежать бесконечного цикла

При использовании `while` нужно быть особенно внимательным — убедитесь, что условие цикла в конечном счёте станет `false`, иначе программа зависнет навсегда:

```yaoxiang
# Опасно! Бесконечный цикл — условие всегда true
# mut x = 1
# while x > 0 {
#     x = x + 1     # x становится всё больше и больше, никогда не станет <= 0
# }

# Правильно — есть явное условие завершения
mut x = 1
while x <= 5 {
    println(x)
    x = x + 1        # x постепенно увеличивается, и при x > 5 цикл завершится
}
```

## Использование `while` для чтения ввода

Классический сценарий использования `while` — обработка ввода неизвестной длины: вы не знаете, сколько строк введёт пользователь, а знаете лишь, что «ввод прекращается на пустой строке»:

```yaoxiang
# Псевдокод, демонстрирующий типичное использование while
# read_line возвращает пустую строку при чтении пустой строки
mut line = read_line()
while line != "" {
    process(line)
    line = read_line()
}
```

Этот шаблон «проверить условие → обработать данные → обновить условие» — основной паттерн использования `while`.
