# if-elif-else

`if-elif-else` — это самый базовый инструмент принятия решений в программировании. Его логика очень интуитивна: **если условие истинно, выполняется определённый блок кода; иначе проверяется следующее условие; если ни одно не выполняется, выбирается путь по умолчанию**.

## Базовый синтаксис

В спецификации синтаксиса определения `if`-выражения и `if`-инструкции полностью совпадают:

```
if Expr Block ('elif' Expr Block)* ('else' Block)?
```

Если перевести на повседневный язык: начинается с `if`, за которым следуют условное выражение и блок кода, затем может идти от нуля до нескольких `elif условие блок`, и в конце может быть необязательный `else блок`.

Простейшая форма — только `if`:

```yaoxiang
if temperature > 30 {
    println("Жарко, включи кондиционер")
}
```

С добавлением `else`:

```yaoxiang
if is_raining {
    println("Возьми зонт")
} else {
    println("Зонт не нужен")
}
```

Несколько условий через `elif`:

```yaoxiang
score = 85

if score >= 90 {
    println("Отлично")
} elif score >= 80 {
    println("Хорошо")
} elif score >= 60 {
    println("Удовлетворительно")
} else {
    println("Нужно подтянуть")
}
```

Обратите внимание: в YaoXiang ключевое слово — `elif`, а не `else if`. Это отражение намеренного стремления языка сохранять набор ключевых слов компактным.

## `if` как выражение

Это одна из важнейших особенностей управляющих конструкций YaoXiang: **`if` может использоваться как выражение и возвращать значение**.

```yaoxiang
# if-выражение: значения веток присваиваются в result
result = if x > 0 {
    "Положительное"
} elif x < 0 {
    "Отрицательное"
} else {
    "Ноль"
}
# result теперь одно из: "Положительное", "Отрицательное" или "Ноль"
```

Когда `if` используется как выражение, типы возвращаемых значений всех веток должны совпадать:

```yaoxiang
score = 88

# Все ветки возвращают String — типы совпадают, всё корректно
grade = if score >= 90 {
    "A"
} elif score >= 80 {
    "B"
} elif score >= 60 {
    "C"
} else {
    "D"
}
println(grade)  # "B"
```

В блоке кода каждой ветки **значение последнего выражения становится возвращаемым значением этой ветки**. Можно также явно использовать `return`, но в ветках обычно достаточно просто написать выражение.

```yaoxiang
# Просто выражение — рекомендуемый способ
category = if age < 18 { "Несовершеннолетний" } else { "Взрослый" }

# Можно и явный return — эффект тот же
category = if age < 18 {
    return "Несовершеннолетний"
} else {
    return "Взрослый"
}
```

Если вы используете `if` только для проверки условия и не нуждаетесь в значении, он становится обычной инструкцией — полностью совместимой с выражением.

## Вложенные `if`

Внутри `if` можно писать ещё один `if`, чтобы обрабатывать многоуровневые проверки условий:

```yaoxiang
age = 25
has_ticket = true

if age >= 18 {
    if has_ticket {
        println("Добро пожаловать")
    } else {
        println("Сначала купите билет")
    }
} else {
    println("Несовершеннолетним нужен сопровождающий")
}
```

При вложенных выражениях в YaoXiang отсутствует неоднозначность «висячего else», свойственная языку C: каждый `else` всегда относится к ближайшему ещё непарному `if`.

## Комбинирование условий с булевыми операторами

В условиях можно использовать `and`, `or`, `not` для объединения нескольких проверок:

```yaoxiang
username = "admin"
password = "123456"

# and: оба условия должны выполняться
if username == "admin" and password == "123456" {
    println("Вход выполнен")
}

# or: достаточно выполнения хотя бы одного условия
if role == "admin" or role == "moderator" {
    println("Есть права администратора")
}

# not: отрицание
if not is_banned {
    println("Можно писать")
}

# Комбинированное использование
if (age >= 18 and age <= 60) or is_vip {
    println("Можно участвовать в мероприятии")
}
```

По приоритету операторов: `not` выше, чем `and`, а `and` выше, чем `or`. Если сомневаетесь — ставьте скобки, чтобы намерение было яснее.

## Итоги

| Пункт | Описание |
|------|------|
| Базовая структура | `if условие { ... } elif условие { ... } else { ... }` |
| `elif` | В YaoXiang используется `elif`, а не `else if` |
| Выражение | `if` может возвращать значение; типы всех веток должны совпадать |
| Возвращаемое значение ветки | Значение последнего выражения в блоке ветки становится возвращаемым |
| Вложенность | Внутри `if` можно писать ещё один `if`; неоднозначности висячего else нет |
| Булевы операторы | Комбинирование условий с помощью `and`, `or`, `not` |
