# Модель владения

YaoXiang не использует сборку мусора (GC) и не требует аннотаций времени жизни. Её безопасность работы с памятью основана на **пяти концепциях и одном градиенте**.

## Пять концепций и один градиент

```
Посмотреть /        Забрать         Общее          Скопировать     Системный
Изменить на месте                  владение                        уровень
    │              │              │              │              │
   &T            Move           ref          clone()        unsafe
  &mut T         Нулевое        Автоматически  Явное         *T
  Токен нулевого копирование     компилятором  глубокое      Ответственность
  размера                         (Rc/Arc)      копирование   пользователя
```

## Move: передача владения по умолчанию

В YaoXiang **присваивание = передача владения**. Это поведение по умолчанию, без копирования:

```yaoxiang
p = Point(1.0, 2.0)
p2 = p              // Move! Владение p передаётся p2
                    // После этого p больше недоступен

// Хотите изменить p2? Используйте mut
mut p3 = Point(3.0, 4.0)
shift(p3, 1.0, 1.0)    // Изменение на месте
```

Передача параметров и возврат из функций тоже являются Move:

```yaoxiang
// Параметр: передаётся через Move
process: (p: Point) -> Point = {
    p.transform()
    p                  // Возврат через Move — без копирования
}

// Вызов
p = Point(1.0, 2.0)
result = process(p)    // p передан через Move
```

## &T / &mut T: токены заимствования

Если вы не хотите забирать владение, а лишь временно «посмотреть» (`&T`) или «изменить на месте» (`&mut T`), компилятор автоматически создаёт **токен заимствования нулевого размера**:

```yaoxiang
data = [1, 2, 3, 4, 5]

// Компилятор автоматически передаёт токен &List(Int) — владение не забирается
print(data.len())    // 5
print(data)          // ✅ data по-прежнему доступна, мы лишь посмотрели
```

`&T` и `&mut T` — это **типы нулевого размера** — они существуют на этапе компиляции и исчезают во время выполнения. Вам не нужно вручную писать `&`; компилятор сам решает, что использовать, в зависимости от контекста:

```yaoxiang
// Доступ только для чтения → автоматически &T
print: (point: &Point) -> Void = {
    print("({point.x}, {point.y})")
}

// Изменяемый доступ → автоматически &mut T
shift: (point: &mut Point, dx: Float, dy: Float) -> Void = {
    point.x = point.x + dx
    point.y = point.y + dy
}

mut p = Point(1.0, 2.0)
print(p)                // Передаётся &Point
shift(p, 1.0, 1.0)      // Передаётся &mut Point
```

**Ключевое различие**: `&T` копируем (разделяемое неизменяемое владение), `&mut T` — нет (исключительное изменяемое владение). Это не особое правило — это два свойства типов.

## ref: совместное использование между областями видимости

Когда вам нужно **одновременно удерживать** значение в нескольких местах, используйте `ref`:

```yaoxiang
data = [1, 2, 3, 4, 5]

// ref создаёт разделяемое владение
shared = ref data

// Компилятор автоматически выбирает счётчик ссылок:
// - без пересечения задач → Rc (однопоточный счётчик ссылок)
// - пересечение задач → Arc (атомарный счётчик ссылок)
spawn {
    use(shared)    // Пересечение задач! Компилятор автоматически выберет Arc
}

// Вам не нужно знать разницу между Rc и Arc
// Компилятор выберет за вас
```

## clone(): явное глубокое копирование

Когда вам нужна независимая копия, явно вызовите `clone()`:

```yaoxiang
original = [1, 2, 3]
backup = original.clone()   // Глубокое копирование — независимая копия

// Изменения не влияют друг на друга
original[0] = 10
print(backup[0])    // 1 — не затронуто
```

`clone()` вызывается явно — вы чётко указываете, что хотите копию, в отличие от некоторых языков, где копирование выполняется по умолчанию.

## Без времени жизни

В YaoXiang нет времён жизни `'a`. Это проектное решение основано на ключевом наблюдении:

> Задача проверки конфликтов заимствования по сути эквивалентна верификации утверждений Хоара. Достаточно передать её конвейеру доказательств в проверщике типов — отдельный механизм проверки заимствований не нужен.

Вам не нужно указывать `'a` и разбираться во временах жизни — компилятор автоматически проверяет безопасность владения на этапе проверки типов.

## Без GC

Во всей модели владения нет сборки мусора. Момент освобождения всей памяти определяется на этапе компиляции:

- **После Move** → исходная переменная становится недоступной, RAII автоматически освобождает память
- **После ref** → память освобождается, когда счётчик ссылок достигает нуля
- **При выходе из области видимости** → стековые переменные освобождаются автоматически

Никаких пауз GC, никаких накладных расходов во время выполнения.

## Резюме

| Операция | Ключевое слово / синтаксис | Копирование? | Когда использовать |
|------|------------|--------|--------|
| Забрать владение | Поведение по умолчанию | Без копирования | Передача параметров, присваивание |
| Посмотреть | Автоматически `&T` | Токен нулевого размера | Доступ только для чтения |
| Изменить на месте | Автоматически `&mut T` | Токен нулевого размера | Изменяемый доступ |
| Разделяемое владение | `ref` | Счётчик ссылок | Между областями видимости / задачами |
| Явное копирование | `.clone()` | Глубокое копирование | Когда нужна независимая копия |
| Сырой указатель | `unsafe` + `*T` | Вручную | Системные операции |

**Запомните**: Move — по умолчанию, ref — для совместного использования, clone — исключение. Три правила — и прощайся с GC.
