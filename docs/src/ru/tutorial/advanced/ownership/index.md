# Модель владения

YaoXiang не использует сборку мусора (GC) и не требует аннотаций времени жизни. Её безопасность памяти основана на **пяти концепциях и одном градиенте**.

## Пять концепций, один градиент

```
Взглянуть/изменить на месте     Забрать         Разделяемое владение     Скопировать        Системный уровень
        │                          │                     │                     │                     │
       &T                        Move                  ref                 clone()             unsafe
      &mut T                    без копирования      автоматич. компилятора   явное глубокое        *T
   токен нулевого размера       по умолчанию         выбор Rc/Arc            копирование         ответств. польз.
```

## Move: передача владения по умолчанию

В YaoXiang **присваивание = передача владения**. Это поведение по умолчанию, без копирования:

```yaoxiang
p = Point(1.0, 2.0)
p2 = p              # Move! владение p передаётся p2
                    # после этого p больше нельзя читать

# Хотите изменить p2? Используйте mut
mut p3 = Point(3.0, 4.0)
shift(p3, 1.0, 1.0)    # изменение на месте
```

Передача параметров и возврат из функции также являются Move:

```yaoxiang
# Параметр: Move при передаче
process: (p: Point) -> Point = {
    p.transform()
    p                  # Move при возврате — без копирования
}

# Вызов
p = Point(1.0, 2.0)
result = process(p)    # p перемещён
```

## &T / &mut T: токены заимствования

Если вы не хотите забирать владение, а лишь временно «взглянуть» (`&T`) или «изменить на месте» (`&mut T`), компилятор автоматически создаёт **токены заимствования нулевого размера**:

```yaoxiang
data = [1, 2, 3, 4, 5]

# Компилятор автоматически передаёт токен &List(Int) — владение не забирается
println(data.len())    # 5
println(data)          # ✅ data всё ещё доступна, на неё лишь взглянули
```

`&T` и `&mut T` — это **типы нулевого размера**: они существуют во время компиляции и исчезают во время выполнения. Вам не нужно писать `&` вручную — компилятор автоматически решает по контексту использования:

```yaoxiang
# Доступ только для чтения → автоматически &T
print: (point: &Point) -> Void = {
    println("(${point.x}, ${point.y})")
}

# Изменяемая модификация → автоматически &mut T
shift: (point: &mut Point, dx: Float, dy: Float) -> Void = {
    point.x = point.x + dx
    point.y = point.y + dy
}

mut p = Point(1.0, 2.0)
print(p)                # передаётся &Point
shift(p, 1.0, 1.0)      # передаётся &mut Point
```

**Ключевое различие**: `&T` является копируемым (разделяемый доступ только для чтения), а `&mut T` — некопируемым (эксклюзивный изменяемый доступ). Это не специальное правило — это два свойства типа.

## ref: разделяемое владение между областями видимости

Когда вам нужно **одновременно владеть** значением в нескольких местах, используйте `ref`:

```yaoxiang
data = [1, 2, 3, 4, 5]

# ref создаёт разделяемое владение
shared = ref data

# Компилятор автоматически выбирает счётчик ссылок:
# - без пересечения задач → Rc (однопоточный подсчёт ссылок)
# - пересечение задач    → Arc (атомарный подсчёт ссылок)
spawn {
    use(shared)    # пересечение задач! компилятор автоматически выберет Arc
}

# Вам не нужно знать разницу между Rc и Arc
# компилятор выбирает автоматически
```

## clone(): явное глубокое копирование

Когда вам нужна независимая копия, явно вызовите `clone()`:

```yaoxiang
original = [1, 2, 3]
backup = original.clone()   # глубокое копирование — независимая копия

# Каждый массив независим
original[0] = 10
println(backup[0])    # 1 — не затронут
```

`clone()` является явным — вы точно указываете, что хотите копировать, в отличие от некоторых языков, где копирование происходит по умолчанию.

## Без времени жизни

YaoXiang не имеет времён жизни `'a`. Это дизайнерское решение исходит из ключевого наблюдения:

> Задача конфликтов заимствования по сути эквивалентна верификации утверждений Хоара. Её можно решать единообразно через конвейер доказательств проверщика типов, без необходимости в отдельном фреймворке проверки заимствований.

Вам не нужно аннотировать `'a` и не нужно понимать времена жизни — компилятор автоматически проверяет безопасность владения на этапе проверки типов.

## Без GC

Вся модель владения обходится без сборки мусора. Время освобождения всей памяти определяется во время компиляции:

- **После Move** → исходная переменная становится недоступной, RAII освобождает ресурс автоматически
- **После ref** → освобождение при обнулении счётчика ссылок
- **Конец области видимости** → переменные стека освобождаются автоматически

Нулевые паузы GC, нулевые накладные расходы во время выполнения.

## Итог

| Операция | Ключевое слово / синтаксис | Копирование? | Когда использовать |
|----------|---------------------------|--------------|---------------------|
| Забрать владение | поведение по умолчанию | без копирования | передача параметров, присваивание |
| Взглянуть | автоматически `&T` | токен нулевого размера | доступ только для чтения |
| Изменить на месте | автоматически `&mut T` | токен нулевого размера | изменяемая модификация |
| Разделяемое владение | `ref` | подсчёт ссылок | между областями видимости / между задачами |
| Явное копирование | `.clone()` | глубокое копирование | нужна независимая копия |
| Сырой указатель | `unsafe` + `*T` | вручную | системные операции |

**Запомните**: Move — по умолчанию, ref — для совместного владения, clone — исключение. Три правила — и прощай GC.
