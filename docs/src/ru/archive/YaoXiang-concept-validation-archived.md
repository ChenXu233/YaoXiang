## 1. Обзор языка

### 1.1 Цели проектирования

YaoXiang（爻象）— экспериментальный универсальный язык программирования, стремящийся объединить следующие характеристики:

- **Типы — это всё**: значения, функции, модули, generics — всё является типом, типы — первоклассные сущности
- **Математическая абстракция**: единая абстрактная структура на основе теории типов
- **Абстракции с нулевой стоимостью**: высокая производительность, без GC, модель владения обеспечивает безопасность памяти
- **Естественный синтаксис**: читаемость как у Python, приближённая к естественному языку
- **Незаметный асинхрон**: без явного await, компилятор обрабатывает автоматически
- **Дружественный к ИИ**: строгая структурированность, чёткий AST, лёгкость парсинга и модификации

### 1.2 Основная философия дизайна

```
┌─────────────────────────────────────────────────────────────┐
│                    Философия дизайна YaoXiang               │
├─────────────────────────────────────────────────────────────┤
│  Всё — типы → Единая абстракция → Типы как данные → Доступно в runtime  │
│                                                              │
│  Модель владения → Абстракции с нулевой стоимостью → Без GC → Высокая производительность  │
│                                                              │
│  Синтаксис Python → Ощущение естественного языка → Читаемость → Дружелюбность для новичков  │
│                                                              │
│  Автоматический вывод → Минимум ключевых слов → Краткость → Дружественность к ИИ  │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 Позиционирование языка

| Измерение | Позиционирование |
|------|------|
| Парадигма | Мультипарадигма（функциональная + императивная + объектно-ориентированная）|
| Система типов | Зависимые типы + параметрический полиморфизм |
| Управление памятью | Владение + RAII（без GC）|
| Модель компиляции | AOT-компиляция（опционально JIT）|
| Целевые сценарии | Системное программирование, разработка приложений, ИИ-辅助编程 |

## 2. Верификация основных концепций

### 2.1 Выполнимость «всё — типы»

#### Теоретическое обоснование

В теории типов типы можно рассматривать как утверждения, а значения — как доказательства. Этот изоморфизм Карри — Ховарда раскрывает глубокую связь между типами и значениями. YaoXiang распространяет эту идею до предела:

```
Значения — экземпляры типов
Типы — экземпляры типов (метатипы)
Функции — отображения входных типов в выходные
Модули — композиции типов
Generics — фабрики типов
```

#### Примеры верификации

```yaoxiang
# Значения — экземпляры типов
x: Int = 42
# x — экземпляр типа Int

# Типы — экземпляры типов
MyList: type = List(Int)
# MyList — экземпляр type (метатипа)

# Функции — отображения между типами
add(Int, Int) -> Int = (a, b) => a + b
# add — экземпляр типа (Int, Int) -> Int

# Модули — композиции типов (модули используют файлы)
# Math.yx
pi: Float = 3.14159
sqrt(Float) -> Float = (x) => { ... }
# Модуль Math — это тип пространства имён
```

#### Вывод верификации

✅ **Выполнимо** — «всё — типы» имеет прочную математическую основу (теория типов, теория категорий), на практике может быть реализовано через единое представление типов.

### 2.2 Высокая производительность с зависимыми типами

#### Вызовы

Языки с зависимыми типами (Agda, Idris) обычно имеют низкую производительность, потому что:

1. Сложная проверка типов
2. Представление типов в runtime
3. Полная проверка исчерпываемости образца

#### Решение YaoXiang

```yaoxiang
# Стирание типов при компиляции (опционально)
# Информация о типах в runtime загружается по требованию

# Гарантия абстракций с нулевой стоимостью
identity<T>(T) -> T = (x) => x
# Компилируется в прямой возврат, без дополнительных накладных расходов

# Оптимизация на уровне типов
type Nat = { n: Int }
# Компилируется в обычный целочисленный тип, без дополнительной обёртки
```

#### Механизмы гарантии производительности

| Механизм | Описание |
|------|------|
| Мономорфизация | Generic-функции при компиляции раскрываются в конкретные версии |
| Встраивание (inline) | Простые функции автоматически встраиваются |
| Размещение в стеке | Малые объекты по умолчанию размещаются в стеке |
| Анализ побега | Большие объекты размещаются в куче |
| Условное стирание типов | Опциональная информация о типах в runtime |

#### Вывод верификации

✅ **Выполнимо** — через тщательно спроектированные стратегии компиляции можно сохранить возможности зависимых типов и при этом достичь высокой производительности.

### 2.3 Выполнимость незаметного асинхрона

#### Основная идея

```yaoxiang
# Модель автоматического await
# При вызове функции компилятор автоматически определяет асинхронные зависимости
# и вставляет соответствующие барьеры синхронизации

fetch_user(Int) -> User spawn = (id) => {
    database.query("SELECT * FROM users WHERE id = ?", id)
}

display_user(Int) -> String = (id) => {
    user = fetch_user(id)  # Автоматическое ожидание результата
    "User: " + user.name   # Гарантирует готовность user
}
```

#### Процесс автоматической обработки компилятором

```
Исходный код
   ↓
Проверка типов + Анализ асинхронных зависимостей
   ↓
Идентификация spawn-вызовов
   ↓
Генерация конечного автомата
   ↓
Автоматическая вставка await-точек
   ↓
Оптимизация барьеров синхронизации
   ↓
Целевой код
```

#### Вывод верификации

✅ **Выполнимо** — аналогично корутинам Kotlin и async/await в Rust, но с автоматическим управлением на этапе компиляции, что снижает нагрузку на программиста.

## 3. Проектирование системы типов

### 3.1 Иерархия типов

```
┌─────────────────────────────────────────────────────────────┐
│                    Иерархия типов YaoXiang                   │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  type (метатип)                                              │
│    │                                                         │
│    ├── 原类型 (Primitive Types)                              │
│    │   ├── Void                                              │
│    │   ├── Bool                                              │
│    │   ├── Int (8/16/32/64/128)                             │
│    │   ├── Uint (8/16/32/64/128)                            │
│    │   ├── Float (32/64)                                    │
│    │   ├── Char, String                                     │
│    │   └── Bytes                                            │
│    │                                                         │
│    ├── 复合类型 (Composite Types)                             │
│    │   ├── struct { fields }                                │
│    │   ├── union { variants }                               │
│    │   ├── enum { variants }                                │
│    │   ├── tuple (T1, T2, ...)                              │
│    │   ├── list [T], dict [K->V]                            │
│    │   └── option [T]                                       │
│    │                                                         │
│    ├── Функциональные типы (Function Types)                  │
│    │   fn (T1, T2, ...) -> R                                │
│    │                                                         │
│    ├── Обобщённые типы (Generic Types)                       │
│    │   List[T], Map[K, V], etc.                             │
│    │                                                         │
│    ├── Зависимые типы (Dependent Types)                      │
│    │   type { n: Nat } -> type                              │
│    │   Vec[n: Nat, T]                                       │
│    │                                                         │
│    └── Типы модулей (Module Types)                           │
│        mod { exports }                                      │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 3.4 Информация о типах в runtime

```yaoxiang
# Рефлексия типов
fn describe(t: type) -> String {
    match t {
        struct { fields } -> "Struct with " + fields.length + " fields"
        union { variants } -> "Union with " + variants.length + " variants"
        enum { variants } -> "Enum with " + variants.length + " cases"
        list { element } -> "List of " + element.name
        fn { params, ret } -> "Function: (" + params + ") -> " + ret
        primitive { name } -> "Primitive: " + name
    }
}

# Проверка типа
fn is_number(t: type) -> Bool {
    t == Int or t == Float or t == Number
}

# Проверка экземпляра типа
value: type = ...
if value has_type Int {
    print("It's an integer")
}

# Преобразование типов
fn safe_cast[T, U](value: T, target: type) -> option[U] {
    if value has_type target {
        some(value as U)
    } else {
        none
    }
}
```

## 4. Владение и модель памяти

### 4.1 Принципы владения

```yaoxiang
# По умолчанию — неизменяемая ссылка
process(ref Data) -> Void = (data) => {
    # data доступен только для чтения
    # нельзя изменять поля data
    # нельзя передавать владение data
}

# Изменяемая ссылка
modify(mut Data) -> Void = (data) => {
    # можно изменять поля data
    # не может быть других активных ссылок
}

# Передача владения
consume(Data) -> Void = (data) => {
    # владение data передаётся внутрь
    # по завершении функции data уничтожается
}

# Заимствование с возвратом
borrow_field(ref Data) -> ref Field = (data) => ref data.field
```

### 4.2 Время жизни

```yaoxiang
# Явные аннотации времени жизни (для сложных случаев)
longest<'a>(&'a str, &'a str) -> &'a str = (s1, s2) => {
    if s1.length > s2.length { s1 } else { s2 }
}

# Автоматический вывод времени жизни
first<T>(ref List[T]) -> ref T = (list) => ref list[0]
```

### 4.3 Умные указатели

```yaoxiang
# Box — размещение в куче
heap_data: Box[List[Int]] = Box.new([1, 2, 3])

# Rc — подсчёт ссылок
shared: Rc[Data] = Rc.new(data)

# Arc — атомарный подсчёт ссылок (потокобезопасный)
thread_safe: Arc[Data] = Arc.new(data)

# RefCell — внутренняя изменяемость
internal_mut: RefCell[Data] = RefCell.new(data)
```

### 4.4 Гарантии безопасности памяти

```yaoxiang
# Проверки на этапе компиляции
unsafe_example() -> Void = () => {
    data: Data = ...
    ref1 = ref data
    ref2 = ref data  # Ошибка компиляции! Несколько активных ссылок

    mut_data = mut data
    ref_mut = ref mut_data
```

## 6. Проектирование синтаксиса

### 6.1 Ключевые слова (17 слов)

В YaoXiang определено 17 ключевых слов, которые зарезервированы и не могут использоваться как идентификаторы.

| # | Ключевое слово | Назначение | Пример |
|---|--------|------|------|
| 1 | `type` | Определение типа | `type Point = { x: Int, y: Int }` |
| 2 | `pub` | Публичный экспорт | `pub add(Int, Int) -> Int = ...` |
| 3 | `use` | Импорт модуля | `use std.io` |
| 4 | `spawn` | Метка асинхронности | `fetch(String) -> T spawn = ...` |
| 5 | `ref` | Неизменяемая ссылка | `process(ref Data) -> Void = ...` |
| 6 | `mut` | Изменяемая ссылка | `modify(mut Data) -> Void = ...` |
| 7 | `if` | Условное ветвление | `if x > 0 { ... }` |
| 8 | `elif` | Множественные условия | `elif x == 0 { ... }` |
| 9 | `else` | Блок по умолчанию | `else { ... }` |
| 10 | `match` | Образец匹配 | `match x { 0 -> "zero" }` |
| 11 | `while` | Цикл с условием | `while i < 10 { ... }` |
| 12 | `for` | Итерационный цикл | `for item in items { ... }` |
| 13 | `return` | Возврат значения | `return result` |
| 14 | `break` | Выход из цикла | `break` |
| 15 | `continue` | Продолжение цикла | `continue` |
| 16 | `as` | Преобразование типа | `x as Float` |
| 17 | `in` | Проверка принадлежности/列表推导式 | `x in [1, 2, 3]`, `[x * 2 for x in list]` |

**Альтернатива бесконечному циклу:**

```yaoxiang
# Использование while True вместо ключевого слова loop
while True {
    input = read_line()
    if input == "quit" {
        break
    }
    process(input)
}
```

### 6.2 Зарезервированные слова

Зарезервированные слова — это предопределённые языком специальные значения, которые нельзя использовать как идентификаторы, но они не являются ключевыми словами (не используются в синтаксических конструкциях).

| Зарезервированное слово | Тип | Описание |
|--------|------|------|
| `true` | Bool | Логическая истина |
| `false` | Bool | Логическая ложь |
| `null` | Void | Пустое значение |
| `none` | Option | Вариант без значения типа Option |
| `some(T)` | Option | Вариант со значением типа Option (функция) |
| `ok(T)` | Result | Вариант успеха типа Result (функция) |
| `err(E)` | Result | Вариант ошибки типа Result (функция) |

```yaoxiang
# Булевы значения
flag = true
flag = false

# Использование типа Option
maybe_value: option[String] = none
maybe_value = some("hello")

# Использование типа Result
result: result[Int, String] = ok(42)
result = err("error message")
```

### 6.3 Переменные и присваивание

```yaoxiang
# Автоматический вывод типа
x = 42                    # Int
name = "YaoXiang"         # String
pi = 3.14159              # Float
is_valid = true           # Bool

# Явные аннотации типа (опционально)

price: Float = 19.99

# Неизменяемость (по умолчанию)
x = 10
x = 20  # Ошибка компиляции!

# Изменяемые переменные
mut count = 0
count = count + 1  # OK
```

## 10. Риски и вызовы

### 10.1 Технические риски

| Риск | Вероятность | Влияние | Меры смягчения |
|------|--------|------|----------|
| Слишком долгая компиляция из-за зависимых типов | Средняя | Высокое | Инкрементная компиляция, кэширование |
| Сложная семантика автоматического await | Средняя | Среднее | Постепенная реализация |
| Кривая обучения модели владения | Низкая | Среднее | Дружественные подсказки компилятора |
| Слишком сложная система типов | Средняя | Высокое | Приоритет упрощённого подмножества |

### 10.2 Вызовы реализации

### 10.3 Риски проектирования языка

- **Риск**: Слишком мощная система типов может привести к увеличению времени компиляции
- **Смягчение**: Предоставить выбор режима проверки типов
- **Риск**: Ограничения синтаксиса могут повлиять на гибкость
- **Смягчение**: Сохранить простоту ядра, опциональные расширения
