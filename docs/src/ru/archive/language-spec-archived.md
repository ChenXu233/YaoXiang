## Глава 2: Лексическая структура

### 2.1 Исходные файлы

Исходные файлы YaoXiang должны использовать кодировку UTF-8. Исходные файлы обычно имеют расширение `.yx`.

### 2.2 Классификация лексем

| Категория | Описание | Пример |
|------|------|------|
| Идентификатор | Начинается с буквы или подчёркивания | `x`, `_private`, `my_var` |
| Ключевое слово | Зарезервированное слово языка | `Type`, `pub`, `use` |
| Литерал | Фиксированное значение | `42`, `"hello"`, `true` |
| Оператор | Знак операции | `+`, `-`, `*`, `/` |
| Разделитель | Синтаксический разделитель | `(`, `)`, `{`, `}`, `,` |

### 2.3 Ключевые слова

YaoXiang определяет минимальное количество ключевых слов:

```
pub    use    spawn
ref    mut    if     elif
else   match  while  for    return
break  continue as     in     unsafe
```

Эти ключевые слова имеют особое значение в любом контексте и не могут использоваться в качестве идентификаторов.

### 2.4 Зарезервированные слова

| Зарезервированное слово | Тип | Описание |
|--------|------|------|
| `Type` | Type | Метатип |
| `true` | Bool | Логическая истина |
| `false` | Bool | Логическая ложь |
| `void` | Void | Пустое значение |
| `some(T)` | Option | Значение variant типа Option |
| `ok(T)` | Result | Вариант успеха Result |
| `err(E)` | Result | Вариант ошибки Result |

### 2.5 Идентификаторы

Идентификаторы начинаются с буквы или подчёркивания, последующие символы могут быть буквами, цифрами или подчёркиваниями. Идентификаторы чувствительны к регистру.

Специальные идентификаторы:
- `_` используется как заполнитель, обозначающий игнорирование значения
- Идентификаторы, начинающиеся с подчёркивания, обозначают приватные члены

### 2.6 Литералы

#### 2.6.1 Целые числа

```
Decimal     ::= [0-9][0-9_]*
Octal       ::= 0o[0-7][0-7_]*
Hex         ::= 0x[0-9a-fA-F][0-9a-fA-F_]*
Binary      ::= 0b[01][01_]*
```

#### 2.6.2 Числа с плавающей точкой

```
Float       ::= [0-9][0-9_]* '.' [0-9][0-9_]* ([eE][+-]?[0-9][0-9_]*)?
```

#### 2.6.3 Строки

```
String      ::= '"' ([^"\\] | EscapeSequence)* '"'
Escape      ::= '\\' ([nrt'"\\] | UnicodeEscape)
Unicode     ::= 'u' '{' HexDigit+ '}'
```

#### 2.6.4 Коллекции

```
List        ::= '[' Expr (',' Expr)* ']'
Dict        ::= '{' String ':' Expr (',' String ':' Expr)* '}'
Set         ::= '{' Expr (',' Expr)* '}'
```

#### 2.6.5 List comprehension

```
ListComp    ::= '[' Expr 'for' Identifier 'in' Expr (',' Expr)* ('if' Expr)? ']'
```

#### 2.6.6 Проверка принадлежности

```
Membership  ::= Expr 'in' Expr
```

### 2.7 Комментарии

```
// Однострочный комментарий

/* Многострочный комментарий
   может занимать несколько строк */
```

### 2.8 Правила отступов

Код должен использовать отступ в 4 пробела, использование символа Tab запрещено. Это обязательное синтаксическое правило.

## Глава 3: Типы

### 3.1 Классификация типов

```
TypeExpr    ::= PrimitiveType
              | RecordType
              | EnumType
              | InterfaceType
              | TupleType
              | FnType
              | GenericType
              | TypeRef
              | TypeUnion
              | TypeIntersection
              | ConstrainedType
              | AssociatedType
```

### 3.2 Примитивные типы

| Тип | Описание | Размер по умолчанию |
|------|------|----------|
| `Type` | Метатип | 0 байт |
| `Void` | Пустое значение | 0 байт |
| `Bool` | Логическое значение | 1 байт |
| `Int` | Знаковое целое число | 8 байт |
| `Uint` | Беззнаковое целое число | 8 байт |
| `Float` | Число с плавающей точкой | 8 байт |
| `String` | Строка UTF-8 | Переменный |
| `Char` | Символ Unicode | 4 байта |
| `Bytes` | Сырые байты | Переменный |

Целые числа с битовой шириной: `Int8`, `Int16`, `Int32`, `Int64`, `Int128`
Числа с плавающей точкой с битовой шириной: `Float32`, `Float64`

### 3.3 Record types

**Унифицированный синтаксис**：`Name: Type = { field1: Type1, field2: Type2, ... }`

```
RecordType  ::= '{' FieldList? '}'
FieldList   ::= Field (',' Field)* ','?
Field       ::= Identifier ':' TypeExpr
            |  Identifier                 # Ограничение интерфейса
```

```yaoxiang
// Простой record type
Point: Type = { x: Float, y: Float }

// Пустой record type
Empty: Type = {}

// Обобщённый record type
Pair: Type[T] = { first: T, second: T }

// Record type с реализацией интерфейса
Point: Type = {
    x: Float,
    y: Float,
    Drawable,
    Serializable
}
```

**Правила**：
- Record types определяются с помощью фигурных скобок `{}`
- Имя поля непосредственно следует за ним через двоеточие и тип
- Имя интерфейса внутри тела типа означает реализацию этого интерфейса

#### 3.3.1 Значения полей по умолчанию

Поля типа могут иметь значения по умолчанию, которые можно не указывать при конструировании:

```yaoxiang
// Поля со значениями по умолчанию - необязательны при конструировании
Point: Type = {
    x: Float = 0,
    y: Float = 0
}

// Использование
Point()           # → Point(x=0, y=0)
Point(x=1)       # → Point(x=1, y=0)
Point(x=1, y=2) # → Point(x=1, y=2)

// Поля без значений по умолчанию - обязательны при конструировании
Point2: Type = {
    x: Float,
    y: Float
}

// Использование
Point2(x=1, y=2) # ✓
Point2()          # ✗ Ошибка
```

**Правила**：
- `field: Type = expression` → имеет значение по умолчанию, необязательно при конструировании
- `field: Type` → не имеет значения по умолчанию, обязательно при конструировании

#### 3.3.2 Встроенные привязки

Внутри определения типа можно напрямую привязывать методы:

```yaoxiang
// Способ 1: Привязка внешней функции
distance: (a: Point, b: Point) -> Float = { ... }
Point: Type = {
    x: Float = 0,
    y: Float = 0,
    distance = distance[0]    # Привязка к позиции 0
}
// Вызов：p1.distance(p2) → distance(p1, p2)

// Способ 2: Анонимная функция + привязка позиции
Point: Type = {
    x: Float = 0,
    y: Float = 0,
    distance: ((a: Point, b: Point) -> Float)[0] = ((a, b) => {
        dx = a.x - b.x
        dy = a.y - b.y
        return (dx * dx + dy * dy).sqrt()
    })
}
// Синтаксис：((params) => body)[position]
// Вызов：p1.distance(p2) → distance(p1, p2)
```

### 3.4 Перечисляемые типы (Variant types)

```
EnumType    ::= '{' Variant ('|' Variant)* '}'
Variant     ::= Identifier (':' TypeExpr)?
```

**Синтаксис**：`Name: Type = { Variant1 | Variant2(params) | ... }`

```yaoxiang
// Варианты без параметров
Color: Type = { red | green | blue }

// Варианты с параметрами
Option: Type[T] = { some(T) | none }

// Смешанный вариант
Result: Type[T, E] = { ok(T) | err(E) }

// Варианты без параметров эквивалентны конструкторам без параметров
Bool: Type = { true | false }
```

### 3.5 Интерфейсные типы

```
InterfaceType ::= '{' FnField (',' FnField)* ','?
FnField       ::= Identifier ':' FnType
FnType        ::= '(' ParamTypes? ')' '->' TypeExpr
```

**Синтаксис**：Интерфейс — это record type, все поля которого являются функциональными типами

```yaoxiang
// Определение интерфейса
Drawable: Type = {
    draw: (Surface) -> Void,
    bounding_box: () -> Rect
}

Serializable: Type = {
    serialize: () -> String
}

// Пустой интерфейс
EmptyInterface: Type = {}
```

**Реализация интерфейса**：Тип реализует интерфейс путём перечисления имён интерфейсов в конце определения

```yaoxiang
// Тип, реализующий интерфейс
Point: Type = {
    x: Float,
    y: Float,
    Drawable,        # Реализует интерфейс Drawable
    Serializable     # Реализует интерфейс Serializable
}
```

**Прямое присваивание интерфейсу**：Конкретный тип может напрямую присваиваться переменной интерфейсного типа (структурное подтипирование)

```yaoxiang
// Прямое присваивание (конкретный тип известен во время компиляции → вызов без накладных расходов)
d: Drawable = Circle(1)
d.draw(screen)        # После компиляции: прямой вызов circle_draw, без vtable

// Возврат из функции (конкретный тип неизвестен во время компиляции → вызов через vtable)
d: Drawable = get_shape()
d.draw(screen)        # Поиск метода через vtable

// Интерфейс как параметр функции
process: (d: Drawable) -> Void = d.draw(screen)
```

**Стратегия оптимизации компиляции**：

| Сценарий | Результат вывода | Способ вызова |
|------|----------|------|
| Прямое присваивание конкретного типа | Конкретный тип определён | Прямой вызов (без накладных расходов) |
| Возврат из функции | Неизвестен | vtable |
| Гетерогенная коллекция | Несколько типов | vtable |

### 3.7 Функциональные типы

### 3.8 Обобщённые типы

#### 3.8.1 Синтаксис обобщённых параметров

```
GenericType     ::= Identifier '[' TypeArgList ']'
TypeArgList     ::= TypeExpr (',' TypeExpr)* ','?
GenericParams   ::= '[' Identifier (',' Identifier)* ']'
                 |  '[' Identifier ':' TypeBound (',' Identifier ':' TypeBound)* ']'
TypeBound       ::= Identifier
                 |  Identifier '+' Identifier ('+' Identifier)*
```

#### 3.8.2 Определение обобщённых типов

```yaoxiang
// Базовый обобщённый тип
Option: Type[T] = {
    some: (T) -> Self,
    none: () -> Self
}

Result: Type[T, E] = {
    ok: (T) -> Self,
    err: (E) -> Self
}

List: Type[T] = {
    data: Array[T],
    length: Int,
    push: [T](self: List[T], item: T) -> Void,
    get: [T](self: List[T], index: Int) -> Option[T]
}
```

#### 3.8.3 Вывод типов

```yaoxiang
// Компилятор автоматически выводит обобщённые параметры
numbers: List[Int] = List(1, 2, 3)  # Компилятор выводит List[Int]
```

### 3.9 Ограничения типов

#### 3.9.1 Одиночное ограничение

```
ConstrainedType ::= '[' Identifier ':' TypeBound ']' TypeExpr
```

```yaoxiang
// Определение интерфейсного типа (как ограничения)
Clone: Type = {
    clone: (Self) -> Self
}

// Использование ограничения
clone: [T: Clone](value: T) -> T = value.clone()
```

#### 3.9.2 Множественные ограничения

```yaoxiang
// Синтаксис множественных ограничений
combine: [T: Clone + Add](a: T, b: T) -> T = {
    a.clone() + b
}

// Сортировка обобщённого контейнера
sort: [T: Clone + PartialOrd](list: List[T]) -> List[T] = {
    result = list.clone()
    quicksort(&mut result)
    return result
}
```

#### 3.9.3 Ограничения функционального типа

```yaoxiang
// Ограничения для функций высшего порядка
call_twice: [T, F: Fn() -> T](f: F) -> (T, T) = (f(), f())

compose: [A, B, C, F: Fn(A) -> B, G: Fn(B) -> C](a: A, f: F, g: G) -> C = g(f(a))
```

### 3.10 Ассоциированные типы

#### 3.10.1 Определение ассоциированного типа

```
AssociatedType ::= Identifier ':' TypeExpr
```

```yaoxiang
// Iterator trait (использование синтаксиса record type)
Iterator: Type[T] = {
    Item: T,                    # Ассоциированный тип
    next: (Self) -> Option[T],
    has_next: (Self) -> Bool
}

// Использование ассоциированного типа
collect: [T, I: Iterator[T]](iter: I) -> List[T] = {
    result = List[T]()
    while iter.has_next() {
        if let Some(item) = iter.next() {
            result.push(item)
        }
    }
    return result
}
```

#### 3.10.2 Обобщённые ассоциированные типы (GAT)

```yaoxiang
// Более сложные ассоциированные типы
```

### 3.11 Компиляционные обобщения

#### 3.11.1 Ограничения литеральных типов

```
LiteralType   ::= Identifier ':' Int          # Константа времени компиляции
CompileTimeFn ::= '[' Identifier ':' Int ']' '(' Identifier ')' '->' TypeExpr
```

**Основной дизайн**：Параметр обобщения `[n: Int]` + параметр значения `(n: n)` различают константу времени компиляции и значение времени выполнения.

```yaoxiang
// Факториал времени компиляции: параметр должен быть литералом, известным во время компиляции
factorial: [n: Int](n: n) -> Int = {
    match n {
        0 => 1,
        _ => n * factorial(n - 1)
    }
}

// Массив констант времени компиляции
StaticArray: Type[T, N: Int] = {
    data: T[N],      # Массив с размером, известным во время компиляции
    length: N
}

// Способ использования
arr: StaticArray[Int, factorial(5)]  # Компилятор вычисляет factorial(5) = 120 во время компиляции
```

#### 3.11.2 Массивы констант времени компиляции

```yaoxiang
// Использование для типов матриц
Matrix: Type[T, Rows: Int, Cols: Int] = {
    data: Array[Array[T, Cols], Rows]
}

// Проверка размерностей времени компиляции
identity_matrix: [T: Add + Zero + One, N: Int](size: N) -> Matrix[T, N, N] = {
    // ...
}
```

## Глава 3 (продолжение): Пояснения к дизайну синтаксиса

### 3.17 Связь именованных функций и Lambda

**Основное понимание**：Именованные функции и лямбда-выражения — это одно и то же! Единственное различие: именованная функция просто даёт имя лямбда-выражению.

```yaoxiang
// Эти два варианта по сути полностью идентичны
add: (a: Int, b: Int) -> Int = a + b           # Именованная функция (рекомендуется)
add: (a: Int, b: Int) -> Int = (a, b) => a + b  # Lambda форма (полностью эквивалентна)
```

**Модель синтаксического сахара**：

```
// Именованная функция = Lambda + имя
name: (Params) -> ReturnType = body

// По сути это
name: (Params) -> ReturnType = (params) => body
```

**Ключевой момент**：Когда сигнатура полностью объявляет типы параметров, имена параметров в заголовке лямбда становятся избыточными и могут быть опущены.

### 3.18 Правила области видимости параметров

**Параметры перекрывают внешние переменные**：Область видимости параметров в сигнатуре перекрывает тело функции, внутренняя область имеет более высокий приоритет.

```yaoxiang
x = 10  # Внешняя переменная
double: (x: Int) -> Int = x * 2  # ✅ Параметр x перекрывает внешний x, результат 20
```

### 3.19 Расположение аннотаций типов

Аннотации типов могут находиться в любом из следующих мест**, достаточно указать хотя бы в одном**:

| Расположение | Форма | Описание |
|----------|------|------|
| Только в сигнатуре | `double: (x: Int) -> Int = x * 2` | ✅ Рекомендуется |

#### 5.9.1 Семантика: каждая итерация связывает новое значение

Семантика цикла for в YaoXiang отличается от традиционных языков: **каждая итерация связывает новое значение, а не изменяет одну и ту же переменную**.

```yaoxiang
// Пример: for i in 1..5
for i in 1..5 {
    print(i)
}
```

**Процесс выполнения**：

| Итерация | Поведение переменной цикла |
|------|----------------|
| Первая | Создаётся новая привязка `i = 1`, выполняется тело, печатается 1 |
| Вторая | Создаётся новая привязка `i = 2` (предыдущая привязка уничтожена), выполняется тело, печатается 2 |
| Третья | Создаётся новая привязка `i = 3`, выполняется тело, печатается 3 |
| Четвёртая | Создаётся новая привязка `i = 4`, выполняется тело, печатается 4 |
| Конец | Тело цикла завершено, привязка уничтожается |

**Ключевой момент**：После завершения каждой итерации привязка, созданная в этой итерации, уничтожается. Следующая итерация — это совершенно новая привязка, не имеющая никакой связи с предыдущей.

#### 5.9.2 Разница между for и for mut

| Синтаксис | Изменяемость переменной цикла | Описание |
|------|----------------|------|
| `for i in 1..5` | Неизменяемая | В теле цикла нельзя изменять привязку |
| `for mut i in 1..5` | Изменяемая | В теле цикла можно изменять привязку |

```yaoxiang
// ✅ Допустимо: каждая итерация связывает новое значение, изменение не требуется
for i in 1..5 {
    print(i)  # Читаем значение i
}

// ❌ Ошибка: неизменяемая привязка, нельзя изменять
for i in 1..5 {
    i = i + 1  # Ошибка: нельзя изменять неизменяемую привязку
}

// ✅ Допустимо: использование for mut позволяет изменять привязку
for mut i in 1..5 {
    i = i + 1  # Разрешено изменение
}
```

#### 5.9.3 Проверка затенения

Переменная цикла for не может затенять существующую переменную во внешней области видимости:

```yaoxiang
// ❌ Ошибка: i уже объявлена во внешней области
i = 10
for i in 1..5 {
    print(i)
}

// ✅ Правильно: использовать другое имя переменной
i = 10
for j in 1..5 {
    print(j)
}
```

Код ошибки：`E2013 - Cannot shadow existing variable`

#### 5.9.4 Сравнение с другими языками

| Язык | Семантика переменной цикла for |
|------|------------------|
| YaoXiang | Каждая итерация связывает новое значение |
| Rust | Изменяет одну и ту же переменную (нужен mut) |
| Python | Изменяет одну и ту же переменную (mut не нужен) |
| C/C++ | Изменяет одну и ту же переменную (нужен указатель или ссылка) |

**Обоснование дизайна**：YaoXiang использует семантику привязки, потому что:
1. После завершения каждой итерации переменная в теле цикла уничтожается
2. Следующая итерация — это совершенно новая привязка
3. Это безопаснее, не нужно учитывать состояние между итерациями

## Глава 6: Функции

### 6.1 Унифицированная модель функций

**Основной синтаксис**：`name: type = value`

YaoXiang использует **унифицированную модель объявлений**: переменные, функции и методы используют одну и ту же форму `name: type = value`.

```
Declaration   ::= Identifier ':' Type '=' Expression
FunctionDef   ::= Identifier GenericParams? '(' Parameters? ')' '->' Type '=' (Expression | Block)
GenericParams ::= '[' Identifier (',' Identifier)* ']'
Parameters    ::= Parameter (',' Parameter)*
Parameter     ::= Identifier ':' TypeExpr
```

### 6.2 Объявление переменных

```yaoxiang
// Базовый синтаксис
x: Int = 42
name: String = "YaoXiang"
mut counter: Int = 0

// Вывод типов
y = 100  # Выводится как Int
```

### 6.3 Определение функций

#### 6.3.1 Полный синтаксис

```yaoxiang
// Имена параметров объявляются в сигнатуре
add: (a: Int, b: Int) -> Int = {
    return a + b
}

// Один параметр
inc: (x: Int) -> Int = x + 1

// Функция без параметров
main: () -> Void = {
    print("Hello")
}

// Тело из нескольких строк
calc: (x: Float, y: Float, op: String) -> Float = {
    return match op {
        "+" => x + y,
        "-" => x - y,
        _ => 0.0
    }
}
```

#### 6.3.2 Правила возврата

```yaoxiang
// Не-void тип возврата - необходимо использовать return
add: (a: Int, b: Int) -> Int = {
    return a + b
}

// Void тип возврата - return необязателен
print: (msg: String) -> Void = {
    // return не нужен
}

// Однострочное выражение - возвращаем значение напрямую, return не нужен
greet: (name: String) -> String = "Hello, ${name}!"
```

### 6.4 Обобщённые функции

```yaoxiang
// Определение обобщённой функции
map: [T, R](list: List[T], f: Fn(T) -> R) -> List[R] = {
    result = List[R]()
    for item in list {
        result.push(f(item))
    }
    return result
}

// Использование ограничений типов
clone: [T: Clone](value: T) -> T = value.clone()

// Несколько параметров типов
combine: [T, U](a: T, b: U) -> (T, U) = (a, b)
```

### 6.5 Определение методов

#### 6.5.1 Методы типа

**Синтаксис**：`Type.method: (self: Type, ...) -> Return = ...`

```yaoxiang
// Метод типа: привязан к определённому типу
Point.draw: (self: Point, surface: Surface) -> Void = {
    surface.plot(self.x, self.y)
}

Point.serialize: (self: Point) -> String = {
    return "Point(${self.x}, ${self.y})"
}

// Использование синтаксического сахара для методов
p: Point = Point(1.0, 2.0)
p.draw(screen)           # Синтаксический сахар → Point.draw(p, screen)
```

#### 6.5.2 Обычные методы

**Синтаксис**：`name: (Type, ...) -> Return = ...` (без привязки к типу)

```yaoxiang
// Обычный метод: не привязан к типу, существует как независимая функция
distance: (p1: Point, p2: Point) -> Float = {
    dx = p1.x - p2.x
    dy = p1.y - p2.y
    (dx * dx + dy * dy).sqrt()
}
```

### 6.6 Привязка методов

#### 6.6.1 Ручная привязка

**Синтаксис**：`Type.method = function[positions]`

```yaoxiang
// Привязка к позиции 0 (по умолчанию)
Point.distance = distance[0]

// Привязка к позиции 1
Point.transform = transform[1]

// Привязка к нескольким позициям
Point.scale = scale[0, 1]

// Использование заполнителя
Point.calc = func[0, _, 2]
```

#### 6.6.2 Автоматическая привязка через pub

Функции, объявленные с `pub`, компилятор автоматически привязывает к типам, определённым в том же файле:

```yaoxiang
// Объявление с pub, компилятор автоматически привязывает
pub distance: (p1: Point, p2: Point) -> Float = {
    dx = p1.x - p2.x
    dy = p1.y - p2.y
    (dx * dx + dy * dy).sqrt()
}

// Компилятор автоматически выводит:
// 1. Point определён в текущем файле
// 2. Параметр функции содержит Point
// 3. Выполняется Point.distance = distance[0]

// Вызов
d = distance(p1, p2)           # Функциональный стиль
d2 = p1.distance(p2)           # Синтаксический сахар ООП
```

### 6.7 Правила привязки методов

| Правило | Описание |
|------|------|
| Позиции начинаются с 0 | `func[0]` привязывается к первому параметру (индекс 0) |
| Максимальная позиция | Должна быть < количества параметров функции |
| Отрицательный индекс | `[-1]` означает последний параметр |
| Заполнитель | `_` пропускает эту позицию, значение предоставляется пользователем |

### 6.8 Поддержка каррирования

Привязка естественным образом поддерживает каррирование. Когда при вызове предоставлено меньше параметров, чем ожидается, возвращается функция, принимающая оставшиеся параметры:

```yaoxiang
// Исходная функция: 5 параметров
calculate: (scale: Float, a: Point, b: Point, x: Float, y: Float) -> Float = ...

// Привязка: Point.calc = calculate[1, 2]
// После привязки остались параметры: scale, x, y

// Сценарии вызова
p1.calc(2.0, 10.0, 20.0)       # Предоставлено 3 параметра → прямой вызов
p1.calc(2.0)                    # Предоставлен 1 параметр → возвращается (Float, Float) -> Float
p1.calc()                       # Предоставлено 0 параметров → возвращается (Float, Float, Float) -> Float
```

### 6.9 Spawn функции и аннотации

#### 6.9.1 Spawn функции

```
SpawnFn     ::= Identifier ':' FnType 'spawn' '=' Lambda
FnType      ::= '(' ParamTypes? ')' '->' TypeExpr ('@' Annotation)?
Annotation  ::= 'block' | 'eager'
```

**Аннотации функций**：

| Аннотация | Расположение | Поведение |
|------|------|------|
| `@block` | После типа возврата | Отключает оптимизацию параллелизма, полностью последовательное выполнение |
| `@eager` | После типа возврата | Принудительное энергичное вычисление |

**Примеры синтаксиса**：

```
// Spawn функция: может выполняться параллельно
fetch_data: (url: String) -> JSON spawn = { ... }

// @block синхронная функция: полностью последовательное выполнение
main: () -> Void @block = { ... }

// @eager энергичная функция: выполняется немедленно
compute: (n: Int) -> Int @eager = { ... }
```

#### 6.9.2 Spawn блок

Явно объявленная область параллелизма, задачи внутри блока выполняются параллельно:

```
SpawnBlock  ::= '(' Pattern (',' Pattern)* ')' '=' 'spawn' '{' Expr (',' Expr)* '}'
```

**Пример**：

```
// Spawn блок: явный параллелизм
(result_a, result_b) = spawn {
    parse(fetch("url1")),
    parse(fetch("url2"))
}
```

#### 6.9.3 Spawn цикл

Цикл с параллелизмом данных, тело цикла параллельно выполняется для всех элементов данных:

```
SpawnFor    ::= Identifier '=' 'spawn' 'for' Identifier 'in' Expr '{' Expr '}'
```

**Пример**：

```
// Spawn цикл: параллелизм данных
results = spawn for item in items {
    process(item)
}
```

#### 6.9.4 Оператор распространения ошибок

```
ErrorPropagate ::= Expr '?'
```

**Пример**：

```
process: (p: Point) -> Result[Data, Error] = {
    data = fetch_data()?      # Автоматическое распространение ошибки
    transform(data)?
}
```

## Глава 8: Управление памятью

### 8.1 Модель владения

YaoXiang использует **модель владения** для управления памятью, каждый значение имеет единственного владельца:

| Семантика | Описание | Синтаксис |
|------|------|------|
| **Move** | Семантика по умолчанию, передача владения | `p2 = p` |
| **ref** | Совместное использование (Arc с подсчётом ссылок) | `shared = ref p` |
| **clone()** | Явное копирование | `p2 = p.clone()` |

### 8.2 Семантика Move (по умолчанию)

```yaoxiang
// Присваивание = Move (нуль-копирование)
p: Point = Point(1.0, 2.0)
p2 = p              # Move, p становится недействительным

// Передача параметров функции = Move
process: (p: Point) -> Void = {
    // Владение p передаётся внутрь
}

// Возврат значения = Move
create: () -> Point = {
    p = Point(1.0, 2.0)
    return p        # Move, владение передаётся
}
```

### 8.3 Ключевое слово ref (Arc)

Ключевое слово `ref` создаёт **указатель с подсчётом ссылок** (Arc), используемый для безопасного совместного использования:

```yaoxiang
// Создание Arc
p: Point = Point(1.0, 2.0)
shared = ref p      # Arc, потокобезопасный

// Совместный доступ
spawn(() => print(shared.x))   # ✅ Безопасно

// Arc автоматически управляет временем жизни
// Когда shared выходит из области видимости, счётчик становится нулём и происходит освобождение
```

**Характеристики**：
- Потокобезопасный подсчёт ссылок
- Автоматическое управление временем жизни
- Безопасное пересечение границ spawn

### 8.4 Явное копирование clone()

```yaoxiang
// Явное копирование значения
p: Point = Point(1.0, 2.0)
p2 = p.clone()      # p и p2 независимы

// Оба могут изменяться без влияния друг на друга
p.x = 0.0           # ✅
p2.x = 0.0          # ✅
```

### 8.5 Блоки unsafe

Блоки `unsafe` позволяют использовать сырые указатели для системного программирования:

```yaoxiang
// Тип сырого указателя
PtrType ::= '*' TypeExpr

// Блок unsafe
UnsafeBlock ::= 'unsafe' '{' Stmt* '}'
```

**Пример**：

```yaoxiang
p: Point = Point(1.0, 2.0)

// Сырые указатели можно использовать только в блоках unsafe
unsafe {
    ptr: *Point = &p     # Получить сырой указатель
    (*ptr).x = 0.0       # Разыменование
}
```

**Ограничения**：
- Сырые указатели можно использовать только в блоках `unsafe`
- Пользователь гарантирует отсутствие висячих указателей и использования после освобождения
- Не участвуют в проверках Send/Sync

### 8.7 БНФ семантики владения

```bnf
// === Выражения владения ===

// Move (по умолчанию)
MoveExpr     ::= Expr

// ref Arc
RefExpr      ::= 'ref' Expr

// clone
CloneExpr    ::= Expr '.clone' '(' ')'

// === Сырые указатели (только unsafe) ===

PtrType       ::= '*' TypeExpr
UnsafeBlock   ::= 'unsafe' '{' Stmt* '}'
```

### 8.8 Ограничения Send / Sync

| Ограничение | Семантика | Описание |
|------|------|------|
| **Send** | Можно безопасно передавать между потоками | Значение можно переместить в другой поток |
| **Sync** | Можно безопасно совместно использовать между потоками | Неизменяемая ссылка может быть передана в другой поток |

**Автоматический вывод**：

```
// Правила вывода Send
Struct[T1, T2]: Send ⇐ T1: Send и T2: Send

// Правила вывода Sync
Struct[T1, T2]: Sync ⇐ T1: Sync и T2: Sync
```

**Ограничения типов**：

| Тип | Send | Sync | Описание |
|------|------|------|------|
| `T` (значение) | ✅ | ✅ | Неизменяемые данные |
| `ref T` | ✅ | ✅ | Arc потокобезопасен |
| `*T` | ❌ | ❌ | Сырой указатель небезопасен |

## Глава 8 (продолжение): Ограничения системы типов

### 8.7 Ограничения Send/Sync

YaoXiang использует ограничения типов в стиле Rust для обеспечения безопасности параллелизма:

| Ограничение | Семантика | Описание |
|------|------|------|
| **Send** | Можно безопасно передавать между потоками | Значение можно переместить в другой поток |
| **Sync** | Можно безопасно совместно использовать между потоками | Неизменяемая ссылка может быть передана в другой поток |

**Иерархия ограничений**：

```
Send ──► Можно безопасно передавать между потоками
  │
  └──► Sync ──► Можно безопасно совместно использовать между потоками
       │
       └──► Типы, удовлетворяющие Send + Sync, могут автоматически работать параллельно

Arc[T] реализует Send + Sync (потокобезопасный подсчёт ссылок)
Mutex[T] обеспечивает внутреннюю изменяемость
```

### 8.8 Потокобезопасные типы

| Тип | Семантика | Параллельная безопасность | Описание |
|------|------|----------|------|
| `T` | Неизменяемые данные | ✅ Безопасно | Тип по умолчанию, многозадачное чтение без гонок |
| `Ref[T]` | Изменяемая ссылка | ⚠️ Требуется синхронизация | Обозначен как изменяемый для параллелизма, компилятор проверяет использование блокировок |
| `Atomic[T]` | Атомарный тип | ✅ Безопасно | Низкоуровневые атомарные операции, безблокировочный параллелизм |
| `Mutex[T]` | Обёртка мьютекса | ✅ Безопасно | Автоматическая блокировка/разблокировка, гарантированная компилятором |
| `RwLock[T]` | Обёртка читатель-писатель | ✅ Безопасно | Оптимизация для сценариев с большим количеством чтения |

**Синтаксис**：

```
Mutex[T]    # Изменяемые данные, обёрнутые в мьютекс
Atomic[T]   # Атомарный тип (только для Int, Float и т.д.)
RwLock[T]   # Обёртка читатель-писатель
```

**Синтаксический сахар with**：

```
with mutex.lock() {
    // Критическая секция: защищена Mutex
    ...
}
```

## Глава 9: Обработка ошибок

### 9.1 Тип Result

```
Result: Type[T, E] = ok(T) | err(E)
```

**Конструкторы вариантов**：

| Вариант | Синтаксис | Описание |
|------|------|------|
| `ok(T)` | `ok(value)` | Успешное значение |
| `err(E)` | `err(error)` | Значение ошибки |

### 9.2 Тип Option

```
Option: Type[T] = some(T) | none
```

**Конструкторы вариантов**：

| Вариант | Синтаксис | Описание |
|------|------|------|
| `some(T)` | `some(value)` | Значение есть |
| `none` | `none` | Значения нет |

### 9.3 Распространение ошибок

```
ErrorPropagate ::= Expr '?'
```

Оператор `?` автоматически распространяет ошибки типа Result:

```
// При успехе возвращает значение, при неудаче возвращает err выше
data = fetch_data()?

// Эквивалентно
data = match fetch_data() {
    ok(v) => v
    err(e) => return err(e)
}
```

## Глава 10: Привязка методов

### 10.1 Обзор привязки

YaoXiang использует **чисто функциональный дизайн**, все операции реализуются через функции. Механизм привязки связывает функции с типами, позволяя вызывающему вызывать функцию как метод.

```
Объявление привязки ::= Тип '.' Идентификатор '=' Имя функции '[' Список позиций ']'
Список позиций ::= Позиция (',' Позиция)* ','?
Позиция     ::= Целое число (начиная с 0) | Отрицательное целое | Заполнитель
```

**Основные правила**：
- Индексы позиций начинаются с **0**
- По умолчанию привязывается к позиции **0** (первый параметр)
- Поддерживаются отрицательные индексы `[-1]` для обозначения последнего параметра
- Объединённая привязка нескольких позиций `[0, 1, 2]`
- Заполнитель `_` означает пропуск этой позиции

### 10.2 Синтаксис привязки

**Синтаксис**：
```
Type.method = func[position]
Type.method = func[0, 1, 2]    # Привязка к нескольким позициям
Type.method = func[0, _, 2]   # Использование заполнителя
Type.method = func[-1]        # Отрицательный индекс (последний параметр)
```

**Семантика**：
- `Type.method = func[0]` означает, что при вызове `obj.method(arg)`, `obj` привязывается к параметру 0 функции `func`
- Оставшиеся параметры заполняются в исходном порядке

### 10.3 Примеры привязки

```yaoxiang
// === Базовая привязка ===

// Исходная функция
distance: (a: Point, b: Point) -> Float = {
    dx = a.x - b.x
    dy = a.y - b.y
    (dx * dx + dy * dy).sqrt()
}

// Привязка к типу Point (позиция 0)
Point.distance = distance[0]

// Использование
p1 = Point(3.0, 4.0)
p2 = Point(1.0, 2.0)
d = p1.distance(p2)  # → distance(p1, p2)

// === Привязка к нескольким позициям ===

// Исходная функция
calculate: (scale: Float, a: Point, b: Point, x: Float, y: Float) -> Float = ...

// Привязка нескольких позиций
Point.calc_scale = calculate[0]      # Привязка только scale
Point.calc_both = calculate[1, 2]    # Привязка двух параметров Point

// Использование
f = p1.calc_scale(2.0)  # → calculate(2.0, p1, _, _, _)
result = f(p2, 10.0, 20.0)  # → calculate(2.0, p1, p2, 10.0, 20.0)

// === Каррирование (автоматический возврат функции при недостатке параметров) ===

// Привязка одного параметра
Point.distance_to = distance[0]

// Использование - не предоставлен второй параметр, возвращается каррированная функция
f = p1.distance_to(p2)  # → distance(p1, p2) прямой вызов
f2 = p1.distance_to()   # → distance(p1, _) возвращает функцию (Point) -> Float
result = f2(p2)         # → distance(p1, p2)
```

### 10.4 Правила привязки

**Правила позиций**：
| Правило | Описание |
|------|------|
| Позиции начинаются с 0 | `func[0]` привязывается к первому параметру (индекс 0) |
| Максимальная позиция | Должна быть < количества параметров функции |
| Отрицательный индекс | `[-1]` означает последний параметр |
| Заполнитель | `_` пропускает эту позицию |

**Проверка типов**：
```yaoxiang
// ✅ Допустимая привязка
Point.distance = distance[0]          # distance(Point, Point)
Point.calc = calculate[1, 2]          # calculate(Float, Point, Point, ...)

// ❌ Недопустимая привязка (ошибка компиляции)
Point.wrong = distance[5]             # 5 >= 2 (количество параметров)
Point.wrong = distance[0, 0]          # Повторяющаяся позиция (если не разрешено)
Point.wrong = distance[-2]            # -2 выходит за пределы диапазона
```

### 10.5 Автоматическая привязка

Для функций, определённых в модуле и имеющих первый параметр типа модуля, автоматически поддерживается синтаксис вызова метода:

```yaoxiang
// === Point.yx ===
Point: Type = { x: Float, y: Float }

// Первый параметр Point, автоматическая поддержка вызова метода
distance: (a: Point, b: Point) -> Float = { ... }
add: (a: Point, b: Point) -> Point = { ... }

// === main.yx ===
use Point

p1 = Point(3.0, 4.0)
p2 = Point(1.0, 2.0)

// ✅ Автоматическая привязка: p1.distance(p2) → distance(p1, p2)
d = p1.distance(p2)
// ✅ p1.add(p2) → add(p1, p2)
p3 = p1.add(p2)
```

**Правила автоматической привязки**：
- Функция определена в файле модуля
- Тип параметра 0 функции совпадает с именем модуля
- Функция должна быть `pub`, чтобы автоматически привязываться за пределами модуля

### 10.6 Связь привязки и каррирования

Привязка естественным образом поддерживает каррирование. Когда при вызове предоставлено меньше параметров, чем ожидается, возвращается функция, принимающая оставшиеся параметры:

```yaoxiang
// Исходная функция: 5 параметров
calculate: (scale: Float, a: Point, b: Point, x: Float, y: Float) -> Float = ...

// Привязка: Point.calc = calculate[1, 2]
// После привязки остались параметры: scale, x, y

// Сценарии вызова
p1.calc(2.0, 10.0, 20.0)              # Предоставлено 3 параметра → прямой вызов
p1.calc(2.0)                          # Предоставлен 1 параметр → возвращается (Float, Float) -> Float
p1.calc()                             # Предоставлено 0 параметров → возвращается (Float, Float, Float) -> Float
```
