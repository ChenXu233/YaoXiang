## Глава 1: Введение

### 1.1 Область применения

Этот документ определяет синтаксис и семантику языка программирования YaoXiang. Он является авторитетным справочником по языку, ориентированным на разработчиков компиляторов и инструментария.

Руководства и примеры кода см. в каталоге [tutorial/](../tutorial/).

### 1.2 Соответствие

Программа или реализация считается соответствующей спецификации YaoXiang, если она удовлетворяет всем правилам, определённым в этом документе.

## Глава 2: Лексическая структура

### 2.1 Исходный файл

Исходные файлы YaoXiang должны использовать кодировку UTF-8. Исходные файлы обычно имеют расширение `.yx`.

### 2.2 Классификация лексических единиц

| Категория | Описание | Пример |
|------|------|------|
| Идентификаторы | Начинаются с буквы или подчёркивания | `x`, `_private`, `my_var` |
| Ключевые слова | Зарезервированные слова, предопределённые языком | `Type`, `pub`, `use` |
| Литералы | Фиксированные значения | `42`, `"hello"`, `true` |
| Операторы | Символы операций | `+`, `-`, `*`, `/` |
| Разделители | Синтаксические разделители | `(`, `)`, `{`, `}`, `,` |

### 2.3 Ключевые слова

YaoXiang определяет минимальный набор ключевых слов:

```
pub    use    spawn
ref    mut    if     elif
else   match  while  for    return
break  continue as     in     unsafe
```

Эти ключевые слова имеют специальное значение в любом контексте и не могут использоваться в качестве идентификаторов.

### 2.4 Зарезервированные слова

| Зарезервированное слово | Тип | Описание |
|--------|------|------|
| `Type` | Type | мета-тип |
| `true` | Bool | булево значение истина |
| `false` | Bool | булево значение ложь |
| `void` | Void | пустое значение |
| `some(T)` | Option | value variant Option |
| `ok(T)` | Result | value variant Result (успех) |
| `err(E)` | Result | value variant Result (ошибка) |

### 2.5 Идентификаторы

Идентификаторы начинаются с буквы или подчёркивания, последующие символы могут быть буквами, цифрами или подчёркиваниями. Идентификаторы чувствительны к регистру.

Специальные идентификаторы:
- `_` используется как заполнитель, обозначающий игнорирование значения
- Идентификаторы, начинающиеся с подчёркивания, обозначают приватные члены

### 2.6 Литералы

#### 2.6.1 Целые числа

```
Decimal     ::= [0-9][0-9_]*
Octal       ::= 0o[0-7][0-7_]*
Hex         ::= 0x[0-9a-fA-F][0-9a-fA-F_]*
Binary      ::= 0b[01][01_]*
```

#### 2.6.2 Числа с плавающей точкой

```
Float       ::= [0-9][0-9_]* '.' [0-9][0-9_]* ([eE][+-]?[0-9][0-9_]*)?
```

#### 2.6.3 Строки

```
String      ::= '"' ([^"\\] | EscapeSequence)* '"'
Escape      ::= '\\' ([nrt'"\\] | UnicodeEscape)
Unicode     ::= 'u' '{' HexDigit+ '}'
```

#### 2.6.4 Коллекции

```
List        ::= '[' Expr (',' Expr)* ']'
Dict        ::= '{' String ':' Expr (',' String ':' Expr)* '}'
Set         ::= '{' Expr (',' Expr)* '}'
```

#### 2.6.5 List comprehension (генератор списков)

```
ListComp    ::= '[' Expr 'for' Identifier 'in' Expr (',' Expr)* ('if' Expr)? ']'
```

#### 2.6.6 Проверка принадлежности

```
Membership  ::= Expr 'in' Expr
```

### 2.7 Комментарии

```
// однострочный комментарий

/* многострочный комментарий
   может охватывать несколько строк */
```

### 2.8 Правила отступов

Код должен использовать отступ в 4 пробела; использование символов табуляции запрещено. Это обязательное синтаксическое правило.

## Глава 3: Типы

### 3.1 Классификация типов

```
TypeExpr    ::= PrimitiveType
              | StructType
              | EnumType
              | InterfaceType
              | TupleType
              | FnType
              | GenericType
              | TypeRef
              | TypeUnion
              | TypeIntersection
              | ConstrainedType
              | AssociatedType
```

### 3.2 Primitive type (примитивные типы)

| Тип | Описание | Размер по умолчанию |
|------|------|----------|
| `Type` | мета-тип | 0 байт |
| `Void` | пустое значение | 0 байт |
| `Bool` | булево значение | 1 байт |
| `Int` | знаковое целое | 8 байт |
| `Uint` | беззнаковое целое | 8 байт |
| `Float` | число с плавающей точкой | 8 байт |
| `String` | UTF-8 строка | переменный |
| `Char` | Unicode символ | 4 байта |
| `Bytes` | сырые байты | переменный |

Целые с указанием разрядности: `Int8`, `Int16`, `Int32`, `Int64`, `Int128`
Числа с плавающей точкой с указанием разрядности: `Float32`, `Float64`

### 3.3 Record type (записи)

**Единый синтаксис**: `Name: Type = { field1: Type1, field2: Type2, ... }`

```
RecordType  ::= '{' FieldList? '}'
FieldList   ::= Field (',' Field)* ','?
Field       ::= Identifier ':' TypeExpr
            |  Identifier                 # ограничение интерфейса
```

```yaoxiang
// Простой record type
Point: Type = { x: Float, y: Float }

// Пустой record type
Empty: Type = {}

// Record type с дженериками
Pair: Type[T] = { first: T, second: T }

// Record type, реализующий интерфейс
Point: Type = {
    x: Float,
    y: Float,
    Drawable,
    Serializable
}
```

**Правила**:
- Record type определяется с помощью фигурных скобок `{}`
- После имени поля сразу идёт двоеточие и тип
- Имя интерфейса, указанное в теле типа, означает реализацию этого интерфейса

#### 3.3.1 Значения полей по умолчанию

Поля типа могут иметь значения по умолчанию; при конструировании они могут быть указаны опционально:

```yaoxiang
// Поля со значениями по умолчанию - опциональны при конструировании
Point: Type = {
    x: Float = 0,
    y: Float = 0
}

// Использование
Point()           # → Point(x=0, y=0)
Point(x=1)       # → Point(x=1, y=0)
Point(x=1, y=2) # → Point(x=1, y=2)

// Поля без значений по умолчанию - обязательны при конструировании
Point2: Type = {
    x: Float,
    y: Float
}

// Использование
Point2(x=1, y=2) # ✓
Point2()          # ✗ Ошибка
```

**Правила**:
- `field: Type = expression` → есть значение по умолчанию, при конструировании опционально
- `field: Type` → нет значения по умолчанию, при конструировании обязательно

#### 3.3.2 Builtin binding (встроенные привязки)

В теле определения типа можно напрямую привязывать методы:

```yaoxiang
// Способ 1: Привязка через ссылку на внешнюю функцию
distance: (a: Point, b: Point) -> Float = { ... }
Point: Type = {
    x: Float = 0,
    y: Float = 0,
    distance = distance[0]    # Привязка к позиции 0
}
// Вызов: p1.distance(p2) → distance(p1, p2)

// Способ 2: Анонимная функция + привязка к позиции
Point: Type = {
    x: Float = 0,
    y: Float = 0,
    distance: ((a: Point, b: Point) -> Float)[0] = ((a, b) => {
        dx = a.x - b.x
        dy = a.y - b.y
        return (dx * dx + dy * dy).sqrt()
    })
}
// Синтаксис: ((params) => body)[position]
// Вызов: p1.distance(p2) → distance(p1, p2)
```

### 3.4 Enum type (перечисления, типы-варианты)

```
EnumType    ::= '{' Variant ('|' Variant)* '}'
Variant     ::= Identifier (':' TypeExpr)?
```

**Синтаксис**: `Name: Type = { Variant1 | Variant2(params) | ... }`

```yaoxiang
// Варианты без параметров
Color: Type = { red | green | blue }

// Варианты с параметрами
Option: Type[T] = { some(T) | none }

// Смешанные
Result: Type[T, E] = { ok(T) | err(E) }

// Вариант без параметров эквивалентен конструктору без параметров
Bool: Type = { true | false }
```

### 3.5 Interface type (интерфейсы)

```
InterfaceType ::= '{' FnField (',' FnField)* ','?
FnField       ::= Identifier ':' FnType
FnType        ::= '(' ParamTypes? ')' '->' TypeExpr
```

**Синтаксис**: Интерфейс — это record type, все поля которого являются типами функций

```yaoxiang
// Определение интерфейса
Drawable: Type = {
    draw: (Surface) -> Void,
    bounding_box: () -> Rect
}

Serializable: Type = {
    serialize: () -> String
}

// Пустой интерфейс
EmptyInterface: Type = {}
```

**Реализация интерфейса**: Тип реализует интерфейс, указывая его имя в конце определения

```yaoxiang
// Тип, реализующий интерфейсы
Point: Type = {
    x: Float,
    y: Float,
    Drawable,        # Реализация интерфейса Drawable
    Serializable     # Реализация интерфейса Serializable
}
```

**Прямое присваивание интерфейсу**: Конкретный тип может быть напрямую присвоен переменной типа интерфейса (структурная подтипизация)

```yaoxiang
// Прямое присваивание (конкретный тип определяется на этапе компиляции → вызов без накладных расходов)
d: Drawable = Circle(1)
d.draw(screen)        # После компиляции: прямой вызов circle_draw, без vtable

// Возвращаемое значение функции (неизвестно на этапе компиляции → вызов через vtable)
d: Drawable = get_shape()
d.draw(screen)        # Поиск метода через vtable

// Интерфейс как параметр функции
process: (d: Drawable) -> Void = d.draw(screen)
```

**Стратегии оптимизации на этапе компиляции**:

| Сценарий | Результат вывода | Способ вызова |
|------|----------|----------|
| Прямое присваивание конкретного типа | Конкретный тип определим | Прямой вызов (без накладных расходов) |
| Возвращаемое значение функции | Неизвестно | vtable |
| Гетерогенная коллекция | Несколько типов | vtable |

### 3.6 Tuple type (кортежи)

```
TupleType   ::= '(' TypeList? ')'
TypeList    ::= TypeExpr (',' TypeExpr)* ','?
```

### 3.7 Function type (типы функций)

```
FnType      ::= '(' ParamList? ')' '->' TypeExpr
ParamList   ::= TypeExpr (',' TypeExpr)*
```

### 3.8 Generic type (дженерики)

#### 3.8.1 Синтаксис параметров дженериков

```
GenericType     ::= Identifier '[' TypeArgList ']'
TypeArgList     ::= TypeExpr (',' TypeExpr)* ','?
GenericParams   ::= '[' Identifier (',' Identifier)* ']'
                 |  '[' Identifier ':' TypeBound (',' Identifier ':' TypeBound)* ']'
TypeBound       ::= Identifier
                 |  Identifier '+' Identifier ('+' Identifier)*
```

#### 3.8.2 Определение дженерик-типа

```yaoxiang
// Базовый дженерик-тип
Option: Type[T] = {
    some: (T) -> Self,
    none: () -> Self
}

Result: Type[T, E] = {
    ok: (T) -> Self,
    err: (E) -> Self
}

List: Type[T] = {
    data: Array[T],
    length: Int,
    push: [T](self: List[T], item: T) -> Void,
    get: [T](self: List[T], index: Int) -> Option[T]
}
```

#### 3.8.3 Type inference (вывод типов)

```yaoxiang
// Компилятор автоматически выводит параметры дженериков
numbers: List[Int] = List(1, 2, 3)  # Компилятор выводит List[Int]
```

### 3.9 Type constraint (ограничения типов)

#### 3.9.1 Single constraint (единичное ограничение)

```
ConstrainedType ::= '[' Identifier ':' TypeBound ']' TypeExpr
```

```yaoxiang
// Определение типа интерфейса (как ограничение)
Clone: Type = {
    clone: (Self) -> Self
}

// Использование ограничения
clone: [T: Clone](value: T) -> T = value.clone()
```

#### 3.9.2 Multiple constraint (множественные ограничения)

```yaoxiang
// Синтаксис множественных ограничений
combine: [T: Clone + Add](a: T, b: T) -> T = {
    return a.clone() + b
}

// Сортировка дженерик-контейнера
sort: [T: Clone + PartialOrd](list: List[T]) -> List[T] = {
    result = list.clone()
    quicksort(&mut result)
    return result
}
```

#### 3.9.3 Ограничения типа функции

```yaoxiang
// Ограничения для функций высшего порядка
call_twice: [T, F: Fn() -> T](f: F) -> (T, T) = (f(), f())

compose: [A, B, C, F: Fn(A) -> B, G: Fn(B) -> C](a: A, f: F, g: G) -> C = g(f(a))
```

### 3.10 Associated type (ассоциированные типы)

#### 3.10.1 Определение ассоциированного типа

```
AssociatedType ::= Identifier ':' TypeExpr
```

```yaoxiang
// Iterator trait (с использованием синтаксиса record type)
Iterator: Type[T] = {
    Item: T,                    # ассоциированный тип
    next: (Self) -> Option[T],
    has_next: (Self) -> Bool
}

// Использование ассоциированного типа
collect: [T, I: Iterator[T]](iter: I) -> List[T] = {
    result = List[T]()
    while iter.has_next() {
        if let Some(item) = iter.next() {
            result.push(item)
        }
    }
    return result
}
```

#### 3.10.2 Дженерик-ассоциированные типы (GAT)

```yaoxiang
// Более сложные ассоциированные типы
Container: Type[T] = {
    Item: T,
    IteratorType: Iterator[T],  # ассоциированный тип также дженерик
    iter: (Self) -> IteratorType
}
```

### 3.11 Compile-time generics (дженерики времени компиляции)

#### 3.11.1 Ограничения литеральных типов

```
LiteralType   ::= Identifier ':' Int          # константа времени компиляции
CompileTimeFn ::= '[' Identifier ':' Int ']' '(' Identifier ')' '->' TypeExpr
```

**Основная идея**: Использование параметра дженерика `[n: Int]` + параметра значения `(n: n)` для разделения констант времени компиляции и значений времени выполнения.

```yaoxiang
// Факториал времени компиляции: аргумент должен быть литералом, известным на этапе компиляции
factorial: [n: Int](n: n) -> Int = {
    return match n {
        0 => 1,
        _ => n * factorial(n - 1)
    }
}

// Массив с константой времени компиляции
StaticArray: Type[T, N: Int] = {
    data: T[N],      # Размер массива известен на этапе компиляции
    length: N
}

// Использование
arr: StaticArray[Int, factorial(5)]  # Компилятор вычисляет factorial(5) = 120 на этапе компиляции
```

#### 3.11.2 Массивы с константами времени компиляции

```yaoxiang
// Использование типа матрицы
Matrix: Type[T, Rows: Int, Cols: Int] = {
    data: Array[Array[T, Cols], Rows]
}

// Проверка размерности на этапе компиляции
identity_matrix: [T: Add + Zero + One, N: Int](size: N) -> Matrix[T, N, N] = {
    // ...
}
```

### 3.12 Conditional type (условные типы)

#### 3.12.1 Условный тип If

```
IfType        ::= 'If' '[' BoolExpr ',' TypeExpr ',' TypeExpr ']'
```

```yaoxiang
// Типовый If
If: Type[C: Bool, T, E] = match C {
    True => T,

// Пример: ветвление на этапе компиляции

### 3.14 Type intersection (пересечение типов)

```
TypeIntersection ::= TypeExpr '&' TypeExpr

**Синтаксис**: Пересечение типов `A & B` представляет тип, который одновременно удовлетворяет и A, и B.

// Базовая специализация: использование перегрузки функций (автоматический выбор компилятора)

## Глава 3 (продолжение): Пояснения по дизайну синтаксиса

### 3.17 Связь между именованными функциями и лямбда-выражениями

**Ключевое понимание**: Именованные функции и лямбда-выражения — это одно и то же! Единственное отличие в том, что именованная функция — это лямбда, которой дали имя.

```yaoxiang
// Эти две формы по сути полностью идентичны
add: (a: Int, b: Int) -> Int = a + b           # Именованная функция (рекомендуется)
add: (a: Int, b: Int) -> Int = (a, b) => a + b  # Лямбда-форма (полностью эквивалентна)
```

**Модель синтаксического сахара**:

```
// Именованная функция = Лямбда + имя
name: (Params) -> ReturnType = body

// По сути это
name: (Params) -> ReturnType = (params) => body
```

**Ключевой момент**: Когда в сигнатуре полностью объявлены типы параметров, имена параметров в заголовке лямбды становятся избыточными и могут быть опущены.

### 3.18 Правила области видимости параметров

**Параметры перекрывают внешние переменные**: Область видимости параметров в сигнатуре перекрывает тело функции; внутренняя область видимости имеет более высокий приоритет.

```yaoxiang
x = 10  # Внешняя переменная
double: (x: Int) -> Int = x * 2  # ✅ Параметр x перекрывает внешний x, результат равен 20
```

### 3.19 Расположение аннотаций типов

#### 5.9.1 Семантика: каждая итерация — это новое связывание

Семантика цикла for в YaoXiang отличается от традиционных языков: **каждая итерация — это новое связывание, а не изменение одной и той же переменной**.

```yaoxiang
// Пример: for i in 1..5
for i in 1..5 {
    print(i)
}
```

**Процесс выполнения**:

| Итерация | Поведение переменной цикла |
|------|----------------|
| 1-я | Создаётся новое связывание `i = 1`, выполняется тело цикла, печатается 1 |
| 2-я | Создаётся новое связывание `i = 2` (предыдущее связывание уничтожено), выполняется тело цикла, печатается 2 |
| 3-я | Создаётся новое связывание `i = 3`, выполняется тело цикла, печатается 3 |
| 4-я | Создаётся новое связывание `i = 4`, выполняется тело цикла, печатается 4 |
| Завершение | Тело цикла завершается, связывание уничтожается |

**Ключевой момент**: После завершения каждой итерации связывание, созданное в этой итерации, уничтожается. Следующая итерация — это совершенно новое связывание, никак не связанное с предыдущим.

#### 5.9.2 Разница между for и for mut

| Синтаксис | Изменяемость переменной цикла | Описание |
|------|----------------|------|
| `for i in 1..5` | Неизменяемая | Связывание нельзя изменять внутри тела цикла |
| `for mut i in 1..5` | Изменяемая | Связывание можно изменять внутри тела цикла |

```yaoxiang
// ✅ Допустимо: каждая итерация создаёт новое связывание, изменения не требуются
for i in 1..5 {
    print(i)  # Чтение значения i
}

// ❌ Ошибка: неизменяемое связывание нельзя изменять
for i in 1..5 {
    i = i + 1  # Ошибка: нельзя изменить неизменяемое связывание
}

// ✅ Допустимо: использование for mut разрешает изменение связывания
for mut i in 1..5 {
    i = i + 1  # Изменение разрешено
}
```

#### 5.9.3 Проверка затенения

Переменная цикла for не может затенять переменную, уже существующую во внешней области видимости:

```yaoxiang
// ❌ Ошибка: i уже объявлено во внешней области видимости
i = 10
for i in 1..5 {
    print(i)
}

// ✅ Правильно: использование другого имени переменной
i = 10
for j in 1..5 {
    print(j)
}
```

Код ошибки: `E2013 - Cannot shadow existing variable`

#### 5.9.4 Сравнение с другими языками

| Язык | Семантика переменной цикла for |
|------|------------------|
| YaoXiang | Каждая итерация — новое связывание |
| Rust | Изменение одной и той же переменной (требуется mut) |
| Python | Изменение одной и той же переменной (mut не требуется) |
| C/C++ | Изменение одной и той же переменной (требуется указатель или ссылка) |

**Обоснование дизайна**: YaoXiang использует семантику связывания, потому что:
1. После каждой итерации переменные внутри тела цикла уничтожаются
2. Следующая итерация — это совершенно новое связывание
3. Это безопаснее, не нужно учитывать состояние между итерациями

## Глава 6: Функции

### 6.1 Единая модель функций

**Основной синтаксис**: `name: type = value`

YaoXiang использует **унифицированную модель объявления**: переменные, функции и методы используют одну и ту же форму `name: type = value`.

```
Declaration   ::= Identifier ':' Type '=' Expression
FunctionDef   ::= Identifier GenericParams? '(' Parameters? ')' '->' Type '=' (Expression | Block)
GenericParams ::= '[' Identifier (',' Identifier)* ']'
Parameters    ::= Parameter (',' Parameter)*
Parameter     ::= Identifier ':' TypeExpr
```

### 6.2 Объявление переменных

```yaoxiang
// Базовый синтаксис
x: Int = 42
name: String = "YaoXiang"
mut counter: Int = 0

// Вывод типа
y = 100  # Выводится как Int
```

### 6.3 Определение функций

#### 6.3.1 Полный синтаксис

```yaoxiang
// Имена параметров объявляются в сигнатуре
add: (a: Int, b: Int) -> Int = {
    return a + b
}

// Один параметр
inc: (x: Int) -> Int = x + 1

// Функция без параметров
main: () -> Void = {
    print("Hello")
}

// Многострочное тело функции
calc: (x: Float, y: Float, op: String) -> Float = {
    return match op {
        "+" => x + y,
        "-" => x - y,
        _ => 0.0
    }
}
```

#### 6.3.2 Правила возврата

```yaoxiang
// Возвращаемый тип не Void - требуется использовать return
add: (a: Int, b: Int) -> Int = {
    return a + b
}

// Возвращаемый тип Void - использование return опционально
print: (msg: String) -> Void = {
    // return не требуется
}

// Однострочное выражение - возвращает значение напрямую, без return
greet: (name: String) -> String = "Hello, ${name}!"
```

### 6.4 Дженерик-функции

```yaoxiang
// Определение дженерик-функции
map: [T, R](list: List[T], f: Fn(T) -> R) -> List[R] = {
    result = List[R]()
    for item in list {
        result.push(f(item))
    }

}

// Использование ограничений дженериков
clone: [T: Clone](value: T) -> T = value.clone()

// Несколько параметров типа
combine: [T, U](a: T, b: U) -> (T, U) = (a, b)
```

### 6.5 Определение методов

#### 6.5.1 Методы типа

**Синтаксис**: `Type.method: (self: Type, ...) -> Return = ...`

```yaoxiang
// Метод типа: привязан к конкретному типу
Point.draw: (self: Point, surface: Surface) -> Void = {
    surface.plot(self.x, self.y)
}

Point.serialize: (self: Point) -> String = {
    return "Point(${self.x}, ${self.y})"
}

// Использование синтаксического сахара для методов
p: Point = Point(1.0, 2.0)
p.draw(screen)           # Синтаксический сахар → Point.draw(p, screen)
```

#### 6.5.2 Обычные методы

**Синтаксис**: `name: (Type, ...) -> Return = ...` (не привязан к типу)

```yaoxiang
// Обычный метод: не привязан к типу, существует как независимая функция
distance: (p1: Point, p2: Point) -> Float = {
    dx = p1.x - p1.x
    dy = p2.y - p2.y
    return (dx * dx + dy * dy).sqrt()
}
```

### 6.6 Привязка методов

#### 6.6.1 Ручная привязка

**Синтаксис**: `Type.method = function[positions]`

```yaoxiang
// Привязка к позиции 0 (по умолчанию)
Point.distance = distance[0]

// Привязка к позиции 1
Point.transform = transform[1]

// Привязка к нескольким позициям
Point.scale = scale[0, 1]

// Использование заполнителя
Point.calc = func[0, _, 2]
```

#### 6.6.2 Автоматическая привязка pub

Функции, объявленные с `pub`, автоматически привязываются компилятором к типам, определённым в том же файле:

```yaoxiang
// Объявление с pub, автоматическая привязка компилятором
pub distance: (p1: Point, p2: Point) -> Float = {
    dx = p1.x - p1.x
    dy = p2.y - p2.y
    return (dx * dx + dy * dy).sqrt()
}

// Компилятор автоматически выводит:
// 1. Point определён в текущем файле
// 2. Параметры функции содержат Point
// 3. Выполняется Point.distance = distance[0]

// Вызов
d = distance(p1, p2)           # Функциональная форма
d2 = p1.distance(p2)           # Синтаксический сахар ООП
```

### 6.7 Правила привязки методов

| Правило | Описание |
|------|------|
| Позиции начинаются с 0 | `func[0]` привязывает 1-й параметр (индекс 0) |
| Максимальная позиция | Должна быть < количества параметров функции |
| Отрицательные индексы | `[-1]` означает последний параметр |
| Заполнитель | `_` пропускает эту позицию, предоставляется пользователем |

### 6.8 Поддержка каррирования

Привязка естественным образом поддерживает каррирование. Когда при вызове передаётся меньше аргументов, чем осталось параметров, возвращается функция, принимающая оставшиеся параметры:

```yaoxiang
// Исходная функция: 5 параметров
calculate: (scale: Float, a: Point, b: Point, x: Float, y: Float) -> Float = ...

// Привязка: Point.calc = calculate[1, 2]
// После привязки оставшиеся параметры: scale, x, y

// Сценарии вызова
p1.calc(2.0, 10.0, 20.0)       # Передано 3 параметра → прямой вызов
p1.calc(2.0)                    # Передан 1 параметр → возвращает (Float, Float) -> Float
p1.calc()                       # Передано 0 параметров → возвращает (Float, Float, Float) -> Float
```

### 6.9 Функции spawn и аннотации

#### 6.9.1 Функция spawn (параллельная функция)

```
SpawnFn     ::= Identifier ':' FnType 'spawn' '=' Lambda
FnType      ::= '(' ParamTypes? ')' '->' TypeExpr ('@' Annotation)?
Annotation  ::= 'block' | 'eager'
```

**Аннотации функций**:

| Аннотация | Расположение | Поведение |
|------|------|------|
| `@block` | После возвращаемого типа | Отключает параллельную оптимизацию, полностью последовательное выполнение |
| `@eager` | После возвращаемого типа | Принудительное немедленное вычисление |

**Примеры синтаксиса**:

```
// Функция spawn: может выполняться параллельно
fetch_data: (url: String) -> JSON spawn = { ... }

// Функция @block: полностью последовательное выполнение
main: () -> Void @block = { ... }

// Функция @eager: немедленное выполнение
compute: (n: Int) -> Int @eager = { ... }
```

#### 6.9.2 Блок spawn

Явно объявленная область параллелизма, задачи внутри блока выполняются параллельно:

```
SpawnBlock  ::= '(' Pattern (',' Pattern)* ')' '=' 'spawn' '{' Expr (',' Expr)* '}'
```

**Пример**:

```
// Блок spawn: явный параллелизм
(result_a, result_b) = spawn {
    parse(fetch("url1")),
    parse(fetch("url2"))
}
```

#### 6.9.3 Цикл spawn

Цикл с параллелизмом по данным, тело цикла выполняется параллельно на всех элементах данных:

```
SpawnFor    ::= Identifier '=' 'spawn' 'for' Identifier 'in' Expr '{' Expr '}'
```

**Пример**:

```
// Цикл spawn: параллелизм по данным
results = spawn for item in items {
    process(item)
}
```

#### 6.9.4 Оператор error propagation (распространения ошибок)

```
ErrorPropagate ::= Expr '?'
```

**Пример**:

```
process: (p: Point) -> Result[Data, Error] = {
    data = fetch_data()?      # Автоматическое распространение ошибки
    return transform(data)?
}
```

## Глава 8: Управление памятью

### 8.1 Модель владения (ownership)

YaoXiang использует **модель владения** для управления памятью, каждое значение имеет единственного владельца:

| Семантика | Описание | Синтаксис |
|------|------|------|
| **Move** | Семантика по умолчанию, передача владения | `p2 = p` |
| **ref** | Общее владение (Arc со счётчиком ссылок) | `shared = ref p` |
| **clone()** | Явное копирование | `p2 = p.clone()` |

### 8.2 Семантика Move (по умолчанию)

```yaoxiang
// Присваивание = Move (без копирования)
p: Point = Point(1.0, 2.0)
p2 = p              # Move, p становится недействительным

// Передача параметров в функцию = Move
process: (p: Point) -> Void = {
    // Владение p передаётся внутрь
}

// Возвращаемое значение = Move
create: () -> Point = {
    p = Point(1.0, 2.0)
    return p        # Move, владение передаётся
}
```

### 8.3 Ключевое слово ref (Arc)

Ключевое слово `ref` создаёт **указатель со счётчиком ссылок** (Arc) для безопасного совместного использования:

```yaoxiang
// Создание Arc
p: Point = Point(1.0, 2.0)
shared = ref p      # Arc, потокобезопасный

// Совместный доступ
spawn(() => print(shared.x))   # ✅ Безопасно

// Arc автоматически управляет временем жизни
// Когда shared выходит из области видимости, счётчик обнуляется и память освобождается
```

**Особенности**:
- Потокобезопасный подсчёт ссылок
- Автоматическое управление временем жизни
- Безопасно跨越 границ spawn

### 8.4 Явное копирование clone()

```yaoxiang
// Явное копирование значения
p: Point = Point(1.0, 2.0)
p2 = p.clone()      # p и p2 независимы

// Оба могут быть изменены, не влияя друг на друга
p.x = 0.0           # ✅
p2.x = 0.0          # ✅
```

### 8.5 Блоки unsafe

Блоки `unsafe` разрешают использование сырых указателей для системного программирования:

```yaoxiang
// Тип сырого указателя
PtrType ::= '*' TypeExpr

// Блок unsafe
UnsafeBlock ::= 'unsafe' '{' Stmt* '}'
```

**Пример**:

```yaoxiang
p: Point = Point(1.0, 2.0)

// Сырые указатели могут использоваться только в блоках unsafe
unsafe {
    ptr: *Point = &p     # Получение сырого указателя
    (*ptr).x = 0.0       # Разыменование
}
```

**Ограничения**:
- Сырые указатели могут использоваться только в блоках `unsafe`
- Пользователь гарантирует отсутствие висячих указателей и использования после освобождения
- Не участвуют в проверке Send/Sync

### 8.7 BNF синтаксиса ownership

```bnf
// === Выражения ownership ===

// Move (по умолчанию)
MoveExpr     ::= Expr

// ref Arc
RefExpr      ::= 'ref' Expr

// clone
CloneExpr    ::= Expr '.clone' '(' ')'

// === Сырые указатели (только unsafe) ===

PtrType       ::= '*' TypeExpr
UnsafeBlock   ::= 'unsafe' '{' Stmt* '}'
```

### 8.8 Ограничения Send / Sync

| Ограничение | Семантика | Описание |
|------|------|------|
| **Send** | Можно безопасно передавать между потоками | Значение может быть перемещено в другой поток |
| **Sync** | Можно безопасно разделять между потоками | Неизменяемая ссылка может быть разделена между потоками |

**Автоматическое выведение**:

```
// Правила выведения Send
Struct[T1, T2]: Send ⇐ T1: Send и T2: Send

// Правила выведения Sync
Struct[T1, T2]: Sync ⇐ T1: Sync и T2: Sync
```

**Ограничения типов**:

| Тип | Send | Sync | Описание |
|------|------|------|------|
| `T` (значение) | ✅ | ✅ | Неизменяемые данные |
| `ref T` | ✅ | ✅ | Arc потокобезопасен |
| `*T` | ❌ | ❌ | Сырой указатель небезопасен |

## Глава 8 (продолжение): Ограничения системы типов

### 8.7 Ограничения Send/Sync

YaoXiang использует ограничения типов в стиле Rust для обеспечения безопасности параллелизма:

| Ограничение | Семантика | Описание |
|------|------|------|
| **Send** | Можно безопасно передавать между потоками | Значение может быть перемещено в другой поток |
| **Sync** | Можно безопасно разделять между потоками | Неизменяемая ссылка может быть разделена между потоками |

**Иерархия ограничений**:

```
Send ──► Можно безопасно передавать между потоками
  │
  └──► Sync ──► Можно безопасно разделять между потоками
       │
       └──► Типы, удовлетворяющие Send + Sync, могут автоматически быть параллельными

Arc[T] реализует Send + Sync (потокобезопасный подсчёт ссылок)
Mutex[T] обеспечивает внутреннюю изменяемость
```

### 8.8 Потокобезопасные типы

| Тип | Семантика | Потокобезопасность | Описание |
|------|------|----------|------|
| `T` | Неизменяемые данные | ✅ Безопасно | Тип по умолчанию, чтение несколькими задачами без гонок |
| `Ref[T]` | Изменяемая ссылка | ⚠️ Требуется синхронизация | Помечена как параллельно изменяемая, проверка блокировок на этапе компиляции |
| `Atomic[T]` | Атомарный тип | ✅ Безопасно | Низкоуровневые атомарные операции, параллелизм без блокировок |
| `Mutex[T]` | Обёртка мьютекса | ✅ Безопасно | Автоматическая блокировка/разблокировка, гарантия на этапе компиляции |
| `RwLock[T]` | Обёртка блокировки чтения-записи | ✅ Безопасно | Оптимизация для сценариев "много читателей, мало писателей" |

**Синтаксис**:

```
Mutex[T]    # Изменяемые данные, обёрнутые мьютексом
Atomic[T]   # Атомарный тип (только Int, Float и т.д.)
RwLock[T]   # Обёртка блокировки чтения-записи
```

**Синтаксический сахар with**:

```
with mutex.lock() {
    // Критическая секция: защищена Mutex
    ...
}
```

## Глава 9: Обработка ошибок

### 9.1 Тип Result

```
Result: Type[T, E] = ok(T) | err(E)
```

**Конструкторы вариантов**:

| Вариант | Синтаксис | Описание |
|------|------|------|
| `ok(T)` | `ok(value)` | Успешное значение |
| `err(E)` | `err(error)` | Значение ошибки |

### 9.2 Тип Option

```
Option: Type[T] = some(T) | none
```

**Конструкторы вариантов**:

| Вариант | Синтаксис | Описание |
|------|------|------|
| `some(T)` | `some(value)` | Есть значение |
| `none` | `none` | Нет значения |

### 9.3 Распространение ошибок

```
ErrorPropagate ::= Expr '?'
```

Оператор `?` автоматически распространяет ошибки типа Result:

```
// При успехе возвращает значение, при неудаче возвращает err наверх
data = fetch_data()?

// Эквивалентно
data = match fetch_data() {
    ok(v) => v
    err(e) => return err(e)
}
```

## Приложение B: Описание различий с реализацией кода

> В этом разделе описаны известные различия между спецификацией языка и текущей реализацией кода.

### B.1 Ключевые слова

| Ключевое слово | Статус в спецификации | Реализация в коде | Описание |
|--------|---------|---------|------|
| `struct` | Удалено | ❌ Отсутствует | Используется единый синтаксис `Name: Type = {...}` |
| `enum` | Удалено | ❌ Отсутствует | Используется синтаксис вариантов `Name: Type = { A \| B \| C }` |
| `type` | Удалено | ❌ Отсутствует | В качестве ключевого слова мета-типа используется `Type` (с заглавной буквы) |

### B.2 Синтаксические различия

| Элемент синтаксиса | Спецификация | Реализация в коде | Описание |
|---------|------|---------|------|
| Разделитель match arm | `->` | `=>` | Используется `=>` (FatArrow) |
| Определение функции | `name(types) -> type = (params) => body` | Две формы | Поддерживается тип-центричная `name: Type = (params) =>` |
| Interface type | `type Serializable = [ serialize() -> String ]` | ❌ Не реализовано | Синтаксис с квадратными скобками ожидает реализации |

### B.3 Ожидающие реализации функции

Следующие функции, описанные в спецификации, ещё не реализованы в коде:

| Функция | Приоритет | Описание |
|------|--------|------|
| Единый синтаксис типов `Name: Type = {...}` | P0 | RFC-010: единый синтаксис заменяет `type Name = ...` |
| Синтаксис типов с фигурными скобками | P0 | `Point: Type = { x: Float, y: Float }` |
| Interface type | P1 | `Drawable: Type = { draw() -> Void }` |
| List comprehension | P2 | `[x for x in list if condition]` |
| Распространение ошибок `?` | P1 | Автоматическое распространение ошибок типа Result |
| Ключевое слово `ref` | P1 | Совместное использование через Arc с подсчётом ссылок |
| Блок `unsafe` | P1 | Сырые указатели и системное программирование |
| Тип сырого указателя `*T` | P1 | Синтаксис типа сырого указателя |
| Семантика `clone()` | P1 | Явное копирование |
| Аннотация `@block` | P1 | Гарантия последовательного выполнения |
| Функция `spawn` | P1 | Маркер параллельной функции |
| Блок `spawn {}` | P1 | Явная область параллелизма |
| Цикл `spawn for` | P1 | Цикл с параллелизмом по данным |
| Ограничения Send/Sync | P2 | Проверка потокобезопасных типов |
| Типы Mutex/Atomic | P2 | Потокобезопасные типы данных |
| Визуализация графа ошибок | P3 | Отслеживание распространения параллельных ошибок |
| **Система дженерик-типов** | P1 | RFC-011 |
| Базовые дженерики `[T]` | P1 | Параметры дженерик-типов и мономорфизация |
| Ограничения типов `[T: Clone]` | P2 | Система единичных/множественных ограничений |
| Ассоциированные типы `Item: T` | P3 | Поддержка GAT |
| Дженерики времени компиляции `[N: Int]` | P3 | Ограничения литеральных типов |
| Условные типы `If[C, T, E]` | P3 | Вычисления на уровне типов |
| Специализация перегрузки функций | P2 | Специализация по платформе и типу |
| Синтаксис методов `Type.method` | P1 | `Point.draw: (...) -> ... = ...` |

### B.4 Не реализуемые функции

Следующие функции в стиле Rust **не будут реализованы**:

| Функция | Причина |
|------|------|
| Времена жизни `'a` | Нет концепции ссылок, времена жизни не нужны |
| Borrow checker | ref = Arc заменяет |
| Синтаксис заимствования `&T` | Используется семантика Move |
| Изменяемое заимствование `&mut T` | Используется mut + Move |

## Глава 10: Привязка методов

### 10.1 Обзор привязки

YaoXiang использует **чисто функциональный дизайн**, все операции реализованы через функции. Механизм привязки связывает функции с типами, позволяя вызывающей стороне вызывать функцию как метод.

```
Объявление привязки ::= Тип '.' Идентификатор '=' ИмяФункции '[' СписокПозиций ']'
СписокПозиций ::= Позиция (',' Позиция)* ','?
Позиция     ::= Целое (начиная с 0) | ОтрицательноеЦелое | Заполнитель
```

**Основные правила**:
- Индексы позиций начинаются с **0**
- По умолчанию привязка к позиции **0** (первый параметр)
- Поддерживаются отрицательные индексы `[-1]` для последнего параметра
- Многопозиционная комбинированная привязка `[0, 1, 2]`
- Заполнитель `_` указывает на пропуск этой позиции

### 10.2 Синтаксис привязки

**Синтаксис**:
```
Type.method = func[position]
Type.method = func[0, 1, 2]    # Многопозиционная привязка
Type.method = func[0, _, 2]   # Использование заполнителя
Type.method = func[-1]        # Отрицательный индекс (последний параметр)
```

**Семантика**:
- `Type.method = func[0]` означает, что при вызове `obj.method(arg)` объект `obj` привязывается к 0-му параметру `func`
- Остальные параметры заполняются в исходном порядке

### 10.3 Примеры привязки

```yaoxiang
// === Базовая привязка ===

// Исходная функция
distance: (a: Point, b: Point) -> Float = {
    dx = a.x - a.x
    dy = b.y - b.y
    (dx * dx + dy * dy).sqrt()
}

// Привязка к типу Point (позиция 0)
Point.distance = distance[0]

// Использование
p1 = Point(3.0, 4.0)
p2 = Point(1.0, 2.0)
d = p1.distance(p2)  # → distance(p1, p2)

// === Многопозиционная привязка ===

// Исходная функция
calculate: (scale: Float, a: Point, b: Point, x: Float, y: Float) -> Float = ...

// Привязка к нескольким позициям
Point.calc_scale = calculate[0]      # Привязка только scale
Point.calc_both = calculate[1, 2]    # Привязка двух параметров Point

// Использование
f = p1.calc_scale(2.0)  # → calculate(2.0, p1, _, _, _)
result = f(p2, 10.0, 20.0)  # → calculate(2.0, p1, p2, 10.0, 20.0)

// === Каррирование (автоматический возврат функции при нехватке аргументов)===

// Привязка одного параметра
Point.distance_to = distance[0]

// Использование - без передачи второго параметра, возвращается каррированная функция
f = p1.distance_to(p2)  # → distance(p1, p2) прямой вызов
f2 = p1.distance_to()   # → distance(p1, _) возвращает функцию (Point) -> Float
result = f2(p2)         # → distance(p1, p2)
```

### 10.4 Правила привязки

**Правила позиций**:
| Правило | Описание |
|------|------|
| Позиции начинаются с 0 | `func[0]` привязывает 1-й параметр (индекс 0) |
| Максимальная позиция | Должна быть < количества параметров функции |
| Отрицательные индексы | `[-1]` означает последний параметр |
| Заполнитель | `_` пропускает эту позицию, предоставляется пользователем |

**Проверка типов**:
```yaoxiang
// ✅ Допустимая привязка
Point.distance = distance[0]          # distance(Point, Point)
Point.calc = calculate[1, 2]          # calculate(Float, Point, Point, ...)

// ❌ Недопустимая привязка (ошибка компиляции)
Point.wrong = distance[5]             # 5 >= 2 (количество параметров)
Point.wrong = distance[0, 0]          # Повторяющиеся позиции (если не разрешено)
Point.wrong = distance[-2]            # -2 за пределами диапазона
```

### 10.5 Автоматическая привязка

Для функций, определённых в модуле, где первый параметр имеет тип, совпадающий с именем модуля, автоматически поддерживается синтаксис вызова метода:

```yaoxiang
// === Point.yx ===
Point: Type = { x: Float, y: Float }

// Первый параметр — Point, автоматически поддерживается вызов метода
distance: (a: Point, b: Point) -> Float = { ... }
add: (a: Point, b: Point) -> Point = { ... }

// === main.yx ===
use Point

p1 = Point(3.0, 4.0)
p2 = Point(1.0, 2.0)

// ✅ Автоматическая привязка: p1.distance(p2) → distance(p1, p2)
d = p1.distance(p2)
// ✅ p1.add(p2) → add(p1, p2)
p3 = p1.add(p2)
```

**Правила автоматической привязки**:
- Функция определена в файле модуля
- 0-й параметр функции соответствует имени модуля
- Функция должна быть `pub` для автоматической привязки за пределами модуля

### 10.6 Связь между привязкой и каррированием

Привязка естественным образом поддерживает каррирование. Когда при вызове передаётся меньше аргументов, чем осталось параметров, возвращается функция, принимающая оставшиеся параметры:

```yaoxiang
// Исходная функция: 5 параметров
calculate: (scale: Float, a: Point, b: Point, x: Float, y: Float) -> Float = ...

// Привязка: Point.calc = calculate[1, 2]
// После привязки оставшиеся параметры: scale, x, y

// Сценарии вызова
p1.calc(2.0, 10.0, 20.0)              # Передано 3 параметра → прямой вызов
p1.calc(2.0)                          # Передан 1 параметр → возвращает (Float, Float) -> Float
p1.calc()                             # Передано 0 параметров → возвращает (Float, Float, Float) -> Float
```
