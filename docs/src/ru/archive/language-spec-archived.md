## Глава 1：Введение

### 1.1 Область применения

В данном документе определены синтаксис и семантика языка программирования YaoXiang. Он служит авторитетным справочником по языку для разработчиков компиляторов и инструментов.

Учебные материалы и примеры кода приведены в [Руководстве по YaoXiang](../guide/YaoXiang-book.md) и в каталоге [tutorial/](../tutorial/).

Программа или реализация считается соответствующей спецификации YaoXiang, если она удовлетворяет всем правилам, определённым в данном документе.

## Глава 2：Лексическая структура

### 2.1 Исходные файлы

Исходные файлы YaoXiang должны использовать кодировку UTF-8. Исходные файлы обычно имеют расширение `.yx`.

### 2.2 Классификация лексем

| Категория | Описание | Примеры |
|-----------|----------|---------|
| Идентификаторы | Начинаются с буквы или подчёркивания | `x`, `_private`, `my_var` |
| Ключевые слова | Зарезервированные слова языка | `Type`, `pub`, `use` |
| Литералы | Постоянные значения | `42`, `"hello"`, `true` |
| Операторы | Знаки операций | `+`, `-`, `*`, `/` |
| Разделители | Синтаксические разделители | `(`, `)`, `{`, `}`, `,` |

### 2.3 Ключевые слова

YaoXiang определяет минимальное количество ключевых слов：

```
pub    use    spawn
ref    mut    if     elif
else   match  while  for    return
break  continue as     in     unsafe
```

Эти ключевые слова имеют особое значение в любом контексте и не могут использоваться в качестве идентификаторов.

### 2.4 Зарезервированные слова

| Зарезервированное слово | Тип | Описание |
|------------------------|-----|----------|
| `Type` | Type | Метатип |
| `true` | Bool | Логическая истина |
| `false` | Bool | Логическая ложь |
| `void` | Void | Пустое значение |
| `some(T)` | Option | Вариант значения Option |
| `ok(T)` | Result | Вариант успеха Result |
| `err(E)` | Result | Вариант ошибки Result |

### 2.5 Идентификаторы

Идентификаторы начинаются с буквы или подчёркивания, последующие символы могут быть буквами, цифрами или подчёркиваниями. Идентификаторы чувствительны к регистру.

Специальные идентификаторы：
- `_` используется как заполнитель, означающий игнорирование значения
- Идентификаторы, начинающиеся с подчёркивания, обозначают приватные члены

### 2.6 Литералы

#### 2.6.1 Целые числа

```
Decimal     ::= [0-9][0-9_]*
Octal       ::= 0o[0-7][0-7_]*
Hex         ::= 0x[0-9a-fA-F][0-9a-fA-F_]*
Binary      ::= 0b[01][01_]*
```

#### 2.6.2 Числа с плавающей точкой

```
Float       ::= [0-9][0-9_]* '.' [0-9][0-9_]* ([eE][+-]?[0-9][0-9_]*)?
```

#### 2.6.3 Строки

```
String      ::= '"' ([^"\\] | EscapeSequence)* '"'
Escape      ::= '\\' ([nrt'"\\] | UnicodeEscape)
Unicode     ::= 'u' '{' HexDigit+ '}'
```

#### 2.6.4 Коллекции

```
List        ::= '[' Expr (',' Expr)* ']'
Dict        ::= '{' String ':' Expr (',' String ':' Expr)* '}'
Set         ::= '{' Expr (',' Expr)* '}'
```

#### 2.6.5 Генератор списка

```
ListComp    ::= '[' Expr 'for' Identifier 'in' Expr (',' Expr)* ('if' Expr)? ']'
```

#### 2.6.6 Проверка вхождения

```
Membership  ::= Expr 'in' Expr
```

### 2.7 Комментарии

```
// Однострочный комментарий

/* Многострочный комментарий
   может занимать несколько строк */
```

### 2.8 Правила отступов

Код должен использовать отступ в 4 пробела, использование символа Tab запрещено. Это обязательное синтаксическое правило.

## Глава 3：Типы

### 3.1 Классификация типов

```
TypeExpr    ::= PrimitiveType
              | StructType
              | EnumType
              | InterfaceType
              | TupleType
              | FnType
              | GenericType
              | TypeRef
              | TypeUnion
              | TypeIntersection
              | ConstrainedType
              | AssociatedType
```

### 3.2 Примитивные типы

| Тип | Описание | Размер по умолчанию |
|-----|----------|---------------------|
| `Type` | Метатип | 0 байт |
| `Void` | Пустое значение | 0 байт |
| `Bool` | Логическое значение | 1 байт |
| `Int` | Знаковое целое число | 8 байт |
| `Uint` | Беззнаковое целое число | 8 байт |
| `Float` | Число с плавающей точкой | 8 байт |
| `String` | Строка UTF-8 | Переменный |
| `Char` | Символ Unicode | 4 байта |
| `Bytes` | Сырые байты | Переменный |

Целые числа с указанием разрядности：`Int8`, `Int16`, `Int32`, `Int64`, `Int128`
Числа с плавающей точкой с указанием разрядности：`Float32`, `Float64`

### 3.3 Типы записей

**Единый синтаксис**：`Name: Type = { field1: Type1, field2: Type2, ... }`

```
RecordType  ::= '{' FieldList? '}'
FieldList   ::= Field (',' Field)* ','?
Field       ::= Identifier ':' TypeExpr
            |  Identifier                 # Ограничение интерфейса
```

```yaoxiang
// Простой тип записи
Point: Type = { x: Float, y: Float }

// Пустой тип записи
Empty: Type = {}

// Тип записи с обобщениями
Pair: Type[T] = { first: T, second: T }

// Тип записи, реализующий интерфейсы
Point: Type = {
    x: Float,
    y: Float,
    Drawable,
    Serializable
}
```

**Правила**：
- Типы записей определяются с помощью фигурных скобок `{}`
- Имя поля непосредственно followed by двоеточие и тип
- Имя интерфейса внутри тела типа означает его реализацию

#### 3.3.1 Значения полей по умолчанию

Поля типа могут иметь значения по умолчанию, которые можно указать при конструировании：

```yaoxiang
// Поле со значением по умолчанию - необязательно при конструировании
Point: Type = {
    x: Float = 0,
    y: Float = 0
}

// Использование
Point()           # → Point(x=0, y=0)
Point(x=1)       # → Point(x=1, y=0)
Point(x=1, y=2) # → Point(x=1, y=2)

// Поле без значения по умолчанию - обязательно при конструировании
Point2: Type = {
    x: Float,
    y: Float
}

// Использование
Point2(x=1, y=2) # ✓
Point2()          # ✗ Ошибка
```

**Правила**：
- `field: Type = expression` → имеет значение по умолчанию, необязательно при конструировании
- `field: Type` → не имеет значения по умолчанию, обязательно при конструировании

#### 3.3.2 Встроенные привязки

Внутри определения типа можно напрямую привязывать методы：

```yaoxiang
// Способ 1: ссылка на внешнюю функцию
distance: (a: Point, b: Point) -> Float = { ... }
Point: Type = {
    x: Float = 0,
    y: Float = 0,
    distance = distance[0]    # Привязка к позиции 0
}
// Вызов：p1.distance(p2) → distance(p1, p2)

// Способ 2: анонимная функция + позиционная привязка
Point: Type = {
    x: Float = 0,
    y: Float = 0,
    distance: ((a: Point, b: Point) -> Float)[0] = ((a, b) => {
        dx = a.x - b.x
        dy = a.y - b.y
        return (dx * dx + dy * dy).sqrt()
    })
}
// Синтаксис：((params) => body)[position]
// Вызов：p1.distance(p2) → distance(p1, p2)
```

### 3.4 Перечисляемые типы（типы вариантов）

```
EnumType    ::= '{' Variant ('|' Variant)* '}'
Variant     ::= Identifier (':' TypeExpr)?
```

**Синтаксис**：`Name: Type = { Variant1 | Variant2(params) | ... }`

```yaoxiang
// Вариант без параметров
Color: Type = { red | green | blue }

// Вариант с параметрами
Option: Type[T] = { some(T) | none }

// Смешанный
Result: Type[T, E] = { ok(T) | err(E) }

// Вариант без параметров эквивалентен конструктору без параметров
Bool: Type = { true | false }
```

### 3.5 Типы интерфейсов

```
InterfaceType ::= '{' FnField (',' FnField)* ','?
FnField       ::= Identifier ':' FnType
FnType        ::= '(' ParamTypes? ')' '->' TypeExpr
```

**Синтаксис**：Интерфейс — это тип записи, все поля которого являются функциональными типами

```yaoxiang
// Определение интерфейса
Drawable: Type = {
    draw: (Surface) -> Void,
    bounding_box: () -> Rect
}

Serializable: Type = {
    serialize: () -> String
}

// Пустой интерфейс
EmptyInterface: Type = {}
```

**Реализация интерфейсов**：Тип реализует интерфейс, перечисляя имена интерфейсов в конце определения

```yaoxiang
// Тип, реализующий интерфейсы
Point: Type = {
    x: Float,
    y: Float,
    Drawable,        # Реализует интерфейс Drawable
    Serializable     # Реализует интерфейс Serializable
}
```

**Прямое присваивание интерфейсу**：Конкретный тип можно напрямую присвоить переменной типа интерфейса（структурное подтипирование）

```yaoxiang
// Прямое присваивание（конкретный тип определяется во время компиляции → вызов без накладных расходов）
d: Drawable = Circle(1)
d.draw(screen)        # После компиляции: прямой вызов circle_draw, без vtable

// Возврат функции（конкретный тип неизвестен во время компиляции → вызов через vtable）
d: Drawable = get_shape()
d.draw(screen)        # Поиск метода через vtable

// Интерфейс как параметр функции
process: (d: Drawable) -> Void = d.draw(screen)
```

**Стратегия оптимизации во время компиляции**：

| Сценарий | Результат вывода | Способ вызова |
|----------|------------------|---------------|
| Прямое присваивание конкретного типа | Конкретный тип определён | Прямой вызов（без накладных расходов） |
| Возврат функции | Неизвестен | vtable |
| Гетерогенная коллекция | Несколько типов | vtable |

### 3.8 Обобщённые типы

### 3.9 Ограничения типов

#### 3.9.1 Единичное ограничение

```
ConstrainedType ::= '[' Identifier ':' TypeBound ']' TypeExpr
```

```yaoxiang
// Определение типа интерфейса（в качестве ограничения）
Clone: Type = {
    clone: (Self) -> Self
}

// Использование ограничения
clone: [T: Clone](value: T) -> T = value.clone()
```

#### 3.9.2 Множественные ограничения

```yaoxiang
// Синтаксис множественных ограничений
combine: [T: Clone + Add](a: T, b: T) -> T = {
    a.clone() + b
}

// Сортировка обобщённого контейнера
sort: [T: Clone + PartialOrd](list: List[T]) -> List[T] = {
    result = list.clone()
    quicksort(&mut result)
    return result
}
```

#### 3.9.3 Ограничения функциональных типов

```yaoxiang
// Ограничения для функций высшего порядка

compose: [A, B, C, F: Fn(A) -> B, G: Fn(B) -> C](a: A, f: F, g: G) -> C = g(f(a))

### 3.10 Ассоциированные типы

#### 3.10.1 Определение ассоциированных типов

```

```

```yaoxiang
// Iterator trait（синтаксис типов записей）
Iterator: Type[T] = {
    Item: T,                    # Ассоциированный тип
    next: (Self) -> Option[T],
    has_next: (Self) -> Bool
}

// Использование ассоциированного типа
collect: [T, I: Iterator[T]](iter: I) -> List[T] = {
    result = List[T]()
    while iter.has_next() {
        if let Some(item) = iter.next() {
            result.push(item)
        }
    }
    return result
}
```

### 3.11 Обобщения времени компиляции

#### 3.11.1 Ограничения литеральных типов

```
LiteralType   ::= Identifier ':' Int          # Константа времени компиляции
CompileTimeFn ::= '[' Identifier ':' Int ']' '(' Identifier ')' '->' TypeExpr
```

**Основной дизайн**：Параметр обобщения `[n: Int]` + параметр значения `(n: n)` различают константу времени компиляции и значение времени выполнения.

```yaoxiang

    match n {
        0 => 1,
        _ => n * factorial(n - 1)
    }
}

// Массив констант времени компиляции
StaticArray: Type[T, N: Int] = {
    data: T[N],      # Массив с размером, известным во время компиляции
    length: N
}

// Использование
arr: StaticArray[Int, factorial(5)]  # Компилятор вычисляет factorial(5) = 120 во время компиляции
```

#### 3.11.2 Массивы констант времени компиляции

## Глава 3（продолжение）：Пояснения к дизайну синтаксиса

### 3.17 Отношение между именованными функциями и Lambda

**Основное понимание**：Именованные функции и лямбда-выражения — это одно и то же! Единственное различие заключается в том, что именованная функция даёт имя лямбда-выражению.

```yaoxiang
// По сути они полностью идентичны
add: (a: Int, b: Int) -> Int = a + b           # Именованная функция（рекомендуется）
add: (a: Int, b: Int) -> Int = (a, b) => a + b  # Lambda форма（полностью эквивалентно）
```

**Модель синтаксического сахара**：

```
// Именованная функция = Lambda + имя
name: (Params) -> ReturnType = body

// По сути
name: (Params) -> ReturnType = (params) => body
```

**Ключевой момент**：Когда сигнатура полностью объявляет типы параметров, имена параметров в заголовке лямбда-выражения становятся избыточными и могут быть опущены.

### 3.18 Правила области видимости параметров

**Параметры перекрывают внешние переменные**：Область видимости параметров в сигнатуре перекрывает тело функции, внутренняя область видимости имеет более высокий приоритет.

```yaoxiang
x = 10  # Внешняя переменная
double: (x: Int) -> Int = x * 2  # ✅ Параметр x перекрывает внешний x, результат 20
```

### 3.19 Расположение аннотаций типов

Аннотации типов могут располагаться в любом из следующих мест**, достаточно указать хотя бы в одном**：

| Расположение аннотации | Форма | Описание |

| В обоих местах | `double: (x: Int) -> Int = (x: Int) => x * 2` | ✅ Избыточно, но допустимо |

#### 5.9.1 Семантика: каждая итерация связывает новое значение

Семантика цикла for в YaoXiang отличается от традиционных языков：**каждая итерация связывает новое значение, а не модифицирует одну и ту же переменную**.

```yaoxiang
// Пример：for i in 1..5
for i in 1..5 {
    print(i)
}
```

**Процесс выполнения**：

| Итерация | Поведение переменной цикла |
|----------|----------------------------|
| 1-я | Создаётся новая связь `i = 1`, выполняется тело цикла, печатается 1 |
| 2-я | Создаётся новая связь `i = 2`（предыдущая связь уничтожена）, выполняется тело цикла, печатается 2 |
| 3-я | Создаётся новая связь `i = 3`, выполняется тело цикла, печатается 3 |
| 4-я | Создаётся новая связь `i = 4`, выполняется тело цикла, печатается 4 |
| Конец | Тело цикла завершено, связь уничтожена |

**Ключевой момент**：После завершения каждой итерации связь, созданная в этой итерации, уничтожается. Следующая итерация — это совершенно новая связь, не имеющая никакого отношения к предыдущей итерации.

#### 5.9.2 Разница между for и for mut

| Синтаксис | Изменяемость переменной цикла | Описание |
|-----------|-------------------------------|----------|
| `for i in 1..5` | Неизменяемая | Нельзя модифицировать связь в теле цикла |
| `for mut i in 1..5` | Изменяемая | Можно модифицировать связь в теле цикла |

```yaoxiang
// ✅ Допустимо: каждая итерация связывает новое значение, модификация не нужна
for i in 1..5 {
    print(i)  # Чтение значения i
}

// ❌ Ошибка: неизменяемая связь, нельзя модифицировать
for i in 1..5 {
    i = i + 1  # Ошибка: нельзя модифицировать неизменяемую связь
}

// ✅ Допустимо: использование for mut позволяет модифицировать связь
for mut i in 1..5 {
    i = i + 1  # Допускается модификация
}
```

#### 5.9.3 Проверка затенения

Переменная цикла for не может затенять переменную, уже существующую во внешней области видимости：

```yaoxiang
// ❌ Ошибка: i уже объявлена во внешней области
i = 10
for i in 1..5 {
    print(i)
}

// ✅ Правильно: использование другого имени переменной
i = 10
for j in 1..5 {
    print(j)
}
```

Код ошибки：`E2013 - Cannot shadow existing variable`

#### 5.9.4 Сравнение с другими языками

| Язык | Семантика переменной цикла for |
|------|--------------------------------|
| YaoXiang | Каждая итерация связывает новое значение |
| Rust | Модифицирует одну и ту же переменную（нужен mut） |
| Python | Модифицирует одну и ту же переменную（mut не нужен） |
| C/C++ | Модифицирует одну и ту же переменную（нужен указатель или ссылка） |

**Обоснование дизайна**：YaoXiang использует семантику связывания, потому что：
1. После завершения каждой итерации переменные внутри тела цикла уничтожаются
2. Следующая итерация — это совершенно новая связь
3. Это безопаснее, не нужно учитывать состояние между итерациями

## Глава 6：Функции

### 6.1 Единая модель функций

**Основной синтаксис**：`name: type = value`

YaoXiang использует **единую модель объявлений**：переменные, функции и методы используют одинаковую форму `name: type = value`.

```
Declaration   ::= Identifier ':' Type '=' Expression
FunctionDef   ::= Identifier GenericParams? '(' Parameters? ')' '->' Type '=' (Expression | Block)
GenericParams ::= '[' Identifier (',' Identifier)* ']'
Parameters    ::= Parameter (',' Parameter)*
Parameter     ::= Identifier ':' TypeExpr
```

### 6.2 Объявление переменных

```yaoxiang
// Базовый синтаксис
x: Int = 42
name: String = "YaoXiang"
mut counter: Int = 0

// Вывод типа
y = 100  # Выводится как Int
```

### 6.3 Определение функций

#### 6.3.1 Полный синтаксис

```yaoxiang
// Имена параметров объявляются в сигнатуре
add: (a: Int, b: Int) -> Int = {
    return a + b
}

// Один параметр
inc: (x: Int) -> Int = x + 1

// Функция без параметров
main: () -> Void = {
    print("Hello")
}

// Многострочное тело функции
calc: (x: Float, y: Float, op: String) -> Float = {
    return match op {
        "+" => x + y,
        "-" => x - y,
        _ => 0.0
    }
}
```

#### 6.3.2 Правила возврата

```yaoxiang
// Не Void тип возврата - необходимо использовать return
add: (a: Int, b: Int) -> Int = {
    return a + b
}

// Void тип возврата - return необязателен
print: (msg: String) -> Void = {
    // return не нужен
}

// Однострочное выражение - возвращает значение напрямую, return не нужен
greet: (name: String) -> String = "Hello, ${name}!"
```

### 6.4 Обобщённые функции

```yaoxiang
// Определение обобщённой функции
map: [T, R](list: List[T], f: Fn(T) -> R) -> List[R] = {
    result = List[R]()
    for item in list {
        result.push(f(item))
    }
    return result
}

// Использование ограничений обобщений
clone: [T: Clone](value: T) -> T = value.clone()

// Параметры нескольких типов
combine: [T, U](a: T, b: U) -> (T, U) = (a, b)
```

### 6.5 Определение методов

#### 6.5.1 Методы типа

**Синтаксис**：`Type.method: (self: Type, ...) -> Return = ...`

```yaoxiang
// Метод типа: привязан к конкретному типу
Point.draw: (self: Point, surface: Surface) -> Void = {
    surface.plot(self.x, self.y)
}

Point.serialize: (self: Point) -> String = {
    return "Point(${self.x}, ${self.y})"
}

// Использование синтаксического сахара для методов
p: Point = Point(1.0, 2.0)
p.draw(screen)           # Синтаксический сахар → Point.draw(p, screen)
```

#### 6.5.2 Обычные методы

**Синтаксис**：`name: (Type, ...) -> Return = ...`（не привязаны к типу）

```yaoxiang
// Обычный метод: не привязан к типу, существует как независимая функция
distance: (p1: Point, p2: Point) -> Float = {
    dx = p1.x - p2.x
    dy = p1.y - p2.y
    (dx * dx + dy * dy).sqrt()
}
```

### 6.6 Привязка методов

#### 6.6.1 Ручная привязка

**Синтаксис**：`Type.method = function[positions]`

```yaoxiang
// Привязка к позиции 0（по умолчанию）
Point.distance = distance[0]

// Привязка к позиции 1
Point.transform = transform[1]

// Привязка нескольких позиций
Point.scale = scale[0, 1]

// Использование заполнителя
Point.calc = func[0, _, 2]
```

#### 6.6.2 Автоматическая привязка через pub

Функции, объявленные с `pub`, автоматически привязываются компилятором к типам, определённым в том же файле：

```yaoxiang
// Объявление с pub, компилятор автоматически привязывает
pub distance: (p1: Point, p2: Point) -> Float = {
    dx = p1.x - p2.x
    dy = p1.y - p2.y
    (dx * dx + dy * dy).sqrt()
}

// Компилятор автоматически выводит：
// 1. Point определён в текущем файле
// 2. Параметры функции содержат Point
// 3. Выполняется Point.distance = distance[0]

// Вызов
d = distance(p1, p2)           # Функциональный стиль
d2 = p1.distance(p2)           # ООП синтаксический сахар
```

### 6.7 Правила привязки методов

| Правило | Описание |
|---------|----------|
| Позиции начинаются с 0 | `func[0]` привязывает первый параметр（индекс 0） |
| Максимальная позиция | Должна быть < количества параметров функции |
| Отрицательный индекс | `[-1]` обозначает последний параметр |
| Заполнитель | `_` пропускает эту позицию |

### 6.8 Поддержка каррирования

Привязка естественно поддерживает каррирование. Когда при вызове предоставлено меньше параметров, чем оставшихся, возвращается функция, принимающая оставшиеся параметры：

```yaoxiang
// Исходная функция: 5 параметров
calculate: (scale: Float, a: Point, b: Point, x: Float, y: Float) -> Float = ...

// Привязка：Point.calc = calculate[1, 2]
// После привязки оставшиеся параметры：scale, x, y

// Сценарии вызова
p1.calc(2.0, 10.0, 20.0)       # Предоставлено 3 параметра → прямой вызов
p1.calc(2.0)                    # Предоставлен 1 параметр → возврат (Float, Float) -> Float
p1.calc()                       # Предоставлено 0 параметров → возврат (Float, Float, Float) -> Float
```

### 6.9 Функции параллельного выполнения и аннотации

#### 6.9.1 Функции spawn（функции параллельного выполнения）

```
SpawnFn     ::= Identifier ':' FnType 'spawn' '=' Lambda
FnType      ::= '(' ParamTypes? ')' '->' TypeExpr ('@' Annotation)?
Annotation  ::= 'block' | 'eager'
```

**Аннотации функций**：

| Аннотация | Расположение | Поведение |
|-----------|--------------|-----------|
| `@block` | После типа возврата | Отключает оптимизацию параллелизма, полностью последовательное выполнение |
| `@eager` | После типа возврата | Принудительное энергичное вычисление |

**Примеры синтаксиса**：

```
// Функция spawn: может выполняться параллельно
fetch_data: (url: String) -> JSON spawn = { ... }

// @block синхронная функция: полностью последовательное выполнение
main: () -> Void @block = { ... }

// @eager энергичная функция: немедленное выполнение
compute: (n: Int) -> Int @eager = { ... }
```

#### 6.9.2 Блок spawn

Явно объявленная область параллелизма, задачи внутри блока выполняются параллельно：

```
SpawnBlock  ::= '(' Pattern (',' Pattern)* ')' '=' 'spawn' '{' Expr (',' Expr)* '}'
```

**Пример**：

```
// Блок spawn: явный параллелизм
(result_a, result_b) = spawn {
    parse(fetch("url1")),
    parse(fetch("url2"))
}
```

#### 6.9.3 Цикл spawn

Цикл с параллелизмом данных, тело цикла выполняется параллельно для всех элементов данных：

```
SpawnFor    ::= Identifier '=' 'spawn' 'for' Identifier 'in' Expr '{' Expr '}'
```

**Пример**：

```
// Цикл spawn: параллелизм данных
results = spawn for item in items {
    process(item)
}
```

#### 6.9.4 Оператор распространения ошибок

```
ErrorPropagate ::= Expr '?'
```

**Пример**：

```
process: (p: Point) -> Result[Data, Error] = {
    data = fetch_data()?      # Автоматическое распространение ошибки
    transform(data)?
}
```

## Глава 8：Управление памятью

### 8.1 Модель владения

YaoXiang использует **модель владения** для управления памятью, каждое значение имеет единственного владельца：

| Семантика | Описание | Синтаксис |
|-----------|----------|-----------|
| **Move** | Семантика по умолчанию, передача владения | `p2 = p` |
| **ref** | Совместное использование（Arc с подсчётом ссылок） | `shared = ref p` |
| **clone()** | Явное копирование | `p2 = p.clone()` |

### 8.2 Семантика Move（по умолчанию）

```yaoxiang
// Присваивание = Move（нулевое копирование）
p: Point = Point(1.0, 2.0)
p2 = p              # Move, p становится недействительным

// Передача параметров функции = Move
process: (p: Point) -> Void = {
    // Владение p передаётся
}

// Возврат значения = Move
create: () -> Point = {
    p = Point(1.0, 2.0)
    return p        # Move, владение передаётся
}
```

### 8.3 Ключевое слово ref（Arc）

Ключевое слово `ref` создаёт **указатель с подсчётом ссылок**（Arc）для безопасного совместного использования：

```yaoxiang
// Создание Arc
p: Point = Point(1.0, 2.0)
shared = ref p      # Arc, потокобезопасный

// Совместный доступ
spawn(() => print(shared.x))   # ✅ Безопасно

// Arc автоматически управляет жизненным циклом
// При выходе shared из области видимости счётчик становится нулём и происходит освобождение
```

**Характеристики**：
- Потокобезопасный подсчёт ссылок
- Автоматическое управление жизненным циклом
- Безопасность через границы spawn

### 8.4 Явное копирование через clone()

```yaoxiang
// Явное копирование значения
p: Point = Point(1.0, 2.0)
p2 = p.clone()      # p и p2 независимы

// Оба могут быть изменены без влияния друг на друга
p.x = 0.0           # ✅
p2.x = 0.0          # ✅
```

### 8.5 Блоки кода unsafe

Блок кода `unsafe` позволяет использовать сырые указатели для системного программирования：

```yaoxiang
// Тип сырого указателя
PtrType ::= '*' TypeExpr

// Блок unsafe
UnsafeBlock ::= 'unsafe' '{' Stmt* '}'
```

**Пример**：

```yaoxiang
p: Point = Point(1.0, 2.0)

// Сырые указатели можно использовать только в блоках unsafe
unsafe {
    ptr: *Point = &p     # Получение сырого указателя
    (*ptr).x = 0.0       # Разыменование
}
```

**Ограничения**：
- Сырые указатели можно использовать только в блоках `unsafe`
- Пользователь гарантирует отсутствие висячих указателей и использования после освобождения
- Не участвуют в проверках Send/Sync

### 8.7 БНФ семантики владения

```bnf
// === Выражения владения ===

// Move（по умолчанию）
MoveExpr     ::= Expr

// ref Arc
RefExpr      ::= 'ref' Expr

// clone
CloneExpr    ::= Expr '.clone' '(' ')'

// === Сырые указатели（только unsafe）===

PtrType       ::= '*' TypeExpr
UnsafeBlock   ::= 'unsafe' '{' Stmt* '}'
```

### 8.8 Ограничения Send / Sync

| Ограничение | Семантика | Описание |
|-------------|-----------|----------|
| **Send** | Можно безопасно передавать между потоками | Значение можно переместить в другой поток |
| **Sync** | Можно безопасно совместно использовать между потоками | Неизменяемая ссылка может быть передана в другой поток |

**Автоматический вывод**：

```
// Правила вывода Send
Struct[T1, T2]: Send ⇐ T1: Send 且 T2: Send

// Правила вывода Sync
Struct[T1, T2]: Sync ⇐ T1: Sync 且 T2: Sync
```

**Ограничения типа**：

| Тип | Send | Sync | Описание |
|-----|------|------|----------|
| `T`（значение） | ✅ | ✅ | Неизменяемые данные |
| `ref T` | ✅ | ✅ | Arc потокобезопасен |
| `*T` | ❌ | ❌ | Сырой указатель небезопасен |

## Глава 8（продолжение）：Ограничения системы типов

### 8.7 Ограничения Send/Sync

YaoXiang использует ограничения типов в стиле Rust для обеспечения безопасности параллелизма：

| Ограничение | Семантика | Описание |
|-------------|-----------|----------|
| **Send** | Можно безопасно передавать между потоками | Значение можно переместить в другой поток |
| **Sync** | Можно безопасно совместно использовать между потоками | Неизменяемая ссылка может быть передана в другой поток |

**Иерархия ограничений**：

```
Send ──► Можно безопасно передавать между потоками
  │
  └──► Sync ──► Можно безопасно совместно использовать между потоками
       │
       └──► Типы, удовлетворяющие Send + Sync, могут автоматически использовать параллелизм

Arc[T] реализует Send + Sync（потокобезопасный подсчёт ссылок）
Mutex[T] обеспечивает внутреннюю изменяемость
```

### 8.8 Типы для безопасности параллелизма

| Тип | Семантика | Безопасность параллелизма | Описание |
|-----|-----------|---------------------------|----------|
| `T` | Неизменяемые данные | ✅ Безопасно | Тип по умолчанию, чтение несколькими задачами без гонок |
| `Ref[T]` | Изменяемая ссылка | ⚠️ Нужна синхронизация | Отмечен как параллельно изменяемый, компилятор проверяет использование блокировок |
| `Atomic[T]` | Атомарный тип | ✅ Безопасно | Низкоуровневые атомарные операции, безблокировочный параллелизм |
| `Mutex[T]` | Обёртка мьютекса | ✅ Безопасно | Автоматическая блокировка/разблокировка, компилятор гарантирует |
| `RwLock[T]` | Обёртка блокировки чтения-записи | ✅ Безопасно | Оптимизация для сценариев с частым чтением и редкой записью |

**Синтаксис**：

```
Mutex[T]    # Обёртка мьютекса для изменяемых данных
Atomic[T]   # Атомарный тип（только для Int, Float и т.д.）
RwLock[T]   # Обёртка блокировки чтения-записи
```

**Синтаксический сахар with**：

```
with mutex.lock() {
    // Критическая секция: защищена Mutex
    ...
}
```

```

**Конструкторы вариантов**：

| Вариант | Синтаксис | Описание |
|---------|-----------|----------|
| `some(T)` | `some(value)` | Имеет значение |

### 9.3 Распространение ошибок

```
ErrorPropagate ::= Expr '?'
```

Оператор `?` автоматически распространяет ошибки типа Result：

```

data = fetch_data()?

// Эквивалентно
data = match fetch_data() {
    ok(v) => v
    err(e) => return err(e)
}
```

## Глава 10：Привязка методов

### 10.1 Обзор привязки

YaoXiang использует **чисто функциональный дизайн**, все операции реализуются через функции. Механизм привязки связывает функции с типами, позволяя вызывающему коду вызывать функции как методы.

```
Объявление привязки ::= Тип '.' Идентификатор '=' ИмяФункции '[' СписокПозиций ']'
СписокПозиций ::= Позиция (',' Позиция)* ','?
Позиция     ::= Целое число（начиная с 0） | Отрицательное целое | Заполнитель
```

**Основные правила**：
- Индексы позиций начинаются с **0**
- По умолчанию привязка к позиции **0**（первый параметр）
- Поддерживаются отрицательные индексы `[-1]` для обозначения последнего параметра
- Множественная совместная привязка `[0, 1, 2]`
- Заполнитель `_` означает пропуск этой позиции

### 10.2 Синтаксис привязки

**Синтаксис**：
```
Type.method = func[position]
Type.method = func[0, 1, 2]    # Привязка нескольких позиций
Type.method = func[0, _, 2]   # Использование заполнителя
Type.method = func[-1]        # Отрицательный индекс（последний параметр）
```

**Семантика**：
- `Type.method = func[0]` означает, что при вызове `obj.method(arg)`, `obj` привязывается к параметру `func` с индексом 0
- Оставшиеся параметры заполняются в исходном порядке

### 10.3 Примеры привязки

```yaoxiang
// === Базовая привязка ===

// Исходная функция
distance: (a: Point, b: Point) -> Float = {
    dx = a.x - b.x
    dy = a.y - b.y
    (dx * dx + dy * dy).sqrt()
}

// Привязка к типу Point（позиция 0）
Point.distance = distance[0]

// Использование
p1 = Point(3.0, 4.0)
p2 = Point(1.0, 2.0)
d = p1.distance(p2)  # → distance(p1, p2)

// === Привязка нескольких позиций ===

// Исходная функция
calculate: (scale: Float, a: Point, b: Point, x: Float, y: Float) -> Float = ...

// Привязка нескольких позиций
Point.calc_scale = calculate[0]      # Только привязка scale
Point.calc_both = calculate[1, 2]    # Привязка двух параметров Point

// Использование
f = p1.calc_scale(2.0)  # → calculate(2.0, p1, _, _, _)
result = f(p2, 10.0, 20.0)  # → calculate(2.0, p1, p2, 10.0, 20.0)

// === Каррирование（автоматический возврат функции при недостатке параметров）===

// Привязка одного параметра
Point.distance_to = distance[0]

// Использование - без указания второго параметра возвращается каррированная функция
f = p1.distance_to(p2)  # → distance(p1, p2) прямой вызов
f2 = p1.distance_to()   # → distance(p1, _) возврат функции (Point) -> Float
result = f2(p2)         # → distance(p1, p2)
```

### 10.4 Правила привязки

**Правила позиций**：
| Правило | Описание |
|---------|----------|
| Позиции начинаются с 0 | `func[0]` привязывает первый параметр（индекс 0） |
| Максимальная позиция | Должна быть < количества параметров функции |
| Отрицательный индекс | `[-1]` обозначает последний параметр |
| Заполнитель | `_` пропускает эту позицию |

**Проверка типов**：
```yaoxiang
// ✅ Допустимая привязка
Point.distance = distance[0]          # distance(Point, Point)
Point.calc = calculate[1, 2]          # calculate(Float, Point, Point, ...)

// ❌ Недопустимая привязка（ошибка компиляции）
Point.wrong = distance[5]             # 5 >= 2（количество параметров）
Point.wrong = distance[0, 0]          # Повторяющаяся позиция（если не разрешено）
Point.wrong = distance[-2]            # -2 выходит за пределы
```

### 10.5 Автоматическая привязка

Для функций, определённых в модуле и имеющих первый параметр типа модуля, автоматически поддерживается синтаксис вызова методов：

```yaoxiang
// === Point.yx ===
Point: Type = { x: Float, y: Float }

// Первый параметр - Point, автоматическая поддержка вызова методов
distance: (a: Point, b: Point) -> Float = { ... }
add: (a: Point, b: Point) -> Point = { ... }

// === main.yx ===
use Point

p1 = Point(3.0, 4.0)
p2 = Point(1.0, 2.0)

// ✅ Автоматическая привязка：p1.distance(p2) → distance(p1, p2)
d = p1.distance(p2)
// ✅ p1.add(p2) → add(p1, p2)
p3 = p1.add(p2)
```

**Правила автоматической привязки**：
- Функция определена в файле модуля
- Тип нулевого параметра функции совпадает с именем модуля
- Функция должна быть `pub`, чтобы автоматически привязываться за пределами модуля

### 10.6 Отношение привязки и каррирования

Привязка естественно поддерживает каррирование. Когда при вызове предоставлено меньше параметров, чем оставшихся, возвращается функция, принимающая оставшиеся параметры：

```yaoxiang
// Исходная функция: 5 параметров
calculate: (scale: Float, a: Point, b: Point, x: Float, y: Float) -> Float = ...

// Привязка：Point.calc = calculate[1, 2]
// После привязки оставшиеся параметры：scale, x, y

// Сценарии вызова
p1.calc(2.0, 10.0, 20.0)              # Предоставлено 3 параметра → прямой вызов
p1.calc(2.0)                          # Предоставлен 1 параметр → возврат (Float, Float) -> Float
p1.calc()                             # Предоставлено 0 параметров → возврат (Float, Float, Float) -> Float
```
