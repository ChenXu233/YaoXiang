# Схема агрессивной переработки архитектуры фронтенда YaoXiang (RFC-версия)

> Версия: 3.0 | Дата: 2026-01-29 | Статус: На основе требований RFC

>
> **Основная цель**: На основе архитектуры с низким уровнем связывания обеспечить полную поддержку реализации требований RFC-004/010/011

## 📋 Цели переработки

### Основные цели

- **Снижение связывания**: Устранение сильных зависимостей между модулями, реализация слабосвязанной архитектуры
- **Поддержка RFC**: Полная поддержка требований к дизайну и путям реализации трёх RFC
- **Оптимизация многоуровневой структуры файлов**: Чёткая многоуровневая архитектура, каждый уровень с единственной ответственностью
- **Повышение удобства сопровождения**: Разделение больших файлов, чёткие обязанности
- **Усиление расширяемости**: Резервирование пространства для расширения будущих функций, таких как RFC-012

### Матрица поддержки RFC

| RFC | Основные требования | Степень поддержки переработки | Расположение реализации |
|-----|---------------------|-------------------------------|-------------------------|
| **RFC-004** | Синтаксис много позиционных привязок, интеллектуальная привязка, автокарринг | 95% | `statements/bindings.rs`, `core/lexer/literals.rs` |
| **RFC-010** | Унифицированный синтаксис, синтаксис generics, определение типов | 90% | `statements/declarations.rs`, `types/parser.rs` |
| **RFC-011** | Решение ограничений, мономорфизация, система generics | 100% | `type_system/*`, `constraints.rs`, `unify.rs` |

### Критерии успеха

- [ ] Все файлы ограничены 500 строками
- [ ] Зависимости между модулями чёткие, без циклических зависимостей
- [ ] Требования к реализации всех трёх RFC на 100% поддерживаются архитектурой
- [ ] Упрощён публичный API, скрыта внутренняя реализация
- [ ] Покрытие тестами более 85%
- [ ] Время компиляции сокращено на 20% (благодаря лучшей модуляции)

## 🏗️ Дизайн новой архитектуры

### 1. Схема многоуровневой архитектуры

```
┌─────────────────────────────────────┐
│           Frontend API              │  ← Уровень публичного интерфейса
│        (frontend/mod.rs)            │
├─────────────────────────────────────┤
│  Lexer → Parser → TypeCheck → Const│  ← Уровень конвейера
│     │        │         │       │   │
│     ▼        ▼         ▼       ▼   │
├─────────────────────────────────────┤
│          Shared Utilities           │  ← Уровень общих утилит
│    (error, span, diagnostic)        │
├─────────────────────────────────────┤
│        Core Algorithm Layer         │  ← Уровень базовых алгоритмов
│  (type_system, const_eval, parse)   │
│                                     │
│  ▸ RFC-004: Поддержка解析绑定        │
│  ▸ RFC-010: Унифицированный синтаксис│
│  ▸ RFC-011: Полная система generics │
└─────────────────────────────────────┘
```
